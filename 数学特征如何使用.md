# 一、execution_noise_penalty：推荐的工业级做法

## 1️⃣ 先给你一句总定义（可以写进宪法）

> **execution_noise_penalty = 对“当前市场噪声 / 结构不稳定度”的连续折价因子
> 只影响“怎么做”，不影响“做不做”。**

这句话非常重要，它解决了你所有“是不是又在 gate”的心理负担。

---

## 2️⃣ 输入特征（你现在这 4 个就够了）

我们假设你已经有这些标准化前的原始量：

* `hurst_raw`
* `wpt_entropy_raw`
* `spectrum_width_raw`
* `hilbert_phase_var_raw`

### 🚫 注意

**不要直接用原值**
必须做三件事：窗口 → 稳定化 → 相对化

---

## 3️⃣ 时间窗口设计（抗抖 + 抗漂移）

这是很多系统失败的地方，我直接给你推荐参数区间：

```text
base_tf = 1h / 4h （你现在是 4h，非常合适）

rolling_window:
  short: 20 bars   （响应）
  long:  120 bars  （背景基线）
```

为什么两个窗口？

> **我们关心的不是“现在噪声高不高”
> 而是“现在相对最近这段历史，是不是更吵了”**

---

## 4️⃣ 单特征稳定化（关键）

### 以 WPT entropy 为例（其他同理）

```python
wpt_rel = (wpt_short_mean - wpt_long_mean) / wpt_long_std
```

然后：

```python
wpt_score = clip(sigmoid(wpt_rel), 0, 1)
```

解释一下语义：

* `≈0.5`：正常环境
* `→1`：比近期明显更吵
* `→0`：结构反而更清晰（少见，但好事）

### 对四个特征都做这一步

你会得到：

```text
hurst_score
wpt_score
spectrum_score
hilbert_score
```

⚠️ 注意：

* **Hurst 是反向的**（越低越噪）
* 所以它通常是 `1 - hurst_score`

---

## 5️⃣ 合成 execution_noise_penalty（最终形态）

### 推荐公式（稳健、可解释）

```python
noise_raw = (
    0.35 * wpt_score +
    0.30 * spectrum_score +
    0.20 * hilbert_score +
    0.15 * hurst_score
)
```

然后做一个 **非常重要的钳制**：

```python
execution_noise_penalty = clip(noise_raw, 0.0, 0.8)
```

> **永远不要允许它到 1**
> 否则你就在用它当 gate

---

## 6️⃣ 再加一道“惯性滤波”（防 execution 抖动）

```python
execution_noise_penalty_smooth =
    0.7 * prev_penalty + 0.3 * current_penalty
```

这是**减震器的减震器**。

## 3️⃣ 一个非常干净的执行映射（直接用）

假设你原始 execution 参数是：


### 引入 noise_penalty 后：


## 4️⃣ 那 soft gate 什么时候才“过滤”？

> **soft gate 只允许“退化为 NO_OP”，
> 不允许“否定某个子策略存在”**

例如：

```python

    NO_OP
```

这是 execution 物理限制，不是策略判断。
