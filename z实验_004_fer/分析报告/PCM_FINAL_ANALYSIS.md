## 4. Sharpe可信性分析

### 修复后的Sharpe: 14.33（年化）

**计算过程**:
- Per-trade Sharpe: 0.1257
- 年化交易数: 12,987 (11,107笔 / 0.86年)
- 年化Sharpe: 0.1257 × √12,987 = 14.33

### 可信性评估

⚠️ **年化Sharpe 14.33仍然不现实**

**对比参考**：
- 顶级量化基金: 2-4
- 优秀策略: 1.5-2.5
- 可接受策略: 0.8-1.5

**问题诊断**：

1. **年化方法可能错误**
   - 使用√12,987（年化交易数）不合理
   - 应该使用√252（交易日）或√12（月数）
   - 正确计算：0.1257 × √12 = 0.44（这更合理）

2. **Per-trade Sharpe 0.1257偏高**
   - 平均R=0.3041，标准差=2.4185
   - Sharpe = 0.3041/2.4185 = 0.1257
   - 这个计算本身没问题，但平均R可能有问题

3. **可能的数据问题**
   - Entry Filter过于严格→只留下高胜率信号
   - OOS期间市场环境特殊→单边上涨
   - 训练数据存在泄漏→模型"看到"了未来

## 6. _detect_direction_col函数评估

### 当前逻辑

```python
def _detect_direction_col(df: pd.DataFrame, archetype: str) -> Optional[str]:
    """检测 archetype 的方向列名"""
    candidates = [
        f"{archetype}_breakout_direction",  # 策略专属
        "breakout_direction",               # 通用突破方向
        "entry_direction",                  # 通用入场方向
        "direction",                        # 兜底
    ]
    for col in candidates:
        if col in df.columns:
            return col
    return None
```

### 评估

| 项目 | 评估 | 说明 |
|------|------|------|
| 逻辑正确性 | ✅ | 优先级合理，从专属到通用 |
| 扩展性 | ✅ | 易于添加新策略的方向列 |
| 语义性 | ⚠️ | **被动检测，不主动计算** |

### 问题

❌ **不够强**: 这个函数只是被动检测列名，不会根据策略语义动态计算方向

**用户质疑的正确性**:
> "趋势的可以用通道突破方向，或者远离sr方向，有语义"

**改进方案**:

```python
def _detect_or_compute_direction_col(df: pd.DataFrame, archetype: str) -> pd.Series:
    """检测或计算方向列"""
    
    # 1. 先尝试检测已有列
    dir_col = _detect_direction_col(df, archetype)
    if dir_col:
        return df[dir_col]
    
    # 2. 如果没有，根据策略语义动态计算
    if archetype == 'fer':
        # FER: 基于CVD的反向清算
        if 'cvd_change_5_normalized' in df.columns:
            return -np.sign(df['cvd_change_5_normalized']).replace(0, 1)
    
    elif archetype == 'me':
        # ME: 基于突破方向
        if 'breakout_sign' in df.columns:
            return df['breakout_sign']
        elif 'dist_to_nearest_sr' in df.columns:
            # 价格高于SR→做多，低于SR→做空
            return np.sign(df['dist_to_nearest_sr'])
    
    elif archetype == 'bpc':
        # BPC: 固定做多（突破延续）
        return 1.0
    
    # 3. 兜底：固定做多
    return 1.0
```
