## 3. Entry Filter严格度分析

### 诊断结果：❌ 极其严格（通过率4.3%）

**数据统计**：
- OOS期间ME样本总数: **3,694**
- PCM回测ME交易数: **158**
- **通过率: 4.3%**

**Cherry-Picking效应**：
```
通过率4.3% → 只有最高置信度信号通过 → 胜率虚高
```

**Entry Filter配置**：
```yaml
- id: orderflow_confirm
  enabled: true
  conditions:
    - feature: vpin
      operator: ">="
      value: 0.50  # 要求VPIN >= P50
```

**严格性分析**：
- VPIN >= P50 意味着订单流至少在中位数以上
- 结合ME的Gate规则，双重筛选导致极低通过率
- 4.3%通过率 → 只留下"绝对确定"的信号

**对比参考**：
- 正常策略通过率: 10-20%
- 严格策略通过率: 5-10%
- ME当前: **4.3%** (极其严格)

**后果**：
1. ✅ 胜率虚高（82.3%）- cherry-picking最佳信号
2. ❌ 交易数过少（158笔）- 错过大量机会
3. ❌ 策略鲁棒性差 - 过度依赖极端条件

## 5. 解决方案（三维度）

### 维度1: 语义合理性

**当前问题**：ME固定做多，缺乏语义方向判断

**解决方案**：
```python
# 基于突破方向动态决定
if 'breakout_sign' in df.columns:
    direction = df['breakout_sign']
elif 'dist_to_nearest_sr' in df.columns:
    # 价格高于SR→做多，低于SR→做空
    direction = np.sign(df['dist_to_nearest_sr'])
else:
    direction = 1.0  # 兜底做多
```

**语义解释**：
- 突破上轨 → 做多（向上突破）
- 突破下轨 → 做空（向下突破）
- 远离SR向上 → 做多
- 远离SR向下 → 做空

### 维度2: 逻辑对立性

**FER vs ME 方向对比**：

| 策略 | 方向逻辑 | 语义 | 对立性 |
|------|----------|------|--------|
| FER | `-sign(CVD)` | 博弈失败反向 | ✅ 反向 |
| ME | `sign(breakout_dir)` | 突破方向 | ✅ 正向 |

**验证**：
- FER：CVD正(多头失败) → 做空
- ME：突破向上 → 做多
- 两者逻辑独立且对立 ✅

### 维度3: 计算独立性

**当前状态**：
- ME: 使用`breakout_sign`或`dist_to_nearest_sr`
- FER: 使用`cvd_change_5_normalized`

**验证**：
- ✅ 两者特征完全不同
- ✅ 计算逻辑独立
- ✅ 无共用方向信号
