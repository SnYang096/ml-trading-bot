如何生成 drift 报告
  1. 前置：目标策略的 regime.yaml 需已有 last_calibration.plateaus（Tier-0 校准写入）。仅 B·Trend 独立 regime.yaml 走此路径；无 plateaus 时本页漂移列显示「未监测」。 
    python scripts/regime_threshold_calibrate.py --strategies tpc --dry-run
# 确认后加 --apply 写回 regime.yaml
  2. 监测：用近期特征窗口 parquet（与 feature bus / 研究快照列一致）跑漂移脚本： 
    python scripts/regime_drift_monitor.py \
  --strategies tpc,bpc,me,srb \
  --window-parquet results/<run>/recent_features.parquet \
  --strategies-root live/highcap/config/strategies
  3. 输出：写入 results/regime_drift_monitor/<时间戳>/drift_report.json； 有分位漂出 plateau 时进程退出码为 2。本页读取最新一份报告后点「刷新」。

详见 docs/strategy/regime_layer.md · Tier-0 运维。
