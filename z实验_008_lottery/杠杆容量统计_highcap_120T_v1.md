# 杠杆容量统计 · highcap 120T · v1

> 历史上哪些入场点能撑 100x / 50x / 20x / 10x / 5x 杠杆而不爆仓？哪些特征能识别这些位置？
> 这份报告是**基于 OHLCV 的第一版普查**，订单流/Funding/OI 特征在 v2 引入。

**脚本**：
- `scripts/analyze_leverage_capacity.py` — 生成逐 symbol 分桶 + 特征 lift + 决策树规则
- `scripts/summarize_leverage_capacity.py` — 跨 symbol 聚合报告

**样本**：
- Symbols: `BTC/ETH/SOL/BNB/XRP/ADA`（highcap universe）
- Timeframe: 120T（2h bar）
- Horizons: `H ∈ {12, 48, 120}` 即 24h / 4d / 10d
- Periods: 全历史 2020-01~2026-02、bull_2020_2021（2020-07~2021-05）、bull_2023_2024（2023-10~2024-03）
- 方向：每根 bar 同时评估 long 与 short

**口径**：
- 入场价 = bar_t close；未来 H 根 bar 的 high/low 决定 MAE/MFE
- 可支撑杠杆 `L_max = (1 - MMR) / MAE_pct`，MMR = 0.4%
- 分桶：`≥100x / 50–100x / 20–50x / 10–20x / 5–10x / <5x`

---

## 1. 结论 TL;DR

1. **"哪里能加 100x"强依赖于「看多久」**：
   - H=12（24h）：~29% 的随机 bar 可撑 100x → 但 MFE 中位仅 3.7%，**赔率低的伪 100x**。
   - H=48（4 天）：~14% 可撑 100x。
   - H=120（10 天）：~9% 可撑 100x，**但 P90 MFE ~50%**，开始有肥尾。

2. **牛市 × 做多 × 长窗 = 彩票真正集中区**：
   - 2020–2021 bull × long × H=120 ≥100x：**MFE 中位 36%、P90 107%**。
   - 2023–2024 bull × long × H=120 ≥100x：MFE 中位 20%、P90 41%。
   - 这才是"把 100x 当彩票"值得复制的位置。

3. **牛市 × 做空 = 绞杀区**：
   - 2020–2021 bull × short × H=120：**<5x 桶占 39%**（MAE > 20% 很常见），≥100x 只 4%。
   - 结论：下一轮牛市做空彩票 **不要用 100x**。

4. **识别 ≥100x 位置的稳健 OHLCV 特征**（2020–21 bull、long、H=120）：
   - `atr_pct` 最低分位（lift **2.22**）
   - `bar_range_pct` 最低分位（lift 2.00）
   - `rv_20` 最低分位（lift 1.99）
   - `ema1200_slope_100` 温和正（lift 1.93）
   - `close_ema1200_dist` 轻微正偏离（lift 1.73）
   - `ema200_slope_20` 小正斜率（lift 1.69）

   一句话语义：**"低波动压缩 + 温和上行趋势 + 未追高"** —— 这与 BPC 的"突破后回踩压缩"、ME 的"加速前的安静期"都吻合。

5. **已知警告**：
   - **相邻 bar 高度自相关**，同一段趋势会贡献多个 ≥100x 点，真实独立机会 **远少于** 占比数。
   - 未扣资金费 / 手续费 / 强平滑点；保守估计会把 H=120 的 ≥100x 占比再砍一半。
   - 未考虑"入场瞬间的闪跌"——实盘还要加 `快损熔断`。

## 3. 反向识别特征（谁能识别 ≥100x 位置）

下列「lift」= 该 feature 分位段内命中 ≥100x 桶的占比 / 基准占比。lift>1 表示该段富集 ≥100x。

### 3.1 2020–2021 bull × long × H=120（最值得复制的"彩票位置"）

| feature | quantile | lift | 位段中值 |
|---|---:|---:|---:|
| `atr_pct` | q0（最低） | **2.22** | 0.96% |
| `bar_range_pct` | q0 | 2.00 | 0.66% |
| `rv_20` | q0 | 1.99 | 0.64% |
| `ema1200_slope_100` | q1（小正） | 1.93 | +0.02% / bar |
| `close_ema1200_dist` | q1（轻微正） | 1.73 | +5.7% |
| `ema200_slope_20` | q2（温和上行） | 1.69 | −0.02%（弱） |

**语义**：牛市里能撑 100x 的点 = **低波动压缩中 + EMA1200 温和上行 + 价格不追高**。这和你现有的 BPC（压缩+延续）、ME（加速前的安静）语义高度一致。

### 3.2 2020–2021 bull × long × H=48

同样 `atr_pct` q0 / `bar_range_pct` q0 / `rv_20` q0 lift 最高（1.6–1.7）。波动分位信息在 48 bar 尺度就已经很强。

### 3.3 全历史 × long × H=12

`atr_pct` q0 lift=1.62，`bar_range_pct` q0 lift=1.59，`rv_20` q0 lift=1.52。**不论周期，低波动都是能撑高杠杆的第一前提**。

## 4. 已知问题 & v2 方向

### 已知问题

1. **样本自相关**：逐 bar 样本高度相关，占比数字是"时间占比"不是"独立机会数"。下次统计需做：
   - 每段趋势只保留第一个 ≥100x 点
   - 或 block bootstrap 估置信区间
2. **未扣 funding**：H=120 ≈ 10 天；funding 年化 20% 时约扣 0.55%，足以把 ≥100x 占比吃掉相当一块。
3. **未模拟入场瞬时跳变**：实盘还要配 `<=10s 快损熔断`、交易所托管 SL。
4. **MMR 近似**：用了固定 0.4%；真实按名义阶梯变化。

### v2 要扩展

- [ ] 加入**订单流特征**：`cvd_roll20`、`vpin`、`shd`、`ofci`（你现有 feature store 已有）。
- [ ] 加入**杠杆脆弱特征**：OI 变动率、funding z-score、long/short 比、orderbook imbalance（LV 语义）。
- [ ] **去重**：每段同向趋势只留第一个 ≥100x 点。
- [ ] **Funding cost 扣除**：H=120 按真实 funding 序列扣。
- [ ] 叠加 **策略信号点（BPC/TPC/ME）** 做对比：策略信号 vs 全网格，lift 有多大差异？
- [ ] **决策树 OOS**：用 2020–2022 训规则，2023–2024 测 precision / recall。

## 5. 给 L3 彩票设计的直接建议（基于 v1 数据）

对照你在 `实施文档_04_整体投资架构` 里的 L3 定义（ME 100x × 6 份）：

1. **标的选择**：BTC/ETH/SOL 这三个在 H=120 ≥100x 桶里占比最高、MFE 尾最肥；XRP/ADA 在 2023–24 牛市表现更弱。

2. **入场口径（≥100x 门控候选）**：
   - `atr_pct` 处于 20 个窗口内的最低 10–20% 分位
   - `ema1200_slope_100` 为温和正（第 10–30 分位）
   - `close_ema1200_dist` 处于 0~+10% 轻微上偏离
   - 与现有 ME 的"加速前的安静段"高度一致

3. **方向约束**：**只做多**。牛市 short ≥100x 占比极低（4–6%），<5x 桶却高达 20–40%，期望极差。

4. **份数控制**：H=120 下即便最佳位置也只有 ~15% 能真正撑 100x；按你现有的"6 份 × 100x" 节奏，6 份里期望命中 ~1 份。预算耗尽即停是对的。

5. **持仓时长**：10 天窗口的 MFE 中位 20–36%、P90 40–107%。**3–10 天持有 + 追踪止盈** 是捕捉肥尾的合理区间；比"立即 TP"更能吃到尾。

## 6. 复现

```bash
# 生成逐 symbol 的 samples / bucket / feature_lift / 决策树规则
python scripts/analyze_leverage_capacity.py \
    --symbols BTCUSDT,ETHUSDT,SOLUSDT,BNBUSDT,XRPUSDT,ADAUSDT \
    --timeframe 120T \
    --horizons 12,48,120 \
    --start-date 2020-01-01 \
    --end-date 2026-02-28 \
    --output-dir reports/leverage_capacity_v1

# 生成跨 symbol 聚合报告
python scripts/summarize_leverage_capacity.py \
    --root reports/leverage_capacity_v1 \
    --output reports/leverage_capacity_v1/SUMMARY.md
```

输出结构：

```
reports/leverage_capacity_v1/
├── SUMMARY.md                    ← 跨 symbol 聚合报告
├── global_bucket_counts.csv      ← 逐 symbol × horizon × period 分桶计数
├── global_bucket_counts_agg.csv  ← 跨 symbol 聚合
└── <SYMBOL>_120T/
    ├── samples_H{12,48,120}.parquet            ← 逐样本 (ts, side, mae, mfe, L_max, features)
    ├── bucket_counts_H{H}_{period}.csv
    ├── feature_lift_H{H}_{period}.csv
    └── tree_rules_H{H}_{period}.md             ← 浅层决策树可读规则
```
