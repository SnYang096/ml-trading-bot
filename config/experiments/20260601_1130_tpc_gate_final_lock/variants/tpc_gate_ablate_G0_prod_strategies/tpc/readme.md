# 一、TPC 同时开多个 symbol

**可以同时开多个，不需要等前一个盈利才开下一个。**

但需要一个总风险上限，比如同时最多持有 3–4 个仓位，每笔风险 1%，总风险敞口不超过 3–4%。

不能无限制同时开，原因是 crypto 各币种相关性很高，同时开 10 个仓位看起来分散，实际上 BTC 一跌全部止损，是在做同一个方向的集中暴露。

---

# 二、不断止损出局

这个现象本身就是诊断信号，直接告诉你问题在哪一层。

**连续止损有两种情况，性质完全不同：**

**情况一：行情本身在震荡，没有趋势。**

这不是系统问题，是 regime 问题。TPC 是趋势系统，震荡市里本来就应该少做或不做。这时候需要的不是调小止损，而是加一个 regime filter，在震荡市直接禁止入场。

**情况二：行情有趋势，但你总是在错误的位置进场。**

这是 entry filter 的问题。进场位置不好，止损设得再宽也会被洗出。这时候需要提高入场的结构质量，而不是调整止损。

**连续止损本身不能说明止损太小，只能说明「入场质量」或「regime 判断」有问题。**

---

# 三、max drawdown 太大，是否减小止损

**减小止损不能解决 max drawdown 的问题，只会制造新问题。**

止损小了之后会发生什么：在正常的 pullback 里被洗出，然后行情继续走，你踏空了。为了不踏空你会重新进场，然后再被洗出。结果是**止损次数增加，总亏损不变甚至更大，同时错过了真正的行情**。

**max drawdown 的正确解法是仓位管理，不是止损收紧。**

每笔风险从 1% 降到 0.5%，max drawdown 直接减半，同时止损位置不变，被洗的概率也不变。这是唯一不破坏系统结构的降低 drawdown 方法。
