# Sharpe 比率分析：为什么长时间段表现较差？

## 📊 最终结果汇总

### 四个策略的最终 Sharpe 结果（Pruned/SFFS 后）

| 策略 | Baseline Sharpe | 最终 Sharpe | 改进 | Return% | Trades | 评估 |
|------|----------------|------------|------|---------|--------|------|
| **compression_breakout** | -1.05 | **-1.07 ± 0.83** | -1.9% | -111.35% | 10.0 | ❌ 表现差 |
| **sr_breakout** | 0.75 | **0.82 ± 0.93** | +8.5% | 7.49% | 101.0 | ✅ 表现良好 |
| **sr_reversal_rr_reg_long** | 0.25 | **0.93 ± 0.44** | +273% | 6.55% | 36.8 | ⭐ 表现优秀 |
| **trend_following** | -0.07 | ~-0.07 | - | - | - | ⚠️ 表现差 |

## 🔍 为什么 Sharpe 这么低？

### 1. **训练日期范围过长（3年）**

所有策略的训练日期范围都是：**`2023-01-01` → `2025-12-31`**

这个 3 年的长周期包含了：
- **2023 年**：可能表现较好（牛市/震荡市）
- **2024-2025 年**：可能包含不利的市场环境（熊市/高波动）

**影响**：
- 短时间段（几个月）可能只覆盖了有利的市场环境
- 长时间段包含了更多不利环境，导致整体 Sharpe 下降
- 策略在不同市场环境下表现差异大

### 2. **市场环境变化**

金融市场是非平稳过程，不同时期的市场特征不同：

| 时期 | 市场特征 | 策略表现 |
|------|---------|---------|
| 2023 年 | 可能：牛市/震荡市 | 策略可能表现较好 |
| 2024-2025 年 | 可能：熊市/高波动/监管变化 | 策略可能表现较差 |

### 3. **策略特性差异**

不同策略对市场环境的敏感度不同：

- **compression_breakout**: 
  - 依赖压缩-突破模式
  - 在低波动市场表现好，高波动市场可能失效
  - Sharpe = -1.07（表现最差）

- **sr_breakout**:
  - 依赖支撑/阻力突破
  - 相对稳健
  - Sharpe = 0.82（表现良好）

- **sr_reversal_rr_reg_long**:
  - 依赖支撑/阻力反转
  - 改进最大（+273%）
  - Sharpe = 0.93（表现最好）

- **trend_following**:
  - 依赖趋势延续
  - 在震荡市表现差
  - Sharpe = -0.07（表现差）

## 🎯 核心发现：树模型无法适应 Regime Shift

### 关键观察

**你的观察非常准确**：
> "单纯的树策略模型无法适应 regime shift，单独训练几个月预测下几个月还不错"

这确实是树模型的核心问题！

### 为什么树模型无法适应 Regime Shift？

#### 1. **硬阈值依赖历史分布**

树模型使用硬阈值（如 `cvd_long <= -34895`），这些阈值是**基于历史数据分布拟合的**：

```python
# 训练期（2023年，低波动）
if cvd_long <= -34895:  # 阈值是历史拟合的
    veto_tc = True  # 训练期准确率90%

# 测试期（2024年，高波动）
# 数据分布变化：cvd_long均值从-10000变为-30000
# 阈值-34895从2.5%分位数变为20%分位数
# 结果：准确率下降到30%（大幅失效）
```

**问题**：
- 硬阈值依赖历史数据分布
- 当市场 regime 变化时，数据分布也变化
- 阈值失效，模型表现大幅下降

#### 2. **局部拟合，无法泛化**

树模型通过**局部划分**学习模式，每个规则只适用于训练期的数据分布：

| 时期 | 数据分布 | 规则有效性 |
|------|---------|-----------|
| **训练期（几个月）** | 单一 regime（如低波动） | ✅ 规则有效 |
| **测试期（下几个月，相同 regime）** | 相同 regime | ✅ 规则仍然有效 |
| **测试期（下几个月，不同 regime）** | 不同 regime（如高波动） | ❌ 规则失效 |

**结果**：
- 短时间段（几个月）→ 可能只覆盖单一 regime → 表现好
- 长时间段（3年）→ 包含多种 regime → 表现差

#### 3. **无法自适应**

树模型是**静态的**，一旦训练完成，规则就固定了：
- 无法根据当前市场状态自动调整
- 需要手动重新训练或调整阈值
- 无法适应 regime shift

### 解决方案

#### 方案1：滚动训练（Rolling Training）⭐ **推荐**

**核心思想**：定期重新训练模型，用最近几个月的数据预测下几个月

```bash
# 每3-6个月重新训练一次
make rolling \
  ROLLING_CONFIG=config/strategies/sr_reversal_rr_reg_long \
  SYMBOL=BTCUSDT \
  INITIAL_TRAIN_MONTHS=6  # 用6个月训练，预测下1个月
```

**优势**：
- ✅ 模型始终使用最新的市场状态
- ✅ 自动适应 regime shift
- ✅ 短时间段表现好（几个月训练 → 几个月预测）

**工作流程**：
```
第1次: 训练=[1-6月], 测试=7月 → 模型1
第2次: 训练=[2-7月], 测试=8月 → 模型2
第3次: 训练=[3-8月], 测试=9月 → 模型3
...
```

#### 方案2：Regime-Aware 模型

**核心思想**：先识别市场 regime，再使用对应的模型

```python
# Stage 1: Regime Detection
regime = regime_detector.predict(features)  # 0=震荡, 1=趋势, 2=爆发

# Stage 2: Conditional Model
if regime == 0:
    signal = mean_revert_model.predict(features)
elif regime == 1:
    signal = trend_model.predict(features)
else:
    signal = breakout_model.predict(features)
```

**优势**：
- ✅ 不同 regime 使用不同模型
- ✅ 自动适应市场状态变化
- ✅ 可解释性强

#### 方案3：使用相对阈值（分位数）

**核心思想**：用相对位置（分位数）替代绝对阈值

```python
# 树模型（硬阈值）
if cvd_long <= -34895:  # 硬阈值
    veto_tc = True

# 改进（分位数阈值）
cvd_long_q20 = quantile(cvd_long, window=200, q=0.2)
if cvd_long <= cvd_long_q20:  # 相对位置（20%分位数）
    veto_tc = True
```

**优势**：
- ✅ 阈值自适应数据分布
- ✅ 即使数据分布变化，20%分位数仍然对应"极端值"
- ✅ 更稳健，不容易受到 Regime Shift 影响

## 📈 结论

1. **最终结果比 Baseline 好**：
   - sr_reversal: +273% 改进 ⭐
   - sr_breakout: +8.5% 改进
   - compression_breakout: -1.9%（仍然很差）

2. **长时间段导致 Sharpe 下降**：
   - 3 年周期包含了多种市场环境
   - 短时间段可能只覆盖有利环境
   - 需要按时间段分别分析

3. **树模型无法适应 Regime Shift** ⚠️ **核心问题**：
   - 硬阈值依赖历史分布
   - 局部拟合，无法泛化到不同 regime
   - 短时间段（几个月）表现好，长时间段（3年）表现差
   - **解决方案**：滚动训练、Regime-Aware 模型、相对阈值

4. **策略选择很重要**：
   - sr_reversal 表现最好（Sharpe = 0.93）
   - compression_breakout 表现最差（Sharpe = -1.07）
   - 不同策略对市场环境敏感度不同
