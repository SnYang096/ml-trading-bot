## 📋 系统概述

### 设计目标
- 实现 BPC、ME、FER 三个策略的统一实盘运行框架
- 通过配置驱动决策，避免重复代码
- 支持策略优先级仲裁和资源竞争管理
- 保持与现有单策略系统的兼容性

### 核心架构

```
实盘入口 (run_live.py)
    ↓
策略初始化层
    ├── BPCLiveStrategy (传统)
    ├── GenericLiveStrategy (新)
    │   ├── BPC archetype
    │   ├── ME archetype  
    │   └── FER archetype
    ↓
PCM 仲裁层 (LivePCM)
    ├── 策略优先级: FER > ME > BPC
    ├── 资源管理: 最大 slot 数控制
    ├── 信号仲裁: 高优先级策略优先执行
    ↓
特征计算层 (IncrementalFeatureComputer)
    ├── 流式特征计算
    ├── 多策略特征融合
    ├── 实时数据更新
    ↓
执行层 (OrderManager)
    ├── 订单管理
    ├── 风险控制
    ├── 执行反馈
```

## 🏗️ 核心组件设计

### 1. GenericLiveStrategy 通用策略引擎

#### 设计理念
- **配置驱动**: 所有决策逻辑通过 YAML 配置文件定义
- **模块化**: 5个独立评估器组件，职责分离
- **可扩展**: 支持新策略类型快速接入

#### 核心组件

```python
class GenericLiveStrategy:
    def __init__(self, strategy_name: str, strategies_root: str):
        self.strategy_name = strategy_name
        self.strategies_root = strategies_root
        self._direction_eval = DirectionEvaluator()      # 方向判断
        self._gate_eval = GateEvaluator()              # 结构性过滤
        self._entry_filter_checker = EntryFilterChecker()  # 入场时机
        self._evidence_scorer = EvidenceScorer()       # 信号强度评分
        self._execution_param_gen = ExecutionParamGenerator()  # 执行参数
```

#### 5个评估器详解

**1. DirectionEvaluator (方向评估器)**
```yaml
# direction.yaml 示例
rules:
  - name: "突破方向"
    condition: "price > sma_200"
    direction: 1  # 多头
  - name: "回调支撑"
    condition: "price < sma_20 and rsi < 30"
    direction: 1
fallback: reject  # 无匹配则拒绝
```

**2. GateEvaluator (门禁评估器)**
```yaml
# gate.yaml 示例
rules:
  - name: "波动率过滤"
    condition: "volatility_20 > 0.02"
  - name: "成交量确认"  
    condition: "volume_15min > volume_20_mean * 1.5"
combination_mode: and  # 必须同时满足
```

**3. EntryFilterChecker (入场过滤器)**
```yaml
# entry_filters.yaml 示例
filters:
  - name: "布林带宽度"
    condition: "bb_width < 0.05"
  - name: "流动性静默"
    condition: "liquidity_silence > 60"
combination_mode: or  # 满足其一即可
```

**4. EvidenceScorer (证据评分器)**
```yaml
# evidence.yaml 示例
features:
  - name: "price_momentum"
    weight: 0.3
    quantile_thresholds: [0.2, 0.8]
  - name: "volume_profile"
    weight: 0.4
    quantile_thresholds: [0.1, 0.9]
scoring_method: weighted_sum
```

**5. ExecutionParamGenerator (执行参数生成器)**
```yaml
# execution.yaml 示例
default_params:
  slippage_limit: 0.001
  size_multiplier: 1.0
  max_position_size: 0.1
risk_controls:
  max_drawdown: 0.05
  position_limit: 2
```

### 2. LivePCM 仲裁层

#### 仲裁逻辑
```python
class LivePCM:
    def __init__(self, archetype_priority: List[str], capacity_limit: int):
        self.archetype_priority = archetype_priority  # ['fer', 'me', 'bpc']
        self._capacity_limit = capacity_limit
        self._active_positions = {}  # symbol -> archetype
    
    def decide(self, features, symbol) -> List[TradeIntent]:
        intents = []
        for archetype in self.archetype_priority:
            if self._can_allocate_slot(symbol):
                intent = self._strategies[archetype].decide(features, symbol)
                if intent and intent.action != "hold":
                    intents.append(intent)
                    self._allocate_slot(symbol, archetype)
                    break  # 高优先级策略先执行
        return intents
```

#### 资源管理策略
- **Slot 限制**: 最大持仓币种数控制
- **优先级抢占**: 高优先级策略可抢占低优先级仓位
- **状态同步**: 确保仓位状态在策略间一致

### 3. 特征计算层优化

#### 多策略特征融合
```python
class IncrementalFeatureComputer:
    def __init__(self, archetypes_dir: str):
        # 自动检测所有策略的特征需求
        self.required_features = self._detect_features(archetypes_dir)
        
    def compute_features(self, bar_data: pd.Series) -> Dict[str, float]:
        # 统一计算所有策略所需特征
        features = {}
        for feature_name in self.required_features:
            features[feature_name] = self._compute_feature(feature_name, bar_data)
        return features
```

#### 性能优化
- **特征缓存**: 避免重复计算
- **增量更新**: 只计算变化的特征
- **并行计算**: 支持多核特征计算

## 🛡️ 风险控制设计

### 多层风控机制

#### 1. 策略层风控
- **方向过滤**: direction.yaml 严格控制入场方向
- **结构过滤**: gate.yaml 防止异常市场条件
- **时机过滤**: entry_filters.yaml 优化入场时机

#### 2. 系统层风控
- **仓位限制**: capacity_limit 控制总持仓数
- **优先级管理**: 高优先级策略可抢占资源
- **异常处理**: 策略异常时自动降级

#### 3. 执行层风控
- **滑点控制**: slippage_limit 限制最大滑点
- **仓位大小**: size_multiplier 动态调整仓位
- **止损机制**: 集成交易所止损订单

#### 实时监控
