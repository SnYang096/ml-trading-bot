
## 📊 策略优先级

三策略竞争同一交易槽位，按以下优先级排序（PCM 仲裁）：
1. **LV (Low Volatility)** - 低波动率策略 (暂缓)
2. **FER (Fading Efficiency Reversion)** - 效率反转策略 (4H)
3. **ME (Market Efficiency)** - 市场效率策略 (1H)
4. **BPC (Breakout Pattern Confirmation)** - 突破模式确认策略 (4H)

## ⚙️ 关键配置

### 环境变量（已在 start_live.sh 中配置）
| 变量 | 默认值 | 说明 |
|------|--------|------|
| `MLBOT_LIVE_TRADE_SIZE` | `0.0` | 0=观察模式 |
| `MLBOT_LIVE_MODE` | `three_strategies` | 三策略多时间框架模式 |
| `MLBOT_STRATEGIES_ROOT` | `live/highcap/config/strategies` | 策略配置目录 |
| `MLBOT_PCM_REGIME_CONFIG` | `live/highcap/config/pcm_regime.yaml` | PCM Regime 配置 |
| `MLBOT_CONSTITUTION_YAML` | `live/highcap/config/constitution/constitution.yaml` | 宪法配置 |
| `MLBOT_BPC_WINDOW_MINUTES` | `15` | 特征计算窗口 |
| `MLBOT_ORDER_MODE` | `test` | test/paper/live |

### timeframe 配置（从 meta.yaml 动态读取）
- BPC: `config/strategies/bpc/meta.yaml` → `timeframe: "240T"` (4H)
- ME:  `config/strategies/me/meta.yaml`  → `timeframe: "60T"` (1H)
- FER: `config/strategies/fer/meta.yaml` → `timeframe: "240T"` (4H)

### 风控参数
- **最大持仓数**: 2个交易槽位 (constitution.yaml)
- **单笔交易量**: 由 `MLBOT_LIVE_TRADE_SIZE` 控制
- **策略竞争**: PCM仲裁机制，高优先级策略优先进入

## 🛡️ 安全建议

### 首次运行
1. **务必先用观察模式运行24小时**，确认系统稳定性
2. **从小仓位开始**，建议初始 `MLBOT_LIVE_TRADE_SIZE=0.001`
3. **密切监控日志**，关注异常和错误信息
4. **验证信号质量**，确认 ME 用 1H 特征、BPC/FER 用 4H 特征

## 🚨 紧急处理

### 停止运行
```bash
# 优雅停止
Ctrl+C

# 强制停止
pkill -f "run_live.py"
```

### 故障排查
- **tick数据不足**: `bash live/scripts/prepare_warmup_ticks.sh highcap 1 --fill-gap`
- **连接问题**: 检查网络和 API 密钥
- **特征计算错误**: 检查 warmup 数据完整性和存储路径
- **内存溢出**: 监控系统资源使用情况
