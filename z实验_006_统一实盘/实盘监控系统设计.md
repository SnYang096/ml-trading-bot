# 一个三层“决策面板”结构, 让你每天只需要看 3 个灯号。

好。

我们把那 10 个指标压缩成：

# 🎛 三层“决策面板”

每天只看 3 个灯号。

不讨论细节，不调参数。
只判断：

> 继续 / 降速 / 停止

---

# 第一层：资本生存灯（红线层）

### 🟥 灯号 1：Capital Integrity

**输入指标：**

* 当前 DD / 回测最大 DD
* Rolling 30 笔 Expectancy
* 连续亏损笔数

**规则建议（示例）：**

🟢 绿灯：

* DD < 0.8 × 回测最大 DD
* Rolling E > 0

🟡 黄灯：

* 0.8 ~ 1.3 × 回测最大 DD
* Rolling E 接近 0

🔴 红灯：

* > 1.3 × 回测最大 DD
* Rolling E 连续为负
* 连续亏损 > 统计 95% 分位

---

### 行动逻辑：

* 🟢 → 正常运行
* 🟡 → 降杠杆 30%
* 🔴 → 暂停系统

这层只回答一个问题：

> 系统还活着吗？

---

# 第二层：机制健康灯（结构层）

### 🟦 灯号 2：Edge Health

**输入指标：**

* Archetype 分解 PnL
* 胜率变化
* 盈亏比变化
* MFE / MAE 比率变化
* 持仓时间变化

我们压缩成两个核心比率：

1. 当前 30 天 Sharpe / 回测中位 Sharpe
2. 当前 MFE/MAE 比率 / 历史中位数

---

🟢 绿灯：

* Sharpe > 0.7 × 历史中位
* MFE/MAE 未恶化

🟡 黄灯：

* Sharpe 下降但 >0
* MFE 略降

🔴 红灯：

* Sharpe 接近 0 或负
* MFE 显著下降 + MAE 上升

---

### 行动逻辑：

* 🟢 → 不动
* 🟡 → 降权弱 archetype
* 🔴 → 暂停该 archetype

注意：

这是“局部降解”，
不是全系统熔断。

---

# 第三层：市场环境灯（外部层）

### 🟩 灯号 3：Market Compatibility

输入指标：

* 信号密度变化
* 波动率分位
* LV 触发频率
* OI / funding 极端性（若有）

压缩为：

1. 当前信号数量 / 历史中位数
2. 当前波动率分位

---

🟢 绿灯：

* 信号密度正常
* 波动率处于适配区间

🟡 黄灯：

* 信号显著减少
* 波动率极低或极高

🔴 红灯：

* 信号极端异常
* 市场进入非结构态（极端插针 / 单边无回撤）

---

### 行动逻辑：

* 🟢 → 正常
* 🟡 → 降仓
* 🔴 → 只保留最稳 archetype

---

# 🧠 每天你只看：

```
Capital:   🟢 / 🟡 / 🔴
Edge:      🟢 / 🟡 / 🔴
Market:    🟢 / 🟡 / 🔴
```

---

# 三层优先级逻辑

优先级从上到下：

1️⃣ Capital
2️⃣ Edge
3️⃣ Market

如果 Capital 红灯，下面不用看。

如果 Capital 绿灯但 Edge 红灯 → 降权。

如果 Capital 绿灯、Edge 绿灯、Market 黄灯 → 轻微降速。

---

# 这套面板的真正价值

它的作用不是：

> 提升收益

而是：

> 防止情绪干扰

当亏损时：

你不用问：

* 要不要调参数？
* 要不要加模型？
* 是不是失效？

你只看灯号。

---

# 最关键的一点

面板的阈值必须固定半年。

不能：

* 因为一次亏损就修改灯号规则
* 因为收益下降就放宽阈值

否则你会进入：

> 自我解释循环
