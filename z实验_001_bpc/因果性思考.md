# 因果性分层 × BPC 系统审计

> 核心判断：长期有效的不是具体信号，而是——
> **系统对 alpha 死亡的免疫力。**

---

# 一、因果性 Level 分级框架

**硬判定标准**：是否依赖"人类行为持续不变"？

| Level | 依赖什么 | 半衰期 | 策略姿势 |
|-------|----------|--------|----------|
| **0** | 只依赖"市场存在交易"这一事实。不依赖人类心理，不依赖参与者结构 | 永恒 | 永久持有，只优化执行 |
| **1** | 依赖市场结构 + 激励机制。不依赖具体指标形式 | 5-10年 | 核心持仓，监控结构变化 |
| **2** | 明确依赖行为偏差。可被学习、拥挤、反向利用 | 1-3年 | 动态仓位，准备退化 |
| **3** | 纯统计相关。某时间段的订单流模式、特定新闻反应 | 6-18月 | 小仓位试错，快速迭代 |

**工程化用途**：看到任何新 feature，立刻定位"最坏情况下它会退化到哪一层"。

**策略失效诊断**：
- 夏普 < 历史均值 - 2σ → 触发诊断
- 结构性改变（新监管/规则） → 永久放弃
- 拥挤交易（同质化竞争） → Level 0/1 降仓等待，其他暂停
- 市场状态切换（低波→高波） → 加状态过滤器

**资金分配**：守正出奇
- 70% → Level 0/1 核心组合
- 20% → 探索新因子
- 10% → 捕捉短期套利

## 二、真正"长期有效"的三类东西（10 年尺度）

### ✅ 1. 分层架构本身（Level 0.5）

```
Gate:      "能不能做"  → 约束层，永恒有价值
Evidence:  "下多少"    → 调整层，具体因子会死但框架不死
Entry:     "现在吗"    → 等待层，拒绝在信息不充分时行动
Execution: "怎么执行"  → 风控层，处理市场的物理属性
```

为什么是 Level 0.5？因为这套分层解决的是一个永恒问题：

> "预测永远不可靠，但约束永远有价值。"

这与做市商的 quoting/hedging 分层、CTA 的 signal/sizing/overlay、
高频的 alpha/risk/throttle **是同构的**。

**只会被"细化"，不会被"淘汰"。**

### ✅ 2. Entry Filter / Plateau / 稳定区思想（Level 1）

不是在找 entry，而是在**拒绝在"信息不充分时行动"**。

- plateau（平坦高原）
- 鲁棒区（CV 约束）
- confidence（宽度→置信度）
- 宽容侧推荐（错误成本不对称）

本质是在编码一句话：

> **"当世界对我不敏感时，我才出手。"**

具体阈值会变，但 plateau / 稳定区 / 不追极值 **永远不会过时**。

### ✅ 3. Execution 的"防爆仓宪法"（Level 0）

处理的是市场的**物理属性**，不是统计规律：
- 流动性不连续
- 滑点
- 极端行情
- 人类无法忍受的回撤路径

不依赖人类心理，不依赖参与者结构，只依赖"市场存在交易"。

execution 参数表 / regime-aware sizing / 失败模式兜底 = **只会被压利润，不会失效**。

## 三、BPC 系统各组件因果性审计

### 🟢 Level 0-1：长期可信（可以跟着走 5-10 年）

| 组件 | 因果机制 | 评级 |
|------|----------|------|
| **分层架构本身** | 对不确定性的工程化处理 | Level 0.5 |
| **BPC 结构**（突破→回踩→延续） | 流动性机制：突破触发止损单→价格真空→回踩是流动性回填 | Level 1 |
| **Execution 风控** | trailing stop、tiers、timeout — 风险管理永远有效 | Level 0 |
| **Guardrail: price_position > 0.9** | 极端位置买入→均值回归压力 — 物理约束 | Level 0-1 |
| **Guardrail: vol_compression < 0.3** | 无量突破=假突破 — 没有真实资金推动 | Level 1 |
| **Plateau / 稳定区思想** | 不追极值、鲁棒性优先、宽容侧推荐 | Level 1 |
| **Router / Execution 职责切分** | 分层解耦 — 架构级永恒 | Level 0.5 |
| **regime-aware sizing** | 不同环境不同仓位 — 物理属性 | Level 1 |

### 🟡 Level 2：中期有效（1-3 年，必须随时准备退役）

| 组件 | 因果机制 | 风险 |
|------|----------|------|
| **Entry: bpc_vol_pullback_confirm** | 回踩缩量=卖方耗尽 — 供需逻辑 | 行为偏差会被对手盘学习 |
| **Entry: bpc_cvd_absorption** | 订单流吸收=买方底部接盘 | BTC上中等可信，因果性取决于市场结构 |
| **Gate: direction_crowding (OFCI)** | 拥挤→反转风险 | 机制合理，但OFCI具体算法是统计构造 |
| **Evidence: sma_200_position** | 自我实现预言 — 因为所有人都看 | 正因广泛使用，既是信号也是靶标 |
| **Entry: wick_absorption** | 影线=底部拒绝 — K线形态 | 最容易被学术论文杀死的模式 |
| **Entry: bb_compression** | 波动率压缩→爆发 — GARCH事实 | 参数化方式可被替代 |
| **Entry: momentum_confirm** | RSI/动量反转 | 极度拥挤的信号 |

**正确姿势**：放在 Entry / Router，**只作为必要非充分条件**，
永远不让它们单独决定方向或仓位。

### 🟡 Level 3：非因果但允许存在（只允许在 Evidence 层）

> **核心纠偏：“数学特征 = 无用”这个结论推得过头了。**
>
> Evidence 层的任务不是“解释为什么赚钱”，
> 而是“区分哪些 trade 更容易失败”——这是一个**反向任务（Failure detection）**，
> 天然允许是统计性的。

| 组件 | 因果性 | 合法使用方式 |
|------|----------|----------------|
| **Evidence: wpt_exhaustion_score** | 非因果，但能检测“这不是常态世界” | 风险状态估计 → 降仓/延迟 |
| **Evidence: spectrum_price_flatness** | 非因果，但能识别“价格在说谎” | 器械判断 → 降权 |
| **Evidence: evt_scale_right** | EVT 告诉你“当前已进入历史极端尾部” | 极端状态警报 → 放弃 entry |
| **Evidence: macd_signal_atr** | Level 2（行为偏差型），不是 Level 3 | 趋势确认 → 结合其他特征 |
| **Entry: wpt_absorption** | 非因果，但能检测“能量吸收模式” | 确认信号 → 只做必要非充分条件 |
| **Noise Penalty (wpt/spectrum/hilbert/hurst)** | 非因果，但是“噪声程度”的合理估计 | 降仓/拉宽止损 → 风控辅助 |
| **具体阈值** (0.55, 0.67, 0.72...) | 全部是历史数据拟合 | plateau 鲁棒区间 → 而非 argmax |

**关键前提（权力边界）**：
- ✅ 不能决定方向
- ✅ 不能单独决定仓位
- ✅ 必须可被整体关闭（fallback 方案）
- ✅ 只允许在 Evidence 层使用，不允许泄漏到 Gate/Entry

**它们是燃料，不是发动机。但燃料也有价值——只要不让它当方向盘。**

# 四、核心问题：不是因果性，而是权力结构

### 关键区分：三件本该分开的事

| 容易混淆 | 实际含义 |
|----------|----------|
| 因果性 vs 可替代性 | 数学特征不是"假"，而是"可被更底层的特征替代" |
| 决策权重 vs 信息来源 | Evidence 提供的是"调整信息"，不是"方向决策" |
| "数学特征"在不同层的合法性 | 在 Gate/Entry 层非法，在 Evidence 层合法（Failure detection） |

### Evidence 的真实角色

```
Evidence ≠ Alpha       → 不负责"为什么赚钱"
Evidence ≠ Direction    → 不决定方向
Evidence = Failure detector → "区分哪些 trade 更容易失败"
```

Failure detection **本来就允许是统计性的**。

例如 EVT 告诉你："当前 price path 已进入历史极端尾部"。
EVT 不告诉你为什么，但它告诉你"这不是常态世界"——
这对降仓、延迟、放弃 entry 是极其有价值的信息。

### 🚫 Evidence 层铁律（禁止清单）

> **Evidence 层的任何输出，都不能在单一 trade 上把 NO → YES 翻转。**

Evidence 只能让你：

| ✅ 允许 | ❌ 禁止 |
|---------|--------|
| YES → 小一点（降仓） | 原本不该做 → 因为 Evidence 好看就做 |
| YES → 晚一点（延迟） | 被 Gate 否决的 trade → Evidence 翻案 |
| YES → 条件更苛刻（收紧） | 单独决定方向或仓位 |

**把这条写进系统设计，未来基本不可能"数学特征反噬"。**

### 真正的风险不在因果性，而在权力边界

> **最有因果基础的组件负责「生存与约束」，
> 最没因果基础的组件如果进入「决策核心」，那才是危险。**

这不是对系统的判决，而是对演化方向的警告。

### 应对方案

1. **OOS 验证**：不是 holdout，是完全未见过的时间段
2. **因果标签**：给每个特征打上「因果型 / 统计型 / 风险型」标签，明确"最多能影响哪一层"
3. **权力边界**：统计型特征只允许降权/延迟，不允许决定方向或主导仓位
4. **降级方案**：当数学特征 OOS 退化时，回退到只用 Gate + Entry Filter + Execution

## 五、按"可长期性"重新归类

### 🟢 可以跟你走 10 年的
- 分层架构本身
- Router / Execution 的职责切分
- plateau / 稳定区 / 鲁棒性思想
- execution 风控规则
- regime-aware sizing

### 🟡 可能活 1–3 年
- 回踩结构 + 缩量确认
- 部分订单流衍生指标（CVD）
- 行为型 exhaustion
- 方向拥挤度

### 🟡 非因果但合法（只允许在 Evidence 层）
- 数学特征（WPT/Spectrum/EVT）— 作为风险状态估计器

- 具体阈值（全部是拟合的）
- MACD 的具体 alpha（行为偏差型，会被拥挤）
- 单一市场微观结构模式

## 六、因果性会退化 — 持续监控

当某个模式被广泛使用 → 会被对手盘利用 → 变成"假信号"。

```
持续评估因果置信度 
  → 高置信 (Level 0-1): 统计建模，核心仓位
  → 中置信 (Level 2):   必要非充分条件，不独立决策
  → 低置信 (Level 3):   小仓位，快速迭代
  → 置信度下降:         立即降级策略
```

**验证方法**：
- 反事实测试：如果信号消失，价格会回撤吗？
- 跨市场验证：同一信号在不同交易所表现一致吗？
- 时间稳定性：模式能存活多久？（过拟合 vs 真实 alpha）

# 七、终极自检原则

> **最不确定的东西，权力必须最小；
> 最确定的东西，必须负责生存。**

成熟量化系统的终态：

> **因果负责活下去，
> 统计负责少犯错，
> 架构负责不自杀。**

每加一个组件，必须回答三件事：

| 自检问题 | 不合格则 |
|----------|----------|
| 它退化时会不会杀死系统？ | 必须降权或加 fallback |
| 它有没有越权影响方向/仓位？ | 必须限制在合法层级 |
| 关掉它，系统还能不能跑？ | 不能则违反可降级原则 |

通过这三条，系统不是"可能长期有效"，而是**很难整体失败**。
