## Pipeline 总览

训练是**两轮迭代**流程，不是线性的。Prefilter/Direction 分析依赖 predictions.parquet，
而 predictions.parquet 由训练管线产出（OHLCV + 标签 + 特征 + 模型预测值 pred）。

```
═══ Round 0: 冷启动 (首次训练，无 prefilter/direction 校准) ═══

Step 1: Feature Store 构建
  ↓
Step 2: Gate 冷启动训练 (无 prefilter) → 产出 predictions.parquet
  ↓
═══ Round 1: 校准轮 (用 predictions.parquet 分析，更新配置) ═══
  ↓
Step 3: Prefilter 语义分层分析 (验证 archetype 特征区分力)
Step 4: Direction 验证 (方向模块质量检查)
  ↓ (人工审核，更新 prefilter.yaml / direction.yaml)
  ↓
═══ Round 2: 正式训练 (带 prefilter + direction) ═══
  ↓
Step 5: Gate 正式训练 + 优化
  ↓
Step 6: Evidence 训练 + 优化
  ↓
Step 7: Entry Filter 优化
  ↓
Step 8: Execution 优化
  ↓
Step 9: PCM 联合回测
```

> **predictions.parquet 内容**: OHLCV + 标签 (forward_rr, success_no_rr_extreme) + 训练用特征列 + 模型预测值 pred + split 标记。
> FS 里没有标签列，所以分析脚本不能直接用 FS，必须用 predictions.parquet。

首次训练，目的是产出 `predictions.parquet` 供后续 Prefilter/Direction 分析。

### Step 3: Prefilter 语义分层分析

> **前置依赖**: Step 2 产出的 `predictions.parquet`

验证 BPC archetype 特征的区分力，确认 pre_filter 条件是否合理。

```bash
python scripts/analyze_archetype_feature_stratification.py \
  --logs ${COLD_RESULT_DIR}/predictions.parquet \
  --strategy bpc \
  --config config/strategies/bpc/prefilter.yaml
```

**关注指标**：
- 正信号（high=good）: bad rate 差异 > 5% → 可做 pre_filter 正向条件
- 反信号（high=bad）: bad rate 差异 > 5% → 可做反向 hard_gate
- 参考: `gate_semantic_prefilter_design.md` 中 BPC 部分分析结论

**操作**: 审核报告后，更新 `config/strategies/bpc/prefilter.yaml` 和 `archetypes/gate.yaml` 中的 hard_gate 规则。

### Step 4: Direction 验证

> **前置依赖**: Step 2 产出的 `predictions.parquet`

BPC 方向由 `direction.yaml` 确定性规则决定 (bpc_breakout_direction: Donchian 突破方向)。
需验证方向模块不是有害的。

```bash
# 全量验证 (所有策略)
python z实验_005_统一研究/direction_strict_validation.py

# 单策略验证 (指定 predictions.parquet)
python z实验_005_统一研究/direction_strict_validation.py \
  --logs ${COLD_RESULT_DIR}/predictions.parquet \
  --strategy bpc
```

**Phase 1 通过标准** (覆盖率)：
- 覆盖率 100% (每个 bar 都有方向)
- Short 占比 > 15%

**Phase 2 通过标准** (方向质量)：
- `median(rr_in_direction) > median(forward_rr_long)` → 方向模块优于 always-long
- `mean(rr_in_direction) > 0` → 按方向交易总体不亏
- Bad rate(按方向) < Bad rate(always-long) → 踩坑率降低
- Short 子集样本量 >= 1080 → 统计可信

**操作**: 如果 Phase 2 不通过，检查 `direction.yaml` 规则或暂时禁用 `direction_aware`。

> ⚠️ **BPC 注意**: BPC direction = Donchian breakout direction，在牛市主导的 crypto 数据中，
> short 占比可能很低甚至为 0。这是 BPC 策略的固有特性（趋势跟随），不一定是 bug。
> 如果 short < 15%，需评估是否禁用 direction_aware 回退到 always-long。

### Step 5: Gate 正式训练 (方向感知)

> **前置依赖**: Step 3 Prefilter 分析 + Step 4 Direction 验证通过后，带校准配置重新训练。

**方向感知** (`direction_aware: true`):
- `forward_rr = forward_rr_long × direction` → 再判断 < -0.8
- Short 时 MFE/MAE 互换: `MFE_short = MAE_long, MAE_short = MFE_long`
- Gate/Evidence/no_opportunity 三种标签均已启用方向感知
- 配置: `labels_rr_extreme.yaml`, `labels_return_tree.yaml`, `labels_no_opportunity.yaml`

```bash
# Gate 正式训练 (与 Step 2 命令相同，但此时 prefilter/direction 已校准)
mlbot train final --no-docker \
  --config config/strategies/bpc \
  --features config/strategies/bpc/features_gate.yaml \
  --labels config/strategies/bpc/labels_rr_extreme.yaml \
  --symbol BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT,XRPUSDT,ADAUSDT \
  --timeframe 240T --data-path data/parquet_data \
  --start-date 2023-01-01 --end-date 2026-01-01 \
  --holdout-start-date 2024-05-01 --holdout-end-date 2026-01-01 --seed 42

# 记录训练输出目录
GATE_RESULT_DIR="results/train_final_<timestamp>_rr_extreme/bpc"

# Gate 应用 archetype 规则
mlbot gate apply-archetype \
  --logs ${GATE_RESULT_DIR}/predictions.parquet --strategy bpc

# Gate 优化 (plateau 搜索)
python scripts/optimize_gate_unified.py --strategy bpc \
  --logs ${GATE_RESULT_DIR}/logs_gated.parquet \
  --output ${GATE_RESULT_DIR}/gate_optimization.json
```
