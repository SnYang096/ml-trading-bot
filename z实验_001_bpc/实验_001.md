> 完整流水线：数据 → 训练 → Gate/Evidence/Entry/Execution 优化 → 配置审核 → 实盘
> 详细原理和参数见 [实验_001详细记录.md](./实验_001详细记录.md)
> 更新日期：2026-02-23
> 关键变更：对齐 FER 最新流程 — `--prepare-only` / `--promote` / gate_draft / Direction 随机方向基线 + 置换检验

## Pipeline 总览

训练是**两轮迭代**流程。Prefilter/Direction 分析只需要 **features + labels**（不需要模型），
通过 `--prepare-only` 直接导出全周期 `features_labeled.parquet`，无需先跑完整训练。

```
═══ Round 0: 数据准备 (--prepare-only, 不训练模型) ═══

Step 1: Feature Store 构建
  ↓
Step 2: 导出 features_labeled.parquet (--prepare-only, all-long)
  ↓
═══ Round 1: 校准轮 (用 features_labeled.parquet 分析，更新配置) ═══
  ↓
Step 3: Prefilter 语义分层分析 (环境条件, direction-agnostic, --promote → archetypes/)
Step 4: Direction 验证 (内部计算 rr×direction, 随机方向基线, --promote → archetypes/)
  ↓
═══ Round 2: 正式训练 (管线重新计算 direction-aware 标签) ═══
  ↓
Step 5: Gate 正式训练 → gate_draft.yaml → apply(--gate-path) → optimize(--promote) → archetypes/
  ↓
Step 6: Evidence 训练 + 优化
  ↓
Step 7: Entry Filter 优化
  ↓
Step 8: Execution 优化
  ↓
Step 9: PCM 联合回测
```

> **重要**: Step 2 产出的 `features_labeled.parquet` 是 **all-long 方向**的分析数据。
> 即使 labels 配置中 `direction_aware: true`，如果 direction.yaml 尚未创建，管线会自动 fallback 到 all-long 并输出警告。
> 分析脚本 (Step 3/4) 在 all-long 数据上正常工作。正式训练 (Step 5+) 会从 feature store 重新计算 direction-aware 标签。

> **features_labeled.parquet 内容**: 全周期 (train+test) 的特征列 + 标签 (success_no_rr_extreme) + 元数据 (timestamp, symbol 等)。
> forward_rr 为 all-long 方向，供分析脚本使用。正式训练时管线会重新计算 direction-aware 标签。
> 与 predictions.parquet 的区别: 没有模型预测值 pred，但覆盖全周期（不仅是 holdout）。

### Step 3: Prefilter 语义分层分析

> **前置依赖**: Step 2 产出的 `features_labeled.parquet`（全周期数据）

验证 BPC archetype 特征的区分力，确认 pre_filter 条件是否合理。
脚本自动读取 `config/strategies/bpc/prefilter.yaml` 的 candidates 列表，
并将结果自动回写到该文件的 `last_evaluation` 段。

```bash
# Mode A (推荐): 近 6 月选特征 + 全量 AND 覆盖率模拟

PREPARE_DIR="results/train_final_20260223_215911_rr_extreme/bpc"

python scripts/analyze_archetype_feature_stratification.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc --select-recent 6 --promote

# Mode B: 指定时间范围 + 时间稳定性
python scripts/analyze_archetype_feature_stratification.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc \
  --start-date 2025-07-01 --end-date 2026-01-01 \
  --temporal

# 全特征发现: 扫描 ALL 数值特征

PREPARE_DIR="results/train_final_20260223_215911_rr_extreme/bpc"

python scripts/analyze_archetype_feature_stratification.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc --all-features --select-recent 6
```

**关注指标**：
- 正信号（high=good）: bad rate 差异 > 5% → 可做 pre_filter 正向条件
- 反信号（high=bad）: bad rate 差异 > 5% → 可做反向 hard_gate
- temporal CV < 0.3 稳定，< 0.5 一般，>= 0.5 不稳
- Tier 自动分级: T1(composite >= 0.80) / T2(>= 0.70) / T3(>= 0.50)
- AND 累积模拟: 逐条 AND 后剩余样本量 > 1080 才可用

**操作**: 审核报告后，更新 `config/strategies/bpc/archetypes/prefilter.yaml`。

### Step 4: Direction 验证 + 特征发现

> **前置依赖**: Step 2 产出的 `features_labeled.parquet`

BPC 方向由 `direction.yaml` 确定性规则决定。需验证方向模块优于随机方向，并可用全特征扫描发现新的方向候选。

```bash
# ── 基础验证: 覆盖率 + 方向质量 ──
python z实验_005_统一研究/direction_strict_validation.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc

# ── 候选对比: 读 direction.yaml 的候选，按 Med.RR 排名 (vs 随机方向) ──
python z实验_005_统一研究/direction_strict_validation.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc --compare-features

# ── 候选对比 + 时间稳定性 + 自动生成 direction_rules (推荐) ──
python z实验_005_统一研究/direction_strict_validation.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc --compare-features --temporal --promote

# ── 全特征发现: 扫描 ALL 数值特征 ──
python z实验_005_统一研究/direction_strict_validation.py \
  --logs ${PREPARE_DIR}/features_labeled.parquet \
  --strategy bpc --compare-features --all-features --temporal
```

**Phase 1 通过标准** (覆盖率)：
- 覆盖率 100% (每个 bar 都有方向)
- Short 占比 > 15%

**Phase 2 通过标准** (方向质量, 基线: **随机方向**)：
- `median(rr × direction) > 0` → 方向信号优于随机方向
- 置换检验 p < 0.05 → 统计显著
- `mean(rr × direction) > 0` → 按方向交易总体不亏
- Short 子集样本量 >= 1080 → 统计可信

**Direction promote 标准** (自动生成 direction_rules):
- `median_rr_in_dir > 0` 且 `p_random < 0.05` — 必须同时满足
- `20% <= short_pct <= 80%` — 不能是假方向或常数偏差
- `temporal_cv < 1.5` + 未完全衰减 — 时间稳定

**操作**: `--promote` 会自动从 candidates 选出合格特征生成 `direction_rules` 并复制到 `archetypes/direction.yaml`。

> ⚠️ **BPC 方向特殊处理**:
>
> BPC 不使用自动 promote，而是**手动指定** `bpc_breakout_direction` (raw)。原因：
>
> 1. **语义约束**: BPC = Breakout-Pullback-Continuation，方向来自结构突破 (Donchian channel)，是 deterministic 的语义规则，不应被统计优化。
>
> 2. **统计表现差不等于错误**: `bpc_breakout_direction` 在当前数据上 CV=2.26, decay=3/3，但这是趋势跟踪的正常特征：
>    - **胜率低** (50% 左右) — 结构突破成功率本就不高
>    - **靠盈亏比赚钱** — 成功时走出趋势，盈亏比 > 1.5；失败时快速止损
>    - **CV 高是市场依赖** — 趋势市好，震荡市差，这是正常的 regime 依赖
>    - **decay 反映近期震荡** — 不代表特征失效，只是近期没有趋势
>
> 3. **center_sign 强度分数是错误的**: 自动选择器所选的 `bpc_score_continuation` (center_sign) 有 short%=89%，本质是常数做空偏差，不是真方向信号。已增加 `short% <= 80%` 护栏拦截。
>
> **结论**: BPC direction 手动指定 `bpc_breakout_direction` (raw) + `macd_atr` (兜底)，即使统计表现不佳，这是语义正确的选择。自动 promote 时若无合格候选，不会覆盖手动规则。

### Step 5: Gate 正式训练 (方向感知 + Prefilter)

> **前置依赖**: Step 3 Prefilter 分析 + Step 4 Direction 验证通过后，带校准配置重新训练。

**方向感知** (`direction_aware: true`):
- `forward_rr = forward_rr_long × direction` → 再判断 < -0.8
- Short 时 MFE/MAE 互换
- 配置: `labels_rr_extreme.yaml`, `labels_return_tree.yaml`

**Prefilter 训练** (`--archetype-prefilter`):
- Gate Train **必须带 prefilter**，只在符合 BPC archetype 的样本上学习
- 语义: Prefilter = “BPC 是否适用”，Gate = “假设 BPC 适用，这是好机会吗”
- 不带 prefilter 会学到通用特征，而不是 BPC 特有特征

**自动产出 gate_draft.yaml**:
- 训练完成后自动提取树分裂规则，生成 `gate_draft.yaml`
- 输出位置: `config/strategies/bpc/gate_draft.yaml` + `results/.../risk_gate_draft.yaml`
- 已是 `when/then/action` 格式，可直接用于 apply + optimize

```bash
# Gate 正式训练 (带 archetype prefilter)

mlbot train final --no-docker \
  --config config/strategies/bpc \
  --features config/strategies/bpc/features_gate.yaml \
  --labels config/strategies/bpc/labels_rr_extreme.yaml \
  --archetype-prefilter config/strategies/bpc/archetypes/prefilter.yaml \
  --symbol BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT,XRPUSDT,ADAUSDT \
  --timeframe 240T --data-path data/parquet_data \
  --start-date 2023-01-01 --end-date 2026-01-01 \
  --holdout-start-date 2024-11-01 --holdout-end-date 2026-01-01 --seed 42

# 记录训练输出目录
GATE_RESULT_DIR="results/train_final_20260223_234340_rr_extreme/bpc"
# ── 审核 gate_draft.yaml 语义方向后，用 draft 进行 apply + optimize ──
mlbot gate apply-archetype \
  --logs ${GATE_RESULT_DIR}/predictions.parquet --strategy bpc \
  --gate-path config/strategies/bpc/gate_draft.yaml

# Gate 优化 + 自动 promote 到 archetypes/gate.yaml
# 注: 必须传 --prefilter 才会应用过滤，确保 plateau 在生产分布上验证

GATE_RESULT_DIR="results/train_final_20260223_234340_rr_extreme/bpc"

python scripts/optimize_gate_unified.py --strategy bpc \
  --logs ${GATE_RESULT_DIR}/logs_gated.parquet \
  --output ${GATE_RESULT_DIR}/gate_optimization.json \
  --gate-path config/strategies/bpc/gate_draft.yaml \
  --prefilter config/strategies/bpc/archetypes/prefilter.yaml \
  --promote

# 用优化后的 archetypes/gate.yaml 重新 apply (最终版)
mlbot gate apply-archetype \
  --logs ${GATE_RESULT_DIR}/predictions.parquet --strategy bpc \
  --gate-path config/strategies/bpc/archetypes/gate.yaml
```

**Gate 流程** (全自动): 训练 → gate_draft.yaml → apply(--gate-path) → optimize(--promote) → archetypes/gate.yaml → re-apply

### Step 6: Evidence 训练

```bash
# Evidence 训练 (不带 prefilter — 全量训练 → 更多候选)
mlbot train final --no-docker \
  --config config/strategies/bpc \
  --features config/strategies/bpc/features_evidence.yaml \
  --labels config/strategies/bpc/labels_return_tree.yaml \
  --symbol BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT,XRPUSDT,ADAUSDT \
  --timeframe 240T --data-path data/parquet_data \
  --start-date 2023-01-01 --end-date 2026-01-01 \
  --holdout-start-date 2024-11-01 --holdout-end-date 2026-01-01 --seed 42

# 记录训练输出目录
EVIDENCE_RESULT_DIR="results/train_final_<timestamp>_return_tree/bpc"
# 需要先 gate apply
EVIDENCE_RESULT_DIR="results/train_final_20260223_235150_return_tree/bpc"

mlbot gate apply-archetype \
  --logs ${EVIDENCE_RESULT_DIR}/predictions.parquet \
  --out ${EVIDENCE_RESULT_DIR}/logs_gated.parquet \
  --gate-path config/strategies/bpc/archetypes/gate.yaml \
  --strategy bpc

# Evidence 优化 (自动从 candidates 加载 + 数据验证方向 + auto-promote)
python scripts/optimize_evidence_plateau.py --strategy bpc \
  --candidates ${EVIDENCE_RESULT_DIR}/evidence_candidates.yaml \
  --predictions ${EVIDENCE_RESULT_DIR}/predictions.parquet \
  --logs ${EVIDENCE_RESULT_DIR}/logs_gated.parquet \
  --output ${EVIDENCE_RESULT_DIR}/evidence_optimization.json \
  --promote
```

> **流程说明**: `--candidates` 从 Return Tree 训练导出的 `evidence_candidates.yaml` 加载候选特征，`--predictions` 自动用数据判定方向，`--promote` 优化成功后自动写入 `archetypes/evidence.yaml`。
