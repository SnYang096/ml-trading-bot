## 架构特点

### 1. 统一性
- **训练阶段**: 使用向量化批处理（快速）
- **实盘/回测**: 使用事件驱动（实时、统一）
- **代码复用**: 特征计算逻辑可复用

### 2. 内存效率
- **批处理**: GB 级内存占用
- **事件驱动**: MB 级内存占用
- **节省**: 90%+ 内存节省

### 3. 实时性
- **批处理**: 需要等待数据积累
- **事件驱动**: tick 到达即处理
- **延迟**: 从秒级降到毫秒级

### 4. 可维护性
- **模块化**: 特征计算、信号生成、交易执行分离
- **清晰**: 按事件类型组织代码
- **扩展**: 易于添加新特征和策略

## 使用示例

### 实盘交易

```python
from src.time_series_model.live.event_driven_strategy import EventDrivenStrategy
from nautilus_trader.model import InstrumentId, BarType

# 创建策略
strategy = EventDrivenStrategy(
    strategy_name="sr_reversal",
    instrument_id=InstrumentId.from_str("BTCUSDT-PERP.BINANCE"),
    bar_types={"15T": bar_type_15m, "1H": bar_type_1h},
    trade_size=0.001,
    check_interval_minutes=15,
    vpin_bucket_volume_usd=100000.0,
)

# 添加到交易节点
node.trader.add_strategy(strategy)
node.run()
```

## 重要说明

### 统一框架 vs 策略特定实现

**统一框架** (`EventDrivenStrategy` / `EventDrivenBacktest`)：
- ✅ 适合快速测试和原型验证
- ✅ 通过 `strategy_name` 加载不同模型
- ❌ 无法处理策略特定的止损/止盈/仓位逻辑
- **定位**：开发阶段的快速测试工具

**策略特定实现** (`sr_reversal_backtest.py` 等)：
- ✅ 适合真实场景和生产环境
- ✅ 实现策略特定的止损/止盈/仓位逻辑
- ✅ 精确控制策略参数
- **定位**：生产环境的实际使用

**详细说明**：请参考 [策略架构说明](./策略架构说明.md) 和 [问题解答](./问题解答.md)
