## 2. EventDrivenStrategy 支持多策略 ✅

**问题**：`EventDrivenStrategy` 可以通过加载不同模型支持不同策略，用 `strategy_name="sr_reversal"` 控制。

**答案**：✅ **是的，完全正确！**

`EventDrivenStrategy` 通过以下方式支持多策略：

1. **通过 `strategy_name` 加载不同配置**：
   ```python
   strategy = EventDrivenStrategy(
       strategy_name="sr_reversal",  # 加载 config/strategies/sr_reversal/
       # ...
   )
   ```

2. **通过 `model_path` 加载不同模型**：
   ```python
   strategy = EventDrivenStrategy(
       strategy_name="sr_reversal",
       model_path="models/sr_reversal/model.pkl",  # 加载 sr_reversal 模型
       # ...
   )
   ```

3. **自动查找默认模型**：
   ```python
   # 如果不指定 model_path，会自动查找：
   # models/{strategy_name}/model.pkl
   strategy = EventDrivenStrategy(
       strategy_name="sr_reversal",  # 自动加载 models/sr_reversal/model.pkl
       # ...
   )
   ```

**使用示例**：
```python
# SR 反转策略
sr_strategy = EventDrivenStrategy(
    strategy_name="sr_reversal",
    model_path="models/sr_reversal/model.pkl",
    # ...
)

# SR 突破策略
breakout_strategy = EventDrivenStrategy(
    strategy_name="sr_breakout",
    model_path="models/sr_breakout/model.pkl",
    # ...
)
```

## 3. 统一回测 vs 策略特定回测 ✅

**问题**：每个策略实际上有不同的参数（仓位、止损、止盈等），统一的回测和实盘可能解决不了这些差异。可以把这个统一的回测和实盘当测试用，真实场景还是要差异化吗？

**答案**：✅ **完全正确！你的理解非常准确！**

### 统一框架的定位

**统一框架** (`EventDrivenStrategy` / `EventDrivenBacktest`) 的定位是：
- ✅ **快速测试和原型验证**
- ✅ **特征工程测试**
- ✅ **模型性能评估**
- ❌ **不适合真实场景和生产环境**

### 策略特定实现的必要性

**策略特定实现** (`sr_reversal_backtest.py` 等) 的必要性：

1. **止损/止盈差异**：
   - SR 反转：止损 1×ATR，止盈 2×ATR
   - SR 突破：止损 1.5×ATR，止盈 3×ATR
   - 压缩突破：止损基于压缩区间边界
   - 趋势跟随：止损基于趋势线

2. **仓位管理差异**：
   - SR 反转：`position_size ∝ P(success)`
   - SR 突破：基于突破强度
   - 压缩突破：基于压缩强度
   - 趋势跟随：基于趋势动量

3. **加仓/减仓逻辑差异**：
   - SR 反转：❌ 不加仓，✅ 可减仓（CVD 相位转负）
   - SR 突破：✅ 加仓（突破确认后），❌ 不减仓
   - 压缩突破：✅ 加仓，✅ 减仓
   - 趋势跟随：✅ 加仓（趋势确认后），✅ 减仓（趋势减弱时）

4. **风险控制差异**：
   - 每个策略有不同的风险控制逻辑
   - 每个策略有不同的最大持仓周期
   - 每个策略有不同的最小置信度阈值

### 使用建议

#### 开发阶段：使用统一框架
