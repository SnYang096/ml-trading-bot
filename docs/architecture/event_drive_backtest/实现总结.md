# 事件驱动相关实现总结

本文档与 `事件驱动架构说明.md` 配套，记录**当前仓库真实状态**（2026 年对齐用）。旧版中列举的 `event_driven_strategy.py`、`event_driven_backtest.py`、`run_nautilus_strategy.py` 及 `backtesting/sr_*_backtest.py` **已不在本仓库路径下**；请勿再以旧文档为准。

## 1. 事件回测（与实盘对齐）

| 项 | 说明 |
|----|------|
| 入口 | `scripts/event_backtest.py` |
| 主类 | `EventBacktester`：多策略列表、`LivePCM`、按 timeframe 分组的 `IncrementalFeatureComputer` |
| 持仓 | `PositionSimulator` → `position_logic.build_position_dict` / `enforce_position`（与实盘共享实现） |
| 配置 | 策略目录 `config/strategies/<id>/`（含 `meta.yaml`、archetypes）；宪法与 PCM 读 `config/constitution/`、`config/pcm_regime.yaml` 等 |

## 2. 实盘与增量特征

| 项 | 说明 |
|----|------|
| 单策略管线 | `src/time_series_model/live/generic_live_strategy.py` |
| 特征 | `incremental_feature_computer.py`（流式 API + 批量 `compute_features_dataframe`） |
| 实盘监听 | `src/live_data_stream/order_flow_listener.py`（与回测共享 `position_logic` 思路） |
| WebSocket | `src/live_data_stream/websocket_client.py` |

## 3. 测试

- 目录：`tests/event_driven/`（增量特征、WebSocket 客户端、集成流）。
- 收集测试：`python -m pytest tests/event_driven/ --collect-only`。

事件回测脚本本身另有 `tests/unit/test_event_*.py` 等多处单测，不在 `tests/event_driven/` 目录下。

## 4. 文档索引（本目录）

| 文件 | 用途 |

| `事件驱动架构说明.md` | 总览 + 真实路径 |

## 5. 使用示例（事件回测）

```bash
python scripts/event_backtest.py --strategy bpc,fer,me-long --days 180

python scripts/event_backtest.py --strategy fer --symbols BTCUSDT --days 90 \
  --export trades.csv --output result.json
```

完整参数见 `--help`。
