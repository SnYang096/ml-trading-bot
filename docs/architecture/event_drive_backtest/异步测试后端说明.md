# 异步测试后端说明

## 一、什么是 Trio？

### 1.1 Trio 简介

**Trio** 是一个 Python 异步编程库，类似于 `asyncio`，但设计理念不同：

- **asyncio**：Python 标准库，基于事件循环
- **Trio**：第三方库，强调"结构化并发"（structured concurrency）

### 1.2 主要特点

1. **结构化并发**：确保所有并发任务都在可控范围内
2. **更好的错误处理**：自动清理资源，避免泄漏
3. **更直观的 API**：代码更易理解和调试
4. **高性能**：适合高并发场景

## 二、AnyIO 和 Trio 的关系

### 2.1 AnyIO 是什么？

**AnyIO** 是一个异步抽象层，可以在不同的异步后端之间切换：

- **asyncio 后端**：使用 Python 标准库 `asyncio`
- **trio 后端**：使用 `trio` 库

### 2.2 为什么需要 AnyIO？

1. **代码兼容性**：同一份代码可以在不同后端运行
2. **测试覆盖**：确保代码在不同后端都能正常工作
3. **灵活性**：可以根据需求选择后端

## 三、在测试中的使用

### 3.1 pytest-anyio 插件

`pytest-anyio` 插件会自动用两种后端运行异步测试：

```python
@pytest.mark.anyio
async def test_something():
    # 这个测试会运行两次：
    # 1. 使用 asyncio 后端
    # 2. 使用 trio 后端（如果已安装）
    pass
```

### 3.2 测试输出

当你运行测试时，会看到：

```
test_something[asyncio] PASSED
test_something[trio] FAILED  # 如果 trio 未安装
```

## 四、是否需要安装 Trio？

### 4.1 对于你的项目

**不需要安装 trio**，原因：

1. ✅ **asyncio 后端已足够**：你的代码使用 `asyncio`，asyncio 后端测试已通过
2. ✅ **trio 是可选**：trio 后端测试失败不影响核心功能
3. ✅ **减少依赖**：不安装 trio 可以减少项目依赖

### 4.2 什么时候需要 trio？

只有在以下情况下才需要安装 trio：

1. **需要测试 trio 兼容性**：确保代码能在 trio 后端运行
2. **实际使用 trio**：在生产环境中使用 trio 而不是 asyncio
3. **完整的测试覆盖**：希望所有后端测试都通过

## 五、解决方案

### 5.1 方案 1：忽略 trio 后端测试（推荐）

**最简单的方法**：只运行 asyncio 后端测试

```bash
# 只运行 asyncio 后端测试
pytest tests/features/test_binance_tick_aggregator.py -v -k "asyncio"
```

或者修改测试，只使用 asyncio：

```python
@pytest.mark.asyncio  # 只使用 asyncio，不使用 anyio
async def test_something():
    pass
```

### 5.2 方案 2：安装 trio（可选）

如果你想所有测试都通过：

```bash
pip install trio
```

然后所有测试都会运行两次（asyncio 和 trio）。

### 5.3 方案 3：跳过 trio 后端测试

在测试中跳过 trio 后端：

```python
import pytest

@pytest.mark.anyio
async def test_something(anyio_backend):
    if anyio_backend == "trio":
        pytest.skip("跳过 trio 后端测试")
    # 测试代码
```
