# 币安 BTCUSDT 合约交易成本计算

本文档详细说明如何在系统中计算币安 BTCUSDT 永续合约的交易成本，用于回测、模拟和实盘成本估算。

## 一、成本计算公式

### 代码中的成本公式

```python
cost = cost_per_turnover * turnover + slippage_bps * abs(target_exposure) * 1e-4
```

其中：
- `turnover = abs(target_exposure - prev_exposure)`：仓位变化量（相对于权益的比例）
- `cost_per_turnover`：每单位换手率的成本（默认 0.0002 = 2 bps）
- `slippage_bps`：每单位仓位的额外成本，包含滑点和资金费率（默认 0.0）
- `target_exposure`：目标仓位（相对于权益的比例）

### 实现位置

- **代码位置**：`src/time_series_model/rl/sim_env_3action.py:179-182`
- **配置位置**：`src/time_series_model/rl/sim_env_3action.py:SimEnvConfig`
- **默认值**：
  - `cost_per_turnover = 0.0002`（2 bps）
  - `slippage_bps = 0.0`

## 二、币安实际费用结构

### 1. 交易手续费（Trading Fee）

币安 BTCUSDT 永续合约的交易手续费：

| 订单类型 | 手续费率 | 说明 |
|---------|---------|------|
| Maker（挂单） | 0.02% = 2 bps | 提供流动性，限价单 |
| Taker（吃单） | 0.04% = 4 bps | 消耗流动性，市价单 |
| Maker（BNB 折扣） | 0.015% = 1.5 bps | 使用 BNB 支付，25% 折扣 |
| Taker（BNB 折扣） | 0.03% = 3 bps | 使用 BNB 支付，25% 折扣 |

**映射到代码参数**：
- 主要使用限价单（Maker）：`cost_per_turnover = 0.0002`（2 bps）
- 主要使用市价单（Taker）：`cost_per_turnover = 0.0004`（4 bps）
- 使用 BNB 支付 + 限价单：`cost_per_turnover = 0.00015`（1.5 bps）

**注意**：交易手续费按**交易额**计算，即 `turnover` 部分。

---

### 2. 资金费率（Funding Rate）

币安永续合约每 8 小时收取一次资金费率：

| 项目 | 说明 |
|-----|------|
| 收取频率 | 每 8 小时一次（00:00, 08:00, 16:00 UTC） |
| 费率范围 | 通常在 -0.01% 到 0.01% 之间（-10 bps 到 +10 bps） |
| 支付方向 | 多头支付资金费率，空头收取资金费率 |
| 计算方式 | `funding_cost = funding_rate * exposure * notional_value` |

**映射到代码参数（4H 时间框架）**：
- 每个 4H bar ≈ 0.5 个资金费率周期
- 平均资金费率：约 0.0001（1 bps）可计入 `slippage_bps`
- 如果做多：`slippage_bps = 0.0001`（支付资金费率）
- 如果做空：`slippage_bps = -0.0001`（收取资金费率，但通常设为 0，因为成本是绝对值）

**注意**：资金费率按**持仓额**计算，即 `abs(target_exposure)` 部分。

---

### 3. 滑点（Slippage）

滑点取决于订单大小和市场深度：

| 订单大小 | 滑点范围 | 说明 |
|---------|---------|------|
| 小单（< 1 BTC） | < 0.01% = < 1 bps | BTCUSDT 流动性好，滑点很小 |
| 中单（1-10 BTC） | 0.01-0.05% = 1-5 bps | 需要分批执行 |
| 大单（> 10 BTC） | 0.05-0.2% = 5-20 bps | 可能需要 TWAP/VWAP 执行 |
| 超大单（> 50 BTC） | 0.2-1.0% = 20-100 bps | 需要算法执行 |

**映射到代码参数**：
- 小单：`slippage_bps = 0.0001`（1 bps）
- 中单：`slippage_bps = 0.0003`（3 bps）
- 大单：`slippage_bps = 0.001`（10 bps）或更高

**注意**：滑点按**持仓额**计算，即 `abs(target_exposure)` 部分。

## 三、计算示例

### 示例 1：从空仓到 50% 多头（限价单，小单）

**场景**：
- 从 0% 仓位变到 50% 多头
- 使用限价单（Maker）
- 订单大小 < 1 BTC

**参数设置**：
- `cost_per_turnover = 0.0002`（2 bps，Maker）
- `slippage_bps = 0.0001`（1 bps，资金费率 + 小单滑点）
- `turnover = 0.5`（50% 仓位变化）
- `target_exposure = 0.5`（50% 多头）

**计算过程**：
```python
# 交易手续费
trading_fee = cost_per_turnover * turnover
            = 0.0002 * 0.5
            = 0.0001 (1 bps)

# 资金费率 + 滑点
funding_slippage = slippage_bps * abs(target_exposure) * 1e-4
                 = 0.0001 * 0.5 * 1e-4
                 = 0.00000005 (0.005 bps，可忽略)

# 总成本
total_cost = trading_fee + funding_slippage
           ≈ 0.0001 (1 bps)
```

**成本分解**：
- 交易手续费：1 bps
- 资金费率 + 滑点：≈ 0 bps（很小）
- **总成本：约 1 bps**

---

### 示例 2：从空仓到 50% 多头（市价单，小单）

**场景**：
- 从 0% 仓位变到 50% 多头
- 使用市价单（Taker）
- 订单大小 < 1 BTC

**参数设置**：
- `cost_per_turnover = 0.0004`（4 bps，Taker）
- `slippage_bps = 0.0001`（1 bps）
- `turnover = 0.5`
- `target_exposure = 0.5`

**计算过程**：
```python
trading_fee = 0.0004 * 0.5 = 0.0002 (2 bps)

```

**总成本：约 2 bps**

### 示例 3：从 30% 多头到 80% 多头（限价单，中单）

**场景**：
- 从 30% 仓位变到 80% 多头
- 使用限价单（Maker）
- 订单大小 1-10 BTC

**参数设置**：
- `cost_per_turnover = 0.0002`（2 bps）
- `slippage_bps = 0.0003`（3 bps，中单滑点）

**计算过程**：
```python

funding_slippage = 0.0003 * 0.8 * 1e-4 = 0.000000024 (可忽略)
total_cost ≈ 0.0001 (1 bps)
```

**总成本：约 1 bps**

### 示例 4：100% 仓位翻转（从 100% 多头到 100% 空头）

**场景**：
- 从 100% 多头变到 100% 空头
- 使用限价单（Maker）

**总成本：约 4 bps**

## 五、成本阈值说明

### max_cost_mean: 0.002（20 bps）

**含义**：所有步骤的平均成本不超过 0.2%（20 bps）

**计算示例**：
- 如果每步平均成本 = 20 bps
- 一年约 250 个交易日，每天 6 步（4H 时间框架）= 1500 步
- 总成本 = 0.002 × 1500 = 3.0（300%）

**说明**：这个阈值用于防止策略因过度交易而吃掉所有收益。

---

### max_cost_p95: 0.010（100 bps）

**含义**：95% 的步骤成本不超过 1.0%（100 bps）

**何时会超过**：
- 极端换手：100% 仓位翻转 + 市价单 = 0.0004 × 2.0 = 0.0008（8 bps）
- 高滑点：大单（> 10 BTC）执行 = 0.002（20 bps）或更高
- 极端资金费率：罕见但可能出现的资金费率峰值 = 0.01（100 bps）每 8h
- 组合情况：8 bps（手续费）+ 20 bps（滑点）+ 100 bps（极端资金费率）= 128 bps > 100 bps 限制

**说明**：这个阈值用于捕获执行问题，而不是正常交易成本。

## 六、实现建议

### 1. 动态成本估算

根据实际交易情况动态调整参数：

```python
def estimate_cost(
    turnover: float,
    target_exposure: float,
    order_type: str = "maker",  # "maker" or "taker"
    order_size_btc: float = 1.0,
    use_bnb: bool = False
) -> float:
    """
    估算交易成本
    
    Args:
        turnover: 仓位变化量 (0-2.0)
        target_exposure: 目标仓位 (-1.0 to 1.0)
        order_type: 订单类型
        order_size_btc: 订单大小（BTC）
        use_bnb: 是否使用 BNB 支付
    
    Returns:
        总成本（fraction of equity）
    """
    # 交易手续费
    if order_type == "maker":
        fee_rate = 0.00015 if use_bnb else 0.0002  # 1.5 bps or 2 bps
    else:
        fee_rate = 0.0003 if use_bnb else 0.0004  # 3 bps or 4 bps
    
    trading_fee = fee_rate * turnover
    
    # 滑点估算
    if order_size_btc < 1.0:
        slippage_bps = 0.0001  # 1 bps
    elif order_size_btc < 10.0:
        slippage_bps = 0.0003  # 3 bps
    else:
        slippage_bps = 0.001  # 10 bps
    
    # 资金费率（假设平均 1 bps per 4H）
    funding_bps = 0.0001
    
    # 总滑点（滑点 + 资金费率）
    total_slippage_bps = slippage_bps + funding_bps
    
    # 滑点成本
    slippage_cost = total_slippage_bps * abs(target_exposure) * 1e-4
    
    # 总成本
    total_cost = trading_fee + slippage_cost
    
    return total_cost
```

### 2. 实时成本监控

在实盘中监控实际成本，并与估算值对比：

```python
def record_actual_cost(
    symbol: str,
    order_id: str,
    executed_price: float,
    executed_qty: float,
    order_type: str,
    funding_rate: float,
    estimated_cost: float
) -> Dict[str, float]:
    """
    记录实际交易成本
    
    Returns:
        实际成本字典
    """
    # 计算实际手续费
    if order_type == "maker":
        fee_rate = 0.0002
    else:
        fee_rate = 0.0004
    
    notional = executed_price * executed_qty
    actual_fee = fee_rate * notional
    
    # 计算实际滑点（如果有预期价格）
    # slippage = abs(executed_price - expected_price) / expected_price
    
    # 计算资金费率成本
    # funding_cost = funding_rate * notional * exposure_duration
    
    # 总实际成本
    actual_cost = actual_fee  # + slippage + funding_cost
    
    # 与估算值对比
    cost_error = actual_cost - estimated_cost
    
    return {
        "actual_cost": actual_cost,
        "estimated_cost": estimated_cost,
        "cost_error": cost_error,
        "fee": actual_fee,
    }
```

### 3. 成本阈值检查

在 Constitution 中实现成本检查：

```python
def validate_cost(
    rolling_mean_cost: float,
    rolling_p95_cost: float,
    max_cost_mean: float = 0.002,
    max_cost_p95: float = 0.010
) -> Tuple[bool, List[str]]:
    """
    检查成本是否超过阈值
    
    Returns:
        (is_valid, violations)
    """
    violations = []
    
    if rolling_mean_cost > max_cost_mean:
        violations.append(f"mean_cost {rolling_mean_cost:.4f} > {max_cost_mean:.4f}")
    
    if rolling_p95_cost > max_cost_p95:
        violations.append(f"p95_cost {rolling_p95_cost:.4f} > {max_cost_p95:.4f}")
    
    return len(violations) == 0, violations
```
