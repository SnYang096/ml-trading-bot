👉 它们决定 **“放大 or 缩小”，而不是“进 or 不进”**

* 高谱熵 → 止损易被扫

**例子：**

```yaml

```

# 一、Tradeability Head

> **不开仓，也能救命的模块**

## 1️⃣ Tradeability Head 的哲学定位（非常重要）

先说一句硬话：

> **Tradeability ≠ Regime ≠ Signal**
> Tradeability 回答的是：
> 👉 *“现在这个市场，值得我暴露风险吗？”*

它的权力只有三种：

* ✅ 允许交易
* ⚠️ 降速 / 降仓
* ⛔ 冻结（NO_TRADE）

**它永远不产生方向，也永远不触发交易。**

---

## 2️⃣ Tradeability Head 看什么？（你提到的那些“该退居二线的指标”）

这正是 WPT / Hurst / Hilbert / Spectrum 的**正位**。

### 合法输入（全部是“结构稳定性描述”，不是 alpha）

| 特征                      | 作用       |
| ----------------------- | -------- |
| Hurst                   | 是否具备可持续性 |
| Spectrum entropy        | 是否多频混沌   |
| Hilbert phase stability | 相位是否可预测  |
| WPT energy dispersion   | 能量是否集中   |
| BB width percentile     | 是否处在可展开区 |

---

## 3️⃣ Tradeability Head 的最小可执行 DSL

```yaml
tradeability_head:
  allow_trade:
    all:
      - hurst > 0.52
      - spectrum_entropy < 0.75
      - hilbert_phase_stability > 0.6

  size_multiplier:
    expr: clip(
      0.5
      + 0.5 * hurst
      + 0.3 * wpt_energy_concentration,
      0.3,
      1.2
    )

  hard_freeze:
    any:
      - spectrum_entropy > 0.9
      - wpt_energy_concentration < 0.2
```

### 行为解释

* **allow_trade = false** → 所有 Archetype 强制 NO_TRADE
* **size_multiplier < 0.5** → 即便 Signal 满分，也只是“试探仓”
* **hard_freeze** → 风控层直接接管

👉 这是**市场不配合时的“安全带”**。
