# 树模型规则导出与维护方法

## 📋 概述

本文档阐述如何从树模型中提取规则、如何将树模型特征迁移到规则类系统，以及如何维护和评估规则类系统。

**核心哲学**：
- 树模型 = 高效的"假设发现器"
- 规则系统 = 可维护的"因果假设库"
- 规则导出 ≠ 复制 Sharpe，而是为了**可归因性 (Attributability)**

---

## 一、树模型规则导出的语义性价值

### 1.1 核心价值：可归因性

树模型规则导出的核心价值在于**实盘失效时的可归因性**，而不是复制树模型的 Sharpe。

#### 场景对比

**树模型（黑盒）**：
```python
# 实盘 Sharpe 从 1.5 降到 0.3
# 问题：不知道是哪个因素变了
可能原因:
  - 市场 regime 变化？
  - 某个特征失效？
  - 数据分布漂移？
→ 无法定位问题
```

**规则类（可归因）**：
```python
# 实盘 Sharpe 从 1.5 降到 0.3
# 分析每个规则的贡献
fbf_vp_skewness_confirms: Sharpe 0.8 → 0.2 ❌ 这个语义失效！
fbf_wick_exhaustion: Sharpe 0.5 → 0.4 ✅ 仍然 work
fbf_sr_distance: Sharpe 0.6 → 0.5 ✅ 基本稳定

→ 定位问题: "VP偏度"这个语义在当前市场失效
→ 解决方案: 暂时禁用或调整阈值
```

#### 新标的迁移

**树模型**：
```python
# 在 BTC 上训练的模型，应用到 ETH
→ Sharpe 可能从 1.5 变成 -0.3
→ 不知道是哪些假设在 ETH 上不成立
```

**规则类**：
```python
# 在 BTC 上 work 的规则，应用到 ETH
分析:
  - sr_distance 规则: ✅ 通用（所有标的都有 SR）
  - vp_skewness 规则: ❌ ETH 的 VP 分布不同
  - wick_exhaustion 规则: ✅ 通用

→ 只调整"VP相关规则"的阈值，其他保持不变
```

### 1.2 红线原则

#### ⚠️ 红线 1：规则可归因 ≠ 因果已被证明

**正确理解**：
- 规则系统给出的是：**"哪一类语义假设失效了"**
- 不是：**"这个语义在任何 regime 下都必然不成立"**

**正确用法**：
```yaml
# VP skewness 在当前市场失效
→ 降权 / 禁用 / 收紧阈值
→ 等 regime 切换后重新放行
```

**错误用法**：
```yaml
# ❌ 直接删除特征、删除语义
# 应该保留语义，只调整阈值或禁用
```

#### ⚠️ 红线 2：不要让"规则 Sharpe"反向绑架你

**正确问法**：
- ✅ "它是不是一个**必要条件 / 保护条件 / 稳定器**？"
- ❌ "这个规则单独 Sharpe 很低，是不是不重要？"

**示例**：
```yaml
# jump_risk_high 规则
# 单独 Sharpe 可能很低，但它是重要的保护条件
# 移除后可能导致极端市场下的巨大亏损
```

## 二、多个相似特征的聚合使用

### 2.1 核心洞察

**关键理解**：多个 SR 特征不是"相似特征"，而是**不同层级的 SR 定义**。

```python
SR 层级系统:
  L1_structural: ['swing_high', 'swing_low']      # 结构性 SR（最强）
  L2_volume: ['vol_hal_high', 'vol_hal_low', 'poc']  # 成交密集区（中强）
  L3_pattern: ['zigzag_peak', 'zigzag_trough']    # 形态 SR（较弱）
```

### 2.2 推荐方案：分层 SR 系统

> **⚠️ 重要说明（2026-01-27 更新）**：
> 
> **不采用预设层级权重的原因**：
> - 所有 SR（swing、HAL、POC、zigzag）都是**心理锚点**，需要通过市场反应（SQS）来验证强度
> - 预设层级权重过于主观，不符合数据驱动理念
> - 当前实现通过 **SQS 测试**动态评估 SR 强度，更客观、更自适应
> - 不同市场、不同时期，同一 SR 的强度可能不同，应该让市场告诉你哪个 SR 更有效
> 
> **当前实现的优势**：
> - ✅ 所有 SR 平等对待，通过 SQS 竞争
> - ✅ 市场会告诉你哪个 SR 更有效（数据驱动）
> - ✅ 适合 ML 系统：让模型学习哪些 SR 更重要
> 
> **文档中的分层设计**更适合规则系统（需要明确的优先级），而当前实现更适合 ML 系统（数据驱动）。

#### 方案 A：最近 SR + 层级（执行系统推荐）

> **注意**：此方案为概念性设计，当前实现不采用预设层级权重，而是通过 SQS 动态评估。

```python
class HierarchicalSRSystem:
    """
    分层 SR 系统：不同特征代表不同"强度"的 SR
    """
    
    def __init__(self):
        self.sr_levels = {
            'L1_structural': ['swing_high', 'swing_low'],
            'L2_volume': ['vol_hal_high', 'vol_hal_low', 'poc'],
            'L3_pattern': ['zigzag_peak', 'zigzag_trough'],
        }
    
    def calculate_sr_distance(self, sample):
        """
        找"最近的有效 SR"，而不是平均
        """
        # Step 1: 计算到各层级 SR 的距离（ATR 归一化）
        distances = {}
        atr = sample['atr']
        
        for level, sr_types in self.sr_levels.items():
            for sr_type in sr_types:
                if sr_type in sample:
                    distances[sr_type] = abs(sample[sr_type] - sample['close']) / atr
        
        # Step 2: 分上下方向
        current_price = sample['close']
        upper_srs = {
            k: v for k, v in distances.items()
            if any(x in k for x in ['high', 'peak']) or 
               (k.replace('_distance', '') in sample and 
                sample[k.replace('_distance', '')] > current_price)
        }
        lower_srs = {
            k: v for k, v in distances.items()
            if any(x in k for x in ['low', 'trough']) or
               (k.replace('_distance', '') in sample and 
                sample[k.replace('_distance', '')] < current_price)
        }
        
        # Step 3: 返回最近的 SR 及其层级
        nearest_upper = min(upper_srs.items(), key=lambda x: x[1]) if upper_srs else (None, float('inf'))
        nearest_lower = min(lower_srs.items(), key=lambda x: x[1]) if lower_srs else (None, float('inf'))
        
        nearest = nearest_upper if nearest_upper[1] < nearest_lower[1] else nearest_lower
        
        return {
            'nearest_sr_type': nearest[0],
            'nearest_sr_distance': nearest[1],
            'nearest_sr_level': self._get_sr_level(nearest[0]),
            'sr_side': 'upper' if nearest_upper[1] < nearest_lower[1] else 'lower'
        }
    
    def _get_sr_level(self, sr_type):
        for level, types in self.sr_levels.items():
            if sr_type in types:
                return level
        return 'unknown'
```

#### 方案 B：SR 质量分数（SQS）- 多 SR 重合

```python
class SRQualityScore:
    """
    SR 质量分数：多个 SR 重合的地方 = 更强的 SR
    """
    
    def calculate_sqs(self, sample, distance_threshold=0.5):
        """
        计算 SR 质量分数
        
        Args:
            distance_threshold: SR 重合的距离阈值（ATR 倍数）
        
        Returns:
            sqs: SR 质量分数 [0, 1]
        """
        # Step 1: 收集所有 SR 价位
        sr_levels = {
            'swing_high': sample.get('swing_high'),
            'swing_low': sample.get('swing_low'),
            'vol_hal_high': sample.get('vol_hal_high'),
            'vol_hal_low': sample.get('vol_hal_low'),
            'poc': sample.get('poc'),
            'zigzag_peak': sample.get('zigzag_peak'),
            'zigzag_trough': sample.get('zigzag_trough'),
        }
        
        # 过滤 None
        sr_levels = {k: v for k, v in sr_levels.items() if v is not None}
        
        # Step 2: 归一化到 ATR 单位
        current_price = sample['close']
        atr = sample['atr']
        
        sr_distances = {
            k: (v - current_price) / atr 
            for k, v in sr_levels.items()
        }
        
        # Step 3: 找最近的 SR
        nearest_sr_name = min(sr_distances.items(), key=lambda x: abs(x[1]))[0]
        nearest_sr_distance = sr_distances[nearest_sr_name]
        
        # Step 4: 计算有多少其他 SR 在附近
        overlap_count = sum(
            1 for name, dist in sr_distances.items()
            if name != nearest_sr_name and abs(dist - nearest_sr_distance) < distance_threshold
        )
        
        # Step 5: 计算质量分数（加权）
        weights = {
            'swing_high': 3.0,
            'swing_low': 3.0,
            'vol_hal_high': 2.0,
            'vol_hal_low': 2.0,
            'poc': 2.0,
            'zigzag_peak': 1.0,
            'zigzag_trough': 1.0,
        }
        
        weighted_overlap = sum(
            weights.get(name, 1.0) for name, dist in sr_distances.items()
            if abs(dist - nearest_sr_distance) < distance_threshold
        )
        
        max_possible_weight = sum(weights.values())
        sqs = weighted_overlap / max_possible_weight if max_possible_weight > 0 else 0.0
        
        return {
            'sqs': sqs,
            'overlap_count': overlap_count,
            'nearest_sr_type': nearest_sr_name,
            'nearest_sr_distance': abs(nearest_sr_distance),
            'contributing_srs': [
                name for name, dist in sr_distances.items()
                if abs(dist - nearest_sr_distance) < distance_threshold
            ]
        }
```

### 2.3 在 Archetype 中使用

```yaml
FailedBreakoutFade:
  preconditions:
    - fbf_near_high_quality_sr:
        reason: "靠近高质量 SR（多层级重合）"
        when:
          all_of:
            - sr_quality_score:  # SQS
                quantile_gte: 0.6  # 至少 60 分位的 SR 质量
            
            - nearest_sr_distance:  # 距离
                quantile_lt: 0.3  # 距离最近 SR < 0.3 ATR
            
            - nearest_sr_level:  # 层级
                in: ['L1_structural', 'L2_volume']  # 必须是强 SR
        then:
          action: require
    
    # 可选：如果 SR 质量极高，放宽其他条件
    - fbf_exceptional_sr:
        reason: "SR 质量极高时，其他条件可放宽"
        when:
          all_of:
            - sr_quality_score: {quantile_gte: 0.85}
            - overlap_count: {gte: 3}  # 至少 3 个 SR 重合
        then:
          action: allow_relax  # 允许放宽其他 preconditions
```

### 2.5 SQS 使用原则

**⚠️ 红线**：SQS 不要直接参与方向决策，只做强度/置信度调节

```yaml
# ❌ 不推荐
direction = sign(sqs_hal_high * something)  # 或 sqs_hal_low

# ✅ 推荐用法
# - 决定是否允许
# - 决定是否放宽
# - 决定 RR / size / entry aggressiveness
```

### 2.6 SR 特征使用指南

#### 2.6.1 相关特征列表

| 特征名 | 含义 | 取值范围 | 涵盖范围 | 用途 |
|--------|------|---------|---------|------|
| `dist_to_nearest_sr` | 到最近 SR 的距离（ATR 归一化） | [-3, 3] ATR | 所有 SR | 判断是否靠近 SR |
| `direction_to_nearest_sr` | 方向（价格在 SR 上方/下方） | -1 (below) / +1 (above) | 所有 SR | 判断方向 |
| `sr_strength_max` | **所有 SR 的最大强度（SQS）** | [0, 1] | **15 个 SR**（swing, zigzag, HAL, POC, Bollinger, OLS, VWAP） | **判断 SR 质量（最全面）** |
| `sqs_hal_high` | HAL high 的 SQS | [0, 1] | **仅 HAL high** | HAL high 质量（**单个 SR**） |
| `sqs_hal_low` | HAL low 的 SQS | [0, 1] | **仅 HAL low** | HAL low 质量（**单个 SR**） |

#### 2.6.1.1 计算方式区别

**`sqs_hal_high` / `sqs_hal_low`**：
- 只针对 **HAL high/low 这一个特定的 SR 价格**计算 SQS
- 使用 `calculate_sqs` 函数，传入 `hal_high` 或 `hal_low` 作为 `sr_price`
- 只计算这一个 SR 的质量

**`sr_strength_max`**：
- 通过 `_compute_boundary_strengths` 计算**所有 SR 边界的 SQS**
- 涵盖 **15 个 SR 类型**：
  - swing_high_s, swing_low_s, swing_high_l, swing_low_l
  - zigzag_high, zigzag_low
  - hal_high, hal_low
  - poc_level
  - boll_upper, boll_lower
  - ols_upper, ols_lower, ols_mid
  - vwap_level
- 对每个 SR 都调用 `calculate_sqs` 计算 SQS
- 取所有 SR 的**最大 SQS**
- **额外考虑**：
  - 边界重合（cluster_bonus）：如果多个 SR 在附近，有加成
  - 压缩质量（compression_bonus）：压缩期 SR 更有效

**注意**：`sqs_f`（通用的 SQS）已被删除，因为：
- 只涵盖 HAL（high 和 low），不够全面
- `sr_strength_max` 涵盖所有 SR 类型（15 个），更全面
- 当前实现使用 `sr_strength_max` 作为 SR 质量的通用指标

#### 2.6.2 推荐使用方式

**1. 在 Gate Rules 中使用（execution_archetypes.yaml）**

```yaml
# ✅ 推荐：使用 dist_to_nearest_sr
- id: lsr_liquidity_sweep_detected
  phase: preconditions
  when:
    any_of:
    - dist_to_nearest_sr:
        quantile_lt: 0.2  # 距离 SR < 0.2 ATR
    min_matches: 1

# ✅ 推荐：使用 sr_strength_max 判断 SR 质量
- id: fbf_near_quality_sr
  phase: preconditions
  when:
    all_of:
    - sr_strength_max:
        quantile_gte: 0.6  # SR 强度 >= 60 分位
    - dist_to_nearest_sr:
        quantile_lt: 0.3  # 距离 < 0.3 ATR
    min_matches: 2
```

**2. 在树策略中使用**

```yaml
# features.yaml
features:
  - sr_strength_max_f  # 输出: sr_strength_max, dist_to_nearest_sr, direction_to_nearest_sr
  - sqs_hal_high_f     # 可选：如果特别关注 HAL
  - sqs_hal_low_f      # 可选：如果特别关注 HAL
```

**3. 特征选择建议**

- **必须用**：`sr_strength_max_f`（包含 `sr_strength_max`, `dist_to_nearest_sr`, `direction_to_nearest_sr`）
  - `sr_strength_max` 涵盖所有 SR 类型（15 个），取最大值，并考虑重合和压缩
- **可选**：`sqs_hal_high_f`, `sqs_hal_low_f`（如果特别关注 HAL 质量）
  - 只针对 HAL，不涵盖其他 SR
- **不推荐**：单独使用 `sqs_f`（只涵盖 HAL，不如 `sr_strength_max` 全面）

#### 2.6.3 当前使用情况检查

**execution_archetypes.yaml**：
- ✅ 使用了 `dist_to_nearest_sr`（正确，已经是 ATR 归一化）
- ✅ 使用了 `sr_strength_max`（已添加，用于判断 SR 质量）

#### 2.6.4 距离特征的区别

**`dist_to_nearest_sr`**：
- **来源**：`sr_strength_max_f` 或 `sr_strength_max_close_f` 的输出
- **计算**：`(sr_price - close) / atr`（ATR 归一化）
- **单位**：ATR 倍数（如 1.5 表示 1.5 倍 ATR）
- **取值范围**：典型 [-3, 3] ATR
- **符号**：正数表示价格在 SR 上方，负数表示在下方
- **✅ 推荐使用**：已经是 ATR 归一化，直接使用


**`direction_to_nearest_sr`**：
- **来源**：`sr_strength_max_f` 或 `sr_strength_max_close_f` 的输出
- **计算**：`1.0 if current_price < nearest_price else -1.0`
- **含义**：
  - `+1.0`：价格在 SR **下方**（向上接近 SR）
  - `-1.0`：价格在 SR **上方**（向下接近 SR）
- **用途**：判断价格相对于 SR 的位置方向

**使用建议**：
- ✅ **推荐**：直接使用 `dist_to_nearest_sr`（从 `sr_strength_max_f` 获取，已经是 ATR 归一化）
- ✅ **可选**：使用 `direction_to_nearest_sr` 判断方向（如果需要）

#### 2.6.5 使用原则总结

1. **距离判断**：用 `dist_to_nearest_sr`（从 `sr_strength_max_f` 获取，已经是 ATR 归一化）
2. **质量判断**：用 `sr_strength_max`（涵盖所有 15 个 SR 类型，取最大值，并考虑重合和压缩）
3. **方向判断**：用 `direction_to_nearest_sr`（如果需要）
4. **HAL 特定**：用 `sqs_hal_high` / `sqs_hal_low`（如果特别关注 HAL，但通常不需要）
5. **不要**：
   - 用 `sqs_hal_high` / `sqs_hal_low` 做方向决策（只做强度/置信度调节）
   - 用 `sqs_hal_high` / `sqs_hal_low` 代替 `sr_strength_max`（只涵盖 HAL，不够全面）

## 三、规则类特征重要性判断

### 3.1 核心挑战

```
树模型: feature_importance, SHAP 直接给出
规则类: 没有这些工具，怎么办？
```

**解决方案**：消融实验 (Ablation Study)

### 3.2 方法 1：单规则消融（最基础，必做）

```python
class RuleImportanceAnalyzer:
    """
    规则重要性分析器
    """
    
    def __init__(self, archetype_config, data):
        self.config = archetype_config
        self.data = data
    
    def ablation_study(self, archetype_name):
        """
        逐个移除规则，观察影响
        """
        # Baseline: 所有规则都用
        baseline_result = self.run_archetype(archetype_name, rules='all')
        
        results = {
            'baseline': {
                'sharpe': baseline_result['sharpe'],
                'win_rate': baseline_result['win_rate'],
                'trade_rate': baseline_result['trade_rate'],
            }
        }
        
        # 逐个移除规则
        all_rules = self.config[archetype_name]['when_then_rules']
        
        for rule in all_rules:
            rule_id = rule['id']
            
            # 移除这个规则后运行
            ablated_result = self.run_archetype(
                archetype_name, 
                exclude_rules=[rule_id]
            )
            
            # 计算影响
            impact = {
                'sharpe_change': ablated_result['sharpe'] - baseline_result['sharpe'],
                'win_rate_change': ablated_result['win_rate'] - baseline_result['win_rate'],
                'trade_rate_change': ablated_result['trade_rate'] - baseline_result['trade_rate'],
            }
            
            results[rule_id] = {
                'ablated_metrics': ablated_result,
                'impact': impact,
                'importance': self._calculate_importance(impact)
            }
        
        return results
    
    def _calculate_importance(self, impact):
        """
        计算规则重要性分数
        
        重要性 = Sharpe 变化的绝对值（移除后 Sharpe 下降越多，越重要）
        """
        return -impact['sharpe_change']  # 负号：下降越多，重要性越高
```

### 3.3 方法 2：特征贡献分解（进阶）

```python
class FeatureContributionAnalyzer:
    """
    分析每个特征（不是规则）的贡献
    """
    
    def analyze_feature_contribution(self, archetype_name):
        """
        分析每个特征对 archetype 的贡献
        """
        # Step 1: 提取 archetype 用到的所有特征
        features_used = self._extract_features_from_rules(archetype_name)
        # 例如: ['sr_distance_normalized', 'vp_skewness', 'wick_exhaustion', ...]
        
        # Step 2: 为每个特征计算贡献
        contributions = {}
        
        for feature in features_used:
            # 找到使用这个特征的所有规则
            rules_using_feature = self._find_rules_using_feature(
                archetype_name, 
                feature
            )
            
            # Baseline: 这些规则都启用
            baseline = self.run_archetype(archetype_name, rules='all')
            
            # Ablation: 禁用这些规则
            ablated = self.run_archetype(
                archetype_name,
                exclude_rules=rules_using_feature
            )
            
            # 计算贡献
            contributions[feature] = {
                'sharpe_contribution': baseline['sharpe'] - ablated['sharpe'],
                'rules_using': rules_using_feature,
                'usage_count': len(rules_using_feature)
            }
        
        # Step 3: 排序
        ranked = sorted(
            contributions.items(),
            key=lambda x: x[1]['sharpe_contribution'],
            reverse=True
        )
        
        return {
            'feature_contributions': dict(ranked),
            'most_important': ranked[0][0] if ranked else None,
            'least_important': ranked[-1][0] if ranked else None
        }
```

**重要理解**：
- Feature contribution ≠ trading edge
- Feature contribution = semantic dependency

它回答的是：
- ✅ "这个 archetype 依赖哪些语义维度？"
- ❌ 不是："这个特征单独能不能赚钱？"

### 3.4 方法 3：边际贡献分析（最严谨，可选）

```python
class MarginalContributionAnalyzer:
    """
    计算每个规则的边际贡献（类似 SHAP，但用于规则）
    """
    
    def approximate_shapley(self, archetype_name, n_samples=100):
        """
        用蒙特卡洛抽样近似 Shapley 值
        
        ⚠️ 警告：精确 Shapley 值计算量巨大（2^n）
        - 5 个规则: 32 次运行
        - 10 个规则: 1024 次运行
        - 20 个规则: 1,048,576 次运行（不现实）
        
        因此使用蒙特卡洛近似
        """
        import numpy as np
        
        all_rules = self.config[archetype_name]['when_then_rules']
        n_rules = len(all_rules)
        
        shapley_estimates = {rule['id']: [] for rule in all_rules}
        
        for _ in range(n_samples):
            # 随机排列规则
            perm = np.random.permutation(n_rules)
            
            # 按顺序加入规则，记录每个规则的边际贡献
            current_sharpe = 0
            current_rules = []
            
            for i in perm:
                rule_id = all_rules[i]['id']
                current_rules.append(rule_id)
                
                # 运行当前规则集
                result = self.run_archetype(
                    archetype_name,
                    include_rules=current_rules
                )
                
                # 边际贡献
                marginal = result['sharpe'] - current_sharpe
                shapley_estimates[rule_id].append(marginal)
                
                current_sharpe = result['sharpe']
        
        # 平均
        shapley_values = {
            rule_id: np.mean(estimates)
            for rule_id, estimates in shapley_estimates.items()
        }
        
        return shapley_values
```

**使用建议**：
- 用在 **Top rules**（最重要的规则）
- 用在 **回溯分析**（历史数据）
- **不用**在日常迭代（计算成本高）

## 四、树模型特征迁移到规则类

### 4.1 核心原则

**不需要"仔细移植"，但需要战略性借鉴**

#### 判断标准

```
新特征 X（如 vp_skewness）
  ↓
问题 1: 它是否有独特的"语义"？
  ├─ 是 → 继续
  └─ 否（和现有特征语义重复）→ 不加
  ↓
问题 2: 它是否"入场时可观测"？
  ├─ 是 → 继续
  └─ 否（需要未来数据）→ 不加
  ↓
问题 3: 它是否能"解释为市场结构"？
  ├─ 是 → 继续
  └─ 否（只是统计相关）→ 不加
  ↓
问题 4: 加了它后，规则是否更"可解释"？
  ├─ 是 → 加入
  └─ 否（变得更复杂）→ 不加
```

### 4.2 推荐迁移的特征

#### ✅ 应该添加的 2 个特征

**1. `vp_skewness`（成交量分布偏度）⭐⭐⭐**

```yaml
语义: 成交量分布是否偏向一侧

FBF 场景应用:
  if vp_skewness > 0.5:  # 上方成交密集
    # 假突破向上更可能失败
    fbf_direction = SHORT
    fbf_confidence *= 1.2
  
  elif vp_skewness < -0.5:  # 下方成交密集
    fbf_direction = LONG
    fbf_confidence *= 1.2

AER 场景应用:
  if abs(vp_skewness) > 0.7:
    # 强偏度 = 趋势仍强 = 不是衰竭
    skip_aer("trend still strong")
```

**2. `vp_entropy`（成交分散度）⭐⭐**

```yaml
语义: 市场共识程度

高 entropy（分散）→ 多空博弈激烈 → 反转时机
低 entropy（集中）→ 市场共识强 → 压缩后爆发

FBF 场景:
  if vp_entropy > quantile(0.7):
    # 成交分散 = SR 附近博弈激烈 = 反转概率高
    fbf_confidence *= 1.3

ME 场景:
  if vp_entropy < quantile(0.3):
    # 成交集中 = 市场共识强 = 突破后动量足
    me_confidence *= 1.2
```

#### ❌ 不需要添加的特征

| 特征 | 原因 |
|------|------|
| `vp_width_ratio` | 和 `bb_width` 高度相关 |
| `vp_lv_ratio` | 和 `liquidity_void_risk` 重复 |
| `vp_hv_ratio` | 和 SR 相关特征重复 |
| `vp_poc_deviation` | 已经有 `dist_to_nearest_sr` |

FailedBreakoutFade:

### 6.1 核心原则

4. **特征迁移 = 战略性借鉴，不是全部移植**
