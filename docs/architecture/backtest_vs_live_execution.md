# 回测与实盘执行层对比

## 概述

本文档详细对比Execution层在回测和实盘中的不同使用方式，重点分析Tier选择和数学特征调整的应用场景。

## 回测执行流程

### 1. 数据处理
- **批量处理**：一次性加载整个历史数据集
- **特征计算**：使用完整的数据集计算所有特征
- **流式模拟**：虽然是批量数据，但数学特征计算使用流式模式避免未来函数

### 2. Gate层决策
- **输入**：仅使用结构/订单流特征（如bpc_score_breakout, cvd_divergence等）
- **目的**：模拟历史上的交易准入决策
- **特点**：不使用数学特征，避免过拟合

### 3. Evidence层评估
- **输入**：仅基于结构/订单流特征计算证据分数
- **目的**：评估历史数据中BPC信号的质量
- **特点**：保持alpha质量评估的纯粹性

### 4. Execution层应用
- **evidence_score**：来自Evidence层的alpha质量评分
- **math_features**：用于计算execution_noise_penalty的数学特征
- **Tier选择**：根据evidence_score选择档位
- **参数调整**：根据noise_penalty调整执行参数
- **记录**：详细记录每个决策及其参数调整

### 5. 回测特定功能
- **参数敏感性分析**：测试不同Tier参数的效果
- **多情景模拟**：模拟不同市场条件下的表现
- **绩效归因**：分析数学特征调整对绩效的影响

## 实盘执行流程

### 1. 数据处理
- **流式处理**：逐根K线处理实时数据
- **特征计算**：基于历史状态和当前数据计算特征
- **实时更新**：特征值实时更新，状态持续维护

### 2. Gate层决策
- **输入**：仅使用结构/订单流特征
- **目的**：实时判断是否进入交易
- **特点**：毫秒级响应，严格的风控

### 3. Evidence层评估
- **输入**：基于实时计算的结构/订单流特征
- **目的**：实时评估当前BPC信号质量
- **特点**：快速计算，及时反馈

### 4. Execution层应用
- **evidence_score**：实时计算的alpha质量评分
- **math_features**：实时计算的数学特征用于noise_penalty
- **Tier选择**：实时选择执行档位
- **参数调整**：实时调整执行参数
- **订单管理**：立即执行交易指令

### 5. 实盘特定功能
- **风险管理**：实时监控和干预
- **订单优化**：最优执行算法
- **状态同步**：与交易所状态同步

## 配置文件使用差异

### 回测配置
```yaml
# config/backtest/bpc_execution_tiers.yaml
version: 2
noise_penalty:
  enabled: true
  short_window: 20
  long_window: 120
  weights:
    wpt_entropy: 0.35
    spectrum_width: 0.30
    hilbert_phase_var: 0.20
    hurst_inverse: 0.15
  max_penalty: 0.8
  # 回测特定参数
  sensitivity_analysis:
    enabled: true
    parameters:
      - sl_r: [0.8, 1.0, 1.2]
      - tp_r: [2.0, 2.5, 3.0]

tiers:
  - name: "强证据"
    score_min: 0.70
    params:
      sl_r: 0.8
      tp_r: 3.0
      size_multiplier: 1.2
```

### 实盘配置
```yaml
# config/live/bpc_execution_tiers.yaml
version: 2
noise_penalty:
  enabled: true
  short_window: 20
  long_window: 120
  weights:
    wpt_entropy: 0.35
    spectrum_width: 0.30
    hilbert_phase_var: 0.20
    hurst_inverse: 0.15
  max_penalty: 0.8
  # 实盘特定参数
  risk_limits:
    max_drawdown: 0.15
    max_position_size: 10000

tiers:
  - name: "强证据"
    score_min: 0.70
    params:
      sl_r: 0.8  # 固定参数，经过回测验证
      tp_r: 3.0
      size_multiplier: 1.2
```

## 起始参数差异

### 回测起始参数
- **完整历史数据**：从指定日期开始的完整数据
- **初始状态**：通常为零状态或预设状态
- **参数范围**：可能包含参数优化的多个设置

### 实盘起始参数
- **当前市场数据**：从当前时间点开始的实时数据
- **实时状态**：从上次运行保存的状态恢复
- **固定参数**：经过回测验证的最优参数

## 核心相同点

尽管应用场景不同，但以下核心要素在回测和实盘中保持一致：

1. **分层架构**：Gate/Evidence/Execution三层分离
2. **数学特征使用原则**：仅在Execution层使用，不参与决策
3. **Noise Penalty计算**：相同的算法和权重
4. **Tier选择逻辑**：相同的档位选择标准
5. **参数调整规则**：相同的调整公式

## 风险控制差异

### 回测风险控制
- **事后分析**：通过历史数据分析风险
- **参数优化**：寻找最优参数组合
- **压力测试**：模拟极端市场条件

### 实盘风险控制
- **事前控制**：实时监控和干预
- **硬性限制**：不可突破的风险阈值
- **应急机制**：自动暂停和人工干预

## 性能要求差异

### 回测性能
- **吞吐量优先**：尽可能快地处理大量历史数据
- **精度优先**：精确复现历史行情
- **资源充足**：可以使用更多计算资源

### 实盘性能
- **延迟优先**：毫秒级响应时间
- **稳定性优先**：持续稳定的性能
- **资源受限**：在有限资源下运行

## 总结

回测和实盘虽然在数据处理方式、配置使用和性能要求上有所不同，但Execution层的核心逻辑保持一致。数学特征调整机制在两种场景下都发挥着相同的作用：根据市场噪声水平动态调整执行参数，确保在不同市场环境下都能以适当的风险水平进行交易。
