# 为什么用统计规则管线而不是端到端模型

> 当前方案（Gate 规则 + Evidence 统计筛选）对比端到端树模型 / MLP 直出信号的设计决策记录。

---

## 一句话结论

**当前方案在 crypto 小样本、非平稳场景下是正确选择。**
端到端模型的理论天花板更高，但在当前数据量下不可达 — 找到的"非线性交互"99% 是过拟合。

---

## 方案对比

| 维度              | 当前方案 (统计规则 + Evidence)                  | 端到端模型 (LightGBM/MLP 直出信号) |
| ----------------- | ----------------------------------------------- | ---------------------------------- |
| **信号来源**      | Prefilter → Gate(统计规则) → Evidence(Spearman) | 模型预测值 > 阈值 → 开仓           |
| **模型角色**      | 辅助（只提供 SHAP∩Gain 特征重要性）             | 核心（预测值直接决定开仓）         |
| **可解释性**      | 每条规则可读、可审计                            | 黑盒                               |
| **IS→OOS 衰减**   | 低（统计规则 + robustness 验证）                | 高（模型拟合训练集分布）           |
| **理论天花板**    | 低 — 只能捕获线性/单调信号                      | 高 — 能捕获非线性交互              |
| **实际 OOS 表现** | 更稳定                                          | 经常 IS 很好，OOS 崩               |

---

## 三个核心原因

### 1. 数据量不够支撑端到端模型

BPC 一个 holdout 只有 ~300 行 gate-passed 数据。LightGBM 5-fold CV 每 fold 训练集约 250 行。
这个量级下模型学的是噪声，不是 alpha。端到端模型需要数万行才有意义，
当前 6 币种 × 2 年 × 240T 时间框架凑不出足够样本。

### 2. 非平稳性杀死端到端模型的泛化

Crypto 市场微结构每季度大幅变化（流动性、波动率 regime、杠杆水平）。
端到端模型拟合的是 IS 期间的特征-收益映射，这个映射在 OOS 大概率失效。
统计规则方案只提取"环境特征与收益的单调关系"（Spearman + quintile），
对分布漂移的敏感度低得多。

### 3. 模型已在最合适的位置发挥作用

LightGBM 在管线里做的是 SHAP∩Gain 特征选择 — 找出哪些特征重要。
然后统计方法接手做阈值优化和规则生成。
这比让模型直接出预测值更稳健，因为：

> **特征重要性排序比预测值本身更 robust to distribution shift。**

---

## 当前方案的局限

统计规则只能捕获"A 特征高→收益好"这种单调关系，
Gate 的 2D compound rule 最多捕获二维交互。
如果 alpha 藏在 5+ 个特征的非线性交互里，当前方案捕获不到。

**但这个天花板在当前数据量下是不可达的。**

---

## 什么时候该切换到端到端模型

满足以下条件时可考虑引入端到端模型：

1. **数据量**: gate-passed trades 达到 10,000+ 行（加更多币种 / 缩短时间框架）
2. **泛化验证**: walk-forward 验证证明模型 OOS Sharpe 衰减 < 30%
3. **特征稳定**: 特征工程成熟到不需要频繁改动

在此之前，统计规则管线是更安全的选择。
