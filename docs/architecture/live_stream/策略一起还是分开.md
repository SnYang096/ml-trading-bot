## 🎯 直接结论（不绕）

> **6 个 archetype 必须共用一个 Nautilus Strategy（一个 adapter），
> 绝对不要拆成 6 个独立 strategy_name 去跑。**

**但：**
这 **不等于** 把逻辑写死在一个类里。

正确形态是：

> **一个 Strategy 进程

* 一个 Router
* 多个 Archetype 模块（纯执行）
* 强隔离的统计与风控**

## 一、为什么“拆成 6 个 strategy_name”是错的（非常重要）

如果你做成：

```bash
nautilus live --strategy BreakoutPullback
nautilus live --strategy FailedBreakoutFade
...
```

你会**立刻**踩 5 个结构性坑。

---

### ❌ 1️⃣ Router 被“假拆分”掉了

你现在最重要的资产是：

> **Router = 世界观裁决器**

如果你拆成 6 个 strategy：

* 每个 strategy **隐含假设自己一定是对的**
* Router 不存在或退化为 config

这在研究期看不出来，**实盘必爆**。

---

### ❌ 2️⃣ 多 strategy = 隐性对赌

Nautilus 的多个 strategy：

* 各自计算 position
* 各自下单
* 各自统计 PnL

你会得到这种灾难结构：

```text
TREND 认为该追
MEAN 认为该反
→ 账户在同一时刻自对冲
→ 表面低波动，实则慢性流血
```

**这是量化版精神分裂。**

---

### ❌ 3️⃣ 风控无法“总量级一致”

你的风控本质是：

> **“不让我死”，不是“单策略好看”**

拆 strategy 之后：

* max drawdown
* leverage cap
* risk per day

全部是 **局部最优**，不是账户级最优。

---

### ❌ 4️⃣ 回测结果会严重误导你

拆策略回测，得到的其实是：

> “如果我永远只做这一种世界观，会怎样？”

但现实是：

> **你每天都在选择世界观**

拆回测会**系统性高估**稳定性。

---

### ❌ 5️⃣ 未来扩展直接崩盘

等你以后加：

* Vol Mean
* Options overlay
* Cross-asset netting

拆 strategy 会让你 **全部推倒重来**。

---

## 二、那为什么“一个 Strategy + 多 archetype”是对的？

因为它**忠实还原了你的真实交易过程**：

```
同一时间
同一账户
同一市场
→ 只能相信一种世界观
→ 只能采取一种行为方式
```

这是 **交易的物理现实**。

---

## 三、推荐的 Nautilus 架构（你可以直接照这个搭）

### 1️⃣ Nautilus 层：**只有一个 Strategy**

```python
class MetaRouterStrategy(Strategy):
    def on_bar(self, bar):
        features = self.feature_engine.update(bar)
        
        state = self.state_heads(features)        # mfe / mae / vol / time
        regime = self.router.decide(features, state)  # TREND / MEAN / NO
        
        if regime == NO:
            self.execution_manager.flat_or_hold()
            return

        archetype = self.execution_manager.select(regime, state)
        orders = archetype.generate_orders(state, features)
        self.submit(orders)
```

**关键点：**

* Nautilus 只认识一个 Strategy
* 所有世界观冲突在**内部解决**

---

### 2️⃣ Archetype = 纯执行模块（无市场信仰）

```python
class FailedBreakoutFade(ExecutionArchetype):
    def is_applicable(self, state, features):
        return (
            state.structure_failed
            and state.absorption
            and state.vol != SHOCK
        )

    def generate_orders(self, state, features):
        return OrderPlan(...)
```

它 **不知道自己是 MEAN 还是 TREND**
它只知道：**“现在我这种行为合不合法”**

---

### 3️⃣ Router 是唯一的“裁决者”

```python
class Router:
    def decide(self, features, state):
        if self.no_trade(features, state):
            return NO
        
        if self.trend_world(features, state):
            return TREND
        
        if self.mean_world(features, state):
            return MEAN
```

> **任何 archetype 不得 override Router**

这是你要写进“系统宪法”的一句话。

---

### 4️⃣ Execution Manager = 隔离与统计中枢

```python
class ExecutionManager:
    def select(self, regime, state):
        candidates = self.archetypes[regime]
        return best_candidate(candidates, state)

    def update_stats(self, archetype_id, pnl, mfe, mae):
        self.stats[archetype_id].update(...)
```

**这里你可以做到：**

* 每个 archetype 独立统计
* 独立启停
* 独立降权

但 **不独立下单**。

## 四、那怎么“测试”6 个 archetype？（你最关心的）

### ✅ 正确测试方式（非常重要）

#### 1️⃣ 统一 Strategy

```bash
nautilus backtest --strategy MetaRouterStrategy
```

#### 2️⃣ 用 config / flag 控制启用集合

```yaml
enabled_archetypes:
  TREND:
    - BreakoutPullbackContinuation
    - MomentumExpansion
  MEAN:
    - FailedBreakoutFade
    - LiquiditySweepRejection
    - AuctionExhaustionReversal
```

#### 3️⃣ 做 **ablation test**

* 关掉某一个 archetype
* 看系统整体变化
* 而不是单独跑它

---

### ❌ 错误测试方式

```bash
# 不要这么干
nautilus backtest --strategy FailedBreakoutFade
```

这只能用于 **unit test，不是系统验证**。

## 五、什么时候才允许“单 archetype strategy”？

* 不做风险判断
