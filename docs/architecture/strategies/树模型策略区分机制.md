# 树模型策略区分机制

本文档详细说明树模型如何区分不同策略，澄清"树规则是直接塞入特征，只是标签和训练目标不一样"的误解。

## 核心结论

树模型通过**标签定义 + 特征筛选**双重机制区分不同策略，**不是**简单地把相同特征塞进去。即使某些特征名称相同，在不同策略中：
- 特征的重要性不同
- 特征的阈值不同  
- 特征的组合方式不同

---

## 一、标签层面的区分（核心机制）

每个策略有**独特的标签生成逻辑**，这是区分策略的最核心机制。

### 1.1 sr_reversal_rr_reg_long（SR反转）

- **标签类型**：连续R/R标签（回归任务）
- **标签函数**：`compute_sr_reversal_rr_continuous_label_with_weights`
- **优化目标**：最大化R/R比率（Risk/Reward）
- **标签范围**：连续值（0到max_rr，通常max_rr=3.0）
- **标签定义**：
  ```python
  R_R = min(max(MFE / MAE, 0), max_rr)
  # MFE: 最大有利偏移
  # MAE: 最大不利偏移
  ```
- **特点**：关注SR区域的反转机会，优化高R/R成功率

### 1.2 sr_breakout（SR突破）

- **标签类型**：连续R/R标签（回归任务）
- **标签函数**：`compute_sr_breakout_label`
- **优化目标**：突破质量（MFE/MAE）
- **标签范围**：连续值（0到max_rr）
- **标签定义**：与sr_reversal类似，但关注的是"突破"而非"反转"
- **特点**：区分"真突破跑远" vs "假突破回抽"

### 1.3 compression_breakout（压缩区突破）

- **标签类型**：三元标签（多分类任务）
- **标签函数**：`compute_compression_breakout_label`
- **优化目标**：方向+质量（-1/0/+1）
- **标签范围**：离散值{-1, 0, 1}
- **标签定义**：
  ```python
  label = +1  # 向上有效突破（收盘站稳 + Vol > 20日均值）
  label = -1  # 向下有效突破
  label = 0   # 假突破/回补/震荡
  ```
- **特点**：同时捕捉方向与突破质量，0是主动拒绝信号

### 1.4 trend_following（趋势跟踪）

- **标签类型**：百分位标签（回归任务）
- **标签函数**：`compute_trend_following_label`
- **优化目标**：趋势相对强度（rank）
- **标签范围**：连续值[0, 1]
- **标签定义**：
  ```python
  label = rank(future_return_50, window=200)  # 0.0 到 1.0
  ```
- **特点**：对数据分布不敏感，适应不同波动率，跨品种/跨周期标准化

### 1.5 标签差异的影响

**关键点**：虽然特征可能重叠，但**不同的标签定义和训练目标**使得模型学习到不同的模式。

例如：
- `sr_reversal`和`sr_breakout`都使用连续R/R标签，但：
  - `sr_reversal`关注"反转"场景，模型会学习"订单流背离"等模式
  - `sr_breakout`关注"突破"场景，模型会学习"放量确认"等模式
- `compression_breakout`使用三元标签，模型必须同时判断方向和真假
- `trend_following`使用百分位标签，模型学习的是相对强度而非绝对值

## 二、特征层的人为筛选（重要补充）

树模型**不是**简单地把所有特征塞进去，而是有**三层筛选机制**：

### 2.1 Pool A（基础必需特征）

- **位置**：`config/strategies/<strategy>/features_base.yaml`
- **作用**：标签生成和回测必需，**不参与搜索优化**
- **示例**：
  - `atr_f`：ATR（所有策略都需要，用于标签生成和回测）
  - `poc_hal_features_close_f`：SR反转策略需要，输出`dist_to_nearest_sr`（标签生成必需）
- **特点**：
  - 每个策略的基础特征可能不同
  - 这些特征是"基础设施"，不是alpha来源
  - 始终包含，不参与特征搜索

### 2.2 Pool B（数据驱动候选池）

- **生成方式**：`mlbot analyze factor-eval`
- **筛选标准**：IC/IR-based filtering + correlation removal
- **输出位置**：`results/pools/<strategy>/pool_b/<tag>/features_pool_b.yaml`
- **特点**：
  - **策略特定**：每个策略的Pool B不同
  - 通过IC/IR筛选，只保留与**该策略标签**相关的特征
  - 数据驱动，客观评估
  - 包含数学特征、TA-Lib特征、数值特征等

**重要**：Pool B是**策略特定的**，因为：
- 不同策略的标签不同
- IC/IR计算基于不同标签
- 因此筛选出的特征集合不同

### 2.3 Semantic Groups（人工维护语义特征）

- **位置**：
  - 优先：`config/feature_groups_<strategy_dir>_semantic.yaml`
  - 备选：`config/feature_groups.yaml`
- **内容**：人类可理解的语义因子
  - compression（压缩）
  - ignition（点火）
  - absorption（吸收）
  - exhaustion（衰竭）
- **特点**：
  - **策略特定**：不同策略关注不同的语义场景
  - 需要人工维护和设计
  - 从Pool B深度加工而来
  - 经过验证，已知对特定策略有效

**示例**：
- `sr_reversal`可能关注`vpin_exhaustion_scene_score`（衰竭信号）
- `compression_breakout`可能关注`vpin_compression_score`（压缩信号）
- `trend_following`可能关注`funding_ignition_score`（点火信号）

### 2.4 特征组合搜索

- **工具**：`mlbot diagnose feature-group-search`
- **算法**：greedy/halving/beam/sffs/pipeline
- **输入**：Pool A + Pool B + Semantic groups
- **输出**：`features_suggested*.yaml`
- **过程**：
  1. 加载Pool A（基础特征，必须包含）
  2. 加载Pool B（数据驱动候选，策略特定）
  3. 加载Semantic groups（人工维护，策略特定）
  4. 在合并后的候选空间中搜索最佳组合
  5. 输出最终的特征配置

### 2.5 特征筛选的层次结构

```
特征全集 (features_all.yaml)
    ↓
[factor-eval筛选] → Pool B (策略特定)
    ↓
[人工维护] → Semantic Groups (策略特定)
    ↓
[特征搜索] → features_suggested*.yaml (策略特定)
    ↓
[训练] → 树模型 (策略特定)
```

**结论**：树模型通过**标签+特征筛选**双重机制区分策略，不是简单"塞特征"。

## 三、模型层面的区分

虽然不同策略可能使用不同的模型类型和参数，但**核心区分机制是标签定义**，而非模型选择。

### 3.1 模型类型选择

- **sr_reversal**：通常使用XGBoost（二元或回归）
- **sr_breakout**：通常使用XGBoost（回归）
- **compression_breakout**：通常使用CatBoost（多分类）
- **trend_following**：通常使用LightGBM（回归）

### 3.2 训练参数差异

不同策略的训练参数可能不同：
- 学习率
- 树深度
- 正则化参数
- 样本权重策略

但这些差异是**次要的**，核心区分仍然是标签定义。

### 3.3 模型层面的影响

即使使用相同的模型类型和参数：
- 不同的标签 → 不同的损失函数 → 不同的优化方向
- 不同的特征池 → 不同的输入空间 → 不同的学习模式
- 因此，模型会学习到**策略特定的模式**

## 四、规则导出的策略特异性

导出的规则具有**策略特异性**，这是理解规则映射的关键。

### 4.1 同一特征在不同策略中的不同含义

**示例**：`cvd_long <= -34895`

- 在`trend_following`中：
  - 这是**负规则**（coef < 0）
  - 语义：长期订单流负 → 趋势不健康
  - 用途：TC Gate（否决）
  
- 在`sr_reversal`中：
  - 这可能是**正规则**（coef > 0）
  - 语义：长期订单流负 → 反转机会
  - 用途：FR信号（允许）

### 4.2 规则组合的策略特异性

不同策略的规则组合方式不同：

- **sr_reversal**：关注"订单流背离 + 结构确认"
  - 典型规则：`cvd_long <= -34895 AND vp_poc_deviation <= -0.025`
  
- **sr_breakout**：关注"放量 + 订单流支持"
  - 典型规则：`cvd_change_5 > 4805 AND volume_ratio > 1.87`
  
- **compression_breakout**：关注"流动性陷阱 + 假突破"
  - 典型规则：`liquidity_void_retracement <= 0.038 AND cvd_change_5 <= -848`
  
- **trend_following**：关注"趋势健康度 + 衰竭信号"
  - 典型规则：`bb_width_normalized <= 4.86 AND cvd_long <= -34895`（负规则）

### 4.3 映射规则到Execution时的注意事项

因此，映射规则到Execution时，必须：

1. **考虑规则的来源策略**：同一特征在不同策略中可能有相反的含义
2. **理解规则的语义上下文**：不能只看特征名称，要看规则在策略中的角色
3. **不能简单照搬阈值**：阈值是历史拟合的，需要泛化为分位数或相对位置

## 五、总结

### 5.1 树模型区分策略的三层机制

1. **标签定义**（核心）：不同的标签生成逻辑 → 不同的优化目标
2. **特征筛选**（重要）：策略特定的Pool B和Semantic groups → 不同的输入空间
3. **模型选择**（次要）：不同的模型类型和参数 → 不同的学习方式

### 5.2 关键澄清

**误解**：树规则是直接塞入特征，只是标签和训练目标不一样，这样就能区分不同规则了吗？

**澄清**：
- ✅ 标签和训练目标不同 → **这是核心区分机制**
- ✅ 特征层做了人为筛选 → **Pool B和Semantic groups都是策略特定的**
- ✅ 即使特征名称相同，在不同策略中的含义和重要性也不同

**结论**：树模型通过**标签+特征筛选**双重机制区分策略，不是简单"塞特征"。

### 5.3 规则导出的对应关系

树策略导出的规则可以对应到TC/TE/FR/ET，但需要注意：

- **sr_reversal** → **FR**（Failure Reversion）：检测假突破/订单流背离
- **sr_breakout** → **TE**（Trend Expansion）：验证突破质量/放量确认
- **compression_breakout** → **TE**（Trend Expansion）：压缩突破确认/流动性陷阱
- **trend_following** → **TC/ET**（Trend Continuation/Exhaustion Turn）：
  - 正规则 → TC（趋势健康度验证）
  - 负规则/衰竭规则 → ET（趋势末端反转）
