# 策略架构说明

## 概述

本项目采用**分层架构**，区分**统一框架**和**策略特定实现**：

1. **统一框架**（基础层）：`EventDrivenStrategy` 和 `EventDrivenBacktest`
   - 提供事件驱动的基础架构
   - 支持特征计算、信号生成、交易执行
   - **适用于快速测试和原型验证**

2. **策略特定实现**（应用层）：`sr_reversal_backtest.py`、`sr_breakout_backtest.py` 等
   - 实现策略特定的逻辑（止损、止盈、仓位管理等）
   - 针对每个策略的独特需求
   - **适用于真实场景和生产环境**

## 架构层次

```
┌─────────────────────────────────────────┐
│  策略特定实现（应用层）                   │
│  - sr_reversal_backtest.py              │
│  - sr_breakout_backtest.py              │
│  - compression_breakout_backtest.py     │
│  - trend_following_backtest.py          │
│  - 策略特定的止损/止盈/仓位逻辑           │
└─────────────────────────────────────────┘
           ↓ 继承/使用
┌─────────────────────────────────────────┐
│  统一框架（基础层）                      │
│  - EventDrivenStrategy                 │
│  - EventDrivenBacktest                 │
│  - IncrementalFeatureComputer          │
│  - 通用的事件驱动架构                    │
└─────────────────────────────────────────┘
```

## 统一框架 vs 策略特定实现

### 统一框架 (`EventDrivenStrategy` / `EventDrivenBacktest`)

**用途**：
- ✅ 快速测试新策略
- ✅ 原型验证
- ✅ 特征工程测试
- ✅ 模型性能评估

**特点**：
- 通过 `strategy_name` 加载不同模型
- 通用的信号生成逻辑
- 简单的仓位管理（固定仓位）
- 基础的止损/止盈（可选）

**局限性**：
- ❌ 无法处理策略特定的止损/止盈逻辑
- ❌ 无法处理策略特定的仓位管理
- ❌ 无法处理策略特定的加仓/减仓逻辑
- ❌ 无法处理策略特定的风险控制

**示例**：
```python
# 统一框架：快速测试
strategy = EventDrivenStrategy(
    strategy_name="sr_reversal",  # 加载 sr_reversal 模型
    instrument_id=instrument_id,
    bar_types={"15T": bar_type_15m, "1H": bar_type_1h},
    trade_size=0.001,  # 固定仓位
    # 没有策略特定的止损/止盈逻辑
)
```

### 策略特定实现 (`sr_reversal_backtest.py` 等)

**用途**：
- ✅ 真实场景回测
- ✅ 生产环境实盘
- ✅ 策略优化
- ✅ 风险控制

**特点**：
- 策略特定的止损/止盈逻辑
- 策略特定的仓位管理（基于置信度、ATR 等）
- 策略特定的加仓/减仓逻辑
- 策略特定的风险控制

**示例**：
```python
# SR 反转策略特定回测
result = backtest_sr_reversal(
    df=df,
    predictions=predictions,
    sr_type_col="sr_type",
    stop_loss_r=1.0,      # 止损：1×ATR
    take_profit_r=2.0,     # 止盈：2×ATR
    max_holding_bars=50,   # 最大持仓周期
    min_confidence=0.3,    # 最小置信度
    base_position_size=0.1,  # 基础仓位
    enable_reduce_position=True,  # 允许减仓
    # ... 更多策略特定参数
)
```

## 策略差异对比

### SR 反转策略 (`sr_reversal_backtest.py`)

**特点**：
- 止损：入场价 ± 1×ATR（反向）
- 止盈：入场价 ± 2×ATR（同向）
- 仓位：`position_size ∝ P(success)`
- 加仓：❌ 通常不加仓
- 减仓：若价格未快速脱离，且 CVD 相位转负，可提前部分止盈

**参数**：
```python
stop_loss_r: float = 1.0      # 止损倍数
take_profit_r: float = 2.0     # 止盈倍数
max_holding_bars: int = 50     # 最大持仓周期
min_confidence: float = 0.3    # 最小置信度
base_position_size: float = 0.1  # 基础仓位
enable_reduce_position: bool = True  # 允许减仓
```

### SR 突破策略 (`sr_breakout_backtest.py`)

**特点**：
- 止损：突破点 ± 止损倍数×ATR
- 止盈：突破点 ± 止盈倍数×ATR
- 仓位：基于突破强度
- 加仓：✅ 支持加仓（突破确认后）
- 减仓：❌ 通常不减仓

**参数**：
```python
stop_loss_r: float = 1.5      # 止损倍数（通常更大）
take_profit_r: float = 3.0     # 止盈倍数（通常更大）
breakout_threshold: float = 0.7  # 突破阈值
add_position_size: float = 0.0  # 加仓规模
```

### 压缩突破策略 (`compression_breakout_backtest.py`)

**特点**：
- 止损：压缩区间边界
- 止盈：基于压缩幅度
- 仓位：基于压缩强度
- 加仓：✅ 支持加仓
- 减仓：✅ 支持减仓

**参数**：
```python
compression_threshold: float = 0.8  # 压缩阈值
breakout_confirmation: int = 3       # 突破确认周期
stop_loss_r: float = 1.0
take_profit_r: float = 2.5
```

### 趋势跟随策略 (`trend_following_backtest.py`)

**特点**：
- 止损：趋势线或移动平均线
- 止盈：基于趋势强度
- 仓位：基于趋势动量
- 加仓：✅ 支持加仓（趋势确认后）
- 减仓：✅ 支持减仓（趋势减弱时）

**参数**：
```python
trend_strength_threshold: float = 0.6  # 趋势强度阈值
momentum_window: int = 20              # 动量窗口
stop_loss_r: float = 1.2
take_profit_r: float = 3.0
```

## 使用建议

### 1. 开发阶段

**使用统一框架**：
```python
# 快速测试新策略
strategy = EventDrivenStrategy(
    strategy_name="new_strategy",
    model_path="models/new_strategy/model.pkl",
    # ... 基础配置
)
```

**优势**：
- 快速迭代
- 无需实现策略特定逻辑
- 专注于特征和模型

### 2. 优化阶段

**使用策略特定实现**：
```python
# 实现策略特定的回测
result = backtest_sr_reversal(
    df=df,
    predictions=predictions,
    # ... 策略特定参数
)
```

**优势**：
- 精确控制止损/止盈
- 实现策略特定的仓位管理
- 优化策略参数

### 3. 生产环境

**使用策略特定实现**：
```python
# 实盘策略（继承 EventDrivenStrategy 并添加策略特定逻辑）
class SRReversalStrategy(EventDrivenStrategy):

        pass

        # 策略特定的仓位管理
        pass
    
    def _manage_position(self, ...):
        # 策略特定的止损/止盈逻辑
        pass
```

## 总结

| 特性 | 统一框架 | 策略特定实现 |
|------|---------|-------------|
| **用途** | 快速测试、原型验证 | 真实场景、生产环境 |
| **止损/止盈** | 基础（可选） | 策略特定 |
| **仓位管理** | 固定仓位 | 策略特定（基于置信度、ATR 等） |
| **加仓/减仓** | ❌ 不支持 | ✅ 支持 |
| **风险控制** | 基础 | 策略特定 |
| **开发速度** | 快 | 慢 |
| **精确度** | 低 | 高 |

**建议**：
- **开发阶段**：使用统一框架快速迭代
- **优化阶段**：使用策略特定实现精确优化
- **生产环境**：使用策略特定实现确保准确性
