# 策略模块架构说明

## 概述

本系统实现了**四类交易机会的独立策略模块**，每个策略都有：
- ✅ 独立的目录结构
- ✅ 专属的特征工程
- ✅ 专属的标签设计
- ✅ 专属的模型选择

## 四类策略对比

| 策略类型 | 标签类型 | 标签定义 | 模型 | 任务类型 | 核心特征 |
|---------|---------|---------|------|---------|---------|
| **SR 反转** | 二元标签 | 是否 ≥2R 成功 | XGBoost | Binary | SR 强度 + 流动性验证 |
| **SR 突破** | 连续标签 | 实现 R/R (MFE/MAE) | XGBoost | Regression | 突破质量 + 动能持续性 |
| **压缩区突破** | 三元标签 | -1/0/+1 (方向+质量) | CatBoost | Multiclass | 压缩强度 + 突破触发 |
| **趋势跟踪** | 百分位标签 | Rank (0-1) | LightGBM | Regression | 趋势强度 + 资金验证 |

## 标签设计详解

### 1. SR 反转：二元标签（≥2R 成功率）

**标签定义：**
```python
label = 1.0  # 在 SR 区入场后，50根K线内先触达 +2R 止盈
label = 0.0  # 先触达 -1R 止损
label = NaN  # 未触发（超时）
```

**特点：**
- ✅ 直接优化策略核心：高 R/R 成功率
- ✅ 逻辑清晰，与实盘一致
- ✅ 适合高 R/R、固定风控的离散机会

### 2. SR 突破：连续标签（实现 R/R）

**标签定义：**
```python
R_R = min(max(MFE / MAE, 0), 3.0)
# MFE: 最大有利偏移
# MAE: 最大不利偏移
```

**特点：**
- ✅ 保留盈利质量信息
- ✅ 区分"真突破跑远" vs "假突破回抽"
- ✅ 适合盈利幅度差异大的场景

### 3. 压缩区突破：三元标签（方向+质量）

**标签定义：**
```python
label = +1  # 向上有效突破（收盘站稳 + Vol > 20日均值）
label = -1  # 向下有效突破
label = 0   # 假突破/回补/震荡
```

**特点：**
- ✅ 同时捕捉方向与突破质量
- ✅ 0 是主动拒绝信号，不是"不确定"
- ✅ 适合方向与真假必须同时判断的临界点

### 4. 趋势跟踪：百分位标签（Rank）

**标签定义：**
```python
label = rank(future_return_50, window=200)  # 0.0 到 1.0
```

**特点：**
- ✅ 对数据分布不敏感，适应不同波动率
- ✅ 跨品种/跨周期标准化
- ✅ 适合趋势跟踪、多品种标准化

## 特征工程工具

### WPT（小波包变换）

**用途：**
- 多尺度分解（price、volume、CVD）
- 趋势提取（最低频子带）
- 残差计算（去趋势后的波动）

**关键函数：**
- `wpt_decompose()`: WPT 分解
- `extract_wpt_features()`: 提取 WPT 特征

### Hilbert 变换

**用途：**
- 对去趋势后的信号做 Hilbert 变换
- 提取瞬时相位、包络、频率
- 计算相位差（CVD vs Price）以捕捉领先-滞后关系

**关键函数：**
- `hilbert_transform()`: Hilbert 变换
- `compute_phase_lead()`: 计算相位领先
- `extract_hilbert_features()`: 提取 Hilbert 特征

### Hurst 指数

**用途：**
- 计算全序列 Hurst 指数
- 滚动 Hurst（捕捉市场状态切换）
- WPT 子带上的 Hurst

**关键函数：**
- `compute_hurst_rs()`: R/S 方法
- `compute_hurst_dfa()`: DFA 方法（推荐）
- `extract_hurst_features()`: 提取 Hurst 特征

### 频谱分析

**用途：**
- 主频提取（周期长度）
- 频谱平坦度（压缩度）
- 高频能量占比（噪声强度）

**关键函数：**
- `compute_spectrum_features()`: 计算频谱特征
- `extract_spectrum_features()`: 提取频谱特征

`strategy_trainer.py` 支持：
- **模型类型**：XGBoost、CatBoost、LightGBM
- **任务类型**：回归、二分类、多分类

## 核心设计原则

### 1. 标签设计 = 策略 DNA

**每个标签都直接优化策略的真实目标：**
- SR 反转：是否值得冒 1R 博 2R
- SR 突破：预期盈利质量（R/R）
- 压缩区突破：突破方向 + 真假
- 趋势跟踪：趋势相对强度

### 2. 特征工程 = 多尺度市场状态 × 流动性验证 × 动态节奏感知

**特征分组：**
- **多尺度 SR 结构**（WPT 驱动）：WPT 低频 POC、中频 Swing、高频 ZigZag
- **价量动态**（CVD + Volume）：滚动 CVD、Take Buy Ratio、VPER
- **时频信号处理**：WPT 能量、Hurst、Hilbert 相位差、Spectrum 主频
- **传统技术指标**：ROC、ATR、Bollinger Band Width
- **Mamba 时序嵌入**：长程上下文（可选）

### 3. 模型选择 = 根据标签类型和策略特点

**推荐配置：**
- **SR 反转**：XGBoost Binary（二元标签，高 R/R）
- **SR 突破**：XGBoost Regression（连续标签，盈利质量）
- **压缩区突破**：CatBoost Multiclass（三元标签，样本不平衡）
- **趋势跟踪**：LightGBM Regression（百分位标签，跨品种标准化）

## 使用示例

### 1. 训练 SR 反转模型

```python
from src.strategies.sr_reversal.train import main
import sys

sys.argv = [
    "train.py",
    "--symbol", "BTCUSDT",
    "--horizon", "50",
    "--timeframe", "15T",
    "--feature-type", "baseline,default",
]

main()
```

### 2. 使用特征工程工具

```python
from src.strategies.utils.wpt_features import extract_wpt_features
from src.strategies.utils.hilbert_features import extract_hilbert_features

# WPT 特征
df = extract_wpt_features(
    df,
    price_col="close",
    volume_col="volume",
    cvd_col="cvd",
)

# Hilbert 特征
df = extract_hilbert_features(
    df,
    price_fluctuation_col="wpt_price_fluctuation",
    cvd_fluctuation_col="wpt_cvd_fluctuation",
)
```


## 总结

这套架构实现了：
- ✅ **逻辑解耦**：每个策略独立，无内部冲突
- ✅ **风险隔离**：SR 反转失效 ≠ 趋势模型失效
- ✅ **特征聚焦**：SR 模型专注结构，趋势模型专注动量
- ✅ **适应性强**：调度器自动切换"模式权重"
- ✅ **可解释性高**：每个信号都有明确来源

**核心思想：你不需要一个"万能模型"，而需要一个由多个专家组成的决策委员会。**
