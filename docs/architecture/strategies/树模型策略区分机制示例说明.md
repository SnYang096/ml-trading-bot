# 树模型策略区分机制 - 具体示例说明

本文档通过**具体例子**说明树模型如何通过标签定义和特征筛选区分策略，以及规则如何具有策略特异性。

---

## 示例1：同一特征在不同策略中的相反含义

### 特征：`cvd_long <= -34895`

这是最典型的例子，说明**同一特征在不同策略的规则中可能有相反的含义**。

#### 在 `trend_following` 策略中

**规则**：
```
coef=-0.0799752, support=0.144: 
bb_width_normalized <= 4.85935 
AND cvd_long <= -34895.86719 
AND cvd_long > -64485.69531
```

**语义解读**：
- **系数为负**（coef=-0.0799752）→ 这是**负规则**（降低预测值）
- **语义**：波动不放大（bb_width_normalized <= 4.86）+ 长期订单流持续负（cvd_long <= -34895）→ **趋势已经死了**
- **用途**：**TC Gate（否决）** - 当这个条件满足时，否决Trend Continuation

**为什么是负规则？**
- `trend_following`的标签是"趋势相对强度"（rank）
- 标签值越高，表示趋势越强
- 当订单流负且波动压缩时，趋势强度应该降低
- 因此这个规则是**负规则**，用于否决不健康的趋势

#### 在 `sr_reversal_rr_reg_long` 策略中

**规则**：
```
coef=0.102949, support=0.077: 
cvd_medium <= 8999.94189 
AND cvd_medium > -49748.96094 
AND vp_width_ratio > 0.56304
```

虽然这个规则没有直接包含`cvd_long <= -34895`，但`sr_reversal`策略中，**订单流负往往是正信号**。

**语义解读**：
- **系数为正**（coef=0.102949）→ 这是**正规则**（提高预测值）
- **语义**：订单流负 + 结构确认 → **反转机会**
- **用途**：**FR信号（允许）** - 当这个条件满足时，允许Failure Reversion

**为什么是正规则？**
- `sr_reversal`的标签是"R/R比率"（连续值）
- 标签值越高，表示R/R越好
- 当订单流负但价格可能上涨时，这是"背离"信号，可能带来反转机会
- 因此这个规则是**正规则**，用于识别反转机会

### 对比总结

| 策略 | 规则类型 | 系数符号 | 语义 | 用途 |
|------|---------|---------|------|------|
| **trend_following** | 负规则 | coef < 0 | 趋势不健康 | TC Gate（否决） |
| **sr_reversal** | 正规则 | coef > 0 | 反转机会 | FR信号（允许） |

**关键洞察**：
- 同一特征（`cvd_long <= -34895`）在不同策略中**含义相反**
- 这是因为**标签定义不同**：
  - `trend_following`：优化"趋势强度" → 订单流负是坏信号
  - `sr_reversal`：优化"R/R比率" → 订单流背离可能是好信号（反转机会）

## 示例2：不同策略的Pool B差异

### Pool B的生成过程

每个策略的Pool B是通过`mlbot analyze factor-eval`生成的，**基于该策略的标签**进行IC/IR筛选。

#### `sr_reversal_rr_reg_long` 的Pool B

**标签**：连续R/R标签（`rr_label`）
- 标签值 = min(max(MFE/MAE, 0), 3.0)
- 优化目标：最大化R/R比率

**Pool B筛选标准**：
- 计算每个特征与`rr_label`的IC（Information Coefficient）
- 计算IR（IC的均值/标准差）
- 只保留 |IR| >= 0.05 且 |t-stat| >= 1.96 的特征

**典型的Pool B特征**（示例）：
```yaml
# results/pools/sr_reversal_rr_reg_long/pool_b/features_pool_b.yaml
feature_pipeline:
  requested_features:
    - cvd_features_f          # 订单流特征（与R/R相关）
    - vp_poc_deviation        # POC偏离（结构确认）
    - vp_entropy              # Volume profile熵（结构集中度）
    - vpin_exhaustion_scene_score  # VPIN衰竭场景（反转信号）
    - trade_cluster_absorption_scene_score  # 交易聚类吸收场景
    # ... 其他与R/R相关的特征
```

**为什么这些特征被选中？**
- 这些特征与`rr_label`（R/R比率）有较高的IC
- 例如：`vpin_exhaustion_scene_score`高时，往往对应反转机会，R/R可能更好

#### `trend_following` 的Pool B

**标签**：百分位标签（`label`）
- 标签值 = rank(future_return_50, window=200)
- 优化目标：趋势相对强度（rank）

**Pool B筛选标准**：
- 计算每个特征与`label`（rank）的IC
- 只保留 |IR| >= 0.05 且 |t-stat| >= 1.96 的特征

**典型的Pool B特征**（示例）：
```yaml
# results/pools/trend_following/pool_b/features_pool_b.yaml
feature_pipeline:
  requested_features:
    - cvd_features_f          # 订单流特征（与趋势强度相关）
    - bb_width_normalized     # Bollinger Band宽度（波动率）
    - macd_signal             # MACD信号（趋势动量）
    - funding_ignition_score  # Funding点火场景（趋势加速）
    - funding_exhaustion_scene_score  # Funding衰竭场景（趋势末端）
    - wpt_cvd_fluctuation     # WPT订单流波动（趋势稳定性）
    # ... 其他与趋势强度相关的特征
```

**为什么这些特征被选中？**
- 这些特征与`label`（趋势相对强度rank）有较高的IC
- 例如：`funding_ignition_score`高时，往往对应趋势加速，rank可能更高

### Pool B差异的原因

| 策略 | 标签类型 | 标签含义 | Pool B特征类型 |
|------|---------|---------|---------------|
| **sr_reversal** | 连续R/R | MFE/MAE比率 | 反转信号、结构确认、订单流背离 |
| **trend_following** | 百分位rank | 趋势相对强度 | 趋势动量、波动率、资金费率场景 |

**关键洞察**：
- 不同策略的**标签不同** → IC/IR计算基于不同标签 → **筛选出的Pool B不同**
- 即使某些特征名称相同（如`cvd_features_f`），在不同策略的Pool B中：
  - **重要性不同**：在`sr_reversal`中可能IC=0.08，在`trend_following`中可能IC=0.05
  - **组合方式不同**：在`sr_reversal`中与`vp_poc_deviation`组合，在`trend_following`中与`bb_width_normalized`组合

## 示例3：不同策略的Semantic Groups差异

### Semantic Groups的定义

Semantic Groups是人工维护的语义特征组，位于：
- `config/feature_groups_<strategy_dir>_semantic.yaml`（策略特定）
- 或 `config/feature_groups.yaml`（通用）

### `sr_reversal_rr_reg_long` 的Semantic Groups

**关注场景**：反转、衰竭、吸收

**典型的Semantic Groups**（示例）：
```yaml
# config/feature_groups_sr_reversal_rr_reg_long_semantic.yaml
semantic__vpin_exhaustion:
  - vpin_exhaustion_scene_score    # VPIN衰竭场景（反转信号）
  
semantic__vpin_absorption:
  - vpin_absorption_scene_score    # VPIN吸收场景（反转确认）
  
semantic__trade_cluster_absorption:
  - trade_cluster_absorption_scene_score  # 交易聚类吸收（反转确认）
  
semantic__liquidity_void:
  - liquidity_void_retracement      # 流动性真空回撤（假突破检测）
```

**为什么关注这些场景？**
- `sr_reversal`关注"反转机会"
- 衰竭、吸收场景往往对应反转信号
- 流动性真空往往对应假突破，需要检测

### `trend_following` 的Semantic Groups

**关注场景**：点火、压缩、衰竭

**典型的Semantic Groups**（示例）：
```yaml
# config/feature_groups_trend_following_semantic.yaml
semantic__funding_ignition:
  - funding_ignition_score          # Funding点火场景（趋势加速）
  
semantic__funding_compression:
  - funding_compression_score       # Funding压缩场景（趋势等待）
  
semantic__funding_exhaustion:
  - funding_exhaustion_scene_score  # Funding衰竭场景（趋势末端）
  
semantic__funding_absorption:
  - funding_absorption_score        # Funding吸收场景（趋势过热）
```

**为什么关注这些场景？**
- `trend_following`关注"趋势延续"
- 点火场景对应趋势加速（正信号）
- 压缩场景对应趋势等待（中性）
- 衰竭场景对应趋势末端（负信号，用于ET）
- 吸收场景对应趋势过热（负信号，用于风控）

### Semantic Groups差异的原因

| 策略 | 关注场景 | 语义特征类型 | 用途 |
|------|---------|-------------|------|
| **sr_reversal** | 反转、衰竭、吸收 | VPIN场景、交易聚类场景 | 识别反转机会 |
| **trend_following** | 点火、压缩、衰竭 | Funding场景 | 识别趋势状态 |

**关键洞察**：
- 不同策略关注**不同的市场场景**
- Semantic Groups是**策略特定的**，由人工根据策略逻辑维护
- 即使使用相同的语义特征节点（如`vpin_scene_semantic_scores_f`），不同策略可能只关注其中的某些输出列

## 示例4：规则组合的策略特异性

### `sr_reversal` 的典型规则组合

**规则模式**：订单流背离 + 结构确认

**典型规则**：
```
coef=0.0943697, support=0.128: 
vp_poc_deviation <= -0.02497 
AND vp_entropy <= 0.96382 
AND cvd_normalized > -0.057
```

**语义解读**：
- `vp_poc_deviation <= -0.025`：POC偏离负（volume集中在某区域下方）
- `vp_entropy <= 0.96`：Volume profile熵低（volume集中）
- `cvd_normalized > -0.057`：订单流标准化值不太负
- **组合语义**：结构集中 + 订单流不太负 → **反转机会**

**为什么这样组合？**
- `sr_reversal`关注"反转机会"
- 需要同时检测"结构集中"（volume集中）和"订单流背离"（价格与订单流不一致）
- 因此规则组合了volume profile特征和订单流特征

### `sr_breakout` 的典型规则组合

**规则模式**：放量确认 + 订单流支持

**典型规则**：
```
coef=0.144323, support=0.030: 
cvd_change_5 > 4805.8501 
AND volume_ratio > 1.87706
```

**语义解读**：
- `cvd_change_5 > 4805`：短期订单流推进（5根K线的CVD变化）
- `volume_ratio > 1.87`：成交量比率高（放量）
- **组合语义**：放量 + 订单流支持 → **真实突破**

**为什么这样组合？**
- `sr_breakout`关注"突破质量"
- 需要同时验证"放量"（成交量支持）和"订单流支持"（真实买盘）
- 因此规则组合了成交量特征和订单流特征

### `compression_breakout` 的典型规则组合

**规则模式**：流动性陷阱 + 假突破检测

**典型规则**：
```
coef=-0.0761822, support=0.081: 
liquidity_void_retracement <= 0.03782 
AND cvd_change_5 <= -848.44751 
AND volume_ratio > 1.65399
```

**语义解读**：
- `liquidity_void_retracement <= 0.038`：流动性真空后回撤很浅
- `cvd_change_5 <= -848`：短期订单流下降
- `volume_ratio > 1.65`：成交量比率高
- **组合语义**：流动性真空 + 订单流不支持 + 放量 → **假突破/流动性陷阱**

**为什么这样组合？**
- `compression_breakout`关注"压缩区突破"
- 需要检测"假突破"（流动性陷阱）
- 因此规则组合了流动性特征、订单流特征和成交量特征

### `trend_following` 的典型规则组合

**规则模式**：趋势健康度 + 衰竭检测

**典型规则（TC Gate）**：
```
coef=-0.0799752, support=0.144: 
bb_width_normalized <= 4.85935 
AND cvd_long <= -34895.86719
```

**语义解读**：
- `bb_width_normalized <= 4.86`：波动率压缩（Bollinger Band宽度小）
- `cvd_long <= -34895`：长期订单流负
- **组合语义**：波动不放大 + 订单流负 → **趋势不健康**（TC Gate否决）

**典型规则（ET信号）**：
```
coef=0.0460151, support=0.526: 
funding_exhaustion_scene_score > 0.00204 
AND macd_signal <= 0.42512
```

**语义解读**：
- `funding_exhaustion_scene_score > 0.002`：Funding衰竭场景分数高
- `macd_signal <= 0.425`：MACD信号弱
- **组合语义**：资金费率衰竭 + 动量弱 → **趋势末端反转**（ET信号）

**为什么这样组合？**
- `trend_following`关注"趋势延续"
- 需要同时检测"趋势健康度"（用于TC Gate）和"趋势衰竭"（用于ET信号）
- 因此规则组合了波动率特征、订单流特征、资金费率场景特征和动量特征

### 规则组合差异总结

| 策略 | 规则模式 | 组合特征类型 | 用途 |
|------|---------|-------------|------|
| **sr_reversal** | 订单流背离 + 结构确认 | Volume profile + 订单流 | 识别反转机会 |
| **sr_breakout** | 放量确认 + 订单流支持 | 成交量 + 订单流 | 验证突破质量 |
| **compression_breakout** | 流动性陷阱 + 假突破 | 流动性 + 订单流 + 成交量 | 检测假突破 |
| **trend_following** | 趋势健康度 + 衰竭 | 波动率 + 订单流 + 资金费率场景 + 动量 | TC Gate / ET信号 |

**关键洞察**：
- 不同策略的规则组合方式不同
- 这是因为**标签定义不同** → **优化目标不同** → **关注的模式不同**
- 即使使用相同的特征，不同策略会以**不同的方式组合**它们

## 总结：三层机制如何协同工作

### 1. 标签定义（核心区分机制）

- **sr_reversal**：连续R/R标签 → 优化"反转机会的R/R"
- **sr_breakout**：连续R/R标签 → 优化"突破质量的R/R"
- **compression_breakout**：三元标签 → 优化"方向+质量"
- **trend_following**：百分位标签 → 优化"趋势相对强度"

### 2. 特征筛选（重要区分机制）

- **Pool B**：基于不同标签的IC/IR筛选 → 每个策略的Pool B不同
- **Semantic Groups**：人工维护，策略特定 → 不同策略关注不同场景

### 3. 规则导出（策略特异性）

- 同一特征在不同策略中可能有**相反的含义**
- 不同策略的规则组合方式不同
- 映射到Execution时必须考虑**规则的来源策略**

### 完整示例：`cvd_long <= -34895`的完整路径

```
特征: cvd_long <= -34895
    ↓
[标签定义差异]
    ↓
trend_following (标签: 趋势强度rank)
    → Pool B筛选: 与rank的IC计算
    → Semantic Groups: funding场景
    → 规则: 负规则 (coef < 0)
    → 语义: 趋势不健康
    → 用途: TC Gate (否决)
    ↓
sr_reversal (标签: R/R比率)
    → Pool B筛选: 与R/R的IC计算
    → Semantic Groups: vpin场景
    → 规则: 正规则 (coef > 0)
    → 语义: 反转机会
    → 用途: FR信号 (允许)
```

**关键洞察**：
- 同一特征，通过**不同的标签定义**和**不同的特征筛选**，最终导出**不同含义的规则**
- 这就是为什么树模型能够区分不同策略，而不是简单"塞特征"

## 实际应用建议

### 1. 理解规则时必须看来源策略

当看到一个规则时，必须知道它来自哪个策略：
- 来自`trend_following`的负规则 → 可能是TC Gate（否决）
- 来自`sr_reversal`的正规则 → 可能是FR信号（允许）

### 2. 映射规则到Execution时必须考虑语义上下文

不能只看特征名称，要看：
- 规则的系数符号（正规则/负规则）
- 规则的来源策略
- 规则在策略中的角色

### 3. 不能简单照搬阈值

导出的阈值是历史拟合的，需要：
- 泛化为分位数（如`quantile(cvd_long, 0.2)`）
- 或使用相对位置（如`is_in_low_quantile(cvd_long, 0.2)`）

详见：[树策略导出的可泛化规则](树策略导出的可泛化规则.md)
