# VPIN 多维特征测试说明

## 概述

本文档说明如何测试新添加的 VPIN 多维聚合特征，验证其计算正确性、有用性和无未来信息泄露。

## 测试文件

测试文件位于：`tests/test_vpin_multi_dimensional_features.py`

## 测试内容

测试包含 7 个测试用例，全面验证多维特征的各个方面：

### 1. 特征存在性测试 (`test_multidimensional_features_exist`)

**验证内容：**
- 所有新增的多维特征是否存在
- 原有特征是否保留（向后兼容性）

**新增特征列表：**
- `vpin_max` - 峰值（关键！）
- `vpin_last` - 最新值
- `vpin_min` - 最小值
- `vpin_std` - 标准差
- `vpin_count` - 事件数
- `vpin_signed_imbalance_last` - Signed imbalance 最新值
- `vpin_signed_imbalance_max` - Signed imbalance 峰值

### 2. 计算正确性测试 (`test_calculation_correctness`)

**验证内容：**
- `vpin_max >= vpin_mean`（峰值不小于均值）
- `vpin_min <= vpin_mean`（最小值不大于均值）
- `vpin_max >= vpin_min`（峰值不小于最小值）
- `vpin_count > 0`（事件数大于0）
- `vpin_std >= 0`（标准差非负）
- 峰值位置的 `vpin_max` 确实是最高的

### 3. 峰值信号保留测试 (`test_peak_signal_preservation`)

**验证内容：**
- 峰值信号没有被均值稀释
- `vpin_max` 明显大于 `vpin_mean`
- 峰值位置的 `vpin_max` 明显高于平均值

**关键验证：**
- 如果只使用均值，峰值信号可能被稀释
- 使用 `vpin_max` 应该保留峰值信号
- 峰值应该明显高于平均值（至少 0.9 倍）

### 4. 事件数区分测试 (`test_event_count_distinction`)

**验证内容：**
- `vpin_count` 能帮助区分突发事件和持续活跃
- 高峰值 + 低事件数 = 突发事件
- 高峰值 + 高事件数 = 持续活跃

**有用性验证：**
- 验证 `vpin_count` 提供了额外的信息维度
- 帮助理解信号的类型（突发事件 vs 持续活跃）

### 5. 无未来信息泄露测试 (`test_no_future_information_leak`)

**验证内容：**
- 修改未来数据不应该影响当前时刻的特征
- 修改过去数据应该影响当前时刻的特征
- 时间对齐的正确性

**测试方法：**
1. 修改 t=15 之后的 tick 数据（未来数据）
2. 计算特征
3. 恢复原始数据
4. 修改 t=15 之前的 tick 数据（过去数据）
5. 计算特征
6. 验证：两次计算中 t=15 的特征值应该不同（因为修改了过去数据）
7. 验证：修改未来数据不应该影响当前特征

### 6. 统计一致性测试 (`test_statistical_consistency`)

**验证内容：**
- 所有 K 线位置的统计关系一致
- `vpin_max >= vpin_min`
- `vpin_mean >= vpin_min`
- `vpin_max >= vpin_mean`
- `vpin_std >= 0`
- `vpin_count >= 0`

### 7. 特征值域合理性测试 (`test_feature_value_ranges`)

**验证内容：**
- `vpin`, `vpin_max`, `vpin_min`, `vpin_last` 在 [0, 1] 范围内
- `vpin_std >= 0`
- `vpin_count >= 0`

## 测试要点说明

### 1. 峰值信号保留

这是**最关键的测试**，验证：
- 均值可能稀释峰值信号（如 0.8 被稀释到 0.3）
- `vpin_max` 应该保留峰值信号（如 0.8）
- 峰值应该明显高于平均值

### 2. 无未来信息泄露

这是**最重要的安全性测试**，验证：
- 时间对齐的正确性
- 不使用未来数据计算当前特征
- 符合因果性要求

### 3. 计算正确性

验证统计量的数学正确性：
- 最大值 >= 均值 >= 最小值
- 标准差 >= 0
- 事件数 >= 0

### 4. 特征有用性

验证特征提供了有意义的信息：
- `vpin_count` 能区分突发事件和持续活跃
- `vpin_max` 能捕捉峰值信号
- 多维特征组合能提供更丰富的信息

## 故障排查

### 如果测试失败

1. **检查特征是否存在**
   - 确认 `vpin_features` 已正确计算
   - 检查 `config/feature_dependencies.yaml` 中的 `output_columns`

2. **检查计算逻辑**
   - 查看 `src/features/time_series/utils_order_flow_features.py`
   - 确认多维统计特征的计算逻辑正确

3. **检查时间对齐**
   - 确认时间对齐逻辑正确（右对齐）
   - 验证没有使用未来数据

### 常见问题

**Q: 测试显示峰值没有被保留？**
A: 检查 `vpin_max` 的计算逻辑，确保使用 `max()` 而不是 `mean()`

**Q: 测试显示有未来信息泄露？**
A: 检查时间对齐逻辑，确保使用右对齐（`kline_start <= event_time < kline_end`）

**Q: 测试显示特征值域不合理？**
A: 检查特征值的计算，确保在合理范围内
