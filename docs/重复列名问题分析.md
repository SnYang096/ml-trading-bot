# 重复列名问题分析

## 问题现象

在特征计算过程中，出现了重复列名（如 `'poc', 'poc', 'poc', 'poc', 'poc'`），导致：
1. `ValueError: cannot reindex on an axis with duplicate labels`
2. `ValueError: Cannot set a DataFrame with multiple columns to the single column`

## 根本原因

### 1. 多个特征输出相同的列名

在 `config/feature_dependencies.yaml` 中，有两个特征都输出了 `poc` 列：

```yaml
sqs_hal_high:
  output_columns: ["sqs_hal_high", "hal_high", "poc"]  # 输出 poc 列

sqs_hal_low:
  output_columns: ["sqs_hal_low", "hal_low", "poc"]    # 也输出 poc 列
```

### 2. 特征计算逻辑

两个特征都调用了 `add_poc_hal_dimensionless_features` 函数：

- `compute_sqs_hal_high()` → 调用 `add_poc_hal_dimensionless_features()` → 创建 `poc` 列
- `compute_sqs_hal_low()` → 调用 `add_poc_hal_dimensionless_features()` → 也创建 `poc` 列

### 3. 特征合并时的冲突

当这两个特征都被计算时：

1. **第一次合并**（`sqs_hal_high`）：
   ```python
   result_df = pd.concat([result_df, sqs_hal_high_result], axis=1)
   # result_df 现在包含: ['sqs_hal_high', 'hal_high', 'poc', ...]
   ```

2. **第二次合并**（`sqs_hal_low`）：
   ```python
   result_df = pd.concat([result_df, sqs_hal_low_result], axis=1)
   # 如果 sqs_hal_low_result 也包含 'poc' 列，就会产生重复
   # result_df 现在包含: [..., 'poc', 'sqs_hal_low', 'hal_low', 'poc', ...]
   ```

### 4. 按月计算时的额外问题

在按月计算时：
- 每个月份的数据可能都包含了这两个特征的结果
- 当合并月份数据时，如果某些月份有重复的 `poc` 列，`reindex` 操作会失败

## 解决方案

### 已实施的修复

1. **在数据加载和计算后立即处理重复列名**：
   - 在 `_compute_single_feature_worker_monthly` 中，从缓存加载或计算后立即去重

2. **在合并前处理重复列名**：
   - 在所有合并位置（按月合并、特征合并）都添加了重复列名检查和处理

3. **在函数入口处理重复列名**：
   - 在 `enhance_wpt_vol_features` 函数开始时处理重复列名

### 建议的长期解决方案

1. **特征设计层面**：
   - 考虑将 `poc` 作为独立的基础特征，而不是多个特征的副产品
   - 或者使用不同的列名（如 `poc_hal_high`, `poc_hal_low`）

2. **合并逻辑优化**：
   - 在合并特征时，如果列已存在，使用 `update` 或 `combine_first` 而不是 `concat`
   - 或者在合并前检查列是否已存在，只添加新列

3. **配置层面**：
   - 在 `feature_dependencies.yaml` 中明确标记共享列（如 `poc`），避免重复计算
