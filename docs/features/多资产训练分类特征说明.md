# 多资产训练分类特征说明

## 问题

当特征支持多资产训练（通过`_symbol`分组）时，模型是否也需要将`_symbol`作为分类特征（categorical feature）？

**答案：是的！** 这是多资产统一建模的最佳实践。

## 为什么需要分类特征？

### 方案对比

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **方案A**：不将`_symbol`作为特征 | 只使用归一化后的特征，模型学习通用模式 | 可以泛化到新资产 | 无法学习资产特定的行为差异 |
| **方案B**：将`_symbol`作为分类特征 ✅ | 将`_symbol`作为分类特征输入模型 | 可以学习通用模式 + 资产特定模式 | 对新资产泛化能力可能略差 |

### 推荐方案：方案B（分类特征）

**原因**：
1. **学习资产差异**：不同资产（BTC、ETH、SOL）可能有不同的行为模式
2. **统一模型**：一个模型覆盖所有资产，而不是为每个资产单独训练
3. **LightGBM优势**：原生支持分类特征，能自动学习类别共性与个性

**示例**：
- 通用规则："压缩突破后上涨"适用于所有资产
- 资产特定规则："只有ETH对gas price敏感"

## 实现方式

### 1. 特征提取阶段

特征已经按`_symbol`分组计算，避免跨资产数据泄露：

```python
# 在特征提取时，按资产分组计算
for symbol in df["_symbol"].unique():
    df_asset = df[df["_symbol"] == symbol]
    # 计算特征（避免跨资产数据泄露）
    features = extract_features(df_asset)
```

### 2. 模型训练阶段

将`_symbol`作为分类特征传递给LightGBM：

```python
from src.features.utils_categorical_features import prepare_categorical_features_for_lightgbm

# 自动检测分类特征
categorical_feature_names, categorical_feature_indices = prepare_categorical_features_for_lightgbm(
    df=df_train,
    feature_names=feature_cols,
    known_categorical=["_symbol"],  # 已知的分类特征
)

# 训练模型时传递分类特征
trainer = LightGBMTrainer(model_type="classification")
trainer.train(
    X=df_train[feature_cols],
    y=df_train["label"],
    categorical_features=categorical_feature_names,  # 传递分类特征
)
```

### 3. LightGBM Dataset创建

在创建`lgb.Dataset`时指定分类特征：

```python
train_data = lgb.Dataset(
    X_train,
    label=y_train,
    categorical_feature=cat_feature_indices,  # 分类特征索引
    free_raw_data=False,
)
```

## 注意事项

### 1. 单资产 vs 多资产

- **单资产**：如果只有1个资产，`_symbol`是常数，会被自动排除
- **多资产**：如果有多个资产，`_symbol`会被自动检测为分类特征

### 2. 分类特征编码

LightGBM支持：
- **字符串类型**：直接使用（LightGBM内部自动hash）
- **整数类型**：如果已经是整数编码，也可以使用

**推荐**：直接使用字符串（如"BTCUSDT"），LightGBM会自动处理。

### 3. 参数调优

对于多资产训练，建议调整以下LightGBM参数：

```python
params = {
    "min_data_per_group": 30,  # 每个symbol至少30样本才split
    "cat_smooth": 10,  # 平滑稀有symbol
    "max_cat_to_onehot": 4,  # 类别数<=4时使用one-hot，否则使用hash
}
```

### 4. 验证效果

训练后检查特征重要性：

```python
# 如果_symbol排名靠前，说明资产差异显著
feature_importance = model.feature_importance(importance_type='gain')
print(f"_symbol importance: {feature_importance['_symbol']}")
```

## 总结

✅ **推荐做法**：
1. 特征提取时按`_symbol`分组（避免数据泄露）
2. 训练时将`_symbol`作为分类特征（学习资产差异）
3. 使用统一模型（一个模型覆盖所有资产）

这样既能利用多资产数据，又能捕捉资产特定模式，是专业量化团队常用的**multi-asset unified modeling**范式。
