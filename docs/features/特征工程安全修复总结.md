# 特征工程安全修复总结

## 修复的问题

### 1. 数据泄露问题修复

#### ✅ `utils_wpt_features.py`
**问题**：`extract_wpt_features()` 直接对整个序列做 WPT，有数据泄露
**修复**：
- 改为滚动窗口实现（`window=100`）
- 只使用历史数据 `[i-window, i)` 计算当前时刻 `i` 的特征
- 使用 `shift(1)` 确保时间对齐
- 添加 `return_reconstructed_price` 参数支持按需返回

#### ✅ `utils_hilbert_features.py`
**问题**：`extract_hilbert_features()` 直接对整个序列做 Hilbert 变换，有数据泄露
**修复**：
- 改为滚动窗口实现（`window=64`）
- 只使用历史数据计算当前时刻的特征
- 使用 `shift(1)` 确保时间对齐

#### ✅ `utils_spectrum_features.py`
**问题**：对 volume 和 cvd 使用全序列计算，有数据泄露
**修复**：
- 价格频谱：已使用滚动窗口（安全）
- 成交量频谱：改为滚动窗口
- CVD 频谱：改为滚动窗口
- 所有特征都使用 `shift(1)` 确保时间对齐

#### ✅ `utils_hurst_features.py`
**问题**：对 cvd 和 volume 使用全序列计算，有数据泄露
**修复**：
- 价格 Hurst：已使用滚动窗口（安全）
- CVD Hurst：改为滚动窗口
- Volume Hurst：改为滚动窗口
- 所有特征都使用 `shift(1)` 确保时间对齐

## 修复后的特征工程架构

### 核心原则
1. **滚动窗口**：所有特征都使用滚动窗口计算，只使用历史数据
2. **时间对齐**：使用 `shift(1)` 确保特征与标签时间对齐
3. **归一化**：所有特征都经过归一化处理（z-score、IQR 等）
4. **无数据泄露**：严格禁止使用未来信息

### 文件结构

```
src/features/time_series/
├── baseline_features.py          # 基线特征（SR、压缩等）
```

### 使用方式

1. **策略特征文件**：直接调用 `utils_*_features.py` 中的函数
2. **批量特征工程**：使用 `ComprehensiveFeatureEngineer`，它内部使用 `enhanced_features.py`
3. **特征加载器**：通过 `feature_function_mapping.py` 调用 `utils_*_features.py`

## 注意事项

1. **窗口大小**：
   - WPT: `window=100`（默认）
   - Hilbert: `window=64`（默认）
   - Spectrum: `rolling_window=64`（默认）
   - Hurst: `rolling_window=50`（默认）

2. **最小数据要求**：
   - WPT: 需要 `2^level` 个数据点（level=4 需要 16 个点）
   - Hilbert: 需要至少 10 个数据点
   - Spectrum: 需要至少 10 个数据点
   - Hurst: 需要至少 10 个数据点

3. **NaN 处理**：
   - 所有特征在 `shift(1)` 后，NaN 被填充为默认值（0.0 或 0.5）
   - NaN 表示历史数据不足，这是正常的
