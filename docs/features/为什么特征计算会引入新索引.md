# 为什么特征计算会引入新索引？

## 问题现象

在特征计算过程中，训练集和测试集可能引入新的索引，导致：
- 训练集引入了10个新索引（在2025-06-29和2025-06-30）
- 测试集引入了1076个新索引（在2025-04到2025-05之间）
- 这些新索引导致训练集和测试集完全重叠

## 根本原因

### 1. 按月计算时的索引合并问题

**场景**：特征按月份拆分计算，然后合并

```python
# 原始输入数据（训练集）
df_raw_train.index = [
    '2025-01-01 00:00:00',
    '2025-01-01 04:00:00',
    ...
    '2025-06-30 20:00:00'  # 最后一个索引
]

# 特征计算过程（按月拆分）
monthly_dfs = {
    '2025-01': df_jan,      # 1月的数据
    '2025-02': df_feb,      # 2月的数据
    ...
    '2025-06': df_jun,      # 6月的数据
}

# 问题：如果某个特征函数返回的DataFrame包含额外的索引
# 例如：滚动窗口特征可能在边界处引入新索引
feature_result_jan = compute_some_feature(df_jan)
# feature_result_jan.index 可能包含：
# ['2025-01-01 00:00:00', ..., '2025-01-31 20:00:00', '2025-02-01 00:00:00']  # 多了一个！

# 合并时使用 pd.concat(axis=0)
combined_result = pd.concat([feature_result_jan, feature_result_feb, ...], axis=0).sort_index()
# 结果：combined_result 包含了 '2025-02-01 00:00:00'，这个索引不在原始 df_raw_train 中！
```

### 2. 特征函数返回额外索引的情况

**示例1：滚动窗口特征的边界处理**

```python
def compute_rolling_feature(df):
    # 使用滚动窗口计算特征
    result = df['close'].rolling(window=20).mean()
    # 问题：如果滚动窗口需要"未来"数据来填充边界，可能引入新索引
    # 或者某些实现可能使用 reindex 或 resample，引入额外索引
    return result.to_frame('feature')
```

**示例2：时间序列重采样**

```python
def compute_resampled_feature(df):
    # 如果特征函数内部使用了 resample
    resampled = df.resample('4H').mean()
    # resample 可能创建新的时间索引（即使原始数据中没有）
    return resampled
```

**示例3：特征函数使用 reindex**

```python
def compute_feature_with_reindex(df):
    # 某些特征函数可能使用 reindex 来对齐时间
    new_index = pd.date_range(start=df.index.min(), end=df.index.max() + pd.Timedelta(hours=4), freq='4H')
    result = df.reindex(new_index)  # 引入了新索引！
    return result
```

### 3. 合并操作保留所有索引

**关键代码位置**：`parallel_computer.py` 第312、318、320行

```python
# 按月计算后合并
combined_series = pd.concat([r[i] for r in monthly_results.values()], axis=0).sort_index()
# 或
result_df = pd.concat(monthly_results.values(), axis=0).sort_index()

# pd.concat(axis=0) 会保留所有索引，包括：
# - 原始输入数据的索引
# - 特征函数返回的额外索引
```

### 4. 特征合并到主DataFrame时

**关键代码位置**：`parallel_computer.py` 第902、1006行

```python
# 合并特征结果到主DataFrame
result_df = pd.concat([result_df, combined_result[new_cols]], axis=1)

# 如果 combined_result 包含新索引，pd.concat(axis=1) 会：
# - 保留 result_df 的所有索引
# - 保留 combined_result 的所有索引
# - 取并集（union），而不是交集（intersection）
```

## 具体示例

### 示例：训练集引入新索引

```python
# 原始训练集（df_raw_train）
原始索引范围: 2025-01-01 00:00:00 到 2025-06-30 20:00:00
原始样本数: 1086

# 特征计算后（df_train）
新索引范围: 2025-01-01 00:00:00 到 2025-06-30 20:00:00
新样本数: 1096  # 多了10个！

# 新增的索引（示例）
新增索引: [
    '2025-06-29 12:00:00',  # 不在原始数据中
    '2025-06-29 08:00:00',  # 不在原始数据中
    '2025-06-30 20:00:00',  # 这个可能在，但重复了
    '2025-06-30 08:00:00',  # 不在原始数据中
    '2025-06-29 16:00:00',  # 不在原始数据中
]
```

### 为什么会出现这种情况？

1. **滚动窗口的边界填充**：
   ```python
   # 某个特征使用滚动窗口
   rolling_mean = df['close'].rolling(window=20, min_periods=1).mean()
   # 如果特征函数使用了 forward fill 或 backward fill，可能引入边界索引
   ```

2. **时间对齐操作**：
   ```python
   # 某些特征需要对齐到特定时间点
   aligned_index = pd.date_range(start=..., end=..., freq='4H')
   result = df.reindex(aligned_index, method='ffill')  # 引入了新索引
   ```

3. **缓存数据包含额外索引**：
   ```python
   # 如果使用了按月缓存，缓存的数据可能包含相邻月份的数据
   cached_result = load_monthly_cache('2025-06')
   # cached_result 可能包含 '2025-07-01 00:00:00' 等索引
   ```

## 解决方案

### 1. 在特征计算后过滤索引（已实现）

在 `load_features_from_requested` 中：
```python
# 保存原始索引
original_indices = set(df.index)

# 特征计算
result_df = self.computer.compute_features_parallel(...)

# 过滤掉新索引
new_indices = set(result_df.index) - original_indices
if new_indices:
    result_df = result_df.loc[result_df.index.isin(original_indices)]
```

### 2. 在合并时使用索引交集（可选方案）

```python
# 合并时只保留原始索引
result_df = pd.concat([result_df, feature_df[new_cols]], axis=1)
result_df = result_df.loc[original_indices]  # 只保留原始索引
```

### 3. 确保特征函数不引入新索引（最佳实践）

特征函数应该：
- 只返回输入DataFrame的索引
- 不使用 `reindex` 引入新索引
- 不使用 `resample` 创建新时间点
- 滚动窗口特征应该保持原始索引

## 总结

特征计算引入新索引的主要原因：
1. **按月计算合并时**：某些月份的特征结果包含额外索引
2. **特征函数实现**：使用了 reindex、resample 等操作
3. **缓存数据**：缓存可能包含相邻时间范围的数据
4. **合并操作**：`pd.concat` 默认取索引的并集，而不是交集

**修复方法**：在特征计算后，过滤掉不在原始输入数据中的索引，确保只保留原始数据分割的完整性。
