# 订单流特征缓存优化

## 问题分析

从训练日志可以看到，VPIN 和 Trade Clustering 特征计算非常慢：

```
📊 Computing real VPIN from ticks (7 files)...
   -> [1/7] BTCUSDT_2024-12.parquet
   -> [2/7] BTCUSDT_2025-01.parquet
   ...
📊 Computing trade clustering from tick files (monthly batches)...
   ✅ Loaded 2024-12: 30968 ticks
   ✅ Computed 2024-12: 30968 features
   ✅ Loaded 2025-01: 55419150 ticks  # 非常慢！
   ✅ Computed 2025-01: 55419150 features
```

### 问题 1: VPIN 缓存未生效

**原因：**
- `extract_order_flow_features` 函数调用 `compute_vpin_from_cached_ticks` 时，**没有传递 `monthly_cache_dir` 参数**
- `compute_vpin_from_cached_ticks` 虽然有默认值 `"cache/features/monthly"`，但可能：
  1. 缓存目录不存在
  2. 缓存键不匹配（参数变化导致）
  3. 首次计算（缓存不存在）

**代码位置：**
- `src/features/time_series/utils_order_flow_features.py:429` - 调用时缺少 `monthly_cache_dir` 参数

### 问题 2: Trade Clustering 完全没有缓存

**原因：**
- `extract_trade_clustering_features` 函数中，按月处理时**完全没有缓存机制**
- 每次都要重新加载和计算所有月份的 tick 数据（可能上千万条记录）

**代码位置：**
- `src/features/time_series/utils_order_flow_features.py:1042-1119` - 按月循环计算，无缓存

## 解决方案

### 方案 1: 修复 VPIN 缓存传递

修改 `extract_order_flow_features` 函数，添加 `monthly_cache_dir` 参数并传递给 `compute_vpin_from_cached_ticks`：

```python
def extract_order_flow_features(
    df: pd.DataFrame,
    ticks: Optional[pd.DataFrame] = None,
    ticks_loader_json: Optional[str] = None,
    ...
    monthly_cache_dir: Optional[str] = "cache/features/monthly",  # 新增参数
) -> pd.DataFrame:
    ...
    elif ticks_loader_json:
        ...
        vpin_series = compute_vpin_from_cached_ticks(
            cache_files=tick_files,
            start_ts=loader_params["start_ts"],
            end_ts=loader_params["end_ts"],
            bucket_volume=vpin_bucket_volume,
            n_buckets=vpin_n_buckets,
            adaptive=vpin_adaptive,
            lookback_minutes=loader_params.get("lookback_minutes", 60),
            monthly_cache_dir=monthly_cache_dir,  # 传递缓存目录
        )
```

### 方案 2: 为 Trade Clustering 添加缓存

参考 VPIN 的缓存机制，为 Trade Clustering 添加按月缓存：

1. **在 `tick_loader.py` 中添加缓存函数**（类似 VPIN）：
   - `_get_monthly_trade_clustering_cache_key()`
   - `_load_monthly_trade_clustering_cache()`
   - `_save_monthly_trade_clustering_cache()`

2. **修改 `extract_trade_clustering_features` 函数**：
   - 添加 `monthly_cache_dir` 参数
   - 在按月循环中检查缓存，存在则加载，不存在则计算并保存

3. **修改 `extract_order_flow_features` 函数**：
   - 调用 `extract_trade_clustering_features` 时传递 `monthly_cache_dir`

### 方案 3: 通过 ParallelFeatureComputer 传递缓存目录

由于 `vpin_features` 是通过 `ParallelFeatureComputer` 计算的，需要确保 `monthly_cache_dir` 能正确传递：

1. `ParallelFeatureComputer` 已经有 `monthly_cache_dir` 支持
2. 需要在 `_build_call_args` 或 `_compute_single_feature_worker_monthly` 中，将 `monthly_cache_dir` 注入到 `compute_params` 中
3. 或者修改 `extract_order_flow_features` 函数签名，从 `compute_params` 中读取 `monthly_cache_dir`

## 注意事项

1. **缓存键设计**：需要考虑所有影响计算结果的参数（bucket_volume, window_size, 时间范围等）
2. **缓存失效**：如果 tick 数据更新，需要清除对应月份的缓存
3. **缓存大小**：Trade Clustering 缓存可能较大，需要监控磁盘空间
4. **跨月连续性**：Trade Clustering 需要维护跨月状态，缓存时需要保存状态信息
