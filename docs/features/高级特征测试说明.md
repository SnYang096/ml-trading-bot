## 公共代码抽取

### utils_normalization.py

创建了公共归一化模块 `src/features/time_series/utils_normalization.py`，包含：

1. **normalize_series(x)**: Z-score归一化
   - 输入：numpy array 或 pandas Series
   - 输出：归一化后的序列（均值=0，标准差=1）
   - 用途：将不同量级的序列归一化到相同尺度

2. **normalize_by_group(df, value_col, group_col, method)**: 按组归一化
   - 支持多资产数据（按`_symbol`分组）
   - 支持`zscore`和`minmax`两种方法
   - 确保不同价格水平的资产可以一起训练

### 使用示例

```python
from src.features.time_series.utils_normalization import normalize_series, normalize_by_group

# 单序列归一化
normalized = normalize_series(prices)

# 多资产按组归一化
normalized = normalize_by_group(df, value_col="close", group_col="_symbol", method="zscore")
```

## 归一化验证

### GARCH特征
- **天然归一化**：基于收益率计算，不同资产的值在相似范围内
- **验证**：检查`garch_persistence`和`garch_leverage_gamma`在不同资产间的分布相似性

### DTW特征
- **价格归一化**：内部使用Z-score归一化价格窗口和模板
- **验证**：不同价格水平的资产，DTW距离分布应该相似

### EVT特征
- **收益率归一化**：基于收益率计算，天然归一化
- **验证**：检查`evt_tail_shape`在不同资产间的分布相似性

## 性能要求

- **GARCH**: 200个样本 < 10秒
- **DTW**: 100个样本 < 5秒
- **EVT**: 200个样本 < 10秒

## 运行测试

```bash
# 在Docker中运行
make test-advanced-features

# 或直接运行pytest
pytest tests/test_advanced_features.py -v
```

## 注意事项

1. **依赖包**：
   - GARCH需要`arch`包
   - DTW需要`dtaidistance`包
   - EVT需要`scipy`包（通常已安装）

2. **数据要求**：
   - GARCH: 至少`window + 10`个样本
   - DTW: 至少`window`个样本
   - EVT: 至少`window + 10`个样本

3. **多资产训练**：
   - 所有特征都支持多资产数据（通过`_symbol`列识别）
   - 确保按资产分组计算，避免跨资产数据泄露
