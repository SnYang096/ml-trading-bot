# 滚动归一化（Rolling Normalization）说明

## 📖 什么是滚动归一化？

滚动归一化是一种**时序数据归一化方法**，它只使用**历史窗口内的数据**计算统计量（均值、标准差等），而不是使用整个数据集。

## ⚠️ 为什么需要滚动归一化？

### 问题：全局归一化的未来信息泄露（Look-ahead Bias）

在时序数据中，如果使用**全局归一化**（使用整个数据集计算统计量），会导致严重的**未来信息泄露**问题：

#### 1. 训练阶段的问题

```python
# ❌ 错误方式：全局归一化
# 假设数据集：2020-01-01 到 2023-12-31
# 训练集：2020-2022，测试集：2023

# 全局归一化会使用整个数据集（包括2023年的测试集）计算 mean 和 std
mean = df["feature"].mean()  # 使用了2023年的数据！
std = df["feature"].std()    # 使用了2023年的数据！

# 然后用这个统计量归一化训练集
train_normalized = (train["feature"] - mean) / std
```

**问题**：训练集被归一化时，使用了测试集的信息！模型在训练时"看到"了未来数据。

#### 2. 测试/部署阶段的问题

```python
# 测试时，仍然使用全样本的统计量
test_normalized = (test["feature"] - mean) / std  # mean/std 包含了测试集本身

# 在线部署时，无法获取未来数据的统计量！
# 因为未来数据还不存在！
```

**问题**：
- 测试时：统计量包含了测试集本身，导致测试结果不真实
- 在线部署：无法使用（需要未来数据）

#### 3. 性能虚高

由于训练时使用了未来信息，模型在验证集上表现会**虚高**，但实际部署时性能会**大幅下降**。

### 解决方案：滚动归一化

滚动归一化只使用**历史窗口**内的数据计算统计量：

```python
# ✅ 正确方式：滚动归一化
# 在时间点 t，只使用 [t-window, t-1] 的数据计算统计量

for t in range(window, len(df)):
    # 只使用历史窗口数据
    window_data = df["feature"][t-window:t]
    mean_t = window_data.mean()  # 只用历史数据
    std_t = window_data.std()    # 只用历史数据
    
    # 归一化当前值
    normalized[t] = (df["feature"][t] - mean_t) / std_t
```

**优势**：
- ✅ 训练时：只使用历史数据，无未来信息泄露
- ✅ 测试时：只使用历史数据，结果真实
- ✅ 在线部署：可以使用（只需要历史数据）

## 📊 滚动归一化示例

### 示例1：防止未来信息泄露

```python
import pandas as pd
import numpy as np
from src.features.time_series.utils_normalization import normalize_by_group

# 创建时序数据
dates = pd.date_range("2020-01-01", periods=500, freq="D")
df = pd.DataFrame({
    "date": dates,
    "price": np.random.randn(500).cumsum() + 100,
})

# ❌ 全局归一化（错误）
global_norm = normalize_by_group(
    df,
    value_col="price",
    method="zscore",
    window=None,  # 使用全样本
)
# 警告：会触发未来信息泄露警告！

# ✅ 滚动归一化（正确）
rolling_norm = normalize_by_group(
    df,
    value_col="price",
    method="zscore",
    window=252,  # 使用过去252天的数据
)
# 无警告，安全使用
```

### 示例2：适应市场状态变化

滚动归一化还能**自动适应市场状态变化**：

```python
# 创建有状态变化的数据
dates = pd.date_range("2020-01-01", periods=400, freq="D")

# 前200天：低波动期（std=1）
period1 = np.random.randn(200) * 1 + 100

# 后200天：高波动期（std=5）
period2 = np.random.randn(200) * 5 + 100

df = pd.DataFrame({
    "date": dates,
    "price": np.concatenate([period1, period2]),
})

# 滚动归一化（window=30）
rolling_norm = normalize_by_group(
    df,
    value_col="price",
    method="zscore",
    window=30,
)

# 结果：
# - 低波动期：归一化后标准差≈1
# - 高波动期：归一化后标准差≈1
# 滚动归一化自动适应了每个时期的波动率！
```

## 🎯 如何选择窗口大小？

窗口大小的选择取决于：

1. **数据频率**
   - 日频数据：`window=252`（1年，约252个交易日）
   - 周频数据：`window=52`（1年，52周）
   - 小时数据：`window=168`（1周，7×24小时）

2. **数据特性**
   - 波动率变化快：较小的窗口（如60-90天）
   - 波动率变化慢：较大的窗口（如252-504天）

3. **业务需求**
   - 需要快速适应市场变化：较小窗口
   - 需要稳定、平滑的特征：较大窗口

### 推荐配置

```python
# 日频数据（推荐）
window = 252  # 1年

# 周频数据（推荐）
window = 52   # 1年

# 小时数据（推荐）
window = 168  # 1周

# 分钟数据（推荐）
window = 1440  # 1天（24×60分钟）
```

## 💡 最佳实践

### 1. 始终使用滚动归一化（时序数据）

```python
# ✅ 推荐
normalized = normalize_by_group(
    df,
    value_col="feature",
    method="zscore",
    window=252,  # 明确指定窗口
)
```

### 2. 避免全局归一化（除非明确需要）

```python
# ❌ 不推荐（除非是EDA或非时序场景）
normalized = normalize_by_group(
    df,
    value_col="feature",
    window=None,  # 会触发警告
)
```

### 3. 多资产场景

```python
# ✅ 多资产滚动归一化
normalized = normalize_by_group(
    df,
    value_col="feature",
    group_col="_symbol",  # 按资产分组
    method="zscore",
    window=252,  # 每个资产独立滚动归一化
)
```

### 4. 处理窗口不足的情况

滚动归一化的前 `window-1` 行可能因窗口不足而为 NaN，建议：

```python
normalized = normalize_by_group(
    df,
    value_col="feature",
    window=252,
    fillna=True,  # 填充NaN为0
)
```

## 🔍 验证滚动归一化

可以通过以下方式验证滚动归一化是否正确：

1. **检查前window行**：应该被填充或标记
2. **检查统计量**：每个窗口内的归一化结果应该均值≈0，标准差≈1
3. **时间分割测试**：严格按时间分割训练/测试集，验证无未来信息泄露

## ⚠️ 总结

**滚动归一化是时序数据归一化的标准做法**，它：

1. ✅ 防止未来信息泄露
2. ✅ 适应市场状态变化
3. ✅ 支持在线部署
4. ✅ 提供真实的模型性能评估

**全局归一化在时序数据中应该避免使用**，除非：
- EDA（探索性数据分析）
- 非时序场景（横截面数据）
- 严格按时间分割的训练/测试集，且只在训练集上计算统计量
