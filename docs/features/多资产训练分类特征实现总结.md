# 多资产训练分类特征实现总结

## ✅ 已完成的更新

### 1. 创建分类特征工具模块

**文件**: `src/features/utils_categorical_features.py`

提供三个核心函数：
- `detect_categorical_features()`: 自动检测分类特征
- `prepare_categorical_features_for_lightgbm()`: 为LightGBM准备分类特征（名称和索引）
- `validate_categorical_features()`: 验证分类特征有效性

### 2. 更新LightGBMTrainer支持分类特征

**文件**: `src/time_series_model/strategies/models/lightgbm_model.py`

#### 更新的方法：

1. **`_prepare_features()`**
   - 添加`categorical_features`参数
   - 不再只选择数值列，同时保留分类特征列

2. **`prepare_data()`**
   - 添加`categorical_features`参数
   - 传递给`_prepare_features()`

3. **`train()`**
   - 添加`categorical_features`参数
   - **自动检测**：如果未提供且存在`_symbol`列（且唯一值>1），自动检测为分类特征
   - 存储分类特征到`self._categorical_features`供后续使用
   - 在创建`lgb.Dataset`时传递`categorical_feature`参数

4. **`optimize_hyperparameters()`**
   - 添加`categorical_features`参数
   - 在超参数优化时也使用分类特征

5. **`optimize_hyperparameters_for_q50_constraint()`**
   - 添加`categorical_features`参数
   - 在Q50约束优化时也使用分类特征

6. **`predict()`**
   - 使用存储的`self._categorical_features`确保预测时特征一致

## 📊 工作原理

### 1. 特征准备阶段

```python
# _prepare_features 会保留分类特征
numeric_cols = X.select_dtypes(include=[np.number]).columns.tolist()
cat_cols = [col for col in categorical_features if col in X.columns]
all_cols = numeric_cols + cat_cols  # 合并数值和分类特征
```

### 2. LightGBM Dataset创建

```python
# 计算分类特征的索引
cat_feature_indices = [
    i for i, col in enumerate(X_train.columns)
    if col in categorical_features
]

# 创建Dataset时传递分类特征
train_data = lgb.Dataset(
    X_train,
    label=y_train,
    categorical_feature=cat_feature_indices,  # 关键！
    free_raw_data=False,
)
```

### 3. LightGBM如何处理分类特征

- **字符串类型**：LightGBM内部自动hash编码
- **整数类型**：如果已经是整数编码，也可以使用
- **Split方式**：LightGBM会为分类特征创建category split，例如：
  - "如果 asset ∈ {BTC, ETH} 且 compression_duration > 10 → buy"
  - "如果 asset == SOL → 忽略 volume_anomaly"

## ⚠️ 注意事项

1. **单资产 vs 多资产**：
   - 单资产：`_symbol`是常数，会被自动排除
   - 多资产：`_symbol`会被自动检测为分类特征

2. **特征列顺序**：
   - 确保训练和预测时特征列顺序一致
   - 分类特征索引基于列顺序计算

3. **LightGBM参数建议**：
   ```python
   params = {
       "min_data_per_group": 30,  # 每个symbol至少30样本才split
       "cat_smooth": 10,  # 平滑稀有symbol
       "max_cat_to_onehot": 4,  # 类别数<=4时使用one-hot
   }
   ```

4. **数据泄露预防**：
   - 特征提取时按`_symbol`分组（已实现）
   - 训练时使用`_symbol`作为分类特征（已实现）
   - 两者结合确保无数据泄露
