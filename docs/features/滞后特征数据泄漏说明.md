# 滞后特征数据泄漏说明

## 核心结论

**✅ 滞后特征（Lag Features）不使用未来信息，完全安全**

## 什么是滞后特征？

### 定义

**滞后特征（Lag Features）**：使用**过去**的值作为当前时刻的特征。

### 代码实现

```python
# 计算基础波动率（使用历史数据）
vol_base = returns.rolling(window=20, min_periods=10).std()

# 创建滞后特征
for lag in [1, 2, 3]:
    result[f"vol_lag_{lag}"] = vol_base.shift(lag)
```

### pandas `shift()` 方法说明

**`shift(lag)` 的含义**：
- `shift(1)`：将数据**向后**移动1期，即使用**1期之前**的值
- `shift(2)`：将数据**向后**移动2期，即使用**2期之前**的值
- `shift(-1)`：将数据**向前**移动1期，即使用**1期之后**的值（⚠️ 这是未来信息！）

## 时间对齐示例

假设我们有以下数据：

| 时间 | 波动率 (vol_base) | vol_lag_1 | vol_lag_2 | vol_lag_3 |
|------|------------------|-----------|-----------|-----------|
| t-3  | 0.05             | NaN       | NaN       | NaN       |
| t-2  | 0.06             | 0.05      | NaN       | NaN       |
| t-1  | 0.04             | 0.06      | 0.05      | NaN       |
| **t** | **0.03**         | **0.04**  | **0.06**  | **0.05**  |
| t+1  | 0.07             | 0.03      | 0.04      | 0.06      |

**在时间 t**：
- `vol_lag_1` = 0.04（t-1 时刻的波动率）✅ **过去信息**
- `vol_lag_2` = 0.06（t-2 时刻的波动率）✅ **过去信息**
- `vol_lag_3` = 0.05（t-3 时刻的波动率）✅ **过去信息**

**结论**：所有滞后特征都使用**过去**的值，**不使用未来信息**。

## 为什么需要滞后特征？

### 1. 波动率聚集性（Volatility Clustering）

波动率具有**自相关性**：
- 今天高波动，明天很可能也高
- 这是 GARCH 模型的核心假设

### 2. 捕捉历史模式

滞后特征帮助模型学习：
- 短期记忆（lag_1）：昨天波动率对今天的影响
- 中期记忆（lag_2-3）：更早的波动率模式

## 与未来信息的对比

### ❌ 错误：使用未来信息

```python
# ⚠️ 错误：使用未来信息
result["vol_future_1"] = vol_base.shift(-1)  # 使用 t+1 的值
result["vol_future_2"] = vol_base.shift(-2)  # 使用 t+2 的值
```

**问题**：在时间 t，我们无法知道 t+1 或 t+2 的波动率，这是**数据泄漏**。

### ✅ 正确：使用滞后特征

```python
# ✅ 正确：使用过去信息
result["vol_lag_1"] = vol_base.shift(1)   # 使用 t-1 的值
result["vol_lag_2"] = vol_base.shift(2)   # 使用 t-2 的值
```

**安全**：在时间 t，我们**已经知道** t-1 和 t-2 的波动率，这是**历史信息**。

## 验证方法

### 1. 代码检查

```python
# 检查 shift 的方向
vol_lag_1 = vol_base.shift(1)  # ✅ 正数 = 向后移动 = 使用过去
vol_future_1 = vol_base.shift(-1)  # ❌ 负数 = 向前移动 = 使用未来
```

### 2. 时间对齐测试

```python
# 在时间 t，检查特征值
t = 100
print(f"vol_base[t] = {vol_base.iloc[t]}")        # 当前波动率
print(f"vol_lag_1[t] = {vol_lag_1.iloc[t]}")      # 应该是 vol_base[t-1]
print(f"vol_base[t-1] = {vol_base.iloc[t-1]}")    # 验证：应该相等

# 如果 vol_lag_1[t] == vol_base[t-1]，说明正确使用了过去信息
assert vol_lag_1.iloc[t] == vol_base.iloc[t-1], "时间对齐错误！"
```

### 3. 相关性测试

使用 `scripts/verify_feature_correlation.py` 进行滞后测试：
- 如果特征已经正确 `shift(1)`，再滞后 1-2 根，相关性应该**缓慢衰减**
- 如果相关性**急剧下降**，可能是时间对齐问题

## 实际应用

### 波动率预测模型

```python
# 在时间 t，预测未来波动率
# 使用的特征：
X_t = [
    vol_lag_1[t],    # t-1 时刻的波动率 ✅
    vol_lag_2[t],    # t-2 时刻的波动率 ✅
    vol_lag_3[t],    # t-3 时刻的波动率 ✅
    garch_vol[t],    # GARCH 预测的波动率 ✅
    atr[t],          # 当前 ATR ✅
]

# 预测目标：
y_t = future_volatility[t]  # 未来波动率（标签，不是特征）
```

**关键点**：
- 所有特征都使用**t 时刻及之前**的信息
- 预测目标是**未来**的波动率（这是正确的，因为标签可以使用未来信息）

## 总结

| 特征类型 | 时间对齐 | 是否泄漏 | 说明 |
|---------|---------|---------|------|
| **滞后特征** | `shift(1)` = 使用 t-1 的值 | ✅ **不泄漏** | 使用过去信息，完全安全 |
| **当前特征** | 使用 t 的值 | ✅ **不泄漏** | 使用当前信息，完全安全 |
| **未来特征** | `shift(-1)` = 使用 t+1 的值 | ❌ **泄漏** | 使用未来信息，禁止使用 |
| **标签** | 使用 t+1 到 t+H 的值 | ✅ **正确** | 标签可以使用未来信息 |

**核心原则**：
- ✅ **特征**：只能使用**当前及过去**的信息
- ✅ **标签**：可以使用**未来**的信息（用于监督学习）
- ❌ **禁止**：特征使用未来信息（数据泄漏）
