# 交互特征策略配置说明

## 📋 概述

交互特征（Interaction Features）是两个特征的乘积，用于捕捉"状态 × 动量"的组合效应。本文档说明各个策略中配置的交互特征及其用途。

## 🎯 交互特征列表

### 1. `liquidity_void_x_wpt_risk`
**公式**: `liquidity_void_detected × wpt_false_breakout_risk`

**用途**: 流动性真空 × WPT 假突破风险

**适用策略**:
- ✅ **SR Breakout**: 识别假突破（流动性真空 + 高假突破风险）
- ✅ **Compression Breakout**: 识别假突破

**解读**:
- 高值：流动性真空 + 高假突破风险 → 假突破概率高
- 低值：无流动性真空或低假突破风险 → 真突破概率高

---

### 2. `compression_energy_x_ofi_short`
**公式**: `compression_energy × ofi_short`

**用途**: 压缩能量 × 订单流强度

**适用策略**:
- ✅ **Compression Breakout**: 压缩区突破确认

**解读**:
- 高值：高压缩能量 + 强订单流 → 突破概率高
- 低值：低压缩能量或弱订单流 → 突破概率低

---

### 3. `hurst_x_trend_r2`
**公式**: `hurst_price_rolling × trend_r2_20`

**用途**: Hurst 指数 × 趋势 R²

**适用策略**:
- ✅ **SR Breakout**: 确认突破质量（趋势持续性 + 趋势强度）
- ✅ **Trend Following**: 确认趋势持续性

**解读**:
- 高值：高 Hurst（趋势性强）+ 高 R²（趋势质量好）→ 强趋势
- 低值：低 Hurst（均值回复）或低 R²（趋势质量差）→ 弱趋势

---

### 4. `evt_x_trend_r2`
**公式**: `evt_tail_shape × trend_r2_20`

**用途**: EVT 尾部风险 × 趋势强度

**适用策略**:
- ✅ **Trend Following**: 风险管理（尾部风险 + 趋势强度）
- ✅ **SR Reversal**: 反转信号确认

**解读**:
- 高值：高尾部风险 + 强趋势 → 高风险趋势（需谨慎）
- 低值：低尾部风险或弱趋势 → 相对安全

---

### 5. `vpin_x_compression`
**公式**: `vpin × compression_energy`

**用途**: VPIN × 压缩能量

**适用策略**:
- ✅ **Compression Breakout**: 压缩区突破确认

**解读**:
- 高值：高 VPIN（知情交易）+ 高压缩能量 → 突破概率高
- 低值：低 VPIN 或低压缩能量 → 突破概率低

---

### 6. `sma_slope_x_price_pos`
**公式**: `sma_200_slope × (close / sma_200)`

**用途**: 均线斜率 × 价格位置

**适用策略**:
- ✅ **Trend Following**: 趋势确认（均线斜率 + 价格相对位置）

**解读**:
- 高值：均线上行 + 价格在均线上方 → 强上升趋势
- 低值：均线下行或价格在均线下方 → 弱趋势或下降趋势

---

### 7. `vpin_x_wick_upper`
**公式**: `vpin × wick_upper_ratio`

**用途**: VPIN × 上影线占比

**适用策略**:
- ✅ **SR Reversal**: 反转信号（高 VPIN + 长上影线 = 强卖压）

**解读**:
- 高值：高 VPIN（知情交易）+ 长上影线 → 强卖压，可能反转
- 低值：低 VPIN 或短上影线 → 卖压弱

---

### 8. `vpin_x_wick_lower`
**公式**: `vpin × wick_lower_ratio`

**用途**: VPIN × 下影线占比

**适用策略**:
- ✅ **SR Reversal**: 反转信号（高 VPIN + 长下影线 = 强买压）

**解读**:
- 高值：高 VPIN（知情交易）+ 长下影线 → 强买压，可能反转
- 低值：低 VPIN 或短下影线 → 买压弱

---

## 📊 策略配置总结

### SR Breakout 策略
```yaml
- liquidity_void_x_wpt_risk  # 识别假突破
- hurst_x_trend_r2  # 确认突破质量
```

**用途**:
- 识别假突破（流动性真空 + 高假突破风险）
- 确认突破质量（趋势持续性 + 趋势强度）

---

### Trend Following 策略
```yaml
- hurst_x_trend_r2  # 确认趋势持续性
- evt_x_trend_r2  # 风险管理
- sma_slope_x_price_pos  # 趋势确认
```

**用途**:
- 确认趋势持续性（Hurst + 趋势 R²）
- 风险管理（尾部风险 + 趋势强度）
- 趋势确认（均线斜率 + 价格位置）

---

### SR Reversal 策略
```yaml
- vpin_x_wick_upper  # VPIN × 上影线
- vpin_x_wick_lower  # VPIN × 下影线
- vpin_x_wick_upper_rank  # Rank transform 版本
- vpin_x_wick_lower_rank  # Rank transform 版本
- evt_x_trend_r2  # 反转信号确认
```

**用途**:
- 识别反转信号（VPIN + 影线）
- 反转信号确认（尾部风险 + 趋势强度）

---

### Compression Breakout 策略
```yaml
- liquidity_void_x_wpt_risk  # 识别假突破
- compression_energy_x_ofi_short  # 压缩区突破确认
- vpin_x_compression  # 压缩区突破确认
```

**用途**:
- 识别假突破（流动性真空 + 高假突破风险）
- 压缩区突破确认（压缩能量 + 订单流强度）
- 压缩区突破确认（VPIN + 压缩能量）

## 🔍 使用建议

### 1. 交互特征的组合使用

交互特征通常与基础特征组合使用：
- 基础特征：提供单一维度的信息
- 交互特征：捕捉两个维度的组合效应

**示例**:
```python
# SR Breakout 策略
if (liquidity_void_detected > 0.5 and 
    wpt_false_breakout_risk > 0.7 and
    liquidity_void_x_wpt_risk > 0.35):  # 交互特征
    # 假突破概率高
    signal = "false_breakout"
```

### 2. Rank Transform 版本

对于某些交互特征，提供了 Rank Transform 版本（`_rank` 后缀）：
- 原始版本：绝对值
- Rank 版本：相对排名（0-1），更稳健，跨品种可比

**建议**:
- 单品种模型：使用原始版本
- 多品种模型：使用 Rank 版本

### 3. 特征重要性

交互特征的重要性通常低于基础特征，但能提供额外的信息：
- 基础特征：主要信号
- 交互特征：辅助确认

**建议**:
- 先确保基础特征质量
- 再添加交互特征提升模型性能
