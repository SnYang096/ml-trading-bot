    if "ticks_loader_json" in error_msg.lower():

## 特征计算流程（改进后）

```
1. 加载原始数据 (train_strategy_pipeline.py)
   └─> load_raw_data()  # K 线数据

2. 配置 Tick 加载器 (train_strategy_pipeline.py)
   └─> configure_tick_loader_for_features()
       ├─> list_tick_files()  # 列出 tick 文件
       ├─> serialize_tick_loader_params()  # 序列化为 JSON
       └─> 验证配置（新增：详细的错误处理）

3. 创建特征加载器 (train_strategy_pipeline.py)
   └─> StrategyFeatureLoader()
       └─> 加载 feature_dependencies.yaml

4. 计算特征 (StrategyFeatureLoader)
   └─> load_features_from_requested()
       ├─> 检查 required_columns（新增：更详细的诊断）
       ├─> 添加缺失的基础列
       ├─> compute_features_parallel()
       │   ├─> 按月拆分数据（如果支持）
       │   ├─> 检查缓存（使用版本控制）
       │   ├─> 调用特征计算函数
       │   │   ├─> 对于需要 tick 的特征：
       │   │   │   ├─> 从 ticks_loader_json 反序列化参数（新增：详细错误处理）
       │   │   │   └─> 调用 tick_loader.py 中的函数
       │   │   └─> 对于其他特征：
       │   │       └─> 直接调用特征计算函数
       │   └─> 合并结果（新增：改进的错误处理）
       └─> 验证和过滤（新增：索引对齐、数据类型检查）
           ├─> 过滤新索引
           ├─> 重新对齐索引
           └─> 检查数据类型
```

## 改进效果

### 1. 错误诊断更清晰
- **之前**：`ValueError: Invalid JSON format`
