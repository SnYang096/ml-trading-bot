# 特征工程文件功能说明

## 文件功能对比

### 1. `enhanced_features.py` (EnhancedFeatureEngineer 类)

**功能：**
- 提供**类方法**形式的特征工程
- 内部实现了 WPT、Hurst、Hilbert、Spectrum 功能
- 作为 `ComprehensiveFeatureEngineer` 的一部分被调用
- 自动处理标准化、缩放、特征命名等

**使用方式：**
```python
from src.features.time_series.enhanced_features import EnhancedFeatureEngineer

engineer = EnhancedFeatureEngineer()
df = engineer.add_wavelet_packet_features(df)  # 类方法
df = engineer.add_hurst_features(df)
df = engineer.add_hilbert_features(df)
```

**特点：**
- ✅ 集成度高，自动处理所有细节
- ✅ 与 `ComprehensiveFeatureEngineer` 配合使用
- ⚠️ 需要实例化类
- ⚠️ 输出特征名称有固定前缀（如 `wpt_*`, `hurst_*`）

### 2. `utils_*_features.py` (独立工具函数)

**功能：**
- 提供**独立函数**形式的特征工程工具
- 更灵活，可以单独使用
- 适合策略专属特征工程

**使用方式：**
```python
from src.features.time_series.utils_wpt_features import extract_wpt_features
from src.features.time_series.utils_hilbert_features import extract_hilbert_features

df = extract_wpt_features(df)  # 独立函数
df = extract_hilbert_features(df)
```

**特点：**
- ✅ 灵活，可以单独使用
- ✅ 适合策略专属特征工程
- ✅ 函数式编程风格
- ⚠️ 需要手动处理特征命名和组合

## 功能重复分析

### WPT 功能

**enhanced_features.py:**
- `add_wavelet_packet_features()` - 类方法
- 对所有信号源（close, open, volume, cvd, taker_buy_ratio）做 WPT
- 自动生成特征名称（如 `wpt_close_energy_*`, `wpt_volume_energy_*`）
- 自动标准化和缩放

**utils_wpt_features.py:**
- `wpt_decompose()` - 独立函数，返回字典
- `extract_wpt_features()` - 独立函数，直接操作 DataFrame
- 更灵活，可以自定义参数和输出

**结论：** 功能有重叠，但使用场景不同

### Hilbert 功能

**enhanced_features.py:**
- `add_hilbert_features()` - 类方法
- 对所有信号源做 Hilbert 变换
- 自动生成特征名称（如 `hilbert_close_phase_*`）

**utils_hilbert_features.py:**
- `hilbert_transform()` - 独立函数
- `compute_phase_lead()` - 计算相位领先
- `extract_hilbert_features()` - 独立函数
- 更专注于相位领先分析

**结论：** `utils_hilbert_features.py` 提供了更专业的相位领先分析功能

### Hurst 功能

**enhanced_features.py:**
- `add_hurst_features()` - 类方法
- `calculate_hurst_exponent()` - 内部方法
- 对所有信号源计算 Hurst

**utils_hurst_features.py:**
- `compute_hurst_rs()` - R/S 方法
- `compute_hurst_dfa()` - DFA 方法（推荐）
- `extract_hurst_features()` - 独立函数

**结论：** `utils_hurst_features.py` 提供了 DFA 方法，更稳健

### Spectrum 功能

**enhanced_features.py:**
- `add_spectral_features()` - 类方法（如果存在）
- 可能包含在 enhanced_features 中

**utils_spectrum_features.py:**
- `compute_spectrum_features()` - 独立函数
- `extract_spectrum_features()` - 独立函数
- 提供主频、平坦度等特征

**结论：** `utils_spectrum_features.py` 提供了更完整的频谱分析功能

## 推荐使用方案

### 方案 1：统一使用 enhanced_features.py（推荐用于综合特征工程）

**适用场景：**
- 使用 `ComprehensiveFeatureEngineer` 进行综合特征工程
- 需要自动处理所有细节（标准化、缩放、命名）
- 批量处理多个信号源

**优点：**
- ✅ 集成度高，使用简单
- ✅ 自动处理标准化和缩放
- ✅ 特征命名统一

**缺点：**
- ⚠️ 灵活性较低
- ⚠️ 需要实例化类

### 方案 2：使用 utils_*_features.py（推荐用于策略专属特征）

**适用场景：**
- 策略专属特征工程（如 `sr_reversal/features.py`）
- 需要灵活控制特征生成过程
- 需要自定义参数和输出

**优点：**
- ✅ 灵活，可以单独使用
- ✅ 函数式编程风格
- ✅ 适合策略专属特征工程

**缺点：**
- ⚠️ 需要手动处理特征命名和组合

### 方案 3：混合使用（当前实现）

**当前策略 features.py 的实现：**
- 使用 `utils_*_features.py` 进行策略专属特征工程
- 使用 `ComprehensiveFeatureEngineer` 进行基础特征工程

**优点：**
- ✅ 灵活性高
- ✅ 策略可以自定义特征生成过程

## 建议

### 1. 保留两种方式

**理由：**
- `enhanced_features.py` 适合综合特征工程（通过 `ComprehensiveFeatureEngineer` 使用）
- `utils_*_features.py` 适合策略专属特征工程（灵活、可定制）

### 2. 明确使用场景

**使用 `enhanced_features.py` 当：**
- 通过 `ComprehensiveFeatureEngineer` 进行综合特征工程
- 需要批量处理多个信号源
- 需要自动标准化和缩放

**使用 `utils_*_features.py` 当：**
- 策略专属特征工程（如 `sr_reversal/features.py`）
- 需要灵活控制特征生成过程
- 需要自定义参数和输出

### 3. 可选：整合 utils 到 enhanced_features

如果觉得重复，可以考虑：
- 将 `utils_*_features.py` 中的函数作为 `EnhancedFeatureEngineer` 的静态方法
- 或者将 `utils_*_features.py` 作为 `enhanced_features.py` 的辅助模块

## 当前实现状态

**策略 features.py 文件：**
- ✅ 使用 `utils_*_features.py` 进行策略专属特征工程
- ✅ 使用 `ComprehensiveFeatureEngineer` 进行基础特征工程

**这是合理的，因为：**
- 策略需要灵活控制特征生成过程
- `utils_*_features.py` 提供了更专业的工具函数（如相位领先分析、DFA Hurst）
- `enhanced_features.py` 更适合批量处理

## 总结

**没有完全重复，使用场景不同：**

| 文件 | 使用场景 | 特点 |
|------|---------|------|
| `enhanced_features.py` | 综合特征工程（通过 ComprehensiveFeatureEngineer） | 集成度高，自动处理 |
| `utils_*_features.py` | 策略专属特征工程 | 灵活，可定制 |

**建议：保留两种方式，根据使用场景选择。**
