# 特征工程重复实现分析

## 发现的重复文件

### 1. `feature_engineering_enhanced.py` vs `enhanced_features.py`
- **状态**：完全重复（都是 ~1985 行）
- **使用情况**：
  - `comprehensive_features.py` 使用的是 `enhanced_features.py`
  - `feature_engineering_enhanced.py` 没有被使用
- **建议**：删除 `feature_engineering_enhanced.py`

### 2. WPT 特征实现重复
- **`utils_wpt_features.py`**：
  - `extract_wpt_features()`: ❌ **有数据泄露**（全序列处理）
  - `wpt_decompose()`: ✅ 基础函数，无问题
- **`enhanced_features.py`**：
  - `add_wavelet_packet_features()`: ✅ 滚动窗口，安全
  - `calculate_wavelet_packet_features()`: ✅ 滚动窗口，安全

### 3. Hurst 特征实现
- **`utils_hurst_features.py`**: 独立工具函数
- **`enhanced_features.py`**: 类方法，滚动窗口实现
- **状态**：两者都安全，但实现方式不同

### 4. Hilbert 特征实现
- **`utils_hilbert_features.py`**: 独立工具函数
- **`enhanced_features.py`**: 类方法
- **状态**：需要检查是否有数据泄露

### 5. Spectrum 特征实现
- **`utils_spectrum_features.py`**: 独立工具函数
- **`enhanced_features.py`**: 类方法
- **状态**：需要检查是否有数据泄露

## 修复计划

1. **删除重复文件**：`feature_engineering_enhanced.py`
2. **修复数据泄露**：`utils_wpt_features.py` 的 `extract_wpt_features()` 改为滚动窗口
3. **统一实现**：确保所有特征都使用滚动窗口，无数据泄露
4. **归一化检查**：确保所有特征都经过归一化处理
