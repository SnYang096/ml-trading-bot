# Q50约束优化说明

## optimize_hyperparameters_for_q50_constraint() 的作用

### 背景

在分位数回归模型中，我们通常训练三个模型：
- **Q10模型**：预测10%分位数（下界）
- **Q50模型**：预测中位数（50%分位数）
- **Q90模型**：预测90%分位数（上界）

### 问题

理论上，Q50应该位于Q10和Q90之间：
```
Q10 ≤ Q50 ≤ Q90
```

但在实际训练中，由于模型独立训练，可能出现：
- Q50 < Q10（中位数低于下界，不合理）
- Q50 > Q90（中位数高于上界，不合理）

### 解决方案

`optimize_hyperparameters_for_q50_constraint()` 专门优化Q50模型的超参数，确保：
```
Q50损失 ≤ max(Q10损失, Q90损失)
```

这样Q50预测值会更合理地位于Q10和Q90之间。

### 支持分类特征的意义

现在支持分类特征（如`_symbol`）意味着：

1. **多资产训练**：在优化Q50模型时，也会考虑不同资产的差异
2. **更准确的约束**：不同资产可能有不同的波动率分布，分类特征帮助模型学习资产特定的Q50约束
3. **统一优化**：一个优化过程同时考虑通用模式和资产特定模式

### 使用场景

主要用于：
- **波动率预测**：预测未来波动率的分位数（Q10, Q50, Q90）
- **风险量化**：VaR（Value at Risk）和ES（Expected Shortfall）计算
- **仓位管理**：根据波动率分位数调整仓位大小

### 代码示例

```python
# 训练Q50模型
q50_trainer = LightGBMTrainer(model_type="quantile", quantile_alpha=0.5)

# 优化Q50约束（确保Q50在Q10和Q90之间）
best_params = q50_trainer.optimize_hyperparameters_for_q50_constraint(
    X=X_train,
    y=y_train,
    n_trials=20,
    n_splits=3,
    categorical_features=["_symbol"],  # 支持多资产
)

# 使用优化后的参数训练
q50_trainer.params.update(best_params)
q50_trainer.train(X_train, y_train)
```

### 总结

- **目的**：确保Q50预测值合理（在Q10和Q90之间）
- **方法**：通过优化超参数，使Q50损失不超过Q10/Q90的最大损失
- **分类特征支持**：让优化过程考虑不同资产的差异，提高多资产场景下的准确性
