# 特征依赖管理和按需加载方案

## 需求分析

### 核心需求
1. **按业务策略加载指定特征**：业务可以选择自己需要的特征
2. **研究阶段和实盘阶段统一**：确保两个阶段使用相同的特征计算逻辑
3. **解决特征依赖关系**：自动处理特征之间的依赖和计算顺序
4. **按需计算**：只计算需要的特征，避免不必要的计算开销

### 当前问题
- ❌ 特征依赖关系不明确
- ❌ 无法按需加载特征
- ❌ 研究阶段和实盘阶段可能不一致
- ❌ 特征计算顺序不清晰

### 2. 依赖解析算法

```python
def resolve_dependencies(requested_features, registry):
    """
    解析特征依赖关系，返回计算顺序
    
    Args:
        requested_features: 请求的特征列表
        registry: 特征注册表
    
    Returns:
        computation_order: 计算顺序（拓扑排序结果）
    """
    # 1. 收集所有需要的特征（包括依赖）
    all_needed = set(requested_features)
    queue = list(requested_features)
    
    while queue:
        feature = queue.pop(0)
        if feature in registry:
            deps = registry[feature]["dependencies"]
            for dep in deps:
                if dep not in all_needed:
                    all_needed.add(dep)
                    queue.append(dep)
    
    # 2. 构建依赖图
    graph = {f: [] for f in all_needed}
    in_degree = {f: 0 for f in all_needed}
    
    for feature in all_needed:
        if feature in registry:
            deps = registry[feature]["dependencies"]
            for dep in deps:
                if dep in all_needed:
                    graph[dep].append(feature)
                    in_degree[feature] += 1
    
    # 3. 拓扑排序
    queue = [f for f in all_needed if in_degree[f] == 0]
    result = []
    
    while queue:
        feature = queue.pop(0)
        result.append(feature)
        for neighbor in graph[feature]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    return result
```

### 3. 特征计算调度器

```python
class FeatureScheduler:
    def __init__(self, registry):
        self.registry = registry
        self.computed_cache = {}
    
    def compute_features(self, df, requested_features, fit=True):
        """
        按需计算特征
        
        Args:
            df: 输入 DataFrame
            requested_features: 请求的特征列表
            fit: 是否拟合（研究阶段=True，实盘阶段=False）
        
        Returns:
            df_with_features: 包含计算特征的 DataFrame
        """
        # 1. 解析依赖
        computation_order = resolve_dependencies(requested_features, self.registry)
        
        # 2. 按顺序计算
        result_df = df.copy()
        for feature_name in computation_order:
            if feature_name in self.computed_cache:
                # 使用缓存
                continue
            
            feature_info = self.registry[feature_name]
            compute_func = feature_info["compute_func"]
            
            # 检查所需列是否存在
            required_cols = feature_info.get("required_columns", [])
            missing_cols = [c for c in required_cols if c not in result_df.columns]
            if missing_cols:
                raise ValueError(f"Missing required columns for {feature_name}: {missing_cols}")
            
            # 计算特征
            result_df = compute_func(result_df, fit=fit)
            
            # 缓存
            self.computed_cache[feature_name] = True
        
        # 3. 只返回请求的特征列
        output_cols = []
        for feature_name in requested_features:
            feature_info = self.registry[feature_name]
            output_cols.extend(feature_info.get("output_columns", [feature_name]))
        
        return result_df[list(df.columns) + output_cols]
```

### 4. 研究阶段和实盘阶段统一

```python
class UnifiedFeatureEngineer:
    """
    统一的特征工程器，支持研究阶段和实盘阶段
    """
    def __init__(self, feature_config=None):
        self.registry = self._load_registry()
        self.scheduler = FeatureScheduler(self.registry)
        self.fitted_scalers = {}  # 保存拟合的 scaler
    
    def fit_transform(self, df, requested_features):
        """
        研究阶段：拟合并转换
        
        Args:
            df: 训练数据
            requested_features: 请求的特征列表
        
        Returns:
            df_with_features: 包含特征的 DataFrame
        """
        return self.scheduler.compute_features(df, requested_features, fit=True)
    
    def transform(self, df, requested_features):
        """
        实盘阶段：只转换（使用已拟合的参数）
        
        Args:
            df: 实时数据
            requested_features: 请求的特征列表（应与研究阶段一致）
        
        Returns:
            df_with_features: 包含特征的 DataFrame
        """
        return self.scheduler.compute_features(df, requested_features, fit=False)
```

## 优势总结

1. **依赖自动处理**：自动解析和计算依赖特征
2. **按需加载**：只计算需要的特征，提高效率
3. **研究实盘统一**：使用相同的计算逻辑
4. **易于扩展**：新增特征只需注册
5. **配置灵活**：支持策略级别的特征配置

## 注意事项

1. **依赖关系准确性**：需要仔细分析特征依赖关系
2. **计算顺序**：确保依赖特征先计算
3. **缓存管理**：避免内存泄漏
4. **错误处理**：依赖缺失时的错误提示
5. **向后兼容**：保持与现有代码的兼容性
