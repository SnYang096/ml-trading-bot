## 🚀 使用方式

### 1. 在策略配置中请求特征

```yaml
# config/strategies/sr_reversal/features.yaml
feature_pipeline:
  requested_features:
    # 衍生特征
    - sr_strength_combined
    - sr_distance_normalized
    - cvd_slope_5
    - atr_ratio
    # ...
```

### 2. 特征会自动加载

系统会：
1. 解析依赖关系
2. 按拓扑排序计算特征
3. 自动处理并行计算和缓存

## ⚠️ 注意事项

1. **依赖特征必须存在**：
   - `dist_to_zz_high` 需要 `zz_high_value`（来自 ZigZag 特征）
   - `bb_width_ratio` 需要 `bb_upper`, `bb_lower`, `bb_middle`（来自 Bollinger Bands）
   - `compression_score` 需要 `bb_width_ratio`

2. **某些特征可能因为依赖缺失而无法计算**：
   - 如果 `zz_high_value` 不存在，`dist_to_zz_high` 会返回默认值（0.0）
   - 如果 `bb_upper` 不存在，`bb_width_ratio` 会返回默认值（0.0）

3. **特征计算顺序**：
   - 系统会自动处理依赖关系，按正确顺序计算
   - 不需要手动指定计算顺序
