# 缓存版本控制使用说明

## 工作原理

### 1. 版本号的作用

缓存版本号用于**自动失效旧缓存**。当特征计算逻辑改变时，更新版本号可以确保使用新的计算逻辑，而不是旧的缓存数据。

### 2. 如何更新版本号

**位置**：`src/features/loader/parallel_computer.py` 第 398 行

```python
# 缓存版本控制（用于失效旧缓存）
# 当特征计算逻辑改变时，更新此版本号以失效旧缓存
# v5: 改进错误处理和流程验证，添加索引对齐检查
self.cache_version = "v5"
```

**更新方法**：
1. 修改特征计算逻辑（例如：修复 bug、改进算法）
2. 将 `self.cache_version` 从 `"v5"` 改为 `"v6"`（或更高版本）
3. 添加注释说明版本变化的原因

**示例**：
```python
# v6: 修复了 EVT 特征的 NaN 处理逻辑，使用基于实际数据的保守估计
self.cache_version = "v6"
```

### 3. 自动清理机制

**工作原理**：
1. 系统在初始化时检查版本标记文件（`.cache_version`）
2. 如果检测到版本号变化（例如：从 v5 改为 v6），**自动删除所有旧缓存文件**
3. 更新版本标记文件为当前版本

**效果**：
- ✅ **无需手动删除旧缓存**：版本号改变时，旧缓存会自动被清理
- ✅ **节省磁盘空间**：旧缓存文件不会占用空间
- ✅ **确保数据一致性**：只使用当前版本的缓存

### 4. 版本标记文件

**位置**：`cache/features/monthly/.cache_version`

**内容**：当前缓存版本号（例如：`v5`）

**作用**：
- 记录上次使用的缓存版本
- 用于检测版本号变化
- 触发自动清理机制

## 使用场景

### 场景1：修复特征计算 Bug

```python
# 之前：v5
self.cache_version = "v5"

# 修复 bug 后：v6
# v6: 修复了 rolling_std 计算中的数值溢出问题
self.cache_version = "v6"
```

**结果**：
- 旧缓存（v5）自动被清理
- 新缓存（v6）使用修复后的逻辑计算

### 场景2：改进特征算法

```python
# 之前：v5
self.cache_version = "v5"

# 改进算法后：v6
# v6: 优化了 VPIN 计算性能，使用更高效的算法
self.cache_version = "v6"
```

**结果**：
- 旧缓存（v5）自动被清理
- 新缓存（v6）使用改进后的算法计算

### 场景3：添加新功能

```python
# 之前：v5
self.cache_version = "v5"

# 添加新功能后：v6
# v6: 添加了索引对齐验证，确保输出索引与输入一致
self.cache_version = "v6"
```

**结果**：
- 旧缓存（v5）自动被清理
- 新缓存（v6）包含新功能的验证逻辑

## 手动清理缓存

如果需要手动清理所有缓存（不依赖版本号），可以使用：

```python
feature_loader.computer.clear_cache(memory=True, disk=True, old_versions=True)
```

**参数说明**：
- `memory=True`: 清除内存缓存
- `disk=True`: 清除磁盘缓存
- `old_versions=True`: 清除旧版本的按月缓存（默认 True）

## 常见问题

### Q1: 我需要手动删除旧缓存吗？

**A**: **不需要**。当你更新 `cache_version` 时，系统会自动检测版本变化并清理旧缓存。

### Q2: 版本号改变后，旧缓存文件还在吗？

**A**: **不会**。版本号改变时，所有旧缓存文件会被自动删除。

### Q3: 如何知道当前使用的缓存版本？

**A**: 查看 `cache/features/monthly/.cache_version` 文件，或查看代码中的 `self.cache_version`。

### Q4: 如果我想保留旧缓存怎么办？

**A**: 不要更新版本号，或者手动备份旧缓存文件。

### Q5: 版本号可以回退吗？

**A**: 可以，但不推荐。如果回退版本号，系统会认为版本改变了，会清理所有缓存（包括你想要的旧版本）。

## 最佳实践

1. **每次修改特征计算逻辑时，更新版本号**
   - 修复 bug → 更新版本号
   - 改进算法 → 更新版本号
   - 添加新功能 → 更新版本号

2. **在版本号注释中说明变化原因**
   ```python
   # v6: 修复了 EVT 特征的 NaN 处理逻辑
   self.cache_version = "v6"
   ```

3. **版本号格式建议**
   - 使用语义化版本：`v5`, `v6`, `v7`
   - 或者使用日期：`v20250101`
   - 保持一致性

4. **测试新版本**
   - 更新版本号后，运行一次特征计算
   - 确认新缓存正常工作
   - 检查是否有错误

## 总结

- ✅ **自动清理**：版本号改变时，旧缓存自动被清理
- ✅ **无需手动删除**：不再需要手动删除旧缓存文件
- ✅ **简单易用**：只需更新代码中的版本号即可
- ✅ **节省空间**：旧缓存不会占用磁盘空间
