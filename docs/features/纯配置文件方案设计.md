# 纯配置文件方案设计（简化版）

## 用户问题

**为什么要构建特征注册表（方案 4）？能不能直接在配置文件设置依赖，然后再搞个策略配置文件？**

## 答案：完全可以！这是更简化的方案

### 方案对比

| 方案 | 特征依赖定义位置 | 策略配置位置 | 复杂度 |
|------|----------------|------------|--------|
| **方案 2 + 方案 4** | Python 注册表 | YAML 配置文件 | ⭐⭐⭐ |
| **纯配置文件方案** | YAML 配置文件 | YAML 配置文件 | ⭐⭐ |

### 纯配置文件方案的优势

1. ✅ **更简单**：只需要配置文件，不需要 Python 注册表
2. ✅ **配置集中**：所有配置都在一个地方
3. ✅ **易于修改**：修改 YAML 即可，不需要改代码
4. ✅ **完全分离**：配置和代码完全分离

#### 4. 特征加载器实现

```python
# strategy_feature_loader.py
"""
基于纯配置文件的特征加载器
"""

import yaml
import pandas as pd
from typing import List, Dict, Optional
from pathlib import Path

from feature_function_mapping import get_compute_func
from src.features.time_series.comprehensive_features import ComprehensiveFeatureEngineer


class StrategyFeatureLoader:
    """
    基于纯配置文件的特征加载器
    
    特点：
    1. 特征依赖关系在 YAML 中定义
    2. 策略特征集在 YAML 中定义
    3. 自动解析依赖，按需计算
    """
    
    def __init__(
        self,
        feature_deps_path: str = "config/feature_dependencies.yaml",
        strategy_config_path: str = "config/strategy_features.yaml",
    ):
        """
        初始化特征加载器
        
        Args:
            feature_deps_path: 特征依赖配置文件路径
            strategy_config_path: 策略配置文件路径
        """
        self.feature_deps = self._load_yaml(feature_deps_path)
        self.strategy_config = self._load_yaml(strategy_config_path)
        self.computed_cache = {}  # 缓存已计算的特征
    
    def _load_yaml(self, path: str) -> Dict:
        """加载 YAML 配置文件"""
        with open(path, 'r', encoding='utf-8') as f:
            return yaml.safe_load(f)
    
    def resolve_dependencies(self, requested_features: List[str]) -> List[str]:
        """
        解析特征依赖关系，返回计算顺序（拓扑排序）
        
        Args:
            requested_features: 请求的特征列表
        
        Returns:
            computation_order: 计算顺序
        """
        # 1. 收集所有需要的特征（包括依赖）
        all_needed = set(requested_features)
        queue = list(requested_features)
        
        while queue:
            feature = queue.pop(0)
            if feature in self.feature_deps["features"]:
                deps = self.feature_deps["features"][feature].get("dependencies", [])
                for dep in deps:
                    if dep not in all_needed:
                        all_needed.add(dep)
                        queue.append(dep)
        
        # 2. 构建依赖图
        graph = {f: [] for f in all_needed}
        in_degree = {f: 0 for f in all_needed}
        
        for feature in all_needed:
            if feature in self.feature_deps["features"]:
                deps = self.feature_deps["features"][feature].get("dependencies", [])
                for dep in deps:
                    if dep in all_needed:
                        graph[dep].append(feature)
                        in_degree[feature] += 1
        
        # 3. 拓扑排序
        queue = [f for f in all_needed if in_degree[f] == 0]
        result = []
        
        while queue:
            feature = queue.pop(0)
            result.append(feature)
            for neighbor in graph[feature]:
                in_degree[neighbor] -= 1
                if in_degree[neighbor] == 0:
                    queue.append(neighbor)
        
        # 4. 检查循环依赖
        if len(result) != len(all_needed):
            raise ValueError("Circular dependency detected!")
        
        return result
    
    def load_strategy_features(
        self,
        df: pd.DataFrame,
        strategy_name: str,
        fit: bool = True,
    ) -> pd.DataFrame:
        """
        为指定策略加载特征
        
        Args:
            df: 输入 DataFrame
            strategy_name: 策略名称（sr_reversal, sr_breakout, compression_breakout, trend_following）
            fit: 是否拟合（研究阶段=True，实盘阶段=False）
        
        Returns:
            df_with_features: 包含计算特征的 DataFrame
        """
        if strategy_name not in self.strategy_config["strategies"]:
            raise ValueError(f"Unknown strategy: {strategy_name}")
        
        strategy_config = self.strategy_config["strategies"][strategy_name]
        result_df = df.copy()
        
        # 1. 生成基础特征（如果需要）
        if strategy_config.get("base_feature_types"):
            print(f"   🔧 Generating base features ({strategy_config['base_feature_types']})...")
            engineer = ComprehensiveFeatureEngineer(
                feature_types=",".join(strategy_config["base_feature_types"])
            )
            result_df = engineer.engineer_all_features(result_df, fit=fit)
        
        # 2. 解析依赖，确定计算顺序
        requested_features = strategy_config["requested_features"]
        computation_order = self.resolve_dependencies(requested_features)
        
        print(f"   📊 Computing {len(computation_order)} features (including dependencies)...")
        
        # 3. 按顺序计算特征
        for feature_name in computation_order:
            if feature_name in self.computed_cache:
                continue  # 已计算，跳过
            
            if feature_name not in self.feature_deps["features"]:
                print(f"     ⚠️  Warning: Feature '{feature_name}' not found in dependencies config, skipping...")
                continue
            
            feature_info = self.feature_deps["features"][feature_name]
            
            # 检查所需列
            required_cols = feature_info.get("required_columns", [])
            missing_cols = [c for c in required_cols if c not in result_df.columns]
            if missing_cols:
                print(f"     ⚠️  Warning: Missing required columns for {feature_name}: {missing_cols}, skipping...")
                continue
            
            # 获取计算函数
            compute_func_name = feature_info["compute_func"]
            compute_func = get_compute_func(compute_func_name)
            
            # 获取计算参数
            compute_params = feature_info.get("compute_params", {})
            
            # 计算特征
            try:
                if compute_params:
                    result_df = compute_func(result_df, **compute_params)
                else:
                    result_df = compute_func(result_df)
                
                # 验证输出列是否存在
                output_cols = feature_info.get("output_columns", [feature_name])
                missing_output = [c for c in output_cols if c not in result_df.columns]
                if missing_output:
                    print(f"     ⚠️  Warning: Expected output columns {missing_output} not found after computing {feature_name}")
                else:
                    print(f"     ✅ Computed {feature_name}")
                
                # 缓存
                self.computed_cache[feature_name] = True
            except Exception as e:
                print(f"     ❌ Error computing {feature_name}: {e}")
                continue
        
        # 4. 只返回请求的特征列（以及它们的输出列）
        output_cols = []
        for feature_name in requested_features:
            if feature_name in self.feature_deps["features"]:
                feature_info = self.feature_deps["features"][feature_name]
                output_cols.extend(feature_info.get("output_columns", [feature_name]))
        
        # 保留原始列和计算的特征列
        all_cols = list(df.columns) + [c for c in output_cols if c in result_df.columns]
        return result_df[all_cols]
    
    def get_strategy_features(self, strategy_name: str) -> List[str]:
        """
        获取策略的特征列表（包括依赖）
        
        Args:
            strategy_name: 策略名称
        
        Returns:
            feature_list: 特征列表（包括依赖）
        """
        if strategy_name not in self.strategy_config["strategies"]:
            raise ValueError(f"Unknown strategy: {strategy_name}")
        
        requested_features = self.strategy_config["strategies"][strategy_name]["requested_features"]
        return self.resolve_dependencies(requested_features)
```

## 方案对比

### 方案 2 + 方案 4（原方案）

**优点：**
- ✅ 类型安全（函数直接引用）
- ✅ IDE 支持好
- ✅ 可以在代码中验证依赖关系

**缺点：**
- ⚠️ 需要维护 Python 注册表
- ⚠️ 配置分散（Python + YAML）

### 纯配置文件方案（新方案）

**优点：**
- ✅ **更简单**：只需要配置文件
- ✅ **配置集中**：所有配置都在 YAML 中
- ✅ **易于修改**：修改 YAML 即可
- ✅ **完全分离**：配置和代码完全分离

**缺点：**
- ⚠️ 需要函数映射表（但很小，只需维护一次）
- ⚠️ 配置文件可能变得很大（但可以分模块）

## 最终推荐

### 推荐：纯配置文件方案（更简单）

**理由：**
1. ✅ **更简单**：只需要配置文件，不需要 Python 注册表
2. ✅ **配置集中**：所有配置都在 YAML 中，易于管理
3. ✅ **易于修改**：修改 YAML 即可，不需要改代码
4. ✅ **完全分离**：配置和代码完全分离

**实现步骤：**
1. 创建 `feature_dependencies.yaml`（特征依赖配置）
2. 创建 `strategy_features.yaml`（策略特征配置）
3. 创建 `feature_function_mapping.py`（函数映射表，只需维护一次）
4. 实现 `StrategyFeatureLoader`（特征加载器）

**预计时间：3-5天**（比原方案更快）

## 总结

**用户的建议是对的！** 纯配置文件方案更简单，更适合当前需求。不需要构建 Python 注册表，直接在配置文件中定义依赖关系即可。
