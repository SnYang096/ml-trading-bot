# 组合特征分类说明

## 问题：交互特征和通用组合特征有什么区别？

### 答案：它们都是"组合特征"，只是组合方式不同

我已经将它们统一到 `utils_interaction_features.py` 中，分为两类：

## 两类组合特征

### 1. **交互特征（Interaction Features）**
**定义**：两个特征的**乘积**（状态 × 动量）

**示例**：
- `vpin_x_compression` = `vpin` × `compression_energy`
- `liquidity_void_x_wpt_risk` = `liquidity_void_detected` × `wpt_false_breakout_risk`
- `vpin_x_wick_upper` = `vpin` × `wick_upper_ratio`

**设计原则**：
- 状态 × 动量：环境特征 × 行为特征
- 风险 × 强度：脆弱性指标 × 趋势强度
- 多时间尺度融合：短周期信号 × 长周期 regime

**用途**：捕捉"在XX环境下，YY信号是否可靠"的高阶逻辑

### 2. **衍生特征（Derived Features）**
**定义**：单个特征的变换或两个特征的**其他运算**（比值、差值、归一化等）

**示例**：
- `sr_distance_normalized` = `dist_to_nearest_sr` / `atr`（比值）
- `dist_to_zz_high` = `abs(close - zz_high_value)`（差值）
- `cvd_slope_5` = `cvd` 的滚动窗口斜率（变换）
- `atr_ratio` = `atr` / `close`（比值）
- `compression_score` = `1 / (1 + bb_width_ratio)`（变换）

**用途**：归一化、距离计算、滚动统计等

## 统一后的结构

### 文件组织

```
src/features/time_series/utils_interaction_features.py
├── 交互特征（两个特征的乘积）
│   ├── compute_liquidity_void_x_wpt_risk
│   ├── compute_compression_energy_x_ofi_short
│   ├── compute_vpin_x_compression
│   └── ...
└── 衍生特征（单个特征的变换或两个特征的其他运算）
    ├── compute_sr_strength_combined
    ├── compute_sr_distance_normalized
    ├── compute_dist_to_zz_high
    ├── compute_cvd_slope
    └── ...
```

### 配置文件定义

```yaml
# config/feature_dependencies.yaml
features:
  # 交互特征（两个特征的乘积）
  vpin_x_compression:
    compute_func: compute_vpin_x_compression
    dependencies: ["vpin_features", "wpt_volume_energy"]
    ...
  
  # 衍生特征（单个特征的变换或两个特征的其他运算）
  sr_distance_normalized:
    compute_func: compute_sr_distance_normalized
    dependencies: ["atr"]
    ...
  
  cvd_slope_5:
    compute_func: compute_cvd_slope
    dependencies: []
    ...
```

### 策略配置使用

```yaml
# config/strategies/sr_reversal/features.yaml
requested_features:
  # 交互特征（按需选择）
  - vpin_x_wick_upper
  - vpin_x_wick_lower
  
  # 衍生特征（按需选择）
  - sr_strength_combined
  - sr_distance_normalized
  - cvd_slope_5
  - atr_ratio
```

## 为什么统一？

1. **都是组合特征**：都是基于已有特征计算的新特征
2. **都支持按需加载**：可以在 `requested_features` 中按需选择
3. **都支持并行计算**：系统会自动处理依赖和并行
4. **都支持缓存**：计算结果会被缓存

**区别只是组合方式**：
- 交互特征 = 乘积（状态 × 动量）
- 衍生特征 = 其他运算（比值、差值、变换）

## 总结

| 项目 | 交互特征 | 衍生特征 |
|------|---------|---------|
| **组合方式** | 两个特征的乘积（×） | 单个特征的变换或两个特征的其他运算（/、-、abs、rolling等） |
| **设计目的** | 捕捉"环境 × 行为"的高阶逻辑 | 归一化、距离计算、滚动统计 |
| **文件位置** | `utils_interaction_features.py` | `utils_interaction_features.py`（已合并） |
| **配置文件** | `feature_dependencies.yaml` | `feature_dependencies.yaml` |
| **是否公用** | ✅ 是（所有策略可用） | ✅ 是（所有策略可用） |
| **按需加载** | ✅ 是（在 requested_features 中选择） | ✅ 是（在 requested_features 中选择） |

**关键点**：
- 两类特征已经统一到同一个文件中
- 都是公用的，所有策略都可以使用
- 都支持按需加载、并行计算、缓存
- 区别只是组合方式不同（乘积 vs 其他运算）
