# 扩展波动率特征说明

## lag_periods 参数与输出特征的关系

### 配置参数
```yaml
extended_volatility_features:
  compute_params:
    window: 20
    lag_periods: [1, 2, 3, 5, 10]  # 滞后周期列表
```

### 生成的滞后特征
`lag_periods: [1, 2, 3]` 会生成以下特征：
- `vol_lag_1` - 1期前的波动率（最重要，捕捉短期波动率聚集性）
- `vol_lag_2` - 2期前的波动率（有用，捕捉中期波动率记忆）
- `vol_lag_3` - 3期前的波动率（有用，捕捉中期波动率记忆）

**优化说明**：
- 已精简为只保留短期滞后（1-3期），降低过拟合风险
- 长期滞后（5-10期）理论支撑较弱，可能冗余
- 详见 `滞后特征优化建议.md`

### 代码实现
```python
# 3. Lag features of volatility
vol_base = returns.rolling(window=window, min_periods=window//2).std()
for lag in lag_periods:
    result[f"vol_lag_{lag}"] = vol_base.shift(lag)
```

### 完整输出特征列表

#### 1. 历史波动率特征（4个）
- `vol_historical_5` - 5期滚动标准差
- `vol_historical_10` - 10期滚动标准差
- `vol_historical_20` - 20期滚动标准差
- `vol_historical_60` - 60期滚动标准差

#### 2. ATR相关特征（17个）
- `vol_atr_price_ratio` - ATR/Price比率
- `vol_atr_ma_5`, `vol_atr_ma_10`, `vol_atr_ma_20` - ATR移动平均
- `vol_atr_std_5`, `vol_atr_std_10`, `vol_atr_std_20` - ATR标准差
- `vol_atr_max_5`, `vol_atr_max_10`, `vol_atr_max_20` - ATR最大值
- `vol_atr_min_5`, `vol_atr_min_10`, `vol_atr_min_20` - ATR最小值
- `vol_atr_ratio_ma20` - ATR相对于20期均值的比率
- `vol_atr_change` - ATR变化率
- `vol_atr_change_abs` - ATR绝对变化

#### 3. 滞后特征（3个）⭐
- `vol_lag_1` - 1期前的波动率（最重要）
- `vol_lag_2` - 2期前的波动率
- `vol_lag_3` - 3期前的波动率

**注意**：已精简为只保留短期滞后，降低过拟合风险。长期滞后（5-10期）已移除。

#### 4. 趋势特征（4个）
- `vol_trend_slope_5` - 5期波动率趋势斜率
- `vol_trend_slope_10` - 10期波动率趋势斜率
- `vol_trend_slope_20` - 20期波动率趋势斜率
- `vol_acceleration` - 波动率加速度

#### 5. 移动平均特征（6个）
- `vol_ma_5`, `vol_ma_10`, `vol_ma_20` - 波动率简单移动平均
- `vol_ema_5`, `vol_ema_10`, `vol_ema_20` - 波动率指数移动平均

#### 6. Regime特征（2个）
- `vol_zscore` - 波动率Z-score
- `vol_percentile_rank` - 波动率百分位排名

#### 7. 范围特征（4个）
- `vol_range_10`, `vol_range_20` - 波动率范围
- `vol_range_ratio_10`, `vol_range_ratio_20` - 波动率范围比率

#### 8. 动量特征（3个）
- `vol_momentum_3` - 3期波动率动量
- `vol_momentum_5` - 5期波动率动量
- `vol_momentum_10` - 10期波动率动量

### 总计
**共43个特征**（已优化），其中：
- 滞后特征：3个（由 `lag_periods` 参数控制，已精简）
- 其他特征：40个（固定生成）

**优化说明**：
- 原配置有45个特征（5个滞后特征）
- 现精简为43个特征（3个滞后特征）
- 降低过拟合风险，保持模型简洁性

### 如何验证特征是否生成

1. **检查配置**：确认 `output_columns` 中包含所有 `vol_lag_*` 特征
2. **检查数据**：加载特征后，查看 DataFrame 的列名
3. **代码验证**：
```python
# 检查滞后特征
lag_cols = [col for col in df.columns if col.startswith('vol_lag_')]
print(f"滞后特征: {lag_cols}")
# 应该输出: ['vol_lag_1', 'vol_lag_2', 'vol_lag_3', 'vol_lag_5', 'vol_lag_10']
```

### 注意事项

1. **参数传递**：`lag_periods` 通过 `compute_params` 传递，确保配置正确
2. **特征命名**：滞后特征命名格式为 `vol_lag_{lag_period}`
3. **修改滞后周期**：如果要修改滞后周期，只需修改 `lag_periods` 列表，特征会自动生成

### 示例：修改滞后周期

如果想使用不同的滞后周期，例如 `[1, 2, 3, 5]`：

```yaml
extended_volatility_features:
  compute_params:
    lag_periods: [1, 2, 3, 5]  # 修改这里
```

会生成：
- `vol_lag_1`, `vol_lag_2`, `vol_lag_3`, `vol_lag_5`

**注意**：
1. 同时需要更新 `output_columns` 列表，添加新的特征名称
2. 建议只保留短期滞后（1-3期），长期滞后可能过拟合
3. 详见 `滞后特征优化建议.md` 了解详细分析
