# 特征定义位置说明

## 问题：interaction_features 为什么不是公用的？

### 当前情况

`interaction_features` 定义在 `config/feature_dependencies.yaml` 中，这是**全局特征定义文件**，所有策略都可以使用。

### 特征定义的位置

#### 1. **全局特征定义** (`config/feature_dependencies.yaml`)
- ✅ **所有策略共享**
- ✅ **自动并行计算和缓存**
- ✅ **按需加载**

**包含的特征：**
- 基础技术指标（ATR, RSI, MACD 等）
- WPT 特征
- Hilbert 特征
- Hurst 特征
- VPVR 特征
- **VPIN 特征**（`vpin_features`）
- **交互特征**（`interaction_features`）← **这里是公用的！**

#### 2. **策略特定配置** (`config/strategies/{strategy}/features.yaml`)
- 定义该策略需要哪些特征
- 定义 post_processors（组合和衍生特征）
- 定义特征选择器

**示例：**
```yaml
# config/strategies/sr_reversal/features.yaml
feature_pipeline:
  requested_features:
    - sr_strength_max
    - vpin_features        # ← 使用全局定义的 vpin_features
    - interaction_features # ← 使用全局定义的 interaction_features
  post_processors:
    - module: src.time_series_model.strategies.sr_reversal.features
      function: build_sr_reversal_features  # ← 策略特定的组合特征
```

## 工作流程

### 之前（在 post_processors 中处理）

```python
# build_sr_reversal_features (post_processor)
def build_sr_reversal_features(df):
    # 1. 计算基础特征（顺序执行，无并行）
    df = extract_wpt_features(df, ...)
    df = extract_hilbert_features(df, ...)
    df = extract_order_flow_features(df, ...)
    df = extract_interaction_features(df, ...)  # ← 在这里调用
    
    # 2. 组合和衍生特征
    # ...
    
    return df
```

**问题：**
- ❌ 顺序执行，无并行
- ❌ 无缓存
- ❌ 总是计算所有特征

### 现在（在 requested_features 中定义）

```yaml
# features.yaml
requested_features:
  - vpin_features        # ← 系统自动计算（并行+缓存）
  - interaction_features # ← 系统自动计算（并行+缓存）

post_processors:
  - build_sr_reversal_features  # ← 只做组合和衍生
```

**流程：**
1. 系统读取 `requested_features`
2. 自动解析依赖关系（`interaction_features` 依赖 `vpin_features` 等）
3. 按层级分组，同层并行计算
4. 自动缓存结果
5. 最后执行 `post_processors`（只做组合和衍生）

**优势：**
- ✅ 自动并行计算
- ✅ 自动缓存
- ✅ 按需加载
- ✅ 任何策略都可以使用 `interaction_features`

## 如何让其他策略也使用 interaction_features？

### 方法 1：在策略配置中添加（推荐）

```yaml
# config/strategies/sr_breakout/features.yaml
feature_pipeline:
  requested_features:
    - sr_strength_max
    - vpin_features        # ← 使用全局定义
    - interaction_features # ← 使用全局定义
```

### 方法 2：在 post_processors 中调用（不推荐）

```python
# 不推荐：会失去并行和缓存优势
def build_sr_breakout_features(df):
    df = extract_interaction_features(df, ...)  # ← 顺序执行
    return df
```

## 总结

| 项目 | 位置 | 是否公用 | 说明 |
|------|------|---------|------|
| `vpin_features` | `feature_dependencies.yaml` | ✅ 是 | 所有策略可用 |
| `interaction_features` | `feature_dependencies.yaml` | ✅ 是 | 所有策略可用 |
| `build_sr_reversal_features` | `sr_reversal/features.py` | ❌ 否 | 策略特定的组合特征 |

**关键点：**
- `interaction_features` **已经是公用的**，定义在全局配置文件中
- 任何策略都可以在 `requested_features` 中添加 `interaction_features`
- 系统会自动处理依赖、并行计算和缓存
- `post_processors` 只应该做策略特定的组合和衍生特征
