# ME Rolling Sim 对比报告：`20260413_130313` vs `20260412_112742`

**对比目的**：评估在「宽止损 + SHAP 保护更多与假动量相关的特征节点」之后，ME 慢滚动 `rolling_sim` 的 OOS 表现相对此前锁定基线的变化。

**旧基线 run**：`results/me/research_roll.features_on/_rolling_sim/20260412_112742`  
**新 run**：`results/me/research_roll.features_on/_rolling_sim/20260413_130313`

**新 run 相对旧基线的主要配置差异**（见 `config/prod_train_pipeline_2h_slow_me_only.yaml` 已入库版本）：

1. **执行层**：`event_backtest.sym_r_by_family.me` 由 `"2.0:1:6.0"` 调整为 **`"3.0:1:6.0"`**（提高 `initial_r` 下沿，允许更宽初始止损）。
2. **特征层**：`shap_feature_selection.protected_nodes` 在 `atr_f` 之外增加  
   **`bpc_soft_phase_f`**（含 BB 压缩、回撤/持续等 BPC 软相位列）、**`footprint_basic_f`**（含 `fp_delta_poc` 等）、**`vpin_features_f`**（含 `vpin_trend` 等），避免 SHAP 将假动量相关信号剪到只剩 2～3 维。
3. **风控**：`config/constitution/constitution.yaml` 中的 kill switch / 日开仓节流等（若与 ME 同仓验证）仍按仓库当前版本生效；本报告以 rolling 产物为准。

## 4.  结论（工程视角）

1. **整段 stitched**：新配置在 **总 R（+155.96 vs +99.48）与交易数** 上优于旧基线，可作为当前 ME 慢滚动的**主推配置**（与 tag 一致）。
2. **2024-11**：从亏损转为盈利，**优先满足此前「假动量月」的改进目标**。
3. **分月波动**：部分月份（如 2023-12、2024-01、2024-03、2024-10）新 run 弱于旧 run，建议后续用 **分月 funnel / 阈值漂移** 做针对性复查，而不是单次 stitched 即宣告所有月份单调改进。
