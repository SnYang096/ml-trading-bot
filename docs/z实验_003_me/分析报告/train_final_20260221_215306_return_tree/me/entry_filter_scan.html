🔬 边际贡献分析 (Marginal Contribution)

边际交易 = 该 filter 触发但其他所有 filter 都未触发的交易。
如果边际 snotio ≈ baseline 且 stop_rate ≈ baseline → 该 filter 没有独立过滤能力，不值得启用。

🔍 Loss Rate / Stop Rate 分析

Loss Rate = 亏损交易占比 (R < 0)。Stop Rate = 触发止损占比 (R ≈ -SL)。
好的 Entry Filter 应同时降低两者。如果 snotio 提升但 stop_rate 不变 → filter 只是挑 easy trade（伪改进）。

🔍 Worst-10% Trade 分析

Entry Filter 的真正功劳 = 把最烂的交易赶出去。
如果 worst-10% 明显改善但 snotio 不动 → filter 在精准狙击坏 trade。
如果 worst-10% 不动但 snotio 变好 → filter 可能在挑 easy trade（警惕过拟合）。
层级 KPI 定位: Entry Filter → snotio / loss_rate / stop_rate | Evidence → failure rate / drawdown | Execution → Sharpe / Calmar / MDD | 全系统 → OOS Sharpe
