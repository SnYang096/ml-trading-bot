## 产品决策（当前）

- **先不改配置**：不把斜率硬性并进 `direction.yaml` / Gate，等业务上要先验证 fat-tail 捕获链路。
- **动机**：微观入场过滤对 **单笔分布** 可能有帮助，但对 **肥尾行情能否出现、能否骑住** 通常 **边际更小**；优先级放在「有没有大趋势 / 极端行情」与仓位、PCM、加仓与风控上更划算。

上述取舍 **大体正确**：fat-tail PnL 往往来自少数大波段；入场若仅过滤「均线走平阶段的线上多头」，对尾部总量的改变可能有限（需事件回测量化后才能精确说「差别多大」）。

## 统计结论摘要（120T，与生产 EMA1200 语义对齐）

### 多头（条件：`ema_pos > 0`）

- **`slope > 0` 相对 `slope ≤ 0`**：前瞻 spot 收益 **更高**，且 **horizon 越长差距越大**（5d～60d 样本：BTC/ETH/SOL pooled）。
- **`slope ≤ 0` 且仍在均线上方** 的 bar **很少**，多为「线上但慢均线已走平/拐头」——更像噪声区而非延续段。
- **含义**：若将来要做「多头斜率过滤」，统计上与「延续」叙事一致；代价是触发更少，需回测权衡。

### 空头（条件：`ema_pos < 0`）

- **不可照搬多头结论**。同一样本里：`slope < 0` 区间的 spot 前瞻 **不低于** `slope ≥ 0`（甚至更高），与「裸空 timing」的简单直觉不一致。
- **原因**：crypto 长样本 regime、现货前瞻 ≠ 永续空单 PnL、反弹段混杂等。
- **含义**：空单是否加斜率、如何加，必须以 **带符号持仓回测**（event_backtest / trades 分层）为准，不能单靠本条 spot 切片。
