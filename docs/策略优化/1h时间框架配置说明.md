# 1h 时间框架配置说明

## 概述

从 4h (240T) 切换到 1h (60T) 时间框架，主要需要调整以下参数：

## 参数调整对照表

| 参数 | 4h (240T) | 1h (60T) | 说明 |
|------|-----------|----------|------|
| `timeframe` | `240T` | `60T` | 数据重采样时间框架 |
| `max_holding_bars` | 50 | 200 | 保持相同的持仓天数（约 8.3 天） |
| `freq` (backtest.yaml) | `"4H"` | `"1H"` | 回测频率标识（可选，实际使用命令行参数） |
| VPIN tick 模式 | `auto` | `auto` | 自动启用（60T <= 120分钟） |

## 关键调整

### 1. max_holding_bars 计算

- **4h**: 50 bars × 4h = 200h ≈ 8.3 天
- **1h**: 200 bars × 1h = 200h ≈ 8.3 天

保持相同的持仓天数，确保策略逻辑一致。

### 2. VPIN 自动启用

根据 `SR_BASELINE_TICK_MODE ?= auto` 的逻辑：
- 60T (1h) <= 120分钟 → **自动启用真实 tick 数据**
- 无需手动设置 `--tick-data-mode on`

### 3. 特征计算

所有特征计算都是**时间框架无关**的：
- ATR、SR strength、SQS 等基于价格和成交量，不依赖时间框架
- 窗口大小（如 ATR window=14）表示 14 根 K 线，自动适配

### 4. 回测参数

`config/strategies/sr_reversal/backtest.yaml` 中的 `freq: "4H"` 主要用于标识，实际回测使用命令行传入的 `timeframe` 参数。

## 使用方法

### 方法 1: 通过 Makefile 变量覆盖

```bash
make ts-sr-reversal-rule-baseline SR_BASELINE_TIMEFRAME=60T
```

### 方法 2: 直接修改 Makefile 默认值

```makefile
SR_BASELINE_TIMEFRAME ?= 60T  # 改为 1h
```

### 方法 3: 使用专门的 1h 测试命令

```bash
make ts-sr-reversal-1h-baseline
```

## 预期效果

切换到 1h 后，预期会看到：

1. **更多交易信号**：1h 时间框架会产生更多 K 线，SR 接触机会增加
2. **更频繁的交易**：相同时间段内，1h 的交易次数约为 4h 的 4 倍
3. **更细粒度的止损/止盈**：1h 的 ATR 通常小于 4h，止损/止盈距离更小

## 注意事项

1. **数据量增加**：1h 数据量是 4h 的 4 倍，特征计算和回测时间会相应增加
2. **参数调优**：可能需要重新优化 `max_holding_bars`、`stop_loss_r`、`take_profit_r` 等参数
3. **VPIN 计算**：确保 tick 数据可用，否则 VPIN 会报错退出
