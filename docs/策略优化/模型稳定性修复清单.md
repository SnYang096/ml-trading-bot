# 模型稳定性修复清单

## 问题
每次训练完测试，结果都不一样，ML模型不稳定。

## 已实施的修复 ✅

### 1. 固定所有随机种子

#### ML模型（分类）
```python
params = {
    ...
    "random_state": 42,
    "bagging_seed": 42,
    "feature_fraction_seed": 42,
    "data_random_seed": 42,
}
```

#### 波动率模型（回归）
```python
# 在 train_volatility_model 中

    model.params["random_state"] = 42

    model.params["bagging_seed"] = 42

    model.params["feature_fraction_seed"] = 42

    model.params["data_random_seed"] = 42
```

#### 全局随机种子
```python

```

如果修复成功，两次运行应该：
- ✅ AUC 完全相同

- ✅ 预测值分布完全相同

## 如果仍然不稳定

### 可能的原因

1. **数据文件变化**
   - 检查数据文件是否每次加载都相同
   - 检查数据文件是否有时间戳或版本差异

2. **特征计算顺序**
   - 检查特征依赖关系是否稳定
   - 检查特征计算是否有并行化导致的随机性

3. **模型初始化**
   - 检查 LightGBMTrainer 内部是否有其他随机性
   - 检查是否有其他库（如 sklearn）的随机性

4. **数据预处理**
   - 检查是否有数据清洗步骤引入了随机性
   - 检查是否有缺失值处理引入了随机性
