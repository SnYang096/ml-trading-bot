# 未来波动率标签问题修复说明

## 问题现象

从模型对比日志看到：
```
Predicted mean: 0.975, Actual mean: 0.000
```

实际波动率均值显示为0，但调试信息显示：
```
Mean: 0.00660396
Median: 0.00627368
```

## 根本原因

**单位转换错误**！

### 问题分析

1. **`future_volatility_label` 返回的是什么？**
   - 返回的是**相对波动率**（RMS of returns）
   - 例如：0.0066 = 0.66% 的相对波动率
   - 计算公式：`RMS = sqrt(mean(returns^2))`

2. **预测波动率是什么？**
   - `pred_vol_relative` 是相对波动率（例如0.0075 = 0.75%）
   - `pred_vol = pred_vol_relative * prices` 转换为绝对波动率（例如0.0075 * 100000 = 750）

3. **错误的比较方式**
   ```python
   # ❌ 错误：直接把相对波动率除以ATR
   future_vol_relative = future_vol_valid / (atr_valid + 1e-8)
   # future_vol_valid = 0.0066 (相对波动率)
   # atr_valid = 1200 (绝对价格)
   # future_vol_relative = 0.0066 / 1200 = 0.0000055 ≈ 0
   ```

4. **正确的比较方式**
   ```python
   # ✅ 正确：先转换为绝对波动率，再除以ATR
   future_vol_absolute = future_vol_valid * prices_valid  # 0.0066 * 100000 = 660
   future_vol_relative = future_vol_absolute / (atr_valid + 1e-8)  # 660 / 1200 = 0.55
   ```

## 修复方案

### 修复前（错误）

```python
# 错误的单位转换
pred_vol_relative = pred_vol_valid / (atr_valid + 1e-8)
future_vol_relative = future_vol_valid / (atr_valid + 1e-8)  # ❌ 错误：future_vol_valid是相对波动率
```

### 修复后（正确）

```python
# 正确的单位转换
# future_vol_valid 是相对波动率，需要先转换为绝对波动率
prices_valid = df_temp.loc[df_temp.index[valid_mask], "close"].values
future_vol_absolute = future_vol_valid * prices_valid  # 转换为绝对波动率
future_vol_relative = future_vol_absolute / (atr_valid + 1e-8)  # 转换为相对ATR的比率

# pred_vol_valid 已经是绝对波动率
pred_vol_relative = pred_vol_valid / (atr_valid + 1e-8)
```

## 验证

修复后，应该看到：
- `Actual mean` 不再是0，而是合理的值（例如0.5-1.5之间）
- `Actual median` 也不再是0
- 预测误差（MAE, RMSE）会变得更合理
- 相关性会提高（因为单位匹配了）

## 总结

**问题根源**：单位不匹配
- `future_volatility_label` 返回相对波动率（RMS of returns）
- 但比较时直接除以ATR（绝对价格），导致结果接近0

**解决方案**：先转换为绝对波动率，再除以ATR
- `future_vol_absolute = future_vol_relative * prices`
- `future_vol_relative_to_atr = future_vol_absolute / ATR`
