> **重要**：`mlbot nnmultihead` 的 `train` / `predict` / `eval` / `build-feature-store` / `pipeline-3action-e2e` / `feature-group-search` 等子命令 **必须** 传 `--task-spec`（TaskSpec v1），不再支持仅传 `--config` 指向 `config/nnmultihead/...` 目录。仓库示例：`config/tasks/minimal_path_primitives_task_spec.yaml`。下文若仍出现 `--config .../path_primitives_...`，请改为等价的 `--task-spec` + 可选 `--base-config` 覆盖。

## 核心命令组

### nnmultihead

NN 多头模型（Path Primitives）相关命令。

#### `train` ⭐⭐⭐
**训练 NN 多头模型**

```bash
mlbot nnmultihead train --no-docker \
  --task-spec config/tasks/minimal_path_primitives_task_spec.yaml \
  --symbols BTCUSDT,ETHUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2024-12-31 \
  --feature-store-root feature_store \
  --feature-store-layer nnmh_tree_union_all_240T_v2 \
  --output-dir results/nnmultihead/my_run_name
```

**产物**: `model.pt`, `report.html`, `metrics.json`

---

#### `predict` ⭐⭐⭐
**模型推理：产出 primitives 预测**

```bash
mlbot nnmultihead predict --no-docker \
  --task-spec config/tasks/minimal_path_primitives_task_spec.yaml \
  --symbols BTCUSDT,ETHUSDT \
  --timeframe 240T \
  --start-date 2025-07-01 \
  --end-date 2025-12-31 \
  --model results/nnmultihead/my_run_name/model.pt \
  --output results/nnmultihead/my_run_name/preds_2025H2 \
  --feature-store-root feature_store \
  --feature-store-layer nnmh_tree_union_all_240T_v2
```

**产物**: `preds_*.parquet` (包含 dir_prob, mfe, mae, t_to_mfe)

---

#### `pipeline-3action-e2e` ⭐⭐⭐
**一键主链路：完整流程**

```bash
mlbot nnmultihead pipeline-3action-e2e --no-docker \
  --task-spec config/tasks/minimal_path_primitives_task_spec.yaml \
  --symbols BTCUSDT,ETHUSDT \
  --timeframe 240T \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --model results/nnmultihead/my_run_name/model.pt \
  --returns-source rr_execution \
  --out results/nnmh_e2e/my_run
```

**作用**: 自动执行 `predict → mode-3action → build-logs → run-e2e`

### visualize

可视化命令。

#### `feature-indicators` ⭐
**生成特征指标可视化**（用于检查数据区间与特征可用性；特征类型由 `config/visualization/feature_indicators.yaml` 控制）

- **不加 `--strategy-config`**：只加载原始 OHLCV，报告里 Hurst/Hilbert/WPT/Spectral 等会显示「0 列 / Not found」（因为原始数据没有这些列）。
- **加 `--strategy-config`**：会跑该策略的特征管道（或从 Feature Store 读），报告里会显示该策略下的 Hurst/Hilbert/WPT/Spectral 等特征列，并可用于检查数据/特征是否正确。

```bash
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-12-31 \
  --strategy-config config/strategies/compression_breakout \
  --use-cache --force-rebuild

选项	说明
--use-cache	使用 FeatureStore 缓存（默认不使用，每次新算）
--force-rebuild	强制删除现有缓存并重算（需配合 --use-cache）

# 仅原始数据（Hurst/Hilbert/WPT/Spectral 会显示 Not found）
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T

# 指定日期范围
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-05-31

# 按策略跑特征管道，报告里会显示 Hurst/Hilbert/WPT/Spectral 等（需该策略 features 里包含对应特征）
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-05-31 \
  --strategy-config config/strategies/compression_breakout
```

输出：`results/feature_indicators/BTCUSDT_240min_feature_indicators_*.html`，用浏览器打开即可查看。报告内除状态表外，还会为每种有数据的特征类型生成**时间序列图**（Hurst、Hilbert、WPT、Spectral 等），可在配置 `config/visualization/feature_indicators.yaml` 的 `display.show_plots` 中开关。
