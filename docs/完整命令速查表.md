## 核心命令组

### rule

规则路由命令（3-action: NO_TRADE/MEAN/TREND）。

#### `mode-3action` ⭐⭐⭐
**Router：把 preds → mode_3action.parquet**

```bash
mlbot rule mode-3action --no-docker \
  --preds results/nnmultihead/my_run_name/preds \
  --model results/nnmultihead/my_run_name/model.pt \
  --output results/nnmultihead/my_run_name/mode_3action.parquet \
  --mfe-min 0.125 \
  --eff-min 1.043 \
  --dir-conf-trend-min 0.064
```

**产物**: `mode_3action.parquet` (包含 NO_TRADE/MEAN/TREND 决策)

---

### research

**B 系统 / 树通道 R&D 统一入口**（① 假设筛查、draft 标定、显式 promote）。完整矩阵见 [`strategy/R&D工具矩阵_CN.md`](strategy/R&D工具矩阵_CN.md)。

#### `scan` ⭐⭐⭐
**条件 / 特征 label 效应筛查**

```bash
mlbot research scan condition-set --strategy tpc --layer gate \
  --parquet results/train_final/tpc/.../features_labeled.parquet \
  --label success_no_rr_extreme \
  --filter 'tpc_semantic_chop<=0.4' \
  --condition 'H: abs(ema_1200_position)>0.10' \
  --output results/rd_loop/tpc/quick_scan/regime.md

mlbot research scan feature-plateau --strategy tpc --feature tpc_pullback_depth \
  --operator '<=' --grid '0.5,0.6,0.7,0.75' --parquet ... --output ...
```

子模式：`condition-set` · `feature-plateau` · `pair-scan`

---

#### `ic` ⭐⭐⭐
**IC 衰减（树通道定 H）**

```bash
mlbot research ic --strategy fast_scalp --parquet ... \
  --features pulse_z,macd_atr,bb_width_normalized_pct \
  --horizons 1,3,5,10,20 --target forward_rr --output results/.../ic_decay.md
```

树通道亦可：`mlbot analyze factor-eval --ic-decay-lags ...`

---

#### `plateau` ⭐⭐⭐
**阈值平坦高原（label / snotio / lift）**

```bash
# Gate deny lift（B 系统 gate 精调）
mlbot research plateau --kpi lift --strategy tpc --layer gate \
  --parquet ... --feature tpc_semantic_chop --operator gt \
  --grid '0.2,0.3,0.4,0.5' --output results/.../gate_lift.json

# Entry snotio（ME/SRB entry filter）
mlbot research plateau --kpi snotio --snotio-mode proxy \
  --strategy srb --layer entry --feature vol_persistence \
  --operator '<=' --grid '0,1,0.1' --parquet ... --output ...

# Prefilter / entry label proxy（默认 --kpi label）
mlbot research plateau --strategy tpc --feature pulse_z \
  --operator '<=' --grid '0,1,0.1' --parquet ... --output ...
```

### visualize

可视化命令。

#### `feature-indicators` ⭐
**生成特征指标可视化**（用于检查数据区间与特征可用性；特征类型由 `config/visualization/feature_indicators.yaml` 控制）

- **不加 `--strategy-config`**：只加载原始 OHLCV，报告里 Hurst/Hilbert/WPT/Spectral 等会显示「0 列 / Not found」（因为原始数据没有这些列）。
- **加 `--strategy-config`**：会跑该策略的特征管道（或从 Feature Store 读），报告里会显示该策略下的 Hurst/Hilbert/WPT/Spectral 等特征列，并可用于检查数据/特征是否正确。

```bash
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-12-31 \
  --strategy-config config/strategies/compression_breakout \
  --use-cache --force-rebuild

选项	说明
--use-cache	使用 FeatureStore 缓存（默认不使用，每次新算）
--force-rebuild	强制删除现有缓存并重算（需配合 --use-cache）

# 仅原始数据（Hurst/Hilbert/WPT/Spectral 会显示 Not found）
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T

# 指定日期范围
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-05-31

# 按策略跑特征管道，报告里会显示 Hurst/Hilbert/WPT/Spectral 等（需该策略 features 里包含对应特征）
mlbot visualize feature-indicators --no-docker \
  --symbol BTCUSDT \
  --timeframe 240T \
  --start-date 2023-01-01 \
  --end-date 2025-05-31 \
  --strategy-config config/strategies/compression_breakout
```

输出：`results/feature_indicators/BTCUSDT_240min_feature_indicators_*.html`，用浏览器打开即可查看。报告内除状态表外，还会为每种有数据的特征类型生成**时间序列图**（Hurst、Hilbert、WPT、Spectral 等），可在配置 `config/visualization/feature_indicators.yaml` 的 `display.show_plots` 中开关。
