# 系统每层假设与实盘监控

> 创建时间: 2026-02-20
> 核心理念: 监控假设是否成立，比直接监控 PnL 更早发现问题。
> 假设失效 = 需要调整对应层的信号。

---

## 总览

```
信号生成链:
  特征 → Prefilter → Gate → Evidence → Direction → Entry Filter → Execution Tier
                                                                        ↓
  PCM 仲裁: Regime 检测 → 优先级排序 → Slot 分配
                                                                        ↓
  宪法层: Kill-switch / DD 上限 / 风险预算
```

每层都有一个核心假设。假设成立 → 该层提供正向贡献。假设失效 → 该层变成噪声甚至负贡献。

---

## Layer 1: 特征层

| 项目 | 内容 |
|------|------|
| **假设** | 特征能有效区分"好机会"和"坏机会"，且在实盘数据上的分布与训练数据一致 |
| **失效信号** | ① 特征分布偏移（mean/std 超出训练期 2σ）② 特征 importance 排名大幅变化 ③ 新上线 symbol 的特征分布与训练集不同 |
| **监控指标** | `feature_drift_zscore`: 每日计算关键特征的 rolling z-score，报警阈值 ±3σ |
| **调整方向** | 重新计算 quantiles / 重训模型 / 排查数据源变化（交易所 API 改版等） |

---

## Layer 2: 语义预筛选 (Prefilter)

| 项目 | 内容 |
|------|------|
| **假设** | 预筛选条件能准确界定 archetype 适用场景，过滤掉的 bar 确实不含该 archetype 的 alpha |
| **失效信号** | ① 预筛选通过率骤降（市场状态变了，但筛选条件没跟上）② 被过滤掉的 bar 事后 R 反而高 ③ 筛选后样本量不足统计可信（< 1080） |
| **监控指标** | `prefilter_pass_rate`: 每日/周统计通过率，与回测期对比 |
| **调整方向** | 放宽/收紧阈值 / 重跑 Plateau 搜索 / 检查上游特征是否漂移 |

---

## Layer 3: Gate (结构性条件过滤)

| 项目 | 内容 |
|------|------|
| **假设** | Gate 通过的信号 bad rate 显著低于全量信号（Gate Lift > 1.0） |
| **失效信号** | ① Gate Lift 降至 < 1.0（Gate 不再区分好坏）② Gate 通过率过低（< 5%，错过太多机会）③ Gate 通过率过高（> 50%，失去筛选意义） |
| **监控指标** | `gate_pass_rate`, `gate_lift`（rolling 30天窗口） |
| **调整方向** | 重新优化 Gate 规则阈值 / 检查 Gate 依赖的特征是否漂移 |

---

## Layer 4: Evidence (信号质量评分)

| 项目 | 内容 |
|------|------|
| **假设** | Evidence score 与事后 R 正相关（高分信号更好）；分位数阈值在实盘中仍然有效 |
| **失效信号** | ① 高 evidence 信号的事后 R 不比低 evidence 好 ② 实盘 evidence 分布与回测差异大（quantile 偏移）③ evidence score 集中在某个窄区间（失去区分力） |
| **监控指标** | `evidence_r_correlation`（rolling），`evidence_quantile_drift` |
| **调整方向** | 重新计算 quantile / 重训 Evidence 模型 / 检查输入特征漂移 |

---

## Layer 5: Direction (方向判断)

| 项目 | 内容 |
|------|------|
| **假设** | 方向判断函数正确率 > 50%（好于随机），且方向错误时止损能兜底 |
| **失效信号** | ① 方向正确率 < 50%（系统性反向）② 某个 symbol 的方向正确率显著偏低 ③ long/short 胜率严重不对称（市场结构变了） |
| **监控指标** | `direction_accuracy`（per symbol, per archetype, rolling 30天） |
| **调整方向** | 检查方向判断依赖的特征 / 切换方向算法 / 检查 symbol 是否进入新的市场结构 |

---

## Layer 6: Entry Filter (入场过滤)

| 项目 | 内容 |
|------|------|
| **假设** | Entry Filter 通过的信号事后 R 优于未过滤信号 |
| **失效信号** | ① Entry Filter 过滤掉的信号事后 R 反而更高 ② 过滤率太高（> 70%，几乎不出信号） |
| **监控指标** | `entry_filter_lift`, `entry_filter_pass_rate`（rolling） |
| **调整方向** | 调整 Entry Filter 阈值 / 重新 Plateau 搜索 |

---

## Layer 7: Execution Tier (执行分层)

| 项目 | 内容 |
|------|------|
| **假设** | 高 evidence → 更激进的执行参数（更大 size / 更宽止损）是合理的，高 tier 的风险调整收益 ≥ 低 tier |
| **失效信号** | ① 高 tier 的平均 R 反而低于低 tier ② 高 tier 的 max DD 显著恶化 ③ tier 分布严重偏向某一端 |
| **监控指标** | `per_tier_mean_r`, `per_tier_win_rate`, `tier_distribution` |
| **调整方向** | 调整 tier evidence_min 阈值 / 调整 tier 参数（size_multiplier, stop_loss） |

---

## Layer 8: PCM 仲裁 (优先级 + Regime)

| 项目 | 内容 |
|------|------|
| **假设** | ① 条件严格性排序 (LV > FER > ME > BPC) 在冲突时选出更优信号 ② Regime 检测正确识别市场状态 ③ 冲突率保持低位 (< 10%) |
| **失效信号** | ① 被拒信号的事后 R 系统性高于被选信号 ② Regime 检测频繁误判（HIGH_VOL 实际不是高波动）③ 冲突率骤升（策略间不再互补） |
| **监控指标** | `pcm_conflict_rate`, `counterfactual_r`（被拒信号 R），`regime_accuracy` |
| **调整方向** | 调整优先级排序 / 调整 Regime 检测阈值 / 检查 archetype 语义边界是否模糊 |

---

## Layer 9: 宪法层 (Constitution)

| 项目 | 内容 |
|------|------|
| **假设** | ① 4% 日亏上限足以覆盖尾部风险 ② 20% max DD 是可接受的最大回撤 ③ 2 slot 限制足够分散风险 |
| **失效信号** | ① Kill-switch 频繁触发（策略本身有问题，不是极端事件）② 接近 DD 上限但未触发（阈值太宽松）③ 全部 slot 长期被一个 archetype 占满 |
| **监控指标** | `kill_switch_trigger_count`, `max_dd_distance`（当前 DD 离上限的距离），`slot_archetype_distribution` |
| **调整方向** | 收紧/放宽宪法参数 / 增加 slot 数 / 检查底层策略质量 |

---

## 实盘监控仪表板建议

### 日频检查 (每天收盘后)

| 检查项 | 数据源 | 告警条件 |
|--------|--------|---------|
| 特征漂移 | 实盘特征 vs 训练期统计 | 任何关键特征 z-score > 3 |
| Gate 通过率 | 实盘 Gate 日志 | 偏离回测期均值 > 2σ |
| Direction 准确率 | 实盘成交记录 | 7 日 rolling < 45% |
| PnL / DD | 宪法层 | 日亏 > 2% (预警) / 4% (kill-switch) |

### 周频检查 (每周末)

| 检查项 | 数据源 | 告警条件 |
|--------|--------|---------|
| Evidence 区分力 | 实盘高/低 evidence 交易的 R 对比 | 相关系数 < 0.05 |
| PCM 冲突分析 | PCM 日志 | 冲突率 > 15% 或被拒信号 R 系统性 > 被选 |
| Tier 分布 | 执行日志 | 某 tier 占比 > 80% |
| Prefilter 通过率 | 筛选日志 | 偏离回测期 > 30% |

### 月频检查 (每月末)

| 检查项 | 数据源 | 告警条件 |
|--------|--------|---------|
| 全层假设验证 | 累计实盘数据 | 任何层的 Lift < 1.0 |
| Regime 准确性 | 事后对比 regime vs 实际市场状态 | 误判率 > 30% |
| archetype 正交性 | 各 archetype 收益相关系数 | 相关系数 > 0.5（不再互补） |

---

## 假设失效 → 调整路径

```
假设失效
  ├─ 特征层 → 重算 quantile / 检查数据源
  ├─ Prefilter → 重搜阈值 / 放宽条件
  ├─ Gate → 重优化规则 / 更新特征
  ├─ Evidence → 重训模型 / 更新 quantile
  ├─ Direction → 切换算法 / 排查 symbol
  ├─ Entry Filter → 调整阈值
  ├─ Execution → 调整 tier 参数
  ├─ PCM → 调整优先级 / Regime 阈值
  └─ Constitution → 收紧风控参数
```

**核心原则**: 哪层假设先失效，就先调哪层。不要因为 PnL 下降就盲目重训所有模型。
