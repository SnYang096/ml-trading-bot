## 二、信号栈（4 层 + 方向过滤）

```
全量数据
    │
    ▼
Layer 0: Prefilter  ─── archetype 结构检测
    │  (人工假设 + bin sweep，2-4 条规则)
    │  目标: mean_rr_pass > mean_rr_all + 5%
    │  特征: 只用结构特征（趋势/突破/回踩/波动率）
    │  NOT: 订单流/funding/CVD（那是 Gate 的原料）
    │  通过率: 10-30%
    ▼
Layer 1: Direction Filter  ─── 确定性方向门
    │  Long config: direction = short → 跳过
    │  Short config: direction = long → 跳过
    │  作用: 提高置信度，不是提高准确率
    ▼
Layer 2: Gate  ─── 尾部风险删除
    │  LightGBM (label: rr_extreme < -0.8R)
    │  SHAP∩Gain 发现特征，统计验证规则
    │  目标: deny 那些"必然大亏"的交易
    │  deny 率: 10-40%
    ▼
Layer 3: Entry Filter  ─── 入场时机
    │  snotio = mean(R-multiples) 优化
    │  特征×阈值穷举搜索
    │  OR 组合 + plateau 稳定性验证
    ▼
Execution  ─── 参数优化
    initial_r / activation_r / trail_r
    MeanR 目标 >= 0.25R（覆盖手续费+滑点 0.18R）
```

## 三、为什么删除 Evidence

Evidence 在结构上必然失效，这不是 bug，是架构矛盾：

```
全量数据 (~15000 bars)
    │ Prefilter: 保留 archetype 区域
    │ Gate: 淘汰 80-95%（ME 只保留 6.5%）
    │ Entry Filter: 再淘汰 ~80%
    ▼
最终执行: ~0.7% 留存

Evidence 试图在 0.7% 里区分好坏
但 Gate 已用同源特征淘汰了所有"明显差"的信号
→ 剩余信号在 evidence 特征维度上是同质的
```

实测结果 (evidence单调性验证.md):
- Spearman r = -0.05，p = 0.48 → 与 R 无关
- 最低分 bin mean_R = +0.16，最高分 bin mean_R = -0.25 → 方向反转
- Gate 放行子集重训后：r = -0.068，更差
- 删除 Evidence → Sharpe 不变或微升

## 四、Long/Short 分配置设计

### 阶段一（立即可做）：方向作为硬过滤器

保持当前单一配置，仅在执行层加方向判断：
- direction=long → 只允许 long 入场
- direction=short → 只允许 short 入场（如未启用 short，则跳过）

### 阶段二（下轮迭代）：分方向 Prefilter + Gate

```
config/strategies/bpc/archetypes/
├── prefilter_long.yaml    # 向上突破结构特征
├── prefilter_short.yaml   # 向下突破结构特征
├── gate_long.yaml         # 多头尾部风险
├── gate_short.yaml        # 空头尾部风险
├── entry_filters_long.yaml
├── entry_filters_short.yaml
├── direction.yaml         # 确定性方向规则（只验证，不训练）
└── execution.yaml
```

优点：
- Long/short 各自发现最优特征
- 空头稳定性差时可直接 `enabled: false`
- 训练数据更纯净（长/短趋势的特征分布本质不同）

当前优先级: 阶段一先做（代码改动小），阶段二作为优化项。

## 五、各 Archetype 的 Prefilter 设计原则

### Prefilter 特征只能来自结构层

```
✅ 可用:            ❌ 不可用:
trend_strength      CVD / funding_rate
breakout_strength   OI_change
pullback_depth      vpin / footprint
atr_percentile      SHAP 自动发现的任意特征
volatility_regime
```

### BPC Prefilter（突破-回踩-延续）
```yaml
conditions:
  - bpc_volume_compression_pct >= 0.3   # 回踩时成交量压缩
  - price_position <= 0.9               # 价格未过度偏离
  # 候选: bpc_pullback_depth ∈ [0.2, 0.6]
  # 候选: bpc_bb_compression > 0.4（布林带压缩）
```

### ME Prefilter（动能扩张）
```yaml
conditions:
  - atr_percentile >= 0.85   # 高波动率（P85+，ATR 扩张中）
  # 候选: me_momentum_spike > 1.2
  # 候选: me_range_expansion > 1.4
```

### FER Prefilter（失败反转）
```yaml
conditions:
  any_of:
    - fer_trapped_longs >= 4.48    # 多头被困
    - fer_trapped_shorts >= 3.77   # 空头被困
```


## 八、Prefilter 训练目标：为什么用 >0.8R

用户的直觉正确：Prefilter 应该找"好的数据分布"而不是"坏的分布"。

```
Prefilter 问题: 这个 bar 属于这个 archetype 吗？
            → 评估: pass 区的 mean_rr 是否显著高于 deny 区
            → 标签: forward_rr（clipped at -2R/+2R，避免极端值主导）

Gate 问题: 这笔交易会不会大亏？
        → 评估: deny 区是否浓缩了 rr_extreme
        → 标签: rr_extreme（<-0.8R = True）
```

两者 **互补**，不互换：
- Prefilter 找好区间（正向筛选）
- Gate 排除坏交易（负向过滤）

## 九、稳定性机制

| 机制                        | 目的                             | 频率             |
| --------------------------- | -------------------------------- | ---------------- |
| SHAP 8-fold 稳定性筛选      | Gate 特征跨 regime 稳定          | 每次训练自动     |

| MeanR >= 0.25R 门槛         | 排除 activation_r 过小的退化参数 | Execution 优化时 |
