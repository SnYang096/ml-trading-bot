# activation_r 过小导致回测 Sharpe 虚高的原因分析

## 现象

Grid Search 结果中 Best = (initial_r=1.5, activation_r=0.5, trail_r=0.5)：
- WinRate=84.2%，Trades=2249，Sharpe=0.2415，Equity=$4168
- 看似最优，但实盘无法复现

## 根本原因：手续费与滑点吞噬期望收益

### 具体例子

ME 信号触发，入场价=100，ATR=2，参数 initial_r=1.5, activation_r=0.5, trail_r=0.5：

```
initial_sl = 100 - 1.5×2 = 97
activation 条件: 价格 >= 100 + 0.5×2 = 101
activation 后 trail_sl = HWM - 0.5×2 = HWM - 1
```

**回测记录（1min bar 模拟）：**
```
Bar 1: high=101.2 → activation，HWM=101.2，trail_sl=100.2，跳过本 bar SL
Bar 2: high=101.5, low=100.0 → HWM=101.5，trail_sl=100.5，low<trail_sl → 出场
盈利 = (100.5 - 100) / 2 = +0.25R  （实际 MeanR ≈ 0.14R，多次平均）
```

**实盘实际发生：**
```
1. 价格涨过 101 → 系统设置 STOP_MARKET stopPrice=100.2
2. 订单提交延迟 50~200ms，期间价格继续波动
3. 价格回落触碰 100.2 → 以市价成交
4. 实际成交价：100.1（滑点 0.1）
5. 实际盈利 = (100.1 - 100) / 2 = +0.05R（含滑点后）
```

### 实盘损耗量化（以 ATR=2，入场价=100 为例）

| 损耗来源                      | 量级（R）      |
| ----------------------------- | -------------- |
| 订单提交延迟导致的追价        | 0.05~0.15R     |
| 市价单滑点                    | 0.05~0.20R     |
| 手续费往返（0.04% taker × 2） | 约 0.08R       |
| **合计**                      | **0.18~0.43R** |

**期望盈利 0.14R < 损耗下限 0.18R → 期望收益为负。**

### 回测 Equity 虚高的计算

回测总收益：2249笔 × 0.1409R = **317R**

扣除实盘损耗（保守估计）：
- 手续费：2249 × 0.08R = **180R**
- 滑点：2249 × 0.05R = **112R**
- 实际净收益：317 - 180 - 112 = **+25R**（而非 $4168）

## 为什么 WinRate=84.2% 是退化信号

### 理论随机胜率

任何 trailing stop 策略，即使随机入场，其胜率下界为：

```
P(random_win) = initial_r / (initial_r + activation_r)
              = 1.5 / (1.5 + 0.5) = 75%
```

实际 WinRate=84.2%，比随机高 **9.2pp**。

这意味着 ME 的真实 alpha 只有 **9.2pp 的胜率提升**，不是 84.2%。
对比对称参数 (3.0, 2.0, 1.5) 的 WinRate=64.8%：
- 随机下界 = 3.0/(3.0+2.0) = 60%
- 实际超额 = 64.8% - 60% = **4.8pp**（但 MeanR=0.1444 更高，说明每次赢更多）

## 正确的参数选择标准

应优先看 **MeanR 足够覆盖实盘损耗**：

| 参数组          | MeanR   | 实盘最低损耗 | 净期望     |
| --------------- | ------- | ------------ | ---------- |
| (1.5, 0.5, 0.5) | 0.1409R | 0.18R        | **负**     |
| (3.0, 2.0, 1.5) | 0.1444R | 0.18R        | **接近 0** |
| 目标            | ≥ 0.3R  | 0.18R        | **正**     |

## 结论

当前 ME 策略的 MeanR 整体偏低（最高 0.1444R），实盘在手续费和滑点之后期望收益接近 0。

**后续行动：**
1. 用 `--sym-r` 运行对称约束的 Grid Search，在排除退化组合的前提下找最优 R 值
2. 若 MeanR 无法达到 ≥ 0.3R，需考虑从 Entry Filter 层提升信号质量（更高 evidence 阈值），而非在 Execution 层修补
3. 手续费已在 `optimize_event_execution.py` 中通过 `--fee-rate 0.0004` 默认开启，但未来跑对比时建议显式指定
