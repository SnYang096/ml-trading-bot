# 实验 06：ME 方向规则 A/B/C/D/E/F 对比 + 假动量特征识别

**日期**: 2026-04-13
**目标**: 找出 Nov 2024 假动量入场的根本原因，评估方向规则改进方案，挖掘可识别假动量的特征

---

## 1. 背景

ME 策略在 Nov 2024（大选行情）中开仓 7 笔全部止损亏损 (-13.96R)，错过了 Nov 5 日之后的大涨。
此前怀疑 `me_vwap_structure_bypass_direction`（bypass 规则）导致了弱势入场。

## 2. 方向规则 A/B/C/D/E/F 实验

### 2.1 实验设计

使用 `20260412_112742` 最优 rolling_sim 的校准策略，**仅替换 `direction.yaml`**，
在 4 个关键月份（Jul/Oct/Nov/Dec 2024）独立运行 event_backtest：

| 变体 | R1 级联模式 | Bypass | 描述 |
|------|------------|--------|------|
| **A** | first (原始) | 无 | 删除 bypass |
| **B** | first (原始) | MACD过滤 | bypass 需 macd_atr 同向 (dual_position_agree_deadband) |
| **C** | first (原始) | 原始 (无过滤) | **baseline（当前生产配置）** |
| **D** | **all (共识)** | 原始 | R1 需所有信号同向 + bypass 备路径 |
| **E** | **all (共识)** | 无 | 最严格：全信号共识，无 bypass |
| **F** | **all (共识)** | MACD过滤 | 共识 + MACD 确认 bypass |

`consensus_mode: all` 是本次新增代码：
- 文件: `src/time_series_model/live/generic_live_strategy.py` (scalar 路径)
- 文件: `src/time_series_model/live/direction_rule_ops.py` (vectorized 路径)
- 语义: signal_rules 中所有非零信号必须方向一致，否则返回 0

### 2.2 实验结果

```
变体                               Jul       Oct       Nov        Dec       合计
──────────────────────────────────────────────────────────────────────────────────
C (原始: cascade-first + bypass)  +29.0R    +6.2R    -14.0R    +41.7R    +63.0R  ← baseline
A (无bypass)                     +26.3R    +6.2R    -14.0R    +18.6R    +37.2R
B (bypass+MACD过滤)              +26.3R    +6.2R    -14.0R    +18.6R    +37.2R
D (consensus-all + bypass)       +29.0R    +6.2R    -16.7R    +41.7R    +60.3R
E (consensus-all, 无bypass)       +2.9R    +2.6R    +38.6R     +0.2R    +44.3R
F (consensus-all + bypass+MACD)  +30.7R    +2.6R     +3.5R     +9.0R    +45.8R
```

### 2.3 关键发现

1. **A ≡ B**：MACD 过滤 bypass 没有额外效果（bypass 有价值的时刻 MACD 恰好同向）

2. **Nov 的 7 笔亏损全部由 R1 主规则触发，而非 bypass**
   - R1 的级联用 "第一个非零信号" 逻辑
   - `me_accel_5k > 0` 直接触发 LONG，跳过了 `macd_atr`（此时为负）
   - 去掉 bypass (A) 后 Nov 结果完全相同

3. **bypass 是 ME 的核心价值**
   - Dec: bypass 让 BNB 在 12-02T10:00 入场（MACD 未确认前），贡献 +22.7R
   - Jul: bypass 让 XRP 更早入场，贡献额外 +2.7R
   - 删除 bypass (A) 导致 4 个月合计损失 -25.8R

4. **E (consensus only) 抓住了 Nov 大行情 (+38.6R)**
   - 全信号共识拦住了 Nov 1-3 号的弱入场
   - 等到 Nov 5-6 号 MACD 转正后才入场 BNB/ETH/SOL
   - 但代价是 Jul(-26R) 和 Dec(-41.5R) 严重损失

5. **根本矛盾无法通过方向规则解决**
   - Nov 需要等 MACD 确认 → consensus
   - Dec 需要 bypass 在 MACD 确认前入场 → first + bypass
   - 二者不可兼得

## 3. 假动量特征识别分析

### 3.1 方法

从特征库提取所有 33 笔新入场（Jan-Nov 2024）的 353 个特征值，
比较 Nov 假动量入场 (3 笔) vs 其他月盈利入场 (17 笔)：
- Cohen's d (效应量)
- Mann-Whitney U test (非参数检验)
- AUC (区分度)
- 逐阈值分析 (拦截率 + 误杀率)

### 3.2 关键发现：模型为什么看不到假动量

SHAP 剪枝后模型仅保留 3 个特征：`atr_f`, `macro_tp_vwap_1200_position_f`, `volatility_regime_f`
（从 85 个候选中剪到 3 个），模型的 evidence_score 对假动量入场给出 0.50（等同随机）。

### 3.4 最佳单特征过滤：`bpc_bb_compression > 0.3`

| 指标 | 值 |
|------|------|
| Nov 假动量拦截 | **3/3 (100%)** |
| 其他月盈利误杀 | 2/17 (12%)，仅 Apr BTC(+0.32R) 和 Aug ADA(+0.40R) |
| 其他月亏损拦截 | 7/13 (54%) |
| 净R提升 | +7.09R → **+16.62R (+9.53R)** |

**语义解释**：
```
真动量链路：  压缩 (BB窄) → 突破 → 加速 → 扩张 ✓
假动量链路：  波动放大 (BB宽) → 噪音加速 → 止损 ✗
```

Nov 1 入场时 bb_compression=0.15（极低，市场已在宽幅震荡），缺乏压缩突破支撑。

### 3.5 其他有效特征组合

## 4. 实施方案

将 top 3 假动量识别特征节点加入 SHAP `protected_nodes`：
```yaml
protected_nodes:
  - atr_f                       # 原有
  - bpc_soft_phase_f            # bpc_bb_compression（压缩度，假动量最强区分）
  - footprint_basic_f           # fp_delta_poc（订单流POC delta）
  - vpin_features_f             # vpin_trend（VPIN毒性流趋势）
```

预期效果：模型从 3 个特征扩展到 ~6-10 个，能学习 "低压缩 + 负POC delta + 高VPIN" → 低概率入场，
从而在 entry_quantile 阈值处自然过滤假动量信号。
