# 特征因果性与统计相关性分析

> **核心论点：不是所有"有效的特征"都是"真实的因果"。**
> 统计相关性随 regime 漂移，因果性在特定条件下也会退化为历史痕迹。
> 正确的姿态不是"选对特征"，而是**按因果层级分配权力**。

---

## 一、系统中使用的六类特征

BPC 系统当前使用的特征分为六大类：

| 类别 | 典型特征 | 数量级 | 来源 |
|------|---------|--------|------|
| **订单流特征** | VPIN, CVD, Trade Clustering, OFCI, SHD | 40+ | tick 数据逐笔成交 |
| **数学特征** | WPT(小波包), Spectrum(频谱), Hilbert(包络/相位), Hurst(长期记忆), EVT(极值理论) | 20+ | 信号处理 / 统计物理 |
| **Mamba/DL 序列特征** | dl_seq_f0...f63 (Mamba/Transformer embedding) | 64 | 深度学习序列编码器 |
| **结构特征** | SR距离, 回踩深度, 突破状态, 压缩分数, 方向一致性 | 15+ | 价格行为 / 结构检测 |
| **传统指标** | SMA200, BB Width, ATR, ROC, RSI | 10+ | 经典技术分析 |
| **组合语义特征** | soft_phase(阶段概率), flow_intensity, exhaustion_score | 5+ | 多特征加权组合 |

## 二、因果性分级框架

### 判定标准：是否依赖"人类行为持续不变"

| Level | 依赖什么 | 半衰期 | 策略姿势 |
|-------|----------|--------|----------|
| **0** | 只依赖"市场存在交易"这一事实 | 永恒 | 永久持有，只优化执行 |
| **1** | 依赖市场结构 + 激励机制 | 5-10年 | 核心持仓，监控结构变化 |
| **2** | 明确依赖行为偏差，可被学习和反向利用 | 1-3年 | 动态仓位，准备退化 |
| **3** | 纯统计相关，某时间段的模式 | 6-18月 | 小仓位试错，快速迭代 |

### 工程化用途

看到任何新 feature，立刻问：**"最坏情况下它会退化到哪一层？"**

## 三、六类特征的因果性审计

### 3.1 订单流特征

**因果机制**：订单流 → 价格（买单增加需求 → 价格上涨）

| 特征 | 因果 Level | 因果逻辑 | 退化风险 |
|------|-----------|---------|---------|
| **CVD（累积量差）** | Level 1-2 | 主动买卖量差 → 直接反映供需压力 | 机制长期存在，但具体阈值（如 `cvd_change_5 > 4805`）是历史拟合 |
| **VPIN** | Level 1-2 | 知情交易概率 → 检测信息不对称 | 理论基础扎实，但 VPIN 的 bucket 参数是经验选择 |
| **Trade Clustering（连续同向成交）** | Level 2 | 机构拆单行为 → 连续同向 = 大单分批执行 | 行为模式，但算法拆单策略会演化 |
| **OFCI（订单流拥挤指数）** | Level 2-3 | 拥挤 → 反转风险 | 机制合理，但 OFCI 的具体算法是统计构造 |
| **SHD（CVD-price 相关性）** | Level 2-3 | CVD 与价格背离 → 供需分歧 | 相关性本身是统计量，背离持续时间不确定 |

**核心结论**：
- CVD/VPIN 的**机制**是因果的（Level 1）：有人买 → 价格涨，这不会改变
- 但**具体用法**（阈值、时间窗口、组合方式）是统计的（Level 2-3）
- 退化方式：不是"订单流无用了"，而是"这个阈值在新市场不准了"

### 3.2 数学特征（信号处理/统计物理）

**本质**：不描述"为什么"，只描述"是什么状态"

| 特征 | 因果 Level | 实际语义 | 合法用法 |
|------|-----------|---------|---------|
| **WPT（小波包变换）** | Level 3 | 多尺度能量分布 → "价格在哪个频率最活跃" | 噪声检测器（Noise Penalty） |
| **Spectrum（频谱）** | Level 3 | 主频 + 功率谱平坦度 → "价格是有趋势还是随机" | 结构破碎度估计 |
| **Hilbert（相位/包络）** | Level 3 | 瞬时相位 + 包络 → "当前在周期的哪个位置" | 短期相位不稳定检测 |
| **Hurst 指数** | Level 1-2 | 长期记忆性 → "趋势 vs 均值回归" | 背景环境判断（权重最低） |
| **EVT（极值理论）** | Level 1 | 尾部风险建模 → "当前是否在历史极端尾部" | **保险丝**，不是刻度盘 |

**核心结论**：

> **数学特征 ≠ 无用。数学特征 = 非因果但允许存在。**

关键区分：
- 在 **Gate/Entry 层**：数学特征非法（不能决定"做不做"）
- 在 **Evidence 层**：数学特征合法（用于 Failure detection —— "区分哪些 trade 更容易失败"）
- 在 **Noise Penalty**：数学特征合法（用于降仓/拉宽止损）

**EVT 的正确用法**：
```python
# ❌ 错误：线性加权
noise_penalty = 0.35 * wpt + 0.30 * spectrum + 0.20 * hilbert + 0.15 * evt

# ✅ 正确：EVT 是保险丝
if evt_tail_risk > threshold:
    noise_penalty = min(0.8, noise_penalty + 0.2)
```

### 3.3 Mamba/DL 序列特征

| 特征 | 因果 Level | 问题 |
|------|-----------|------|
| **dl_seq_f0...f63** | Level 3 | 64 维 embedding，完全黑盒 |

**为什么不用**（详见架构决策总结）：
1. **黑盒**：无法知道哪个维度学到了什么
2. **IC 不稳定**：跨 regime 漂移严重
3. **维护成本高**：需要 GPU、热启动、定期重训
4. **数据量不够**：加密市场 5 年数据不足以训练稳定的序列模型
5. **embedding 尺度漂移**：不同 seed / 不同市场阶段的 embedding 幅度不同

**唯一合法用法**（未来）：
- 仅当 IC 在 OOS 连续 6 个月 > 0.03 时考虑
- 只做 Evidence 层的辅助评分轴，权重 < 10%

### 3.4 结构特征

**因果机制**：支撑/阻力 → 价格反弹（大量订单集中 → 难以突破）

| 特征 | 因果 Level | 因果逻辑 |
|------|-----------|---------|
| **SR 距离** | Level 1 | 支撑阻力 = 历史订单聚集区 → 流动性不连续 → 价格难突破 |
| **回踩深度** | Level 1-2 | 回踩 = 流动性回填 → 突破后回踩是物理过程 |
| **突破状态** | Level 1 | 突破 = 触发止损单 → 价格真空 → 物理机制 |
| **压缩分数** | Level 1-2 | 压缩 = 波动率收窄 → 能量积累 → GARCH 事实 |
| **方向一致性** | Level 2 | 连续同向 K 线 → 趋势延续 → 行为偏差型 |

**核心结论**：结构特征是最接近因果的一类。BPC 策略的核心（突破→回踩→延续）本身就是一个**流动性机制**：

```
突破触发止损单 → 价格真空 → 回踩是流动性回填 → 延续是新参与者进入
```

这个机制只依赖"市场存在交易"，**Level 0-1**。

### 3.5 传统指标

| 特征 | 因果 Level | 退化风险 |
|------|-----------|---------|
| **SMA 200** | Level 2 | "自我实现预言" — 因为所有人都看，所以有效。但正因广泛使用，既是信号也是靶标 |
| **BB Width** | Level 1-2 | 波动率压缩是 GARCH 事实，但参数化方式（20 期 / 2 标准差）是经验选择 |
| **ATR** | Level 0-1 | 波动率测量 — 物理量，只依赖"市场存在交易" |
| **RSI / MACD** | Level 2-3 | 极度拥挤的信号，被大量论文研究 → 被对手盘利用 |

### 3.6 组合语义特征

| 特征 | 因果 Level | 语义 |
|------|-----------|------|
| **bpc_soft_phase** | Level 1-2 | 多特征投票 → 当前处于 BPC 哪个阶段 |
| **flow_intensity** | Level 2 | Trade Clustering + VPIN 组合 → 订单流强度 |
| **exhaustion_score** | Level 2-3 | 多维耗尽指标组合 → 趋势是否衰竭 |

**核心问题**：组合语义特征的因果性 ≤ 其成分中**最弱的那个**。

## 四、因果性退化 —— 什么时候"因果"变成"历史痕迹"

### 4.1 退化机制

```
因果有效 → 被发现 → 被广泛使用 → 被对手盘学习 → 变成"假信号" → 历史痕迹
```

| 退化阶段 | 表现 | 持续时间 |
|---------|------|---------|
| **1. 发现期** | IC 高且稳定，少数人使用 | 1-2 年 |
| **2. 拥挤期** | IC 开始波动，策略同质化 | 6-12 月 |
| **3. 反向利用期** | IC 反转，对手盘设陷阱 | 3-6 月 |
| **4. 消亡期** | 恢复为噪声，无预测力 | 永久 |

### 4.2 每类特征的典型退化路径

| 特征类别 | 退化方式 | 速度 | 可恢复性 |
|---------|---------|------|---------|
| **订单流** | 算法拆单进化 → 传统 Trade Clustering 失效 | 中（1-3年） | 可以，换检测算法 |
| **数学特征** | 不退化（描述的是状态，不是行为） | N/A | N/A |
| **DL 序列** | 训练数据分布漂移 → embedding 语义漂移 | 快（3-6月） | 需要重训 |
| **结构特征** | 结构机制不变，但"所有人都知道 SR" → 假突破增多 | 慢（5-10年） | 基本不退化 |
| **传统指标** | 极度拥挤 → 被反向利用 | 已经退化 | 仅作为确认，不单独使用 |
| **组合语义** | 成分退化时跟着退化 | 取决于成分 | 替换成分后可恢复 |

### 4.3 因果性也会退化的具体场景

**场景 1：订单流的"CVD 吸收"模式**

```
因果逻辑：回踩时 CVD 不跟随下跌 → 买方在底部接盘 → 延续信号

退化过程：
1. 这个模式被量化基金广泛使用
2. 做市商发现"CVD 吸收后散户必买" → 故意制造虚假吸收
3. CVD 吸收 → 真的有人买 → 但是做市商在上方挂卖单等着
4. 因果逻辑变成：CVD 吸收 → 陷阱
```

**在 BTC 上的中等可信度**：因为加密市场做市商数量有限、拥挤度低于美股。但不保证永远有效。

**场景 2：SMA 200 的"自我实现预言"退化**

```
因果逻辑：价格在 SMA 200 以上 → 多头市场 → 回踩到 200 日线附近买入

退化过程：
1. 所有人都知道 200 日线 → 大量止损单放在 200 日线下方
2. 做市商发现止损单聚集 → 故意打穿 200 日线触发止损 → 然后反弹
3. "在 200 日线买入" → 变成 "被止损后看着涨"
```

**场景 3：波动率压缩 → 爆发的退化**

```
因果逻辑：BB 收窄 = 波动率压缩 → GARCH 事实 → 压缩后必爆发

退化方向：
- GARCH 事实不会退化（这是统计物理）
- 但"压缩后做多/做空"的具体策略会被套利
- 退化不是"压缩不爆发了"，而是"爆发方向越来越难预测"
```

## 五、Regime 切换 —— 为什么统计相关性会变

### 5.1 什么是 Regime Shift

市场状态的根本性变化，导致历史模式的统计关系失效：

| Regime 类型 | 训练期 | 测试期 | 影响 |
|------------|--------|--------|------|
| **波动率 regime** | ATR percentile = 0.3 | ATR percentile = 0.8 | 所有硬阈值失效 |
| **趋势 regime** | 趋势明显、方向一致 | 震荡为主、来回反转 | 趋势跟踪策略全失败 |
| **流动性 regime** | 流动性充足、滑点小 | 流动性枯竭、滑点大 | 执行成本吞噬利润 |
| **订单流 regime** | 散户主导、模式稳定 | 机构算法主导、模式高频变化 | 订单流特征语义漂移 |
| **相关性 regime** | CVD 与 BB Width 相关性 0.7 | 相关性降到 0.3 | 特征组合规则失效 |

### 5.2 硬阈值在 Regime 切换时的致命问题

```python
# 训练期（2023年，低波动）
if cvd_long <= -34895:  # 这个阈值 = 历史 2.5% 分位数
    veto = True  # 准确率 90%

# 测试期（2024年，高波动）
# cvd_long 均值从 -10000 变为 -30000
# -34895 从 2.5% 分位数变成 20% 分位数
# 准确率从 90% 降到 30%
```

**解决方案**：绝对阈值 → 分位数阈值

```python
# ✅ 分位数阈值（自适应）
cvd_q20 = quantile(cvd_long, window=200, q=0.2)
if cvd_long <= cvd_q20:  # 相对位置不变
    veto = True  # 始终对应"极端值"
```

### 5.3 不同市场需要不同方案

| 市场 | 主要 Regime | 有效特征倾向 | 无效特征倾向 |
|------|-----------|------------|------------|
| **BTC 牛市** | 高波动趋势 | 结构特征、CVD 同向 | 均值回归指标、压缩指标 |
| **BTC 熊市** | 高波动震荡 | SR 反转、EVT 极值 | 趋势跟踪、动量指标 |
| **BTC 横盘** | 低波动压缩 | 压缩分数、BB Width | 订单流（信噪比低） |
| **ETH（相对 BTC）** | 高 beta | CVD 交叉、结构相似 | 独立 VPIN（流动性不足） |
| **山寨币** | 极端波动 | 极值理论、结构止损 | 传统指标（噪声太大） |

### 5.4 为什么不能"一套特征打天下"

```
核心矛盾：
  稳定特征（Level 0-1）→ 在所有 regime 都有效 → 但 alpha 小
  激进特征（Level 2-3）→ 在特定 regime alpha 大 → 但 regime 变了就失效
```

**BPC 系统的解决方案**：

1. **Gate 层用 Level 0-1 特征**（结构、物理量）—— 决定"做不做"
2. **Evidence 层允许 Level 2-3 特征**（数学、订单流衍生）—— 只影响"仓位大小"
3. **Noise Penalty 用非因果特征**（WPT/Spectrum）—— 只影响"执行参数"
4. **所有 Level 2-3 特征都有降级方案**：关掉它们，系统退化但不死

## 六、特征权力边界 —— 按因果性分配权力

### 6.1 权力分配原则

> **最不确定的东西，权力必须最小；最确定的东西，必须负责生存。**

| 因果 Level | 允许的最大权力 | 典型用法 |
|-----------|--------------|---------|
| **Level 0** | 生存决策（hard veto, 风控） | Execution 宪法、极端行情自动收缩 |
| **Level 1** | 方向判断 + Gate 否决 | BPC 结构检测、SR 距离 |
| **Level 2** | 仓位调整（Evidence 评分） | CVD 吸收、SMA 位置 |
| **Level 3** | 风险状态估计（Noise Penalty） | WPT 能量、Spectrum 平坦度 |

### 6.2 铁律：Evidence 层的禁止清单

> **Evidence 层的任何输出，都不能在单一 trade 上把 NO → YES 翻转。**

| ✅ 允许 | ❌ 禁止 |
|---------|--------|
| YES → 仓位小一点 | 原本不该做 → 因为 Evidence 好看就做 |
| YES → 晚一点执行 | 被 Gate 否决 → Evidence 翻案 |
| YES → 条件更苛刻 | 单独决定方向或仓位 |

### 6.3 降级方案

```
最低配置（所有统计特征关闭）:
  Gate (hard deny 3 条) → Entry Filter (BB 压缩) → Execution (固定参数)
  ↓
  系统仍然可以交易，Sharpe 降低但不会亏钱

正常配置:
  Gate (3 hard + 9 soft) → Entry Filter (BB OR Liq) → Evidence (9 特征) → Tier → Noise Penalty
  ↓
  更精细的仓位管理，Sharpe 更高
```

## 七、监控体系 —— 持续验证因果性

### 7.1 每个组件的监控指标

| 组件 | 监控指标 | 报警阈值 |
|------|---------|---------|
| **Gate 规则** | 否决率变化 | ±50% vs 历史均值 |
| **Evidence 特征** | 单特征 IC（6 个月滚动） | < 0.01 连续 3 个月 |
| **Entry Filter** | 通过率 + 通过后 Sharpe | Sharpe < 0 连续 2 个月 |
| **Noise Penalty** | 降仓后 vs 不降仓的 Sharpe 差异 | 差异 < 0（降仓反而更差） |

### 7.2 退化诊断流程

```
Sharpe < 历史均值 - 2σ → 触发诊断
  ↓
Step 1: 检查 Gate 否决率 → 是否 regime 变化导致过度否决？
Step 2: 检查 Evidence IC → 哪个特征失效？
Step 3: 检查 Entry Filter → 是否在错误的时机入场？
Step 4: 检查 Noise Penalty → 是否在不该降仓时降仓？
  ↓
定位到具体组件 → 禁用或调整阈值
```

### 7.3 验证方法

1. **反事实测试**：如果信号消失，价格会回撤吗？
2. **跨市场验证**：同一信号在 BTC / ETH / SOL 表现一致吗？
3. **时间稳定性**：模式能存活多久？（OOS 滚动验证）
4. **扰动测试**：阈值 ±20% 后，Sharpe 变化大吗？（plateau 验证）

## 八、总结

### 一句话版本

> **因果负责活下去，统计负责少犯错，架构负责不自杀。**

### 按可长期性分类

| 分类 | 组件 | 预期寿命 |
|------|------|---------|
| 🟢 **可以跟你走 10 年** | 分层架构、职责切分、plateau/稳定区思想、execution 风控、regime-aware sizing | > 10 年 |
| 🟡 **可能活 1-3 年** | 回踩结构+缩量确认、CVD 吸收、方向拥挤度、行为型 exhaustion | 1-3 年 |
| 🟡 **非因果但合法** | 数学特征（WPT/Spectrum/EVT）作为风险状态估计器、Noise Penalty | 不退化，但权力受限 |
| 🔴 **注定短命（但必须做）** | 具体阈值、MACD alpha、单一微观结构模式 | 6-18 月 |

### 资金分配建议

```
70% → Level 0-1 核心组合（结构 + 物理量 + 风控）
20% → 探索新因子（新订单流特征、新结构检测）
10% → 捕捉短期套利（Level 3 特征驱动的短期策略）
```

### 终极自检

每加一个特征/组件，必须回答三件事：

| 自检问题 | 不合格则 |
|----------|----------|
| 它退化时会不会杀死系统？ | 必须降权或加 fallback |
| 它有没有越权影响方向/仓位？ | 必须限制在合法层级 |
| 关掉它，系统还能不能跑？ | 不能则违反可降级原则 |

> **你已经不在找 alpha，你在设计一个允许 alpha 死亡的生态位。**
