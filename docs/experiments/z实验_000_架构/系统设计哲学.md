## 一、认知分层：四句箴言

> 来源：`docs/archive/architecture/系统哲学基础.md`

```
Price  tells you WHAT          — 发生了什么
Volume tells you IF            — 是否被市场接受
Order flow tells you WHO       — 是谁在推动
Math   tells you HOW CAREFULLY — 系统应有多保守
```

这不是诗句，而是**信息论分层**：

| 层 | 问题 | 数据类型 | 系统对应 |
|----|------|----------|----------|
| WHAT | 发生了什么 | Price / Structure | Gate（结构判断） |
| IF | 是否被市场接受 | Volume / Participation | Entry Filter（确认信号） |
| WHO | 是谁在推动 | Order flow / Trade cluster | Evidence（机会排序） |
| HOW CAREFULLY | 系统应有多保守 | Math / Noise / Tail risk | Execution（风控修正） |

**核心原则**：

> **没有任何一层能"越权回答别层的问题"。**

- Gate 保护正确性（Gate protects correctness）
- Evidence 排序机会（Evidence ranks opportunity）
- Execution 决定暴露（Execution decides exposure）
- Math 控制谨慎度（Math controls caution）

这套分权原则约束了系统的一切设计决策：不能让 Evidence 决定方向，不能让 Math 主导仓位，不能让 Execution 覆盖 Gate。

## 二、Popper 可证伪系统

> 来源：`docs/archive/architecture/系统哲学基础.md`、`docs/z实验_001_bpc/因果性思考.md`

系统不是在"调参"，而是在**可证伪地进化**：

```
持续学习循环：假设 → 测试 → 证伪/确证 → 更新
```

| Popper 步骤 | 在系统里的实体 |
|------------|-------------|
| 假设 | Archetype / Rule / Gate |
| 测试 | Tree / Backtest / Failure audit |
| 证伪 | failure_rr_extreme / no_opportunity |
| 更新 | 宪法条款 / rule removal / weight shift |

**关键洞察**：

> **系统不是在"最大化收益"，而是在最小化已知错误之外的未知错误。**

- 我们不 retrain to improve success
- 我们 retrain to eliminate avoidable failure
- 下一轮训练，只针对宪法条款覆盖不到的 failure

这是**智慧系统**的定义，不是策略的定义。

## 三、因果性分级与权力边界

> 来源：`docs/z实验_001_bpc/因果性思考.md`、`docs/z实验_000_架构/特征因果性与统计相关性分析.md`

### 3.1 四级因果性框架

| Level | 依赖什么 | 半衰期 | 策略姿势 |
|-------|----------|--------|----------|
| **0** | 只依赖"市场存在交易"这一事实 | 永恒 | 永久持有，只优化执行 |
| **1** | 依赖市场结构 + 激励机制 | 5-10年 | 核心持仓，监控结构变化 |
| **2** | 明确依赖行为偏差，可被学习、拥挤 | 1-3年 | 动态仓位，准备退化 |
| **3** | 纯统计相关，某时间段的特定模式 | 6-18月 | 小仓位试错，快速迭代 |

**工程化用途**：看到任何新 feature，立刻定位"最坏情况下它会退化到哪一层"。

### 3.2 权力边界原则

> **最不确定的东西，权力必须最小；最确定的东西，必须负责生存。**

这是本系统最重要的哲学原则之一。按因果性分配权力：

| 因果等级 | 允许的权力 | 禁止的权力 |
|---------|-----------|-----------|
| Level 0-1 | 决定生存约束、Gate 否决 | — |
| Level 2 | 作为必要非充分条件 | 单独决定方向或仓位 |
| Level 3 | 只在 Evidence 层降权/延迟 | 不能 NO→YES 翻转 |

### 3.3 Evidence 层铁律

> **Evidence 层的任何输出，都不能在单一 trade 上把 NO → YES 翻转。**

| 允许 | 禁止 |
|------|------|
| YES → 小一点（降仓） | 原本不该做 → 因为 Evidence 好看就做 |
| YES → 晚一点（延迟） | 被 Gate 否决的 trade → Evidence 翻案 |
| YES → 条件更苛刻（收紧） | 单独决定方向或仓位 |

**终极比喻**：统计特征是燃料，不是发动机。但燃料也有价值——只要不让它当方向盘。

## 四、自由度限制哲学

> 来源：`docs/architecture/自由度限制.md`、`docs/architecture/自由度限制-归因-仓位和加仓.md`

### 4.1 核心信条

> **可控的次优解 > 不可控的最优解。**

系统会"自然地"不断增加自由度（分组、择时、加仓次数、动态风险预算、策略切换）。如果不把自由度关进宪法，回撤时你会遇到两类致命问题：

- **不可归因**：你不知道是哪一个自由度在作恶
- **不可回退**：你无法一键回到"更低自由度、更安全"的模式

### 4.2 自由度的分类

**允许的自由度**（最小集合）：
- 是否开仓（Gate 决定）
- 何时开仓（Entry Filter 决定）
- 方向（结构决定，不是模型决定）

**禁止的自由度**：
- 模型决定仓位大小
- 模型决定是否替换仓位
- 模型决定并行数量
- 任何隐式的自由度

### 4.3 硬规则

> **任何新增自由度必须满足：**
> 1. 开/关显式 — 不是隐藏在代码里
> 2. 贡献可统计 — 能测量有没有用
> 3. 经历完整 cycle — 不是只在牛市验证

**哲学本质**：自由度限制不是限制能力，而是限制**犯错的方式数量**。一个只有 3 种犯错方式的系统，远比有 30 种犯错方式的系统更容易修复。

## 五、宪法体系

> 来源：`docs/architecture/自由度限制-归因-仓位和加仓.md`、`docs/archive/architecture/极端情况暴富和生存的问题.md`、`docs/archive/树模型策略知识迁移到多头模型.md`

### 5.1 什么是宪法

宪法 ≠ 策略规则。宪法是**不可违反的硬约束**，所有策略、模型、token 都必须遵守。

宪法要做的不是"让收益更高"，而是把系统变成：
- **可控**：任何自由度可关、可复盘
- **可生存**：硬风险约束永不违反（哪怕错过行情）
- **可跃迁**：满足资格时，允许在 Portfolio 层提高风险预算

### 5.2 仓位宪法核心条款

| 条款 | 内容 |
|------|------|
| **坑位不可突破** | 最大并行仓位数固定，不因行情/模型/自信而改变 |
| **模型无权分仓** | NN 不得决定仓位大小、是否替换、并行数量 |
| **替换必须"有罪"** | 任何仓位被平掉，必须能指出明确的失败原因 |
| **不排序不预测** | 不做横截面排序，不做"未来可能更强"的假设 |

### 5.3 交易宪法七条

1. **Alpha 定义**：Alpha 不存在于方向判断中，存在于"在允许活下来的前提下，提高单位时间收益密度"
2. **权力边界**：模型只表达偏好，不知道风险，不决定仓位
3. **Safety 至高权力**：安全层拥有唯一否决权，可禁止交易、压仓位、降速
4. **Execution 约束**：规则永远覆盖执行层，执行层不能 override 安全层
5. **灭绝回测原则**：每个模块必须经历全市场极端回放
6. **实盘可视化义务**：不看 PnL，才能活得久
7. **进化原则**：任何新增模型或特征，必须先证明"它不会让我死得更快"

### 5.4 个人交易宪法（创伤版）

> 来源：`docs/archive/architecture/极端情况暴富和生存的问题.md`

| 章节 | 核心 |
|------|------|
| **生存优先** | 不允许交易威胁基本生活与尊严。没有"追回损失"这一合法动机 |
| **系统边界** | 无 regime → 无交易。任何仓位升级必须来自系统状态变化，不是 PnL |
| **尊严条款** | 不允许任何一次交易让你事后讨厌自己。夜不能寐的交易 = 非法 |
| **上限条款** | 接受"赚不到最多的钱"，换取"不再被淘汰"。错过牛市 ≠ 错误 |
| **融资冻结** | 在系统稳定、自有资金 Sharpe > 1 之前，永不融资 |

## 六、Plateau 哲学：不追极值

> 来源：`docs/architecture/guides/PLATEAU_OPTIMIZATION_METHODOLOGY.md`、`docs/z实验_001_bpc/因果性思考.md`

### 6.1 核心思想

> **"当世界对我不敏感时，我才出手。"**

不追求 argmax（最优参数点），而是寻找**平坦高原**（plateau）——在一个参数区间内，性能变化不大的稳健区域。

| 旧范式 | Plateau 范式 |
|--------|-------------|
| 最大化 Sharpe | 找 Sharpe > 阈值的**稳定区域** |
| 寻找"最佳参数" | 接受参数模糊性（Parameter Fuzziness） |
| 相信精确值 | 在 plateau 宽容侧取值 |
| Sharpe = 3.0 的尖峰 | Sharpe ∈ [1.5, 2.0] 的宽原 |

### 6.2 三重判据

1. **Mean Sharpe**：区间内的平均性能
2. **Plateau Width**：区间的宽度（越宽越鲁棒）
3. **Trades CV**：交易次数的变异系数（参数变化不应剧烈改变交易数量）

### 6.3 哲学内核

Plateau 不是在找 entry，而是在**拒绝在"信息不充分时行动"**。具体阈值会变，但 plateau / 稳定区 / 不追极值**永远不会过时**。

> **Optuna 用来找点，Plateau 用来画面。**

## 七、可降级原则

> 来源：`docs/z实验_001_bpc/因果性思考.md`、`docs/z实验_000_架构/架构决策总结_为什么用纯规则架构.md`

### 7.1 核心定义

> **任何组件可关闭，系统退化但不死。**

每个组件必须通过"可降级三问"：

| 自检问题 | 不合格则 |
|----------|----------|
| 它退化时会不会杀死系统？ | 必须降权或加 fallback |
| 它有没有越权影响方向/仓位？ | 必须限制在合法层级 |
| 关掉它，系统还能不能跑？ | 不能则违反可降级原则 |

### 7.2 降级路径

系统的降级路径是显式设计的：

```
完整系统: Gate + Evidence + Entry Filter + Tier + Noise Penalty + Execution
    ↓ 关闭 Noise Penalty
安全模式: Gate + Evidence + Entry Filter + Tier + Execution
    ↓ 关闭 Evidence + Tier
基础模式: Gate + Entry Filter + Execution (固定参数)
    ↓ 关闭 Entry Filter
最小模式: Gate + Execution (最保守参数)
    ↓ 触发 kill switch
停机模式: 不交易
```

每一级都是一个**可运行的系统**，而不是一堆残缺的代码。

### 7.3 哲学意义

可降级原则的本质是：**系统的生存不依赖于任何单一组件的正确性**。这是对"完美主义"的彻底否定——不追求每个组件都完美，而是确保任何组件失效时系统仍然活着。

## 八、反身性 vs 自指智慧

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

### 8.1 核心区分

> **反身性是"盲目的自我放大"，自指智慧是"清醒的自我校正"。**

| 特性 | 反身性（索罗斯式） | 自指智慧（侯世达式） |
|------|-----------------|-------------------|
| **反馈类型** | 正反馈（Reinforcing Loop） | 负反馈（Balancing Loop） |
| **目标** | 放大信念 → 自我实现 | 校准认知 → 趋近真实 |
| **结果** | 失稳、泡沫、崩溃 | 稳定、适应、学习 |
| **信息流** | 忽略证伪信号，只强化原假设 | 主动寻找错误，修正模型 |

### 8.2 反身性是"错误的自指"

```
市场上涨 → 投资者相信"趋势将持续" → 更多人买入 → 市场上涨
```

这个循环中没有外部真实锚点。它不是"认识世界"，而是"集体催眠"。一旦现实打破幻觉（流动性枯竭），循环瞬间反转 → 崩盘。

**反身性 = 自指 × 无批判性 = 系统性幻觉**

### 8.3 自指智慧的纠错闭环

```
行动（执行交易）→ 观测（记录 PnL + 市场反应）→ 归因（分析"为何对/错"？）→ 更新（修正规则/阈值）→ 行动
```

关键差异：
- 引入外部真实（PnL、滑点、流动性数据）
- 主动寻找反例（"什么情况下我会错？"）
- 允许系统否定自身（暂停策略、降低仓位）

**自指智慧 = 自指 × 批判理性 = 可持续进化**

### 8.4 KPI 的双面性

> **KPI 本身不决定性质——取决于你如何用它。**

| 行为 | 反身性系统 | 自指智慧系统 |
|------|-----------|------------|
| 看到盈利 | "我太对了！加仓！" | "盈利是否来自真实 edge？" |
| 看到亏损 | "只是回调，坚持！" | "规则在哪种 regime 失效？" |
| KPI 作用 | 仅用于确认成功 | 用于证伪假设 |

**终极比喻**：
- 反身性 = **回音壁**：声音不断反射放大，外面早已天翻地覆
- 自指智慧 = **带校准功能的镜子**：不仅照出你，还告诉你"领带歪了"

## 九、观测即参与

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

### 9.1 核心原则

> **你不是旁观者，而是共谋者——这要求节制与自省。**

系统的订单既是信息输入，也是市场扰动。因此：
- 不是"预测市场"，而是以**最小干预**探测市场状态
- 避免自我实现的幻觉（过度交易导致市场扰动）
- 系统必须**节制**（不频繁交易）和**自省**（知道自己的影响）

### 9.2 工程体现

- **Gate 层**实现"节制"：知道何时不该交易
- **归因能力**实现"自省"：知道哪里出问题
- **最小干预原则**：小单试探、控制冲击成本

### 9.3 量子观测者隐喻

经典物理思维把交易者当成外部观察者。但你是市场的一部分——你的策略本身就在改变市场有效性。忽视这一点，就是忽视反身性的最危险形式。

## 十、反脆弱

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

### 10.1 从"预测"转向"反脆弱"

放弃"找到最优参数"的执念，转向"构建能从不确定性中受益的系统"（塔勒布思想）。

| 旧范式 | 反脆弱范式 |
|--------|-----------|
| 最大化 Sharpe | 最小化尾部损失 |
| 寻找"最佳参数" | 接受参数模糊性 |
| 相信趋势持续 | 为趋势中断做准备 |
| 回测拟合历史 | 压力测试未来 |

### 10.2 终极心法

> **真正的专业，不是知道如何赚最多，而是知道如何活最久。**

- 市场不需要你"正确"，只需要你"还在"
- 放弃对"最优"的执念，转而构建：
  - 多重失效保护（Gate 规则）
  - 跨 regime 适应性（动态阈值）
  - 尾部免疫能力（仓位限制 + 对冲）

> **不要问"哪个参数历史表现最好？"，**
> **而要问"哪个参数在未来最不容易让我爆仓？"**

### 10.3 期权思维

指数加仓 = 隐性卖出深度虚值看跌期权（Short OTM Put）：
- 大部分时间赚小钱
- 一旦趋势反转，亏损指数级放大

**正确姿势**：不做"隐形期权卖方"，而做"显性期权组合构建者"。

## 十一、四条设计原则

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

### 1. 可修复性 > 最优性

- **哲学基础**：智慧不是追求最优，而是知道如何修复
- **工程体现**：分层 KPI、归因能力、针对性修复
- **反面教训**：追求 Sharpe=3.0 的系统，坏了你不知道哪里坏

### 2. 可控性 > 灵活性

- **哲学基础**：节制与自省，知道何时不该行动
- **工程体现**：降低自由度、平坦高原、Gate 层
- **反面教训**：灵活的系统 = 犯错方式太多的系统

### 3. 归因能力 > 扩展能力

- **哲学基础**：自指是复杂性的核心，系统必须能"谈论自身"
- **工程体现**：分层 KPI、自我诊断、自我修复
- **反面教训**：能扩展到 100 个 token 但不知道哪个在亏钱 = 高级形态的自杀

### 4. 系统可维护性 > 模型性能

- **哲学基础**：系统的"自我模型"需要持续内部状态和可归因的决策边界
- **工程体现**：分层架构、统一管理、可诊断性
- **反面教训**：Sharpe=2 但半年后没人看得懂 = 技术债的利滚利

## 十二、哲学作为认知操作系统

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

### 12.1 哲学 = 底层操作系统

| 操作系统（哲学） | 应用表现（交易行为） |
|----------------|-------------------|
| 相信"市场可完全预测" | 过度拟合、频繁调参、忽视尾部风险 → 最终崩盘 |
| 接受"知识有限性" | 设置 Gate、识别平坦高原、主动暂停 → 长期存活 |
| 认为"AI 能替代判断" | 盲信模型输出，不做归因 → 被反身性收割 |
| 理解"观测即扰动" | 小单试探、控制冲击成本 → 滑点更低 |

> **你的哲学，决定了你在"不知道的时候，会怎么做"。**
> **而市场最残酷的惩罚，往往就发生在"你不知道自己不知道"的时候。**

### 12.2 四个血泪教训的哲学根源

| 教训 | 哲学根源 | 正确哲学 | 工程体现 |
|------|----------|----------|----------|
| "这次不一样" | 否认历史规律循环性 | 市场状态会变，人性不变 | Regime 切换 |
| 过度优化 | 混淆"拟合"与"理解" | 奥卡姆剃刀 + 可证伪性 | 平坦高原搜索 |
| 忽视反身性 | 把自己当外部观察者 | 你是市场的一部分 | 反身性监测 |
| 回测找最大 Sharpe | 决定论幻觉 + 归纳法迷信 | 从"预测"转向"反脆弱" | 压力测试 |

> **所有爆仓，都是认知漏洞的变现。**

### 12.3 哲学如何直接提升 PnL

| 哲学洞见 | 工程实践 | 直接收益 |
|---------|---------|---------|

## 十三、怪圈式探索

> 来源：`docs/archive/leagcy/FINAL_SIMPLIFIED_ARCHITECTURE_2026_01.md`

> **从代码出发，抵达哲学；再从哲学返回，照亮代码。**

本项目的认知路径本身就是一个"怪圈"（Strange Loop, 侯世达）：

```
Gate 规则（技术）
    ↓ 为什么这条规则有效？
系统设计的本质（哲学）
    ↓ 这个哲学如何落地？
工程实践（代码）
    ↓ 代码运行后发现新问题
Gate 规则（技术）
    ...
```

系统具备三个"自我模型"特征：
1. **持续内部状态**：分层 KPI 持续跟踪系统状态
2. **基于自我认知调节行为**：通过归因能力知道"哪里出问题"，针对性修复
3. **可归因的决策边界**：每层独立 KPI，可精确定位问题

> **而这，或许就是理性最动人的样子。**
