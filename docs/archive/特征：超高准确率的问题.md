🧩 一、什么是“价格连续性”？为什么它会欺骗模型？
现象：
在任意时间粒度（比如 5 分钟 K 线），价格变化非常小：

时间 Close (BTC) Return (log)
------ ------------- --------------
t-1 60,000 —
t 60,030 ≈ +0.0005
t+1 60,050 ≈ +0.00033

👉 close_t 和 return_{t+1} 虽然不直接相等，但高度相关，因为：
return_{t+1} = log(close_{t+1} / close_t)
而 close_t 本身是 close_{t-1} 的延续

如果特征中包含 close_t、ma_close、price_position 等，模型会“偷看”当前价格水平，从而几乎完美预测下一时刻的微小变动——但这在实盘中不可能做到！

✅ 二、“价格类特征用收益率/差分” —— 具体怎么做？
❌ 错误做法（含未来信息）：
python
特征计算时用了当前 close（t 时刻）
df['price_pos'] = (df['close'] - df['low']) / (df['high'] - df['low']) # ← 用了 t 时刻的 close/high/low
df['ma20'] = df['close'].rolling(20).mean() # ← 包含当前 close
✅ 正确做法（只用 t-1 及之前）：
python
所有特征基于“上一根K线结束时已知的信息”
df['log_ret'] = np.log(df['close'] / df['close'].shift(1)) # 收益率（t 时刻的收益，由 t-1 → t 得到）
特征必须 shift(1)，表示“在 t 时刻做预测时，只能用到 t-1 的统计量”
df['ma20_ret'] = df['log_ret'].rolling(20).mean().shift(1) # 过去20根的平均收益（不含当前）
df['vol60'] = df['log_ret'].rolling(60).std().shift(1) # 过去60根波动率
价格位置也用上一根K线
df['price_pos_prev'] = (
(df['close'].shift(1) - df['low'].shift(1)) /
(df['high'].shift(1) - df['low'].shift(1))
)
🔑 核心原则：
你在 t 时刻要预测 t+1 的收益，那么所有特征必须在 t 时刻“已经知道”——也就是基于 t 及之前的原始数据，但不能包含 t+1 的任何信息。
而由于 close_t 在 t 时刻结束时才确定，严格来说，在 t 时刻刚开始时，你只能用到 close_{t-1}！

但在大多数回测框架中，我们假设：
在 t 根 K 线结束后，立即用该根 K 线的数据生成信号，用于下一根（t+1）交易
所以 close_t 是可用的（因为它在 t 结束时已知）

✅ 因此，只要目标是预测 return_{t+1}，特征可以用 close_t —— 但前提是：不能让特征“泄露”关于 return_{t+1} 的信息。

然而，直接使用 close_t 仍危险，因为它与 return_{t+1} 有强数学关联（通过价格连续性）。所以更安全的做法是：全部转为收益率或标准化量。

🎯 三、“目标变量去连续性” —— 到底是什么意思？

这是更高阶的技巧，目的是剥离价格序列中“确定性”的部分，只留下“可预测的超额部分”。
背景知识：金融收益常有 自相关（autocorrelation）
尤其在高频（5T/15T），return_t 和 return_{t+1} 有微弱正相关
比如：过去 5 分钟涨了，接下来 5 分钟还可能惯性上涨一点点

但这部分“惯性”是市场微观结构导致的确定性成分，不是 alpha。
💡 思路：把目标从 “预测 return” 变成 “预测 return 中无法被历史解释的部分”
方法 1：减去 AR(1) 预测值（推荐）
python
Step 1: 估计一阶自回归系数 φ
returns = df['log_ret'].dropna()
phi = np.corrcoef(returns[:-1], returns[1:])[0, 1] # lag-1 autocorr
Step 2: 构造“去连续性”目标
df['target'] = df['log_ret'].shift(-1) - phi df['log_ret']

df['log_ret'].shift(-1) 是 return_{t+1}
phi df['log_ret'] 是基于 return_t 对 return_{t+1} 的最佳线性预测（AR(1)）
target = 实际值 - 预测值 = “残差”，即无法被惯性解释的部分
📌 现在你的模型任务是：预测“超出价格惯性的那部分收益”
如果模型还能预测这个残差（IC > 0），说明它找到了真正的 alpha，而非利用连续性。
方法 2：直接预测符号（sign），忽略大小
python
df['target'] = np.sign(df['log_ret'].shift(-1))
虽然仍有连续性影响，但至少避免了对微小价格变动的过度拟合
常用于分类模型或 RL 的 reward shaping
方法 3：使用“去趋势价格”（适用于均值回复策略）
python
用移动平均去除趋势

# AR(1) 是什么意思

AR(1) 是 一阶自回归模型（Autoregressive Model of Order 1）的简称，是时间序列分析中最基础、最重要的模型之一。

📌 一句话定义：
AR(1) 模型假设：当前时刻的值，等于一个常数 + 前一时刻值的线性比例 + 随机噪声。

数学表达式如下：

\[
x_t = c + \phi \cdot x_{t-1} + \varepsilon_t
\]

其中：
\( x_t \)：时间 \( t \) 的观测值（比如收益率、价格、温度等）
\( c \)：常数项（可为 0）
\( \phi \)（读作 "phi"）：自回归系数，表示 \( x_{t-1} \) 对 \( x_t \) 的影响强度
\( \varepsilon_t \)：随机误差项（通常假设为白噪声，均值为 0）

🔍 举个金融例子（用收益率）：

假设你有一串 BTC 的 5 分钟对数收益率序列：
..., r_{t-2}, r_{t-1}, r_t, r_{t+1}, ...

如果它服从 AR(1) 模型，那就意味着：

\[
r_{t+1} = \phi \cdot r_t + \varepsilon_{t+1}
\]

👉 下一时刻的收益率，主要由当前收益率乘以一个系数 \(\phi\) 决定，再加上一个随机扰动。
如果 \(\phi = 0.2\)：说明当前涨了，下一刻略微倾向于继续涨（动量效应）
如果 \(\phi = -0.1\)：说明当前涨了，下一刻略微倾向于回调（均值回复）
如果 \(\phi ≈ 0\)：说明前后收益率几乎无关（接近有效市场）
💡 在高频交易中，加密货币的 5T/15T 收益率常有微弱的正自相关（\(\phi \approx 0.05 \sim 0.2\)），这就是“价格惯性”的来源。

🧮 如何估计 \(\phi\)？

最简单的方法：计算滞后 1 阶的自相关系数（lag-1 autocorrelation）

python
import numpy as np
import pandas as pd
假设 returns 是一个 pandas Series，包含对数收益率
returns = np.log(df['close'] / df['close'].shift(1)).dropna()
方法1：用 pandas autocorr
phi = returns.autocorr(lag=1)
方法2：用 numpy 手动计算
r_t = returns[:-1]
r_t1 = returns[1:]
phi = np.corrcoef(r_t, r_t1)[0, 1]

print(f"AR(1) 系数 φ = {phi:.4f}")

例如，你可能得到：φ = 0.12
这意味着：当前 5 分钟收益率每上涨 1%，下一 5 分钟平均上涨 0.12%（仅由惯性导致）。

🎯 为什么在量化中要用 AR(1)？
目的：剥离“确定性惯性”，提取“超额预测信号”
原始目标：预测 \( r_{t+1} \)
但其中有一部分是“可被 \( r_t \) 线性解释的”：\( \hat{r}_{t+1} = \phi \cdot r_t \)
真正需要预测的是残差（residual）：
\[
\text{target} = r_{t+1} - \phi \cdot r_t
\]

✅ 这样做的好处：
消除价格连续性带来的虚假相关
让模型专注学习超越简单惯性的模式（比如量价背离、波动率突变等）
使 IC、准确率等指标更真实，避免“看似很强，实盘失效”

📊 类比理解

场景 类比
------ ------
原始目标 \( r_{t+1} \) 预测“明天天气温度”
AR(1) 部分 \( \phi \cdot r_t \) “今天热，明天大概率也热”（季节/惯性）
残差目标 \( r_{t+1} - \phi r_t \) 预测“明天比惯性预期更热还是更冷？”（异常信号）

你真正想预测的，是异常部分，而不是“因为今天热所以明天热”这种 obvious pattern。

✅ 总结
AR(1) = 用前一时刻的值线性预测当前值
φ（phi）衡量序列的“记忆性”或“惯性”
在量化中，用 AR(1) 残差作为目标变量，可避免被价格连续性欺骗
实操：计算 lag-1 自相关系数 φ，然后构造 target = r_{t+1} - φ * r_t

这是从“回测幻觉”走向“真实 alpha”的关键一步。
