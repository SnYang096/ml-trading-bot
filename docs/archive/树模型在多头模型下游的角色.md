## 📄 Tree Gate Rule Specification

### Rule ID

```
TREND_VETO_01
```

---

### 1️⃣ 作用对象（Scope）

```
trend
```

**可选值**：

* `trend`

* 防止规则“越权干预”

### 2️⃣ 类型（Decision Type）

```
veto
```

**可选值**：

* `veto`      → 强制不交易
* `confirm`   → 必须满足才允许
* `throttle`  → 不否决，但降级执行

### 为什么必须写？

* 决定这条规则**能不能一票否决**
* 决定 false error 的容忍方向

---

### 3️⃣ 输入特征（Inputs）

```

  - dir_prob
  - mfe
  - tau
  - vol

handcrafted:
  - vpin
  - spread_ratio
  - sr_level

router:
  - p_trend
```

### 为什么要明确写？

* 防止 **隐性特征膨胀**
* 防止未来引入 **leakage**
* 方便 debug / ablation

👉 **Tree Gate 的输入是“制度的一部分”，不是随便加的**

---

### 4️⃣ 判定逻辑（Learned / Rule）

```
type: learned_tree
model: xgboost_v1.3
```

或（少数情况下）：

```
type: hard_rule

* hard → 需要人工 review

### 6️⃣ 失败成本定义（Failure Cost）⭐️⭐️⭐️（最重要）

```
false_reject_cost: LOW
false_accept_cost: HIGH
```

或更明确：

```
false_reject:
  impact: missed_opportunity
  tolerance: high

false_accept:
  impact: tail_loss / drawdown
  tolerance: very_low
```

### 为什么这是“灵魂字段”？

它决定了：

* 训练时 **label 偏向**
* threshold 选择
* 是否宁可多 veto

👉 **这是你对市场的“风险哲学”写进系统的地方**

This rule aims to block trend execution

but insufficient MFE under volatility expansion,

and sharp reversals.
```

### 为什么一定要写？
