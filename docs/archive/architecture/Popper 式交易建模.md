## 三、Level 1：Failure Budget（最重要）

> **任何 archetype，都必须允许它“在一部分世界里失败”。**

### 一个可执行定义（强烈建议你用）

对每个 archetype，定义：


解释：

* ❌ 不是“有失败就 deny”
* ✅ 是“**失败覆盖不能超过预算**”

### 怎么算 deny_coverage？

不是样本比例，而是：

* 时间覆盖（months）
* 或 regime 覆盖（vol × trend × liquidity buckets）

> BPC 在 2019–2024 覆盖了 72 个月
> deny rule 覆盖了 31 个月
> 👉 deny_coverage = 43% ❌ 超预算

**结果：即便失败是真的，也不能全部 deny**

## **为什么"失败优先"在根本上是对的**

### **1. 这符合风险管理的基本逻辑**

在任何成熟的系统中（不只是交易）：
- 航空安全：先识别所有可能导致坠机的因素
- 医疗系统：先排除禁忌症，再考虑治疗方案
- 工程设计：先定义失效模式（failure modes），再优化性能

**原因很简单：** 一次致命失败可以抹去无数次成功

---

### **2. 不对称性是真实存在的**

交易中有个残酷的事实：
- 赚钱的方式有限，但赔钱的方式无穷
- 一个好策略可能因为10种原因赚钱
- 但只要有1种致命缺陷就会爆仓

**这种不对称性决定了：** 
- 穷举"成功条件"是不可能的
- 但识别"死亡陷阱"是可行的

---

### **3. 统计上的稳定性差异是真实的**

文档说"失败比成功稳定10倍"，这不是夸张：

**为什么？**
- 成功需要"所有条件都对"（AND逻辑）
- 失败只需要"任一条件错误"（OR逻辑）

在数学上，OR逻辑的模式比AND逻辑的模式：
- 更容易观察
- 更少受噪音影响
- 更跨时间/跨市场稳定

class ContinuousLearningLoop:
    """
    持续学习循环：假设 → 测试 → 证伪/确证 → 更新
    """
    def run_learning_cycle(self, market_data: pd.DataFrame):
        while True:
            # 1. 生成/选择假设
            hypothesis = self.select_hypothesis_to_test()
            
            # 2. 设计严格测试
            test_design = self.design_crucial_test(hypothesis)
            
            # 3. 执行测试
            test_results = self.execute_test(test_design, market_data)
            
            # 4. 更新知识
            if test_results['falsified']:
                self.update_knowledge_base('falsified', hypothesis, test_results)
                new_hypothesis = self.generate_new_hypothesis(hypothesis, test_results)
            else:
                self.update_knowledge_base('corroborated', hypothesis, test_results)
                self.adjust_confidence_level(hypothesis, test_results)
            
            # 5. 等待新数据，继续循环
