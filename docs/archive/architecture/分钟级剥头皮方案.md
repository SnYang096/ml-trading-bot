# 分钟级剥头皮方案（非高频）

> 目标：在分钟/几十分钟/日内尺度提升交易频率，但避免进入 HFT（秒级/毫秒级）。
> 
> 适用场景：你已稳定 4H 结构体系，希望增加日内收益密度。

---

## 1) 时间尺度与可行性

### 推荐梯度（从稳到激进）

- **30–60 分钟（日内结构）**：最稳，信号密度提升但仍保留结构性。
- **5–15 分钟（剥头皮上限）**：可做，但对滑点/成本/噪声更敏感。
- **<5 分钟**：接近半高频，需要更严格执行与成本建模，不建议作为主线。

### 原则

> **分钟级策略的核心不是“信号强度”，而是“执行成本可控性”。**

---

## 1.5) “剥头皮”与“短周期结构交易”的边界

### 结论（可写进文档）

> **不是所有“几分钟/日内交易”都叫剥头皮。**  
> **真正的剥头皮是“微观结构摩擦套利”，而短周期结构交易仍然可以放进你的框架。**

### 可放进框架（短周期结构交易）

- 依赖结构叙事（趋势延续 / 失败 / 回收）
- 有明确的失效点与止损结构
- Router/Gate/Execution 仍然有意义

### 不适合放进框架（真实 scalping）

- 依赖盘口摩擦与队列优势（spread/queue/latency）
- 决策半衰期小于 Router 周期
- 收益完全被手续费与滑点支配

---

## 2) 标的选择（必须严格）

### 推荐
- **BTC / ETH**（深度、成交密度、滑点稳定）
- **Top 10~20 高流动性币**（需单独验证）

### 避免
- 小市值 / Meme / 低深度币（滑点吞噬收益）
- 交易密度低的标的（信号不稳定）

---

## 3) 核心特征清单（分钟级）

### A. 订单流与流动性（必须）
- `buy_volume`, `sell_volume`, `taker_buy_ratio`
- `cvd_slope_short`（短窗口 CVD 斜率）
- `liquidity_void_retracement`（流动性缺口回补）
- `absorption_strength`（大单吃掉但价格不动）
- `tick_count`, `volume_density`（成交密度）

### B. 微结构波动与结构
- `atr_short` / `range_pct`（短波动）
- `compression_score_short`（短压缩）
- `trend_r2_short`（微趋势稳定性）
- `micro_vwap_deviation`（价格偏离微 VWAP）

### C. 执行与成本
- `spread_proxy`（点差代理）
- `slippage_proxy`（滑点代理）
- `volatility_of_vol`（短波动的二阶波动）

---

## 4) 模型选择：树 + MLP 的分工（推荐）

### 推荐结构
- **Regime / Gate：树模型（或规则）**
- **Entry 评分：MLP（小模型）**

理由：
- 剥头皮最怕过拟合噪声 → Gate 必须稳定可解释
- MLP 用于“评分/排序”，不做硬门控

### Regime 怎么做（树 / 规则优先）

**输入（短窗口聚合）**：
- `trend_r2_short`
- `atr_short` / `compression_score_short`
- `taker_buy_ratio_short`
- `absorption_strength_short`

**输出（离散 regime）**：
- `micro_trend` / `micro_mean` / `micro_chop`

示例（规则版）：
```yaml
micro_regime:
  micro_trend:
    - trend_r2_20 > 0.65
    - compression_score_20 < 0.3
  micro_mean:
    - trend_r2_20 < 0.35
    - abs_dev_from_vwap > q70
  micro_chop:
    - trend_r2_20 < 0.45
    - atr_short < q40
```

### MLP 做什么
- 输入：订单流 + 微结构特征（最近 N bars 的统计）
- 输出：entry_score（0~1）
- **不直接决定“是否交易”，只排序/加权**

---

## 5) Gate 设计（核心层）

### 目标
- 过滤执行成本
- 避免微观“伪突破 / 假回归”

### 示例（执行门禁）

```yaml
gate_rules:
  allow_mode: any
  deny_if:
    - spread_proxy > q90
    - slippage_proxy > q85
    - atr_short > q95  # 过度波动，剥头皮不稳
    - tick_count < q20  # 低密度成交
  allow_if:
    - absorption_strength > q70
    - liquidity_void_retracement > q60
    - taker_buy_ratio between [0.45, 0.55]  # 避免单边
```

**关键原则**：
- Gate 用 `deny_if` 严格控制成本
- `allow_if` 使用 OR 提高机会

---

## 6) Execution 设计（快进快出）

### 执行规则
- **Entry**：只在 `micro_regime` 与 `gate` 同时成立时
- **Stop**：小止损（0.8~1.2 * ATR_short）
- **TP**：小止盈（1.0~1.5 * ATR_short）
- **Time stop**：3~6 根 bar 未到目标则退出
- **No pyramid**：禁止加仓

### 示例（执行模板）
```yaml
execution:
  stop_loss: 1.0 * atr_short
  take_profit: 1.2 * atr_short
  time_stop_bars: 4
  max_trades_per_day: 12
  cooldown_bars: 2
```

---

## 7) 是否复用 Router/Gate/Execution？

### 结论
- **可以复用结构，但职责要收敛**

**建议：**
- Router：只做分钟级 regime（不再用 4H 语义）
- Gate：作为主防线（成本、流动性、稳定性）
- Execution：轻仓、快进快出、时间止损

---

## 8) 具体示例：剥头皮 archetype（简化版）

```yaml
archetype: scalp_micro
required_conditions:
  - micro_regime in [micro_trend, micro_mean]

gate_rules:
  deny_if:
    - spread_proxy > q90
    - slippage_proxy > q85
    - tick_count < q20
  allow_if:
    - absorption_strength > q70
    - liquidity_void_retracement > q60
  allow_mode: any

evidence_rules:
  - taker_buy_ratio between [0.45, 0.55]
  - cvd_slope_short > 0

execution:
  stop_loss: 1.0 * atr_short
  take_profit: 1.2 * atr_short
  time_stop_bars: 4
  size_cap: 0.2
  pyramid: false
```

---

## 9) 最小落地建议（不复杂但可用）

### 最小版本（推荐）
- **树模型做 regime + gate**（解释性强）
- **MLP 做 entry_score**（但不直接决定交易）
- **严格执行规则**（小止损 + 时间止损）

### 为什么不是纯 MLP
- 分钟级噪声大
- 解释性差 → 易过拟合
- Execution 与成本要显式约束

---

## 10) 风险与注意事项

- 滑点成本必须内生，否则回测虚高
- 成本比信号重要
- 成交密度不足时，交易应直接 veto
- 不要跨标的统一模型
- 不要把剥头皮当作主收益来源
