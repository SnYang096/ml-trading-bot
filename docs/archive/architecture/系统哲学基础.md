# 极其罕见的「工程级交易哲学」

下文说明这组表述为何成立。

## 一、四句箴言不是口号，而是认知分层

| 箴言 | 含义 |
|------|------|
| **Price tells you WHAT** | 发生了什么 |
| **Volume tells you IF** | 是否被市场接受 |
| **Order flow tells you WHO** | 是谁在推动 |
| **Math tells you HOW CAREFULLY** | 系统应有多保守 |

这不是修辞，而是**信息论分层**：

| 层 | 问题 | 数据类型 |
|---|------|----------|
| WHAT | 发生了什么 | Price / Structure |
| IF | 是否被市场接受 | Volume / Participation |
| WHO | 是谁在推动 | Order flow / Trade cluster |
| HOW CAREFULLY | 系统应有多保守 | Math / Noise / Tail risk |

**关键点**

- 没有任何一层能「越权回答」别层的问题。
- 此前架构上的判断，本质上都在执行这条分权原则。

**可写入系统宪法的表述**

> Gate protects correctness.  
> Evidence ranks opportunity.  
> Execution decides exposure.  
> Math controls caution.

**一句话总结**

> 当系统已经学会约束自己，就不再需要预测自己。

## 二、Popper 系统：你不是在「调参」，而是在可证伪地进化

**持续学习循环**

> 假设 → 测试 → 证伪/确证 → 更新

这不是一句「科研味」的话，在系统里是真实机制：

| Popper 步骤 | 在你系统里的实体 |
|-------------|------------------|
| 假设 | Archetype / Rule / Gate |
| 测试 | Tree / Backtest / Failure audit |
| 证伪 | failure_rr_extreme / no_opportunity |
| 更新 | 宪法条款 / rule removal / weight shift |

**核心表述**

> 「下一轮训练，只针对宪法条款覆盖不到的 failure。」

这在哲学上意味着：系统不是在「最大化收益」，而是在**最小化已知错误之外的未知错误**——这是智慧系统的定义，不是策略。

## 三、可直接使用的系统哲学总结（Manifesto）

以下段落可置于 README、系统总设计文档，或作为整个量化体系的宣言。

### System Philosophy / Manifesto（英文稿）

This system is not optimized for profit, but for survivable learning.

**Market interpretation is layered:**

- Price tells us **WHAT** is happening.
- Volume tells us **IF** the move is accepted.
- Order flow tells us **WHO** is behind it.
- Math tells us **HOW CAREFULLY** we should act.

Each layer answers a non-overlapping question. No feature is allowed to speak outside its epistemic domain.

**Learning via Falsification (Popper System)**

The system evolves through a continuous falsification loop:

> Hypothesis → Test → Falsification / Confirmation → Update

- Hypotheses are expressed as archetypes, rules, and gates.
- Tests are executed via backtests and failure audits.
- Failures are explicitly labeled and categorized.
- Updates only target failures not yet covered by the system constitution.

We do not retrain to improve success. We retrain to eliminate avoidable failure.

**Risk before Return**

- Mathematical features (WPT, Hilbert, Hurst, EVT) are not signals. They are execution modifiers and risk dampeners.
- EVT acts as a circuit breaker, not a predictor.
- The system never asks math whether to trade. It only asks math how much to trust itself.
