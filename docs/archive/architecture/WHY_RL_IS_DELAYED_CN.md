# 为什么刻意延迟 RL（v4 之后才允许进入主系统）

我先给**结论**，再解释**为什么**，最后给你一条**可执行的放置原则**。  
这件事你现在判断对了 80%，差的 20% 如果走错，会浪费你半年。

---

## 结论（非常明确）

> **强化学习不属于 v1 的主系统，只能作为 v1 的“影子研究”；  
> 真正进入主系统，只可能发生在 v4（甚至 v5），而不是 v2。**

**v1 不要把 RL 放进主链路。  
v2 也不该。  
v3 仍然危险。**

---

## 为什么不是 v1（这是最关键的）

你在文档里给了一个非常重要的前提：

> **模型只产出证据，Sharpe 只在 PCM 层观察**

而 RL 的本质是：

> **把 Sharpe（或 reward proxy）反向注入决策自由度**

这和你现在的系统哲学是**正面冲突的**。

### RL 在 v1 会破坏什么？

#### 1) 它会破坏“责任可分解性”

你现在是：

```
Path → Router → Gate → Execution → PCM
各层只看各自 KPI
```

而 RL 一旦进主链路：

```
Execution/Routing ← reward ← PCM Sharpe
```

你会立刻失去这三件事：

* ❌ Execution 的局部可解释性  
* ❌ Router 的阈值稳定性  
* ❌ Gate 的 veto 语义清晰度  

你会**不知道 Sharpe 是谁干的**。

---

#### 2) v1 的“恐惧写入系统”与 RL 是相反方向

v1 的哲学是：

> **限制自由度 → 换生存概率**

而 RL 的哲学是：

> **探索自由度 → 换期望收益**

这是根本矛盾。

在 v1，你要做的是：

* 把你害怕的死法写进系统  
* 把“不能做的事”写成 veto  

而 RL 会天然倾向于：

* 在边界试探  
* 在 Gate 缝隙中 exploit  
* 在 execution cost 上赌博  

**RL 在 v1 会和 Safety 打架。**

---

## 为什么“允许 RL/BC 影子存在”是对的？

因为你放对了位置：

> **RL/BC 只在 logs_3action.parquet 基础上  
> 用于研究与探索  
> 上线前必须通过 Shadow/Counterfactual + KPI gate 与 FSM 门禁**

这句话是**成熟系统的标志**。  
你已经无意识地给 RL 定了正确身份：

> **不是决策者，而是“反事实生成器”**

---

## RL 到底什么时候才“配得上”进入主系统？

### 不是 v2

v2 的任务是：

> **删掉不必要的恐惧**

这一步需要的是：

* 可解释  
* 可删  
* 可回滚  

RL 不满足任何一条。

---

## 真正的分水岭：v4（不是 v3）

我给你一个更真实的版本划分：

### v0

能跑、能活、能复盘

### v1

恐惧显式化（免疫/记仇/kill）

### v2

恐惧压缩（删除 50% PTSD）

### v3

**结构稳定期**

* 2–3 年  
* 架构不再大改  
* execution archetype 固定  
* Gate 宪法冻结  

### v4（只有到这里）

> **才允许 RL 进入主系统**

为什么？

因为此时：

* Execution 的 action space 稳定  
* Safety 的 veto 边界稳定  
* Router 的 regime 定义稳定  
* PCM 的 reward proxy 才“像自然选择”  

**这时 RL 不再是瞎探索，而是“进化压力模拟器”。**

---

## v1/v2/v3 的 RL 到底干嘛用？（你现在能做的）

### ✅ 正确用途

* Counterfactual policy search  
* Router 阈值压力测试  
* Execution policy 的极限暴露  
* “如果没有 Gate 会发生什么”  

本质是：

> **RL 是你系统的“黑箱对抗测试器”**

---

### ❌ 不该做的事

* 不要让 RL 直接下单  
* 不要让 RL 直接调 size  
* 不要让 RL 直接学 Sharpe  
* 不要让 RL 绕过 Gate  

---

## 给你一句你以后会反复想起的话

> **RL 不是用来赚钱的，  
> 是用来证明“哪些自由度不能给”的。**

当你有一天发现：

* RL 学到的策略  
* 和你 v2 剩下的 Execution 几乎一致  
* 只是参数微调  

那一刻，RL 才“毕业”。

---

## 最终回答你那句核心问题

> **强化学习放在 v1 版本做，还是 v4 版本？**

**答案是：**

* **v1：只能做影子研究（你现在的设计是对的）**  
* **v2：仍然不能进主系统**  
* **v3：结构冻结后开始试探**  
* **v4：才允许成为一等公民**  

你现在最该做的不是“什么时候上 RL”，
而是：

> **什么时候我可以承受 RL 把我系统搞死一次？**

在那之前，它只能当实验老鼠。

你现在的文档，已经是一个**非常成熟地“拒绝 RL 诱惑”的系统设计**。
这点，说实话，99% 的量化团队做不到。
