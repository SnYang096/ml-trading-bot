## 问题1：未来波动率标签计算

### 现象

从模型对比日志看到：
```
Predicted mean: 0.964, Actual mean: 0.000
```

实际波动率均值为0，这明显不正常。

### 可能原因

1. **数据对齐问题**：
   - 未来波动率标签和预测波动率的时间索引不对齐
   - 导致比较时大部分是NaN

2. **计算逻辑问题**：
   - `future_volatility_label` 函数可能有问题
   - 或者数据本身有问题

3. **单位不匹配**：
   - 预测波动率是绝对值（价格 × 相对波动率）
   - 未来波动率标签可能是相对值
   - 需要统一单位

## 问题2：保本止损未触发

### 当前保本止损逻辑

对于Long信号：
1. **初始止损**：`entry_price - 1.25 × pred_vol`
2. **保本触发条件**：价格达到 `entry_price + 1.25 × pred_vol`
3. **保本止损**：移动到 `entry_price`（保本）

### 为什么未触发？

#### 原因1：触发条件过于严格

需要价格移动 `1.25 × pred_vol` 才能触发保本，这个距离可能太大。

**示例**：
- 如果 `pred_vol = 1000`（绝对波动率）
- 触发点 = `entry_price + 1.25 × 1000 = entry_price + 1250`
- 如果价格只移动了800，就达不到触发点

#### 原因2：交易在触发前就结束了

可能的情况：
1. **先触达止盈**：价格达到 `entry_price + 3.0 × pred_vol`，交易结束
2. **先触达止损**：价格回到 `entry_price - 1.25 × pred_vol`，交易结束
3. **达到最大持仓期**：在触发前就达到 `max_holding_bars`

#### 原因3：预测波动率偏小

如果预测波动率偏小，触发点会更远，更难达到。

### 分析建议

```python
# 分析保本触发情况
for trade in trades:
    entry_price = trade.entry_price
    pred_vol = trade.predicted_vol
    trigger_level = entry_price + 1.25 * pred_vol
    
    # 检查是否达到触发点
    max_price = df.loc[trade.entry_idx:trade.exit_idx, "high"].max()
    reached_trigger = max_price >= trigger_level
    
    if not reached_trigger:
        # 分析为什么没达到
        distance_to_trigger = trigger_level - max_price
        print(f"未达到触发点，距离: {distance_to_trigger}")
```

## 问题3：Optuna优化有效性

### 核心问题

**用Optuna优化R/R参数是否有效？还是只是"炼丹"？**

### 答案：取决于优化方法

#### ❌ 无效的方法（过拟合）

1. **在测试集上直接优化**：
   ```python
   # ❌ 错误：在测试集上优化
   study.optimize(lambda trial: evaluate_on_test_set(trial), n_trials=100)
   ```
   - 这是典型的过拟合
   - 找到的参数只对测试集有效

2. **使用未来信息**：
   ```python
   # ❌ 错误：使用未来波动率优化
   study.optimize(lambda trial: evaluate_with_future_vol(trial), n_trials=100)
   ```
   - 实盘无法获得未来信息
   - 优化结果无法复现

#### ✅ 有效的方法（真实有效）

1. **时间序列交叉验证（TSCV）**：
   ```python
   # ✅ 正确：使用TSCV
   def objective(trial):
       stop_loss_r = trial.suggest_float("stop_loss_r", 0.5, 2.0)
       take_profit_r = trial.suggest_float("take_profit_r", 1.0, 5.0)
       
       # 使用TSCV，每个fold的验证集都是"未来"数据
       tscv = TimeSeriesSplit(n_splits=5)
       scores = []
       for train_idx, val_idx in tscv.split(df_train):
           model = train_model(df_train.iloc[train_idx])
           results = evaluate(df_train.iloc[val_idx], model, stop_loss_r, take_profit_r)
           scores.append(results["sharpe_ratio"])
       
       return np.mean(scores)
   
   study = optuna.create_study(direction="maximize")
   study.optimize(objective, n_trials=50)
   ```

2. **滚动窗口优化**：
   ```python
   # ✅ 正确：滚动窗口优化
   window_size = 1000
   step_size = 200
   
   all_params = []
   for start in range(0, len(df) - window_size, step_size):
       window_df = df.iloc[start:start+window_size]
       
       # 在每个窗口上独立优化
       study = optuna.create_study(direction="maximize")
       study.optimize(lambda trial: objective_for_window(window_df, trial), n_trials=30)
       
       best_params = study.best_params
       all_params.append(best_params)
   
   # 使用参数的中位数作为最终参数
   final_stop_loss_r = np.median([p["stop_loss_r"] for p in all_params])
   final_take_profit_r = np.median([p["take_profit_r"] for p in all_params])
   ```

3. **样本外测试**：
   ```python
   # ✅ 正确：样本外测试
   # Step 1: 只在训练集上优化
   study.optimize(objective, n_trials=50)  # 只在训练集上
   best_params = study.best_params
   
   # Step 2: 在完全独立的测试集上验证（不参与优化）
   final_results = evaluate(df_test_unseen, model, **best_params)
   ```

### 有效性判断标准

#### ✅ 有效的优化应该满足：

1. **时间序列划分**：训练集 < 测试集（时间上）
2. **交叉验证**：使用TSCV，避免未来信息泄漏
3. **样本外测试**：测试集不参与优化
4. **参数稳定性**：不同时间窗口优化出的参数应该相似
5. **性能一致性**：训练集和测试集性能差距不应过大（<20%）

#### ❌ 无效的优化特征：

1. **在测试集上优化**：直接过拟合测试集
2. **使用未来信息**：实盘无法复现
3. **参数不稳定**：不同窗口优化出完全不同的参数（如 stop_loss_r 从0.5到2.0）
4. **性能差距大**：训练集Sharpe=2.0，测试集Sharpe=0.3（过拟合）
