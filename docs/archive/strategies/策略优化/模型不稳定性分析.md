# ML模型不稳定性分析

## 问题描述

每次训练完测试，结果都不一样，ML模型不稳定。

## 根本原因分析

### 1. 数据量不足 ⚠️

**当前数据量**：
- 总样本数：3102
- 特征数：37
- 正样本：951 (30.66%)
- 负样本：2151 (69.34%)

**问题**：
- 样本/特征比：83.8（建议 > 100-200）
- 正样本/特征比：25.7（建议 > 50-100）
- 对于时间序列数据，3102个样本可能不足以稳定训练

**影响**：
- 小样本导致模型对训练数据敏感
- 交叉验证时不同折的差异会被放大
- 特征噪声影响更大

### 2. 随机种子未固定 ⚠️

**问题**：
- LightGBM 的 `random_state` 未设置
- `bagging_fraction` 和 `feature_fraction` 的随机性未控制
- 交叉验证划分可能有随机性

**影响**：
- 每次训练结果不同
- 无法复现实验结果
- 难以判断模型改进是否有效

### 3. 模型预测能力弱 ⚠️

**当前表现**：
- AUC: 0.4977 ± 0.0733（接近随机 0.5）
- 说明模型几乎没有学到有效信号

**影响**：
- 当模型预测能力很弱时，小的随机性变化会导致结果差异很大
- 预测值分布不稳定（min=0.1792, max=0.5794）
- 阈值选择对结果影响很大

### 4. 交叉验证折数不足 ⚠️

**当前设置**：
- `n_splits=5`（5折交叉验证）

**问题**：
- 对于3102个样本，5折意味着每折约620个样本
- 验证集太小，导致评估不稳定
- 不同折之间的差异会被放大

## 验证方法

### 稳定性测试

运行10次相同的训练，检查：
- AUC 的标准差（应该 < 0.01）
- 预测值的分布（应该一致）
- 交易结果的差异（应该 < 5%）

### 代码示例

```python
# 稳定性测试
results = []
for seed in range(10):
    np.random.seed(seed)
    model = train_ml_model(X_train, y_train, ...)
    results.append(evaluate_model(model, X_test, y_test))

# 检查稳定性
aucs = [r['auc'] for r in results]
print(f"AUC mean: {np.mean(aucs):.4f}, std: {np.std(aucs):.4f}")
assert np.std(aucs) < 0.01, "Model is unstable!"
```

## 总结

模型不稳定的主要原因：
1. **数据量不足**（3102样本 vs 37特征）
2. **随机种子未固定**
3. **模型预测能力弱**（AUC ≈ 0.5）
4. **CV折数不足**（5折）

**立即行动**：
- 固定随机种子
- 增加CV折数到10

**长期改进**：
- 增加数据量（多符号、多时间框架）
- 特征选择
- 集成学习
