# 最终目录结构设计（已实施）

## 核心设计理念

1. **特征工程独立管理**：`src/features/` 专门负责特征工程
2. **策略在模型模块下**：`src/time_series_model/strategies/` 包含时序策略
3. **数据工具独立**：`src/data_tools/` 只负责数据加载

## 模块职责划分

### 1. `src/features/` - 特征工程模块

**职责：**
- 所有特征工程代码的统一管理
- 时序特征和横截面特征的分离
- 可以被多个模块使用（strategies、backtesting、analysis）

**包含文件：**
- `baseline_features.py`: 基础 SR 和压缩特征
- `enhanced_features.py`: 增强特征（WPT、Hurst、Hilbert、Spectrum）
- `comprehensive_features.py`: 综合特征工程器
- `feature_engineering.py`: 基础特征工程
- `feature_engineering_talib.py`: TA-Lib 特征
- `dl_sequence_features.py`: 深度学习序列特征

### 2. `src/time_series_model/strategies/` - 时序策略模块

**职责：**
- 策略的标签设计
- 策略的模型训练
- 具体策略的实现

**包含：**
- `labels/`: 4类标签（SR反转、SR突破、压缩区突破、趋势跟踪）
- `models/`: 统一模型训练器（支持 XGBoost/CatBoost/LightGBM）
- `strategies/`: 4个具体策略实现

### 3. `src/data_tools/` - 数据工具模块

**职责：**
- 数据加载
- 数据预处理
- 基础工具函数

## 导入路径示例

### 特征工程


```makefile

python3 -m src.time_series_model.strategies.strategies.sr_reversal.train
```

## 优势

### 1. 改动更小

- ✅ 利用现有的 `time_series_model` 目录
- ✅ 不需要创建新的 `strategies` 目录
- ✅ 减少目录层级

### 2. 职责清晰

- **features/**：专门负责特征工程
- **time_series_model/strategies/**：专门负责策略实现
- **data_tools/**：专门负责数据加载

### 3. 易于扩展

- 新增时序策略：在 `time_series_model/strategies/strategies/` 下添加
- 新增横截面策略：可以创建 `cross_sectional_model/strategies/`（未来）
- 特征工程独立，策略可复用

### 4. 避免重复

- ✅ 特征工程代码统一管理，不再分散在多个地方
- ✅ 不再有 `data_tools` 和 `strategies` 的重复实现
