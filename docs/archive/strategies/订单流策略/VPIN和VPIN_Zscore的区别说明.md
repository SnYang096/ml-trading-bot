# VPIN 和 VPIN Z-score 的区别说明

## 一、核心区别

### 1. `vpin`（原始值）
- **范围**：[0, 1]
- **含义**：绝对的不平衡度
- **计算**：`|buy_volume - sell_volume| / total_volume`
- **特点**：
  - 0 = 完全平衡
  - 1 = 完全不平衡（全买或全卖）
  - 值越大，不平衡越严重

### 2. `vpin_zscore_20`（Z-score 标准化）
- **范围**：(-∞, +∞)，通常 [-3, 3]
- **含义**：相对于最近20期的异常程度
- **计算**：`(vpin - mean(vpin_last_20)) / std(vpin_last_20)`
- **特点**：
  - 0 = 正常水平（等于平均值）
  - > 0 = 高于平均水平（异常高）
  - < 0 = 低于平均水平（异常低）
  - 绝对值越大，异常程度越高

## 二、实际应用场景对比

### 场景 1：市场长期高波动
假设某市场长期 VPIN 都在 0.6-0.8 之间（高波动市场）

| 时刻 | vpin | vpin_zscore_20 | 含义 |
|------|------|----------------|------|
| t1 | 0.7 | 0.0 | 正常水平（等于平均值） |
| t2 | 0.8 | +2.0 | **异常高**（突破信号） |
| t3 | 0.6 | -1.5 | 低于平均水平 |

**结论**：
- `vpin = 0.8` 在绝对值上很高，但在该市场中是"正常"的
- `vpin_zscore = +2.0` 表示相对于历史，这是**异常高**，更可能是突破信号

### 场景 2：市场长期低波动
假设某市场长期 VPIN 都在 0.2-0.4 之间（低波动市场）

| 时刻 | vpin | vpin_zscore_20 | 含义 |
|------|------|----------------|------|
| t1 | 0.3 | 0.0 | 正常水平 |
| t2 | 0.5 | +3.0 | **异常高**（突破信号） |
| t3 | 0.2 | -1.0 | 低于平均水平 |

**结论**：
- `vpin = 0.5` 在绝对值上不算很高，但在该市场中是**异常高**
- `vpin_zscore = +3.0` 能更好地捕捉这种异常

## 三、交叉特征的区别

### `vpin_x_trade_cluster_max_buy_run`
```python
vpin (0-1) × max_buy_run (0-100)
```
- **含义**：高不平衡度 × 连续买入
- **适用场景**：需要识别"高"的订单流不平衡（绝对值）
- **问题**：在不同市场环境下，相同的 vpin 值可能代表不同的异常程度

### `vpin_zscore_x_trade_cluster_max_buy_run`
```python
vpin_zscore (-3 to +3) × max_buy_run (0-100)
```
- **含义**：异常高的不平衡度 × 连续买入
- **适用场景**：需要识别"异常"的订单流不平衡（相对值）
- **优势**：能适应不同市场环境，自动识别异常情况

## 四、为什么不同策略选择不同？

### SR 突破策略 → `vpin_zscore_x_trade_cluster_max_buy_run`
**原因**：
- 突破需要识别"异常"信号，而不仅仅是"高"信号
- 不同市场环境下，突破的 VPIN 阈值不同
- Z-score 能自动适应市场环境，识别真正的异常

**示例**：
- 低波动市场：vpin=0.5 可能已经是异常高（zscore=+3）
- 高波动市场：vpin=0.7 可能只是正常水平（zscore=0）

### 压缩区突破策略 → `vpin_x_trade_cluster_max_buy_run`
**原因**：
- 压缩区突破需要识别"高"的订单流，不一定是异常
- 压缩期通常 VPIN 较低，突破期 VPIN 较高
- 使用绝对值可以更好地捕捉这种"从低到高"的变化

**示例**：
- 压缩期：vpin=0.2（低）
- 突破期：vpin=0.6（高）
- 即使 zscore 不高，但绝对值的变化已经很明显

## 五、建议

### 方案 A：统一使用 Z-score 版本（推荐）✅
**理由**：
1. **更通用**：Z-score 既能识别高值，也能识别异常值
2. **自适应**：自动适应不同市场环境
3. **减少冗余**：避免两个相似特征

**修改**：
- 压缩区突破策略也使用 `vpin_zscore_x_trade_cluster_max_buy_run`
- 移除 `vpin_x_trade_cluster_max_buy_run`（或保留作为备选）

### 方案 B：保留两个，但明确用途
**理由**：
1. **不同场景**：绝对值 vs 相对值
2. **模型选择**：让模型自动选择有用特征

**建议**：
- 两个都加载，让模型通过特征重要性选择
- 如果某个特征重要性很低，再移除

## 六、最终建议

**推荐方案 A**：统一使用 Z-score 版本

**原因**：
1. Z-score 版本更通用，能适应不同市场环境
2. 减少特征冗余，降低模型复杂度
3. 用户建议的特征就是 Z-score 版本，说明其重要性

**实施**：
- 压缩区突破策略改为使用 `vpin_zscore_x_trade_cluster_max_buy_run`
- 保留 `vpin_x_trade_cluster_max_buy_run` 在配置文件中作为备选（注释掉）
