# 简化训练函数使用指南

## 当前 `train_lightgbm_model` 的功能

`train_lightgbm_model` 函数支持以下任务类型：

1. **多类分类（Multiclass）**：3类分类（0=Hold, 1=Long, 2=Short）
2. **二分类（Binary）**：2类分类（0/1）
3. **回归（Regression）**：连续值预测（收益率、波动率等）

## 你需要的三种模型

### 1. 三分类模型（多空Hold）

```python
from time_series_model.utils.training import train_lightgbm_model

# 方法1：使用 preserve_multiclass=True
model = train_lightgbm_model(
    X_train,
    y_train,  # 标签：0=Hold, 1=Long, 2=Short
    X_val=X_val,
    y_val=y_val,
    preserve_multiclass=True,  # 关键：保留3类分类
    params={"num_class": 3},  # 明确指定3类
)

# 方法2：明确指定 task_type
model = train_lightgbm_model(
    X_train,
    y_train,
    X_val=X_val,
    y_val=y_val,
    task_type="multiclass",  # 明确指定任务类型
    params={"num_class": 3},
)
```

### 2. 波动率模型（回归）

```python
model = train_lightgbm_model(
    X_train,
    y_train,  # 波动率值（连续值）
    X_val=X_val,
    y_val=y_val,
    task_type="regression",  # 明确指定回归任务
    params={"objective": "regression", "metric": "rmse"},
)
```

### 3. 收益率模型（回归）

```python
model = train_lightgbm_model(
    X_train,
    y_train,  # 收益率值（连续值）
    X_val=X_val,
    y_val=y_val,
    task_type="regression",  # 明确指定回归任务
    params={"objective": "regression", "metric": "rmse"},
)
```

## 函数复杂性说明

### 为什么这么复杂？

1. **自动检测任务类型**：根据标签自动判断是分类还是回归
2. **历史兼容性**：为了兼容旧代码，默认将3类分类转换为2类分类
3. **GPU/CPU 回退**：自动处理 CUDA 错误，回退到 CPU
4. **分类特征处理**：支持多种分类特征输入格式

### 简化建议

如果你只需要这三种模型，可以：

1. **明确指定 `task_type`**：避免自动检测的复杂性
2. **使用 `preserve_multiclass=True`**：确保3类分类不被转换为2类
3. **直接使用 LightGBM**：如果函数太复杂，可以直接使用 `lgb.train()`
