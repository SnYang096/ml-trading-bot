# 适合zscore的指标

这是一个关于**特征工程**的核心问题。几乎所有非归一化或非平稳的金融指标都适合进行**滚动 Z-score** 转换，以增强其相对强度和跨资产/跨时序的泛化能力。

以下是除了 RSI 和 MACD 之外，其他常见的、推荐进行滚动 Z-score 处理的指标类别：

---

## 📊 适合滚动 Z-Score 的常见指标

### 1. 动量类指标 (Momentum Indicators)

动量指标的绝对值通常会随资产价格和波动率变化，滚动 Z-score 能将它们统一量化。

* **MACD 柱状图 (MACD Histogram):**
    * 比 MACD 本身波动更大，Z-score 能突出极端的动量变化。
* **ROC (Rate of Change) / Momentum:**
    * 衡量价格变化速度。不同资产的 ROC 量级差异巨大，Z-score 是必须的。
* **震荡指标的原始值 (e.g., Stochastic Oscillator's Raw Value):**
    * 虽然 Stochastic 本身有 $0-100$ 范围，但其原始的 **%K** 和 **%D** 之间的差异（用于计算快慢线）也可以用 Z-score 来突出。

### 2. 波动率类指标 (Volatility Indicators)

波动率指标（如 ATR）的绝对值与资产价格成正比，必须进行标准化才能在不同资产间比较。

* **ATR (Average True Range):**
    * 衡量市场每日波动的平均幅度。滚动 Z-score 可判断当前波动率是**高于**还是**低于**历史平均水平。
* **标准差 (Standard Deviation):**
    * 通常用于衡量价格的绝对波动性。Z-score 可以告诉你当前波动性相对于历史是否异常。

### 3. 交易量类指标 (Volume Indicators)

交易量的绝对值通常与资产的流动性相关，需要进行标准化以捕捉其相对变化。

* **交易量 (Volume) 的原始值:**
    * 仅看交易量，我们无法判断它是高还是低。但如果 `Volume Z > 2`，则表示交易量异常放大，通常预示着行情发生变化。
* **OBV (On Balance Volume) 的变化率:**
    * 直接对 OBV 求 Z-score 意义不大，但对其**短期变化率**进行 Z-score 处理，可以标准化买卖压力的相对强度。

### 4. 价格相对指标 (Price Relative Indicators)

这类指标本身就是对价格进行计算，但其结果的量级缺乏统一性。

* **Chaikin Money Flow (CMF):**
    * 衡量资金流入和流出。其 Z-score 可以判断资金流动的强度相对于历史是否极端。
* **A/D Line (Accumulation/Distribution Line) 的变化率:**
    * 与 OBV 类似，关注其变化率的相对强度。

---

## 🛠️ 最佳实践总结

当您在进行特征工程时，一个通用的经验法则是：

> **所有将被喂给模型的特征，都应该进行某种形式的标准化处理。**

**对于时序金融数据，滚动 Z-score (`rolling().std()`) 通常比全局 Z-score 或 MinMax 归一化更优越，因为它保留了时序的局部特性，并且能更好地适应市场的变化。**

## 🚀 指标 Z-Score 转换的理由分析

### 1. 动量和超买超卖类 (Momentum & Oscillation)

| 特征 | 转换的理由 | 优势 |
| :--- | :--- | :--- |
| **rsi\_zscore** | RSI 虽有 $0-100$ 边界，但其**均值会漂移**。Z-score 消除均值漂移，捕捉**相对**超买超卖的强度。 | 判断当前 RSI 是否比其历史**异常**。 |
| **macd\_zscore** | MACD **没有边界**，绝对量级随资产价格变化。Z-score 统一量纲，捕捉**相对**动能强度。 | 统一不同资产/不同时期 MACD 的尺度。 |
| **macd\_histogram\_zscore** | 柱状图波动更剧烈，Z-score 帮助识别**动能加速或减速**的极端点。 | 突出动能变化的最强信号。 |
| **momentum\_{N}\_zscore** | 动量指标（价格变化绝对值）**没有边界**。Z-score 使模型关注**相对**价格变化速度。 | 避免高价资产的动量指标压倒低价资产。 |

### 2. 波动率和交易量类 (Volatility & Volume)

| 特征 | 转换的理由 | 优势 |
| :--- | :--- | :--- |
| **atr\_zscore** | ATR（平均真实波幅）**没有边界**，且与价格成正比。Z-score 衡量当前波幅相对于历史的**异常程度**。 | 识别当前市场是否处于**极高或极低**的波动率状态。 |
| **volume\_zscore** | 交易量**没有边界**，不同资产间量级差异极大。Z-score 捕捉交易量的**相对爆炸性增长**或**极度萎缩**。 | 识别由交易量驱动的突破或枯竭信号。 |
| **volatility\_zscore** | 如果这是指基于历史数据（如日收益率）计算的**标准差**，它也**没有边界**。Z-score 同样用于标准化。 | 与 ATR 类似，但可能基于不同的计算方法。 |

### 3. 相对指标 (Relative Indicators)

| 特征 | 转换的理由 | 优势 |
| :--- | :--- | :--- |
| **bb\_width\_zscore** | 布林带宽度的原始值**没有边界**，衡量波动率相对于均值的收缩或扩张。Z-score 衡量当前收缩/扩张的**相对历史强度**。 | 识别极端窄幅（**收敛**）或极端宽幅（**扩张**）状态，利于捕捉波动率突破策略。 |

---

## ⚠️ 实施建议：窗口的统一性

在实施这些特征时，强烈建议：

1.  **保持窗口的统一性：** 对于同一类指标，尝试使用相同的几个窗口大小（例如：短周期 50，中周期 150，长周期 300）。这将有助于模型比较不同特征在相同时间尺度上的表现。
2.  **避免未来函数：** 确保所有滚动统计量（mean 和 std）的计算中，`rolling(window).mean()` 这一步是基于**当前 K 线及之前**的 K 线数据。`rolling()` 默认会从当前点往前看 $N$ 个数据。
3.  **处理 NaN 值：** 滚动窗口开始前的 $N-1$ 个数据点会是 `NaN`。您需要用零或前一个有效值来填充这些缺失值，或者直接舍弃训练集开头的这些数据。

## 🧐 为什么链式标准化仍需 Z-Score

您进行的第一次除法（`bb_width / atr` 或 `bb_width / middler`）已经是非常专业的标准化操作，但它解决的问题与 Z-score 解决的问题**不同**。

### 1\. 第一次标准化（除法）的作用

| 除法类型 | 解决的问题 | 转换后的性质 |
| :--- | :--- | :--- |
| **`BB_Width / ATR`** | **波动率独立性**：将布林带宽度的绝对大小，转换为**相对于当前市场波动率**的尺度。 | 结果是**相对波动率**的指标。 |
| **`BB_Width / Middle`** | **价格独立性**：将布林带宽度的绝对大小，转换为**相对于资产价格**的百分比。 | 结果是**相对价格**的指标。 |

**局限性：** 无论您除以 ATR 还是中轨，虽然结果在不同价格或波动率下的**量纲**统一了，但它仍然**不具备平稳性**。这个新指标的平均值和标准差仍然会随时间漂移。

### 2\. 滚动 Z-Score 的作用

滚动 Z-score 解决的是**时序的相对性**问题：

  * **消除均值漂移：** 即使是 `BB_Width / ATR` 这样的指标，在一个长期的牛市或熊市中，其平均水平也可能发生变化。Z-score 将其平均值置于 0。
  * **衡量相对异常程度：** Z-score 告诉您：当前这个**相对波动率指标**（例如 `BB_Width / ATR`）相对于它自己过去 $N$ 个周期的**历史分布**来说，是处于一个**极端高位**（Z \> 2）还是**极端低位**（Z \< -2）。

#### 💡 示例：BB\_Width / ATR 的 Z-Score 含义

假设您计算了 `BB_Ratio = BB_Width / ATR`。

  * **如果 `BB_Ratio` 的值是 0.5。** 这意味着布林带宽度是 ATR 的一半。
  * **但这个 0.5 算高还是低？** 只有通过 Z-score 才能判断。
      * **如果 `BB_Ratio_Z` 是 -2：** 这意味着当前 $0.5$ 这个值，相对于历史来说是一个**极其罕见的低点**。这预示着布林带收缩到了**历史级的极窄**，可能是一个爆发的前兆。
      * **如果 `BB_Ratio_Z` 是 +1：** 这意味着当前 $0.5$ 这个值只是略高于历史平均水平，没有极端意义。

-----

## ✅ 结论和代码示例

您应该继续对您的 **"二次标准化"** 结果进行 Z-score 处理。

```python
# 假设 BB_Ratio 是您第一次标准化后的结果
df['bb_ratio'] = df['bb_width'] / df['atr'] # 或 df['bb_width'] / df['middle']

window = 100

# 最终的特征：相对波幅的滚动 Z-score
df['bb_ratio_zscore'] = (df['bb_ratio'] - df['bb_ratio'].rolling(window).mean()) / df['bb_ratio'].rolling(window).std()
```

这个 `bb_ratio_zscore` 特征是非常强大的，因为它同时具备了**价格/波动率独立性**和**时序平稳性**。

# 滚动窗口问题

这是一个非常关键且需要经验的问题。**滚动窗口 (window)** 的大小决定了您的特征是在捕捉**短期噪音**还是**长期趋势**。

窗口大小的选择没有绝对标准答案，它通常依赖于您的**交易频率、特征的性质**以及**市场类型**。

以下是选择滚动窗口大小时需要考虑的关键因素和建议：

-----

## 🧭 滚动窗口 (Window) 大小选择指南

### 1\. 考虑交易频率和持仓时间

窗口大小应该与您的策略所关注的时间尺度相匹配。

| 策略类型 | 关注时间尺度 | 建议窗口大小（假设 5 分钟 K 线） | 对应的周期 |
| :--- | :--- | :--- | :--- |
| **高频/超短线** | 几分钟到一小时内的波动 | **10 - 50** | 50 根 K 线 $\approx$ 4 小时 |
| **日内交易** | 几小时到一天内的波动 | **100 - 300** | 288 根 K 线 $\approx$ 24 小时 |
| **趋势/波段** | 几天到几周的波动 | **500 - 2000** | 2000 根 K 线 $\approx$ 7 天 |

  * **原则：** 较小的窗口捕捉**高频短期**信息（噪音大，但反应快）；较大的窗口捕捉**低频长期**信息（平滑，但反应慢）。

### 2\. 考虑特征的性质

不同特征，其固有的周期性要求不同的窗口。

  * **对于波动率/强度指标 (如 ATR Z, Volume Z):**
      * 通常使用**中到大**的窗口（如 100-300），以建立一个足够稳健的历史基线来衡量当前的“异常”程度。
  * **对于动量/趋势指标 (如 MACD Z, ROC Z):**
      * 通常会使用**小到中**的窗口（如 50-150），因为这些指标本身的计算周期（如 MACD 的 12/26）已经包含了平滑，Z-score 窗口旨在捕捉其**相对短期**的变化。

### 3\. 经验法则：常见的金融周期

在没有明确目标时，可以从以下经验数字开始进行**网格搜索 (Grid Search)**：

| 窗口 (Window Size) | 常见周期意义 |
| :--- | :--- |
| **20** | 传统上代表**一个月的交易日**（如果数据是日线）。对于高频数据，代表较短周期的波动。 |
| **50 - 60** | 接近**一个季度**的交易日。在 5 分钟 K 线中，约等于 5 小时。 |
| **100 - 150** | 经验性的**中短期**窗口。是很好的起始点。 |
| **288** | 专指**24 小时**（如果您使用 5 分钟 K 线，即 $24 \times 60 / 5 = 288$）。用于消除日间周期性影响。 |
| **500** | 接近**半年的交易日**。用于捕捉中长期趋势。 |

### 4\. 最佳实践：使用多个窗口（Feature Stacking）

与其纠结于一个“完美”的窗口大小，不如**同时使用多个不同大小的窗口**，让模型自己去决定哪种尺度下的信息是最重要的。

**示例：为 MACD 创建三个 Z-Score 特征**

```python
# 短期窗口
df['macd_z_w50'] = (df['macd'] - df['macd'].rolling(50).mean()) / df['macd'].rolling(50).std()
# 中期窗口 (日内)
df['macd_z_w288'] = (df['macd'] - df['macd'].rolling(288).mean()) / df['macd'].rolling(288).std()
# 长期窗口
df['macd_z_w500'] = (df['macd'] - df['macd'].rolling(500).mean()) / df['macd'].rolling(500).std()
```

将这三个特征都输入模型，模型就能同时利用短期的敏感性和长期的稳定性，通常会获得更好的鲁棒性。
