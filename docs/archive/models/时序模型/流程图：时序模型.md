🔍 为什么“不能直接用于训练”？
1. 市场结构剧变（Non-stationarity）
2020–2021：DeFi Summer、机构入场、流动性泛滥 → 高波动 + 强趋势
2022：LUNA 崩盘、FTX 暴雷、宏观加息 → 极端负相关、流动性枯竭
2023：ETF 预期、BTC 减半周期启动 → 新范式

👉 这些 regime 的统计特性（均值、波动、相关性）完全不同。用 2020 年数据训练的模型，在 2024 年可能完全失效。
2. 特征逻辑可能过时
例如：2021 年 exchange_netflow 是强 alpha，但 2023 年 CEX 影响力下降
链上指标、衍生品 funding rate 的定价效率也在变化
3. 幸存者偏差（Survivorship Bias）
2020 年存在的币（如 LUNA、FTT）已归零，但你的数据里可能还保留它们 → 虚假信号

✅ 那么，老数据该怎么用？—— 正确姿势

用途 是否推荐 说明
------ -------- ------
训练生产模型（主训练集） ❌ 不推荐 除非做 滚动窗口训练（只保留最近 1–1.5 年）
FCA 分析（找 timeframe/forward） ✅ 推荐 可观察特征在不同 market regime 下的稳定性

# 如果统一模型在某个时间点开仓了，我如何知道他是根据什么特征开仓的 

这是一个极其关键的问题——你已经从“建模”进入了“可解释性与归因分析”阶段，这是实盘风控和策略迭代的核心。
目标：当统一模型（如 LightGBM）在某时间点对某个币种发出开仓信号时，知道是哪些特征驱动了这个决策。

✅ 正确答案：使用 SHAP（SHapley Additive exPlanations） 做逐样本特征归因

SHAP 是目前金融/量化领域最可靠、理论最扎实的模型解释方法，能告诉你：
每个特征对本次预测的贡献（正/负）
哪些特征最关键
是否符合你的逻辑预期（防“诡异信号”）

🧩 完整流程：从开仓信号到归因报告
步骤 1：训练时保存 SHAP 支持

model.fit(X_train, y_train)

explainer = shap.TreeExplainer(model) # 专为树模型优化，速度快

步骤 2：实盘预测时同时计算 SHAP 值
python
假设 df_live 是最新一根 K 线的特征（已按 symbol 的 timeframe 对齐）

同时计算 SHAP 值（每个样本每个特征的贡献）
