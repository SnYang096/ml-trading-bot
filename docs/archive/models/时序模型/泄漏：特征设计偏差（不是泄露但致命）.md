@staticmethod
def _rolling_percentile(series: pd.Series, window: int, min_periods: int = None) -> pd.Series:
if min_periods is None:
min_periods = window

def _percentile(x: np.ndarray) -> float:
if len(x) < 2 or not np.isfinite(x[-1]):
return np.nan
current = x[-1]
history = x[:-1] # ← 只用历史，排除当前值！
history = history[np.isfinite(history)]
if len(history) == 0:
return np.nan
return (history <= current).sum() / float(len(history))

return series.rolling(window=window, min_periods=min_periods).apply(_percentile, raw=True)

🧭 核心原则：决策时间 vs. 特征可用时间

是否需要 .shift(1)，取决于 你的交易信号是在什么时间点生成，并基于哪些数据。
情况一：T+1 开盘交易（最常见）

但！⚠️ 这只是针对原始指标或基础滚动指标。

❗关键区分：基础滚动指标 vs. 二阶统计特征

类型 示例 是否可用 t 时刻值？ 是否需 shift(1)？
------ ------ ------------------- ------------------
基础滚动指标 sma_20[t] = mean(close[t-19:t+1]) ✅ 可用 ❌ 不需要
ATR / BB 宽度 atr[t], bb_width[t] ✅ 可用 ❌ 不需要

为什么？

因为像 zscore_w288[t] 的计算是：
python

它把 当前值 close[t] 同时用作“被标准化的对象”和“分布的一部分”。

当你用这个 zscore[t] 去预测 未来收益（比如 return[t+1]），模型就“知道”了 close[t] 在最近 288 根 K 线中的相对位置 —— 而这个位置信息依赖于未来的上下文（因为窗口包含 t 本身）。

虽然你在 t 时刻确实知道 close[t]，但 “它在最近 288 根中的百分位”这个信息，在真实交易中无法在 t 时刻精确获得，除非你假设未来没有极端波动改变分布 —— 但这正是过拟合的来源。

更严谨地说：
任何将当前样本包含在滚动统计分母/分布中的特征，都会引入轻微前视偏差（look-ahead bias），即使数据本身是已知的。

这就是为什么 bb_width_zscore_w288 泄漏很强 —— 因为 bb_width[t] 本身没问题，但它的 z-score 把自己放进历史分布里算了。

你困惑的是：“我都已经看到这288根K线了（包括第t根），为什么不能算它的排名？”

直觉上完全合理！但问题不在“能不能算”，而在这个排名是否对未来有预测能力——还是只是对过去的一个描述？

🧩 关键区分：描述性统计 vs. 预测性信号
✅ 你可以合法计算：
close[t]
mean(close[t-287:t+1]) → 这是过去288根的均值
percentileofscore(close[t-287:t+1], close[t]) → 这是 close[t] 在这288根里的排名

✅ 技术上，这些在 t 时刻都能算出来，没有数据缺失。

但 ❗问题在于：这个“排名”包含了当前点自身对分布的影响。

🔍 举个极端例子（帮助理解）

假设过去 287 根收盘价都在 [100, 101] 区间波动，非常平稳。

现在第 288 根（即 t 时刻）突然暴涨到 200。

你计算：
python
window = close[t-287 : t+1] # 共288个点，最后一个是200
rank = percentileofscore(window, 200) # ≈ 100%

于是你得到一个特征：close_percentile = 100%，然后用它去预测 未来收益（比如 t+1 到 t+5 的回报）。

模型发现：“当 close_percentile = 100% 时，未来往往下跌” —— 因为这是一个极端高位。

听起来很合理？

但这里隐藏了一个因果倒置：
不是“高百分位导致未来下跌”，而是“今天暴涨导致百分位变高 + 未来可能回调”。

而“今天暴涨”这个信息，其实已经直接包含在 return[t] 或 close[t]/close[t-1] 里了。

你用一个“由今日极端值扭曲出来的历史排名”，间接泄露了今日的极端变动。

🎯 真正的问题：样本内自参照（self-referential bias）

当你把 x[t] 同时用于：
1. 构建分布（作为分母/样本之一）
2. 作为被评估对象（分子）

你就得到了一个对当前异常值过度敏感的统计量，它在样本内表现极强（因为能捕捉到突变），但在样本外：
如果未来没有同样幅度的突变，这个信号就失效；
更糟的是，它可能在震荡市中频繁给出假信号（因为小波动也会改变局部排名）。

这本质上是一种过拟合：模型学到了“如何识别历史中的极端点”，而不是“如何预测未来”。

✅ 正确做法：用 t-1 及之前的数据评估 t 时刻的位置

更稳健的做法是问：
“在看到今天的价格之前，根据过去287根数据，今天的价位算高吗？”

实现方式：
python
错误（有泄漏）：
percentile_t = percentileofscore(close[t-287:t+1], close[t])
正确（无泄漏）：
historical_dist = close[t-288:t] # 不包含 t
percentile_t = percentileofscore(historical_dist, close[t])

但注意！这个 percentile_t 仍然依赖 close[t]（作为查询值），所以它不能提前知道。

因此，在t 时刻做决策时，你其实无法计算这个“无泄漏的百分位”，除非你愿意延迟一天。

所以实践中，大家采用折中方案：
把整个特征整体 shift(1)，即在 t+1 时刻使用基于 t 及之前数据计算出的百分位，来预测 t+2 的收益。

这样虽然牺牲了一天的时效性，但保证了严格因果。

📊 回到你的泄漏检测结果

你发现 bb_width_zscore_w288 和未来收益相关性高达 +0.1288（p=0），这强烈暗示：
模型通过“当前布林带宽度在近期的相对位置”猜到了未来走势；
而这个“相对位置”之所以有效，很可能是因为它隐含了刚刚发生的波动率突变；
一旦你 shift(1)，这个相关性就会大幅下降（甚至消失），说明它原本就是一种伪信号。

💡 类比理解

想象你在考试前夜背题：
有泄漏的做法：老师把明天考卷混进你今晚做的100道练习题里，你发现“这套题里最难的一道”总是对应某个知识点 → 你以为掌握了规律。
无泄漏的做法：你只用前99道题总结规律，去预测第100道题的难度。

前者看似准确率高，但换一套卷子就崩了。

✅ 总结：为什么“知道排名”仍有问题？

“我在 t 时刻知道所有288根数据，当然能算排名” ✅ 技术上没错
“那这个排名应该可用啊” ❌ 但它是一个被当前极端值污染的历史描述，不是稳定的预测因子

所以，不是不能算，而是这个特征容易导致过拟合和虚假信号。
