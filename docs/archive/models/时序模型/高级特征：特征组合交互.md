🏭 工业级答案：Grouped Time-Series Rank Normalization（分组时序分位标准化）
核心思想：
在每个时间点 t，对所有币种的特征值做 rank(pct=True)，但保留各自的时间序列结构。

这既解决了跨品种量纲差异，又保持了时序一致性，是 Citadel、Jump、Two Sigma 等机构在多资产时序建模中的标准做法。

🔧 具体实现方式（推荐）

python
假设 df 有列: ['symbol', 'timestamp', 'feature1', 'feature2', ...]

def grouped_time_series_rank_norm(df: pd.DataFrame, feature_cols: list) -> pd.DataFrame:
"""
对每个时间点，对所有 symbol 的特征做分位排名
适用于: 多标的共享时序模型
"""
df = df.copy()

# 确保按时间排序
df = df.sort_values('timestamp')

for col in feature_cols:
# 方法1: 横截面分位（推荐）
df[f"{col}_rank"] = df.groupby('timestamp')[col].rank(pct=True, method='average')

# 可选：填充空值（如某币缺失数据）
df[f"{col}_rank"] = df[f"{col}_rank"].fillna(0.5)

return df
📌 为什么这是“工业级”？
优势 说明
------ ------
✅ 消除量纲差异 BTC 的 compression_energy_x_ofi_short=50 和 ETH 的 =5 被映射到 [0,1] 同一尺度
✅ 保留时序动态 每个币自己的 rank 序列仍是连续的（今天 0.8 → 明天 0.3 表示相对强度下降）
✅ 抗极端值 单个币的异常值不会影响整体分布
✅ 模型泛化强 树模型看到的是“相对状态”，而非绝对数值，更容易学到跨品种通用规律

def live_rank_transform(live_features_df: pd.DataFrame, history_df: pd.DataFrame):
"""
live_features_df: 当前时刻各币的新特征值（未标准化）
history_df: 过去 N 天的历史数据（含 timestamp, symbol, features）

返回: 标准化后的特征（仅基于 <= current_time 的数据）
"""
current_time = live_features_df['timestamp'].max()
all_data_until_now = pd.concat([
history_df[history_df['timestamp'] <= current_time],
live_features_df
])

# 只对当前时间点做 rank
current_slice = all_data_until_now[all_data_until_now['timestamp'] == current_time]
for col in feature_cols:
ranks = current_slice.groupby('timestamp')[col].rank(pct=True)
live_features_df[f"{col}_rank"] = ranks.values

return live_features_df
🔒 注意：绝不能把未来数据混入 rank 计算！

✅ 一、Grouped Time-Series Rank vs 多因子 Rank：核心差别确实是 是否保留时序维度
📊 对比表：

维度 多因子横截面模型（Cross-Sectional） 时序模型（Time-Series）
------ ---------------------------------------- ----------------------------
目标 “今天哪个币最强？” → 做横向比较 “BTC 明天会不会涨？” → 做纵向预测
标准化方式 df.groupby('date')[feature].rank(pct=True) 完全一样！
使用方式 选 top N 排名的币做多 把 rank 特征输入模型，预测每个币自身未来收益
是否保留时序？ ❌ 不关心单个币的历史 rank 序列 ✅ 必须保留每个币自己的 rank 时间序列
🔍 关键洞察：
计算方式相同，但建模逻辑不同。
多因子：rank 是选股信号
时序模型：rank 是特征输入，模型仍要学习“当 BTC 的 rank 从 0.3 升到 0.8 时，未来收益如何变化”

✅ 所以你说“保存了时序维度”——完全正确！
在时序模型中，BTC_rank_t-2=0.4, BTC_rank_t-1=0.6, BTC_rank_t=0.9 这个序列本身携带趋势信息，而多因子只看 t 时刻的快照。

# 非常好的问题！这是很多量化开发者容易混淆的关键点。

简明答案：
✅ 原始 OHLCV（Open, High, Low, Close, Volume）本身不应该直接作为特征输入模型，
❌ 更不需要对 raw price 做归一化（如 MinMax / Z-Score），
✅ 但应转化为“尺度无关”的衍生特征后再做标准化（如 Rank）。

🔍 为什么 raw OHLCV 不适合直接归一化？
1. 价格绝对值无预测意义
BTC $80,000 和 ETH $3,000 的绝对价格无法比较
模型无法从 close=80000 学到通用规律
2. 归一化会引入未来函数（Look-Ahead Bias）
python
错误示例：用全局 MinMax 归一化
scaler = MinMaxScaler()
df['close_norm'] = scaler.fit_transform(df[['close']]) # 泄露了未来最高/最低价！

→ 在实盘中无法实现，回测严重过拟合。
3. 树模型不关心绝对值，只关心相对变化
树模型能自动学习 high > close_prev 这类关系
但 close=80000 本身对预测未来涨跌毫无信息量

✅ 正确做法：将 OHLCV 转化为 尺度无关的衍生特征

原始字段 推荐衍生特征 是否需要 Rank？
-------- ------------- ----------------
Open, High, Low, Close - log_return = log(close / close.shift(1))<br>- body_pct = (close - open) / (high - low)<br>- upper_shadow = (high - max(open, close)) / (high - low)<br>- rsv = (close - low) / (high - low) ✅ 是（跨品种时）
Volume - volume_sma_ratio = volume / volume.rolling(20).mean()<br>- volume_quantile_30d = volume.rank(pct=True).rolling(720).mean() ✅ 是
📌 关键原则：
所有特征必须是“相对量”或“比率”，而非“绝对量”

🛠 工业级 OHLCV 特征工程模板（多标的 + 时序模型）

python
def ohlcv_to_features(df: pd.DataFrame) -> pd.DataFrame:
df = df.copy()

# 1. 收益率（核心）
df['log_ret_1'] = np.log(df['close'] / df['close'].shift(1))
df['log_ret_5'] = np.log(df['close'] / df['close'].shift(5))

# 2. K线形态（尺度无关）
df['price_range'] = df['high'] - df['low']
df['body'] = df['close'] - df['open']
df['body_pct'] = df['body'] / (df['price_range'] + 1e-9)
df['upper_shadow'] = (df['high'] - df[['open', 'close']].max(axis=1)) / (df['price_range'] + 1e-9)
df['lower_shadow'] = (df[['open', 'close']].min(axis=1) - df['low']) / (df['price_range'] + 1e-9)

# 3. 量能相对强度
df['vol_ma20'] = df['volume'].rolling(20).mean()
df['volume_ratio'] = df['volume'] / (df['vol_ma20'] + 1e-9)

# 4. RSV（未成熟随机值）
df['rsv'] = (df['close'] - df['low']) / (df['high'] - df['low'] + 1e-9)

return df
然后对这些衍生特征做 Grouped Time-Series Rank
derived_features = [
'log_ret_1', 'body_pct', 'upper_shadow', 'volume_ratio', 'rsv'
]

for col in derived_features:
df[f"{col}_rank"] = df.groupby('timestamp')[col].rank(pct=True)
