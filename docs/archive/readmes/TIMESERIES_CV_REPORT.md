## 🔍 问题诊断

### ❌ 发现的问题
在检查 `ml_project` 的训练代码时，发现了一个**严重的方法论错误**：

#### 问题代码位置：
1. **`src/ml_trading/models/lightgbm_model.py:75-77`**
2. **`examples/optuna_optimization.py:39, 116`**

```python
# ❌ 错误的做法：使用随机分割
X_train, X_val, y_train, y_val = train_test_split(
    X_clean, y_clean, test_size=0.2, random_state=42
)
```

### ⚠️ 问题的严重性

参考文档 [`docs/时间序列cv.md`](docs/时间序列cv.md) 指出：

> **使用 `train_test_split` 会导致：**
> - 训练集可能使用第 90 天的数据
> - 验证集可能使用第 50 天的数据
> - 相当于"**用未来预测过去**"（Look-Ahead Bias）
> - 回测表现虚高，**实盘会大幅亏损**

这是**ML交易系统的生死线**！

---

## ✅ 修复方案

### 修复内容

#### 1. 修改 `lightgbm_model.py`

```python
# ✅ 正确的做法：使用时间序列交叉验证
from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)

for fold, (train_idx, val_idx) in enumerate(tscv.split(X_clean)):
    X_train, X_val = X_clean.iloc[train_idx], X_clean.iloc[val_idx]
    y_train, y_val = y_clean.iloc[train_idx], y_clean.iloc[val_idx]
    
    # 训练模型...
    # 保证：训练集永远在验证集之前
```

**关键改进：**
- ✅ 使用 `TimeSeriesSplit` 代替 `train_test_split`
- ✅ 训练集永远在验证集之前
- ✅ 5折交叉验证，提供稳健的性能估计
- ✅ 避免未来信息泄露

#### 2. 修改 `optuna_optimization.py`

同样使用 `TimeSeriesSplit` 进行超参数优化：
