### 2. 特征并非越多越好

**对比**:
- 基线：142特征 → 91.24%准确率，69.78%收益
- 增强：321特征 → 91.38%准确率，68.59%收益

**教训**:
- 特征增加126% → 性能提升可忽略
- 简单有效 > 复杂理论
- LightGBM特征选择很强

### 3. 时间序列特征必须随时间变化

**Spectral案例**:
- 错误：标量赋值 → 重要性=0
- 正确：时间序列 → 重要性=1072

**黄金法则**:
- ❌ 常数特征 = 无信息量
- ✅ 时间序列 = 有预测价值
- 💡 检查：`df['feature'].nunique()` >> 1

### 4. 基础特征不可替代

**证据**:
- 技术指标类别排名第2（4544重要性）
- Top 10中有4个基础特征
- ATR、BB、Momentum等都很重要

**启示**:
- 高级特征是补充，不是替代
- 基础 + 高级 = 最优组合

## 💡 最重要的三大发现

### 🔥 发现1：Hilbert变换是金融时序分析的杀手锏

**数据**:
- 15个特征贡献48%总重要性
- 单特征`close_hilbert_frequency`占35%
- 平均重要性是Spectral的20倍

**理论**:
- Hilbert提供瞬时频率、相位、振幅
- 完美适配快速变化的加密市场
- 比传统FFT/Spectral更有效

**应用**:
- ✅ 必须保留Hilbert
- ✅ 考虑对更多信号做Hilbert
- ✅ Hilbert应该是特征工程的核心

### 📊 发现2：时间序列特征必须随时间变化

**Spectral案例**:
- 错误实现：标量赋值 → 重要性=0
- 正确实现：时间序列 → 重要性=1072

**黄金法则**:
```python
# ❌ 错误模式
feature_value = calculate_once(entire_data)  # 标量
df['feature'] = feature_value  # 所有行相同

# ✅ 正确模式
feature_series = []
for i in range(len(data)):
    window = data[i-100:i]
    feature_series.append(calculate(window))
df['feature'] = feature_series  # 每行不同
```

**验证方法**:
```python
# 检查特征有效性
if df['feature'].nunique() == 1:
    print("❌ 错误：常数特征")
else:
    print("✅ 正确：时间序列特征")
```

**证据**:
- 基线142特征 vs 增强321特征
- 性能：基线略优（收益69.78% vs 68.59%）
- 效率：基线快2倍以上

**教训**:
