# 信号生成与盈利评估说明

## 📋 概述

本文档说明 `train`、`rolling`、`ts-dim-compare` 三个流程中：
1. **是否使用 `ClassificationStrategyHandler.generate_signals`**
2. **如何判断盈利**
3. **仓位大小如何计算**

---

## 1. 三个流程的信号生成方式

### 1.1 `make rolling` - ✅ 使用 `ClassificationStrategyHandler.generate_signals`

**位置**: `src/time_series_model/pipeline/training/rolling.py`

**代码流程**:
```python
# 1. 训练模型
model_cls = models_dict.get("classification")
model_return = models_dict.get("return")
model_vol = models_dict.get("vol")

# 2. 创建 pipeline stub
pipeline_stub = _SingleTimeframePipeline(
    timeframe="default",
    cls_model=model_cls,
    return_model=model_return,
    vol_model=model_vol,
)

# 3. 使用 ClassificationStrategyHandler 生成信号
strategy_handler = ClassificationStrategyHandler(
    pipeline_stub,
    signal_strength_threshold=0.0,  # 不过滤信号
    confidence_threshold=0.0,        # 不过滤信号
    base_position_size=1.0,         # 100% 仓位
    classification_threshold=0.5,
)
signals_df = strategy_handler.generate_signals(X_test_df, test_slice, timeframe="default")

# 4. 评估信号性能
performance = evaluate_signal_performance(signals_df, y_return_test)
```

**信号生成逻辑**:
- 使用分类模型预测概率 `class_proba`
- 使用收益回归模型预测 `return_pred`
- 使用波动率模型预测 `vol_pred`
- 生成信号：`signal` (0=Hold, 1=Long, -1=Short)
- 计算仓位：`position_size = base_size * |return_pred| * combined_confidence / vol`

---

### 1.2 `make train` - ❌ 不使用 `ClassificationStrategyHandler.generate_signals`

**位置**: `src/time_series_model/pipeline/training/train.py`

**说明**:
- `make train` 主要用于**训练模型**，不进行信号生成和回测
- 它只训练模型并保存，不评估交易性能
- 如果需要回测，需要使用 `make ts-vectorbot-backtest`

---

### 1.3 `make ts-dim-compare` - ❌ 不使用 `ClassificationStrategyHandler.generate_signals`

**位置**: `src/time_series_model/pipeline/dimensionality/dimensionality_comparison.py`

**代码流程**:
```python
# 1. 训练分类模型
model = train_production_lightgbm(X_train, y_train, X_val, y_val)

# 2. 直接使用模型预测
predictions = model.predict(X_test)  # 0=Hold, 1=Long, 2=Short

# 3. 使用 backtest_classification_model 评估
metrics = backtest_classification_model(
    model, X_test, y_test, price_data, horizon=1
)
```

**信号生成逻辑**:
- 直接使用分类模型的预测结果（0=Hold, 1=Long, 2=Short）
- **不使用** `ClassificationStrategyHandler` 的复杂逻辑
- **不使用** 收益回归和波动率模型
- **不使用** `combined_confidence` 和 `signal_strength`

## 2. 盈利判断方式

### 2.1 `rolling` - 使用 `evaluate_signal_performance`

**位置**: `src/time_series_model/utils/training.py`

**核心逻辑**:
```python
def evaluate_signal_performance(
    signals_df: pd.DataFrame,
    future_returns: pd.Series,  # 真实的未来收益率
    initial_capital: float = 100000.0,
) -> Dict[str, Any]:
    # 1. 合并信号和未来收益率
    df = signals_df.join(future_returns.rename("future_return"), how="inner")
    
    # 2. 计算仓位（使用 signal_strength，限制在 [-1, 1]）
    df["position"] = np.clip(df["signal_strength"], -1.0, 1.0)
    
    # 3. 计算每期收益
    df["period_return"] = df["position"] * df["future_return"]
    
    # 4. 计算累计资金曲线
    equity_curve = (1.0 + df["period_return"]).cumprod() * initial_capital
    
    # 5. 计算总收益
    total_return = equity_curve.iloc[-1] / initial_capital - 1.0
    
    # 6. 计算胜率、盈亏比等指标
    win_rate = (df["period_return"] > 0).mean() * 100.0
    profit_factor = positives.sum() / abs(negatives.sum())
    ...
```

**关键点**:
- ✅ 使用**真实的未来收益率** (`future_returns`) 来判断盈利
- ✅ 仓位 = `signal_strength`（限制在 [-1, 1]）
- ✅ 每期收益 = `position * future_return`
- ✅ 累计收益 = `(1 + period_return).cumprod()`

### 2.2 `ts-dim-compare` - 使用 `calculate_strategy_returns_from_predictions`

**位置**: `src/time_series_model/pipeline/dimensionality/backtest_evaluator.py`

**核心逻辑**:
```python
def calculate_strategy_returns_from_predictions(
    predictions: np.ndarray,  # 0=Hold, 1=Long, 2=Short
    price_data: pd.DataFrame,
    horizon: int = 1,
) -> np.ndarray:
    # 1. 计算未来收益率
    forward_returns = (close_prices[i + horizon] / close_prices[i]) - 1.0
    
    # 2. 根据预测计算策略收益
    strategy_returns = np.zeros(len(predictions))
    
    # Long (prediction = 1): 使用正向收益
    long_mask = predictions == 1
    strategy_returns[long_mask] = forward_returns[long_mask]
    
    # Short (prediction = 2): 使用负向收益
    short_mask = predictions == 2
    strategy_returns[short_mask] = -forward_returns[short_mask]
    
    # Hold (prediction = 0): 收益 = 0
    return strategy_returns
```

**关键点**:
- ✅ 使用**价格数据**计算未来收益率
- ✅ Long: `return = forward_return`
- ✅ Short: `return = -forward_return`
- ✅ Hold: `return = 0`
- ❌ **不使用仓位大小**（固定 100% 或 0%）

## 3. 仓位大小计算

### 3.1 `ClassificationStrategyHandler.generate_signals` 的仓位计算

**位置**: `src/time_series_model/strategies/classification_strategy_handler.py`

**公式**:
```python
# 仓位大小 = base_size * |return_pred| * combined_confidence / vol
position_size = (
    self.base_position_size *           # 基础仓位（默认 0.1 = 10%）
    np.abs(signals_df["return_pred"]) * # 预测收益的绝对值
    signals_df["combined_confidence"] / # 综合置信度
    (signals_df["vol"] + 1e-8)         # 预测波动率（避免除零）
)
```

**参数说明**:
- `base_position_size`: 基础仓位倍数（默认 0.1 = 10%）
- `return_pred`: 预测收益的绝对值（越大，仓位越大）
- `combined_confidence`: 综合置信度 = `confidence * return_confidence`
  - `confidence`: 分类置信度 = `|class_proba - 0.5| * 2`
  - `return_confidence`: 收益置信度 = `|return_pred| / (vol + 1e-8)`
- `vol`: 预测波动率（越大，仓位越小）

**示例**:
```python
# 假设：
base_position_size = 0.1
return_pred = 0.02  # 预测 2% 收益
combined_confidence = 0.8  # 80% 置信度
vol = 0.01  # 1% 波动率

# 计算：
position_size = 0.1 * 0.02 * 0.8 / 0.01 = 0.16 = 16%
```

---

### 3.2 `evaluate_signal_performance` 的仓位计算

**位置**: `src/time_series_model/utils/training.py`

**公式**:
```python
# 仓位 = signal_strength（限制在 [-1, 1]）
df["position"] = np.clip(df["signal_strength"], -1.0, 1.0)
```

**说明**:
- `signal_strength` 来自 `ClassificationStrategyHandler.generate_signals`
- `signal_strength = (return_pred * direction) / (vol + 1e-8)`
  - `direction = 1` (如果 `class_proba > 0.5`) 或 `-1` (如果 `class_proba <= 0.5`)
- 限制在 `[-1, 1]` 表示：
  - `1.0` = 100% 做多
  - `-1.0` = 100% 做空
  - `0.0` = 空仓

**注意**: 这里**不使用** `position_size`，而是直接使用 `signal_strength` 作为仓位！

---

### 3.3 `ts-dim-compare` 的仓位计算

**位置**: `src/time_series_model/pipeline/dimensionality/backtest_evaluator.py`

**说明**:
- ❌ **不使用仓位大小**
- 固定仓位：
  - Long (prediction = 1): 100% 做多
  - Short (prediction = 2): 100% 做空
  - Hold (prediction = 0): 0% 空仓

## 4. 总结对比表

| 流程 | 使用 `ClassificationStrategyHandler`? | 信号生成方式 | 盈利判断方式 | 仓位计算 |
|------|-------------------------------------|------------|------------|---------|
| **`make rolling`** | ✅ 是 | `ClassificationStrategyHandler.generate_signals` | `evaluate_signal_performance` | `signal_strength` (限制在 [-1, 1]) |
| **`make train`** | ❌ 否 | 不生成信号（只训练模型） | 不评估盈利 | N/A |
| **`make ts-dim-compare`** | ❌ 否 | 直接使用分类模型预测 | `calculate_strategy_returns_from_predictions` | 固定 100% 或 0% |

## 5. 关键差异说明

### 5.1 `rolling` vs `ts-dim-compare` 的信号生成

| 特性 | `rolling` | `ts-dim-compare` |
|------|-----------|---------------|
| **使用模型** | 分类 + 收益回归 + 波动率 | 仅分类模型 |
| **信号逻辑** | 复杂（考虑置信度、收益、波动率） | 简单（直接使用分类预测） |
| **仓位计算** | 动态（基于收益、置信度、波动率） | 固定（100% 或 0%） |
| **适用场景** | 生产环境、真实回测 | 特征选择、模型对比 |

### 5.2 `rolling` 中的仓位计算不一致问题 ⚠️

**问题**:
1. `ClassificationStrategyHandler.generate_signals` 计算 `position_size`
2. 但 `evaluate_signal_performance` 使用 `signal_strength` 作为仓位
3. **两者不一致！**

**当前实现**:
```python
# ClassificationStrategyHandler 计算 position_size
signals_df["position_size"] = base_size * |return_pred| * combined_confidence / vol

# 但 evaluate_signal_performance 使用 signal_strength
df["position"] = np.clip(df["signal_strength"], -1.0, 1.0)
# signal_strength = (return_pred * direction) / vol
```

**建议**:
- 应该使用 `position_size` 而不是 `signal_strength` 作为仓位
- 或者统一使用 `signal_strength` 并移除 `position_size` 计算
