# 架构讨论：因子依赖关系管理

## 当前问题分析

### 1. 依赖关系的复杂性

当前 `required_features` 机制存在以下问题：

#### 问题 1: 隐式依赖无法控制

```python
# 示例：add_common_derived_features 中的依赖
if "rsi_normalized" in required_features:
    # 但函数内部会先调用 ensure_basic_indicators()
    # 这会计算所有基础指标：rsi, macd, atr, zigzag, bbands 等
    result = ensure_basic_indicators(df.copy())  # ❌ 计算了所有基础指标
    
    # 然后才计算 rsi_normalized
    result["rsi_normalized"] = (result["rsi"] - 50) / 50
```

**问题**：即使只需要 `rsi_normalized`，也会计算所有基础指标。

#### 问题 2: 模块级别的粗粒度控制

```python
# 当前架构：模块级别的控制
if self.use_enhanced or self.use_hurst:
    if not required_features or hurst_features:
        df = self.enhanced_engineer.add_hurst_features(df, hurst_features)
```

**问题**：
- `add_hurst_features()` 可能对所有信号源（close, open, volume, cvd）都计算 Hurst
- 即使只需要 `close_hurst`，也会计算 `open_hurst`, `volume_hurst` 等

#### 问题 3: 依赖链无法追踪

```python
# 示例：zigzag_distance 的依赖链
zigzag_distance = (close - zigzag) / atr
# 依赖：close (数据列) + zigzag (基础指标) + atr (基础指标)
# 但 required_features 中只有 "zigzag_distance"
# 系统无法知道需要先计算 zigzag 和 atr
```

## 架构方案对比

### 方案 1: 显式依赖声明（Dependency Graph）

**核心思想**：每个因子显式声明其依赖关系

```python
# 因子注册表
FACTOR_REGISTRY = {
    "rsi": {
        "dependencies": [],  # 无依赖，直接从 OHLCV 计算
        "module": "talib",
        "compute_fn": compute_rsi,
    },
    "rsi_normalized": {
        "dependencies": ["rsi"],  # 依赖 rsi
        "module": "derived",
        "compute_fn": lambda df: (df["rsi"] - 50) / 50,
    },
    "zigzag": {
        "dependencies": [],
        "module": "base_indicators",
        "compute_fn": compute_zigzag,
    },
    "zigzag_distance": {
        "dependencies": ["zigzag", "atr", "close"],  # 显式声明依赖
        "module": "derived",
        "compute_fn": lambda df: (df["close"] - df["zigzag"]) / df["atr"],
    },
    "close_hurst": {
        "dependencies": ["close"],  # 只需要 close，不需要其他信号源
        "module": "enhanced",
        "compute_fn": lambda df: compute_hurst(df["close"]),
    },
}
```

**优点**：
- ✅ 依赖关系清晰、可追踪
- ✅ 可以自动解析依赖链
- ✅ 只计算真正需要的因子

**缺点**：
- ❌ 需要重构所有因子计算逻辑
- ❌ 维护成本高（每个因子都要注册）
- ❌ 可能遗漏某些依赖关系

### 方案 2: 细粒度模块拆分

**核心思想**：将大模块拆分成更小的、独立的计算单元

```python
# 当前：粗粒度模块
class EnhancedFeatureEngineer:
    def add_hurst_features(self, df):
        # 对所有信号源计算 Hurst
        for source in ["close", "open", "volume", "cvd", "taker_buy_ratio"]:
            df[f"{source}_hurst"] = compute_hurst(df[source])

# 改进：细粒度模块
class HurstFeatureEngineer:
    def compute_close_hurst(self, df):
        return compute_hurst(df["close"])
    
    def compute_open_hurst(self, df):
        return compute_hurst(df["open"])
    
    # 每个信号源独立计算
```

**优点**：
- ✅ 可以精确控制计算哪些因子
- ✅ 模块更独立，易于测试
- ✅ 不需要大的重构

**缺点**：
- ❌ 代码重复（每个信号源都要写一遍）
- ❌ 模块数量增加（管理复杂度上升）
- ❌ 仍然无法解决基础指标的依赖问题

### 方案 3: 懒加载 + 依赖解析（推荐）

**核心思想**：按需计算，自动解析依赖

```python
class LazyFeatureEngineer:
    def __init__(self):
        self.computed_features = set()
        self.dependency_graph = self._build_dependency_graph()
    
    def _build_dependency_graph(self):
        """构建依赖图"""
        return {
            "rsi_normalized": ["rsi"],
            "zigzag_distance": ["zigzag", "atr"],
            "close_hurst": ["close"],
            # ... 所有因子的依赖关系
        }
    
    def compute_feature(self, df, feature_name):
        """懒加载计算特征"""
        if feature_name in self.computed_features:
            return df[feature_name]
        
        # 解析依赖
        deps = self.dependency_graph.get(feature_name, [])
        for dep in deps:
            if dep not in df.columns:
                self.compute_feature(df, dep)  # 递归计算依赖
        
        # 计算当前特征
        df[feature_name] = self._compute(feature_name, df)
        self.computed_features.add(feature_name)
        return df[feature_name]
    
    def compute_features(self, df, required_features):
        """批量计算特征"""
        for feature in required_features:
            self.compute_feature(df, feature)
        return df
```

**优点**：
- ✅ 自动解析依赖，只计算需要的
- ✅ 可以缓存已计算的特征
- ✅ 不需要大的重构（可以逐步迁移）

**缺点**：
- ❌ 需要维护依赖图
- ❌ 递归计算可能有性能问题（需要优化）

### 方案 4: 分层计算 + 显式依赖（混合方案）

**核心思想**：分层管理，每层内部处理依赖

```python
# 分层架构
class FeatureEngineer:
    def __init__(self):
        self.layers = [
            DataLayer(),           # 数据层：OHLCV
            BasicIndicatorLayer(), # 基础指标层：rsi, macd, atr, zigzag
            DerivedLayer(),        # 衍生层：rsi_normalized, zigzag_distance
            EnhancedLayer(),       # 增强层：hurst, wavelet, hilbert
        ]
    
    def compute(self, df, required_features):
        """分层计算"""
        computed = set()
        
        # 1. 分析需要哪些层的特征
        layer_requirements = self._analyze_layer_requirements(required_features)
        
        # 2. 按层计算
        for layer, features in layer_requirements.items():
            # 每层内部处理依赖
            df = layer.compute(df, features, computed)
            computed.update(features)
        
        return df

class BasicIndicatorLayer:
    def compute(self, df, required_features, already_computed):
        """基础指标层：只计算需要的指标"""
        # 不再调用 ensure_basic_indicators()，而是按需计算
        if "rsi" in required_features:
            df["rsi"] = compute_rsi(df["close"])
        if "atr" in required_features:
            df["atr"] = compute_atr(df["high"], df["low"], df["close"])
        # ... 只计算需要的
        return df
```

**优点**：
- ✅ 分层清晰，易于理解
- ✅ 每层内部可以优化
- ✅ 可以逐步重构

**缺点**：
- ❌ 需要重构现有代码
- ❌ 层之间的依赖关系需要明确

## 推荐方案：方案 3 + 方案 4 混合

### 架构设计

```python
# 1. 因子注册表（显式依赖）
FACTOR_REGISTRY = {
    "rsi": {
        "layer": "basic",
        "dependencies": ["close"],
        "compute_fn": "compute_rsi",
    },
    "rsi_normalized": {
        "layer": "derived",
        "dependencies": ["rsi"],
        "compute_fn": "normalize_rsi",
    },
    "zigzag_distance": {
        "layer": "derived",
        "dependencies": ["zigzag", "atr", "close"],
        "compute_fn": "compute_zigzag_distance",
    },
}

# 2. 分层计算引擎
class LayeredFeatureEngineer:
    def __init__(self):
        self.registry = FACTOR_REGISTRY
        self.layers = {
            "basic": BasicLayer(),
            "derived": DerivedLayer(),
            "enhanced": EnhancedLayer(),
        }
    
    def compute(self, df, required_features):
        """智能计算：自动解析依赖"""
        # 1. 解析依赖链
        all_needed = self._resolve_dependencies(required_features)
        
        # 2. 按层分组
        by_layer = self._group_by_layer(all_needed)
        
        # 3. 按层计算
        for layer_name in ["basic", "derived", "enhanced"]:
            if layer_name in by_layer:
                df = self.layers[layer_name].compute(
                    df, 
                    by_layer[layer_name]
                )
        
        return df
    
    def _resolve_dependencies(self, features):
        """解析依赖链"""
        needed = set(features)
        queue = list(features)
        
        while queue:
            feature = queue.pop(0)
            deps = self.registry.get(feature, {}).get("dependencies", [])
            for dep in deps:
                if dep not in needed:
                    needed.add(dep)
                    queue.append(dep)
        
        return needed
```

### 实施步骤

1. **第一阶段：建立注册表**
   - 为常用因子建立依赖注册表
   - 保持现有代码不变

2. **第二阶段：重构基础层**
   - 将 `ensure_basic_indicators()` 改为按需计算
   - 只计算需要的指标

3. **第三阶段：重构衍生层**
   - 将 `add_common_derived_features()` 改为按需计算
   - 使用注册表解析依赖

4. **第四阶段：重构增强层**
   - 将增强特征模块拆分得更细
   - 每个信号源独立计算

3. **长期**：完善方案 1（显式依赖声明），建立完整的因子注册表

这样可以：
