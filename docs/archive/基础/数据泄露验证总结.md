## 关键原则

### 滑动窗口的正确实现
```python
# ✅ 正确方法
for i in range(len(data)):
    if i >= window:
        window_data = data[i-window:i]  # 不包含第i个元素
        feature[i] = compute(window_data)
```

### 为什么用 `[i-window:i]` 而不是 `[i-window:i+1]`？

1. **训练时：** 我们要预测未来（如第i+1时刻的收益）
2. **如果特征包含第i时刻的全部信息，**可能会导致信息泄露
3. **使用 `[i-window:i]`** 确保只用历史数据

**注意：** 在某些场景下，使用 `[i-window:i+1]` 也是可以的，关键看标签的定义。如果标签是 `shift(-forward_bars)`，通常不会泄露。

## 可选优化建议

1. **CVD使用变化率而非绝对值**
   ```python
   # 当前：cumsum()从头累计
   cvd = (buy - sell).cumsum()
   
   # 建议：使用变化率或滚动窗口
   cvd_change = cvd.diff()
   # 或
   rolling_cvd = (buy - sell).rolling(100).sum()
   ```

2. **添加更多前向验证测试**
   - 逐点计算特征，确认每个时刻只依赖历史

3. **考虑Walk-Forward Optimization**
