# 统一归一化解决方案总结

## 问题分析结果

通过深入分析项目中的所有特征工程文件，发现了以下归一化问题：

### 1. 归一化方法不一致
- **TA-Lib特征工程**: 支持分组归一化，使用多种归一化器
- **增强特征工程**: 仅支持StandardScaler，全局归一化
- **改进特征工程**: 支持多种归一化器，全局归一化
- **深度学习序列**: 支持4种归一化策略，已解决前世偏差问题

### 2. 前世偏差问题
- **存在问题的文件**: TA-Lib、增强特征工程、改进特征工程
- **问题表现**: 训练和测试阶段使用相同的全局统计量，导致特征分布不一致
- **影响**: 模型在预测阶段性能下降，引入前世偏差

### 3. 特征分组策略不统一
- **TA-Lib**: 支持特征分组归一化
- **其他文件**: 不支持特征分组，所有特征使用相同归一化策略

### 4. 归一化参数管理不一致
- **深度学习序列**: 不支持参数保存和加载
- **其他文件**: 支持参数保存和加载

## 统一归一化解决方案

### 1. 核心设计理念

#### 1.1 策略模式设计
- 使用策略模式实现不同的归一化方法
- 每种策略都继承自`NormalizationStrategy`基类
- 提供统一的接口：`fit()`, `transform()`, `fit_transform()`

#### 1.2 支持多种归一化策略
- **GlobalNormalization**: 全局归一化（原有方法）
- **RollingWindowNormalization**: 滚动窗口归一化（避免前世偏差）
- **EMANormalization**: 指数移动平均归一化（平滑适应）
- **AdaptiveNormalization**: 自适应归一化（平衡稳定性和适应性）

### 2. 技术实现

#### 2.1 归一化策略基类
```python
class NormalizationStrategy(ABC):
    @abstractmethod
    def fit(self, data: np.ndarray) -> 'NormalizationStrategy':
        """拟合归一化参数"""
        pass
    
    @abstractmethod
    def transform(self, data: np.ndarray) -> np.ndarray:
        """应用归一化"""
        pass
    
    @abstractmethod
    def fit_transform(self, data: np.ndarray) -> np.ndarray:
        """拟合并应用归一化"""
        pass
```

#### 2.3 特征分组功能
- 自动识别特征类型（价格、比率、波动率、成交量、动量、索引等）
- 为不同特征组使用最适合的归一化策略
- 支持自定义特征分组

### 3. 解决的问题

#### 3.1 前世偏差问题
- **滚动窗口归一化**: 使用滑动窗口内的统计量，避免使用未来信息
- **EMA归一化**: 使用指数移动平均，平滑适应数据变化
- **自适应归一化**: 结合全局和局部统计量，平衡稳定性和适应性

#### 3.2 归一化方法不一致
- 提供统一的归一化接口
- 支持多种归一化策略选择
- 保持向后兼容性

#### 3.3 特征分组策略不统一
- 实现自动特征分组
- 支持自定义特征分组
- 为不同特征类型选择最适合的归一化策略

#### 3.4 归一化参数管理不一致
- 统一的参数保存和加载接口
- 支持参数版本管理
- 提供参数验证和检查

### 2. 策略选择指南

#### 2.1 全局归一化
- **适用场景**: 数据分布相对稳定
- **优点**: 计算简单，保证一致性
- **缺点**: 存在前世偏差风险

#### 2.2 滚动窗口归一化
- **适用场景**: 非平稳时间序列，数据分布变化较大
- **优点**: 避免前世偏差，适应数据变化
- **缺点**: 计算复杂度增加

#### 2.3 EMA归一化
- **适用场景**: 平稳时间序列，需要平滑的归一化
- **优点**: 平衡历史信息和当前信息，计算效率高
- **缺点**: 需要调优平滑参数

#### 2.4 自适应归一化（推荐）
- **适用场景**: 生产环境，需要平衡性能和准确性
- **优点**: 平衡稳定性和适应性，减少前世偏差
- **缺点**: 需要调优权重参数

### 3. 参数调优

#### 3.1 窗口大小
- **默认值**: 100
- **调整原则**: 数据变化大时减小窗口，数据稳定时增大窗口

#### 3.2 EMA平滑参数
- **默认值**: 0.01
- **调整范围**: 0.001 - 0.1
- **较小值**: 更平滑，响应较慢
- **较大值**: 更敏感，响应较快

#### 3.3 自适应权重
- **全局权重**: 0.3（默认）
- **局部权重**: 0.7（默认）
- **调整原则**: 数据变化大时增加局部权重
