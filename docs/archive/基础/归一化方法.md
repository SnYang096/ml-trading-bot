
这是一个非常关键且实战性极强的问题！

> **✅ 简短回答：是的，你上面提到的这些“优秀、可解释、实战有效”的特征，在输入机器学习模型前，绝大多数都需要进行归一化（Normalization）或标准化（Standardization），否则模型性能会严重下降，甚至无法收敛。**

但要注意：**不是所有特征都必须归一化，也不是所有归一化方法都适合每种特征类型。**

下面我们来系统讲解：

---

## 📌 一、哪些特征需要归一化？哪些不需要？

| 特征类型 | 是否建议归一化 | 原因 |
|--------|----------------|------|
| **价格距离类**（如 `dist_to_swing_high_pct`） | ✅ 强烈建议 | 单位是百分比，范围相对稳定，适合标准化 |
| **比率类**（如 `volume_ratio`, `upvol/downvol`） | ✅ 建议 | 虽有自然尺度，但极端值可能影响训练 |
| **波动率指标**（如 `bb_width_normalized`, `ATR`） | ✅ 建议 | 不同品种/周期差异大，需统一量纲 |
| **动量类**（如 `ROC`, `CVD slope`） | ✅ 建议 | 数值随时间/品种变化大 |
| **布尔/分类特征**（如 `is_london_open`, `is_inside_bar`） | ❌ 不需要 | 已是 0/1 或 one-hot 编码，无需处理 |
| **时间编码**（如 `hour_sin`, `day_cos`） | ❌ 不需要 | 已在 [-1, 1] 范围内 |
| **排名类 / 分位数类**（如 `volume_quantile_20`） | ⚠️ 可选 | 本身在 [0,1] 内，通常不需归一化 |

📌 **核心原则**：
> **只要特征的数值范围跨越多个数量级（如有的特征在 0~1，有的在 100~1000），就必须归一化，否则模型会被大数值特征主导。**

---

## 🔧 二、推荐的归一化方法（按场景）

### ✅ 1. **Z-Score 标准化（Standardization）**
> 适用于：近似正态分布、无极端异常值的连续特征

```python
from sklearn.preprocessing import StandardScaler

# 公式：z = (x - μ) / σ
x_std = (x - mean(x)) / std(x)
```

- **适用特征**：
  - `roc_5`
  - `vwap_deviation_pct`
  - `volume_anomaly`（如果去除了极端值）
  - `distance_to_swing_high_pct`

- **优点**：保留原始分布形状，均值为0，标准差为1
- **缺点**：对异常值敏感

> 💡 建议搭配 **滑动窗口统计量**（如 rolling mean/std），更适合时序数据。

```python
# 滚动 Z-Score（更适用于金融时间序列）
def rolling_zscore(x, window=20):
    return (x - x.rolling(window).mean()) / x.rolling(window).std()
```

---

### ✅ 2. **Min-Max 归一化（Normalization）**
> 适用于：已知上下界，或希望压缩到 [0,1] 区间

```python
from sklearn.preprocessing import MinMaxScaler

# 公式：x_norm = (x - min) / (max - min)
x_norm = (x - x.min()) / (x.max() - x.min())
```

- **适用特征**：
  - `bb_width_normalized`（历史窗口内）
  - `range_ratio_5bar`
  - `candle_body_ratio_avg_3`

- **优点**：输出在 [0,1]，直观
- **缺点**：受极端值影响大，不适合动态市场

> ⚠️ 注意：使用 **滚动窗口 Min-Max** 更稳健：

```python
def rolling_min_max(x, window=20):
    min_val = x.rolling(window).min()
    max_val = x.rolling(window).max()
    return (x - min_val) / (max_val - min_val + 1e-8)  # 防除零
```

---

### ✅ 3. **Robust Scaling（鲁棒缩放）**
> 适用于：存在大量异常值的特征（如成交量突增）

```python
from sklearn.preprocessing import RobustScaler

# 公式：x_robust = (x - median) / IQR
x_robust = (x - Q50) / (Q75 - Q25)
```

- **适用特征**：
  - `volume_anomaly`
  - `cvd_slope_3`
  - `upvol_downvol_ratio`

- **优点**：对异常值不敏感，稳定性高
- **缺点**：可能损失部分信息

---

### ✅ 4. **Quantile Transformation（分位数变换）**
> 适用于：将任意分布转换为正态或均匀分布

```python
from sklearn.preprocessing import QuantileTransformer

# 将特征映射到标准正态分布
transformer = QuantileTransformer(output_distribution='normal')
x_transformed = transformer.fit_transform(x.reshape(-1,1))
```

- **适用特征**：
  - 高度偏态的特征（如成交量、ATR）
  - 多品种统一建模时（消除品种间尺度差异）

- **优点**：能处理非线性分布
- **缺点**：破坏原始可解释性，慎用！

---

### ✅ 5. **Log / Box-Cox 变换 + 标准化**
> 适用于：右偏分布（长尾），如成交量、波动率

```python
import numpy as np

# 对数变换（先加1防log(0)）
x_log = np.log1p(x)

# 再做 Z-Score
x_log_z = (x_log - x_log.mean()) / x_log.std()
```

- **适用特征**：
  - `volume`
  - `cvd`
  - `ATR`

---

## 🛠️ 三、实战建议：如何选择 & 组合使用？

| 场景 | 推荐方法 |
|------|----------|
| **大多数连续特征** | 滚动 Z-Score（window=20~60） |
| **波动率压缩类** | 滚动 Min-Max（window=20） |
| **成交量、CVD 等易爆发特征** | Robust Scaling 或 Log + Z-Score |
| **多品种联合训练** | Quantile Transformer 或 RobustScaler |
| **希望保持可解释性** | 避免复杂变换，优先用滚动 Z-Score |

---

## 🧩 四、特别提醒：避免“未来信息泄露”

当你使用 **滚动窗口统计量**（如 rolling mean/std/min/max）时，**必须确保只使用当前及之前的数据**。

❌ 错误做法：
```python
scaler.fit_transform(feature)  # 在整个数据集上 fit → 泄露未来信息！
```

✅ 正确做法：
- 使用 `sklearn.pipeline.Pipeline` + `Rolling Window` 自定义变换器
- 或在回测框架中逐点计算（如 `for t in range(n): compute up to t`）
- 推荐库：`tslearn`, `feature-engine`, 或自己实现滚动函数

## ✅ 最终建议

1. **不要对所有特征统一处理**，要根据类型选择合适方法；
2. **优先使用滚动窗口归一化**（如 rolling Z-Score），更适合时序数据；
3. **在训练集上拟合 scaler，在验证/测试集上 transform**，防止数据泄露；
4. **保留原始特征副本**，便于 SHAP 解释时“翻译”回业务语言；
5. **监控归一化后的特征分布**，确保没有异常。

## 📦 代码实现

```python
import numpy as np
import pandas as pd
from sklearn.base import BaseEstimator, TransformerMixin

class FeatureNormalizer(BaseEstimator, TransformerMixin):
    """
    自动特征归一化模块，针对交易因子特征优化设计。
    """

    def __init__(self):
        # 可扩展：定义不同特征类别
        self.log_features = [
            "volume_anomaly", "upvol_downvol_ratio", "bb_width_normalized"
        ]
        self.tanh_features = [
            "poc_deviation_score", "vwap_deviation_pct", "roc_5", "acceleration_3",
            "hma_slope_9", "price_vs_ema_distance", "session_return"
        ]
        self.clip_features = [
            "dist_to_swing_high_pct", "dist_to_swing_low_pct", "range_ratio_5bar"
        ]
        self.zscore_features = [
            "cvd_slope_3", "fractal_resistance_strength", "recent_swing_count_1h"
        ]
        self.decay_features = ["days_since_last_swing_high"]
        self.binary_features = [
            "is_price_at_key_level", "volatility_squeeze_flag",
            "vwap_cross_momentum", "is_london_open", "is_us_open",
            "is_inside_bar", "is_three_inside_up", "is_double_top_test"
        ]
        self.skip_features = ["future_atr_spike_prob", "hour_of_day_sin", "hour_of_day_cos"]
        self.stats_ = {}

    def fit(self, X: pd.DataFrame, y=None):
        X = X.copy()
        # 计算均值和标准差（用于 Z-score）
        for f in self.zscore_features + self.log_features:
            if f in X.columns:
                vals = X[f].replace([np.inf, -np.inf], np.nan).dropna()
                self.stats_[f] = {
                    "mean": vals.mean(),
                    "std": vals.std() + 1e-9,
                }
        return self

    def transform(self, X: pd.DataFrame):
        X = X.copy()

        for col in X.columns:
            if col in self.skip_features:
                continue

            x = X[col].astype(float)

            # 1️⃣ log 归一化 + Z-score
            if col in self.log_features:
                x = np.log1p(np.clip(x, 0, None))
                mean, std = self.stats_[col]["mean"], self.stats_[col]["std"]
                X[col] = (x - mean) / std

            # 2️⃣ tanh 压缩
            elif col in self.tanh_features:
                X[col] = np.tanh(x * 5)

            # 3️⃣ clip 比率类
            elif col in self.clip_features:
                X[col] = np.clip(x, 0, 0.05) / 0.05 if "dist_to_swing" in col else np.clip(x, 0, 2) / 2

            # 4️⃣ Z-score 标准化
            elif col in self.zscore_features:
                mean, std = self.stats_[col]["mean"], self.stats_[col]["std"]
                X[col] = (x - mean) / std

            # 5️⃣ 衰减型
            elif col in self.decay_features:
                X[col] = 1 / (1 + np.clip(x, 0, None))

            # 6️⃣ 二值类
            elif col in self.binary_features:
                X[col] = x.astype(int)

            # 7️⃣ 默认 fallback：tanh
            else:
                X[col] = np.tanh(x)

        return X
```

## ⚙️ 实战建议

📈 **组合层面最佳实践：**

1. 每个维度选 2–3 个代表性强、稳定的特征；
2. 保持总体维度在 30–50 以内（防止噪声特征稀释信号）；
3. 可以引入 “meta feature” 如

   ```python
   df["momentum_strength"] = df["roc_5"] * df["hma_slope_9"]
   df["compression_potential"] = df["bb_width_normalized"]**-1 * df["volume_anomaly"]
   ```
4. 最后通过 PCA / autoencoder / feature importance 进行压缩或筛选。
