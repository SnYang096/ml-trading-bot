非常好的问题 👍——
“**每次训练用多少数据**”是滚动训练系统中**最核心的设计决策之一**，它决定了模型的：

* 泛化能力（是否学会结构规律）
* 稳定性（是否过拟合近期噪声）
* 响应速度（能否及时适应行情）

下面我们按你的体系（BTC、多周期、小波 + Hilbert + LightGBM / Transformer）
给出**精确的量化建议**与背后的理论推导。

## 🧠 一、核心原则：匹配“市场半衰期”

假设市场结构的“平均稳定期”为 ( T_s )，
则训练窗口长度 ( T_w ) 应约为 **2–4 × ( T_s )**。

对于加密货币（尤其BTC）：

* 大周期结构更新约每 2–3 个月；
* 这意味着最优训练窗口约 **6–12 个月**。

✅ **经验黄金法则：**

> 训练数据 ≈ 过去 9–12 个月的滚动窗口

---

## 📊 二、按时间周期（Timeframe）细化建议

| 时间周期          | 每次训练覆盖时长 | 样本数量 (大约)     | 样本代表性            | 说明            |
| ------------- | -------- | ------------- | ---------------- | ------------- |
| **1m (短线)**   | 3–6 个月   | 100k–200k bar | 高频噪声多，需要较多样本     | 高速模型需较多数据缓冲噪声 |
| **5m (中频)**   | 6–9 个月   | 30k–60k bar   | ✅ 最平衡            | 建议主训练窗口       |
| **15m (中低频)** | 9–12 个月  | 10k–20k bar   | ✅ 稳定性最佳          | 推荐用于主趋势识别     |
| **1h (低频)**   | 12–18 个月 | 5k–10k bar    | ✅ 捕捉周期与波段        | 用于方向性模型       |
| **4h+ (长线)**  | 18–24 个月 | 2k–4k bar     | ⚠️ 仅结构研究，不训练短线模型 | 用于多周期对齐信号     |

## ⚙️ 三、滚动训练推荐配置

假设你的系统结构如下：

* **4H 定方向**
* **15m 定结构**
* **1m 入场执行**

那么：

| 模块                                   | 训练窗口       | 数据量     | 更新频率       |
| ------------------------------------ | ---------- | ------- | ---------- |
| Direction Model (4H)                 | 18 个月      | 3k 样本   | 每季度重训      |

| Fusion Layer (Transformer)           | 同步主模型（15m） | 9 个月    | 每季度重训      |
| Risk Model (LightGBM / Sharpe-aware) | 12 个月      | 汇总特征样本  | 每季度重训      |

## 🔍 五、为什么不是“越多数据越好”

很多人误区是：

> “我用2年、3年的数据训练，模型更稳！”

其实这是 **错误的**，因为：

* 市场结构漂移 (drift) 后的旧数据会“污染”模式；
* LightGBM / Transformer 学的是统计特征，不是常识；
* 过老的数据会**拉低特征方差**，导致模型“迟钝”。

✅ 所以：

> 数据量 ≈ 覆盖一个完整市场循环（通常 6–12 个月）
> 是最优的平衡点。
