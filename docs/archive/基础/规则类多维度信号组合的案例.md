## 📌 信号组合命名与定义

---

### 🔹 1. **"Spring Load"（弹簧加载）**
> **核心逻辑**：波动率 + 成交量双重压缩，结构紧致，等待爆发  
> **适用场景**：趋势延续或反转前的极致压缩区

#### ✅ 组成要素：
- `compression_atr_threshold_quantile` < 0.2
- `compression_volume_threshold_quantile` < 0.2
- `structural_compression_body_ratio` < 0.3
- `structural_compression_atr_ratio` < 0.4
- `compression_confidence_threshold` > 0.7

#### 🧩 加分项（用于提升信号强度）：
- `duration_bonus` > 0.1（持续时间长）
- `internal_price_density_threshold` > 0.8（内部价格密集）

#### 💬 命名由来：
像弹簧被压到极限，随时可能反弹。适用于震荡末期。

---

### 🔹 2. **"Silent Accumulation"（静默吸筹）**
> **核心逻辑**：突破前的低波动静默 + 内部价格密集 + 订单吸收迹象  
> **适用场景**：主力吸筹、准备向上突破

#### ✅ 组成要素：
- `pre_breakout_silence_atr_threshold_quantile` < 0.15
- `internal_price_density_threshold` > 0.75
- `order_absorption_bar` > 0.1 或 `order_absorption_tick` > 0.1
- `direction_ordered_entropy_threshold` < 0.3（方向开始有序）

#### 🧩 加分项：
- `cvd_momentum_weak` 存在但未爆发（暗中积累）
- `high_liquidity_time` 为真（主力活动时间）

#### 💬 命名由来：
市场表面平静，但内部有资金悄悄建仓，是“聪明钱”行为。

---

### 🔹 3. **"Breakout Ignition"（突破点火）**
> **核心逻辑**：压缩后首次爆发 + 成交量 spike + CVD 动能确认  
> **适用场景**：趋势启动的第一根强势K线

#### ✅ 组成要素：
- `first_breakout_bar` == True
- `price_breakout` >= 2.0
- `volume_spike` >= 2.0
- `expansion_confidence_threshold` > 0.7

#### 🧩 加分项：
- `volume_and_cvd_aligned` == True（量价CVD三者共振）
- `poc_direction_up` == True（向上突破价值区）
- `volume_dominant` == True（大单主导）

#### 💬 命名由来：
“点火”意味着趋势正式启动，是最佳入场时机之一。

---

### 🔹 4. **"ATR Squeeze + POC Escape"（ATR压缩 + POC逃逸）**
> **核心逻辑**：波动率压缩 + 突破关键价值节点（POC）+ 动量跟进  
> **适用场景**：区间突破、价值区逃离

#### ✅ 组成要素：
- `compression_atr` 权重激活（ATR压缩显著）
- `poc_direction_up` or `poc_direction_down` == True
- `momentum_convergence_price_threshold` 被触发
- `price_breakout` >= 1.5

#### 🧩 加分项：
- `pre_break_proximity` == True（贴近POC边缘突破）
- `cvd_momentum_strong` == True（动量强劲）

#### 💬 命名由来：
结合经典ATR Squeeze与订单流POC理论，兼具技术与微观结构逻辑。

---

### 🔹 5. **"Volatility Vacuum"（波动真空）**
> **核心逻辑**：极低波动 + 极低成交量 + 高价格密度 → 预示剧烈波动即将来临  
> **适用场景**：盘整末端，适合挂单等待突破

#### ✅ 组成要素：
- `volatility_dense_threshold` > 0.8（波动密集但幅度小）
- `compression_volume_threshold_quantile` < 0.1
- `pre_breakout_silence_atr_threshold_quantile` < 0.1
- `internal_price_density_threshold` > 0.9

#### 🧩 加分项：
- `direction_ordered_entropy_threshold` < 0.2（方向即将成型）
- `high_liquidity_time` == True（避免假突破）

#### 💬 命名由来：
市场像“真空”一样缺乏波动，一旦打破，能量巨大。

---

### 🔹 6. **"Momentum Fusion"（动量融合）**
> **核心逻辑**：多种动量信号汇聚 + 压缩释放 + CVD确认  
> **适用场景**：强势趋势中的二次启动

#### ✅ 组成要素：
- `momentum_convergence_price_threshold` 触发
- `cvd_momentum_strong` == True
- `volume_spike` >= 1.5
- `expansion_confidence_threshold` > 0.6

#### 🧩 加分项：
- `poc_direction_up/down` 与趋势一致
- `duration_bonus` 显示前期压缩时间长

#### 💬 命名由来：
多种动量因子“融合”在一起，形成强大推力。

## 一、LightGBM 能自动发现组合信号吗？

### 🔍 原理
LightGBM 是基于决策树的集成模型（GBDT），它通过**分裂节点自动学习特征之间的非线性关系和交互作用**。

### ✅ 能做什么？
- 自动学习 `compression_atr < 0.2` 且 `volume_spike > 2.0` 时上涨概率高
- 捕捉高阶交互，例如：
  ```python
  if (ATR压缩) and (pre_breakout_silence) and (first_breakout_bar):
      return 高收益
  ```
- 在足够多的数据下，它会把这类组合“编码”进树的路径中。

### ❌ 不能做什么？
- **不会给你一个叫 `"Breakout Ignition"` 的标签**。
- **无法告诉你“这个组合为什么有效”** —— 缺乏可解释性。
- 容易过拟合噪声，尤其在金融时间序列中（信噪比低）。
- 对**稀疏但高价值的模式**（如一天只出现1次的吸筹）学习能力弱。

### 📌 结论：
> ✅ **能隐式学习组合信号**  
> ❌ **不能显式输出“信号名称”或“逻辑结构”**  
> ⚠️ **需要大量标注数据（如未来5根K线收益率）来训练**

---

## 二、TabNet 能做得更好吗？

### 🔍 原理
TabNet 是专为表格数据设计的深度模型，使用**sequential attention**机制，对每个决策步骤选择最重要的特征。

### ✅ 能做什么？
- 显式地给出每条样本中**哪些特征被“关注”了**。
- 可以看到模型在做预测时，是否同时关注了：
  - `compression_atr`
  - `pre_breakout_silence`
  - `volume_spike`
  → 这就**接近你定义的 `"Spring Load"` 模式**
- 支持**可解释性分析**（如 Feature Importance per sample）

### 🌟 举个例子：
假设一条数据中，TabNet 的注意力权重显示：
```
Feature                        | Attention Weight
-------------------------------|-----------------
compression_atr                | 0.85
pre_breakout_silence           | 0.80
internal_price_density         | 0.78
volume_spike                   | 0.65
...
```
→ 你可以**反向归纳出**：这很可能就是一个 `"Volatility Vacuum → Ignition"` 模式。

### ❌ 局限性：
- 仍然是统计相关性，不是因果逻辑。
- 注意力高 ≠ 因果驱动（可能只是共现）。
- 训练成本高，调参复杂，对金融小样本不友好。

### 📌 结论：
> ✅ **能部分“发现”并“可视化”特征组合的重要性**  
> ✅ **比 LightGBM 更接近“理解”你设计的信号逻辑**  
> ⚠️ **仍需人工后处理才能提炼出“信号命名”和“交易逻辑”**
