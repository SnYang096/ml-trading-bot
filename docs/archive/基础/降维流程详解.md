## 1. make compare-dim 详细流程

### 1.1 命令格式

```bash
make ts-dim-compare SYMBOL=BTCUSDT \
  START_DATE=2025-05-01 END_DATE=2025-07-31 \
  ENCODING_DIM=16 \
  DIM_COMPARE_FEATURE_TYPE=comprehensive \
  AUTO_ENCODING_GRID=1 \
  AE_AUTO_TUNE=1
```

### 1.2 完整执行流程

#### 阶段 1: 数据加载与特征工程

```python
# 1. 加载市场数据
loader = MarketDataLoader(data_path)
df = loader.load_data(symbol=symbol, start_date=START_DATE, end_date=END_DATE)

# 2. 特征工程（生成 400+ 个原始特征）
comprehensive_engineer = ComprehensiveFeatureEngineer(feature_types=feature_type)
df_features = comprehensive_engineer.engineer_all_features(df, fit=True)

# 3. 创建多周期标签
df_features = create_labels_multi_horizon(df_features, horizons=[1,5,10,15])
```

**输出**:
- 原始特征数量: 400+ 个
- 特征类型: momentum, volatility, mean_reversion, trend, volume, orderflow 等

#### 阶段 2: 四阶段降维流程

```
原始特征 (400+) 
    ↓
阶段 1: 全部特征 (All Features)
    ↓ IC 过滤
阶段 2: IC 过滤特征 (IC-filtered, ~200-300)
    ↓ 相关性去重
阶段 3: 代表特征 (Representatives, ~60-100)
    ↓ Autoencoder 压缩
阶段 4: 压缩特征 (Compressed, 16-64 维)
```

**详细步骤**:

1. **阶段 1: 全部特征**
   - 使用所有 400+ 个特征训练 LightGBM
   - 计算基准性能指标 (R², RMSE, Sharpe)

2. **阶段 2: IC 过滤**
   - 计算每个特征与目标的相关性 (Spearman Correlation)
   - 保留 |IC| > 阈值的特征
   - 通常保留 200-300 个特征

3. **阶段 3: 代表特征选择**
   - 使用贪心相关性过滤 (correlation threshold=0.9)
   - 去除高度相关的冗余特征
   - 保留 60-100 个代表性特征
   - **保存为 `top_factors.json`**

4. **阶段 4: Autoencoder 压缩**
   - 训练 Autoencoder (VAE/Standard AE)
   - 输入: 阶段 3 的代表特征 (60-100 维)
   - 输出: 压缩嵌入 (16-64 维)
   - **保存为 `production_autoencoder.pth`**

#### 阶段 3: 多维度实验

如果启用 `AUTO_ENCODING_GRID=1`:

```python
# 自动生成压缩维度列表
# 基于压缩比: 5x, 10x, 15x, 20x, 30x
compression_ratios = [5, 10, 15, 20, 30]
encoding_dims = [original_dim // ratio for ratio in compression_ratios]
# 例如: [12, 6, 4, 3, 2] (如果 original_dim=60)

# 对每个维度进行实验
for dim in encoding_dims:
    results = run_experiment(encoding_dim=dim)
    grid_results.append(results)
```

**选择最佳维度**:
- 使用 `--selection-metric` 参数 (sharpe | f1 | r2 | composite)
- composite 综合考虑 Sharpe, F1, R², Max Drawdown

#### 阶段 4: 性能评估与保存

```python
# 评估每个阶段的性能
results = {
    "stage1_all_features": evaluate_model(X_all, y),
    "stage2_ic_filtered": evaluate_model(X_ic, y),
    "stage3_representatives": evaluate_model(X_reps, y),
    "stage4_compressed": evaluate_model(X_compressed, y),
}

# 保存结果
save_production_results(results, model, autoencoder, results_dir)
```

## 2. 降维结果保存位置

### 2.1 目录结构

降维结果保存在 `results/production_dimensionality_{SYMBOL}_{FEATURE_TYPE}_{START_DATE}_{END_DATE}/` 目录下。

**示例**:
```
results/production_dimensionality_BTCUSDT_comprehensive_20250501_20250731/
├── production_results.json          # 核心结果文件
├── production_model.pkl              # 最佳 LightGBM 模型
├── production_autoencoder.pth       # 训练好的 Autoencoder
├── top_factors.json                  # 阶段 3 的代表特征列表
├── representative_factors.json      # 同 top_factors.json（兼容格式）
└── {SYMBOL}_{FEATURE_TYPE}_{START_DATE}_{END_DATE}_dimensionality_report.html
```

## 3. make train 如何使用降维结果

### 3.1 基本用法

```bash

make train SYMBOL=BTCUSDT \
  START_DATE=2025-05-01 END_DATE=2025-07-31 \
  TRAIN_USE_TOP_FACTORS=results/production_dimensionality_BTCUSDT_comprehensive_20250501_20250731/top_factors.json

# 方式 2: 使用 Autoencoder（阶段 4 结果）
# 需要手动指定 encoding-dim（从 production_results.json 读取）
# 注意: make train 目前不支持直接使用 autoencoder
# 需要修改代码或使用 auto-rolling-update
```

### 3.2 代码实现流程

在 `src/time_series_model/pipeline/training/train.py` 中：

```python
def main():
    # 1. 解析参数
    args = parse_args()
    
    # 2. 加载数据
    files = _collect_files(args.data, args.data_dir, args.start, args.end, args.symbol)
    raw = _load_many(files)
    
    # 3. 特征工程

        # 加载降维结果中的 top_factors.json

    y = df_features["signal"].values
    model = train_lightgbm(X, y)
    
    # 6. 保存模型
    save_model(model, model_path)
```

## 5. 如何发现漂移

### 5.1 漂移检测模块

系统提供了 `DriftDetector` 类（位于 `src/time_series_model/utils/drift.py`）：

```python
from time_series_model.utils.drift import DriftDetector

# 初始化
detector = DriftDetector(
    js_threshold=0.3,          # Jensen-Shannon 散度阈值
    overlap_threshold=3,        # Top-5 特征重叠阈值
    window_size=5,              # 历史窗口大小
    min_common_features=10      # 最小共同特征数
)

# 在每次滚动训练后记录重要性
for window_end in rolling_windows:
    model = train_model(...)
    importance = model.feature_importance_
    
    # 添加到历史记录
    detector.add_importance(importance)
    
    # 检查是否触发漂移
    triggered, diagnostics = detector.should_trigger()
    
    if triggered:
        print("⚠️ 检测到漂移！")
        print(f"   JS Divergence: {diagnostics['js_divergence']:.3f}")
        print(f"   Top-5 Overlap: {diagnostics['top5_overlap']}")
        print(f"   Importance Change: {diagnostics['importance_change']:.3f}")
        print(f"   当前 Top-5: {diagnostics['latest_top5']}")
        print(f"   历史 Top-5: {diagnostics['historical_top5']}")
        
        # 触发重新降维
        trigger_dimensionality_reduction()
```

### 5.2 漂移检测指标

系统使用以下指标检测漂移：

#### 1. Jensen-Shannon (JS) 散度

```python
from scipy.spatial.distance import jensenshannon

# 比较当前重要性分布 vs 历史平均分布
js_div = jensenshannon(current_importance, historical_importance)

# 阈值: js_div > 0.3 表示漂移
if js_div > 0.3:
    trigger_reselection = True
```

**含义**:
- JS 散度衡量两个概率分布之间的差异
- 0.0 = 完全相同，1.0 = 完全不同
- 阈值 0.3 表示特征重要性分布发生了显著变化

#### 2. Top-5 特征重叠度

```python
top5_current = set(current_importance.nlargest(5).index)
top5_historical = set(historical_importance.nlargest(5).index)
overlap = len(top5_current & top5_historical)

# 阈值: overlap < 3 表示漂移
if overlap < 3:
    trigger_reselection = True
```

**含义**:
- 如果最重要的 5 个特征发生了显著变化，说明模型关注的模式改变了
- 重叠度 < 3 表示至少有 3 个 Top-5 特征被替换

#### 3. 重要性变化幅度

```python
importance_change = np.mean(np.abs(current_importance - historical_importance))

# 阈值: importance_change > 0.1 表示漂移
if importance_change > 0.1:
    trigger_reselection = True
```

### 5.3 完整的漂移检测流程

```python
def detect_drift(rolling_history):
    """检测是否发生漂移"""
    
    # 1. 特征重要性漂移
    js_div = calculate_js_divergence(rolling_history)
    top5_overlap = calculate_top5_overlap(rolling_history)
    
    # 2. 模型性能漂移
    auc_trend = calculate_auc_trend(rolling_history)
    
    # 3. 特征分布漂移（PSI）
    psi_scores = calculate_psi_scores(rolling_history)
    
    # 4. 综合判断
    drift_score = 0.0
    
    if js_div > 0.3:
        drift_score += 0.4
    if top5_overlap < 3:
        drift_score += 0.3
    if auc_trend < -0.02:
        drift_score += 0.2
    if any(psi > 0.2 for psi in psi_scores.values()):
        drift_score += 0.1
    
    return drift_score > 0.5, drift_score
```

## 6. 如何固定长期有效的特征

### 6.1 特征稳定性分析

在降维过程中，系统会分析特征的稳定性：

```python
# 在 dimensionality_comparison.py 中
def analyze_feature_stability(X, y, feature_names, n_splits=5):
    """分析特征在不同时间窗口的稳定性"""
    
    stability_scores = {}
    
    for feature_name in feature_names:
        feature_idx = feature_names.index(feature_name)
        
        # 计算特征在不同时间窗口的 IC
        ic_scores = []
        for train_idx, val_idx in TimeSeriesSplit(n_splits).split(X):
            X_train, X_val = X[train_idx], X[val_idx]
            y_train, y_val = y[train_idx], y[val_idx]
            
            # 计算 IC
            ic = spearmanr(X_val[:, feature_idx], y_val)[0]
            ic_scores.append(ic)
        
        # 稳定性 = IC 的标准差（越小越稳定）
        stability = 1.0 / (np.std(ic_scores) + 1e-8)
        stability_scores[feature_name] = stability
    
    return stability_scores
```

### 6.2 固定核心特征列表

#### 方法 1: 手动指定核心特征

创建一个 `core_features.json` 文件：

```json
{
  "core_features": [
    "rsi_14",
    "macd_12_26_9",
    "atr_14",
    "bb_upper_20",
    "bb_lower_20",
    "volume_sma_20",
    "momentum_5_15",
    "mean_reversion_5_15"
  ],
  "reason": "长期有效、解释性强的技术指标",
  "source": "manual_selection"
}
```

在降维时，强制保留这些特征：

```python
# 在 dimensionality_comparison.py 中
def select_representative_features(X, y, feature_names, core_features=None):
    """选择代表特征，但强制保留核心特征"""
    
    # 1. 计算所有特征的 IC
    ic_scores = calculate_ic_scores(X, y, feature_names)
    
    # 2. 相关性过滤
    selected = []
    for i, name in enumerate(feature_names):
        if name in core_features:
            # 强制保留核心特征
            selected.append(name)
            continue
        
        # 检查相关性
        is_correlated = any(
            abs(correlation(X[:, i], X[:, feature_names.index(s)]) > 0.9)
            for s in selected
        )
        
        if not is_correlated and abs(ic_scores[i]) > 0.1:
            selected.append(name)
    
    return selected
```

#### 方法 2: 基于历史稳定性自动筛选

```python
def select_stable_features(stability_scores, top_k=50):
    """选择稳定性最高的特征"""
    
    # 按稳定性排序
    sorted_features = sorted(
        stability_scores.items(),
        key=lambda x: x[1],
        reverse=True
    )
    
    # 选择 Top-K
    stable_features = [name for name, score in sorted_features[:top_k]]
    
    return stable_features
```

#### 方法 3: 解释性特征优先

在降维时，优先保留具有明确解释性的特征：

```python
# 定义解释性特征类别
interpretable_categories = {
    "momentum": ["rsi_14", "roc_10", "momentum_5_15"],
    "volatility": ["atr_14", "bb_width_20", "std_20"],
    "trend": ["sma_20", "ema_12", "macd_12_26_9"],
    "volume": ["volume_sma_20", "volume_ratio", "obv"],
    "orderflow": ["cvd", "taker_buy_ratio", "imbalance"]
}

def prioritize_interpretable_features(feature_names, interpretable_categories):
    """优先保留解释性特征"""
    
    prioritized = []
    
    # 1. 先添加解释性特征
    for category, features in interpretable_categories.items():
        for feature in features:
            if feature in feature_names and feature not in prioritized:
                prioritized.append(feature)
    
    # 2. 再添加其他特征（按 IC 排序）
    remaining = [f for f in feature_names if f not in prioritized]
    ic_scores = calculate_ic_scores(remaining)
    remaining_sorted = sorted(remaining, key=lambda x: abs(ic_scores[x]), reverse=True)
    
    prioritized.extend(remaining_sorted)
    
    return prioritized
```

## 8. 总结

### 8.1 关键要点

1. **降维结果保存位置**: `results/production_dimensionality_{SYMBOL}_{FEATURE_TYPE}_{START_DATE}_{END_DATE}/`
2. **核心文件**:
   - `production_results.json`: 包含压缩维度和性能指标
   - `top_factors.json`: 代表特征列表（用于 `--use-top-factors`）
   - `production_autoencoder.pth`: Autoencoder 模型（用于 `--use-autoencoder`）
3. **train 和 rolling 如何使用**:
   - 通过 `--use-top-factors` 参数指定 `top_factors.json` 路径
   - 通过 `--use-autoencoder` 和 `--encoding-dim` 参数使用 Autoencoder 压缩（`rolling` 已支持）
4. **漂移检测**: 使用 `DriftDetector` 类，基于 JS 散度、Top-5 重叠度、重要性变化
5. **固定特征**: 创建 `core_features.json`，在降维时强制保留，或基于稳定性自动筛选

### 8.2 最佳实践

1. **降维频率**: 每季度执行一次，或检测到漂移时触发
2. **特征稳定性**: 优先保留解释性强、长期有效的特征
3. **版本管理**: 使用版本控制管理特征列表，支持回滚
4. **监控**: 持续监控特征重要性变化，及时检测漂移
