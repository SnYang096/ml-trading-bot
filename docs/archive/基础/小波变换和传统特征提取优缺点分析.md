- 小波基函数选择（db4, haar, la8）会影响分解效果

- 归一化方式、融合权重都需要调参

> 这些选择会影响最终信号，但没有“唯一正确”的答案，增加了黑箱感。

### ❌ 3. **能量本身没有方向性（需后处理）**
- 小波能量 $|d_j(t)|^2$ 是非负的，只反映“活跃度”，不反映方向（涨 or 跌）
- 你需要结合原始系数符号或其他信号来判断方向

> 比如：高能量 + 正系数 → 上涨趋势；高能量 + 负系数 → 下跌趋势

### ❌ 4. **边缘效应（Edge Effects）**
- 小波分解在序列两端会出现边界失真（尤其在实时交易中，最新数据可能不准）
- 需要补数据（zero-padding, symmetric extension），但这会引入人工噪声

class WaveletFeatureExtractor:
    """小波特征提取器"""
    
    def __init__(self, wavelet='db4', level=4):
        self.wavelet = wavelet
        self.level = level
        self.price_buffer = deque(maxlen=200)
    
    def update(self, new_bar):
        """更新数据"""
        self.price_buffer.append(new_bar.close)
    
    def get_features(self):
        """获取小波特征"""
        if len(self.price_buffer) < 50:  # 最小数据要求
            return {}
        
        prices = np.array(self.price_buffer)
        coeffs = pywt.wavedec(prices, self.wavelet, level=self.level)
        
        return {
            'cA_trend_strength': np.polyfit(np.arange(len(coeffs[0])), coeffs[0], 1)[0],
            'cD1_energy': np.sum(coeffs[1]**2),
            'cD2_energy': np.sum(coeffs[2]**2),
            'multiscale_corr': np.corrcoef(coeffs[0][:len(coeffs[1])], coeffs[1])[0, 1]
        }

def evaluate_features(features_df):
    """评估特征预测能力"""
    # 特征与目标的相关性
    feature_target_corr = features_df.corr()['target'].abs().sort_values(ascending=False)
    
    # 特征重要性（通过随机森林）
    X = features_df.drop('target', axis=1)
    y = features_df['target']
    rf = RandomForestClassifier(n_estimators=100)
    rf.fit(X, y)
    feature_importances = pd.Series(rf.feature_importances_, index=X.columns).sort_values(ascending=False)
    
    # 特征稳定性（滚动相关系数）
    stability_scores = {}
    for feature in X.columns:
        rolling_corr = features_df[feature].rolling(100).corr(features_df['target'])
        stability_scores[feature] = rolling_corr.std()  # 标准差越小越稳定
    
    return {
        'correlation': feature_target_corr,
        'importance': feature_importances,
        'stability': stability_scores
    }


3. 模型评估与选择

def select_best_model(model_performance):
    """基于多指标选择最佳模型"""
    # 定义评估指标权重
    weights = {
        'accuracy': 0.3,
        'precision': 0.2,
        'recall': 0.2,
        'f1': 0.3,
        'sharpe': 0.5,  # 回测夏普比率
        'max_drawdown': -0.5  # 回撤是负向指标
    }
    
    # 计算加权得分
    best_score = -np.inf
    best_model = None
    
    for model_name, metrics in model_performance.items():
        score = 0
        for metric, weight in weights.items():
            # 归一化指标值
            if metric == 'max_drawdown':
                # 回撤越小越好，取负值
                normalized = -metrics[metric]
            else:
                normalized = metrics[metric]
            
            score += normalized * weight
        
        if score > best_score:
            best_score = score
            best_model = model_name
    
    return best_model, best_score

### 🔹 第一阶段：双轨并行 —— 小波 vs. t-digest 特征提取

#### 目标：
- 验证小波是否比 t-digest 更早、更准地捕捉到“压缩结束”和“趋势启动”
- 不改变现有系统逻辑，仅新增小波信号作为对照

#### 架构设计：

```text
原始价格/订单流
     ↓
┌────────────┐     ┌──────────────────────┐
│ t-digest 模块 │ ←→ │ 波动压缩识别（当前系统） │
└────────────┘     └──────────────────────┘
     ↓                             ↓
【传统特征】                     【小波能量特征】
  - 当前波动率                    - E_C(t, τ): 压缩能量
  - 历史分位数                    - E_S(t, τ): 结构能量
  - Z-score / 突破信号             - E_OF(t, τ): 订单流动能
                                - E_total(t, τ): 共振强度

     ↓ 各自生成信号
【Signal_tdigest】 ←→ 【Signal_wavelet】
     ↓
[可视化对比面板] → 观察哪个信号更及时、更少误报
```

#### 实现要点：

1. **保留原有 t-digest 压缩判断逻辑**
   ```python
   # 示例：t-digest 判断是否处于压缩
   if current_vol < tdigest.quantile(0.2):
       in_compression = True
   ```

2. **新增小波压缩能量判断**
   ```python
   # 小波判断：中短期尺度（τ=8~32min）能量持续下降
   if np.mean(E_C[t, scale_idx]) < threshold and trend_downward(E_C):
       in_compression_wavelet = True
   ```

3. **定义“突破”事件**
   - t-digest：价格突破历史分位数（如 90%）
   - 小波：`E_total` 在某个尺度上突增 + `E_OF` 方向一致

4. **打标签记录每个信号触发时间、尺度、强度**

### 🔹 第二阶段：效果评估与对比分析

#### 目标：
- 回测或观察两种方法在历史行情中的表现
- 回答：“谁更早发现突破？”、“谁的误报更少？”

#### 分析维度：

| 指标 | t-digest | 小波 |
|------|---------|------|
| **平均提前量**（分钟） | 0 | +3.2 ⭐ |
| **假信号次数** | 7 | 3 ⭐ |
| **捕获大行情比例** | 60% | 85% ⭐ |
| **对震荡市敏感度** | 高（频繁触发） | 低（能量低）⭐ |
| **参数稳定性** | 强依赖窗口大小 | 较稳定（多尺度冗余） |

#### 推荐分析方法：

1. **人工标注关键转折点**（如趋势反转、大单冲击）
2. 计算两个系统的：
   - 召回率（Recall）：是否抓住了这些转折？
   - 精确率（Precision）：发出的信号中有多少是真的？
   - 提前量（Lead Time）：比价格突破早了几根K线？

3. **绘制叠加图**：
   - K线图 + t-digest 区间 + 小波能量热力图
   - 对比两者在盘整末期的表现

### 🔹 第三阶段：融合升级 —— 规则融合 or 机器学习

#### 选项 A：规则融合（轻量级，推荐先做）

> 如果小波表现更好，可以用它为主；如果各有优势，就融合。

```python
def final_signal():
    signal_strength = 0.0

    # 权重分配（可调）
    if in_compression_tdigest:
        signal_strength += 0.3
    
    if in_high_wavelet_compression_energy:  # 小波压缩能量高
        signal_strength += 0.5

    if order_flow_spike_detected:
        signal_strength += 0.2 * direction_confirmed

    if signal_strength > threshold:
        return "BREAKOUT_WATCH"
```

✅ 优点：透明、可控、易调试

#### 选项 B：机器学习融合（进阶）

> 把 t-digest 特征 和 小波特征 都输入模型，让算法自己学权重。

##### 输入特征示例：

```python
features = [
    # --- t-digest 类 ---
    current_vol / long_term_vol,           # 波动率比率
    tdigest.cdf(price),                   # 当前价格分位数
    vol_change_5min,                      # 近期波动变化

    # --- 小波类 ---
    E_C[tau_8min],                        # 8分钟压缩能量
    E_S[tau_32min],                       # 32分钟趋势能量
    E_OF[tau_4min],                       # 短期订单流动能
    E_total_max_scale,                    # 最强共振尺度能量
    energy_skewness,                      # 能量分布偏度（集中度）

    # --- 其他辅助 ---
    spread, volume_imbalance,            # 流动性因子
]
```

##### 模型选择：
- **LightGBM / XGBoost**：适合表格特征，可解释性强
- **LSTM**：如果你有完整时序能量矩阵（T x Scales），可建模能量演化
- **异常检测模型（Isolation Forest）**：直接检测“能量异动”

🎯 输出目标：
- 二分类：是否将在未来 N 根K线内突破？
- 多分类：盘整 / 上涨 / 下跌 / 震荡

## 💡 关键提醒

1. **不要追求“完美模型”，而要追求“可解释的进步”**
   - 即使小波只比 t-digest **早 1-2 根K线**发现突破，也可能带来巨大优势

2. **小波不是万能的**
   - 在极端噪声下也会误判
   - 但它能帮你**过滤掉很多 t-digest 无法识别的伪压缩**

3. **最终目标不是“用哪个”，而是“怎么组合最稳”**
   - 正如雷达+红外+光学传感器融合一样，多源验证才是王道
