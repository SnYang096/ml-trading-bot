## 🚀 你的三阶段进化路线详解

### ✅ 第一阶段：规则类组合信号（Rule-Based Baseline）
> 目标：建立**可解释、逻辑清晰、有市场含义**的基线策略

#### 🔧 核心任务：
- 将 `Spring Load`、`Breakout Ignition` 等信号编码为函数
- 构建布尔信号或得分信号（0/1 或 0~5 分）
- 回测胜率、盈亏比、最大回撤等指标

#### 📦 推荐库：

| 功能 | 推荐库 | 说明 |
|------|--------|------|
| **信号工程** | `pandas`, `numpy` | 基础数据处理 |
| **K线形态识别** | `talib` (Technical Analysis Library) | ATR、MA、RSI 等指标 |
| **回测框架** | [`backtrader`](https://www.backtrader.com/) 或 [`vectorbt`](https://github.com/polakowo/vectorbt) | 后者支持 pandas pipeline，更灵活 |
| **可视化** | `plotly`, `cufflinks` | 画出信号触发点、压缩区间 |
| **模式命名与管理** | 自定义 Python 类或 `pydantic` | 把每个信号封装成类 |

```python
# 示例：Spring Load 信号
def is_spring_load(df_row, atr_q=0.2, vol_q=0.2):
    return (df_row['compression_atr'] < atr_q and 
            df_row['compression_volume'] < vol_q and
            df_row['structural_compression_body_ratio'] < 0.3)
```

✅ **输出**：一个 `.csv` 或 `DataFrame`，包含每根K线的信号得分

### ✅ 第二阶段：LightGBM 提升（Gradient Boosting Boost）
> 目标：在规则基础上，**用机器学习捕捉非线性交互和边缘案例**

#### 🔧 核心任务：
- 输入：原始特征 + 规则信号（作为高阶特征）
- 输出：未来 N 根 K 线的收益率分类（涨/跌）或回归
- 使用 SHAP 解释模型是否“理解”了你的规则

#### 📦 推荐库：

| 功能 | 推荐库 | 说明 |
|------|--------|------|
| **模型训练** | [`lightgbm`](https://lightgbm.readthedocs.io/) | 快速、高效、支持类别特征 |
| **特征重要性** | [`shap`](https://github.com/shap/shap) | 可视化每条样本的特征贡献 |
| **超参调优** | [`optuna`](https://optuna.org/) 或 [`hyperopt`](http://hyperopt.github.io/hyperopt/) | 自动搜索最优参数 |
| **交叉验证** | `sklearn.model_selection.TimeSeriesSplit` | 时间序列专用 CV，防止未来信息泄露 |
| **集成方式** | `VotingClassifier` / `Stacking` | 可结合多个规则+模型输出 |

```python
import lightgbm as lgb
import shap

model = lgb.LGBMClassifier(n_estimators=1000)
model.fit(X_train, y_train)

# 解释模型
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_test)
shap.summary_plot(shap_values, X_test)
```

✅ **关键技巧**：
- 把 `signal_spring_load`, `signal_ignition` 作为输入特征
- 如果 SHAP 显示这些信号权重高 → 说明模型认可你的逻辑
- 如果发现新组合（如 `pre_breakout_silence + poc_direction_up` 权重高）→ 可反向设计新规则

### ✅ 第三阶段：注意力神经网络（Attention for Long-Term Dependencies）
> 目标：捕捉**跨多个周期的复杂时序模式**，如“静默3天后爆发”

#### 🔧 核心任务：
- 模型能看过去 `T` 根K线（如 T=64）
- 学习哪些时间步和哪些特征最重要
- 发现规则和 GBDT 难以捕捉的**长程依赖**

#### 📦 推荐模型与库：

| 模型 | 推荐实现 | 说明 |
|------|----------|------|
| **Transformer for Tabular Time Series** | [`tsformer`](https://github.com/WenjieDu/TSFormer), [`TabTransformer`](https://github.com/google-research/google-research/tree/master/tab_transformer) | 支持特征+时间双注意力 |
| **SAINT** | https://github.com/somepago/saint | 当前 SOTA 表格模型之一 |
| **Informer / Autoformer** | 用于长序列预测 | 更适合价格预测，不太适合信号分类 |
| **PyTorch + Custom Attention** | 手写 `MultiheadAttention` 层 | 最灵活，可控性强 |

#### 🛠️ 建议架构：
```text
Input: [Batch, Sequence Length T, Features F]
       ↓
Embedding Layer (数值归一化 + 类别嵌入)
       ↓
Transformer Encoder / SAINT
       ↓
Global Average Pooling 或 [CLS] Token
       ↓
MLP Head → 输出：买入/卖出/持有
```

#### 📦 必备库：
- `pytorch` 或 `tensorflow`
- `pytorch-forecasting`（高级接口）
- `tianshou`（如果做强化学习）
- `captum`（PyTorch 的可解释性库，类似 SHAP）

```python
from torch.nn import MultiheadAttention
# 可视化 attention weights
attn_output, attn_weights = multihead_attn(query, key, value)
```

- 是否在 `volume_spike` 出现前就提升了对 `internal_price_density` 的注意力？
- → 这些都可能成为新的**领先信号**
