## 🧩 一、为什么“只看特征重要性”是危险的？

LightGBM/XGBoost 的 `feature_importance` 虽然方便，但有**三大陷阱**：

| 陷阱 | 说明 |
|------|------|
| **1. 偏向高频特征** | 小波包、高频能量这类“变化快”的特征容易被高估 |
| **2. 忽视协同效应** | 某个特征单独不重要，但和另一个组合就很强（如：CVD + PoC） |
| **3. 过拟合信号** | 历史数据中偶然有效的特征，会被误判为“重要” |

> 💥 举个例子：
> 你有一个“过去5分钟BTC涨了且ETH也涨了”的特征，在2024年牛市中重要性很高，但2025年市场结构变了，它就失效了。

---

## 🛠 二、正确的特征筛选流程（专业机构做法）

推荐使用 **五步筛选法**：

```text
原始300特征
     ↓
① 基础过滤（去无效）
     ↓
② 相关性分析（去冗余）
     ↓
③ 模型重要性初筛
     ↓
④ 稳定性与可解释性评估
     ↓
⑤ 在线验证（A/B 测试）
     ↓
最终 30~50 个核心特征
```

---

### ✅ 步骤 1：基础过滤（去无效特征）

#### 目标：删除**无信息量**的特征

```python
# 1. 删除常数特征
from sklearn.feature_selection import VarianceThreshold
sel = VarianceThreshold(threshold=0.01)
X = sel.fit_transform(X)

# 2. 删除唯一值过多的特征（如时间戳）
def remove_high_cardinality(df, threshold=0.9):
    for col in df.columns:
        if df[col].nunique() / len(df) > threshold:
            df.drop(col, axis=1, inplace=True)
    return df

# 3. 删除缺失率 > 30% 的特征
missing_ratio = df.isnull().mean()
valid_cols = missing_ratio[missing_ratio < 0.3].index
```

> 🎯 目标：从 300 → 250 左右

---

### ✅ 步骤 2：相关性分析（去冗余）

#### 目标：删除**高度相关**的特征（避免模型抖动）

```python
import numpy as np
import pandas as pd

# 计算相关性矩阵
corr_matrix = df.corr().abs()

# 找出上三角矩阵中相关性 > 0.9 的特征对
upper = corr_matrix.where(
    np.triu(np.ones(corr_matrix.shape), k=1).astype(bool)
)
to_drop = [column for column in upper.columns if any(upper[column] > 0.9)]

# 删除冗余特征（保留更稳定的那个）
df_reduced = df.drop(to_drop, axis=1)
```

> 💡 技巧：在“小波包”这类分解特征中，通常只保留能量最高的2-3个频段。

> 🎯 目标：250 → 150

---

### ✅ 步骤 3：模型重要性初筛（LightGBM）

#### 目标：用模型识别“有预测力”的特征

```python
from lightgbm import LGBMClassifier
import matplotlib.pyplot as plt

model = LGBMClassifier(n_estimators=100, device='gpu')
model.fit(X_train, y_train)

# 获取重要性
importance = model.feature_importances_
feature_names = X_train.columns

# 排序并画图
indices = np.argsort(importance)[::-1]
plt.bar(range(50), importance[indices[:50]])
plt.xticks(range(50), [feature_names[i] for i in indices[:50]], rotation=90)
plt.show()
```

> 🎯 策略：
> - 保留前 80% 累计重要性的特征
> - 或保留重要性 > 平均值 1.5 倍的特征

> 🎯 目标：150 → 80

---

### ✅ 步骤 4：稳定性与可解释性评估（关键！）

#### 目标：确保特征**长期有效、逻辑清晰**

| 评估维度 | 方法 | 工具 |
|----------|------|------|
| **时间稳定性** | 滚动重要性变化小 | `sklearn.model_selection.TimeSeriesSplit` |
| **经济逻辑** | 是否有交易意义？ | 人工判断（如：CVD 流入 → 看涨） |
| **过拟合检测** | 验证集 vs 测试集重要性差异 < 20% | 滚动回测 |
| **方向一致性** | 特征系数符号稳定（+/-） | SHAP 值分析 |

```python
# 用 SHAP 解释模型
import shap
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X_val)

# 查看某个特征的 SHAP 值分布
shap.summary_plot(shap_values, X_val, plot_type="bar", max_display=20)
```

> 🎯 目标：80 → 50

---

### ✅ 步骤 5：在线验证（A/B 测试）

#### 目标：实盘验证特征价值

```text
策略A：使用旧特征集
策略B：使用新筛选特征集
↓
运行1周，对比：
- 收益率
- 最大回撤
- 夏普比率
- 信号频率
```

> ✅ 只有实盘表现更好，才算“真正有效”。

## ✅ 总结：特征筛选 checklist

| 步骤 | 操作 | 工具 |
|------|------|------|
| 1️⃣ 去无效 | 删除常数、高缺失、高基数特征 | `VarianceThreshold`, `nunique` |
| 2️⃣ 去冗余 | 删除相关性 > 0.9 的特征 | `pandas.corr()` |
| 3️⃣ 初筛 | LightGBM 重要性排序 | `feature_importances_` |
| 4️⃣ 评估 | 稳定性 + SHAP + 逻辑判断 | `SHAP`, `TimeSeriesSplit` |
| 5️⃣ 验证 | A/B 测试实盘表现 | 回测系统 |
