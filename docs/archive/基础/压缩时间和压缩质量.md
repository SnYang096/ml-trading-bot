### 你的观点解析

你说：
> “持续时间对压缩判断似乎没有用，只能说对以后真假突破的概率有点影响”

这可以分解为两个精准的洞察：

1.  **“对压缩判断没有用”**：
    *   正确。判断当前是否处于“压缩”状态，依赖的是**当前的价格波动率、成交量、价格范围等即时特征**（如低 ATR、窄幅震荡、低成交量）。
    *   持续时间是“事后”信息，它不能帮助你定义“什么是压缩”，只能告诉你“这个压缩已经持续了多久”。

2.  **“对以后真假突破的概率有点影响”**：
    *   这正是 `duration_bonus` 这个设计的理论基础——**能量积蓄假说**（Energy Accumulation Hypothesis）。
    *   市场在长期盘整（压缩）后，积累的“势能”可能更大，一旦突破，真突破的概率或后续动能可能更强。

---

### 深入讨论：持续时间真的能预测真假突破吗？

这是一个**有争议且需要谨慎对待**的问题。你的直觉是对的，它的作用很有限。

#### ✅ 理论上的合理性（为什么有人相信）

*   **供需平衡打破**：长时间的盘整意味着多空双方力量均衡。时间越长，意味着更多交易者被“困”在区间内。一旦一方获胜，这些被困的交易者可能集中平仓或追涨杀跌，导致突破更猛烈。
*   **期权Gamma效应**：在传统金融市场，长期盘整可能导致大量期权头寸集中在区间边界。当价格突破时，做市商需要动态对冲，可能加剧趋势。
*   **市场心理**：交易者对“长期盘整后的突破”往往更关注，可能形成自我实现的预言。

#### ❌ 现实中的局限性和陷阱

1.  **“久盘必跌” vs “久盘必破”**：
    *   市场没有义务遵循你的预期。长时间压缩后，可能不是爆发式突破，而是**缓慢的、无动能的假突破**，或者直接在区间内继续盘整更久。
    *   在低流动性或特定事件背景下，“久盘”反而可能预示着参与者失去兴趣，导致突破失败。

2.  **“时间”不是唯一因素**：
    *   一个持续10天的压缩，如果发生在重大新闻（如美联储议息）之前，其意义与一个无消息面驱动的10天盘整完全不同。
    *   **背景信息（Context）** 往往比单纯的持续时间更重要。

3.  **数据过拟合风险**：
    *   如果你在历史数据中发现“压缩超过N天后突破成功率为X%”，这可能是偶然的统计偏差。未来市场结构变化，这个规律可能失效。

4.  **无法量化“能量”**：
    *   我们用“持续时间”作为“能量”的代理变量，但这很粗糙。真正的“势能”还取决于持仓量、订单簿深度、宏观环境等，而不仅仅是时间。

---

### 更务实的应用建议

既然“持续时间”不能保证突破成功，那 `duration_bonus` 还有用吗？答案是：**可以用，但要降低期望，并结合其他证据**。

#### 如何正确使用 `duration_bonus`

1.  **不要作为主要信号**：
    *   它应该是一个**辅助权重**，而不是决策依据。例如，在多个突破信号中，给“长期压缩后突破”的信号稍高一点的优先级。

2.  **结合突破时的确认信号**：
    *   即使压缩了很久，也要看突破时的**成交量、速度、收盘价确认**。
    *   高成交量 + 快速脱离区间 + 收盘站稳 = 更可靠的信号，此时 `duration_bonus` 可以“锦上添花”。

3.  **动态调整期望值**：
    *   长期压缩后的突破，**预期收益（TP）可以设得稍大一些**，因为理论上动能可能更强。
    *   但**止损（SL）仍需严格执行**，因为假突破的风险依然存在。

4.  **考虑“压缩质量”而非仅“压缩时间”**：
    *   一个窄幅、低成交量、多次测试边界的压缩，比一个宽幅、高成交量的“伪压缩”更有价值。
    *   可以设计一个“压缩质量评分”，再乘以“持续时间因子”。

### 结论

你的理解完全正确：

*   **压缩持续时间本身并不能用于判断“当前是否压缩”**——那是由波动率等指标决定的。
*   **它最多只能作为一个弱先验（weak prior），略微提高我们对“真突破”概率的信念**，但绝非保证。

`_calculate_duration_bonus` 这样的函数，其价值不在于“预测未来”，而在于：

> **在多个相似的突破机会中，优先选择那些‘看起来更有潜力’的，即‘酝酿更久’的。**

这是一种**风险管理中的优先级排序**，而不是一种“圣杯”信号。认识到这一点，就能避免过度依赖它，从而构建更稳健的交易系统。

## ✅ 三、如何改进？让你的“压缩质量”更清晰、更有交易意义

### ✅ 改进建议 1：明确提取“Compression Quality Score”

你现在有 `total_score`，但它是用于**判断是否处于压缩状态**的决策分数。

你可以再设计一个独立的 **`compression_quality`** 指标，在确认已进入压缩后，评估其“成色”。

```python
def get_compression_quality(self) -> float:
    """
    计算当前压缩区的质量得分 (0.0 ~ 1.0)
    只有在处于压缩状态时才有意义
    """
    if not self.is_compression() or self._compression_duration < 3:
        return 0.0

    components = {}

    # 1. 波动抑制质量：越低越好
    bw_atr = self._compute_bandwidth_atr(self._atrs[-1])
    atr_quantile = self._t_atr_estimator.cdf(bw_atr)  # 注意：cdf 返回的是 <=x 的概率
    volatility_suppression = 1.0 - atr_quantile  # 值越大表示波动越小

    # 2. 成交量萎缩质量
    bw_vol = self._compute_bandwidth_volume(self._vol_ma_indicator.value)
    vol_quantile = self._t_vol_estimator.cdf(bw_vol)
    volume_dryness = 1.0 - vol_quantile

    # 3. K线实体紧凑性
    recent_body_ratios = list(self._body_ratios)[-self._compression_duration:]
    avg_body_ratio = np.mean(recent_body_ratios)
    body_compactness = 1.0 - min(avg_body_ratio, 1.0)

    # 4. 内部价格密度（新）
    price_range = self._compression_high - self._compression_low
    if price_range > 1e-8:
        total_range = max(self._highs) - min(self._lows)
        confinement_ratio = price_range / (total_range + 1e-8)
        internal_density = 1.0 - confinement_ratio
    else:
        internal_density = 1.0

    # 5. 持续时间质量
    duration_quality = min(self._compression_duration / 50, 1.0)

    # 加权合成（可根据策略偏好调整权重）
    quality = (
        0.3 * volatility_suppression +
        0.2 * volume_dryness +
        0.2 * body_compactness +
        0.15 * internal_density +
        0.15 * duration_quality
    )

    return max(0.0, min(1.0, quality))
```

> 💡 提示：这个函数可以作为 `get_state()` 或诊断接口的一部分返回。

### ✅ 改进建议 2：增加“压缩纯度”检测（Purity Filter）

防止把“下跌末期阴跌”误判为高质量压缩。

```python
def _is_clean_compression(self) -> bool:
    """检查压缩是否干净，排除单边贴边情况"""
    if not self.is_compression():
        return False

    highs = [b.high.as_double() for b in self._bars[-self._compression_duration:]]
    lows = [b.low.as_double() for b in self._bars[-self._compression_duration:]]

    upper_shadow_ratio = np.mean([(h - max(c, o)) / (h - l + 1e-8) for h, l, c, o in zip(highs, lows, closes, opens)])
    lower_shadow_ratio = np.mean([(min(c, o) - l) / (h - l + 1e-8) for h, l, c, o in zip(highs, lows, closes, opens)])

    # 影线不能太偏向一侧
    return abs(upper_shadow_ratio - lower_shadow_ratio) < 0.3
```
