📋 指标说明

  • CV RMSE (交叉验证均方根误差)：收益回归模型的预测误差，越低越好。
    ✅ 优秀：RMSE 较小，说明预测精度高。
    ⚠️ 注意：如果 RMSE 过大，说明模型预测不准确。
  • CV MSE (交叉验证均方误差)：收益回归模型的预测误差平方，越低越好。
  • F1 (F1 Score)：分类模型的方向性预测 F1 分数，综合精确率和召回率。
    阈值：F1 > 0.3 为良好，F1 > 0.5 为优秀。
    ⚠️ F1 = 0 或 None：模型完全无预测能力，自动标记为不可用。
    反映模型对方向（涨/跌）的预测能力。
  • Acc (Accuracy)：分类模型的方向性预测准确率，反映模型预测方向的正确率。Acc > 0.5 表示模型具备预测能力。
  • Prec / Rec：Precision 表示预测为“涨”时的正确率，Recall 表示捕获所有“涨”的能力。
  • AUC (ROC AUC)：方向性预测 ROC 曲线下面积。
    阈值：AUC > 0.6 为良好，AUC > 0.7 为优秀。
    ⚠️ 注意：仅 AUC 优秀不足以判定模型质量，需要结合其他指标。
  • PR-AUC (Precision-Recall AUC)：不平衡数据集上更敏感的指标。
    阈值：PR-AUC > 0.4 为良好，PR-AUC > 0.6 为优秀。
    ⚠️ 注意：仅 PR-AUC 优秀不足以判断模型质量。
  • Quality (模型质量)：综合评估 CV 指标 + OOS 指标 + 模型可用性。
    ✅ 可用：CV 方向指标达标且 OOS 指标达标 (Accuracy≥50%、AUC≥50%、F1≥0.5、IC≥0.05)。
    ⭐ 优秀：F1≥0.5 且 AUC≥0.7 且 PR-AUC≥0.6 均达标。
    ❌ 不可用：CV 指标不达标或 OOS 指标不达标。
    ⚠️ 重要：F1=0 表示模型无预测能力；OOS 指标不达标也需标记为不可用。
  • 评估建议：综合 CV 和 OOS 的 F1、AUC、PR-AUC、IC 等指标，所有指标达到优秀水平时模型质量才算过关。单一指标优秀不足以判断模型质量，OOS 指标更重要。

🧠 模型架构

使用 3 个模型（Classification, Return Regression, Volatility）的组合架构：

  • Classification 模型：预测未来涨跌方向的概率（0-1），用于判断方向。
  • Return Regression 模型：预测未来收益率幅度，用于估计潜在收益。
  • Volatility 模型：预测未来波动率，用于风险调整与仓位管理。

🧭 决策逻辑

signal_strength = p_up * expected_return / expected_volatility

  • p_up：分类模型预测的上涨概率。
  • expected_return：收益回归模型预测的期望收益。
  • expected_volatility：波动率模型预测的期望波动率。

signal_strength 用于仓位分配与交易决策。

💡 使用说明

  • 分类模型评估：关注 Accuracy、F1、AUC、PR-AUC，F1≥0.5 且 AUC≥0.7 且 PR-AUC≥0.6 时认为过关。
  • 收益回归模型评估：关注 RMSE、MSE、R² 等指标，越低越好。
  • 波动率模型评估：关注 RMSE、MSE，波动率预测准确有助于风险管理。
  • 综合评估：优先关注 OOS 指标，因其反映真实场景表现。
  • 信号生成：根据 signal_strength 做仓位分配、交易决策、风险管理。

💡 特征类型比较

  • baseline：基础特征，数量少、计算快，适合快速验证。
  • default：默认特征（TA-Lib + base_indicators），推荐用于生产。
  • enhanced：增强特征，包含更多高级特征但计算较慢。
  • comprehensive：综合特征，最丰富但可能过拟合。

建议比较不同特征类型时重点关注分类 F1、AUC、PR-AUC，以及收益回归的 RMSE、R² 与 OOS 表现。
