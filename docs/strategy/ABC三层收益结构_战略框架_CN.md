# A / B / C 三层收益结构 — 战略框架（笔记）

本文档固化当前对**多账本、多 payoff** 的战略分工，避免在同一套心理上混用「慢凸性」与「结构 alpha」「短周期微观优势」三套逻辑。

---

## 1. 为什么需要分层

单一策略无法在所有 regime 上最优；混淆目标会导致：

- 用 **长期凸性账本**的持仓尺度去扛 **摆动 alpha**，牛熊切换时把已兑现的 alpha 吐成裸露 beta；
- 用 **短周期系统**却突然「格局」，edge 衰减后暴露在库存风险里。

分层后，每层有独立的 **hypothesis、sizing、KPI、心理预期**，最好在子账户或独立风控上**物理隔离**。

### 1.1 为何会拆成 A / B / C：避免让 B 「代班」扛肥尾

拆三层的直接动因来自一次清晰分工：**不能把「整条超级行情的凸性暴露」强加在本来就为 swing 设计的 B 账本（BPC / TPC / ME 这一路）上。**  
若坚持用 B 去「抓 fat-tail」——长拿、放宽失效、偏重 pyramiding——会掉进两类结构性问题，而不是单靠调参数能收口：

1. **执行层与 payoff 错配**  
   B 的 edge 多依赖局部结构、订单流与中短 **continuation**，有效假设通常是 **小时～数日**量级。为模拟肥尾而放大持仓尺度、钝化退出，会变成用 **摆动级 hypothesis** 在扛 **macro 方向头寸**：alpha 段落早已结束的部分仍暴露在 naked beta 下，入场逻辑与持仓尺度不再描述同一世界观。

2. **风险症结难以在「单层 B」里根治**  
   若在 B 上持续堆「更大止损 / 更少止盈 / 更激进加码」，PnL 分布并不会自动变成真正的 long convexity；更常见的是 **edge 耗尽后的方向裸露**在 regime 翻转时一次性大回吐。这类回撤往往被误当成「执行力不够」，实则是 **两种完全不同的收益形态（尾部凸性 vs 结构 alpha）绑在同一账本** 导致的；不调开架构，只调阈值容易陷入长期拧巴。

因此单独列出 **A**：用慢离场、试错小风险、再在趋势自证后做凸性加仓，专门承接 **肥尾与长期 regime**；**B** 明确退回 **只吃 trend swing**，用紧失效与较快兑现服务 **alpha 而非 beta**；**C** 再承担「大 regime 门控下的短周期」。**先对齐账本与 payoff，再在各自层上优化**，比在同一层强行兼容两种圣杯更可执行。

## 2. 三层定义（概要）

| 层次 | 本质 | 典型时间尺度 | 主要盈利来源 | 风险提示 |
| ---- | ---- | ------------ | ------------ | -------- |
| **A** | 长期凸性 / 慢 regime 参与 | 周～月（出场更慢） | 超级周期右尾；**不错过大行情** | 低胜率、长盘整、大幅回撤心理准备；单笔试错 risk 要小 |
| **B** | 结构性 trend **swing alpha** | 小时～数日 | **趋势段内的 continuation / expansion**，_timing_ 重于「拿满整条牛」 | 应用紧失效与较快兑现；少用 A 单的 pyramiding 哲学 |
| **C** | 大 regime 下的**微观状态 alpha**（router） | 分钟～若干小时 | chop 均值回归 / 网格或库存轮转；directional 短时冲动；快止盈 | edge 衰减快；不要与 B 的长期方向叙事混账本 |

### 2.1 各司其职：**A 偏 Beta · B 偏 Alpha · C 偏现金流**

一句话分工（可与上表对照，勿与「每层独立 KPI」割裂理解）：

- **A：抓 Beta（附带凸性语义）**  
  核心是 **在长期 regime / 极少数年份里的方向性暴露**：参与「慢变量」主导的上涨或深度趋势，追求的是 **不要错过超级周期**，而不是在日频上用技巧跑赢市场。常与 **现货、慢出场、试错小但总可在自证后加码**绑定；夏普往往不高，PnL **高度右尾依赖**。
- **B：抓 Alpha**  
  核心是 **结构化 swing / continuation**：同源资产上靠 **timing、失效纪律、摆动兑现**多出一块相对裸方向的统计优势。目标不是替 A 扛完整 Beta，而是 **hypothesis 在时多拿、一过就走**。
- **C：高胜率现金流 + 更小的 Alpha**  
  核心是 **短 holding、快进快出、周转与区间震荡/局部冲动的 harvest**：单笔 **edge 更薄**，靠 **频次与胜率**贡献稳定现金垫；本质是 **更小颗粒的 alpha**，与 B 的中频结构性、A 的长期暴露 **三者勿混账本**。

## 3. A 系统（现货、肥尾账本）

### 3.1 定位

- **不是**高精度择时系统，而是 **slow regime participation**（类 CTA 思路）：  
  「不预测超级大牛，但若发生，要确保在车上。」

### 3.2 入场 / 出场（与设计方向对齐）

- **入场（可相对快）**：例如 **2h 级别 EMA1200 突破**，作为粗过滤即可。  
- **出场（刻意更慢）**：同周期均线易过早下车；更合适用 **周线 / 月线**结构或慢滤波（周线 MA、Donchian、月线更高低点破坏等）。

核心：**盈亏的大头依赖「持仓 × 尾部扩张」**，而非 entry 精妙程度。

### 3.3 风险语义：「试错小」≠「总头寸永远小」

- **单次参与 / 单笔试错**：风险占 NAV 很低（示意：与 CTA 常见 0.25%～0.5% risk/try 量级同一思想），能承受连续假突破与 whipsaw。  
- **真趋势逐步确认后**：允许 **凸性加仓（pyramid into strength）**，总名义暴露可变大；这是对「被证明的正确方向」加风险，区别于「越跌越买」的长期 accumulator 框架（若为纯 beta 囤货，则更贴近 DCA，与 convex trend engine  hypotheses 不同，不宜混账本）。

## 4. B 系统（BPC / TPC / ME → trend swing alpha）

### 4.1 转向目标

三大策略账本目标调整为：**抓取 trend swing / structural continuation 的 alpha，而不是承载长期 beta。**

换言之：edge 多来自「趋势存在时的一段」，而非「整条宏观牛熊」。

### 4.2 与原先问题的关系（备忘）

若将 **摆动级** entry（突破回踩、动量扩张等）套上 **超长持有 + 激进加仓**，则 alpha 消亡后仍会长时间裸露在方向风险里，表现为熊市回吐。  

B 层应更重：**较快 invalidation、合理止盈尺度、审慎加仓、关注 continuation 质量与 volatility  regime**，而不是在同一账本上追求整条肥尾。

---

## 5. C 系统（2h 大 regime 下的短周期）

### 5.1 定位

- 在 **较慢层（例如 2h）**对 **chop vs expansion / directional** 做粗分类；
- **chop**：偏网格 / 区间震荡 / 库存式回转；
- **非 chop**：短时动量、突破跟随等，但 **止盈与持有期仍短**，吃的是「局部状态」而非宏观趋势本体。

本质是 **regime classifier + strategy router**：不同市场状态挂载不同 payoff，而不是一招打天下。

### 5.2 资金与 KPI 预期

- 通常作为 **cashflow / Sharpe 增强**，不宜等同于主财富引擎；  
- edge 半衰期短、微观结构变换快，须独立风控与单独的评估面板。

## 6. C 与「高频做市 / 超低延迟头皮」的差别（必读）

本项目中的 **C** 仍是 **较慢时钟上的统计优势 + regime 路由**（分钟～小时量级、依赖 bar/特征与不追求微秒抢先），与业界常说的 **做市商（MM）、高频套利、超低延迟头皮** 不在同一条赛道。

| 维度 | 典型 HF 做市 / 超低延迟 scalp | **本体系的 C** |
| ---- | ----------------------------- | ------------- |
| **时间粒度** | tick、毫秒～秒级挂单与撤单；与队列位置、瞬时双边报价相关 | **分钟～小时**，多依赖 **较慢 regime 判定**后再选子策略 |
| **核心 edge** | 手续费返还、排队优先、极小价差高频周转、latency arb、短期库存中性 | **regime 条件化**的方向/区间震荡统计优势；单笔持有可仍短，但不是「抢一单」博弈 |
| **基础设施** | 托管机房、邻近交易所、定制化协议栈、Fpga/内核旁路、微秒级监控 | **普通研究/实盘链路**即可；不必要与专业 MM 比拼物理延迟 |
| **费率结构** | 高度依赖 **maker 返佣 / 专属费率档**；微薄利润 × 极大成交量 | edge 一般不建立在「每笔必赚一个 tick + rebate」之上；费率重要但 **不是整套系统的生存前提** |
| **风险形态** | 偏 **short gamma / 捡硬币**：长期稳定小赚、极端行情尾部暴亏需严格库存与熔断 | 偏 **短周期 alpha**：黑天鹅与 regime 错位仍有风险，但 **假设与风控对象与 HF MM 不同** |

**结论**：C 可以是「大周期门控下的中短周期策略族」，但 **不应被要求**在高频做市同一张桌上竞争 **速度与费率档位**；若要进入真正 MM/HFT，需要单列项目、单列成本与单列合规与基础设施评估。

## 7. Payoff engineering 备忘

1. **按相关性而不仅是按收益拆分资金**：若 A/B/C 均在「BTC 暴涨」时才赚钱，本质是方向叠加；理想上希望 **最优环境错峰**（长趋势 vs 摆动 vs 横盘/短线）。  
2. **账本隔离**：子账户 / 独立的 max DD、单独的 dashboard，防止「我该不该格局」的认知污染。  
3. **评价指标分列**：例如 A 看尾部与参与度，B 看 swing 夏普与回撤，C 看周转与衰减监控。

## 9. 具体案例（用于防止混层）

以下案例不是唯一答案，而是用于在复盘时快速判断：当前问题是参数问题，还是账本/payoff 错配问题。

### 案例 1：B 用长持有追 fat-tail，牛熊切换回吐

**场景**：B 账本（BPC / TPC / ME）在趋势段盈利后，因「想拿整条牛」而放宽退出、提高加仓激进度。  
**表面现象**：上涨期收益亮眼，切换为震荡/下行后回吐明显，净值出现「慢涨快吐」。  
**本质原因**：用摆动级 alpha 的 entry hypothesis，承载了长期 beta 暴露。alpha 衰减后，剩下的是方向风险。  
**纠偏动作**：

- 恢复 B 的失效纪律（结构破坏、continuation 质量下降时退出）；
- 把「长期拿住」诉求迁移到 A；
- 在复盘中分列统计：B 的 alpha 段收益 vs 后续 beta 暴露段收益。

### 案例 2：A 连续小亏后抓住主趋势右尾

**场景**：A 按慢过滤参与，前期多次假突破，连续出现小额试错亏损。  
**表面现象**：胜率低、体验差，容易产生「策略无效」错觉。  
**本质原因**：A 的目标不是提高短期胜率，而是保持参与权，等待稀疏但高贡献的 regime 扩张。  
**纠偏动作**：

- 固守「单次试错风险小」而非追求一次压满；
- 仅在趋势自证后做逐级加仓；
- 评价 A 时优先看尾部贡献与是否在主升段在场，而非短期命中率。

### 案例 3：C 在 chop 提供现金流，但扩张期降频

**场景**：C 在 2h 门控判断为 chop 时，短持有策略周转良好；进入扩张阶段后，原有区间回转 edge 下降。  
**表面现象**：chop 阶段高胜率，扩张阶段频繁止损。  
**本质原因**：C 的 edge 是局部状态 alpha，非跨 regime 的通吃引擎。  
**纠偏动作**：

- 将 C 的 regime 路由作为硬约束（chop 与 expansion 分开子策略）；
- expansion 阶段降低 C 权重或切换到短动量子策略；
- 现金流 KPI 与回撤 KPI 分开看，避免为了胜率硬扛不适配环境。

### 案例 4：为什么 C 不是 HFT/MM（速度与费率通道不匹配）

**场景**：C 表现稳定后，尝试按高频做市逻辑要求其进一步提频和压缩每笔收益。  
**表面现象**：换手提升，但摩擦成本和滑点占比上升，收益质量恶化。  
**本质原因**：C 的基础设施、费率档、执行延迟前提并非为微秒级队列竞争设计。  
**纠偏动作**：

- 坚持 C 的分钟～小时级统计优势定位，不与 MM 拼 latency；
- 将费率与滑点作为约束而非唯一 edge 来源；
- 若要做真 HFT/MM，单列项目、单列预算和通道建设。
