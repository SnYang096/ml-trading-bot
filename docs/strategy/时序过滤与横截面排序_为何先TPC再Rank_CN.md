# 为何先时序过滤（TPC 思路）再横截面 Rank？—— 与小资金 A 股组合

> 配套：[`label_scan_vs_IC_说明_CN.md`](label_scan_vs_IC_说明_CN.md)（label / IC）、[`方法论_R_and_D流程_CN.md`](方法论_R_and_D流程_CN.md)（R&D 流程）、[`WORKFLOW_整体架构与管线改进计划_CN.md`](WORKFLOW_整体架构与管线改进计划_CN.md)（B 系统分层）。
>
> 本文回答：**能不能跳过 TPC，直接用 alpha 因子 rank 买前几名？** 以及 **资金少时买几只、ETF 还是个股**。

---

## 0. 一句话

- **横截面 rank（alpha 因子）** 回答：在「都能买」的标的里，**谁相对更强**。
- **时序过滤（TPC / regime / gate）** 回答：在**当前市场环境**下，这类策略**该不该出手、该不该做这只**。

两层解决的是**不同问题**。不是「必须先 TPC」，而是 **alpha 因子通常不知道「今天适不适合做趋势回调」**；在不适合的日子里 rank 前 3，仍可能集体亏钱。

---

## 1. 两种做法各自在干什么

```text
路径 A（只 rank，不过滤）
  全 universe（或 ETF 池）
      → 算因子 f_i（动量、价值、质量…）
      → 按 f 排序，买前 K 名
      → 持有 / 调仓

路径 B（先时序过滤，再 rank）
  universe
      → 时序层：regime / gate / prefilter（TPC 思路）
           「今天这类形态做不做？」「这只票在不在允许的数据空间？」
      → 得到「今日可交易子集」U_t（可能为空）
      → 在 U_t 里用 alpha 因子 rank，买前 K 名
```

| 层 | 典型问题 | 典型输出 | 评价指标 |
|---|---|---|---|
| **时序过滤** | 策略与环境是否匹配？ | 做 / 不做；可交易列表 | 空仓期是否少亏、回撤、regime 分段 R |
| **横截面 rank** | 在可做的里面谁更好？ | 排序、权重 | Panel IC、分组 spread、top−bottom |

**crypto TPC live** 常常是路径 B 的极端：标的很少（6 币），**不做横截面 rank**，每只独立一条规则链。  
**A 股小资金** 更适合：**路径 B 的「先过滤、再在子集里 rank」**，且 K 很小（1～3）。

---

## 2. 为什么不直接用 rank alpha 排序？（四条原因）

### 2.1 Alpha 因子多半是「相对」的，不是「绝对该不该做」

动量、估值、质量等因子给出的是：**A 比 B 强**，不是 **今天做多任何股票都有正期望**。

在熊市、极端震荡、全市场 risk-off 时：

- 因子仍可能排出「相对最强」的 3 只；
- 但这 3 只的 **绝对收益** 仍可为负（只是跌得少）。

时序层（EMA 带、chop、波动 regime）试图在 **β 不利** 时 **少做或不做**，这是 alpha 因子本身通常不编码的信息。

**本仓库实证（TPC vol / ema）**：`vol_persistence` 在 recent 段对 `forward_rr` 为 **正 IC**（高 vol → 高 R），但 label 在 bull 上保留 vol gate 仍略提 success；H 保留 bull 侧 vol gate 是为 **DD**，不是为 rank。说明 **「相对排序」与「环境该不该做」可分离**。

### 2.2 策略形态有「适用域」，rank 不会自动尊重适用域

TPC 类逻辑隐含：**趋势 + 回调延续** 只在部分行情有效（离开 EMA 死区、非极端 chop 等）。

若只用通用 alpha（如 20 日动量）排序：

- 选出的可能是「涨最多的」，不一定是「**符合你执行逻辑**的回调买点」；
- 执行层若是规则/形态入场，**因子 top 名与入场规则** 可能不一致（买了强因子，但当天没有合格 pullback 信号）。

**先时序过滤** = 保证池子里是「**你的执行逻辑今天能合法触发**」的标的；**再 rank** = 在合法集合里选最强。

### 2.3 全市场 rank 在小资金下不可执行，过滤是「缩池」的第一步

A 股 **100 股一手**、资金少时：

- 全市场 rank 前 3 常是 **买不起或流动性差** 的票；
- 必须先 **流动性 + 价格 + 指数成分** 缩池，再 rank。

时序过滤与「缩池」可合并为一步：**只保留今天允许交易且买得起的标的**。  
这不是理论洁癖，是 **资金约束**。

### 2.4 两类信号混用会降低可解释性与回测可信度

若 alpha 因子与 TPC 特征高度相关（都在赌趋势），**只 rank** 等于隐式重复加权趋势，且难以分解：

- 亏钱是因为 **环境错了**（该空仓）；
- 还是 **选股错了**（rank 失效）。

**先过滤、再 rank** 便于复盘：空仓期、regime 段、rank 段分开评估（与双段 event_backtest、分桶 IC 一致）。

## 3. 什么时候可以「不用 TPC，直接 rank」？

| 条件 | 可以偏「直接 rank」 |
|---|---|
| 策略本身就是 **纯因子多头 / 指数增强** | 执行 = 按因子权重调仓，无独立「回调入场」规则 |
| 有 **市场级开关**（如 300 均线上下、波动率目标） | 宏观过滤已覆盖「该不该做」，不必每票 TPC |
| Universe 已是 **ETF / 行业 rotation** | 形态简单，rank 的是 β 暴露，不是单票 pullback |
| 回测证明 **加 TPC 过滤后** top 组 spread 几乎不变、但交易次数大减 | 过滤无边际贡献，可简化 |

| 条件 | 建议 **先时序、再 rank** |
|---|---|
| 执行层是 **规则 / 形态 / TPC 类 archetype** | 与因子选股不是同一套逻辑 |
| 小资金 **1～3 只**，要控回撤 | 环境错时 rank 前 3 仍可能全亏 |
| 需要解释 **何时空仓** | 因子 rank 给不出「今天不做」 |

**折中（推荐小资金起步）**：

```text
轻量时序（1～2 条 regime，不必全套 gate）
    + 小 universe（8～15 个 ETF 或 20～30 只成分股）
    + 1～2 个 alpha rank
    → 持仓 1～3
```

## 4. 小资金：买几只？ETF 还是个股？

（与 rank / 过滤并列的 **组合层** 决策，不改变「先过滤再 rank」的逻辑。）

### 4.1 买几只

| 可投资量级（示意） | 建议持仓数 | 说明 |
|---|---:|---|
| 很小 | **1～2** | 一手约束；接受集中风险 |
| 小 | **2～3** | 在「今日可交易子集」里 rank 前 2～3 |
| 再大 | 3～5 或 2 ETF + 2 股 | 再谈分散 |

**不是越多越好**：手续费、一手门槛、研究精力。  
**「买 3」** 指 **小池里 rank 前 3**，不是全市场 5000 只里挑 3。

### 4.2 ETF vs 个股

| | ETF | 个股 |
|---|---|---|
| 小资金 | ✅ 优先：易买满、个股风险小 | 可选：弹性大、要单票研究 |
| 与 rank | 在 **10～15 个行业/宽基 ETF** 间 rank | 在 **20～30 只** 流动性好的成分股间 rank |
| 与 TPC 时序 | 规则可简化（趋势/波动） | 更贴 pullback/chop 语义 |

**起步模板**：8～12 个 ETF universe → 轻量时序过滤 → rank 前 **1～2** → 双周调仓；验证后再加个股。

### 4.3 空头

小账户、无融券：**不做「空 rank 后 3」**；用 **空仓**（时序层全不通过）或 **降仓** 代替对冲。

## 5. 推荐流程（A 股小资金，与 crypto TPC 对照）

```text
┌─────────────────────────────────────────────────────────┐
│ ① Universe：买得起的 ETF/股（流动性 + 一手）              │
└───────────────────────────┬─────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────┐
│ ② 时序过滤（TPC 思路，可只保留 regime + 1 条 gate）        │
│    输出：今日可交易集合 U_t（允许为空 → 空仓）              │
└───────────────────────────┬─────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────┐
│ ③ 横截面 rank：在 U_t 内用 alpha 因子排序                 │
│    Panel IC 评因子；label/backtest 评规则（若混用）        │
└───────────────────────────┬─────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────┐
│ ④ 持仓：前 K 名（K=1～3），控换手                          │
└─────────────────────────────────────────────────────────┘
```

**crypto TPC（当前 live）**：① 固定 6 币 → ② 每币独立规则链 → **无 ③**（不互相比）→ ④ 有信号则做。  
**A 股小资金**：必须加 ③，且 ① 要按资金重做。

## 8. 实操：买了以后明天又出一堆 TPC 信号，怎么办？

核心原则：**信号频率 ≠ 调仓频率**。  
每天（甚至每根 K 线）都可以 **更新「谁合格、谁排第几」**；**不必每天买卖**。

### 8.1 两套频率，不要绑死

| 类型 | 做什么 | 建议频率（小资金） | 是否下单 |
|---|---|---|---|
| **观察 / 评分** | 时序过滤 → 可交易集 U_t；alpha rank 打分 | 日或周（收盘后算即可） | ❌ 不必每笔信号都交易 |
| **调仓** | 买入 / 卖出 / 换仓 | **月** 或 **双周** | ✅ 只有调仓日改持仓 |

```text
每天/每周：算 U_t 和 rank 分数（记账、监控）
      ↓
每月（或每双周）固定一天：对照「调仓规则」决定是否换仓
      ↓
其余日子：默认 HOLD，不因新出现 TPC 信号就追
```

**明天又多出很多 TPC 合格标的** → 只更新表格里的分数和名次；**若今天不是调仓日，继续持有原 1～3 只**，除非触发 §8.3 的 **强制规则**（环境变坏、跌出缓冲带等）。

### 8.2 推荐起步：周线看环境，每月调 rank（可自管因子）

这是 **自己控制特征因子与频率** 的一种清晰拆法，和小资金、低换手匹配：

| 层 | 用什么 | 频率 | 作用 |
|---|---|---|---|
| **环境（时序）** | 宽基或持仓上的 **周线** regime（如指数 vs 长均线、波动分位） | **每周** 收盘看一次 | 决定下周是否允许 **新开仓**；极端差时可 **清仓观望** |
| **选股（横截面）** | 1～2 个 **alpha 因子**（动量、相对强度等） | **每月** 在 U_t 里 rank | 调仓日换 **前 K 名**（K=1～3） |
| **执行** | 不必上全套 live TPC gate | 调仓日下单 | 避免日频 chase |

说明：

- **「周线 TPC」** 不必等于把 crypto 的 2h gate 原样搬来；可以 **故意简化** 为 1～2 条周线规则（趋势是否还在、是否极端震荡）。  
- **「每月 rank」** = 你自己定的 **组合层节奏**，与 WORKFLOW 里 M 级验证、Q 级改 yaml 的思想一致：慢变参数，快变只用于监控。  
- **日频 TPC** 若仍算，适合 **监控/预警**（例如某持仓股日线 chop 恶化），不必等于 **明天就换票**。

### 8.3 调仓日规则（避免「每天都想换」）

在 **固定调仓日**（例如每月第一个交易日，或每月最后一个周五收盘后下单）执行：

**买入 / 持有**

1. 取当日（或前一收盘）的 **可交易集 U_t**（周线环境允许 + 日线/周线过滤通过）。  
2. 在 U_t 内 **rank**，计划持仓 **前 K 名**（K=1～3）。  
3. **已在持仓且仍在前 K 名内** → 继续持有（降低换手）。  
4. **新进入前 K**、且有空位或资金 → 买入；资金不足则 **只买排名第 1** 的。

**卖出 / 换仓（建议加缓冲，别对名次差 1 就换）**

| 规则 | 建议 |
|---|---|
| **名次缓冲带** | 仅当持仓跌出 **前 K+2 名**（例如 K=3，跌出前 5）才卖出换更强标的 |
| **最小持有期** | 买入后至少持有 **10～20 个交易日**（约 2～4 周），除非触发强制卖出 |
| **强制卖出** | ① 周线环境转 **OFF**（整体不做）；② 该股 **停牌/ST**；③ 单票止损（若你设了风控线） |
| **环境 OFF 时** | **不买新的**；已有仓位可 **全部清掉** 或 **只清环境最恶化的**（二选一，回测里定一种并坚持） |

**明天新出现的 TPC 信号**

- **不在调仓日** → 记入观察列表，**下个月** 再参与 rank。  
- **在调仓日** → 自然进入当日 rank，按上面规则买。  
- **不要**：今天买 A，明天信号显示 B 更强就立刻换 B（小资金手续费和心态都扛不住）。

### 8.4 和 crypto TPC live 的差别（为何你能「每月调」）

| | Crypto TPC live | 你的 A 股小组合 |
|---|---|---|
| 信号 | 2h bar 触发可能较频 | 可日频算、**月频交易** |
| 持仓 | 有信号就评估开仓 | **调仓日** 才改组合 |
| 配置变更 | regime 季度 / watchdog 周度 | rank 因子月调、环境规则季调 |

crypto 上 **规则层** 也要慢变（见 [`方法论_R_and_D流程_CN.md`](方法论_R_and_D流程_CN.md)）；组合层再叠 **月调仓**，并不矛盾。

### 8.5 一张「我明天该干嘛」速查

| 情况 | 动作 |
|---|---|
| 今天不是调仓日，多了很多 TPC 合格 | **不动**，只更新监控表 |
| 今天是调仓日，rank 前 K 与当前持仓一致 | **不动** 或微调现金 |
| 今天是调仓日，持仓跌出 K+2 缓冲 | **卖** 弱换强 |
| 周线环境转 OFF | **不买**；持仓按你选的规则清仓或持有不动 |
| 只有 1 只能买满，rank 第 1 变了 | **下月调仓日** 再换（或触发止损提前换） |

## 9. 要不要同时做空对冲？

**小资金、刚起步：不必须；优先「多头 + 空仓」**。空头是进阶选项，有门槛和成本。

### 9.1 三种仓位形态

| 形态 | 做法 | 适合 |
|---|---|---|
| **纯多头** | 只在 rank 前 K 做多；环境 OFF 时 **空仓** | 资金少、无融券、最简单 |
| **多头 + 指数对冲** | 多头前 K 的同时，用 **沪深300/中证500 期货或 ETF** 对冲一部分 β | 有期货/期权账户、想降大盘波动 |
| **多空对冲（买强空弱）** | 多前 K、空后 K | 融券成本高、标的少、**小资金一般不推荐** |

### 9.2 为何小资金常不做「空后 3」

- **融券**：券源、保证金、费率；很多票融不到。  
- **只有 3 只多头** 时，再空 3 只 = **6 个腿**，保证金和跟踪难度上升。  
- **rank 后 3** 往往是「相对最弱」，不等于 **适合做空**（可能已跌很多、波动大）。

### 9.3 实用建议（按优先级）

1. **默认**：环境 OFF → **空仓**；环境 ON → 只做多 rank 前 **1～3**（ETF 起步更稳）。  
2. **若担心大盘跌、账户允许**：考虑 **1 手指数期货** 或 **反向 ETF** 做 **部分对冲**（需单独学保证金与展期，不在此展开）。  
3. **不建议起步**：「多 3 空 3」的股票多空；等纯多头流程跑顺、有稳定记录再考虑。

对冲解决的是 **β（大盘涨跌）**；**时序过滤** 解决的是 **环境是否适合你的策略**。两者可叠加，但 **不能替代**「不该做时仍 rank 出前 3 多头」的问题。

### 9.4 和「每月调仓」怎么配合

- **每月**：只在环境 ON 时，按 rank 调整 **多头** 前 K。  
- **对冲**：若使用，**调整频率可低于 rank**（例如季度检查期货张数），不必每天跟 TPC 信号改对冲腿数。

## 10. 可写进计划书的频率模板（小资金）

```text
Universe：8～12 个行业/宽基 ETF（或 20 只成分股，且每只够买 1 手）

每周五收盘：
  - 周线环境 ON/OFF（1～2 条规则，自管因子）
  - 若 OFF：下周不新开仓；持仓按 §8.3 决定是否清

每日（可选，仅监控）：
  - 更新 U_t 与 rank 分数，不下单

每月第一个交易日（调仓日）：
  - 若环境 ON：在 U_t 内 rank，持有前 K（K=1～3），按缓冲带换仓
  - 若环境 OFF：不买入；持仓处理按你固定的一种规则

对冲：无（起步）→ 有余力再评估指数期货部分对冲

回顾：每季度看一次 rank 因子是否还有 Panel IC / 分段收益
```

## 11. 摘要

1. **不必迷信「必须先 TPC」**；必须清楚 **alpha rank 不回答「今天该不该做」**。  
2. **先时序、再 rank** = 环境匹配 × 相对强弱；**只 rank** = 只有相对强弱，熊市仍可能做多最不差的几只。  
3. **小资金**：小 universe → 过滤 → rank 前 **1～3** → 优先 **ETF** 起步。  
4. **与 crypto TPC 差异**：TPC live 是 per-symbol 规则链；A 股组合层几乎总要 **横截面**，时序层是 **缩池 + 空仓**。  
5. **买了以后明天又多信号**：**更新分数即可，不必天天换**；用 **月（或双周）调仓日 + 名次缓冲带 + 最小持有期**。  
6. **频率**：可 **周线看环境、每月 rank 调仓**；日频 TPC 用于监控，不必用于下单。  
7. **对冲**：小资金 **不必须**；优先 **环境 OFF 时空仓**；进阶再考虑指数期货，慎做「空后 3」。
