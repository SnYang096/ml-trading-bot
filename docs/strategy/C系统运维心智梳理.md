# 本质差异

**TPC（trend swing）**：等待高置信度的入场结构，入场后用结构止损保护，目标是吃一段完整的 swing，持仓可能是几天到几周，单笔盈亏比高，但入场次数少。

**dual_add_trend**：不等待入场结构，趋势确认就开腿，用小间距快速止盈，持仓可能只有几个小时，单笔利润很小，但交易次数多，靠累积来赚钱。

---

# 用一个比喻

TPC 是**狙击手**：等待最佳时机，一枪一个目标，打完就走。

dual_add 是**割草机**：不挑时机，趋势段内持续来回收割小利润，靠数量积累。

---

# 为什么两者可以共存

它们在同一趋势段里赚钱的方式完全不同：

TPC 在等 pullback 确认，这段时间 dual_add 已经在趋势方向上开了好几个止盈腿了。TPC 入场后拿着仓位，dual_add 可能已经止盈重开了三四次。

**时间颗粒度不同，赚钱节奏不同，但都需要趋势。**

---

# 真正需要注意的

你之前说的软协调规则仍然重要。虽然两者赚钱方式不同，但它们**同时需要趋势**，趋势反转时都会亏损。dual_add 靠快速止盈控制单次亏损，TPC 靠结构止损控制。两者同时在场，趋势反转时总暴露仍然偏高。

所以区别是真实的，共存是合理的，但需要控制**同一 symbol 两者合计的总暴露**。

# 重新定义 dual_add 的真实身份

它不是「趋势跟随」，而是**regime 内的方向发现 + 加仓机制**。

起点是对冲（不预判方向），然后让市场告诉你哪边有力，有力的方向不断加仓，弱势方向清掉。这是一种**自适应方向发现**的逻辑。

后来发现 regime 下直接开方向单更好，说明**在强 regime 信号存在的情况下，对冲开局是冗余的**，因为方向已经知道了，不需要用对冲来探测。

---

# 这个演化过程说明了一件重要的事

**dual_add 的真正 edge 不是「对冲」，而是「在正确的 regime 里顺势加仓」。**

对冲开局只是在没有 regime 信号时的一种保守做法，有了 regime 就不需要了。

---

# 和 TPC 的关系重新理解

两者的本质区别不只是频率，而是：

| 维度             | TPC                         | dual_add                  |
| ---------------- | --------------------------- | ------------------------- |
| 入场触发         | 结构确认（pullback 到位）   | regime 确认（趋势强度够） |
| 持仓逻辑         | 等结构失效才出              | 快速止盈，反转就开反向    |
| 对 regime 的依赖 | 中等（regime 是 prefilter） | 强（regime 直接决定方向） |
| 错误的代价       | 单次大止损                  | 多次小亏损累积            |

---

# 不同账户的隔离是对的

两者账户分离不只是频率考虑，更重要的是**风险语义不同**。

TPC 的风险是「结构失效的单次损失」，dual_add 的风险是「regime 判断错误后的多腿累积损失」。混在一个账户里，max drawdown 的来源会变得模糊，不知道是哪个逻辑在亏。

账户分离让你能清楚地看到每个系统在什么 regime 下赚钱、什么 regime 下亏钱。

---

**dual_add 是 regime 驱动的方向加仓机器，TPC 是结构驱动的单笔 swing，两者账户分离、regime 部分重叠但执行语义完全不同，现在的架构是合理的。**

**现在的 dual_add 开反向单，是历史遗留，不是主动设计选择。**
