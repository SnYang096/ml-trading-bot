# B 系统简化和目标调整

---

## 背景

| 问题 | 回答 |
| ---- | ---- |
| B 系统（BPC/TPC/ME）平均持仓时间？ | 不确定 / 混合 |
| A 与 B 资金分配？ | A 为主（A > 60%） |
| 对 B 的核心期望？ | 稳定月度收益，回撤可控 |

---

## 问题：三个入场点与加仓

BPC（pullback 后 continue）、TPC（大趋势后 pullback）、ME（动量）——回测上 TPC 较稳、ME 最不稳。若减少独立入场、其余作加仓，且加仓随价格上涨，如何取舍？

**确认**：B 的设计原则应是提高置信度（胜率 × 盈亏比），而非抓住所有 fattail/swing，或「确保在车上」。

---

## B 系统的设计原则

**宁可少做，但每次做都是高置信度的。**

Swing 收益来自 **胜率 × 盈亏比** 的累积，不是参与次数。多一个低置信度入场会**稀释平均 edge**，不是多一次机会。

---

## 与 A 系统的对比

| | A（spot_accum） | B（swing） |
| --- | --- | --- |
| **核心逻辑** | 确保在车上 | 确保每次上车都值得 |
| **最怕** | 错过牛市没仓位 | 低置信度入场稀释 edge |
| **对「错过」** | 不能接受 | 必须接受 |

**「确保在车上」是 A 的职责，不是 B 的。** B 若也追求「在车上」，易出现多信号、止损纠结、identity 模糊。A 已兜底，B 只做最确定的一类即可。

---

## 总结

1. B 不追 fattail（难学习、难按层调规则）。
2. B 做 trend swing，因子约 20 bar 有效，数据按牛熊划分。
3. B 的目标是提高 **胜率 × 盈亏比**，而非单纯 total R。
