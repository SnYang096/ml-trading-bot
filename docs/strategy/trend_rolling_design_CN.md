## 1. 策略哲学

在深熊底部精准抄底，用可控杠杆放大趋势收益，在主力成本区（VWAP1200）升级杠杆，
达到目标利润后阶梯减仓。不做短线、不止损割肉、不频繁交易。

**核心理念**：4年只做3次交易，每次赚5-15倍。

## 2. 入场条件 (冠军三重过滤)

```
AND:
  ├── weekly_ema_200_position < -0.05    # 价格低于周线EMA200至少5%（深熊确认）
  ├── EMA1200 金叉 VWAP1200             # 结构拐点确认（EMA1200上穿VWAP1200）
  └── roc_5 > 0 AND roc_20 > 0           # 短中期动量已转正（趋势已启动）
```

**为什么是三重？** Phase A 扫描了24种入场变体，结论：
- 单用深度：0.74x（抄底在半山腰）
- 单用交叉：2.17x（假突破太多）
- 单用动量：1.14x（深熊中频繁假信号）
- 🏆 三者组合：6.58x（精准抄底+拐点确认+趋势验证）

**BTC为什么被排除？** BTC 在回测期内未同时满足三个条件（深熊深度不够-5%），
冠军入场自动过滤了不成熟的入场时机。

## 3. 杠杆管理

### 3.1 初始杠杆：2x

入场即开2x。为什么不是1x？
- 深熊底部入场后，价格反弹空间远大于下跌空间
- 历史回测：2x入场从未爆仓（最低权益距爆仓线350-1200倍）

### 3.2 滚仓升级：+1x 渐进式

```
触发条件 (全部满足):
  ├── 价格从峰值回撤 ≥ 20%
  ├── 当前浮盈 > 0（确保不是在亏损中加杠杆）
  ├── 杠杆 < max_leverage（还有提升空间）
  ├── 仓位notional < $500k cap（未达仓位上限）
  └── 价格在 VWAP1200 ±5% 范围内（主力成本区支撑）
```

**每次roll只加1x**：2x→3x→4x→5x，不跳级。

### 3.3 为什么渐进(+1x)比跳跃(直接到max)好？

回测对比（2020-2026，$10k起始）：

| 策略                      | 总收益    | 平均回报  | 最差DD | 原因                 |
| ------------------------- | --------- | --------- | ------ | -------------------- |
| R2 (2x→3x)                | $322k     | 10.7x     | -37.2% | 保守，收益受限       |
| **R5-渐进 (2x→3x→4x→5x)** | **$731k** | **24.4x** | -55.6% | ✅ 最优               |
| R5-跳跃 (2x→5x)           | $423k     | 14.1x     | -55.6% | 过早满杠杆           |
| R5-直接 (5x入场)          | $258k     | 8.6x      | -49.5% | 最差，入场即最大风险 |

**为什么渐进更好？**
1. **用利润加杠杆，不是用本金**：2x→3x时已有浮盈，加的1x风险来自利润
2. **回撤更可控**：逐步提升，每次roll前已有缓冲垫
3. **避免过早满杠杆**：跳跃式在第一次回撤时就暴露最大风险
4. **notional cap自然分层**：equity越大，cap限制杠杆越低（BTC最高4x，BNB最高3.7x）

**实际滚仓轨迹**：
| 币种 | 轨迹        | 限制原因                           |
| ---- | ----------- | ---------------------------------- |
| BTC  | 2x→3x→4x    | equity=$125k, 5x=$625k > $500k cap |
| SOL  | 2x→3x→4x→5x | equity较低，未触及cap              |
| BNB  | 2x→3x→3.7x  | equity=$135k, 4x=$540k > $500k cap |

### 3.4 仓位上限 (Notional Cap)

```
max_position_notional = $500,000
实际杠杆 = min(目标杠杆, $500k / 当前equity)
```

**为什么设$500k？**
- 防止单币仓位过大（流动性冲击、滑点）
- equity越大，最大可用杠杆越低（自然降杠杆）
- 确保高equity时不会因5x杠杆导致仓位过大

### 3.5 为什么是VWAP1200？

| 滚仓条件         | 总收益    | 最差回撤   | SOL风险减仓 |
| ---------------- | --------- | ---------- | ----------- |
| 无条件(基线)     | $263k     | -53.7%     | 1次         |
| EMA1200附近      | $269k     | -53.2%     | —           |
| **VWAP1200附近** | **$259k** | **-45.0%** | **0次** ✅   |
| EMA200附近       | $270k     | -53.7%     | —           |

VWAP1200 = 1200根bar的市场平均成交价 = 主力成本区。
在VWAP处滚仓意味着在"聪明钱"的成本区加杠杆，不是盲目追。

## 4. 卖出策略 (权益阶梯)

### 4.1 触发条件

```
权益 / 初始本金 ≥ 5.0x  → 开始阶梯卖出
```

### 4.2 上涨过滤（v4修复）

**原Bug**：阶梯卖出仅看权益倍数，不区分涨跌 → 下跌时也触发卖出。
**修复**：双重过滤，只在上涨时卖出：

```
AND:
  ├── 价格 > EMA1200                    # 趋势向上
  └── 离峰值回撤 < 15%                  # 近高点（非深度回调）
```

### 4.3 底仓保护（v4新增）

**原Bug**：阶梯卖出可卖到100%清仓 → 错过第二波上涨。
**修复**：保留至少20%底仓，永不卖完：

```
min_residual = 初始仓位 × 20%           # 永不卖出的底仓
if pos_qty - sell_qty < min_residual:
    sell_qty = max(0, pos_qty - min_residual)  # 限制卖出量
```

**效果**：BTC从37x提升到97x，BNB从68x提升到140x — 底仓参与了第二波上涨。

### 4.4 卖出速度

```
speed = min(5.0, (equity_multiple / 5.0) ^ 1.0)  # 线性加速
sell_fraction = min(max_frac, 0.08 × speed)       # 每次卖剩余仓位的8%×speed

max_frac:
  权益 < 15x  → 0.99（永不全部清仓，保留火种）
  权益 ≥ 15x  → 1.00（允许全平）
```

### 4.3 卖出节奏

| 权益倍数 | speed | 每次卖出% | 效果     |
| -------- | ----- | --------- | -------- |
| 5x       | 1.0   | 8%        | 缓慢减仓 |
| 7.5x     | 1.5   | 12%       | 加速     |
| 10x      | 2.0   | 16%       | 快速     |
| 15x      | 3.0   | 24%       | 可全平   |

**为什么是5x触发？** 参考 spot_accum_simple 逻辑。更低的触发（如3x）会导致过早止盈（R6: $209k vs R2: $259k）。

## 6. 重入机制

### 6.1 标准重入
清仓后允许重新入场，冷却期 60 天（720 bar），防止频繁进出。

### 6.2 第二波重入（v4新增）

利润出场后，无需等待完整入场信号，满足以下条件即可重新上车：

```
AND:
  ├── 上次出场盈利 (last_exit_profit = True)
  ├── 冷却期 ≥ 4天 (48 bars × 2h)
  ├── 价格从峰值回撤 ≥ 12% (c/peak_px < 0.88)
  └── 短期动量确认 (roc5 > 0)
```

**设计思路**：卖在高点后等回撤再上车，不追求抄底，只求参与下一波趋势。

## 6.5 Regime 切换与全周期退出

### 6.5.1 问题：2026年BTC熊市为什么不卖？

BTC 2026年数据：
| 日期    | 价格  | WEMA200 | wema_pos  | EMA1200 | 趋势   |
| ------- | ----- | ------- | --------- | ------- | ------ |
| 2025-10 | $114k | $62k    | +45%      | $111k   | 🟢 牛市 |
| 2025-12 | $87k  | $66k    | +24%      | $104k   | 🟡 转弱 |
| 2026-01 | $87k  | $67k    | +23%      | $97k    | 🟡 转弱 |
| 2026-03 | $67k  | $68k    | **-1.7%** | $83k    | 🔴 熊市 |
| 2026-05 | $76k  | $68k    | +10.6%    | $75k    | 🟡 反弹 |

**当前策略不卖出的原因**：
- 阶梯卖出需要 `price > EMA1200`（上涨过滤），2026年BTC价格低于EMA1200 → 不触发
- 20%底仓永不卖出 → 即使部分卖出，底仓仍持有
- 没有"regime切换→全仓退出"机制

### 6.5.2 完整交易生命周期

```
┌─────────────────────────────────────────────────────────────────┐
│                    完整交易流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│     价格回撤20% + 浮盈 + VWAP附近 → +1x杠杆                      │
│     2x → 3x → 4x → 5x (渐进式)                                  │

│  ③ 阶梯卖出 (Ladder Sell)                                        │

│     保留20%底仓，永不卖完                                         │

│  ④ Regime退出 (Regime Exit) ← 当前缺失！                         │
│     周线EMA200斜率转负 + 价格跌破WEMA → 全仓退出                   │
│     ↓                                                           │
│  ⑤ 观望等待 (Wait)                                               │

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.5.3 各阶段详解

#### ① 入场 (Entry)
- **条件**: wema_pos < -5% + EMA1200金叉VWAP + ROC5>0 + ROC20>0
- **动作**: 以2x杠杆开仓
- **资金**: 100%可用资金
- **风控**: 无（深熊入场本身就是风控）

#### ② 持仓+滚仓 (Hold + Roll)
- **条件**: 价格peak回撤≥20% + 浮盈>0 + VWAP±5% + 杠杆<max
- **动作**: +1x杠杆，重新计算仓位
- **限制**: notional cap $500k
- **轨迹**: 2x→3x→4x→5x

#### ③ 阶梯卖出 (Ladder Sell)
- **条件**: 权益≥5x + 价格>EMA1200 + 离峰值<15%
- **动作**: 卖出5-25%仓位（速度随权益增加）
- **保护**: 保留20%底仓
- **目的**: 锁定利润，降低风险暴露

#### ④ Regime退出 (Regime Exit) — 需要新增
- **信号**: 周线EMA200斜率由正转负 + 价格跌破WEMA200
- **动作**: 全仓退出，不再持有任何仓位
- **目的**: 在熊市确认前离场，保住大部分利润

**BTC 2026年的Regime信号**：
```
周线EMA200斜率(4周):
  2025-10: +3.53%  ← 强牛
  2026-01: +1.31%  ← 动量衰减
  2026-03: -0.02%  ← 转负！卖出信号
```
如果策略有regime退出，应该在2026年3月（斜率转负+价格跌破WEMA）全仓退出，
此时价格~$67k，相比2025年10月高点$114k已回撤41%，但仍保住大部分利润。

#### ⑤ 观望等待 (Wait)
- **条件**: 无持仓
- **动作**: 等待下一次入场信号
- **耐心**: 通常需要等待1-2年

#### ⑥ 重新入场 (Re-entry)
- **条件**: 再次满足入场三重过滤
- **动作**: 回到①，开始新周期

### 6.5.4 Risk Reduce vs Risk On

| 概念            | 触发条件              | 动作               | 冷却期         |
| --------------- | --------------------- | ------------------ | -------------- |
| **Risk Reduce** | 权益从峰值DD≥35%      | 减半仓             | 28天(336 bars) |
| **Risk On**     | 冷却期结束 + 价格恢复 | 继续持有（不加仓） | —              |

**Risk Reduce的目的**：在剧烈下跌时保护本金，不是止损。
- 触发后仓位减半，降低杠杆暴露
- 28天冷却防止连续触发（千刀万剐）
- 冷却期结束后，如果价格恢复，继续持有剩余仓位
- 如果价格继续下跌，不再触发（已减半）

**Risk On的含义**：冷却期结束后恢复正常状态。
- 不是主动加仓，而是"不再受风控限制"
- 允许后续的roll操作继续进行
- 如果满足roll条件，可以继续+1x

## 8. 爆仓风险评估

### 8.1 理论最坏情况

2x杠杆下，价格需瞬间跌50%才爆仓（单bar内）。
加密市场历史上 BTC 最大单日跌幅 ~40%（2020年3月），ETH ~55%（2021年5月）。
但 R2 入场在深熊底部，已释放大部分下行风险。

### 8.2 历史回测最低权益

| 币种 | 入场$10k | 最低权益  | 距爆仓($100) |
| ---- | -------- | --------- | ------------ |
| ETH  | $10,000  | ~$42,469  | 424x         |
| SOL  | $10,000  | ~$35,709  | 357x         |
| BNB  | $10,000  | ~$119,289 | 1192x        |

**结论：几乎不可能爆仓。** 五层防护 + 深熊入场 + 风控冷却期 = 多层安全网。
