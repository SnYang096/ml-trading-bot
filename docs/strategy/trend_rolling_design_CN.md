# 趋势滚仓 (Trend Rolling) 完整设计文档

> 版本: R5-v4 | 日期: 2026-06-17 | 状态: ✅ 实验通过

---

## 1. 策略哲学

在深熊底部精准抄底，用可控杠杆放大趋势收益，在主力成本区（VWAP1200）升级杠杆，
达到目标利润后阶梯减仓。不做短线、不止损割肉、不频繁交易。

**核心理念**：4年只做3次交易，每次赚5-15倍。

---

## 2. 入场条件 (冠军三重过滤)

```
AND:
  ├── weekly_ema_200_position < -0.05    # 价格低于周线EMA200至少5%（深熊确认）
  ├── EMA1200 金叉 VWAP1200             # 结构拐点确认（EMA1200上穿VWAP1200）
  └── roc_5 > 0 AND roc_20 > 0           # 短中期动量已转正（趋势已启动）
```

**为什么是三重？** Phase A 扫描了24种入场变体，结论：
- 单用深度：0.74x（抄底在半山腰）
- 单用交叉：2.17x（假突破太多）
- 单用动量：1.14x（深熊中频繁假信号）
- 🏆 三者组合：6.58x（精准抄底+拐点确认+趋势验证）

**BTC为什么被排除？** BTC 在回测期内未同时满足三个条件（深熊深度不够-5%），
冠军入场自动过滤了不成熟的入场时机。

## 3. 杠杆管理

### 3.1 初始杠杆：2x

入场即开2x。为什么不是1x？
- 深熊底部入场后，价格反弹空间远大于下跌空间
- 历史回测：2x入场从未爆仓（最低权益距爆仓线350-1200倍）

### 3.2 滚仓升级：2x → 3x

```
AND:
  ├── 价格从峰值回撤 ≥ 20%
  ├── 当前浮盈 > 0（确保不是在亏损中加杠杆）
  └── 价格在 VWAP1200 ±5% 范围内（主力成本区支撑）
```

**为什么是VWAP1200？** 实验对比：
| 滚仓条件         | 总收益    | 最差回撤   | SOL风险减仓 |
| ---------------- | --------- | ---------- | ----------- |
| 无条件(基线)     | $263k     | -53.7%     | 1次         |
| EMA1200附近      | $269k     | -53.2%     | —           |
| **VWAP1200附近** | **$259k** | **-45.0%** | **0次** ✅   |
| EMA200附近       | $270k     | -53.7%     | —           |

VWAP1200 = 1200根bar的市场平均成交价 = 主力成本区。
在VWAP处滚仓意味着在"聪明钱"的成本区加杠杆，不是盲目追。

### 4.3 杠杆上限：3x (R2) / 5x (R5)

R2: max_leverage=3.0 (保守)
R5: max_leverage=5.0 + 仓位上限$500k + 风控DD=35%

## 4. 卖出策略 (权益阶梯)

### 4.1 触发条件

```
权益 / 初始本金 ≥ 5.0x  → 开始阶梯卖出
```

### 4.2 上涨过滤（v4修复）

**原Bug**：阶梯卖出仅看权益倍数，不区分涨跌 → 下跌时也触发卖出。
**修复**：双重过滤，只在上涨时卖出：

```
AND:
  ├── 价格 > EMA1200                    # 趋势向上
  └── 离峰值回撤 < 15%                  # 近高点（非深度回调）
```

### 4.3 底仓保护（v4新增）

**原Bug**：阶梯卖出可卖到100%清仓 → 错过第二波上涨。
**修复**：保留至少20%底仓，永不卖完：

```
min_residual = 初始仓位 × 20%           # 永不卖出的底仓
if pos_qty - sell_qty < min_residual:
    sell_qty = max(0, pos_qty - min_residual)  # 限制卖出量
```

**效果**：BTC从37x提升到97x，BNB从68x提升到140x — 底仓参与了第二波上涨。

### 4.4 卖出速度

```
speed = min(5.0, (equity_multiple / 5.0) ^ 1.0)  # 线性加速
sell_fraction = min(max_frac, 0.08 × speed)       # 每次卖剩余仓位的8%×speed

max_frac:
  权益 < 15x  → 0.99（永不全部清仓，保留火种）
  权益 ≥ 15x  → 1.00（允许全平）
```

### 4.3 卖出节奏

| 权益倍数 | speed | 每次卖出% | 效果     |
| -------- | ----- | --------- | -------- |
| 5x       | 1.0   | 8%        | 缓慢减仓 |
| 7.5x     | 1.5   | 12%       | 加速     |
| 10x      | 2.0   | 16%       | 快速     |
| 15x      | 3.0   | 24%       | 可全平   |

**为什么是5x触发？** 参考 spot_accum_simple 逻辑。更低的触发（如3x）会导致过早止盈（R6: $209k vs R2: $259k）。

## 5. 风险管理 (五层防护)

| 层级          | 机制                       | 参数                            |
| ------------- | -------------------------- | ------------------------------- |
| 🛡️ L1 入场     | 深熊5%以下才入场           | weekly_ema_200_position < -0.05 |
| 🛡️ L2 杠杆     | 永不超过3x                 | max_leverage = 3.0              |
| 🛡️ L3 VWAP     | 只在主力成本区滚仓         | near_vwap1200 ±5%               |
| 🛡️ L4 权益止损 | 权益从峰值回撤50% → 减半仓 | eq_dd_stop = 0.50               |
| 🛡️ L5 价格止损 | 价格从入场跌60% → 减半仓   | px_dd_stop = 0.60               |

**冷却期**：L4/L5 触发后有 28 天（336 bar）冷却，防止千刀万剐。

## 6. 重入机制

### 6.1 标准重入
清仓后允许重新入场，冷却期 60 天（720 bar），防止频繁进出。

### 6.2 第二波重入（v4新增）

利润出场后，无需等待完整入场信号，满足以下条件即可重新上车：

```
AND:
  ├── 上次出场盈利 (last_exit_profit = True)
  ├── 冷却期 ≥ 4天 (48 bars × 2h)
  ├── 价格从峰值回撤 ≥ 12% (c/peak_px < 0.88)
  └── 短期动量确认 (roc5 > 0)
```

**设计思路**：卖在高点后等回撤再上车，不追求抄底，只求参与下一波趋势。

## 8. 爆仓风险评估

### 8.1 理论最坏情况

2x杠杆下，价格需瞬间跌50%才爆仓（单bar内）。
加密市场历史上 BTC 最大单日跌幅 ~40%（2020年3月），ETH ~55%（2021年5月）。
但 R2 入场在深熊底部，已释放大部分下行风险。

### 8.2 历史回测最低权益

| 币种 | 入场$10k | 最低权益  | 距爆仓($100) |
| ---- | -------- | --------- | ------------ |
| ETH  | $10,000  | ~$42,469  | 424x         |
| SOL  | $10,000  | ~$35,709  | 357x         |
| BNB  | $10,000  | ~$119,289 | 1192x        |

**结论：几乎不可能爆仓。** 五层防护 + 深熊入场 + 风控冷却期 = 多层安全网。
