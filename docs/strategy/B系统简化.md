# B 系统的设计原则

**宁可少做，但每次做都是高置信度的。**

---

这背后有一个数学逻辑：

Swing 系统的收益来自**胜率 × 盈亏比的累积**，不是来自「参与次数」。

多一个低置信度入场，不是多一次赚钱机会，而是**稀释了你的平均 edge**。

---

# 和 A 系统的对比

这也是为什么 A 和 B 的设计哲学完全相反：

|                      | A（spot_accum）  | B（swing）                |
| -------------------- | ---------------- | ------------------------- |
| **核心逻辑**         | 确保在车上       | 确保每次上车都值得        |
| **最怕的错误**       | 错过牛市没有仓位 | 低置信度入场拉低整体 edge |
| **对「错过」的态度** | 不能接受         | 必须接受                  |

---

**「确保在车上」是 A 系统的职责，不是 B 的。**

B 系统如果也试图「确保在车上」，就会变成你现在的状态：三个入场信号、止损纠结、系统 identity 模糊。

你已经有 A 在兜底了，B 可以放心地只做最确定的那一类。
