
`allow_new_segment` 中 `sym in active → True` 是有意的：同 symbol 补档/续开不占新 slot；跨策略互斥由 daemon 在 place 层 drop。

## 3. Sizing：`40%` 与 `equity_usdt` 的关系

公式（`multileg_sizing.py`）：

```text
unit_notional / equity = segment_dd_target / (max_loss × capital_units)
```

Chop（10kU，`segment_dd_target=0.072`）：

```text
unit_notional = 10000 × 0.072 / (0.03 × 6) = 4000 USDT/档
比例 = 4000/10000 = 40% equity/档
```

要点：

- **`equity_usdt` 只缩放绝对 USDT**，不改变「每档占 equity 的百分比」。
- **`40%` 是名义 notional**，不是「账户亏 40%」。
- 单段**理论**最坏亏损预算 = `segment_dd_target × equity` = **720 USDT（7.2%）**（6 格/4 腿全满且每格都打满 `max_loss`）。
- **实际** chop risk stop 约 **-120 USDT（1.2%）**；见仓位分析文档 §3–4。

Trend 单腿 unit ≈ **90% equity**（9000/10000），满 4 腿 gross **3.6x**；单 symbol 上 chop/trend **互斥**，不会叠加。

## 4. Gross cap 与网格语义（讨论结论 → 方案 A）

### 4.1 演变：1.30 → 1.60 → 2.70

| 阶段 | `max_gross_notional_pct` | 支持 |
| --- | --- | --- |
| 原始（23313f73） | 1.40 | chop 3 档（120%），trend 无法用 |
| 讨论 1 | 1.60 | chop 3+1 flip（160%），trend 1 腿 |
| **方案 A（当前）** | **2.70** | chop 4 档 + **trend 3 腿**（270%） |

### 4.2 多 symbol 并发（2.70 下的现实）

- `max_concurrent_multi_leg_symbols: 6` = 最多 6 个 symbol 各有一段 active segment。
- `max_gross_notional_pct: 2.70` = **全账户合计** gross ≤ 270% equity（10k → 27k USDT）。
- **单 symbol trend 3 腿（27k）就占满整个 pool**；6 symbol 不可能各自打满。
- 2.70 本质是「单 symbol 重度暴露」档位，不是 6 symbol 平分。
- 实盘多 symbol 时，每个 symbol 只能轻量持有（chop 1–2 档级别）。

### 4.3 杠杆感从哪来

名义大于本金即 U 本位合约的 **gross 杠杆暴露**（非 spot 1:1）：

| 策略 | 单 symbol 满段 gross（实际） | 对应 cap |
| --- | --- | --- |
| Chop 4 档（3+1 flip） | 1.6x | max_gross 2.70 |
| Trend 3 腿 | 2.7x | max_gross 2.70 |
| Trend 4 腿 | 3.6x | ❌ > 2.70 + 被 max_gross_leverage 3.0 拦 |

> `max_gross_leverage: 3.0` 是终极天花板（gross ≤ 30,000），2.70 的 pct cap 在更早位置拦截。

Constitution 另有：

```yaml
account_risk_limits:
  max_gross_leverage: 3.0
  max_projected_initial_margin_pct: 0.65
max_drawdown_pct: 0.20  # kill switch
```

**6 symbol 同时 active 的风险**不是线性 `6 × 7.2%`：governor + regime + risk stop 会先限制实际暴露。
