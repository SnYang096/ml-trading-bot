# 代码复用重构实施计划

## 目标
让 `model-comparison` 复用 `strategy-feature-compare` 的代码，确保 breakeven 功能不丢失。

## 当前状态

### model-comparison 的评估逻辑
- `evaluate_rule_based`: 使用 `compute_rr_label` 和 `compute_rr_label_with_details` 计算 breakeven rate
- `evaluate_ml_model`: 使用 `compute_rr_label` 和 `compute_rr_label_with_details` 计算 breakeven rate
- `evaluate_ml_vol_model`: 使用 `compute_adaptive_rr_label_with_predicted_vol` 和详细信息计算 breakeven rate

### strategy-feature-compare 的回测逻辑
- 使用 `run_vectorbt_backtest` 进行完整回测
- `run_vectorbt_backtest` 调用 `simulate_rr_exits` 处理 RR exits
- `simulate_rr_exits` 支持 `use_breakeven_stop` 参数

## 问题
`run_vectorbt_backtest` 在调用 `simulate_rr_exits` 时没有传递 `use_breakeven_stop` 参数。

## 解决方案

### 步骤1：修复 run_vectorbt_backtest 支持 breakeven ✅
- 在 `run_vectorbt_backtest` 中从 `rr_params` 读取 `use_breakeven_stop`
- 传递给 `simulate_rr_exits`

### 步骤2：在 backtest.yaml 中添加 use_breakeven_stop 配置
```yaml
rr:
  use_breakeven_stop: true  # 启用保本止损
```

### 步骤3：重构 model-comparison 使用策略配置（分阶段）

#### 阶段1：最小改动（保留现有评估函数，但使用配置）
- 使用 `run_feature_pipeline` 替代直接调用 feature_loader
- 使用配置中的标签生成函数
- 保留 `evaluate_rule_based`、`evaluate_ml_model`、`evaluate_ml_vol_model` 但从配置读取参数

#### 阶段2：完全迁移（使用 run_vectorbt_backtest）
- 使用 `run_vectorbt_backtest` 替代 `evaluate_rule_based`、`evaluate_ml_model`
- 确保 breakeven rate 可以从回测结果中计算

## 实施顺序

1. ✅ 修复 `run_vectorbt_backtest` 支持 breakeven
2. ⏳ 在 backtest.yaml 中添加 `use_breakeven_stop: true`
3. ⏳ 重构 `model-comparison` 使用 `run_feature_pipeline`
4. ⏳ 重构 `model-comparison` 使用配置中的标签生成函数
5. ⏳ 重构 `model-comparison` 使用配置中的训练函数
6. ⏳ 重构 `model-comparison` 使用 `run_vectorbt_backtest`（可选，如果 breakeven rate 可以从回测结果计算）

## 注意事项

1. **Breakeven Rate 计算**：
   - 当前 `evaluate_*` 函数使用 `compute_rr_label_with_details` 计算 breakeven rate
   - `run_vectorbt_backtest` 返回的是 vectorbt 的回测结果，不包含 breakeven rate
   - 需要确保可以从回测结果中计算 breakeven rate，或者保留一个辅助函数

2. **向后兼容**：
   - 如果 `use_breakeven_stop` 未配置，默认为 `False`（保持向后兼容）

3. **测试**：
   - 运行 `model-comparison` 确保 breakeven rate 仍然正确计算
   - 运行 `strategy-feature-compare` 确保功能不受影响
