# 文件权限问题说明

## 问题：为什么测试文件的组是 root？

### 根本原因

**当前 shell 是 root 用户**，导致：

1. **通过 `write` tool 创建的文件**：
   - 如果 `write` tool 在 root shell 中运行
   - 创建的文件所有者就是 `root:root`

2. **通过 Docker 命令创建的文件**：
   - Makefile 中的 Docker 命令使用 `--user $(HOST_UID):$(HOST_GID)`
   - 当在 root shell 中运行 `make` 时，`HOST_UID=0, HOST_GID=0`
   - Docker 容器以 root 用户运行
   - 容器中创建的文件就是 `root:root`

### 验证

```bash
# 检查当前用户
whoami  # root
id -u    # 0
id -g    # 0

# Makefile 中的 Docker 命令
DOCKER_RUN_NO_TTY := docker run --rm \
    --user $(HOST_UID):$(HOST_GID) \  # 即 --user 0:0 (root)
    ...
```

### 解决方案

#### 方案 1：以 yin 用户运行命令（推荐）

```bash
# 切换到 yin 用户
su - yin
cd /home/yin/trading/ml_trading_bot

# 然后运行 make 命令
make test-volume-profile-volatility-features
```

#### 方案 2：修复已创建文件的权限

```bash
# 修复文件所有者和组
sudo chown -R yin:yin tests/
chmod 664 tests/test_*.py
```

#### 方案 3：在 Makefile 中强制使用 yin 用户

修改 Makefile，强制使用 yin 用户：

```makefile
# 强制使用 yin 用户（如果存在）
YIN_UID ?= $(shell id -u yin 2>/dev/null || echo 1000)
YIN_GID ?= $(shell id -g yin 2>/dev/null || echo 1000)

DOCKER_RUN_NO_TTY := docker run --rm \
    --user $(YIN_UID):$(YIN_GID) \  # 强制使用 yin 用户
    ...
```

### 预防措施

1. **以 yin 用户运行开发命令**（推荐）
2. **使用 `make fix-all-ownership`** 定期修复权限
3. **在 CI/CD 中设置正确的用户权限**

### 总结

文件组是 root 的原因是：
- **当前 shell 是 root 用户**
- **所有创建的文件（无论是通过 write tool 还是 Docker）都以 root 用户创建**
- **解决方案**：以 yin 用户运行命令，或修复已创建文件的权限
