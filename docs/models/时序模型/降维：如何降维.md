# 自动 IC 筛选 + PCA 降维 pipeline
你问到了量化因子工程的“圣杯”问题：
如何在降维的同时，既保留预测能力，又不失可解释性？

这正是顶级量化基金（如 Two Sigma、AQR、文艺复兴）的核心竞争力之一。

我们来系统性地拆解一个最先进的、工业级的“自动 IC 筛选 + PCA 降维 + 可解释性保留”全流程。

🎯 核心目标

目标 要求
------ ------
✅ 降维 从 400 个因子 → 50 个核心因子
✅ 去噪 剔除伪相关、过拟合因子
✅ 保留可解释性 能说清楚“每个主成分代表什么”
✅ 实盘可用 模型可固化、可更新、低延迟

🚀 推荐方案：三阶段融合框架

text
[阶段1] 自动 IC 筛选 → 剔除垃圾因子
↓
[阶段2] 聚类 + 代表因子选择 → 去冗余
↓
[阶段3] 可解释 PCA（或 Sparse PCA） → 生成语义主成分

我们一步步详解。

🧱 阶段1：自动 IC 筛选（剔除无效因子）
目标：从 400 个因子中，筛出“真正有效”的候选池。
✅ 方法：滚动 Rank IC + IR 检验

python
def compute_rolling_ic(df, factor_cols, target_col='future_return', window=120):
ic_df = pd.DataFrame()
for col in factor_cols:
ic = df[col].rolling(window).corr(df[target_col], method='spearman')
ic_df[col] = ic
return ic_df
计算
ic_history = compute_rolling_ic(df, factors, 'fwd_rtn_5d')
筛选条件
valid_factors = []
for col in ic_history.columns:
mean_ic = ic_history[col].mean()
std_ic = ic_history[col].std()
ir = mean_ic / std_ic

# 三重过滤
if (abs(mean_ic) > 0.06 and # IC 绝对值够大
abs(skew(ic_history[col])) < 2 and # IC 分布不要太偏
ir > 0.8): # IR 足够高
valid_factors.append(col)

📌 筛选标准（工业级）：
指标 阈值 说明
------ ------ ------
\ Mean IC\ > 0.06 预测能力显著
IR > 0.8 稳定性好
IC 波动率 < 0.03 避免阶段性有效
IC 偏度 < 2 避免极端值影响

👉 输出：valid_factors（约 100~150 个）

🧱 阶段2：聚类去冗余（去除共线性）
问题：
很多因子高度相关（如 RSI、KDJ、Stochastic 都是动量类）；
PCA 会把它们“平均”成一个成分，但无法解释。
✅ 方法：层次聚类（Hierarchical Clustering） + 代表因子选择
步骤：

1. 计算因子相关性矩阵
python
corr_matrix = df[valid_factors].corr(method='spearman')

2. 层次聚类分组
python
from sklearn.cluster import AgglomerativeClustering

cluster = AgglomerativeClustering(
n_clusters=None,
distance_threshold=0.5, # 相关系数 > 0.5 的归为一类
linkage='average'
)
clusters = cluster.fit_predict(1 - corr_matrix.abs()) # 距离 = 1 - 相关

3. 每类选一个“代表因子”
python
representatives = []
for i in range(clusters.max() + 1):
group = [f for f, c in zip(valid_factors, clusters) if c == i]

# 选 IC 最高、或 IR 最高的作为代表
best_factor = max(group, key=lambda f: ic_history[f].mean() / ic_history[f].std())
representatives.append(best_factor)

📌 常见聚类结果示例：

聚类 代表因子 经济意义
------ ---------- ----------
0 RSI_14 动量强度
1 Hurst_Hurst 长期记忆性
2 Wavelet_Energy_Band3 中频波动能量
3 BB_Width 波动率水平

👉 输出：representatives（约 50~80 个），每个都有明确含义。

🧱 阶段3：可解释 PCA（生成语义主成分）

现在你有 60 个“去冗余”的因子，但还想进一步降维。
❌ 传统 PCA 的问题：
主成分是原始因子的线性组合，但权重没有稀疏性；
你无法说清“PC1 到底代表什么”。
✅ 先进方法1：Sparse PCA（稀疏主成分分析）
让主成分只依赖少数几个因子，提升可解释性。

python
from sklearn.decomposition import SparsePCA

spca = SparsePCA(n_components=20, alpha=0.1, ridge_alpha=0.01)
X_spca = spca.fit_transform(df[representatives])
查看每个主成分由哪些因子构成
components_df = pd.DataFrame(
spca.components_,
columns=representatives
)
筛选权重绝对值 > 0.3 的因子
for i in range(20):
important_factors = components_df.iloc[i].abs() > 0.3
print(f"PC{i+1} 由 {components_df.iloc[i][important_factors].index.tolist()} 驱动")

📌 优势：
PC1 可能只由 RSI, MACD 构成 → 可解释为“动量状态”
PC2 由 Hurst, ADF 构成 → “趋势稳定性”

✅ 先进方法2：正交非负矩阵分解（Orthogonal NMF）
如果你希望主成分是“因子组合”，且权重为正（更符合经济直觉）。

python
from sklearn.decomposition import NMF

nmf = NMF(n_components=20, init='nndsvd', random_state=42)
X_nmf = nmf.fit_transform(df[representatives])
权重非负，可解释为“因子贡献度”

📌 适用场景：
构建“多因子打分卡”；
每个主成分是一个“因子篮子”，如“波动率篮子”、“动量篮子”。

✅ 先进方法3：自编码器 + 因子重要性蒸馏（State-of-the-Art）

如果你追求最先进，可以用：

text

🧩 阶段4：语义命名与监控（保持可解释性）

降维后，你不能只叫它“PC1”，而要赋予经济含义。
