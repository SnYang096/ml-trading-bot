特征泄漏（Feature Leakage） 是机器学习，尤其是量化交易中最隐蔽、最危险的过拟合形式之一。它指的是：模型在训练时使用了在真实预测时刻本不可获得的信息（通常是未来数据），导致回测表现虚高，实盘严重失效。

🔍 一、什么是特征泄漏？—— 核心定义
特征泄漏 = 特征 X(t) 包含了时间 t 之后的信息（如 X(t) 依赖于 y(t+1), price(t+5) 等）
🌰 举例说明：

场景 是否泄漏 为什么
------ -------- --------
用 close[t] 预测 return[t+1] ❌ 不泄漏 t 时刻收盘价已知
用 close[t+1] 预测 return[t+1] ✅ 泄漏 t 时刻无法知道下一根 K 线收盘价
用 rolling(60).mean(close) 在 t 时刻 ❌ 不泄漏（默认 center=False） 只用历史数据
用 rolling(60, center=True).mean(close) ✅ 泄漏 中心对齐，用了未来 30 根数据
用全样本标准化：(x - x.mean()) / x.std() ✅ 泄漏 实盘不知道全局均值和标准差
用未来波动率作为特征 ✅ 严重泄漏 直接偷看了未来

⚠️ 二、特征泄漏的常见类型
1. 时间对齐错误（Time Alignment Error）
滚动窗口设置错误（如 center=True）；
标签与特征未正确错位（如用 t 的特征预测 t 的标签，但标签本身是 t 的 future return）。
2. 全局信息泄漏（Global Statistic Leakage）
python
❌ 泄漏！用了全样本统计量
df['feature_zscore'] = (df['feature'] - df['feature'].mean()) / df['feature'].std()

→ 实盘时无法计算“全样本均值”。

✅ 正确做法：用 expanding window 或 固定参数：
python
✅ 安全：只用历史数据
df['feature_zscore'] = (df['feature'] - df['feature'].expanding().mean()) / df['feature'].expanding().std()
3. 预处理泄漏（Preprocessing Leakage）
在划分训练/测试集前进行标准化、填充缺失值、PCA 等操作；
导致测试集信息“泄露”到训练过程。

🛡️ 三、如何避免特征泄漏？—— 实战指南
✅ 原则 1：所有特征必须在决策时刻“可计算”
决策时刻 = t → 可用数据 ≤ t；
任何涉及 t+1, t+2, ... 的计算都是泄漏。
✅ 原则 2：滚动窗口默认 center=False，慎用 shift(-k)
Pandas .rolling() 默认 center=False，安全；
使用 .shift(-k) 必须明确目的（通常用于构造标签，而非特征）。
✅ 原则 3：禁止在特征工程中使用全样本统计量
替代方案：
expanding()：用历史所有数据；
rolling(window)：用固定历史窗口；
固定阈值（如 ATR > 0.05）。
✅ 原则 4：严格分离训练/测试预处理
使用 sklearn 的 Pipeline 自动防止预处理泄漏：
python
from sklearn.pipeline import Pipeline
pipe = Pipeline([
('scaler', StandardScaler()),
('model', LGBMRegressor())
])
cross_val_score 会自动在每折内 fit/transform

✅ 原则 5：命名清晰，避免混淆
不要把 trailing_volatility 命名为 future_vol；
特征名应体现其时间属性：ma_20_t, vol_60_t_minus_1 等。

🔬 四、如何检测特征泄漏？
方法 1：随机游走测试（Random Walk Test）
生成纯随机价格序列；
用相同特征工程流程提取特征；
训练模型 → 如果 Rank IC 显著 > 0（如 >0.03），说明特征或标签存在泄漏。
方法 2：人工检查高相关特征
如你之前看到的：dl_seq_f14 与未来收益相关性达 -0.19；
对 top 相关特征，逐行检查其计算逻辑是否用到未来数据。
方法 3：时间偏移测试
将整个数据集向后平移 N 步；
如果模型性能大幅下降 → 可能特征依赖特定时间对齐（如月末、周末效应），或存在边界泄漏。

🧪 五、一个安全的特征工程模板（Python）

python
def safe_feature_engineering(df: pd.DataFrame) -> pd.DataFrame:
# 假设 df 按时间排序，包含 open, high, low, close, volume

# 1. 原始价格特征（t 时刻已知）
df['log_return'] = np.log(df['close'] / df['close'].shift(1))

# 2. 滚动统计量（只用历史）
window = 60
df['ma_20'] = df['close'].rolling(20).mean()
df['vol_60'] = df['log_return'].rolling(window).std()
df['atr'] = talib.ATR(df['high'], df['low'], df['close'], timeperiod=14)

# 3. Z-score using expanding window (safe)
df['vol_zscore'] = (df['vol_60'] - df['vol_60'].expanding().mean()) / df['vol_60'].expanding().std()

# 4. 避免：df['feature'] = (df['feature'] - df['feature'].mean()) / df['feature'].std()

return df.dropna()

✅ 总结：特征泄漏防控 checklist

检查项 安全做法
-------- --------
滚动窗口 center=False（默认）
标准化 用 expanding() 或训练集 fit 后 transform 测试集
特征命名 清晰标明时间属性，避免“future”误导
自定义指标 手动验证是否只用历史数据
回测验证 必跑 Random Walk Test
预处理流程 使用 Pipeline 防止 leakage
