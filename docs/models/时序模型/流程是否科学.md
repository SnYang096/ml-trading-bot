## 优化建议

### 1. 在步骤 1 和 2 之间增加：因子组合分析

**新增步骤：** 在 `factor-test` 后，`ts-timeframe-forward-report` 前

```bash
# 因子组合分析（使用 Alphalens）
make cs-factor-eval \
    CS_FACTOR_SYMBOLS=BTCUSDT \
    START_DATE=2024-01-01 \
    END_DATE=2024-12-31 \
    CS_FACTOR_TIMEFRAME=5T \
    CS_FACTOR_HORIZON=24
```

**作用：**
- 评估因子组合的 IC/IR
- 分位数回测分析
- 因子衰减分析
- 更全面的因子有效性评估

### 2. 在步骤 3 中增加：稳定性验证

**优化：** 在 `ts-dim-compare` 中启用稳定性验证

```bash
make ts-dim-compare \
    SYMBOLS=BTCUSDT \
    START_DATE=2024-01-01 \
    END_DATE=2024-12-31 \
    ENABLE_STABILITY_VALIDATION=true \
    VALIDATION_START=2021-01-01 \
    VALIDATION_YEARS=3
```

**作用：**
- 用近期数据选因子，在更长历史数据上验证稳定性
- 避免因子过拟合到特定时间段
- 提高模型的泛化能力

### 3. 在步骤 4 和 5 之间增加：样本外测试

**新增步骤：** 滚动训练后，vectorbot 回测前

```bash
# 使用完全未参与训练的数据进行测试
make ts-vectorbot-backtest \
    BACKTEST_START=2025-01-01 \
    BACKTEST_END=2025-03-31 \
    BACKTEST_MODEL=models/rolling_model.pkl
```

**作用：**
- 确保模型泛化能力
- 避免过拟合
- 更真实的性能评估

### 4. 步骤 7 优化：自动化性能监控

**建议：** 添加自动化监控脚本，定期检查：
- 模型性能下降（R2、AUC、F1）
- 特征分布漂移
- 预测准确率下降
- 自动触发重新训练

## 优化后的完整流程

### 阶段 1: 因子研究与验证

```bash
# 1.1 测试单个因子有效性
make factor-test \
    FACTOR_TEST_FACTORS='rsi_7 macd_12_26' \
    FACTOR_TEST_SYMBOL=BTCUSDT \
    FACTOR_TEST_START_DATE=2024-01-01 \
    FACTOR_TEST_END_DATE=2024-12-31

# 1.2 因子组合分析（新增）
make cs-factor-eval \
    CS_FACTOR_SYMBOLS=BTCUSDT \
    START_DATE=2024-01-01 \
    END_DATE=2024-12-31 \
    CS_FACTOR_TIMEFRAME=5T

# 1.3 找最适合的 timeframe 和 forward-bars
make ts-timeframe-forward-report \
    SYMBOLS=BTCUSDT \
    TF_ANALYSIS_TIMEFRAMES=5T,15T,60T,240T \
    TF_ANALYSIS_FORWARD_BARS=3,6,12,24 \
    TF_ANALYSIS_RUN_TAG=btc_2024
```

### 阶段 2: 特征工程与降维

```bash
# 2.1 测试降维（带稳定性验证）
make ts-dim-compare \
    SYMBOLS=BTCUSDT \
    START_DATE=2024-01-01 \
    END_DATE=2024-12-31 \
    DIM_COMPARE_FEATURE_TYPE=comprehensive \
    DIM_COMPARE_TIMEFRAME=5T \
    ENABLE_STABILITY_VALIDATION=true \
    VALIDATION_START=2021-01-01
```

### 阶段 3: 模型训练

```bash
# 3.1 滚动训练（使用找到的最优参数）
make rolling \
    SYMBOLS=BTCUSDT \
    ROLLING_START=2024-01-01 \
    ROLLING_END=2025-04-30 \
    ROLLING_FREQ=5T \
    ROLLING_FBS=12 \
    ROLLING_FEATURE_TYPE=comprehensive \
    ROLLING_USE_TOP_FACTORS=true \
    ROLLING_TOPK=60
```

### 阶段 4: 回测验证

```bash
# 4.1 VectorBot 回测（样本内）
make ts-vectorbot-backtest \
    BACKTEST_MODEL=models/btcusdt_rolling_model.pkl \
    BACKTEST_SYMBOL=BTCUSDT \
    BACKTEST_START=2024-01-01 \
    BACKTEST_END=2024-12-31

# 4.2 VectorBot 回测（样本外 - 新增）
make ts-vectorbot-backtest \
    BACKTEST_MODEL=models/btcusdt_rolling_model.pkl \
    BACKTEST_SYMBOL=BTCUSDT \
    BACKTEST_START=2025-01-01 \
    BACKTEST_END=2025-03-31

# 4.3 Nautilus 回测（更接近实盘）
make ts-nautilus-backtest \
    SYMBOLS=BTCUSDT \
    START_DATE=2024-01-01 \
    END_DATE=2025-03-31
```

### 阶段 5: 实盘部署与监控

```bash
# 5.1 实盘部署（Nautilus Trader）
# 加载模型，开始实盘交易

# 5.2 定期性能监控（建议每月）

    SYMBOLS=BTCUSDT \
    ROLLING_START=2024-11-01 \
    ROLLING_END=2025-04-30 \
    ROLLING_OUTPUT=results/monitoring/rolling_check_$(shell date +%Y%m%d)

# 对比新旧模型性能，如果下降 >10%，考虑重新训练
```

2. **滚动训练**：能够适应市场变化，保持模型新鲜度

2. **稳定性验证**：在降维阶段验证因子在更长历史数据上的稳定性
