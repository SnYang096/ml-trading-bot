# 三模型系统特征选择说明

## 概述

本系统将交易机会分为三类，每类使用独立的模型和特征集：

1. **SR 反转型模型**：价格回测 SR 区 → 试探 → 快速反向运行
2. **SR 突破型模型**：价格强势穿过 SR 区，继续沿原方向运行
3. **压缩区突破模型**：价格在压缩区内震荡 → 放量突破 → 继续运行

## 一、SR 反转型模型特征选择

### 核心逻辑
反转型机会的核心是：价格在关键供需区获得支撑/阻力，然后快速反向运行。

### 特征分类与理由

#### 1. SR 区域强度特征
**为什么选择：**
- **SQS (Structure Quality Score)**：量化 SR 位的可靠性
  - 测试次数：多次测试的 SR 位更可靠
  - 反应强度：历史反弹/回落幅度
  - 时间衰减：近期测试的 SR 位更有效
- **dist_to_nearest_sr**：当前价格距离最近 SR 的距离
  - 距离越近，反转概率越高
  - 标准化距离便于跨资产比较
- **sr_density**：SR 位密度
  - 多个 SR 位重合（confluence）→ 更强的支撑/阻力
- **sr_confluence**：SR 位重合度
  - 多周期 SR 位对齐 → 高质量反转机会

**特征来源：**
- `baseline` 特征：`sqs_*`, `dist_to_nearest_sr`, `sr_density`, `sr_confluence`
- 计算逻辑：`baseline_features.py` 中的 `_compute_boundary_strengths()`

#### 2. 反转信号特征
**为什么选择：**
- **reversal**, **reversed**：明确的反转信号
- **fake_breakout**：假突破后的反转
- **breakout_status**：突破状态（用于判断是否反转）
- **price_reversed_before_sr**：价格在到达 SR 前就反转
  - 说明市场情绪变化，可能是高质量反转信号

**特征来源：**
- `baseline` 特征：`fake_breakout`, `price_reversed_before_sr`, `breakout_status`
- 计算逻辑：`baseline_features.py` 中的 `_compute_failed_breakout_reversal()`, `add_price_action_features()`

#### 3. SR 边界强度特征
**为什么选择：**
- **sr_strength**, **boundary_strength**：边界整体强度
- **support_sqs**, **resistance_sqs**：支撑/阻力的独立 SQS
  - 双向评估，更准确反映边界质量
- **price_above**, **trend_down**：价格位置和趋势
  - 用于判断边界当前的角色（支撑还是阻力）

**特征来源：**
- `baseline` 特征：`sqs_*`, `*_support_sqs`, `*_resistance_sqs`, `*_price_above`, `*_trend_down`
- 计算逻辑：`baseline_features.py` 中的 `_compute_boundary_strengths()`

#### 4. 成交量确认特征
**为什么选择：**
- **volume_ratio**, **vol_ratio**：成交量比率
  - 反转时成交量应该萎缩（缺乏跟风）或异常放大（恐慌/贪婪）
- **volume_confirmation**：成交量确认
  - 确认反转的有效性

**特征来源：**
- `baseline` 特征：`volume_ratio`, `vol_ratio`
- `enhanced` 特征：`volume_confirmation`（如果有）
- 计算逻辑：`baseline_features.py` 中的 `_compute_boundary_volume_confirmations()`

#### 5. 压缩和结构特征
**为什么选择：**
- **compression_score**, **compression_confidence**：压缩区域质量
  - 压缩区内的反转更可靠（能量积累）
- **compression**：压缩状态
  - 压缩 → 突破/反转的前兆

**特征来源：**
- `baseline` 特征：`compression_score`, `compression_confidence`
- 计算逻辑：`baseline_features.py` 中的压缩区检测

#### 6. 多时间框架特征
**为什么选择：**
- **trend_context**, **trend_4h**：多时间框架趋势
  - 4H/1H 趋势与 15m 反转信号对齐 → 高质量机会
- **volatility_regime**：波动率状态
  - 不同波动率状态下，反转有效性不同

**特征来源：**
- `baseline` 特征：`trend_context`, `trend_4h`, `volatility_regime`
- 计算逻辑：`baseline_features.py` 中的 `add_multi_timeframe_context_features()`

#### 7. 边界质量特征
**为什么选择：**
- **breakout_quality**：突破质量（用于判断是否假突破）
- **breakout_confirmation**：突破确认
- **role_flip**：角色转换概率
  - 支撑/阻力角色转换 → 反转机会

**特征来源：**
- `baseline` 特征：`*_breakout_quality`, `*_breakout_confirmation`, `*_role_flip_prob`
- 计算逻辑：`baseline_features.py` 中的 `_compute_breakout_confirmation_and_role_flip()`

#### 8. 价格行为特征
**为什么选择：**
- **price_action**：价格行为模式
- **candlestick**：蜡烛形态
  - Pin bar、吞没、长影线等 → 反转信号

**特征来源：**
- `baseline` 特征：`price_action_*`, `candlestick_*`（如果有）
- `default` 特征：传统技术指标（RSI、MACD 等）可能也有用

### 排除的特征
- **breakout_speed**, **momentum_decay**：这些更适合突破型模型
- **follow_through**：持续性特征，不适合反转逻辑

### 可增加的 enhanced/default 特征
- **RSI**（`default`）：超买/超卖，反转信号
- **MACD**（`default`）：背离，反转信号
- **Stochastic**（`default`）：超买/超卖
- **Hurst Exponent**（`enhanced`）：趋势持续性，低 Hurst → 反转概率高
- **Wavelet Features**（`enhanced`）：多频率分析，识别反转点
- **Order Flow Imbalance**（`enhanced`）：订单流不平衡，反转前兆

## 二、SR 突破型模型特征选择

### 核心逻辑
突破型机会的核心是：价格强势穿过 SR 区，成交量放大，继续沿原方向运行。

### 特征分类与理由

#### 1. 突破动量特征
**为什么选择：**
- **breakout_speed**：突破速度
  - 快速突破 → 强势信号
- **momentum**：动量
  - 突破时的动量强度
- **momentum_decay**：动量衰减
  - 突破后动量是否持续
- **follow_through**：跟进
  - 突破后是否继续新高/新低

**特征来源：**
- `baseline` 特征：`breakout_speed`, `momentum_decay`, `follow_through_1`, `follow_through_2`
- 计算逻辑：`baseline_features.py` 中的 `_add_breakout_quality_features()`

#### 2. 成交量特征
**为什么选择：**
- **volume_ratio**, **vol_ratio**：成交量比率
  - 突破时成交量应该放大（确认突破有效性）
- **volume_spike**：成交量突增
  - 突破时的成交量突增
- **volume_confirmation**：成交量确认
- **order_flow**：订单流
  - 突破时的订单流方向
- **cvd**：累积成交量差
  - 突破时的订单流不平衡
- **taker_buy_ratio**：主动买入占比
  - 突破时的买卖压力

**特征来源：**
- `baseline` 特征：`volume_ratio`, `vol_ratio`
- `enhanced` 特征：`cvd_*`, `taker_buy_ratio`, `order_flow_*`
- 计算逻辑：`baseline_features.py` 中的 `_compute_boundary_volume_confirmations()`, `feature_engineering_enhanced.py` 中的订单流特征

#### 3. 趋势特征
**为什么选择：**
- **trend**, **trend_strength**：趋势强度
  - 强趋势中的突破更可靠
- **trend_context**, **trend_4h**：多时间框架趋势
  - 大周期趋势与突破方向一致 → 高质量机会
- **momentum_persistence**：动量持续性
  - 突破后动量是否持续

**特征来源：**
- `baseline` 特征：`trend_strength`, `trend_context`, `trend_4h`
- `default` 特征：`ADX`（趋势强度指标）
- 计算逻辑：`baseline_features.py` 中的 `add_multi_timeframe_context_features()`

#### 4. 流动性特征
**为什么选择：**
- **liquidity**：流动性
- **liquidity_pool**：流动性池位置
  - 突破后是否还有流动性池（止损猎物）
- **order_block**：订单块
  - 机构订单块，突破后可能继续运行

**特征来源：**
- `baseline` 特征：`liquidity_*`（如果有）
- `enhanced` 特征：`order_block_*`（如果有）

#### 5. 波动率特征
**为什么选择：**
- **volatility**, **volatility_regime**：波动率状态
  - 波动率扩张 → 突破有效性
- **atr**：平均真实波幅
  - 突破幅度相对于 ATR
- **volatility_expansion**：波动率扩张
  - 突破时的波动率扩张

**特征来源：**
- `baseline` 特征：`volatility`, `volatility_regime`, `atr`
- `default` 特征：`ATR`, `BBANDS`（布林带宽度）

#### 6. 价格行为特征
**为什么选择：**
- **price_action**：价格行为模式
- **breakout_status**：突破状态
- **breakout_confirmation**：突破确认
  - 突破后是否站稳

**特征来源：**
- `baseline` 特征：`breakout_status`, `breakout_confirmation`
- 计算逻辑：`baseline_features.py` 中的 `_compute_breakout_confirmation_and_role_flip()`

### 排除的特征
- **reversal**, **reversed**：这些更适合反转型模型
- **fake_breakout**：假突破，不适合突破逻辑

### 可增加的 enhanced/default 特征
- **ADX**（`default`）：趋势强度，突破时 ADX 应该上升
- **Parabolic SAR**（`default`）：趋势跟踪，突破信号
- **Ichimoku**（`default`）：云图突破
- **Volume Profile**（`enhanced`）：成交量分布，POC 突破
- **Market Microstructure**（`enhanced`）：市场微观结构，突破有效性

## 三、压缩区突破模型特征选择

### 核心逻辑
压缩区突破的核心是：价格在压缩区内震荡（波动率低、成交量萎缩）→ 突然放量突破 → 继续运行。

### 特征分类与理由

#### 1. 压缩指标特征
**为什么选择：**
- **compression_score**：压缩分数
  - 压缩程度（波动率低 → 压缩分数高）
- **compression_confidence**：压缩置信度
  - 压缩区的可靠性
- **bb_width**：布林带宽度
  - 布林带收窄 → 压缩
- **volatility**：波动率
  - 低波动率 → 压缩

**特征来源：**
- `baseline` 特征：`compression_score`, `compression_confidence`
- `default` 特征：`BBANDS`（布林带宽度）
- 计算逻辑：`baseline_features.py` 中的压缩区检测

#### 2. 突破动量特征
**为什么选择：**
- **breakout_speed**：突破速度
  - 压缩区突破应该快速
- **breakout_momentum**：突破动量
  - 突破时的动量强度
- **follow_through**：跟进
  - 突破后是否继续运行

**特征来源：**
- `baseline` 特征：`breakout_speed`, `follow_through_1`, `follow_through_2`
- 计算逻辑：`baseline_features.py` 中的 `_add_breakout_quality_features()`

#### 3. 成交量特征
**为什么选择：**
- **volume**, **volume_spike**：成交量突增
  - 压缩区突破时成交量应该显著放大
- **volume_confirmation**：成交量确认
- **order_flow**：订单流
  - 突破时的订单流方向
- **cvd**：累积成交量差
  - 突破时的订单流不平衡

**特征来源：**
- `baseline` 特征：`volume_ratio`, `vol_ratio`
- `enhanced` 特征：`cvd_*`, `taker_buy_ratio`
- 计算逻辑：`baseline_features.py` 中的 `_compute_boundary_volume_confirmations()`

#### 4. 价格位置特征
**为什么选择：**
- **bb_position**：布林带位置
  - 价格在压缩区内的位置
- **price_action**：价格行为
  - 压缩区内的价格行为模式

**特征来源：**
- `baseline` 特征：`bb_position`（如果有）
- `default` 特征：`BBANDS`（布林带位置）

#### 5. 趋势特征
**为什么选择：**
- **trend**, **trend_strength**：趋势强度
  - 压缩区突破后的趋势延续

**特征来源：**
- `baseline` 特征：`trend_strength`
- `default` 特征：`ADX`, `MA`（均线斜率）

### 可增加的 enhanced/default 特征
- **Bollinger Bands**（`default`）：压缩区检测
- **Keltner Channels**（`default`）：压缩区检测
- **Donchian Channels**（`default`）：压缩区检测
- **Volume Profile**（`enhanced`）：成交量分布，压缩区识别
- **Market Regime**（`enhanced`）：市场状态，压缩区更可能在震荡市

## 四、特征工程类型建议

### 1. Baseline 特征（必须）
- **SR 区域强度**：SQS、距离、密度
- **反转/突破信号**：价格行为特征
- **成交量确认**：volume_ratio、volume_confirmation
- **压缩区质量**：compression_score

### 2. Default 特征（推荐）
- **技术指标**：RSI、MACD、ADX、Bollinger Bands
- **均线系统**：MA、EMA、SMA
- **趋势指标**：Parabolic SAR、Ichimoku

### 3. Enhanced 特征（可选）
- **Hurst Exponent**：趋势持续性（反转型模型）
- **Wavelet Features**：多频率分析（反转型模型）
- **Order Flow**：订单流不平衡（突破型模型）
- **Volume Profile**：成交量分布（压缩突破模型）

### 4. Alpha101 特征（可选）
- **WorldQuant Alpha101**：经典量化因子
- 可能对突破型模型有用

## 五、特征选择策略

### 当前实现
- 基于关键词匹配选择特征
- 每个模型有专门的特征选择函数

### 改进建议
1. **特征重要性分析**：训练后分析特征重要性，动态调整
2. **特征交互**：考虑特征之间的交互作用
3. **特征工程迭代**：根据模型表现，持续优化特征选择

---

## 六、测试建议

### 1. 基础测试
- 只使用 `baseline` 特征
- 评估每个模型的基础表现

### 2. 增强测试
- `baseline + default`：加入传统技术指标
- `baseline + enhanced`：加入高级特征
- `baseline + default + enhanced`：全特征集

### 3. 对比测试
- 对比不同特征组合的表现
- 找出每个模型的最优特征集

---

## 七、总结

### 反转型模型
- **核心特征**：SR 区域强度、反转信号、成交量确认
- **推荐增加**：RSI、MACD、Hurst Exponent、Wavelet Features

### 突破型模型
- **核心特征**：突破动量、成交量放大、趋势延续
- **推荐增加**：ADX、Parabolic SAR、Order Flow、Volume Profile

### 压缩突破模型
- **核心特征**：压缩指标、突破动量、成交量放大
- **推荐增加**：Bollinger Bands、Keltner Channels、Volume Profile

### 关键原则
1. **特征与机会类型对齐**：每个模型只用相关特征
2. **避免特征混淆**：反转型和突破型特征分开
3. **持续优化**：根据模型表现，动态调整特征选择
