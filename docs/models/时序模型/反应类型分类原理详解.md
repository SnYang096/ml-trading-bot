# 反应类型分类原理详解

## 概述

本系统将交易机会分为三类，每类使用独立的模型和标签：

1. **`sr_reaction == "reversal"`** - SR 反转型
2. **`sr_reaction == "breakout"`** - SR 突破型  
3. **`compression_reaction == "compression_breakout"`** - 压缩区突破型

本文档详细解释这三个分类的原理、判断标准和实现逻辑。

---

## 一、SR 反转型 (`sr_reaction == "reversal"`)

### 核心思想

**反转型机会**是 SR 策略的核心盈利来源：价格回测 SR 区（支撑/阻力）→ 试探（wick 进入）→ **快速反向运行**。

### 判断原理

通过观察价格在 SR 区域附近的行为模式来判断：

#### 1. Long 信号（期待从需求区反弹）

**判断逻辑：**

```
在时刻 t，有 Long 信号（signal > 0），期待价格从需求区（zone_price）反弹上涨。

观察未来 lookback_window（默认 5 根 K 线）的价格行为：

1. 如果价格深跌破区域：
   min_low < zone_price - threshold * ATR
   → 说明反转失败，价格继续下跌
   → 分类为 "breakout"（反向失败）

2. 如果价格快速上涨且未深跌：
   max_high > zone_price + 2 * ATR 且 min_low > zone_price - 0.5 * ATR
   → 说明价格强势突破，不是反转
   → 分类为 "breakout"（强势突破）

3. 否则（有下影线试探后上涨）：
   → 价格在需求区附近试探（下影线），然后反弹
   → 分类为 "reversal"（典型反转）
```

**代码实现：**

```python
if signal > 0:  # Long signal (expecting bounce from demand zone)
    # Check for deep penetration below zone (breakout failure)
    if min_low < zone - threshold:
        reactions.iloc[i] = "breakout"  # 反转失败
    # Check for strong upward move without deep penetration
    elif max_high > zone + 2 * atr and min_low > zone - 0.5 * atr:
        reactions.iloc[i] = "breakout"  # 强势突破
    else:
        reactions.iloc[i] = "reversal"  # 典型反转
```

#### 2. Short 信号（期待从供给区回落）

**判断逻辑：**

```
在时刻 t，有 Short 信号（signal < 0），期待价格从供给区（zone_price）回落下跌。

观察未来 lookback_window（默认 5 根 K 线）的价格行为：

1. 如果价格强势突破区域：
   max_high > zone_price + threshold * ATR
   → 说明反转失败，价格继续上涨
   → 分类为 "breakout"（反向失败）

2. 如果价格快速下跌且未突破：
   min_low < zone_price - 2 * ATR 且 max_high < zone_price + 0.5 * ATR
   → 说明价格强势突破，不是反转
   → 分类为 "breakout"（强势突破）

3. 否则（有上影线试探后下跌）：
   → 价格在供给区附近试探（上影线），然后回落
   → 分类为 "reversal"（典型反转）
```

**代码实现：**

```python
else:  # Short signal (expecting rejection from supply zone)
    # Check for strong penetration above zone (breakout failure)
    if max_high > zone + threshold:
        reactions.iloc[i] = "breakout"  # 反转失败
    # Check for strong downward move without penetration
    elif min_low < zone - 2 * atr and max_high < zone + 0.5 * atr:
        reactions.iloc[i] = "breakout"  # 强势突破
    else:
        reactions.iloc[i] = "reversal"  # 典型反转
```

### 关键参数

- **`lookback_window`**（默认 5）：观察未来多少根 K 线来判断反应类型
- **`threshold_factor`**（默认 0.5）：用于判断"深穿透"的 ATR 倍数
  - `threshold = threshold_factor * ATR`
  - 如果价格穿透超过 `threshold`，认为是"深穿透"

### 判断标准总结

| 信号类型 | 条件 1（深穿透） | 条件 2（强势突破） | 否则 |
|---------|----------------|------------------|------|
| **Long** | `min_low < zone - 0.5*ATR` → **breakout** | `max_high > zone + 2*ATR` 且 `min_low > zone - 0.5*ATR` → **breakout** | **reversal** |
| **Short** | `max_high > zone + 0.5*ATR` → **breakout** | `min_low < zone - 2*ATR` 且 `max_high < zone + 0.5*ATR` → **breakout** | **reversal** |

### 实际案例

#### 案例 1：典型反转（Long）

```
时刻 t: 价格在需求区附近，发出 Long 信号
未来 5 根 K 线：
  - 最低价：zone_price - 0.3*ATR（轻微试探，未深跌）
  - 最高价：zone_price + 1.5*ATR（反弹上涨）
  → 分类：reversal ✅
```

#### 案例 2：反转失败（Long）

```
时刻 t: 价格在需求区附近，发出 Long 信号
未来 5 根 K 线：
  - 最低价：zone_price - 1.0*ATR（深跌破区域）
  - 最高价：zone_price + 0.2*ATR（未反弹）
  → 分类：breakout（反向失败）
```

#### 案例 3：强势突破（Long）

```
时刻 t: 价格在需求区附近，发出 Long 信号
未来 5 根 K 线：
  - 最低价：zone_price - 0.2*ATR（未深跌）
  - 最高价：zone_price + 2.5*ATR（快速上涨）
  → 分类：breakout（强势突破，不是反转）
```

## 二、SR 突破型 (`sr_reaction == "breakout"`)

### 核心思想

**突破型机会**是趋势延续的表现：价格强势穿过 SR 区，继续沿原方向运行（非反转）。

### 判断原理

突破型机会的判断与反转型相反：

#### 1. Long 信号（期待从需求区反弹，但实际突破）

**判断逻辑：**

```
在时刻 t，有 Long 信号，但价格行为显示：

1. 深跌破区域（反转失败）：
   min_low < zone_price - threshold * ATR
   → 分类为 "breakout"（反向失败）

2. 快速上涨且未深跌（强势突破）：
   max_high > zone_price + 2 * ATR 且 min_low > zone_price - 0.5 * ATR
   → 分类为 "breakout"（强势突破）
```

#### 2. Short 信号（期待从供给区回落，但实际突破）

**判断逻辑：**

```
在时刻 t，有 Short 信号，但价格行为显示：

1. 强势突破区域（反转失败）：
   max_high > zone_price + threshold * ATR
   → 分类为 "breakout"（反向失败）

2. 快速下跌且未突破（强势突破）：
   min_low < zone_price - 2 * ATR 且 max_high < zone_price + 0.5 * ATR
   → 分类为 "breakout"（强势突破）
```

### 与反转型的区别

| 特征 | 反转型（reversal） | 突破型（breakout） |
|------|------------------|------------------|
| **价格行为** | 试探后反向运行 | 强势穿过，继续原方向 |
| **穿透深度** | 浅试探（< 0.5*ATR） | 深穿透（> 0.5*ATR）或快速突破（> 2*ATR） |
| **交易逻辑** | 逆势交易（高胜率） | 顺势交易（高盈亏比） |
| **特征侧重** | SR 区域强度、反转信号 | 突破动量、成交量放大 |

## 三、压缩区突破型 (`compression_reaction == "compression_breakout"`)

### 核心思想

**压缩区突破**是第三种重要的交易机会：价格在压缩区内震荡（波动率低、成交量萎缩）→ 突然放量突破 → 继续运行。

### 判断原理

通过检测压缩区和突破行为来判断：

#### 1. 压缩区检测

**判断逻辑：**

```python
# 1. 检测是否在压缩区
compression_window = compression_arr[i - lookback_window : i]
avg_compression = np.nanmean(compression_window)

if avg_compression < compression_threshold:  # 默认 0.3
    continue  # 不在压缩区，跳过

# 2. 计算压缩区范围
comp_high = np.nanmax(high_arr[i - lookback_window : i])
comp_low = np.nanmin(low_arr[i - lookback_window : i])
comp_range = comp_high - comp_low
```

**压缩区特征：**
- 波动率低（ATR 相对价格很小）
- 成交量萎缩（相对于平均成交量）
- 价格在窄幅区间内震荡
- Bollinger Band 宽度收窄

#### 2. 突破检测

**判断逻辑：**

```python
# 观察未来 5 根 K 线
for j in range(1, min(5, len(df) - i)):
    future_high = high_arr[i + j]
    future_low = low_arr[i + j]
    future_volume = volume_arr[i + j]
    avg_volume = np.nanmean(volume_arr[max(0, i - 20) : i])
    
    # 检查向上突破
    if future_high > comp_high + breakout_threshold * atr:  # 默认 1.5*ATR
        if future_volume > avg_volume * 1.5:  # 成交量放大 1.5 倍
            reactions.iloc[i] = "compression_breakout"
            break
    
    # 检查向下突破
    if future_low < comp_low - breakout_threshold * atr:
        if future_volume > avg_volume * 1.5:
            reactions.iloc[i] = "compression_breakout"
            break
else:
    # 未突破，仍在压缩区
    reactions.iloc[i] = "compression_hold"
```

### 关键参数

- **`lookback_window`**（默认 10）：用于检测压缩区的回看窗口
- **`compression_threshold`**（默认 0.3）：压缩分数阈值（越低越压缩）
- **`breakout_threshold`**（默认 1.5）：突破幅度阈值（ATR 倍数）
- **`volume_lookback`**（默认 20）：计算平均成交量的回看窗口
- **`min_volume_ratio`**（默认 1.5）：突破时成交量放大倍数

### 判断标准总结

| 条件 | 结果 |
|------|------|
| 不在压缩区 | `np.nan`（跳过） |
| 在压缩区 + 放量突破（> 1.5*ATR） | `"compression_breakout"` |
| 在压缩区 + 未突破 | `"compression_hold"` |

### 实际案例

#### 案例 1：典型压缩区突破

```
时刻 t-10 到 t: 价格在窄幅区间震荡（压缩区）
  - 压缩分数：0.8（高度压缩）
  - 成交量：平均成交量的 0.6 倍（萎缩）

时刻 t+1: 
  - 价格突破压缩区上沿 + 1.8*ATR
  - 成交量：平均成交量的 2.0 倍（放大）
  → 分类：compression_breakout ✅
```

#### 案例 2：压缩区保持

```
时刻 t-10 到 t: 价格在窄幅区间震荡（压缩区）
  - 压缩分数：0.8（高度压缩）

时刻 t+1 到 t+5: 
  - 价格仍在压缩区内
  - 未突破压缩区边界
  → 分类：compression_hold
```

## 四、三个分类的关系与区别

### 关系图

```
交易机会
├── SR 相关机会
│   ├── 反转型（reversal）
│   │   └── 价格回测 SR 区 → 试探 → 反向运行
│   └── 突破型（breakout）
│       └── 价格强势穿过 SR 区 → 继续原方向
└── 压缩区相关机会
    └── 压缩区突破（compression_breakout）
        └── 压缩区震荡 → 放量突破 → 继续运行
```

### 区别对比

| 维度 | 反转型（reversal） | 突破型（breakout） | 压缩区突破（compression_breakout） |
|------|------------------|------------------|--------------------------------|
| **触发条件** | SR 区域附近 | SR 区域附近 | 压缩区内 |
| **价格行为** | 试探后反向 | 强势穿过，继续原方向 | 压缩后放量突破 |
| **成交量** | 萎缩或异常放大 | 放大 | 放大（必须） |
| **波动率** | 正常 | 扩张 | 先压缩后扩张 |
| **交易逻辑** | 逆势（高胜率） | 顺势（高盈亏比） | 趋势启动（高盈亏比） |
| **特征侧重** | SR 强度、反转信号 | 突破动量、趋势延续 | 压缩指标、突破动量 |
| **标签** | `rr_reversal_achieved` | `rr_breakout_achieved` | `rr_achieved` |

### 为什么需要分开？

1. **标签语义冲突**
   - 反转型：期待价格反向运行（逆势）
   - 突破型：期待价格继续原方向（顺势）
   - 混在一起训练 → 模型学到"平均噪声"

2. **特征意义错位**
   - 反转型特征：SR 强度、反转信号
   - 突破型特征：突破动量、趋势延续
   - 压缩突破特征：压缩指标、突破动量
   - 混在一起 → 特征权重互相抵消

3. **风险收益结构不一致**
   - 反转型：高胜率 + 中等 R/R
   - 突破型：低胜率 + 高 R/R
   - 压缩突破：中等胜率 + 高 R/R
   - 统一目标函数无法同时优化

## 五、实现细节

### 1. 数据流

```
原始数据
  ↓
特征工程（生成 signal、nearest_sr、compression_score 等）
  ↓
分类函数
  ├── classify_sr_reaction() → sr_reaction (reversal/breakout)
  └── classify_compression_breakout() → compression_reaction (compression_breakout/compression_hold)
  ↓
标签生成
  ├── compute_rr_label_by_reaction() → rr_reversal_achieved
  ├── compute_rr_label_by_reaction() → rr_breakout_achieved
  └── compute_rr_label() → rr_achieved (for compression_breakout)
  ↓
模型训练
  ├── 反转型模型（只用 reversal 样本）
  ├── 突破型模型（只用 breakout 样本）
  └── 压缩突破模型（只用 compression_breakout 样本）
```

### 2. 关键函数

#### `classify_sr_reaction()`
- **输入**：DataFrame with signals, SR zone prices, OHLCV data
- **输出**：Series with 'reversal' or 'breakout'
- **原理**：观察未来价格行为，判断是反转还是突破

#### `classify_compression_breakout()`
- **输入**：DataFrame with OHLCV data, compression scores
- **输出**：Series with 'compression_breakout' or 'compression_hold'
- **原理**：检测压缩区，然后检测突破行为

#### `compute_rr_label_by_reaction()`
- **输入**：DataFrame with reaction types, signals, OHLCV data
- **输出**：Series with R/R labels (binary or continuous)
- **原理**：根据反应类型过滤样本，计算 R/R 标签

### 3. 时间对齐（严格因果性）

**关键点：所有分类函数都使用"未来数据"来判断反应类型，但这是用于标签生成，不是特征工程。**

```
时刻 t: 生成信号、特征（只用 t 及之前的数据）
  ↓
时刻 t+1 到 t+lookback_window: 观察价格行为（用于分类）
  ↓
时刻 t: 根据未来行为，分类为 reversal/breakout/compression_breakout
  ↓
时刻 t: 生成标签（rr_reversal_achieved 等）
```

**注意：**
- 分类时使用未来数据是**必要的**，因为我们需要知道"实际发生了什么"
- 但训练时，模型只能使用 t 时刻的特征，不能使用未来信息
- 这是标准的监督学习流程：用历史数据训练，用未来数据验证

## 六、参数调优建议

### `classify_sr_reaction()` 参数

- **`lookback_window`**（默认 5）：
  - 太小（< 3）：可能错过延迟反应
  - 太大（> 10）：可能包含过多噪声
  - **建议**：根据时间框架调整（15m → 5，1H → 3）

- **`threshold_factor`**（默认 0.5）：
  - 太小（< 0.3）：容易误判为突破
  - 太大（> 1.0）：容易误判为反转
  - **建议**：根据市场波动率调整（高波动 → 0.7，低波动 → 0.3）

### `classify_compression_breakout()` 参数

- **`compression_threshold`**（默认 0.3）：
  - 太小（< 0.2）：只识别极端压缩
  - 太大（> 0.5）：包含非压缩区
  - **建议**：根据历史数据分布调整

- **`breakout_threshold`**（默认 1.5）：
  - 太小（< 1.0）：容易误判为突破
  - 太大（> 2.0）：错过小突破
  - **建议**：根据 ATR 分布调整

- **`min_volume_ratio`**（默认 1.5）：
  - 太小（< 1.2）：包含假突破
  - 太大（> 2.0）：错过真实突破
  - **建议**：根据成交量分布调整

## 七、总结

### 核心原理

1. **反转型（reversal）**：价格在 SR 区试探后反向运行
   - 判断：浅试探 + 反向运行 → reversal
   - 深穿透或快速突破 → breakout

2. **突破型（breakout）**：价格强势穿过 SR 区，继续原方向
   - 判断：深穿透或快速突破 → breakout
   - 浅试探 + 反向运行 → reversal

3. **压缩区突破（compression_breakout）**：压缩区震荡后放量突破
   - 判断：在压缩区 + 放量突破 → compression_breakout
   - 否则 → compression_hold

### 关键优势

- **逻辑清晰**：每个分类都有明确的判断标准
- **特征对齐**：每个模型只用相关特征
- **标签一致**：每个模型只优化自己的目标
- **可解释性强**：分类逻辑透明，便于调试和优化

### 下一步

1. **参数调优**：根据历史数据优化分类参数
2. **特征工程**：为每个模型设计更精准的特征
3. **模型训练**：独立训练三个模型，评估各自表现
4. **组合策略**：在实盘中组合使用三个模型
