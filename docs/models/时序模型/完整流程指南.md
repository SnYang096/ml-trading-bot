# 时序模型完整流程指南

本文档描述从研究到上线的完整工作流程，所有命令均使用 config-driven 架构。

## 一、命令总览

### 1. 策略训练命令

#### `make train-strategy` - 单次训练
**作用**：使用固定时间范围训练一个模型，适合快速验证和参数调优。

```bash
make train-strategy \
  STRATEGY_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  TIMEFRAME=15T \
  TEST_SIZE=0.15
```

**输出**：
- `results/strategies/{strategy_name}/model.pkl`
- `results/strategies/{strategy_name}/results.json`

---

#### `make rolling` - 滚动训练（推荐用于生产）
**作用**：使用扩展窗口（Expanding Window）按月滚动训练多个模型，模拟真实生产环境。

```bash
make rolling \
  ROLLING_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  INITIAL_TRAIN_MONTHS=6 \
  MIN_TRAIN_MONTHS=3
```

**工作流程**：
```
第1次: 训练=[1-6月], 测试=7月 → 模型1
第2次: 训练=[1-7月], 测试=8月 → 模型2
第3次: 训练=[1-8月], 测试=9月 → 模型3
...
```

**输出**：
- `results/rolling/{strategy_name}/{month}/model.pkl`
- `results/rolling/{strategy_name}/monthly_results.json`

---

### 2. 特征分析命令

#### `make ts-factor-eval` - 单因子评估
**作用**：评估单个或多个因子在时间序列上的 IC、胜率等指标。

```bash
make ts-factor-eval \
  TS_FACTOR_STRATEGY=config/strategies/sr_reversal \
  TS_FACTOR_FACTORS="atr sqs_hal_high" \
  TS_FACTOR_SYMBOL=BTCUSDT \
  TS_FACTOR_TIMEFRAME=15T
```

**输出**：
- 因子 IC 曲线图
- 因子胜率统计
- JSON 报告

---

#### `make ts-dim-compare` - 特征选择（降维）
**作用**：三阶段特征选择，从大量特征中自动选择最优子集。

**三阶段流程**：
1. **Stage 1**: Missing/stability filter（移除缺失率>20%或低方差特征）
2. **Stage 2**: IC ranking（按信息系数排序，选择 top K）
3. **Stage 3**: Correlation-based selection（去除高相关性冗余特征）

```bash
make ts-dim-compare \
  DIM_COMPARE_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  START_DATE=2024-01-01 \
  END_DATE=2024-12-31
```

**输出**：
- `results/dim_compare/{strategy}_{symbol}_{timestamp}/top_factors.json` ← **供 rolling 使用**
- `results/dim_compare/{strategy}_{symbol}_{timestamp}/results.json`

---

#### `make ts-strategy-feature-compare` - 消融实验
**作用**：对比不同特征配置的性能，评估特征组的贡献。

**消融实验原理**：
- 通过移除/添加特征组，对比性能差异
- 量化每个特征组对最终结果的贡献

```bash
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="baseline=config/features/baseline.yaml full=config/features/full.yaml"
```

**输出**：
- `results/strategy_compare/strategy_feature_compare_summary.csv`
- `results/strategy_compare/strategy_feature_compare_summary.json`

---

### 3. 回测命令

#### `make ts-vectorbot-backtest` - VectorBT 回测
**作用**：使用 VectorBT 进行策略回测，评估交易表现。

```bash
make ts-vectorbot-backtest \
  BACKTEST_MODEL=results/strategies/sr_reversal/model.pkl \
  BACKTEST_SYMBOL=BTCUSDT \
  BACKTEST_START=2024-01-01 \
  BACKTEST_END=2024-12-31
```

## 二、完整工作流程

### 阶段1：特征工程与验证

#### 1.1 单因子评估
```bash
# 评估单个因子的有效性
make ts-factor-eval \
  TS_FACTOR_STRATEGY=config/strategies/sr_reversal \
  TS_FACTOR_FACTORS="atr rsi sqs_hal_high" \
  TS_FACTOR_SYMBOL=BTCUSDT
```

**目标**：识别有效因子，排除无效因子。

---

#### 1.2 特征选择（降维）
```bash
# 从大量特征中选择最优子集
make ts-dim-compare \
  DIM_COMPARE_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  START_DATE=2024-01-01 \
  END_DATE=2024-12-31
```

**输出**：`top_factors.json` ← **保存此文件，供后续使用**

**目标**：减少特征维度，提高模型泛化能力。

---

#### 1.3 消融实验（可选）
```bash
# 评估特征组的重要性
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="baseline=config/features/baseline.yaml full=config/features/full.yaml"
```

**目标**：验证特征组的贡献，优化特征配置。

---

### 阶段2：模型训练

#### 2.1 单次训练（快速验证）
```bash

  STRATEGY_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  TEST_SIZE=0.15
```

**目标**：快速验证策略配置是否合理。

---

#### 2.2 滚动训练（生产准备）

**目标**：

## 三、命令对比表

| 命令 | 用途 | 训练方式 | 模型数量 | 适用场景 |
|------|------|----------|----------|----------|
| `train-strategy` | 单次训练 | 固定时间范围 | 1个 | 快速验证、参数调优 |
| `rolling` | 滚动训练 | 扩展窗口 | N个（每月1个） | **生产环境、稳定性评估** |
| `ts-dim-compare` | 特征选择 | 固定时间范围 | 2个（对比用） | 特征降维、top factors 选择 |
| `ts-strategy-feature-compare` | 消融实验 | 固定时间范围 | M个（每个变体1个） | 特征组重要性评估 |
| `ts-factor-eval` | 单因子评估 | 固定时间范围 | 0个 | 因子有效性验证 |

## 四、典型工作流示例

### 示例1：新策略开发流程

```bash
# 1. 评估因子
make ts-factor-eval TS_FACTOR_STRATEGY=config/strategies/sr_reversal TS_FACTOR_FACTORS="atr rsi" SYMBOL=BTCUSDT

# 2. 特征选择
make ts-dim-compare DIM_COMPARE_CONFIG=config/strategies/sr_reversal SYMBOL=BTCUSDT

# 3. 快速验证
make train-strategy STRATEGY_CONFIG=config/strategies/sr_reversal SYMBOL=BTCUSDT

# 4. 滚动训练（生产准备）
make rolling ROLLING_CONFIG=config/strategies/sr_reversal SYMBOL=BTCUSDT

# 5. 回测验证
make ts-vectorbot-backtest BACKTEST_MODEL=results/strategies/sr_reversal/model.pkl SYMBOL=BTCUSDT
```

### 示例2：特征优化流程

```bash
# 1. 消融实验：对比不同特征配置
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="baseline=config/features/baseline.yaml full=config/features/full.yaml"

# 2. 根据结果优化 features.yaml

# 3. 重新训练
make train-strategy STRATEGY_CONFIG=config/strategies/sr_reversal SYMBOL=BTCUSDT
```

## 五、输出文件说明

### 训练输出
- `results/strategies/{strategy}/model.pkl`: 训练好的模型
- `results/strategies/{strategy}/results.json`: 训练结果和评估指标

### 滚动训练输出
- `results/rolling/{strategy}/{month}/model.pkl`: 每个月的模型
- `results/rolling/{strategy}/monthly_results.json`: 所有月份的汇总结果

### 特征选择输出
- `results/dim_compare/{strategy}_{symbol}_{timestamp}/top_factors.json`: **Top factors 列表（供 rolling 使用）**
- `results/dim_compare/{strategy}_{symbol}_{timestamp}/results.json`: 详细结果

### 消融实验输出
- `results/strategy_compare/strategy_feature_compare_summary.csv`: 对比摘要表
- `results/strategy_compare/strategy_feature_compare_summary.json`: 详细结果

## 六、最佳实践

1. **特征工程阶段**：
   - 先用 `ts-factor-eval` 评估单个因子
   - 再用 `ts-dim-compare` 做特征选择
   - 最后用 `ts-strategy-feature-compare` 做消融实验

2. **模型训练阶段**：
   - 先用 `train-strategy` 快速验证
   - 再用 `rolling` 做生产级训练

3. **验证阶段**：
   - 使用 `ts-vectorbot-backtest` 进行回测
   - 分析滚动训练的结果稳定性

4. **持续优化**：
   - 定期运行 `rolling` 更新模型
   - 使用 `ts-strategy-feature-compare` 评估新特征
