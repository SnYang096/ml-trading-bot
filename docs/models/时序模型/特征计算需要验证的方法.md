✅ 你提出的 5 点点评
1. 归一化验证（多资产训练兼容性）
✔️ 正确。不同资产波动率量纲差异巨大（如 BTC vs. 国债），若特征未做跨资产标准化（如 z-score、rank、percentile），模型会偏向高波动资产。
🔍 测试建议：
检查 vol_zscore 或 vol_percentile_approx 在不同资产上是否均值≈0、范围∈[0,1]；
多资产拼接后，同一特征的分布不应出现极端偏移。
2. 是否使用未来数据（Look-Ahead Bias）
✔️ 这是量化建模的“红线”。任何特征在 t 时刻只能依赖 ≤t 的数据。
🔍 测试建议：
对比“流式计算”（逐行 forward-fill）和“批量向量计算”结果是否一致；
用已知未来事件（如某日暴涨）验证：该事件发生前的特征值是否突变？若突变，则存在泄露。
3. 相关性是否不合理
✔️ 高度相关特征虽不一定有害（LightGBM 能处理），但若出现“逻辑矛盾”的相关性（如 vol_ma_5 和 vol_ma_20 相关性 <0.3），可能说明计算错误。
🔍 测试建议：
计算特征间相关矩阵，检查是否符合预期（如短期/长期波动率应正相关且 >0.7）；
异常低/负相关需人工排查。
4. 流式 vs 批量计算一致性
✔️ 极其重要！生产环境往往是流式（online）推理，而训练是批量（offline）。两者必须数学等价。
🔍 测试建议：
用同一段历史数据，分别用 rolling(window).apply()（批量）和循环+状态缓存（流式）计算，逐点比对；
特别注意边界（前 N 行 NaN 处理）、EMA 初始化方式等细节。
5. lag 特征有效性应缓慢衰减，而非断崖下降
✔️ 非常敏锐的观察！如果 vol_lag_1 有用但 vol_lag_2 完全无效，可能是：
数据频率问题（如日频下 lag>1 信息衰减快属正常）；
但若在小时频下 lag_3 就断崖，可能特征计算有 bug（如错误 shift 方向）。
🔍 测试建议：
绘制 lag 特征与目标变量的互信息（MI）或偏自相关（PACF）曲线，应呈指数衰减；
若出现“lag_1 高相关，lag_2 接近 0”，需检查 shift(1) 是否误写成 shift(-1)（典型未来函数 bug）。

🧠 补充：其他关键测试维度（你可能没想到）
6. NaN / Inf / 极端值传播控制
特征计算中若某步出现 NaN（如除零、空窗口 std），应合理处理（如 forward-fill 或设为 0），不能污染后续所有样本。
✅ 测试：人为注入 NaN 到原始 price，检查特征是否局部受影响，而非全局崩溃。
7. 时间对齐（Timestamp Alignment）
在多资产或多频率数据中，确保所有特征基于相同的时间戳对齐（如都用 close 时间）。
❌ 常见 bug：用 15:00 的 ATR 和 16:00 的 return 计算 ratio → 隐蔽的未来函数。
8. 滚动窗口边界行为
rolling(window=20).std() 在前 19 行返回 NaN 是合理的，但有些实现会自动用 expanding 窗口填充，导致早期样本“信息过强”。
✅ 测试：检查前 N 行是否为 NaN，或是否明确使用 min_periods=N。
9. 特征单调性 / 符号一致性（Domain Logic Check）
例如：vol_range_10 = (high - low) / close 应 ≥0；
vol_atr_ratio_20 = atr / ma_price 应 >0；
若出现负值，说明公式或数据源有误。
10. 可复现性（Determinism）
相同输入必须产生相同输出（尤其涉及随机采样如 percentile_approx）。
✅ 测试：多次运行特征提取，结果应 bit-wise 一致（除非明确引入随机性）。
11. 计算效率与内存稳定性
向量化实现不应 OOM；流式实现不应随时间累积误差。
✅ 测试：用 10 年分钟级数据跑一遍，监控内存和耗时。
12. 跨平台/库一致性（可选但高级）
如用 pandas.rolling vs numba 手写 rolling，结果应一致（容忍浮点误差）。

✅ 总结：一个健壮的特征测试清单应包含

类别 测试项
------ -------
逻辑正确性 无未来函数、符号合理、边界处理正确
数值稳健性 无 NaN/Inf、极端值可控、浮点稳定
统计合理性 相关性符合预期、lag 衰减平滑、多尺度有效
工程一致性 流式=批量、可复现、高效
业务兼容性 多资产归一化、时间对齐、单位统一

import numpy as np

辅助函数：生成多资产模拟价格数据（带已知结构）
def make_mock_prices(n_days=200, n_assets=3, seed=42):
np.random.seed(seed)
dates = pd.date_range("2020-01-01", periods=n_days, freq="D")
assets = [f"asset_{i}" for i in range(n_assets)]

# 模拟带趋势+波动的价格（避免纯随机）
prices = {}
for asset in assets:
returns = np.random.normal(0.001, 0.02, n_days)
price = 100 np.exp(np.cumsum(returns))
prices[asset] = price

return pd.DataFrame(prices, index=dates)
辅助函数：流式计算（简化版，仅用于验证一致性）
def compute_vol_raw_streaming(prices, windows=[5, 10, 20, 60]):
"""逐行模拟流式计算 vol_raw（用 rolling std of log returns）"""
log_ret = np.log(prices / prices.shift(1))
result = pd.DataFrame(index=prices.index)
for w in windows:
# 注意：流式中前 w 行为 NaN
result[f"vol_raw_{w}"] = log_ret.rolling(window=w, min_periods=w).std()
return result

def test_feature_normalization_multi_asset():
"""1. 归一化验证：多资产下特征分布合理"""
prices = make_mock_prices(n_assets=5)
features = extract_extended_volatility_features(prices)

# 检查归一化特征范围
assert features["vol_zscore"].min() >= -5, "z-score 超出合理范围"
assert features["vol_zscore"].max() <= 5
assert features["vol_percentile_approx"].between(0, 1).all(), "percentile 不在 [0,1]"

# 多资产下 vol_raw 应有不同量级，但 zscore 应可比

def test_no_lookahead_bias():
"""2. 无未来数据：通过流式 vs 批量一致性验证"""

streaming_vol_raw = compute_vol_raw_streaming(prices)

# 比较关键特征（如 vol_raw_20）
col = "vol_raw_20"
if col in batch_features.columns and col in streaming_vol_raw.columns:
diff = (batch_features[col] - streaming_vol_raw[col]).abs()
# 允许浮点误差，但前 window 行应为 NaN
assert diff.dropna().max() < 1e-10, f"流式与批量计算不一致: {col}"

def test_feature_correlation_reasonableness():
"""3. 相关性合理性：短期与长期波动率应高度正相关"""
prices = make_mock_prices()
features = extract_extended_volatility_features(prices)

# 检查 vol_raw 系列相关性

# vol_raw_5 与 vol_raw_10 相关性应 >0.7

def test_lag_effect_decay_smoothly():
"""5. lag 特征有效性应缓慢衰减"""
prices = make_mock_prices()
features = extract_extended_volatility_features(prices)

# 构造简单目标（如未来5日收益波动）
log_ret = np.log(prices / prices.shift(1))
target = log_ret.rolling(5).std().shift(-5) # 注意：仅用于测试，实际不可用！

lag_cols = ["vol_lag_1", "vol_lag_2", "vol_lag_3"]
correlations = []
for col in lag_cols:
if col in features.columns:
corr = features[col].corr(target, min_periods=50)
correlations.append(corr)

# 转为绝对值（关注预测能力强度）
abs_corr = np.abs(correlations)
# 应大致递减，且无断崖（如 lag1=0.3, lag2=0.01）
if len(abs_corr) >= 2:
assert abs_corr[1] > 0.1 abs_corr[0], "lag_2 相关性断崖式下降"
if len(abs_corr) >= 3:
assert abs_corr[2] > 0.1 abs_corr[1], "lag_3 相关性断崖式下降"
