🔑 关键机制说明
✅ 1. 依赖解析（DAG + 拓扑排序）
resolve_dependencies() 收集所有直接/间接依赖。
使用 Kahn 算法进行拓扑排序，确保先算依赖项。
若存在循环依赖 → 报错。
✅ 2. 层级并行（Level-wise Parallelism）
analyze_dependency_levels() 将特征按“最长依赖链长度”分层。
同一层内无依赖 → 可安全并行。
层间串行 → 保证依赖顺序。
✅ 3. 双层缓存（Memory + Disk）
内存缓存：self.memory_cache[feature_name]，会话内极速复用。
磁盘缓存：cache/features/{md5_key}.pkl，跨会话持久化。
缓存键包含：特征名、DataFrame 哈希、参数、output_columns、代码版本。
✅ 4. 灵活函数调用
通过 feature_function_mapping.py 映射字符串 → 实际函数。
_build_call_args() 支持：
自动传入 df
从 column_mappings 注入指定列为函数参数（如 high=df['high']）
✅ 5. 输出列控制
配置中定义 output_columns（如 MACD → [macd, signal, histogram]）
最终只保留原始列 + 明确声明的输出列，避免污染。

📌 示例：加载 sr_reversal 策略

python
loader = StrategyFeatureLoader()
df_out = loader.load_strategy_features(df, "sr_reversal", fit=True)

内部会：
1. 读取 strategy_features.yaml → 发现需要 ["sr_strength_max", "sqs_hal_high", "sqs_hal_low"]
2. 从 feature_dependencies.yaml 解析出这些特征还依赖 hal_high, hal_low, atr 等
3. 按层级并行计算：先算 atr → 再算 hal_ → 最后算 sqs_ 和 sr_strength_max
4. 结果自动缓存，下次相同输入直接复用
