# 一、框架概览（一句话）

用小波把价格与订单流分解到多尺度，计算每个尺度的**能量 / 动量 / 相位**等量度，将结构（低频）、压缩（中频）与订单流（高频）统一映射到同一“能量空间”，再做归一化与融合，生成多因子信号用于入场/出场/风控。

# 二、核心思想（步骤）

1. 对原始序列（价格、成交量、CVD、imbalance 等）做 DWT（或 CWT）分解，得到各尺度系数：`A_n`（近似）、`D_n...D1`（细节）。
2. 在每个尺度上计算能量 / 波动度 / 瞬时偏移：例如 `E_j(t) = Σ_b |W_j(b,t)|^2` 或简单的滑窗能量 `sum(D_j^2)`。
3. 将每个尺度的能量映射为因子：低频 → 结构因子；中频 → 压缩因子（压缩度 = 高频能量/低频能量）；高频 → 订单流因子。
4. 对各因子做时间上/历史分位归一化（z-score、历史分位、t-digest），并通过 regime（波动/趋势/震荡）动态加权。
5. 用规则（或 ML）融合因子生成交易信号 + 风控条件。

# 三、数学与计算定义（关键公式）

* 小波系数（离散尺度）：
  [
  W_{j,k} = \langle x, \psi_{j,k} \rangle
  ]
* 某尺度即时能量（滑窗长度 (L)）：
  [
  E_j(t) = \sum_{i=t-L+1}^{t} W_{j,i}^2
  ]
* 总能量与高/低频比（压缩度）：
  [
  E_{\text{total}}(t) = \sum_{j=1}^{J} E_j(t) + E_A(t)
  ]
  [
  \text{CompressionScore}(t) = \frac{\sum_{j=1}^{j_h} E_j(t)}{\sum_{j=j_h+1}^{J} E_j(t) + \epsilon}
  ]
  （其中 `j_h` 为划分高频的尺度阈值）
* 结构强度（低频趋势）：
  [
  S(t) = \frac{E_A(t)}{E_{\text{total}}(t)}
  ]
* 订单流动能（高频偏向）：
  [
  O(t) = \frac{\sum_{j=1}^{j_o} \text{sign}(W_j) \cdot |W_j|}{\sum_{j=1}^{j_o} |W_j|} \quad\text{或}\quad O_{energy}(t)=\sum_{j=1}^{j_o} E_j(t)
  ]

# 四、因子清单（建议实现的基本因子）

* `StructureTrend`: 低频近似能量比（长期趋势强度）
* `CompressionScore`: 高频能量占比（低 → 压缩，高 → 扩张）
* `HighFreqBurst`: 高频能量突增（事件触发因子）
* `OrderFlowBias`: 在高频尺度上，买卖不平衡的带符号能量
* `WaveletMomentum_j`: 各尺度细节系数的滑动平均（多尺度动量）
* `PhaseShift`: 低频相位或斜率用于确认结构反转

# 五、归一化与历史适应

* 对每个原始因子使用**历史分位（推荐 t-digest / rolling quantiles）**或 **rolling z-score** 归一化为 [-1,1] 或 0-1。
* 选择窗口长度要注意：短窗口适合高频因子，长窗口适合低频因子。比如 `L_high = 64`，`L_mid = 256`，`L_low = 2048`（根据数据频率调整）。
* 用分位映射可以避免 z-score 在非高斯分布下失灵。

# 六、因子融合（几种方式）

1. **规则加权（简单、透明）**

   ```
   SignalScore = w1 * StructureTrend_norm + w2 * (1 - CompressionScore_norm) + w3 * OrderFlowBias_norm
   ```

   * 例如在趋势市场提升 `w1`，在震荡市场提升 `w2`。

2. **贝叶斯/概率融合**

   * 把每个因子看作对未来方向的证据，计算合成的概率分布（log-odds 相加）。

3. **机器学习（回归/分类）**

   * 用过去样本训练（features = 各尺度能量 + 交叉项），目标可以是短期收益或方向。需要稳健正则化与时间序列交叉验证（walk-forward）。

4. **决策树 / RL**

   * 若要实现动态权重，可用线上学习或强化学习（需大量数据与谨慎的归因/风控）。

# 七、交易逻辑原型（规则流）

* 预条件：满足流动性 & 资金可用性
* 趋势确认：`StructureTrend_norm > T1`
* 压缩释放：`CompressionScore_norm < T2` 且随后 `HighFreqBurst > T3`
* 订单流支持：`OrderFlowBias_norm` 与趋势方向一致
* 开仓：当 三项同时满足 → 按信号方向入场（按权重分配仓位）
* 平仓：目标盈亏或当 `StructureTrend` 反向或 `HighFreqBurst` 表示相反动能
* 止损：根据多尺度 ATR（低频 ATR for max drawdown， 高频 ATR for intraday stop）

```python
    return coeffs

def scale_energy(coeff, window):
    # coeff: 1D array, compute rolling energy
    sq = np.square(coeff)
    return pd.Series(sq).rolling(window).sum().to_numpy()

def compute_multiscale_factors(price, volume, wavelet='db4', level=6, windows=dict(high=64, mid=256, low=1024)):
    coeffs = wavelet_decompose(price, wavelet=wavelet, level=level)
    A = coeffs[0]
    Ds = coeffs[1:]  # list: D_n ... D1
    # Upsample energies back to price length if needed (use pywt.upcoef for precise align) or compute energies over coeff indexes
    # Simpler: compute energy on reconstructed bandpass signals
    energies = []
    for j, d in enumerate(Ds[::-1], start=1):  # D1 first
        band = pywt.upcoef('d', d, wavelet, level=j, take=len(price))
        energies.append(scale_energy(band, windows['high'] if j<=2 else windows['mid']))
    A_rec = pywt.upcoef('a', A, wavelet, level=level, take=len(price))
    E_low = scale_energy(A_rec, windows['low'])
    E_total = np.sum(energies, axis=0) + E_low + 1e-9
    compression = np.sum(energies[:2], axis=0) / (E_low + 1e-9)  # example
    structure = E_low / E_total
    orderflow = None  # 用成交量或CVD的同尺度高频能量替代
    return dict(structure=structure, compression=compression, energies=energies)
```

> 注：上面是概念代码。工程化时建议使用 `pywt.swt`（stationary wavelet transform）或精确的上采样/下采样对齐重构。

# 十、回测与评估要点

* **脆弱性测试**：在不同市场 regime（高波动、低波动、单边）检验权重自适应策略的稳健性。

## 🎯 一、核心概念回顾

小波变换 = **一种可局部化的傅里叶变换**。
它能同时告诉你：

* “什么时候”（time domain）
* “在哪个频率范围”（frequency domain）
* “能量有多强”（amplitude / power）

即：

> 小波变换通过不同尺度（scale）的基函数，把原始信号分解成多层次的“局部波动结构”。

## ⚙️ 二、优点（为什么金融交易喜欢它）

| 优点               | 解释                       | 在交易中的意义                |
| ---------------- | ------------------------ | ---------------------- |
| 🧩 **多尺度分析能力**   | 同时捕捉趋势（低频）与波动（高频）        | 可识别不同周期的趋势、震荡、噪音       |
| 🎯 **时频局部化**     | 信号变化在哪一时刻发生都能被识别         | 比傅里叶更适合处理“突发事件”和“波动爆发” |
| 🪶 **抗噪能力强**     | 可以选择性地忽略高频噪声             | 滤掉市场的微结构噪声，只保留主要波动     |
| 🔍 **趋势与波动自然分离** | 小波低频 = 趋势，高频 = 波动        | 直接实现趋势过滤、波段分解          |
| 🧠 **适合压缩检测**    | 高频能量低 → 市场压缩，高频能量爆发 → 扩展 | 天然适配“压缩/扩张”信号逻辑        |
| 🧮 **数学基础完备**    | 完整的能量守恒和重构性              | 可以安全地做因子提取、去噪、平滑，而不丢信息 |
| 🧰 **易与机器学习融合**  | 小波系数是非常好的特征向量            | 适合输入神经网络或强化学习系统        |

## ⚠️ 三、缺点（为什么很多量化团队不用它）

| 缺点               | 原因                                      | 实际影响与对策                        |
| ---------------- | --------------------------------------- | ------------------------------ |
| 🧠 **参数选择复杂**    | 需要选母小波（如 Haar, Daubechies, Morlet）+ 尺度数 | 不同选择 → 不同结果；需调优或自动选择           |
| ⚙️ **边界效应问题**    | 在信号起始/结束处，小波不完全匹配                       | 高频段失真或“边界伪能量”；需加 padding 或镜像延拓 |
| ⏱ **计算成本高**      | 连续小波变换 (CWT) 的复杂度较高                     | 高频实盘场景中需降采样或使用离散小波 (DWT)       |
| 📉 **解释性弱**      | 系数代表“尺度能量”，不直接等价价格变化                    | 对交易决策需要额外转换或可视化                |
| 🔄 **过度分解风险**    | 多尺度过多 → 噪音反而放大                          | 实际上常用 3–6 层足够，防止过拟合            |
| 🧩 **非线性市场适配有限** | 小波是线性算子                                 | 对非线性跳跃、制度性断点、极端事件刻画不完美         |
| 🧪 **调参过度风险**    | 母小波、尺度、阈值调得太细 → 拟合历史                    | 要注意稳健性和泛化能力                    |
