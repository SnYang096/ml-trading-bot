# 验证特征相关性：区分真实 Alpha 和数据泄漏

## 概述

`scripts/verify_feature_correlation.py` 脚本用于验证特征与未来收益的相关性，帮助区分：
- **真实 Alpha**：特征包含真实可交易的预测信号
- **数据泄漏**：特征包含未来信息，导致虚假高相关性

## 两个关键实验

### 实验 1：滞后测试（Lag Correlation Test）

**目的**：检查特征滞后 1-2 根后相关性是否缓慢衰减（真实信号）还是急剧下降（对齐问题）

**原理**：
- 如果特征已经正确 `shift(1)`，那么再滞后 1-2 根，相关性应该**缓慢衰减**
- 如果相关性**急剧下降**，可能是时间对齐问题，需要检查 `shift(1)` 是否正确应用

**示例结果**：
```
特征                          Lag 0      Lag 1      Lag 2      衰减模式
--------------------------------------------------------------------------------
atr_zscore_w288              0.1535     0.1518     0.1500     缓慢衰减 ✅
atr_percentile               0.0944     0.0944     0.0943     缓慢衰减 ✅
```

### 实验 2：随机打乱测试（Shuffled Correlation Test）

**目的**：检查特征是否只对真实行情有相关性，对随机数据无相关性（证明不是虚假相关）

**原理**：
- 将 `future_return` 随机打乱，检查特征是否仍与之相关
- 如果相关性降至 ~0，说明特征只对真实行情有相关性，不是虚假相关
- 如果相关性仍高，可能是数据泄漏或虚假相关

**示例结果**：
```
特征                         真实相关性    打乱后相关性      衰减比      结论
--------------------------------------------------------------------------------
atr_zscore_w288              0.1535        0.0029        1.89%     真实信号 ✅
atr_percentile               0.0944        0.0009        0.95%     真实信号 ✅
```

## 使用方法

### 方法 1：使用 Makefile（推荐）

```bash
# 使用默认配置
make verify-feature-correlation

# 指定特征列表文件（JSON 或 YAML）
make verify-feature-correlation VERIFY_TOP_FACTORS=custom/leakage_shift_features.json

# 或使用 YAML 配置
make verify-feature-correlation VERIFY_TOP_FACTORS=configs/tests/leakage_shift_test.yaml

# 指定交易符号和时间范围
make verify-feature-correlation \
  RANK_IC_SYMBOL=BTCUSDT \
  FEATURE_EVAL_START_DATE=2023-01-01 \
  FEATURE_EVAL_END_DATE=2025-01-01
```

### 方法 2：直接运行 Python 脚本

```bash
PYTHONPATH=src python3 scripts/verify_feature_correlation.py \
  --data-path data/parquet_data \
  --symbol BTCUSDT \
  --start-date 2023-01-01 \
  --end-date 2025-01-01 \
  --timeframe 240T \
  --top-factors custom/leakage_shift_features.json \
  --horizon 24
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--data-path` | 数据路径 | `data/parquet_data` |
| `--symbol` | 交易符号 | `BTCUSDT` |
| `--start-date` | 开始日期 | `2023-01-01` |
| `--end-date` | 结束日期 | `2025-01-01` |
| `--timeframe` | 时间周期 | `240T` |
| `--top-factors` | 特征列表文件（JSON 或 YAML） | `custom/leakage_shift_features.json` |
| `--horizon` | 预测周期 | `24` |
| `--test-size` | 测试集比例（用于分割数据） | `0.15` |

## 结果解读

### ✅ 通过验证（真实 Alpha）

如果两个实验都通过：
- **滞后测试**：相关性缓慢衰减 → 真实信号
- **随机打乱**：相关性降至 ~0 → 不是虚假相关

**结论**：特征包含真实 Alpha，不是数据泄漏

### ⚠️ 未通过验证（需要检查）

如果实验未通过：
- **滞后测试**：相关性急剧下降 → 检查 `shift(1)` 是否正确应用
- **随机打乱**：相关性仍高 → 可能存在数据泄漏或虚假相关

**建议**：进一步检查特征计算逻辑

## 注意事项

1. **数据量要求**：每个特征至少需要 100 个有效样本才能进行相关性计算
2. **随机种子**：随机打乱测试使用固定随机种子（42），确保结果可复现
3. **训练集验证**：验证使用训练集数据，避免测试集污染
4. **YAML 支持**：特征列表文件支持 JSON 和 YAML 两种格式
