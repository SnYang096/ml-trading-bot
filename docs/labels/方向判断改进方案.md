# 方向判断改进方案

## 🔍 当前问题分析

### 当前方法（Quantile 方法）

**逻辑**：
1. 计算预测值的分位数（pred_quantile）：预测值在最近 30 个样本中的排名（0-1）
2. 计算置信度（confidence_score）：`|pred_quantile - 0.5| * 2`
3. 生成信号：
   - **Long**: `pred_quantile >= 0.9` 且 `confidence >= 0.85`
   - **Short**: `pred_quantile <= 0.1` 且 `confidence >= 0.85`
   - **Hold**: 其他情况

**问题**：
- ❌ **Win Rate 只有 13.5%**（极低，接近随机）
- ❌ **虽然 IC 高（0.2177），但交易表现差**
- ❌ **阈值太极端**（0.9/0.1），导致信号太少
- ❌ **没有直接使用预测值的符号**，可能丢失方向信息

### 问题根源

1. **排序正确但预测值不准确**
   - IC 高说明模型能正确排序（高收益的排在前面）
   - 但预测值本身可能不准确（符号可能错误，或幅度不对）

2. **分位数方法的问题**
   - 只关注预测值的相对排名，不关注绝对值
   - 如果预测值分布与真实收益分布不匹配，方向判断会出错

3. **阈值太极端**
   - 只有 `pred_quantile >= 0.9` 才做多，`<= 0.1` 才做空
   - 这意味着只有最极端的 10% 预测值才会产生信号
   - 可能错过了很多有效的交易机会

## ✅ 改进方案

### 方案 1：直接使用预测值符号（Sign 方法）

**逻辑**：
- 如果 `pred > 0` → Long
- 如果 `pred < 0` → Short
- 仍然使用置信度过滤（`confidence >= 0.85`）

**优点**：
- ✅ 简单直接，不依赖分位数
- ✅ 直接使用预测值的符号，更符合直觉
- ✅ 适用于预测值本身有方向性的情况

**使用方法**：
```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=sign
```

### 方案 2：结合预测值符号和分位数（Hybrid 方法）

**逻辑**：
- 只有当**预测值符号和分位数方向一致**时才交易
- Long: `pred > 0` 且 `pred_quantile >= 0.9` 且 `confidence >= 0.85`
- Short: `pred < 0` 且 `pred_quantile <= 0.1` 且 `confidence >= 0.85`

**优点**：
- ✅ 双重验证，减少误信号
- ✅ 结合了两种方法的优点
- ✅ 更保守，但更可靠

**使用方法**：
```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=hybrid
```

### 方案 3：基于历史表现优化阈值（Optimized 方法）

**逻辑**：
- 在训练数据上优化预测值阈值，最大化方向准确率
- 使用优化后的阈值生成信号

**优点**：
- ✅ 自动优化，适应数据分布
- ✅ 基于历史表现，更可靠
- ✅ 不需要手动调整阈值

**使用方法**：
```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=optimized
```

### 方案 4：校准预测值

**逻辑**：
- 使用 sigmoid 或 Platt scaling 校准预测值
- 使预测值分布与真实收益分布匹配

**优点**：
- ✅ 使预测值更准确地反映真实收益
- ✅ 可以改善方向判断的准确性

**使用方法**：
```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  CALIBRATE_PREDICTIONS=1
```

## 📊 推荐使用顺序

### 第一步：尝试 Sign 方法（最简单）

```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=sign
```

**预期改善**：
- Win Rate 应该从 13.5% 提升到 40-50%
- 如果预测值符号本身准确，这个方法应该最有效

### 第二步：如果 Sign 方法不够好，尝试 Hybrid 方法

```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=hybrid
```

**预期改善**：
- 更保守，但可能更可靠
- 如果 Sign 方法产生太多误信号，Hybrid 可以过滤掉

### 第三步：如果还是不够好，尝试 Optimized 方法

```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=optimized
```

**预期改善**：
- 自动优化阈值，适应数据分布
- 应该能获得最佳的方向准确率

### 第四步：如果预测值分布不匹配，尝试校准

```bash
make ts-r-rank-ic-train \
  RANK_IC_TOP_FACTORS=results/feature_evaluation/top_factors.json \
  SIGNAL_METHOD=sign \
  CALIBRATE_PREDICTIONS=1
```

## 🔧 在 Makefile 中添加支持

需要在 Makefile 中添加参数支持：

```makefile
RANK_IC_SIGNAL_METHOD ?= quantile
RANK_IC_CALIBRATE_PREDICTIONS ?= 0
```

然后在 `ts-r-rank-ic-train` 命令中添加：
```makefile
$(if $(filter 1 true yes,$(RANK_IC_CALIBRATE_PREDICTIONS)),--calibrate-predictions,) \
--signal-method $(RANK_IC_SIGNAL_METHOD) \
```
