# 深度学习序列特征归一化改进总结

## 问题分析

通过深入分析 `DeepLearningSequenceExtractor` 的归一化实现，发现了以下关键问题：

### 1. 前世偏差问题
- **问题描述**: 当前实现使用全局归一化，在训练阶段使用整个数据集的统计量，在预测阶段使用相同的全局统计量
- **具体表现**: 训练数据和测试数据的统计量存在显著差异，导致归一化后的特征分布不一致
- **影响**: 可能导致模型在预测阶段性能下降，引入前世偏差

### 2. 时间序列非平稳性
- **问题描述**: 金融时间序列数据具有非平稳性，数据分布会随时间变化
- **具体表现**: 不同时间窗口内的统计量差异较大
- **影响**: 全局归一化无法适应数据分布的变化

### 3. 归一化策略单一
- **问题描述**: 只提供一种归一化方法（全局标准化）
- **具体表现**: 无法根据不同场景选择最适合的归一化策略
- **影响**: 限制了模型在不同数据条件下的适应性

## 改进方案

### 1. 多种归一化策略

#### 1.1 全局归一化 (Global)
- **原理**: 使用整个数据集的均值和标准差进行归一化
- **优点**: 计算简单，保证特征分布一致性
- **缺点**: 存在前世偏差，无法适应数据分布变化
- **适用场景**: 数据分布相对稳定的场景

#### 1.2 滚动窗口归一化 (Rolling)
- **原理**: 使用滑动窗口内的统计量进行归一化
- **优点**: 避免前世偏差，适应数据分布变化
- **缺点**: 计算复杂度增加，可能引入噪声
- **适用场景**: 非平稳时间序列，数据分布变化较大

#### 1.3 指数移动平均归一化 (EMA)
- **原理**: 使用指数移动平均的统计量进行归一化
- **优点**: 平衡历史信息和当前信息，计算效率高
- **缺点**: 需要调优平滑参数
- **适用场景**: 平稳时间序列，需要平滑的归一化

#### 1.4 自适应归一化 (Adaptive)
- **原理**: 结合全局和局部统计量，使用加权平均
- **优点**: 平衡稳定性和适应性，减少前世偏差
- **缺点**: 需要调优权重参数
- **适用场景**: 生产环境，需要平衡性能和准确性

## 使用建议

### 1. 归一化方法选择

#### 训练阶段
- **推荐**: 使用 `adaptive` 或 `rolling` 归一化
- **原因**: 避免前世偏差，适应数据分布变化

#### 预测阶段
- **推荐**: 使用与训练阶段相同的归一化方法
- **原因**: 保证特征分布一致性

#### 生产环境
- **推荐**: 使用 `adaptive` 归一化
- **原因**: 平衡性能和准确性，适应数据变化

### 2. 参数调优

#### EMA平滑参数
```python
# 默认值: 0.01
# 调整范围: 0.001 - 0.1
# 较小值: 更平滑，响应较慢
# 较大值: 更敏感，响应较快
```

#### 自适应权重
```python
# 全局权重: 0.3 (默认)
# 局部权重: 0.7 (默认)
# 调整原则: 数据变化大时增加局部权重
```

### 3. 特征质量监控

#### 监控指标
- 特征均值范围
- 特征标准差范围
- 特征分布稳定性
- 归一化效果

#### 异常检测

- 计算错误或数值不稳定
