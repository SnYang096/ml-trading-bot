## 设计与实践建议（要点）

### 1) 为什么用 q50 替代均值回归

- q50（分位数 0.5）优化中位数（MAE 家族），对异常值鲁棒；与 q10/q90 同框架，逻辑一致
- 作为“点估计”更稳健：`q50 > 0` 比 MSE 回归的正负更可靠
- 可用一致的可信度定义：`credibility = |q50| / (q90 - q10)`

### 2) 波动率：volatility_model vs ATR（建议融合）

- volatility_model 预测未来，具备前瞻性；ATR 总结过去，稳健可解释
- 实战推荐融合：
  - 保守上界：`final_vol = max(ŷ_vol, ATR_N)`
  - 可信度加权：`final_vol = cred * ŷ_vol + (1 - cred) * ATR_N`
- 策略条件可用：`|q50| > k * final_vol 且 credibility > c`；仓位 `∝ |q50| / final_vol`

### 3) 特征工程/特征选择：统一先行，后期拆分

- 起步：统一特征集（comprehensive / default 等），快速对齐管线
- 追求性能：为不同任务（q50 / vol / q10/90）分别挑选 TopK 特征
- 自动化流程（建议内置到研究脚本）：
  1. 相关性去冗余（|corr| > 0.95）
  2. 训练轻量 LGBM，按 `gain` 排序出重要性
  3. 网格搜索 `TopK`（如 30/50/80）取 OOS 最优
  4. 用选出的特征重训正式模型，写入 artifacts
  5. 月度滚动重算重要性，应对特征漂移

## 使用示例

- 全量特征单次训练（默认 comprehensive）：
```
make train SYMBOL=BTCUSDT
```

- compare-dim 精选特征训练：
```
make train SYMBOL=BTCUSDT \
  TRAIN_USE_TOP_FACTORS=results/.../top_factors.json
```

- 指定 TopK + 排名文件：
```
make train SYMBOL=BTCUSDT \
  TRAIN_TOPK=80 TRAIN_TOPK_SOURCE=results/.../feature_scores.csv
```

- 滚动训练（单 tf/fb）：
```
make rolling SYMBOL=BTCUSDT ROLLING_FREQ=5T ROLLING_FBS=3
```

- 滚动训练 + 独立特征选择（TopK）：
```
make rolling SYMBOL=BTCUSDT \
  ROLLING_TOPK=60 ROLLING_TOPK_SOURCE=results/.../feature_scores.json
```
