# 实盘特征加载流程分析

## 一、当前 YAML 特征加载流程概述

### 1.1 核心组件

- **StrategyFeatureLoader**: 基于 YAML 配置的特征加载器
- **FeatureComputer**: 并行特征计算器（支持内存+磁盘缓存）
- **StrategyConfigLoader**: 策略配置加载器（目录结构）

### 1.2 当前流程

```
1. 初始化 StrategyFeatureLoader
   ├─ 加载 feature_dependencies.yaml（特征依赖定义）
   ├─ 加载策略 features.yaml（策略所需特征列表）
   └─ 初始化 FeatureComputer（并行计算+缓存）

2. 调用 load_features_from_requested()
   ├─ 解析特征依赖（拓扑排序）
   ├─ 按依赖层级并行计算
   ├─ 缓存检查（内存 → 磁盘）
   └─ 返回包含特征的 DataFrame

3. fit 参数控制
   ├─ fit=True: 训练阶段（需要拟合参数，如归一化、分位数等）
   └─ fit=False: 实盘阶段（使用已保存的参数）
```

## 二、实时流场景分析

### 2.1 实时流特点

1. **数据流式到达**：逐条或小批量（如每1分钟1条K线）
2. **低延迟要求**：需要快速计算特征并生成信号
3. **增量计算**：不能等待完整数据集
4. **状态维护**：某些特征需要历史窗口（如MA、ATR等）

### 2.2 当前流程在实时流中的问题

#### ❌ 问题1：并行计算不适合实时流

**现状**：
- `FeatureComputer` 使用 `ProcessPoolExecutor` 或 `ThreadPoolExecutor`
- 适合批量处理完整数据集
- 并行开销 > 单条数据计算成本

**影响**：
- 实时流中单条数据计算，并行化反而增加延迟
- 进程/线程创建开销大

**解决方案**：
- 实时流中禁用并行计算（`max_workers=1` 或直接串行）
- 或提供专门的实时流计算模式

#### ❌ 问题2：缓存机制需要适配

**现状**：
- 缓存键基于整个 DataFrame 的哈希值
- 实时流中每次都是新数据，缓存命中率低

**影响**：
- 缓存失效，无法复用计算结果
- 但某些中间特征（如ATR、MA）可以缓存

**解决方案**：
- 实现增量缓存：缓存中间特征的计算结果
- 使用滑动窗口缓存：维护最近N条数据的特征缓存

#### ❌ 问题3：历史窗口特征的状态管理

**现状**：
- 特征计算函数是无状态的
- 每次调用都从 DataFrame 中提取历史数据

**影响**：
- 实时流中需要维护历史数据窗口
- 某些特征（如WPT、Hilbert）需要较长历史窗口

**解决方案**：
- 实现状态管理器：维护每个特征所需的历史窗口
- 增量更新：只计算新数据，复用历史计算结果

#### ❌ 问题4：fit 参数在实时流中的使用

**现状**：
- 训练时 `fit=True`，实盘时 `fit=False`
- 但实时流中需要确保始终使用 `fit=False`

**影响**：
- 如果误用 `fit=True`，会导致参数漂移
- 需要明确区分训练/实盘模式

**解决方案**：
- 实时流中强制 `fit=False`
- 提供专门的实时流接口，隐藏 fit 参数

#### ⚠️ 问题5：特征依赖解析的开销

**现状**：
- 每次调用都会解析依赖关系（拓扑排序）
- 依赖关系在运行时是固定的

**影响**：
- 实时流中重复解析依赖，增加延迟

**解决方案**：
- 预编译依赖图：在初始化时解析一次，缓存结果
- 实时流中直接使用预编译的依赖顺序

### 3.3 方案3：状态管理器实现

```python
class FeatureStateManager:
    """
    特征状态管理器
    
    维护：
    1. 历史数据窗口（滑动窗口）
    2. 中间特征状态（如归一化参数、分位数等）
    3. 特征计算历史（用于增量计算）
    """
    
    def __init__(self, window_size: int = 1000):
        self.window_size = window_size
        self.history_df = pd.DataFrame()
        self.feature_states = {}  # 每个特征的状态
    
    def append(self, new_data: pd.DataFrame):
        """追加新数据，维护滑动窗口"""
        self.history_df = pd.concat([self.history_df, new_data])
        if len(self.history_df) > self.window_size:
            # 保留最近 window_size 条
            self.history_df = self.history_df.tail(self.window_size)
    
    def get_window(self) -> pd.DataFrame:
        """获取当前历史窗口"""
        return self.history_df.copy()
    
    def update_feature_state(self, feature_name: str, state: Any):
        """更新特征状态（如归一化参数）"""
        self.feature_states[feature_name] = state
    
    def get_feature_state(self, feature_name: str) -> Optional[Any]:
        """获取特征状态"""
        return self.feature_states.get(feature_name)
```

### 3.4 方案4：增量缓存实现

```python
class IncrementalFeatureCache:
    """
    增量特征缓存
    
    缓存策略：
    1. 缓存中间特征的计算结果（如ATR、MA等）
    2. 使用时间戳+特征名作为缓存键
    3. 自动清理过期缓存
    """
    
    def __init__(self, max_size: int = 10000):
        self.cache = {}
        self.max_size = max_size
    
    def get(self, feature_name: str, timestamp: pd.Timestamp) -> Optional[pd.Series]:
        """获取缓存的特征值"""
        key = (feature_name, timestamp)
        return self.cache.get(key)
    
    def set(self, feature_name: str, timestamp: pd.Timestamp, value: pd.Series):
        """设置缓存"""
        key = (feature_name, timestamp)
        self.cache[key] = value
        
        # 清理过期缓存
        if len(self.cache) > self.max_size:
            self._evict_oldest()
    
    def _evict_oldest(self):
        """清理最旧的缓存"""
        # 按时间戳排序，删除最旧的
        sorted_keys = sorted(self.cache.keys(), key=lambda x: x[1])
        for key in sorted_keys[:len(sorted_keys) // 2]:
            del self.cache[key]
```

## 六、与 Nautilus Trader 集成

### 6.1 Nautilus Trader 架构概述

[Nautilus Trader](https://github.com/nautechsystems/nautilus_trader) 是一个高性能的算法交易平台，采用事件驱动架构：

- **事件驱动**：通过 `on_bar()`, `on_tick()`, `on_data()` 等事件处理器响应市场数据
- **策略类**：继承 `Strategy` 基类，实现交易逻辑
- **数据流**：通过 `DataClient` 接收实时市场数据（Bar, QuoteTick, TradeTick）
- **状态管理**：内置缓存和状态管理机制

### 6.2 集成方案设计

#### 6.2.1 核心思路

在 Nautilus Trader 的 `Strategy` 类中集成特征加载器：

1. **初始化阶段**：在 `on_start()` 中初始化特征加载器和状态管理器
2. **数据接收**：在 `on_bar()` 中接收新的 K 线数据
3. **特征计算**：将 Bar 转换为 DataFrame，计算特征
4. **信号生成**：使用计算好的特征生成交易信号
5. **订单执行**：通过 Nautilus Trader 的订单接口执行交易

#### 6.2.2 数据转换流程

```
Nautilus Bar → DataFrame → 特征计算 → 特征 DataFrame → 信号生成 → 订单执行
```

#### 6.3.1 Nautilus Strategy 集成类

```python
from nautilus_trader.model import Bar
from nautilus_trader.trading import Strategy
from nautilus_trader.model import InstrumentId
from nautilus_trader.model import BarType
from nautilus_trader.model import OrderSide

import pandas as pd

from src.features.loader.strategy_feature_loader import StrategyFeatureLoader
from src.strategy_config import StrategyConfigLoader
from src.time_series_model.live.realtime_feature_integration_example import (
    RealtimeFeatureManager

class NautilusStrategyWithFeatures(Strategy):
    """
    集成特征加载的 Nautilus Trader Strategy
    
    特点：
    1. 在 on_start() 中初始化特征管理器
    2. 在 on_bar() 中计算特征并生成信号
    3. 使用训练好的模型进行预测
    """
    
    def __init__(
        self,
        strategy_name: str,
        instrument_id: InstrumentId,
        bar_type: BarType,
        trade_size: float,
        config_base_path: str = "config/strategies",
        history_window: int = 1000,
    ):
        super().__init__()
        self.strategy_name = strategy_name
        self.instrument_id = instrument_id
        self.bar_type = bar_type
        self.trade_size = trade_size
        
        # 特征管理器（在 on_start() 中初始化）
        self.feature_manager: Optional[RealtimeFeatureManager] = None
        
        # 策略配置（在 on_start() 中加载）
        self.strategy_config = None
        
        # 模型（在 on_start() 中加载）
        self.model = None
        
        # 配置路径
        self.config_base_path = config_base_path
        self.history_window = history_window
    
    def on_start(self) -> None:
        """策略启动时初始化"""
        self.log.info(f"🚀 Starting {self.strategy_name} strategy")
        
        # 1. 加载策略配置
        config_path = Path(self.config_base_path) / self.strategy_name
        config_loader = StrategyConfigLoader(config_path)
        self.strategy_config = config_loader.load()
        
        # 2. 初始化特征管理器
        self.feature_manager = RealtimeFeatureManager(
            strategy_name=self.strategy_name,
            history_window=self.history_window,
            config_base_path=self.config_base_path,
        )
        
        # 3. 加载训练好的模型
        # TODO: 从模型文件加载
        # self.model = load_model(f"models/{self.strategy_name}/model.pkl")
        
        # 4. 订阅市场数据
        self.subscribe_bars(self.bar_type)
        
        self.log.info("✅ Strategy initialized")
    
    def on_bar(self, bar: Bar) -> None:
        """
        处理新的 K 线数据
        
        Args:
            bar: Nautilus Trader Bar 对象
        """
        try:
            # 1. 将 Bar 转换为 DataFrame
            new_bar_df = self._bar_to_dataframe(bar)
            
            # 2. 计算特征
            features_df = self.feature_manager.compute_features(new_bar_df)
            
            # 3. 获取最新特征（用于信号生成）
            latest_features = self.feature_manager.get_latest_features()
            
            if latest_features is None or len(latest_features) == 0:
                self.log.warning("⚠️ No features available yet")
                return
            
            # 4. 生成交易信号
            signal = self._generate_signal(latest_features)
            
            # 5. 执行交易
            if signal:
                self._execute_trade(signal, bar)
        
        except Exception as e:
            self.log.error(f"❌ Error processing bar: {e}")
            import traceback
            self.log.error(traceback.format_exc())
    
    def _bar_to_dataframe(self, bar: Bar) -> pd.DataFrame:
        """
        将 Nautilus Trader Bar 转换为 DataFrame
        
        Args:
            bar: Nautilus Trader Bar 对象
        
        Returns:
            DataFrame with columns: timestamp, open, high, low, close, volume
        """
        return pd.DataFrame({
            "timestamp": [bar.ts_event],
            "datetime": [pd.Timestamp.fromtimestamp(bar.ts_event / 1e9)],
            "open": [float(bar.open)],
            "high": [float(bar.high)],
            "low": [float(bar.low)],
            "close": [float(bar.close)],
            "volume": [float(bar.volume)],
            "symbol": [str(self.instrument_id)],
        })
    
    def _generate_signal(self, features_df: pd.DataFrame) -> Optional[Dict[str, Any]]:
        """
        根据特征生成交易信号
        
        Args:
            features_df: 包含特征的 DataFrame（单行）
        
        Returns:
            交易信号字典，包含 side, quantity, price 等信息
        """
        if self.model is None:
            # TODO: 如果没有模型，可以使用规则逻辑
            return None
        
        try:
            # 1. 提取特征列
            feature_cols = self.feature_manager.get_feature_columns()
            X = features_df[feature_cols].values
            
            # 2. 模型预测
            prediction = self.model.predict(X)[0]
            probability = self.model.predict_proba(X)[0] if hasattr(self.model, 'predict_proba') else None
            
            # 3. 生成信号
            signal = {
                "prediction": prediction,
                "probability": probability,
                "features": features_df.iloc[0].to_dict(),
            }
            
            # 4. 根据预测结果决定交易方向
            if prediction == 1:  # 买入信号
                signal["side"] = OrderSide.BUY
            elif prediction == -1:  # 卖出信号
                signal["side"] = OrderSide.SELL
            else:
                return None  # 无信号
            
            return signal
        
        except Exception as e:
            self.log.error(f"❌ Error generating signal: {e}")
            return None
    
    def _execute_trade(self, signal: Dict[str, Any], bar: Bar) -> None:
        """
        执行交易
        
        Args:
            signal: 交易信号
            bar: 当前 K 线数据
        """
        try:
            side = signal["side"]
            quantity = self.instrument.make_qty(self.trade_size)
            
            # 创建市价单
            order = self.order_factory.market(
                instrument_id=self.instrument_id,
                order_side=side,
                quantity=quantity,
            )
            
            # 提交订单
            self.submit_order(order)
            
            self.log.info(
                f"📊 Signal: {side} {quantity} @ {bar.close} "
                f"(prediction: {signal.get('prediction')}, "
                f"prob: {signal.get('probability')})"
            )
        
        except Exception as e:
            self.log.error(f"❌ Error executing trade: {e}")
    
    def on_stop(self) -> None:
        """策略停止时清理资源"""
        self.log.info(f"🛑 Stopping {self.strategy_name} strategy")
        if self.feature_manager:
            self.feature_manager.reset_history()
```

#### 6.4.1 数据流转换

| Nautilus 数据类型 | 转换方法 | DataFrame 列 |
|------------------|---------|--------------|
| `Bar` | `_bar_to_dataframe()` | timestamp, open, high, low, close, volume |
| `QuoteTick` | `_tick_to_dataframe()` | timestamp, bid, ask, bid_size, ask_size |
| `TradeTick` | `_trade_to_dataframe()` | timestamp, price, size, side |

#### 6.4.2 事件处理流程

```
on_start()
  ├─ 加载策略配置
  ├─ 初始化特征管理器

  └─ 订阅市场数据

on_bar(bar)
  ├─ Bar → DataFrame
  ├─ 计算特征
  ├─ 生成信号
  └─ 执行交易

on_stop()
  └─ 清理资源
```

#### 6.4.3 状态管理

- **历史窗口**：通过 `RealtimeFeatureManager` 维护滑动窗口

- **模型状态**：模型参数在训练时保存，实盘时加载

### 6.5 性能优化建议

1. **特征计算优化**
   - 使用 `max_workers=1` 禁用并行计算
   - 预编译依赖图，避免重复解析
   - 增量计算：只计算新数据，复用历史结果

2. **内存管理**
   - 限制历史窗口大小（如 1000 条）
   - 定期清理过期缓存
   - 使用 Nautilus Trader 的内存管理配置

3. **延迟优化**
   - 特征计算放在异步任务中（如果允许）
   - 使用缓存避免重复计算
   - 监控特征计算耗时

```python
            is_demo=False,
        ),
    },
    exec_clients={
        OKX: OKXExecClientConfig(
            api_key=None,
            api_secret=None,
            api_passphrase=None,
            instrument_types=(OKXInstrumentType.SWAP,),
            contract_types=(OKXContractType.LINEAR,),
            margin_mode=OKXMarginMode.CROSS,  # 全仓模式（共享保证金池）
            # margin_mode=OKXMarginMode.ISOLATED,  # 逐仓模式（独立保证金）
            is_demo=False,
        ),
    },
)

# 注册客户端工厂

# OKX 符号格式说明：
# - 现货: BTC-USDT.OKX
# - 永续合约（线性）: BTC-USDT-SWAP.OKX
# - 永续合约（反向）: BTC-USD-SWAP.OKX
# - 期货: BTC-USD-251226.OKX (到期日 YYMMDD)
# - 期权: BTC-USD-251226-100000-C.OKX (到期日-行权价-类型)

    bar_type=bar_type,
```

1. **保证金模式**：

   - `ISOLATED`（逐仓）：每个持仓独立保证金，风险隔离

2. **合约类型**：
   - `LINEAR`（线性）：USDT 保证金，如 BTC-USDT-SWAP
   - `INVERSE`（反向）：币本位保证金，如 BTC-USD-SWAP

3. **现货保证金交易**：

#### 7.4.5 多交易所统一接口

特征管理器会自动处理不同交易所的数据格式差异，因为：

1. **数据转换统一**：`_bar_to_dataframe()` 方法将不同交易所的 Bar 对象转换为统一的 DataFrame 格式
2. **符号格式统一**：Nautilus Trader 使用统一的 `InstrumentId` 格式，内部处理交易所差异
3. **特征计算一致**：特征计算逻辑与交易所无关，只依赖 DataFrame 的列结构

```python
# 同一个策略可以在不同交易所运行
strategies = [
    NautilusStrategyWithFeatures(
        strategy_name="sr_reversal",
        instrument_id=InstrumentId.from_str("BTCUSDT-PERP.BINANCE"),  # Binance
        bar_type=bar_type,
        trade_size=0.001,
    ),
    NautilusStrategyWithFeatures(
        strategy_name="sr_reversal",
        instrument_id=InstrumentId.from_str("BTC-USDT-SWAP.OKX"),  # OKX
        bar_type=bar_type,
        trade_size=0.001,
    ),
    NautilusStrategyWithFeatures(
        strategy_name="sr_reversal",
        instrument_id=InstrumentId.from_str("BTC-USD-PERP.DYDX"),  # dYdX
        bar_type=bar_type,
        trade_size=0.001,
    ),
]

# 添加到同一个 TradingNode
for strategy in strategies:
    node.trader.add_strategy(strategy)
```

### 7.5 关键注意事项

1. **数据一致性**：确保实时流中的特征计算与训练时一致
2. **状态同步**：多策略运行时，状态管理器需要隔离
3. **错误处理**：实时流中特征计算失败时的降级策略
4. **性能监控**：监控特征计算耗时，确保满足实时性要求
5. **Nautilus 配置**：合理配置缓存、对账等参数
6. **模型版本管理**：确保实盘使用的模型版本与训练时一致
7. **交易所特定配置**：
   - OKX：注意保证金模式（CROSS/ISOLATED）和合约类型（LINEAR/INVERSE）
   - Binance：注意账户类型（SPOT/USDT_FUTURES/COIN_FUTURES）
   - dYdX：需要钱包地址和助记词，首次使用需要激活账户
   - Hyperliquid：需要 EVM 私钥
