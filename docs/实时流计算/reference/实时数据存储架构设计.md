# 实时数据存储架构设计

## 一、数据分类和存储策略

### 1.1 数据分类

根据数据特性和使用场景，将数据分为三类：

| 数据类型 | 来源 | 存储方式 | 用途 |
|---------|------|---------|------|
| **历史研究数据** | 平台API批量下载 | Parquet文件 | 训练模型、回测、研究 |
| **实时K线数据** | WebSocket流式 | QuestDB | 实时特征计算、交易 |
| **订单流数据（Tick）** | WebSocket流式 | QuestDB | 实时特征计算、订单流分析 |

### 1.2 存储方案对比

#### 方案1：全部用 Parquet

❌ **不推荐**

**问题：**
- Parquet 是列式存储，适合批量写入，不适合实时流式写入
- 每次写入需要重新写入整个文件，性能差
- 无法高效查询最新数据
- 订单流数据量大，频繁写入会严重影响性能

#### 方案2：全部用数据库（QuestDB）

✅ **推荐**

**优势：**
- QuestDB 专为时序数据设计，支持高并发写入
- 支持 SQL 查询，易于数据合并和查询
- 自动压缩，存储效率高
- 支持实时查询和聚合

**适用场景：**
- 实时 K线数据（1分钟、5分钟等）
- 订单流数据（Tick数据）
- 实时特征数据（可选）

#### 方案3：混合方案（推荐）

✅ **最佳实践**

**架构：**
```
历史数据（研究用）
  └─ Parquet 文件（按月存储）
      ├─ 2024-01.parquet
      ├─ 2024-02.parquet
      └─ ...

实时数据（交易用）
  └─ QuestDB（时序数据库）
      ├─ klines 表（K线数据）
      ├─ ticks 表（订单流数据）
      └─ features 表（实时特征，可选）
```

### 3.1 Warmup 数据加载

**场景：** 启动时需要加载历史数据作为特征计算的窗口

```python
class DataManager:
    """数据管理器 - 处理历史数据和实时数据的合并"""
    
    def __init__(
        self,
        questdb_url: str = "http://localhost:9000",
        parquet_data_path: str = "data/parquet_data",
    ):
        self.questdb_url = questdb_url
        self.parquet_data_path = parquet_data_path
    
    def load_warmup_data(
        self,
        symbol: str,
        timeframe: str,
        warmup_bars: int = 1000,
    ) -> pd.DataFrame:
        """
        加载 Warmup 数据
        
        策略：
        1. 优先从 QuestDB 加载（如果有）
        2. 如果 QuestDB 数据不足，从 Parquet 补充
        3. 合并后返回
        
        Args:
            symbol: 交易对符号
            timeframe: 时间框架（如 '15m'）
            warmup_bars: 需要的 K线数量
        
        Returns:
            DataFrame with historical data
        """
        # 1. 尝试从 QuestDB 加载最近的数据
        questdb_data = self._load_from_questdb(symbol, timeframe, warmup_bars)
        
        if len(questdb_data) >= warmup_bars:
            # QuestDB 数据足够，直接返回
            return questdb_data
        
        # 2. QuestDB 数据不足，从 Parquet 补充
        needed_bars = warmup_bars - len(questdb_data)
        parquet_data = self._load_from_parquet(
            symbol,
            timeframe,
            needed_bars,
            before_timestamp=questdb_data["timestamp"].min() if len(questdb_data) > 0 else None,
        )
        
        # 3. 合并数据
        if len(questdb_data) > 0 and len(parquet_data) > 0:
            # 合并，去重
            combined = pd.concat([parquet_data, questdb_data])
            combined = combined.drop_duplicates(subset=["timestamp"]).sort_values("timestamp")
            return combined.tail(warmup_bars)
        elif len(questdb_data) > 0:
            return questdb_data
        else:
            return parquet_data
    
    def _load_from_questdb(
        self,
        symbol: str,
        timeframe: str,
        limit: int,
    ) -> pd.DataFrame:
        """从 QuestDB 加载数据"""
        import requests
        
        query = f"""
        SELECT * FROM klines
        WHERE symbol = '{symbol}' AND timeframe = '{timeframe}'
        ORDER BY timestamp DESC
        LIMIT {limit}
        """
        
        try:
            response = requests.post(
                f"{self.questdb_url}/exec",
                data=query,
            )
            response.raise_for_status()
            
            # 解析 CSV 响应
            import io
            df = pd.read_csv(io.StringIO(response.text))
            
            if "timestamp" in df.columns:
                df["timestamp"] = pd.to_datetime(df["timestamp"])
            
            return df.sort_values("timestamp")
        
        except Exception as e:
            print(f"⚠️ 从 QuestDB 加载数据失败: {e}")
            return pd.DataFrame()
```

```python
class QuestDBWriter:
    """QuestDB 数据写入器"""
    
    def __init__(self, questdb_url: str = "http://localhost:9000"):
        self.questdb_url = questdb_url
        self.ilp_sender = None  # ILP (InfluxDB Line Protocol) sender
    
    def write_bar(self, bar: Dict[str, Any]):
        """
        写入 K线数据
        
        使用 ILP (InfluxDB Line Protocol) 格式，性能最优
        """
        import socket
        
        ilp_message = (
            f"klines,symbol={bar['symbol']},timeframe={bar['timeframe']} "
            f"open={bar['open']},high={bar['high']},low={bar['low']},"
            f"close={bar['close']},volume={bar['volume']} "
            f"{int(bar['timestamp'] * 1_000_000_000)}\n"  # 纳秒时间戳
        )
        
        self._send_ilp(ilp_message)
    
    def write_tick(self, tick: Dict[str, Any]):
        """写入订单流数据"""
        ilp_message = (
            f"ticks,symbol={tick['symbol']},side={tick['side']} "
            f"price={tick['price']},size={tick['size']},trade_id={tick['trade_id']} "
            f"{int(tick['timestamp'] * 1_000_000_000)}\n"
        )
        
        self._send_ilp(ilp_message)
    
    def _send_ilp(self, message: str):
        """发送 ILP 消息到 QuestDB"""
        import socket
        
        try:
            # QuestDB ILP 默认端口 9009
            sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
            sock.connect(("localhost", 9009))
            sock.sendall(message.encode())
            sock.close()
        except Exception as e:
            print(f"❌ 写入 QuestDB 失败: {e}")
```

## 四、完整实现方案

### 4.1 实时数据管理器

```python
"""
实时数据管理器 - 整合历史数据和实时数据
"""
import pandas as pd
from typing import Optional, Dict, Any
from datetime import datetime, timedelta

from src.data_tools.data_utils import load_raw_data  # 假设你有这个函数


class RealtimeDataManager:
    """
    实时数据管理器
    
    职责：
    1. 从 Parquet 加载历史数据（Warmup）
    2. 从 QuestDB 加载实时数据
    3. 合并数据供特征计算使用
    4. 将实时数据写入 QuestDB
    """
    
    def __init__(
        self,
        symbol: str,
        timeframe: str,
        questdb_url: str = "http://localhost:9000",
        parquet_data_path: str = "data/parquet_data",
        warmup_bars: int = 1000,
    ):
        self.symbol = symbol
        self.timeframe = timeframe
        self.questdb_url = questdb_url
        self.parquet_data_path = parquet_data_path
        self.warmup_bars = warmup_bars
        
        # 数据缓存
        self.history_df: Optional[pd.DataFrame] = None
        self.questdb_writer = QuestDBWriter(questdb_url)
    
    def initialize(self) -> pd.DataFrame:
        """
        初始化：加载 Warmup 数据
        
        Returns:
            包含历史数据的 DataFrame
        """
        print(f"📊 加载 Warmup 数据: {self.symbol} {self.timeframe}")
        
        # 1. 优先从 QuestDB 加载（如果有最近的数据）
        questdb_data = self._load_from_questdb(self.warmup_bars)
        
        if len(questdb_data) >= self.warmup_bars:
            print(f"✅ 从 QuestDB 加载了 {len(questdb_data)} 条数据")
            self.history_df = questdb_data
            return questdb_data
        
        # 2. QuestDB 数据不足，从 Parquet 补充
        print(f"⚠️ QuestDB 数据不足 ({len(questdb_data)} 条)，从 Parquet 补充")
        
        needed_bars = self.warmup_bars - len(questdb_data)
        parquet_data = self._load_from_parquet(needed_bars)
        
        # 3. 合并数据
        if len(questdb_data) > 0 and len(parquet_data) > 0:
            # 合并，去重，排序
            combined = pd.concat([parquet_data, questdb_data])
            combined = combined.drop_duplicates(subset=["timestamp"]).sort_values("timestamp")
            self.history_df = combined.tail(self.warmup_bars)
        elif len(questdb_data) > 0:
            self.history_df = questdb_data
        else:
            self.history_df = parquet_data
        
        print(f"✅ Warmup 数据加载完成: {len(self.history_df)} 条")
        return self.history_df
    
    def append_bar(self, bar: Dict[str, Any]) -> pd.DataFrame:
        """
        追加新的 K线数据
        
        Args:
            bar: K线数据字典
        
        Returns:
            更新后的 DataFrame（包含新数据）
        """
        # 1. 转换为 DataFrame
        new_bar_df = pd.DataFrame([{
            "timestamp": pd.Timestamp.fromtimestamp(bar["timestamp"] / 1000),
            "datetime": pd.Timestamp.fromtimestamp(bar["timestamp"] / 1000),
            "open": float(bar["open"]),
            "high": float(bar["high"]),
            "low": float(bar["low"]),
            "close": float(bar["close"]),
            "volume": float(bar.get("volume", 0)),
            "symbol": self.symbol,
        }])
        
        # 2. 写入 QuestDB（异步，不阻塞）
        try:
            self.questdb_writer.write_bar({
                **bar,
                "symbol": self.symbol,
                "timeframe": self.timeframe,
            })
        except Exception as e:
            print(f"⚠️ 写入 QuestDB 失败: {e}")
        
        # 3. 追加到历史数据
        if self.history_df is None:
            self.history_df = new_bar_df
        else:
            self.history_df = pd.concat([self.history_df, new_bar_df])
            
            # 维护滑动窗口（只保留最近 N 条）
            if len(self.history_df) > self.warmup_bars * 2:
                self.history_df = self.history_df.tail(self.warmup_bars * 2)
        
        return self.history_df
    
    def append_tick(self, tick: Dict[str, Any]):
        """追加订单流数据（只写入 QuestDB，不保留在内存）"""
        try:
            self.questdb_writer.write_tick({
                **tick,
                "symbol": self.symbol,
            })
        except Exception as e:
            print(f"⚠️ 写入 Tick 数据失败: {e}")
    
    def get_dataframe(self) -> pd.DataFrame:
        """获取当前数据 DataFrame"""
        return self.history_df.copy() if self.history_df is not None else pd.DataFrame()
    
    def _load_from_questdb(self, limit: int) -> pd.DataFrame:
        """从 QuestDB 加载数据"""
        import requests
        
        query = f"""
        SELECT 
            timestamp,
            open, high, low, close, volume,
            trade_count, buy_qty, sell_qty, delta,
            taker_buy_ratio, cvd
        FROM klines
        WHERE symbol = '{self.symbol}' AND timeframe = '{self.timeframe}'
        ORDER BY timestamp DESC
        LIMIT {limit}
        """
        
        try:
            response = requests.post(
                f"{self.questdb_url}/exec",
                data=query,
                timeout=5,
            )
            response.raise_for_status()
            
            import io
            df = pd.read_csv(io.StringIO(response.text))
            
            if len(df) > 0:
                df["timestamp"] = pd.to_datetime(df["timestamp"])
                df = df.sort_values("timestamp")
                return df
            
            return pd.DataFrame()
        
        except Exception as e:
            print(f"⚠️ 从 QuestDB 加载数据失败: {e}")
            return pd.DataFrame()
    
    def _load_from_parquet(self, needed_bars: int) -> pd.DataFrame:
        """从 Parquet 文件加载数据"""
        try:
            # 使用现有的数据加载函数
            # 这里需要根据你的实际实现调整
            df = load_raw_data(
                symbol=self.symbol,
                data_path=self.parquet_data_path,
                # 其他参数...
            )
            
            # 只取最近 needed_bars 条
            if len(df) > needed_bars:
                df = df.tail(needed_bars)
            
            return df
        
        except Exception as e:
            print(f"⚠️ 从 Parquet 加载数据失败: {e}")
            return pd.DataFrame()
```

## 六、总结和建议

### 6.1 推荐方案

✅ **混合存储方案**

1. **历史数据（研究用）**：Parquet 文件
   - 按月存储
   - 批量下载和加载
   - 用于训练和回测

2. **实时数据（交易用）**：QuestDB
   - K线数据：实时写入，保留 90 天
   - Tick 数据：实时写入，保留 7 天
   - 特征数据（可选）：实时写入，保留 30 天

3. **数据合并**：
   - 启动时：从 QuestDB + Parquet 加载 Warmup 数据
   - 运行时：实时数据追加到内存，同时写入 QuestDB
   - 特征计算：使用内存中的合并数据

### 6.2 实施步骤

1. **阶段1**：设置 QuestDB
   - 安装和配置 QuestDB
   - 创建表结构

2. **阶段2**：实现数据管理器
   - 实现 `RealtimeDataManager`

   - 实现实时数据写入

3. **阶段3**：集成到交易引擎
   - 在启动时加载 Warmup 数据
   - 实时数据写入 QuestDB

4. **阶段4**：数据维护

1. ✅ **性能**：QuestDB 专为时序数据优化，写入和查询性能优秀
2. ✅ **灵活性**：可以同时使用历史数据和实时数据
3. ✅ **可维护性**：数据自动管理，无需手动处理
4. ✅ **可扩展性**：支持多交易对、多时间框架
