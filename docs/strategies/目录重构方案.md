# 策略模块目录重构方案

## 当前问题

1. **特征重复**：`src/data_tools/feature_engineering_enhanced.py` 和 `src/strategies/utils/` 中都有 WPT、Hurst、Hilbert、Spectrum 实现
2. **职责不清**：`data_tools` 既包含数据加载，又包含特征工程
3. **扩展性差**：未来要加多因子横截面策略，当前结构难以扩展

## 重构目标

1. ✅ **统一时序特征**：将所有时序特征工程统一到 `strategies/time_series/features/`
2. ✅ **分离策略类型**：时序策略 vs 横截面策略
3. ✅ **清晰职责**：`data_tools` 只负责数据加载和基础工具
4. ✅ **易于扩展**：为横截面策略预留空间

## 重构步骤

### 阶段 1：创建新目录结构

1. 创建 `src/strategies/time_series/` 目录
2. 创建 `src/strategies/time_series/features/` 目录
3. 创建 `src/strategies/cross_sectional/` 目录（预留）

### 阶段 2：迁移特征工程代码

1. 将 `data_tools/baseline_features.py` → `strategies/time_series/features/baseline_features.py`
2. 将 `data_tools/feature_engineering_enhanced.py` → `strategies/time_series/features/enhanced_features.py`
3. 整合 `strategies/utils/` 中的工具到 `enhanced_features.py` 或保留为独立工具
4. 更新 `comprehensive_feature_engineering.py` 的导入路径

### 阶段 3：迁移策略代码

1. 将 `strategies/labels/` → `strategies/time_series/labels/`
2. 将 `strategies/models/` → `strategies/time_series/models/`
3. 将 `strategies/sr_reversal/` → `strategies/time_series/strategies/sr_reversal/`
4. 将 `strategies/sr_breakout/` → `strategies/time_series/strategies/sr_breakout/`
5. 将 `strategies/compression_breakout/` → `strategies/time_series/strategies/compression_breakout/`
6. 将 `strategies/trend_following/` → `strategies/time_series/strategies/trend_following/`

### 阶段 4：更新导入路径

1. 更新所有训练脚本的导入
2. 更新 Makefile 中的模块路径
3. 更新文档

### 阶段 5：清理旧代码

1. 删除 `strategies/utils/`（如果已整合）
2. 更新 `data_tools/` 中的引用

## 优势

### 1. 职责清晰

- **data_tools**：数据加载、数据预处理、基础工具
- **strategies/time_series/features**：时序特征工程（统一管理）
- **strategies/cross_sectional/features**：横截面特征工程（独立管理）

### 2. 易于扩展

- 新增时序策略：在 `strategies/time_series/strategies/` 下添加
- 新增横截面策略：在 `strategies/cross_sectional/strategies/` 下添加
- 特征工程独立，策略可复用

### 3. 避免重复

- 时序特征统一在 `time_series/features/` 管理
- 横截面特征统一在 `cross_sectional/features/` 管理
- 共用因子（如 Alpha101）放在 `alpha_factors/`

### 4. 模块化

- 特征工程、标签、模型、策略分离
- 每个模块职责单一，易于测试和维护
