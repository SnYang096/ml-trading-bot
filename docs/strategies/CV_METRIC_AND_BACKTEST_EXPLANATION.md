# CV指标和回测验证说明

## 1. CV指标计算方式

### 1.1 二分类任务

**CV指标 = Pearson相关系数（预测概率 vs 真实标签）**

```python
# 代码位置：src/time_series_model/strategies/models/strategy_trainer.py 第366行
if task_type == "binary":
    pred_val = model.predict_proba(X_val)[:, -1]  # 预测概率（0-1之间）
    metric = np.corrcoef(pred_val, y_val)[0, 1]  # Pearson相关系数
```

**关键点**：
- ✅ **不依赖入场阈值**：CV指标计算的是预测概率与真实标签的线性相关性
- ✅ **不需要实际开仓**：只需要模型预测的概率值
- ✅ **范围**：-1 到 +1
- ✅ **含义**：预测概率越高，真实标签为1的概率也越高（正相关）

**为什么不需要入场阈值？**
- CV指标评估的是模型的**预测能力**，而不是交易策略的盈利能力
- 它衡量的是：模型能否区分"好信号"和"坏信号"
- 即使预测概率只有0.3，如果真实标签确实是0，那么相关性仍然是正确的

### 1.2 回归任务

**CV指标 = Pearson相关系数（预测值 vs 真实值）**

```python
# 代码位置：src/time_series_model/strategies/models/strategy_trainer.py 第358行
if task_type == "regression":
    pred_val = model.predict(X_val)  # 预测的R/R值
    metric = np.corrcoef(pred_val, y_val)[0, 1]  # Pearson相关系数
```

**关键点**：
- ✅ **不依赖入场阈值**：CV指标计算的是预测R/R值与真实R/R值的线性相关性
- ✅ **不需要实际开仓**：只需要模型预测的R/R值
- ✅ **范围**：-1 到 +1
- ✅ **含义**：预测R/R值越高，真实R/R值也越高（正相关）

## 2. 为什么回归（带权重）版本性能下降？

### 2.1 可能原因

#### 原因1：样本分布变化
- **SR过滤后**：只保留SR附近的样本（62.3%）
- **这些样本可能更难预测**：
  - 在SR附近，价格行为更复杂（震荡、假突破等）
  - 反转信号的成功率可能更低
  - 但一旦成功，R/R更高（这正是我们想要的）

#### 原因2：样本数量减少
- **训练样本**：从10,823减少到6,680（-38.3%）
- **可能影响**：
  - 模型学习能力下降
  - 交叉验证的稳定性下降
  - 但这不是主要原因（因为二分类版本样本也减少了，但CV指标提升）

#### 原因3：标签分布变化
- **SR附近的样本**：可能包含更多"失败"的反转信号
- **远离SR的样本**：可能包含更多"成功"的反转信号（但被过滤掉了）
- **结果**：过滤后的样本中，成功信号的比例可能下降

#### 原因4：模型过拟合
- **样本减少**：可能导致模型更容易过拟合
- **但**：二分类版本样本也减少了，但CV指标提升，所以这个原因不太可能

### 2.2 验证方法

1. **检查标签分布**：
   - 无权重版本：正样本 vs 负样本比例
   - 带权重版本：正样本 vs 负样本比例
   - 如果带权重版本的正样本比例下降，说明过滤掉了更多"好信号"

2. **检查预测分布**：
   - 无权重版本：预测值的分布
   - 带权重版本：预测值的分布
   - 如果带权重版本的预测值更集中，说明模型更保守

3. **检查回测结果**：
   - 虽然CV指标下降，但实际交易收益可能更好
   - 因为SR过滤保留了更符合策略逻辑的样本

## 3. 回测验证

### 3.1 回测指标

- **总收益（total_return_pct）**：策略的总收益率
- **夏普比率（sharpe）**：风险调整后的收益（>1.0为优秀）
- **最大回撤（max_drawdown_pct）**：最大亏损幅度
- **交易次数（total_trades）**：总交易次数

### 3.2 为什么需要回测？

- **CV指标**：衡量模型的预测能力（不依赖入场阈值）
- **回测结果**：衡量策略的实际盈利能力（依赖入场阈值、止损止盈等）

**两者可能不一致**：
- CV指标高，但回测收益低：可能是入场阈值设置不当
- CV指标低，但回测收益高：可能是模型预测能力一般，但策略逻辑有效

## 5. 建议

### 5.1 对于回归（带权重）版本性能下降

1. **检查标签分布**：
   - 对比无权重版本和带权重版本的正负样本比例
   - 如果带权重版本的正样本比例下降，考虑调整过滤条件

2. **检查回测结果**：
   - 虽然CV指标下降，但实际交易收益可能更好
   - 如果回测收益提升，CV指标下降可以接受

3. **调整过滤条件**：
   - 尝试增大 `dist_atr_mult`（如从1.5改为2.0）
   - 或者使用更宽松的过滤条件

4. **检查样本权重**：
   - 验证样本权重是否正确计算
   - 验证权重分布是否合理

### 5.2 对于CV指标的理解

1. **CV指标不依赖入场阈值**：
   - 它衡量的是模型的预测能力
   - 不需要实际开仓，只需要预测值

2. **CV指标 vs 回测结果**：
   - CV指标高：模型预测能力强
   - 回测收益高：策略实际盈利能力强
   - 两者可能不一致，需要综合考虑

3. **选择策略的标准**：
   - 优先看回测结果（夏普比率、总收益）
   - CV指标作为参考（模型预测能力）
   - 综合考虑两者

## 6. 总结

1. **CV指标计算**：
   - 二分类：预测概率 vs 真实标签的Pearson相关系数
   - 回归：预测值 vs 真实值的Pearson相关系数
   - **不依赖入场阈值，不需要实际开仓**

2. **回归（带权重）版本性能下降**：
   - 可能原因：样本分布变化、标签分布变化
   - 需要检查：标签分布、预测分布、回测结果

3. **回测验证**：
   - 查看实际交易收益（夏普比率、总收益、最大回撤）
   - 如果回测收益提升，CV指标下降可以接受

4. **集成测试**：
   - 验证SR过滤正确性
   - 验证CV指标计算
   - 验证标签数量对比
