# 标签生成优化说明

## 优化背景

之前的标签生成使用规则过滤（如 `min_sr_strength`, `min_support_score` 等），导致：
- 标签数据过少
- 模型无法学习到完整的信号分布
- 规则过滤可能过滤掉有价值的样本

## 优化方案

### 1. 采用全量扫描标签生成

**之前：** 使用信号过滤规则 → 生成信号 → 计算标签
**现在：** 直接使用 `compute_sr_reversal_label_full_scan` → 对每根 K 线都假设入场 → 计算标签

### 2. 移除 Makefile 中的过滤配置

**已删除的配置：**
- `SR_SIGNAL_MIN_STRENGTH`
- `SR_SIGNAL_MIN_SUPPORT`
- `SR_SIGNAL_MIN_RESISTANCE`
- `SR_SIGNAL_TOLERANCE_MULT`
- `SR_SIGNAL_MIN_TOLERANCE_PCT`
- `SR_SIGNAL_REQUIRE_FIRST_TOUCH`
- `SR_SIGNAL_MAX_TOUCHES`
- `SR_SIGNAL_ZONE_PRECISION`

**原因：**
- 标签生成不再依赖这些过滤规则
- 模型会根据特征学习哪些信号应该使用
- 保留更多训练样本

### 3. 代码更新

#### `src/time_series_model/strategies/labels/sr_reversal_label.py`

1. **SRSignalConfig 默认值改为 0.0**
   - `min_sr_strength: float = 0.0` （不过滤）
   - `min_support_score: float = 0.0` （不过滤）
   - `min_resistance_score: float = 0.0` （不过滤）

2. **添加注释说明**
   - 说明这些配置主要用于诊断/优化脚本
   - 标签生成使用 `compute_sr_reversal_label_full_scan`，不依赖信号过滤

3. **`compute_sr_reversal_label_full_scan` 文档更新**
   - 说明这是推荐的标签生成方式
   - 解释为什么全量扫描更好

#### `Makefile`

1. **删除配置定义**（780-788行）
2. **删除环境变量传递**（809-816行）
3. **添加注释说明**优化原因

## 当前标签生成流程

### SR Reversal 策略

**配置文件：** `config/strategies/sr_reversal/labels.yaml`

```yaml
label_generator:
  module: src.time_series_model.strategies.labels.sr_reversal_label
  function: compute_sr_reversal_label_full_scan  # 全量扫描
  params:
    max_holding_bars: 50
    take_profit_r: 2.0
    stop_loss_r: 1.0
    combine_mode: any_success
```

**工作流程：**
1. 对每根 K 线都假设入场（不预先过滤）
2. 计算做多标签：entry=open[t+1]，TP=+2R，SL=-1R
3. 计算做空标签：entry=open[t+1]，TP=-2R，SL=+1R
4. 按 `combine_mode` 合并（any_success/long_only/short_only）

**优势：**
- ✅ 保留所有可能的训练样本
- ✅ 模型可以根据特征学习哪些信号应该使用
- ✅ 不依赖人工规则，更灵活

## 模型学习方式

### 之前（规则过滤）
```
信号生成 → 规则过滤（min_sr_strength >= 0.5） → 标签计算 → 模型训练
         ↑ 可能过滤掉有价值的样本
```

### 现在（全量扫描）
```
全量扫描 → 标签计算（所有 K 线） → 模型训练
         ↑ 模型根据特征学习哪些信号应该使用
```

**模型会学习：**
- 哪些 SR 强度应该使用
- 哪些支撑/阻力分数应该使用
- 哪些特征组合表示高质量信号

## 注意事项

### 1. 诊断/优化脚本

`SRSignalConfig` 和 `_apply_env_overrides` 仍然保留，用于：
- 诊断脚本（如 `sr_reversal_model_comparison.py`）
- 优化脚本（如 `ts_sr_reversal_optuna.py`）
- 规则基准测试（如 `sr_reversal_rule_baseline.py`）

这些脚本可以通过环境变量设置阈值进行测试，但不影响标签生成。

### 2. 向后兼容

- `_generate_sr_reversal_signals` 函数仍然保留（用于诊断/优化）
- 默认阈值都是 0.0（不过滤）
- 如果需要，仍然可以通过环境变量设置阈值（仅用于诊断）

### 3. 特征重要性

由于不再预先过滤，模型需要学习：
- **SR 强度特征**：`sr_strength_max` 等
- **支撑/阻力特征**：`sqs_hal_low`, `sqs_hal_high` 等
- **其他特征**：VPIN、流动性、频谱等

这些特征应该已经在策略的 `features.yaml` 中配置。
