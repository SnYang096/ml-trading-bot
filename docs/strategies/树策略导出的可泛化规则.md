## 树策略到Execution的映射关系

> 相关文档：[树模型策略区分机制](树模型策略区分机制.md)

### 映射关系表

| 树策略 | 主要对应Execution | 规则类型 | 典型规则示例 |
|--------|------------------|----------|--------------|
| **sr_reversal_rr_reg_long** | **FR** (Failure Reversion) | 假突破检测、订单流背离 | `cvd_long <= -34895 AND vp_poc_deviation <= -0.025` |
| **sr_breakout** | **TE** (Trend Expansion) | 突破质量确认、放量验证 | `cvd_change_5 > 4805 AND volume_ratio > 1.87` |
| **compression_breakout** | **TE** (Trend Expansion) | 压缩突破确认、流动性陷阱 | `liquidity_void_retracement <= 0.038 AND cvd_change_5 <= -848` |
| **trend_following** | **TC** (Trend Continuation) / **ET** (Exhaustion Turn) | 趋势健康度、衰竭检测 | `bb_width_normalized <= 4.86 AND cvd_long <= -34895` (TC veto)<br>`funding_exhaustion_scene_score > 0.002` (ET signal) |

### 映射逻辑说明

#### 为什么sr_reversal对应FR？

- **sr_reversal**关注"SR区域的反转机会"
- 其规则往往检测"假突破/订单流背离"
- 这正是**FR（Failure Reversion）**的核心语义：突破失败后的回归

**典型规则模式**：
- 订单流背离：`cvd_long <= -34895`（长期订单流负，但价格可能上涨）
- 结构确认：`vp_poc_deviation <= -0.025`（POC偏离，volume集中在某区域）
- 组合：检测"看起来像突破，但底层订单流/结构不支持"

#### 为什么sr_breakout/compression_breakout对应TE？

- 两者都关注"突破"场景
- 规则主要验证"突破质量"（放量、订单流支持）
- 这正是**TE（Trend Expansion）**的核心语义：趋势刚放大/加速

**sr_breakout的典型规则模式**：
- 放量确认：`volume_ratio > 1.87`
- 订单流支持：`cvd_change_5 > 4805`（短期订单流推进）
- 组合：验证"真实突破，有订单流和成交量支持"

**compression_breakout的典型规则模式**：
- 流动性陷阱检测：`liquidity_void_retracement <= 0.038`（流动性真空后回撤很浅）
- 假突破警告：`cvd_change_5 <= -848`（订单流不支持）
- 组合：检测"假突破/流动性陷阱"，用于TE的Gate（否决）

#### 为什么trend_following对应TC/ET？

- **trend_following**关注"趋势延续"
- **正规则** → **TC（Trend Continuation）**：趋势健康度验证
- **负规则/衰竭规则** → **ET（Exhaustion Turn）**：趋势末端反转

**TC的典型规则（负规则，用于Gate）**：
- 趋势不健康：`bb_width_normalized <= 4.86 AND cvd_long <= -34895`
  - 语义：波动不放大 + 长期订单流持续负 → 趋势已经死了
  - 用途：TC Gate（否决）

**ET的典型规则（正规则，用于信号）**：
- 衰竭信号：`funding_exhaustion_scene_score > 0.002 AND macd_signal <= 0.425`
  - 语义：资金费率压力存在，但非趋势+非压缩（更像压力释放/衰竭）
  - 用途：ET信号（允许）

### 重要注意事项

1. **规则具有策略特异性**：同一特征在不同策略的规则中可能有**相反的含义**
   - 例如：`cvd_long <= -34895`在trend_following中是负规则（TC veto），但在sr_reversal中可能是正规则（FR信号）

2. **不能简单照搬阈值**：导出的阈值是历史拟合的，需要泛化为分位数或相对位置

3. **必须考虑规则的语义上下文**：不能只看特征名称，要看规则在策略中的角色（正规则/负规则）

详见下文"如何把树规则'翻译'为4类Execution的Gate/Bias"部分。

## 什么叫“明显语义特征”？

本仓库里有一批特征会输出 **0..1 的语义场景分数**（通常命名包含 `scene` / `semantic` / `*_score`），例如：

- `funding_*_score` / `funding_*_scene_score`
- `trade_cluster_*_score`
- `vpin_*_score`
- `wick_*_score`
- `volume_profile_*_score`
- `liquidity_void_*_score`
- `dtw_*_score` 等

它们的共同点：

- **语义明确**：不是“原始价格/成交量数值”，而是对某种 *regime* 的归纳（压缩/点火/吸收/衰竭）。
- **可当作门禁**：适合做“允许/禁止开仓”“加减仓/风控”这种跨策略组件。

## 可泛化的“语义规则”清单（来自 `trend_following` 导出）

下面这些规则直接来自导出的 RuleFit pruned rules（full 版本）。**建议把阈值当作“启发”，不要照抄**（见后文“如何泛化阈值”）。

### 1) Funding Exhaustion（衰竭）作为“反身性风险/切换信号”

- 规则形态（示例）：
  - `funding_exhaustion_scene_score > 0.00204 and macd_signal <= 0.42512`
- 泛化解读：
  - **funding_exhaustion** 的定义是：资金费率压力存在，但同时“非趋势 + 非压缩”（更像“**压力释放/衰竭**”），这类 regime 往往更适合作为 **风险切换/谨慎开仓** 的条件，而不是盲目追趋势。
- 可复用方式：
  - **趋势策略**：当 exhaustion 高时，降低追涨开仓优先级 / 提高入场门槛。
  - **均值回归/反转策略**：exhaustion 高时，反而可能是“允许出手”的 gating 之一（仍需配合结构/流动性证据）。

### 2) Funding Ignition（点火）作为“趋势延续开绿灯”

- 规则形态（示例）：
  - `funding_ignition_score > 0.00042` +（配合 `rsi`、`cvd_*` 的轻量过滤）
- 泛化解读：
  - ignition 描述“资金费率压力 + 低压缩 + 趋势强”，更像 **拥挤但仍在推进的趋势阶段**。
- 可复用方式：
  - **突破/趋势类**：ignition 高时允许更积极的趋势跟随/突破信号。
  - **反转类**：ignition 高时减少逆势抄底（避免跟大趋势对打）。

### 3) Funding Absorption（吸收/拥挤延续）作为“过热风控”

导出规则里 absorption 多次与 `rsi`、`bb_width`、`macd` 联动，呈现出“**拥挤/过热时要更谨慎**”的味道：

- 规则形态（示例）：
  - `funding_absorption_score > 0.08091 ...`（配合 `bb_width_normalized` / `macd`）
  - `funding_absorption_score > 0.06898 and rsi > 55.41 ...`
- 泛化解读：
  - absorption 的定义是 “压力存在 + 压缩存在 + 趋势也强”，这经常意味着 **拥挤延续**，但也可能对应“高波动/易甩人”的阶段。
- 可复用方式：
  - **统一风控组件**：当 absorption 高且价格/动量指标显示过热（如 RSI 高、BB 宽度变大），降低仓位上限/提高止损敏感度。

### 4) Funding Compression（压缩）作为“等待/少动”信号

- 规则形态（示例）：
  - `funding_compression_score <= ...` 与 `bb_width_normalized` 组合出现（弱负向贡献）
- 泛化解读：
  - compression 更像“压力存在但趋势弱”，可能是 **盘整/挤压** 的背景。
- 可复用方式：
  - **突破策略**：把 compression 当作“可能会爆”的背景，但仍需突破触发器（range break、成交量确认等）。
  - **趋势策略**：compression 高时减少追趋势，等待 ignition 信号。

## 为什么这些 funding 分数是“语义可泛化”的？

因为它们不是“拍脑袋命名”，而是有清晰公式定义（见 `src/features/time_series/funding_rate_features.py` 的 `compute_funding_scene_semantic_scores_from_df`）：

- 先构造 funding 压力：`stress := sigmoid((abs_z - z_shift) / z_scale)`
- 然后把压力投影到不同 regime：
  - `funding_compression_score := stress * compression * (1 - trend)`
  - `funding_ignition_score := stress * (1 - compression) * trend`
  - `funding_absorption_score := stress * compression * trend`
  - `funding_exhaustion_scene_score := stress * (1 - compression) * (1 - trend)`

这意味着你可以把它们当作：

- **可解释的 regime embedding**（压缩/点火/吸收/衰竭）
- **跨策略通用的门禁与风控输入**

## 如何把“策略内阈值”泛化成可复用逻辑（推荐做法）

导出的规则阈值（如 `> 0.00204`）只对当时训练窗/数据分布有效。要做跨策略泛化，建议用以下方式替代硬编码阈值：

### 方式 A：用分位数阈值（推荐）

- 在目标训练窗（或 rolling）上计算：
  - `q_hi = quantile(funding_ignition_score, 0.8)`
  - `q_lo = quantile(funding_exhaustion_scene_score, 0.8)` 等
- 然后规则写成：
  - `if funding_ignition_score >= q_hi: ...`

### 方式 B：用 “top-k% 激活” 而不是绝对值

- 例如只在 `funding_absorption_score` 处于过去 N 天 top 10% 时触发风控。

### 方式 C：把它们做成连续的 risk multiplier（不做硬开关）

- `risk_multiplier = 1 - clamp01(funding_absorption_score)`  
  让仓位/下单强度随语义分数平滑变化（更稳健）。

## 可直接抄走的“规则模板”（跨策略）

下面是把树策略导出的 if/else 思路抽象成“组件化模板”的写法（示意）：

### 模板 1：趋势/突破开仓绿灯（Ignition gate）

- **适用**：trend following、breakout、momentum
- **逻辑**：
  - `if funding_ignition_score is high: allow trend entries; else: be conservative`

### 模板 2：过热风控（Absorption + Overheat）

- **适用**：所有策略（尤其是追趋势/追突破）
- **逻辑**：
  - `if funding_absorption_score high and (RSI high or BB width high): reduce size / tighten stops`

### 模板 3：衰竭/切换（Exhaustion switch）

- **适用**：把趋势策略切到“等待”，或给反转策略提供“允许出手”的背景条件
- **逻辑**：
  - `if funding_exhaustion high: avoid chasing; consider mean-reversion setups (with extra confirmation)`
