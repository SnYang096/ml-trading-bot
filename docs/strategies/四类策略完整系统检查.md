# 四类策略完整系统检查

## 一、策略系统架构总览

| 策略类型 | 模型类型 | 标签类型 | 特征集 | 回测方法 | 评估方法 |
|---------|---------|---------|--------|---------|---------|
| **SR 反转** | XGBoost Classifier (Binary) | 二元标签（≥2R 成功率） | SR 强度 + 流动性验证 | 固定止损/止盈回测 | 胜率、平均 R/R |
| **SR 突破** | XGBoost Regressor | 连续标签（实现 R/R） | 突破质量 + 动能持续性 | 动态止损/止盈回测 | MSE、Top Decile R/R |
| **压缩区突破** | CatBoost Classifier (Multiclass) | 三元标签（方向+质量） | 压缩强度 + 突破触发 | 区间高度目标回测 | 准确率、F1-score |
| **趋势跟踪** | LightGBM Regressor | 百分位标签（Rank） | 趋势强度 + 资金验证 | Rank 退出回测 | Rank IC、Top-Bottom Spread |

## 二、详细配置检查

### 2.1 SR 反转策略

#### 模型配置
- **模型类型**：XGBoost Classifier
- **任务类型**：Binary Classification
- **输出**：P(success) = P(label=1)
- **文件位置**：`src/time_series_model/strategies/sr_reversal/train.py`

#### 标签配置
- **标签类型**：二元标签（1.0 = 成功，0.0 = 失败）
- **标签定义**：在 SR 区入场后，50根K线内是否先触达 +2R 止盈 而非 -1R 止损？
- **R 定义**：R = 1×ATR
- **文件位置**：`src/time_series_model/strategies/labels/sr_reversal_label.py`
- **参数**：
  - `hold_bars`: 50
  - `rr_ratio`: 2.0
  - `stop_loss_r`: 1.0
  - `take_profit_r`: 2.0

#### 特征配置
- **特征集**：SR 强度 + 流动性验证 + 波动状态
- **核心特征**：
  - WPT 低频 POC 与当前价距离
  - SR 重叠数（POC+Swing+ZigZag 同价位数量）
  - VPVR 高量节点（PVP）支撑强度
  - CVD 在 SR 区的净买入斜率
  - Hilbert 相位差（CVD vs Price）
  - Take Buy Ratio
- **文件位置**：`src/time_series_model/strategies/sr_reversal/features.py`

#### 回测方法（待实现）
- **方向**：由 SR 类型决定（支撑区 → 做多，阻力区 → 做空）
- **入场点**：价格触及 SR 区边界（如 POC 下沿反弹）
- **止损**：入场价 ± 1×ATR（反向）
- **止盈**：入场价 ± 2×ATR（同向）
- **仓位**：`position_size ∝ P(label=1)`（高置信度重仓）
- **加仓**：❌ 通常不加仓（反转是单波段机会）
- **减仓**：若价格未快速脱离，且 CVD 相位转负，可提前部分止盈

#### 评估方法（待实现）
- **主要指标**：胜率（Win Rate）
- **辅助指标**：平均 R/R、最大回撤、夏普比率
- **评估逻辑**：关注"是否值得冒 1R 风险博 2R 收益"

### 2.2 SR 突破策略

#### 模型配置
- **模型类型**：XGBoost Regressor
- **任务类型**：Regression
- **输出**：Predicted R/R（预期风险回报比）
- **文件位置**：`src/time_series_model/strategies/sr_breakout/train.py`

#### 标签配置
- **标签类型**：连续标签（0 到 max_rr）
- **标签定义**：突破后最大有利偏移 / 最大不利偏移（MFE/MAE），截断在 [0, 3] 区间
- **文件位置**：`src/time_series_model/strategies/labels/sr_breakout_label.py`
- **参数**：
  - `hold_bars`: 50
  - `max_rr`: 3.0
  - `stop_loss_r`: 1.0

#### 特征配置
- **特征集**：突破质量 + 动能持续性 + 真空区识别
- **核心特征**：
  - 突破时成交量/20日均量
  - VPER（Volume-Price Energy Ratio）中频 spike
  - 突破后 3 根 K 线收盘站稳比例
  - WPT 中低频能量比
  - Hurst 指数
  - ROC(5)/ROC(20) 比值
  - 突破方向上的 VPVR 低量节点距离
  - Spectrum 主频迁移
- **文件位置**：`src/time_series_model/strategies/sr_breakout/features.py`

#### 回测方法（待实现）
- **方向**：由突破方向决定（向上突破 → 做多，向下突破 → 做空）
- **入场点**：突破确认（如收盘站稳 + 成交量 spike）
- **止损**：突破点反向 1×ATR 或区间边界
- **止盈**：动态止盈
  - 初始 TP = 入场 + 2×ATR
  - 若模型预测 R/R > 2.0，启用 trailing stop
- **仓位**：`position_size ∝ predicted_R_R`（R/R=2.5 比 R/R=1.2 重仓）
- **加仓**：✅ 若价格回踩不破且 CVD 持续流入，可在 0.5×ATR 处加仓
- **减仓**：当实际 R/R 达到预测值的 80%，减半仓锁定利润

#### 评估方法（待实现）
- **主要指标**：MSE（Mean Squared Error）
- **辅助指标**：Top Decile R/R、平均 R/R、最大回撤
- **评估逻辑**：关注"盈利质量评估"，真突破 vs 假突破是光谱

### 2.3 压缩区突破策略

#### 模型配置
- **模型类型**：CatBoost Classifier
- **任务类型**：Multiclass Classification
- **输出**：{-1, 0, +1}（向下有效突破、假突破/回补、向上有效突破）
- **文件位置**：`src/time_series_model/strategies/compression_breakout/train.py`

#### 标签配置
- **标签类型**：三元标签（-1, 0, +1）
- **标签定义**：
  - `+1`：向上有效突破
  - `-1`：向下有效突破
  - `0`：假突破/回补
- **有效突破条件**：收盘站稳 + Vol > 20日均值
- **文件位置**：`src/time_series_model/strategies/labels/compression_breakout_label.py`
- **参数**：
  - `lookback_window`: 10
  - `confirmation_bars`: 3
  - `volume_lookback`: 20
  - `min_volume_ratio`: 1.0
  - `breakout_threshold`: 1.5

#### 特征配置
- **特征集**：压缩强度 + 突破触发 + 方向确认
- **核心特征**：
  - Bollinger Band Width 分位数（<10%）
  - ATR/MA(ATR) 比值（<0.8）
  - Spectrum 频谱平坦度
  - 最近 3 根 K 线 range 扩张率
  - Volume spike（>2σ）
  - CVD 净流入突变（Z-score > 2）
  - 突破首根 K 线 body/range 比
  - Take Buy Ratio 方向一致性
  - WPT 高频子带能量方向
- **文件位置**：`src/time_series_model/strategies/compression_breakout/features.py`

#### 回测方法（待实现）
- **方向**：直接由标签符号决定（+1 → 多，-1 → 空）
- **入场点**：突破确认 K 线收盘后
- **止损**：区间另一侧边界 或 突破点 ±1×ATR
- **止盈**：
  - 初始 TP = 区间高度（Measured Move）
  - 若后续 CVD 持续，可移动止盈
- **仓位**：`position_size = base_size if label ≠ 0 else 0`（0 不交易）
- **加仓**：❌ 通常不加仓（爆发后常回调）
- **减仓**：价格达到区间高度目标，减半仓

#### 评估方法（待实现）
- **主要指标**：准确率（Accuracy）
- **辅助指标**：F1-score（针对 minority class "有效突破"）、混淆矩阵
- **评估逻辑**：关注"突破有效性分类"，必须同时编码方向 + 真假

### 2.4 趋势跟踪策略

#### 模型配置
- **模型类型**：LightGBM Regressor
- **任务类型**：Regression
- **输出**：Rank（0~1 之间，收益率百分位）
- **文件位置**：`src/time_series_model/strategies/trend_following/train.py`

#### 标签配置
- **标签类型**：百分位标签（0~1）
- **标签定义**：未来 50 根 K 线收益率在滚动 200 根中的分位数
- **文件位置**：`src/time_series_model/strategies/labels/trend_following_label.py`
- **参数**：
  - `horizon`: 50
  - `rank_window`: 200
  - `min_periods`: 50

#### 特征配置
- **特征集**：趋势强度 + 资金验证 + 节奏感知
- **核心特征**：
  - Hurst 指数（>0.6）
  - ADX(14) > 25
  - WPT 低频子带斜率
  - 滚动 5D CVD 趋势（斜率）
  - MFE/MAE 比（历史趋势质量）
  - Volume 与 price 同向率
  - Hilbert 瞬时频率稳定性
  - Spectrum 主频周期长度
  - Mamba embedding（长程上下文）
- **文件位置**：`src/time_series_model/strategies/trend_following/features.py`

#### 回测方法（待实现）
- **方向**：由动量方向决定（ROC(20) > 0 → 做多，ROC(20) < 0 → 做空）
- **入场点**：趋势确认（如 ADX > 25 + Hurst > 0.6）
- **止损**：动态 ATR 止损（如 2×ATR）或 trendline break
- **止盈**：无固定止盈，靠 Rank 下降退出
  - 当最新 Rank < 0.4，平仓
- **仓位**：`position_size ∝ (label - 0.5)`（Rank=0.9 → 重仓；Rank=0.6 → 轻仓）
- **加仓**：✅ 每当 Rank 创新高（>0.95），加仓
- **减仓**：Rank 从高位回落至 0.7 以下，减仓 50%

#### 评估方法（待实现）
- **主要指标**：Rank IC（Spearman Rank Correlation）
- **辅助指标**：Top-Bottom Spread、平均 Rank、最大回撤
- **评估逻辑**：关注"趋势强度标准化"，适应不同波动率品种
