本文档详细说明：

2. **标签和训练目标如何根据策略设计**

## 一、标签如何影响IC计算和Pool B筛选

### 1.1 IC计算的本质

IC（Information Coefficient）衡量的是**特征与标签的相关性**。

**计算公式**（以Spearman rank correlation为例）：
```python
IC = spearmanr(factor_ranks, target_ranks).correlation
```

其中：
- `factor_ranks`：特征的rank（百分位排名）
- `target_ranks`：标签的rank（百分位排名）

**关键点**：IC计算**完全依赖于标签**，不同的标签会导致完全不同的IC值。

### 1.2 具体例子：`cvd_long`特征在不同策略中的IC差异

#### 场景：同一个特征`cvd_long`，不同的标签

假设在某个时间点：
- `cvd_long = -40000`（长期订单流负）
- 未来50根K线内：
  - MFE = 2.5 * ATR（最大有利偏移）
  - MAE = 1.0 * ATR（最大不利偏移）
  - 未来收益率 = +0.05（5%）

#### 在`sr_reversal_rr_reg_long`策略中

**标签定义**：
```python
rr_label = min(max(MFE / MAE, 0), 3.0)
# 在这个例子中：rr_label = min(max(2.5/1.0, 0), 3.0) = 2.5
```

**IC计算**：
- 计算`cvd_long`与`rr_label`的rank correlation
- 如果`cvd_long`负值大时，`rr_label`也高（R/R好）
- 那么IC可能为**正**（例如IC = 0.08）

**为什么IC为正？**
- `sr_reversal`关注"反转机会"
- 当订单流负但价格可能上涨时，这是"背离"信号
- 背离往往带来反转机会，R/R可能更好
- 因此`cvd_long`负值与高R/R正相关

#### 在`trend_following`策略中

**标签定义**：
```python
label = rank(future_return_50, window=200)
# 在这个例子中：如果future_return_50 = +0.05在200个样本中排名第150
# 那么label = 150/200 = 0.75
```

**IC计算**：
- 计算`cvd_long`与`label`（rank）的rank correlation
- 如果`cvd_long`负值大时，`label`低（趋势强度rank低）
- 那么IC可能为**负**（例如IC = -0.05）

**为什么IC为负？**
- `trend_following`关注"趋势强度"
- 当订单流负时，趋势强度应该降低
- 因此`cvd_long`负值与趋势强度rank负相关

### 1.3 Pool B筛选的完整流程

#### Step 1: 计算每个特征与标签的IC

```python
# 伪代码：factor-eval的核心逻辑
for feature in all_features:
    ic_series = []
    for window in rolling_windows:
        ic = spearmanr(
            feature.rank(pct=True),
            label.rank(pct=True)
        ).correlation
        ic_series.append(ic)
    
    ic_mean = mean(ic_series)
    ic_std = std(ic_series)
    ic_ir = ic_mean / ic_std  # Information Ratio
    
    if abs(ic_ir) >= 0.05 and abs(t_stat) >= 1.96:
        qualified_features.append(feature)
```

#### Step 2: 不同标签导致不同的IC值

**sr_reversal的标签（R/R）**：
- `cvd_long`的IC = 0.08（正相关）
- `vpin_exhaustion_scene_score`的IC = 0.12（正相关）
- `vp_poc_deviation`的IC = 0.06（正相关）
- `bb_width_normalized`的IC = 0.02（弱相关，可能被过滤）

**trend_following的标签（rank）**：
- `cvd_long`的IC = -0.05（负相关，但绝对值可能不够）
- `funding_ignition_score`的IC = 0.15（正相关）
- `bb_width_normalized`的IC = 0.08（正相关）
- `vpin_exhaustion_scene_score`的IC = -0.03（负相关，可能被过滤）

#### Step 3: 筛选出不同的Pool B

**sr_reversal的Pool B**（示例）：
```yaml
feature_pipeline:
  requested_features:
    - cvd_features_f                    # IC=0.08
    - vpin_exhaustion_scene_score       # IC=0.12
    - vp_poc_deviation                  # IC=0.06
    - trade_cluster_absorption_scene_score  # IC=0.09
    # bb_width_normalized被过滤（IC=0.02 < 阈值）
```

**trend_following的Pool B**（示例）：
```yaml
feature_pipeline:
  requested_features:
    - cvd_features_f                    # IC=-0.05（可能被过滤，取决于阈值）
    - funding_ignition_score             # IC=0.15
    - bb_width_normalized               # IC=0.08
    - macd_signal                       # IC=0.07
    # vpin_exhaustion_scene_score被过滤（IC=-0.03）
```

### 1.4 根源确实是标签不同

**完整因果链**：

```
策略语义
    ↓
标签定义（根据策略设计）
    ↓
IC计算（基于标签）
    ↓
Pool B筛选（基于IC/IR）
    ↓
特征组合搜索（基于Pool B）
    ↓
最终特征配置（策略特定）
```

**关键洞察**：
- **根源确实是标签不同**
- 标签定义了"优化目标"
- IC计算衡量"特征与优化目标的相关性"
- 因此不同标签 → 不同IC → 不同Pool B

## 二、标签和训练目标如何根据策略设计

### 2.1 标签设计的核心原则

标签设计必须**直接对应策略的核心目标**，而不是简单的"未来涨跌"。

### 2.2 四个策略的标签设计逻辑

#### sr_reversal_rr_reg_long：R/R标签

**策略核心目标**：在SR区域捕捉反转机会，获得高R/R（Risk/Reward）

**标签设计**：
```python
def compute_sr_reversal_rr_continuous_label(df):
    # 模拟入场后的路径
    mfe = max_favorable_excursion  # 最大有利偏移
    mae = max_adverse_excursion    # 最大不利偏移
    
    if mae == 0:
        return 0.0
    
    rr = mfe / mae
    return min(max(rr, 0), 3.0)  # 限制在[0, 3.0]
```

**为什么这样设计？**
- 策略的核心是"高R/R"，不是"绝对收益"
- 标签直接优化R/R比率
- 模型学习"什么情况下R/R会好"

**训练目标**：回归任务，最小化MSE（预测R/R vs 实际R/R）

#### sr_breakout：R/R标签（类似但关注突破）

**策略核心目标**：捕捉突破机会，获得高R/R

**标签设计**：与sr_reversal类似，但：
- 关注"突破"场景而非"反转"
- 可能需要额外的"突破确认"逻辑

**训练目标**：回归任务

#### compression_breakout：三元标签

**策略核心目标**：在压缩区捕捉突破，同时判断方向和真假

**标签设计**：
```python
def compute_compression_breakout_label(df):
    if compression_score > threshold:
        if price_breaks_up and volume_confirms:
            return +1  # 向上有效突破
        elif price_breaks_down and volume_confirms:
            return -1  # 向下有效突破
        else:
            return 0   # 假突破/回补
    else:
        return NaN     # 不在压缩区
```

**为什么这样设计？**
- 策略需要同时判断"方向"和"真假"
- 三元标签（-1/0/+1）同时编码这两个信息
- 0是"主动拒绝"，不是"不确定"

**训练目标**：多分类任务，最小化交叉熵

#### trend_following：百分位标签

**策略核心目标**：捕捉趋势的相对强度，适应不同波动率环境

**标签设计**：
```python
def compute_trend_following_label(df):
    future_return = df['close'].shift(-50) / df['close'] - 1
    label = future_return.rolling(window=200).rank(pct=True)
    return label  # [0, 1]之间的百分位
```

**为什么这样设计？**
- 策略需要"相对强度"，不是"绝对收益"
- 百分位标签对数据分布不敏感
- 可以跨品种/跨周期标准化

**训练目标**：回归任务，最小化MSE（预测rank vs 实际rank）

### 2.3 标签设计的决策树

```
策略类型？
    ↓
┌─────────────────────────────────────┐
│ 需要判断方向？                       │
│   Yes → 多分类标签（-1/0/+1）       │
│   No  → 继续                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 核心是R/R？                          │
│   Yes → 连续R/R标签（MFE/MAE）      │
│   No  → 继续                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 需要跨品种/跨周期？                  │
│   Yes → 百分位标签（rank）           │
│   No  → 绝对收益标签                 │
└─────────────────────────────────────┘
```

### 2.4 训练目标的选择

**标签类型决定训练目标**：

| 标签类型 | 训练目标 | 损失函数 | 模型类型 |
|---------|---------|---------|---------|
| 连续R/R | 回归 | MSE | XGBoost Regression |
| 三元标签 | 多分类 | Cross-Entropy | CatBoost Multiclass |
| 百分位标签 | 回归 | MSE | LightGBM Regression |
| 二元标签 | 二分类 | Binary Cross-Entropy | XGBoost Binary |

## 三、为什么迁移到分层架构，解决了什么问题

### 3.1 树模型架构的问题

#### 问题1：策略耦合

**树模型架构**：
```
特征 → 树模型 → 策略信号（entry/exit）
```

**问题**：
- 每个策略需要**独立的模型和特征选择**
- 新增策略 = 重新训练整个模型
- 策略之间无法共享知识

**具体例子**：
- `sr_reversal`训练了一个模型，输出"反转信号概率"
- `sr_breakout`训练了另一个模型，输出"突破信号概率"
- 两个模型**完全独立**，无法共享"订单流"等通用知识

#### 问题2：难以复用

**场景**：你想同时使用多个策略

**树模型方案**：
```python
# 需要运行多个模型
reversal_signal = reversal_model.predict(features)
breakout_signal = breakout_model.predict(features)
trend_signal = trend_model.predict(features)

# 如何组合？没有统一框架
if reversal_signal > 0.7:
    # 做反转
elif breakout_signal > 0.8:
    # 做突破
# ... 组合逻辑复杂
```

**问题**：
- 每个模型的输出语义不同（概率？信号强度？）
- 难以统一组合
- 没有统一的"市场状态表征"

#### 问题3：难以诊断

**场景**：策略表现不好，想定位问题

**树模型方案**：
- 只能看"模型预测 vs 实际标签"的准确率
- 无法知道是"特征失效"还是"策略逻辑失效"
- 无法知道"市场状态是否适合这个策略"

#### 问题4：难以扩展

**场景**：想新增一个策略

**树模型方案**：
- 需要重新设计标签
- 需要重新筛选特征（Pool B）
- 需要重新训练模型
- 需要重新设计组合逻辑

### 3.2 分层架构的解决方案

#### 新架构：NN多头路径原语 + Router + Execution

```
特征 → NN多头模型（路径原语）
    ↓
    ├─ dir (方向)
    ├─ mfe (最大有利偏移)
    ├─ mae (最大不利偏移)
    ├─ t_to_mfe (到达MFE的时间)
    └─ vol (波动率)
    ↓
Router (regime选择)
    ↓
    ├─ TREND
    ├─ MEAN
    └─ NO_TRADE
    ↓
Execution (策略执行)
    ↓
    ├─ TC (Trend Continuation)
    ├─ TE (Trend Expansion)
    ├─ FR (Failure Reversion)
    └─ ET (Exhaustion Turn)
```

### 3.3 分层架构解决了什么问题

#### 解决1：策略解耦

**新架构**：
- **NN模型**：只学习"市场路径原语"（dir/mfe/mae），不绑定具体策略
- **Router**：根据路径原语选择regime（TREND/MEAN/NO_TRADE）
- **Execution**：根据regime执行具体策略（TC/TE/FR/ET）

**优势**：
- 新增策略只需要**加Router逻辑**或**加Execution逻辑**
- 不需要重新训练NN模型
- 策略之间可以共享"路径原语"知识

**具体例子**：
```python
# NN模型输出（策略无关）
primitives = {
    'dir': 0.7,      # 方向概率
    'mfe_atr': 2.5,  # 最大有利偏移
    'mae_atr': 1.0,  # 最大不利偏移
    't_to_mfe': 20   # 到达MFE的时间
}

# Router根据primitives选择regime（策略无关）
if primitives['dir'] > 0.6 and primitives['mfe_atr'] > 2.0:
    regime = 'TREND'
elif primitives['dir'] < 0.4:
    regime = 'MEAN'
else:
    regime = 'NO_TRADE'

# Execution根据regime执行策略（策略特定）
if regime == 'TREND':
    if primitives['mfe_atr'] / primitives['mae_atr'] > 2.0:
        execute = 'TE'  # Trend Expansion
    else:
        execute = 'TC'  # Trend Continuation
```

#### 解决2：统一表征

**新架构**：
- 所有策略共享**相同的路径原语**
- 路径原语是"策略无关"的市场状态表征

**优势**：
- 可以统一比较不同策略
- 可以统一诊断问题
- 可以统一监控市场状态

**具体例子**：
```python
# 所有策略都看相同的primitives
primitives = nn_model.predict(features)

# 可以统一监控
if primitives['mfe_atr'] < 1.0:
    # 所有策略都应该谨慎（市场没有空间）
    risk_level = 'HIGH'

# 可以统一诊断
if primitives['dir']不稳定:
    # 问题在NN模型，不是某个具体策略
    diagnose_nn_model()
```

#### 解决3：可诊断性

**新架构**：
- **NN模型**：可以诊断"路径原语预测是否准确"
- **Router**：可以诊断"regime选择是否合理"
- **Execution**：可以诊断"策略执行是否有效"

**具体例子**：
```python
# 诊断NN模型
if primitives['mfe_atr']预测误差大:
    # 问题在特征或模型，不是策略逻辑
    retrain_nn_model()

# 诊断Router
if regime选择频繁切换:
    # 问题在Router逻辑，不是NN模型
    adjust_router_thresholds()

# 诊断Execution
if TC执行但实际是假突破:
    # 问题在Execution逻辑，不是NN或Router
    add_gate_rules()
```

#### 解决4：易于扩展

**新架构**：
- 新增策略 = 新增Router逻辑或Execution逻辑
- 不需要重新训练NN模型
- 可以复用现有的路径原语

**具体例子**：
```python
# 新增一个策略：Mean Reversion
# 只需要在Router和Execution中添加逻辑

# Router（新增regime）
if primitives['dir'] < 0.3 and primitives['mfe_atr'] < 1.5:
    regime = 'MEAN_REVERSION'

# Execution（新增策略）
if regime == 'MEAN_REVERSION':
    execute = 'MR'  # Mean Reversion
```

### 3.4 迁移前后的对比

#### 迁移前（树模型架构）

**问题场景**：想同时使用sr_reversal和sr_breakout

```
特征 → sr_reversal模型 → 反转信号
特征 → sr_breakout模型 → 突破信号

如何组合？没有统一框架
```

**问题**：
- 两个模型完全独立
- 输出语义不同（概率？信号强度？）
- 难以统一组合
- 新增策略需要重新训练

#### 迁移后（分层架构）

**解决方案**：
```
特征 → NN模型 → 路径原语（统一表征）
    ↓
Router → 选择regime（TREND/MEAN/NO_TRADE）
    ↓
Execution → 执行策略（TC/TE/FR/ET）
```

**优势**：
- 所有策略共享相同的路径原语
- 可以统一组合和监控
- 新增策略只需要加逻辑，不需要重新训练

### 3.5 具体例子：从树模型到分层架构的迁移

#### 场景：sr_reversal策略

**树模型方案**：
```python
# 训练
reversal_model = train_tree_model(
    features=pool_b_features,
    label=rr_label,  # R/R标签
    task='regression'
)

# 使用
reversal_signal = reversal_model.predict(features)
if reversal_signal > threshold:
    # 做反转交易
```

**问题**：
- 模型直接输出"反转信号"
- 无法知道"为什么"（是订单流？是结构？）
- 无法与其他策略统一组合

**分层架构方案**：
```python
# 训练NN模型（策略无关）
nn_model = train_nn_model(
    features=all_features,
    labels={
        'dir': direction_label,
        'mfe_atr': mfe_label,
        'mae_atr': mae_label
    }
)

# 使用
primitives = nn_model.predict(features)
# primitives = {'dir': 0.3, 'mfe_atr': 2.5, 'mae_atr': 1.0}

# Router选择regime
if primitives['dir'] < 0.4:
    regime = 'MEAN'

# Execution执行策略
if regime == 'MEAN':
    # 检查是否是反转机会
    if primitives['mfe_atr'] / primitives['mae_atr'] > 2.0:
        execute = 'FR'  # Failure Reversion
```

**优势**：
- 可以知道"为什么"（通过primitives）
- 可以与其他策略统一组合
- 可以统一诊断和监控

## 四、总结

### 4.1 完整因果链

```
策略语义
    ↓
标签设计（根据策略核心目标）
    ↓
IC计算（基于标签）
    ↓
Pool B筛选（基于IC/IR）
    ↓
特征组合搜索
    ↓
树模型训练（策略特定）
    ↓
[问题：策略耦合、难以复用、难以诊断、难以扩展]
    ↓
迁移到分层架构
    ↓
NN多头路径原语（策略无关）
    ↓
Router（regime选择）
    ↓
Execution（策略执行）
    ↓
[解决：策略解耦、统一表征、可诊断、易于扩展]
```

### 4.2 关键洞察

1. **根源确实是标签不同**：标签定义了优化目标，IC计算基于标签，因此不同标签导致不同Pool B

2. **标签设计必须对应策略核心目标**：不是简单的"未来涨跌"，而是策略真正关心的指标（R/R、方向+质量、相对强度等）

3. **分层架构解决了树模型的核心问题**：
   - 策略解耦：NN学习通用原语，Router/Execution处理策略逻辑
   - 统一表征：所有策略共享相同的路径原语
   - 可诊断性：可以分层诊断问题
   - 易于扩展：新增策略只需要加逻辑，不需要重新训练

### 4.3 迁移的价值

**树模型的价值**：
- 发现有用的规则和特征组合
- 验证策略逻辑的有效性
- 提供Gate/Detector的规则来源

**分层架构的价值**：
- 统一的市场状态表征
- 策略解耦和复用
- 更好的可诊断性和可扩展性

**两者结合**：
- 树模型发现规则 → 迁移到分层架构的Gate/Execution层
