# 标签逻辑检查报告

## 检查目标
检查 SR 反转标签的动态 R/R 检查逻辑是否适用于其他策略。

## 各策略标签分析

### 1. SR 反转标签 (`sr_reversal_label.py`)
**标签类型**：二元标签（0=失败, 1=成功）

**当前状态**：✅ 已修复
- 使用 `max_holding_bars=50` 作为上限
- **动态检查**：一旦满足 R/R 条件就停止，不需要等到 `max_holding_bars`
- **标签逻辑**：
  - TP 先于 SL 触达 → `1.0`（成功）
  - SL 先于 TP 触达 → `0.0`（失败）
  - 超时（未触达 TP 或 SL）→ `NaN`（排除训练）

**适用性**：✅ 完全适用

---

### 2. SR 突破标签 (`sr_breakout_label.py`)
**标签类型**：连续标签（实现 R/R = MFE/MAE）

**当前状态**：❌ 需要修复
- 使用固定的 `hold_bars=50`
- **问题**：扫描整个 `hold_bars` 期间计算 MFE/MAE，即使价格在早期就触达止损
- **影响**：如果价格在早期触达止损，继续扫描会改变 MAE 的计算，导致标签不准确

**修复建议**：
1. 将 `hold_bars` 改为 `max_holding_bars`（作为上限）
2. **动态检查**：一旦触达止损就停止扫描（因为 MAE 已经确定）
3. **优化逻辑**：
   - 如果触达止损：立即停止，计算 `realized_rr = 0.0`
   - 如果未触达止损：继续扫描直到 `max_holding_bars` 或达到目标 R/R

**适用性**：✅ 适用（需要修复）

---

### 3. 压缩区突破标签 (`compression_breakout_label.py`)
**标签类型**：三元标签（-1=向下突破, 0=假突破/回补, +1=向上突破）

**当前状态**：✅ 不需要动态 R/R 检查
- 使用 `lookback_window` 和 `confirmation_bars` 来确认突破
- **逻辑**：检测突破是否发生，而不是盈亏比
- **特点**：关注的是突破的有效性（价格站稳 + 成交量确认），而不是 R/R

**适用性**：❌ 不适用（标签性质不同）

---

### 4. 趋势跟踪标签 (`trend_following_label.py`)
**标签类型**：百分位标签（Rank，0.0~1.0）

**当前状态**：✅ 不需要动态 R/R 检查
- 使用固定的 `horizon=50` 来计算未来收益
- **逻辑**：计算未来收益在滚动窗口中的百分位排名
- **特点**：关注的是收益排名，而不是盈亏比

**适用性**：❌ 不适用（标签性质不同）

---

### 5. 通用 R/R 标签 (`label_utils.py` 中的 `compute_rr_label`)
**标签类型**：二元标签（0=失败, 1=成功）或连续标签（实现 R/R）

**当前状态**：✅ 已实现动态检查
- 使用 `max_holding_bars` 作为上限
- **动态检查**：一旦 TP 或 SL 触达就停止
- **逻辑**：
  - 二元标签：TP 先于 SL → 1.0，否则 → 0.0
  - 连续标签：计算 MFE/MAE，但一旦触达止损就停止扫描

**适用性**：✅ 完全适用（已正确实现）
