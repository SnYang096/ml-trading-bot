# 数据泄漏（Look-Ahead Bias）问题修复说明

## 问题发现

在4h时间框架的ML+Volatility模型中，胜率达到92.31%，Sharpe Ratio达到2.36，这个结果异常高，可能存在未来函数（look-ahead bias）问题。

## 检查结果

### 🔴 严重问题：特征计算在划分训练/测试集之前使用了fit=True

**位置**: `scripts/diagnostics/sr_reversal_model_comparison.py` 第1338-1351行

**问题描述**:
```python
# ❌ 错误做法：在整个数据集上fit，然后才划分训练/测试集
df_features = strategy_runner.run_feature_pipeline(
    df_raw,
    feature_loader=feature_loader,
    pipeline_cfg=strategy_cfg.features,
    fit=True,  # 使用了整个数据集（包括测试集）来拟合特征
)

# 然后才划分训练/测试集
split_idx = int(len(df_features) * (1 - args.test_size))
df_train = df_features.iloc[:split_idx].copy()
df_test = df_features.iloc[split_idx:].copy()
```

**影响**:
- 特征的统计信息（如标准化、归一化、滚动窗口统计等）使用了测试集的数据
- 这导致测试集的信息被"泄漏"到特征计算中
- 模型在测试集上的表现会被高估

**修复方案**:
```python
# ✅ 正确做法：先划分训练/测试集，然后在训练集上fit，在测试集上transform
split_idx = int(len(df_raw) * (1 - args.test_size))
df_raw_train = df_raw.iloc[:split_idx].copy()
df_raw_test = df_raw.iloc[split_idx:].copy()

# Fit on training set only
df_train = strategy_runner.run_feature_pipeline(
    df_raw_train,
    feature_loader=feature_loader,
    pipeline_cfg=strategy_cfg.features,
    fit=True,  # 只在训练集上fit
)

# Transform test set using fitted features
df_test = strategy_runner.run_feature_pipeline(
    df_raw_test,
    feature_loader=feature_loader,
    pipeline_cfg=strategy_cfg.features,
    fit=False,  # 不在测试集上fit！
)
```

## 其他检查项

### ✅ 波动率标签计算（正确）

波动率模型训练时使用的 `future_volatility_label` 是**正确的**，因为：
- 标签（label）可以使用未来信息，这是监督学习的标准做法
- 在训练时，我们使用未来波动率作为标签来训练模型
- 在评估时，我们使用**预测的波动率**（不是未来波动率）来计算R/R

### ✅ 回测评估逻辑（正确）

在 `evaluate_ml_volatility_model` 函数中：
- 使用 `vol_model.predict(X)` 来预测波动率（第796行）
- 使用 `compute_adaptive_rr_label_with_predicted_vol` 来计算标签（第859行）
- 这些都是基于**预测值**，不是未来值，所以是正确的
