# RSI Inf 值问题诊断报告

## 问题描述

训练日志显示：
- 训练集中 `rsi` 特征有 **70 个 inf 值**
- 出现在 2025-02-01 附近的时间点

## 测试结果

### 1. 直接计算 RSI
- ✅ **无 inf 值**：直接使用 `BaselineFeatureEngineer.compute_rsi()` 计算，结果正常
- ✅ **无 inf 值**：使用 Talib 直接计算，结果正常
- ✅ **无 inf 值**：RSI 滚动 Z-score（w50, w288, w500）计算正常

### 2. 数据质量
- ✅ 价格数据：无 inf，无 NaN
- ✅ RSI 结果：14 个 NaN（正常，因为需要至少 14 个周期才能计算）

### 3. 可能的原因

由于直接计算 RSI 没有 inf，问题可能出现在：

1. **特征计算流程的其他步骤**
   - 特征交互（如 RSI × 其他特征）
   - 特征归一化或标准化
   - 特征组合或衍生特征

2. **数据过滤或后处理**
   - 在特征计算后的数据清理步骤
   - 特征选择或筛选过程

3. **特征管道中的特定操作**
   - 某些特征计算可能修改了 RSI 列
   - 特征依赖关系导致的级联问题

## 建议

1. **检查特征计算流程**
   - 在特征计算的每个步骤后检查 RSI 列
   - 特别关注特征交互和组合步骤

2. **添加调试日志**
   - 在特征计算的关键步骤添加 inf 值检查
   - 记录 inf 值出现的位置和原因

3. **检查特征依赖**
   - 确认是否有其他特征计算修改了 RSI 列
   - 检查特征依赖关系是否正确

4. **数据过滤逻辑**
   - 检查 `drop_inf_rows` 函数是否正确处理了 RSI
   - 确认数据过滤不会意外引入 inf
