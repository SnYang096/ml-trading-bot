#### 你的差距：

## ⚙️ 二、阶段 2：特征比较与评估层

**目标**：比较 t-digest 与 小波 在预测性能和稳定性上的差异。

### 1️⃣ 离线回测评估

使用你现有的 backtest 框架，对比两组特征的信号触发结果。

| 指标                  | 含义           | 目标         |
| ------------------- | ------------ | ---------- |
| 精确率（Precision）      | 信号后价格方向正确比例  | 小波能否减少噪声触发 |
| 稳定性（Stability）      | 信号在不同波段下的一致性 | 是否更平滑      |
| 滞后性（Lag）            | 结构识别时间差      | 小波应更提前     |
| 方差（Signal Variance） | 信号波动程度       | 低噪声特征更稳定   |

> 如果小波能提前识别趋势阶段变化（低滞后 + 高精确），即可证明其结构性优势。

## 一、总体思想（一句话）

用**多尺度时频特征 + 状态感知（regime）**驱动**高置信号的稀疏下单**，配合**严苛风控与动态仓位**，把交易变成“少而稳、每笔期望值正”的可叠加策略池。

---

## 二、四层模块化架构（精简为中低频实盘版）

1. **数据层（Data） — 简洁且可靠**

   * 时序来源：1m/5m/15m K线为主；可选日内成交量、T&S/Book snapshot（若成本允许）。
   * 存储：Parquet+S3（历史），ClickHouse/QuestDB（快速查询）。线上用轻量消息队列（Kafka/Rabbit）缓冲。
   * 校验：时序对齐、重复/缺失自动补全策略（线性/前向填充），单点异常回退到上 N 条均值。

2. **感知层（Features） — 稳健高置信因子集**

   * 结构因子：ZigZag 顶点/支撑阻力、K 线穿越（确认后）
   * 压缩因子：SWT 小波 CompressionScore（短窗口 64~256，5~15min 对应窗口）
   * 订单流简化：成交方向偏向 CVD-like（按时间窗累积），若无 L2 用成交量/买卖比代替
   * 统计因子：ATR（短/中期）、VWAP/偏离、成交量倍数（相对移动平均）
   * 信息论：滑窗熵（Shannon）或 returns 的分位数（tdigest）

   先做“少而精”的因子组合，避免高维过拟合。

3. **状态层（Regime / Estimation） — 决策门控**

   * Kalman 对低频分量做在线平滑（去噪）。
   * HMM（3 状态：Quiet / Building / Trending）或 LightGBM 的状态分类器（概率输出）做市场 regime 判断。
   * 规则：当 state ≠ “Trending” 时禁止追单型策略，只允许均值回归/压缩突破低频策略（降低回撤）。

4. **决策与风控层（Signals & Sizing）**

   * 信号融合：规则优先（结构确认、成交量验证）+ ML 打分（LightGBM 输出置信度）
   * 阈值：贝叶斯优化或 walk-forward 调整；阈值随 regime 动态修正（在 Quiet 阶段提高阈值）
   * 仓位 sizing：固定风险百分比（参见下方公式），分批入场（网格/分片）。
   * 风控：单笔止损（ATR × k）、日内总损失限制、回撤触发停表、最大未平仓量。

5. **执行层（Execution）**

   * 只做限价/IOC（若流动性差）或TWAP分片（若单笔可能冲击市场）。
   * 严格模拟滑点/手续费：回测必须包含真实 cost 模型（percent + fixed + slippage per spread）。

## 三、核心设计原则（工程取舍）

* **薄实时厚批处理**：复杂时频（CWT/EMD）做离线/日级批处理，在线只计算 SWT + a few aggregates，保证延迟与稳定性。
* **少量高置信信号**：优先提高单笔成功率 > 提高交易频率。
* **保守仓位**：小资金优先“资金防爆”，低杠杆，高胜率，复利增长。
* **严格验证流程**：PurgedKFold / walk-forward + transaction-level成本注入 + Monte-Carlo market scenarios。

## 四、详细实现要点（技术细节与参数建议）

### 数据与时间框架

* 主时间框架：**5m 或 15m**（首选 15m 做思考）；子尺度窗口：1m/5m 用于短期冲击检测。
* 特征窗口：

  * SWT window: 128~256（对应 5m => 10-21 小时滑窗可观测结构）
  * ATR period: 14（适配 5m/15m）
  * Volume MA: 20

### Wavelet（SWT）配置（在线友好）

* 母小波：`db4` 或 `db6`，decomp level=4~6。
* compression_score = low_freq_energy / total_energy，阈值需用 walk-forward 校准。
* 滞后处理：采用 causal variant 或用延迟补偿（向后移出信号产生一步延迟，显式记录）。

### HMM / Regime

* HMM 3 states（Quiet / Building / Trending），输入维度： [compression_score, hl_ratio, volume_ratio, momentum]。
* 输出为 state probabilities，用于动态阈值。若 HMM 安装困难，替代：LightGBM 概率分类器 + 阈值映射。

### 因子融合与模型

* 主模型：**LightGBM**（能解释、训练快），目标为短期方向 +置信度（回归概率/score）。
* 训练细节：

  * PurgedKFold 或 时间序列分割（避免未来泄露）
  * Walk-forward CV：滚动窗口，训练窗口 6~12 个月，测试窗口 1~3 个月
  * 特征重要度与 SHAP 分析排除过拟合因子
* 可选：用 LSTM/Transformer 做短期序列建模，但先做 LightGBM 保证稳定性。

### 信号 & 进出场逻辑（示例）

* **入场条件（示例，可调）**：

  1. structure_confirmed（ZigZag 顶点穿越且收盘确认）
  2. compression_score > T1（表示低频能量占优）且短期 momentum 与方向一致
  3. state == Trending 或（state == Building 且 volume_ratio > vol_thresh）
  4. LightGBM score > 0.7（置信度）
* **出场**：

  * 固定止损：`stop = entry_price - sign * k * ATR`（k=1.5~2.5）
  * 盈利目标：多级分批平仓（例如 50% at 1×ATR profit, 50% trailing by 0.8×ATR）
  * 若 regime 变差（prob(state_trending) 降低），立即缩减仓位或全部平仓。

### 仓位/资金管理（严格）

* 单笔风险（绝对） ≤ 0.5% 账户净值（保守）
* 风险基于 ATR：`position_size = (risk_per_trade * equity) / (ATR * tick_value * stop_multiplier)`
* 账户级别：日亏损阈值（例如 3%）触发全天停表；回撤阈值（如 10%）触发冷却与策略复盘。

公式示例（美元计或合约点）：

```
dollar_risk = equity * risk_pct   # e.g., 0.005 * equity
stop_distance = ATR * k           # k = 1.5~2
position_size = floor(dollar_risk / (stop_distance * dollar_per_point))
```

## 五、回测/验证策略（必须严格）

1. **成本模型**：手续费、滑点（spread-based or fixed slippage per order）, partial fills, rejection prob.
2. **Walk-forward CV**：训练窗口 → 验证窗口（滚动），保存每次测试结果。
3. **Purging**：避免 label leakage（事件窗口内 purge）
4. **Stress tests**：历史 stress 时段（如 2020-03）、Monte-Carlo price paths（加噪音、流动性冲击）
5. **统计显著性**：bootstrap returns，报告置信区间、p-values。
6. **Transaction-level logging**：在回测记录每笔委托、成交、滑点、拒单，方便实盘复现。

关键 KPI：年化收益、最大回撤（目标 <15%）、胜率（>55%）、平均盈亏比、Sharpe、Sortino、Calmar。对小资金更看重最大回撤与回撤恢复时间。
