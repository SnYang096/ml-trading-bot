# bpc，me，fer的方向如何定，用什么特征比较好？ 
这是一个**结构级问题**，不是特征堆叠问题。

你现在三类：

* **BPC（TC）**：趋势内延续
* **ME**：动能扩张
* **FER**：失败反转

方向（long / short）绝对不能用同一套逻辑。

因为三者的“方向因果来源”完全不同。

---

# 一句话总结

| Archetype | 方向来自哪里？ |
| --------- | ------- |
| ME        | 动能方向    |
| BPC       | 结构方向    |
| FER       | 失败方向    |

如果混用，会导致 Router 边界崩塌。

---

# 一、ME 的方向 —— 来自动能

ME 的本质是：

> 钱在加速进来

方向必须来自：

### ✅ 动能持续性

适合的特征：

* wpt_energy_slope
* 连续 directional cluster
* 连续 imbalance 同向
* delta rolling sum > 0（或 < 0）
* price velocity percentile

核心判断：

```
if acceleration > threshold
  direction = sign(acceleration)
```

不要用：

* SR 方向
* HTF MA 方向
* 结构 break 方向

ME 方向一定是“现在在冲的方向”。

否则就不是 ME。

---

# 二、BPC（TC）的方向 —— 来自结构

BPC 本质：

> 结构突破后回踩确认

方向应该来自：

### ✅ 结构突破方向

适合特征：

* sr_structure.breakout_direction
* 过去 swing high/low 结构破坏
* volume_profile acceptance side
* 价格位于 value area 上方 / 下方

核心判断：

```
direction = breakout_direction
```

不要用：

* 当下 delta
* wick
* absorption

BPC 是结构逻辑，不是瞬时流逻辑。

---

# 三、FER 的方向 —— 来自失败

FER 方向最特殊。

它永远是：

> 失败方向的反向

适合特征：

* false_breakout direction
* liquidity_sweep side
* aggressive_but_stuck side
* trapped_cluster side

逻辑是：

```
direction = - failed_impulse_direction
```

不要用：

* HTF trend
* MA 方向
* WPT 当前值

FER 的方向必须来自：

> 哪一边失败了

---

# 四、三者方向来源对比（非常关键）

| Archetype | 时间维度 | 因果来源 | 特征类型                         |
| --------- | ---- | ---- | ---------------------------- |
| ME        | 当前   | 加速   | 能量 / delta / velocity        |
| BPC       | 过去结构 | 突破   | SR / VP / breakout           |
| FER       | 刚发生  | 失败   | absorption / sweep / trapped |

三者方向来源不同，是你系统正交的根基。

---

# 五、最小实现建议（工程可落地）

你可以强制：

```
if archetype == ME:
    direction = sign(wpt_energy)

if archetype == TC:
    direction = sr_breakout_direction

if archetype == FER:
    direction = - failed_direction
```

不要让模型学 direction。

direction 应该是 deterministic 的。

模型学 timing / confidence 就够了。

---

# 六、一个非常重要的稳定性建议

千万不要这样：

* ME 方向来自 HTF
* TC 方向来自 delta
* FER 方向来自 RSI

那会变成统计拼凑。

方向必须是 archetype 内生的。

---

> BPC：跟结构走
> FER：跟失败走

三者绝对不能共用 direction head。
