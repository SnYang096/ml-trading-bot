# 数据泄露检查和最佳实践

## 概述

本文档详细说明如何验证机器学习训练流程中是否存在数据泄露（Look-ahead Bias），并提供最佳实践。

## 数据泄露的常见类型

### 1. 特征计算中使用未来信息

**❌ 错误示例 - 对整个序列做小波变换:**
```python
# 这会使用整个序列的信息，包括未来数据！
def compute_wavelet_wrong(data):
    coeffs = pywt.wavedec(data, 'db4', level=4)
    # 返回的特征会包含未来信息
    return compute_features(coeffs)
```

**✅ 正确示例 - 使用滑动窗口:**
```python
def compute_wavelet_safe(data, window=100):
    """逐点计算小波特征，只使用历史数据"""
    features = []
    for i in range(len(data)):
        if i < window:
            features.append(np.zeros(n_features))  # 窗口不足
        else:
            # 关键：只使用 i-window 到 i 的数据（不包括i）
            window_data = data[i - window:i]
            coeffs = pywt.wavedec(window_data, 'db4', level=4)
            features.append(compute_features(coeffs))
    return np.array(features)
```

### 2. 归一化使用测试集信息

**❌ 错误示例:**
```python
# 在整个数据集（包括测试集）上fit scaler
all_data = np.concatenate([train_data, test_data])
scaler = StandardScaler()
scaler.fit(all_data)  # 这会泄露测试集的统计信息！

train_scaled = scaler.transform(train_data)
test_scaled = scaler.transform(test_data)
```

**✅ 正确示例:**
```python
# 只在训练集上fit scaler
scaler = StandardScaler()
train_scaled = scaler.fit_transform(train_data)  # fit + transform
test_scaled = scaler.transform(test_data)         # 只transform
```

### 3. 累计指标的陷阱

**⚠️ 需要注意的情况 - CVD（累计成交量差）:**
```python
# CVD是累计指标
cvd = (buy_qty - sell_qty).cumsum()

# 问题：如果训练和测试数据分开计算，CVD值会不连续
train_cvd = train_data['delta'].cumsum()  # 从0开始累计
test_cvd = test_data['delta'].cumsum()    # 又从0开始累计！

# 解决方案1：使用变化率
cvd_change = cvd.diff()

# 解决方案2：使用滚动窗口内的累计
rolling_cvd = delta.rolling(window=100).sum()

# 当前实现：每个月独立计算CVD（安全）
# 因为我们不跨月份合并数据
```

### 4. 标签创建

**✅ 标签应该使用未来信息（这是正常的）:**
```python
# 创建标签时，应该使用未来收益
df['future_return'] = df['close'].shift(-forward_bars) / df['close'] - 1
df['label'] = (df['future_return'] > threshold).astype(int)

# 关键：训练时要删除包含NaN标签的行
df = df.dropna()
```

## 验证方法

### 方法1: 代码审查

检查所有特征计算是否满足：
- 滑动窗口使用 `data[i-window:i]` 而非 `data[i-window:i+1]` 或 `data[:i]`
- 归一化只在训练集fit
- 没有使用 `.shift(-n)` 创建特征（除了标签）

### 方法2: 时序分割测试

```python
# 将数据按时间分为前后两部分
mid_point = len(data) // 2
train_data = data[:mid_point]
test_data = data[mid_point:]

# 在训练集上fit特征工程器
df_train, engineer = engineer_features(train_data, None, fit=True)

# 在测试集上transform
df_test, _ = engineer_features(test_data, engineer, fit=False)

# 检查：测试集的第一个样本不应该依赖训练集的数据
# 除了scaler的均值/标准差
```

### 方法3: 前向验证

```python
# 逐点计算特征，验证每个时刻的特征只依赖历史数据
for i in range(100, len(data)):
    # 只使用前i个数据点
    historical_data = data[:i]
    
    # 计算第i个时刻的特征
    features_i = compute_features(historical_data)
    
    # 验证：features_i 应该只依赖 data[:i]，不依赖 data[i:]
```

## 常见问题

### Q1: 为什么滑动窗口用 `[i-window:i]` 而不是 `[i-window:i+1]`？

A: 在训练时，我们需要预测未来（如预测第i+1时刻的收益）。如果特征包含第i时刻的数据，可能会产生信息泄露。使用 `[i-window:i]` 确保只用历史数据。

但实际上，在很多场景下，使用 `[i-window:i+1]` 也是可以的，只要标签是 `shift(-forward_bars)`，就不会泄露。关键看你的标签定义。

### Q2: TA-Lib的指标会不会有数据泄露？

A: TA-Lib的所有指标都是因果的（causal），只使用历史数据。例如：
- `EMA`: 指数移动平均
- `RSI`: 相对强弱指标
- `MACD`: 移动平均收敛散度

这些都是安全的，不会使用未来信息。

### Q3: 为什么CVD用cumsum()是安全的？

A: 在我们的实现中，每个月的数据文件独立计算CVD。训练和测试不会跨文件合并，所以不会泄露。

但如果你要合并多个月的数据，需要确保CVD的连续性，或者使用CVD的变化率而非绝对值。

### Q4: Hilbert变换会不会泄露未来信息？

A: `scipy.signal.hilbert` 理论上可能使用整个序列进行FFT。在我们的实现中，已经改为滚动窗口计算，每次只对窗口内的数据做Hilbert变换，因此是安全的。

### Q5: 如何验证我的实现没有数据泄露？

A: 三步验证法：
1. **代码审查**: 检查所有特征计算的循环和索引
2. **时序测试**: 前后半数据分开训练，检查性能是否合理
3. **前向验证**: 逐点计算，确认每个时刻只依赖历史

运行 `verify_no_data_leakage.py` 可以自动完成大部分检查。
