# 三策略统一实盘启动命令

## 🚀 快速启动指南

### 1. 环境准备
```bash
cd /home/yin/trading/ml_trading_bot
source ~/.bashrc  # 或者 source ~/.zshrc
```

### 2. 数据准备（必需）
```bash
# 准备warmup数据（建议6个月历史数据）
bash live/scripts/prepare_warmup_ticks.sh highcap 6 --from-local

# 或者如果缺少数据，使用在线下载
bash live/scripts/prepare_warmup_ticks.sh highcap 6
```

### 3. 观察模式启动（首次推荐）
```bash
# 单币种观察
MLBOT_MIN_TICKS_REQUIRED=20160 PYTHONPATH=. python scripts/run_three_strategies_live.py --mode observe --symbols BTCUSDT

# 多币种观察
MLBOT_MIN_TICKS_REQUIRED=20160 PYTHONPATH=. python scripts/run_three_strategies_live.py --mode observe --symbols BTCUSDT,ETHUSDT
```

### 4. 实盘模式启动
```bash
# 单币种实盘
MLBOT_MIN_TICKS_REQUIRED=20160 PYTHONPATH=. python scripts/run_three_strategies_live.py --mode live --symbols BTCUSDT --trade-size 0.01

# 多币种实盘
MLBOT_MIN_TICKS_REQUIRED=20160 PYTHONPATH=. python scripts/run_three_strategies_live.py --mode live --symbols BTCUSDT,ETHUSDT --trade-size 0.01
```

## 📊 策略优先级

三策略竞争同一交易槽位，按以下优先级排序：
1. **FER (Fading Efficiency Reversion)** - 效率反转策略
2. **ME (Market Efficiency)** - 市场效率策略  
3. **BPC (Breakout Pattern Confirmation)** - 突破模式确认策略

## ⚙️ 关键配置

### 环境变量
- `MLBOT_MIN_TICKS_REQUIRED=20160` - VPIN特征计算所需最少tick数
- `MLBOT_LIVE_STORAGE_BASE=live/highcap/data` - 数据存储路径
- `MLBOT_MAX_SLOTS=2` - 最大持仓数量

### 风控参数
- **最大持仓数**: 2个交易槽位
- **单笔交易量**: 由`--trade-size`参数控制
- **策略竞争**: PCM仲裁机制，高优先级策略优先进入

## 🛡️ 安全建议

### 首次运行
1. **务必先用观察模式运行24小时**，确认系统稳定性
2. **从小仓位开始**，建议初始`--trade-size 0.01`
3. **密切监控日志**，关注异常和错误信息
4. **验证信号质量**，确认交易信号符合预期

### 监控指标
- 特征计算延迟（应在秒级内完成）
- 订单执行成功率
- 策略信号频率和质量
- 账户资金和持仓变化

## 🚨 紧急处理

### 停止运行
```bash
# 优雅停止
Ctrl+C

# 强制停止
pkill -f "run_three_strategies_live.py"
```

### 故障排查
- **tick数据不足**: `bash live/scripts/prepare_warmup_ticks.sh highcap 1 --fill-gap`
- **连接问题**: 检查网络和API密钥
- **特征计算错误**: 检查warmup数据完整性和存储路径
- **内存溢出**: 监控系统资源使用情况

## 💡 提示

- **最佳实践**: 建议在观察模式下运行至少24小时，确认系统稳定后再开启实盘交易
- **性能监控**: 关注特征计算时间和系统资源使用情况
- **日志记录**: 所有交易信号和执行都会被记录，便于后续分析
- **灵活配置**: 可随时调整交易币种和交易量，无需重启系统
