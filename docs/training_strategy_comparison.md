# 多Symbol训练策略对比

## 问题：是否应该将不同加密货币symbol放在一起训练？

### 方法1：混合训练（Multi-Symbol Training）
将所有symbol的数据混合在一起训练一个模型。

#### 优点：
1. **数据量更大**：多个symbol的数据可以显著增加训练样本
2. **学习通用模式**：可能学到跨资产的一般交易模式（如趋势、反转等）
3. **模型泛化能力**：在数据稀缺的symbol上可能表现更好
4. **维护成本低**：只需维护一个模型

#### 缺点：
1. **价格尺度差异**：不同symbol的价格水平差异巨大（BTC $50k vs SOL $100）
2. **波动性差异**：不同资产的波动性差异很大，即使使用ATR归一化也可能残留差异
3. **市场相关性**：不同资产的市场行为可能不同（BTC vs 山寨币）
4. **特征混淆**：模型可能无法区分不同资产的特征，导致预测能力下降
5. **标签分布不均**：不同资产的趋势方向可能不同，导致标签分布混乱

### 方法2：单独训练（Per-Symbol Training）
每个symbol训练独立的模型。

#### 优点：
1. **针对性更强**：模型专门学习该资产的特征和行为
2. **特征一致性**：所有特征都基于同一资产的价格尺度，特征更有意义
3. **避免混淆**：模型不会混淆不同资产的特征
4. **更易解释**：模型学到的规律更容易解释和理解

#### 缺点：
1. **数据量有限**：单个symbol的数据可能不足以训练复杂模型
2. **维护成本高**：需要为每个symbol维护独立的模型
3. **无法共享知识**：无法利用其他资产学到的通用模式

## 技术分析

### 当前Baseline特征工程的特点：

1. **ATR归一化**：使用ATR（平均真实波幅）对距离特征进行归一化
   - `sr_dist_high_s = (close - roll_high_s) / (atr + eps)`
   - 这有助于减少价格尺度的影响

2. **百分比特征**：使用百分比/分位数特征
   - `atr_percentile`：ATR在滚动窗口内的百分位排名
   - `volume_percentile`：成交量的百分位排名
   - 这些特征对价格尺度不敏感

3. **相对特征**：大部分特征都是相对值而非绝对值
   - 压缩比率、价格熵等

### 理论上可以混合训练的原因：

1. **特征已归一化**：ATR归一化和百分比特征使特征对价格尺度不敏感
2. **相对特征**：大部分特征都是相对值，不依赖绝对价格

### 仍然建议分开训练的原因：

1. **市场行为差异**：
   - BTC作为市场领导者，其行为模式与山寨币不同
   - 山寨币的波动性、趋势持续性、反转模式都可能不同

2. **标签分布差异**：
   - 不同资产在同一时期的趋势方向可能不同
   - 混合训练可能导致标签分布混乱，影响模型学习

3. **特征意义差异**：
   - 虽然特征已归一化，但不同资产的"压缩"、"突破"等概念的实际含义可能不同
   - 例如：BTC的5%波动和山寨币的5%波动在实际交易意义上是不同的

## 推荐方案

### 方案1：完全分开训练（推荐）⭐⭐⭐⭐⭐
**适用场景**：有足够的数据，希望获得最佳性能

```bash
# 为每个symbol训练独立模型
make baseline-train SYMBOL=BTCUSDT ...
make baseline-train SYMBOL=ETHUSDT ...
make baseline-train SYMBOL=SOLUSDT ...
```

**优点**：
- 模型针对性最强
- 性能通常最好
- 易于调优和解释

**缺点**：
- 需要为每个symbol维护模型
- 数据量小的symbol可能过拟合

### 方案2：分层训练（推荐）⭐⭐⭐⭐
**适用场景**：既有BTC等主流币，也有小币种

**策略**：
1. **主流币分开训练**：BTC、ETH等数据量大的symbol单独训练
2. **小币种混合训练**：将相似的小币种（如山寨币）放在一起训练

**实现**：
```python
# 主流币：单独训练
make baseline-train SYMBOL=BTCUSDT ...
make baseline-train SYMBOL=ETHUSDT ...

# 小币种：混合训练（需要修改代码支持多symbol）
# 可以创建一个symbols列表，合并数据后训练
```

### 方案3：混合训练（谨慎使用）⭐⭐
**适用场景**：数据量非常小，或者只是做概念验证

**注意事项**：
1. 确保所有symbol的特征工程一致
2. 考虑添加symbol作为特征（one-hot编码）
3. 监控不同symbol的表现差异

**实现**（需要修改代码）：
```python
# 添加symbol特征
symbols = ['BTCUSDT', 'ETHUSDT', 'SOLUSDT']
for symbol in symbols:
    data = load_data(symbol)
    data['symbol_btc'] = (symbol == 'BTCUSDT').astype(int)
    data['symbol_eth'] = (symbol == 'ETHUSDT').astype(int)
    # ... 合并所有数据
```

## 实践建议

### 1. 先分开训练（基准）
```bash
# 为每个symbol训练基准模型
make baseline-train SYMBOL=BTCUSDT ...
make baseline-train SYMBOL=ETHUSDT ...
```

### 2. 对比性能
- 比较每个symbol的CV和OOS性能
- 如果某个symbol性能较差（可能是数据量不足），考虑混合训练

### 3. 如果考虑混合训练
- 先测试：将BTC和ETH混合训练，对比与单独训练的性能
- 如果混合训练性能更好或接近，可以考虑扩大
- 如果性能下降，回到分开训练

### 4. 监控指标
- **CV Accuracy**：交叉验证准确率
- **OOS Accuracy**：样本外准确率（最关键）
- **不同symbol的表现差异**：如果混合训练，监控每个symbol的表现

## 代码支持

当前代码已经支持：
1. ✅ 每个symbol单独训练（通过`--symbol`参数）
2. ✅ 自动生成每个symbol的报告
3. ✅ 汇总报告对比不同配置

如果需要混合训练，可以：
1. 修改`_collect_files`函数，支持多个symbol
2. 在特征工程中添加symbol标识（one-hot编码）
3. 修改标签创建逻辑，确保不同symbol的标签分布合理

## 结论

**推荐：分开训练每个symbol**

原因：
1. 当前特征虽然已归一化，但不同资产的市场行为差异仍然很大
2. 分开训练性能通常更好，且更易解释和维护
3. 如果某个symbol数据不足，可以通过：
   - 使用更长的历史数据
   - 使用更简单的模型
   - 或者将该symbol与其他相似资产混合训练

**特殊情况**：
- 如果某个symbol的数据量非常小（<1000条），可以考虑与相似资产混合训练
- 如果只是做概念验证，混合训练可以快速验证思路
