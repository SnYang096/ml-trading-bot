# LightGBM 多资产训练中的归一化问题

## 🤔 常见误解

**误解**: "LightGBM 是基于树的模型，不需要归一化，所以多资产训练也不需要归一化"

**实际情况**: 这个说法**部分正确，但不够完整**。需要区分两个场景：

---

## 📊 两种场景对比

### 场景 1: 单资产训练

**LightGBM 确实不需要归一化**

- ✅ 树模型对特征尺度不敏感
- ✅ 不会影响模型收敛
- ✅ 不会影响特征重要性计算
- ✅ 模型性能基本不受影响

**示例**:
```python
# 单资产训练：BTC 数据
features = {
    'sma_5': [50000, 50100, 50200],      # 价格范围：50000-50200
    'volume': [1000, 1200, 1100],        # 成交量范围：1000-1200
    'rsi': [50, 55, 60]                  # RSI 范围：50-60
}
# ✅ 不需要归一化，LightGBM 可以正常工作
```

---

### 场景 2: 多资产训练（跨资产）

**LightGBM 仍然需要归一化**

**原因不是为了让模型收敛，而是为了让模型学习跨资产的通用模式**

#### 问题 1: 数值范围差异巨大

```python
# 多资产训练：BTC + ETH + SOL
features = {
    'sma_5': [
        # BTC
        50000, 50100, 50200,  # 范围：50000-50200
        # ETH
        3000, 3010, 3020,      # 范围：3000-3020
        # SOL
        100, 101, 102           # 范围：100-102
    ],
    'volume': [
        # BTC
        1000, 1200, 1100,      # 范围：1000-1200
        # ETH
        5000, 6000, 5500,      # 范围：5000-6000
        # SOL
        10000, 12000, 11000    # 范围：10000-12000
    ]
}
```

**如果不归一化会发生什么？**

1. **模型会被数值大的资产主导**
   - BTC 的 `sma_5` (50000) 会主导 ETH (3000) 和 SOL (100)
   - 模型会学习"BTC 的价格模式"，而不是"通用的价格模式"

2. **无法学习跨资产模式**
   - 模型无法理解"BTC 的 2% 涨幅"和"SOL 的 2% 涨幅"是相同的信号
   - 模型会认为 BTC 的 100 点变化和 SOL 的 100 点变化意义相同（实际上完全不同）

3. **特征重要性被扭曲**
   - 数值大的特征会被认为"更重要"
   - 实际上，相对变化（百分比）才是关键

#### 问题 2: 特征分布不同

```python
# 不同资产的特征分布差异巨大
BTC: {
    'atr': 500,        # ATR 范围：400-600
    'volume': 1000     # 成交量范围：800-1200
}

SOL: {
    'atr': 2,          # ATR 范围：1.5-2.5
    'volume': 10000    # 成交量范围：8000-12000
}
```

**如果不归一化**:
- 模型无法理解"BTC 的 ATR=500"和"SOL 的 ATR=2"可能代表相同的波动率水平
- 模型会认为 ATR=500 总是比 ATR=2 更重要（实际上应该看相对值）

---

## ✅ 正确的做法

### 方法 1: 特征级归一化（推荐）

**将每个特征归一化到相同的尺度**

```python
# 归一化后
features = {
    'sma_5': [
        # BTC: (50000 - 50000) / 100 = 0
        # ETH: (3000 - 3000) / 10 = 0
        # SOL: (100 - 100) / 1 = 0
        # 所有资产的特征都在相同的尺度上
    ]
}
```

**优点**:
- ✅ 不同资产的特征在相同尺度
- ✅ 模型可以学习跨资产的通用模式
- ✅ 特征重要性不会被数值大小影响

**实现**:
```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_normalized = scaler.fit_transform(X)  # 所有资产一起归一化
```

---

### 方法 2: 资产无关的特征设计（更推荐）

**设计本身就资产无关的特征**

```python
# ❌ 不好的特征（资产相关）
'sma_5': 50000  # BTC 的价格，无法跨资产使用

# ✅ 好的特征（资产无关）
'sma_5_pct_close': (sma_5 / close - 1.0)  # 百分比，资产无关
'atr_percentile': 0.8  # 分位数（0-1），资产无关
'sr_dist_high': (close - roll_high) / atr  # ATR 单位，资产无关
```

**优点**:
- ✅ 特征本身就已经归一化
- ✅ 不需要额外的归一化步骤
- ✅ 更符合金融直觉（相对值比绝对值重要）

**当前代码中的实现**:
- ✅ Baseline 特征：使用 ATR 归一化、分位数归一化
- ✅ Default 特征：部分有 `*_pct_close` 版本
- ⚠️ 但部分原始特征（如 `sma_5`, `ema_5`）可能未归一化

## 🔍 验证方法

### 检查特征分布

```python
# 检查不同资产的特征分布是否相似
for symbol in ['BTCUSDT', 'ETHUSDT', 'SOLUSDT']:
    symbol_X = X_df[train_df['symbol'] == symbol]
    print(f"{symbol}:")
    print(f"  特征均值范围: [{symbol_X.mean().min():.2f}, {symbol_X.mean().max():.2f}]")
    print(f"  特征标准差范围: [{symbol_X.std().min():.2f}, {symbol_X.std().max():.2f}]")
    
# 如果归一化正确，不同资产的均值/标准差应该相似
# 如果差异巨大（>1000），可能是未归一化的特征
```

### 检查特征值范围

```python
# 检查是否有未归一化的特征
for col in X_df.columns:
    col_values = X_df[col].dropna()
    if len(col_values) > 0:
        col_abs_max = col_values.abs().max()
        if col_abs_max > 1000:
            print(f"⚠️ {col}: 值范围过大 ({col_abs_max:.2f})，可能需要归一化")
```

## 📝 总结

### LightGBM 是否需要归一化？

| 场景 | 是否需要归一化 | 原因 |
|------|---------------|------|
| **单资产训练** | ❌ 不需要 | 树模型对尺度不敏感 |
| **多资产训练** | ✅ **需要** | 让不同资产的特征在相同尺度，学习跨资产模式 |

### 为什么多资产需要归一化？

**不是因为模型需要，而是因为数据需要**

1. **数值范围差异**：不同资产的价格/成交量差异巨大
2. **学习跨资产模式**：模型需要理解"2% 涨幅"在不同资产中意义相同
3. **特征重要性**：避免被数值大的资产主导

### 最佳实践

1. **优先使用资产无关的特征**（如百分比、分位数、ATR 单位）
2. **如果必须使用原始特征，进行归一化**
3. **检查特征分布，确保不同资产的特征在相似尺度**
