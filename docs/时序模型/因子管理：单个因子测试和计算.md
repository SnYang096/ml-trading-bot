# 因子管理：单个因子测试和计算

## 问题背景

之前的因子管理系统存在以下问题：
1. **只能测试大类**：只能通过 `feature_type` 选择整个特征类别（如 `comprehensive`, `baseline` 等）
2. **无法测试单个因子**：无法单独测试某个因子的效果
3. **实盘计算不便**：实盘时需要计算所有特征，无法只计算需要的因子

## 解决方案

新增了两个工具来解决这些问题：

### 1. 单个因子测试工具 (`test_single_factor.py`)

**功能**：
- 测试单个或多个指定因子的效果
- 计算因子的 IC、IR、相关性等指标
- 分析因子的分布特征
- 生成详细的因子分析报告

**用法**：

```bash
# 测试单个因子
make factor-test \
  FACTOR_TEST_FACTORS="rsi_7" \
  FACTOR_TEST_SYMBOL=BTCUSDT \
  FACTOR_TEST_START_DATE=2024-01-01 \
  FACTOR_TEST_END_DATE=2024-12-31

# 测试多个因子
make factor-test \
  FACTOR_TEST_FACTORS="rsi_7 zigzag_normalized macd" \
  FACTOR_TEST_SYMBOL=BTCUSDT \
  FACTOR_TEST_START_DATE=2024-01-01 \
  FACTOR_TEST_END_DATE=2024-12-31 \
  FACTOR_TEST_OUTPUT_DIR=results/factor_tests/my_test

# 直接使用 Python 脚本
python scripts/factor_management/test_single_factor.py \
  --factors rsi_7 zigzag_normalized \
  --data-path /data/parquet_data \
  --symbol BTCUSDT \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --output-dir results/factor_tests/test_001
```

**输出**：
- `factor_test_results.json`: JSON 格式的详细结果
- `factor_test_report.txt`: 人类可读的文本报告

**报告内容包括**：
- 每个因子的 IC 值（多个预测周期）
- IC 的滚动窗口统计（IC IR）
- 因子的分布特征
- 异常值分析
- 是否归一化

### 2. 计算指定因子工具 (`compute_specific_factors.py`)

**功能**：
- 只计算指定的因子，不计算其他因子（节省计算资源）
- 支持从文件或数据目录加载数据
- 输出因子值到 CSV、Parquet 或 JSON 格式
- **适合实盘使用**：只计算需要的因子

**用法**：

```bash
# 从文件计算因子
make factor-compute \
  FACTOR_COMPUTE_FACTORS="rsi_7 macd" \
  FACTOR_COMPUTE_INPUT=data/btcusdt_2024.parquet \
  FACTOR_COMPUTE_OUTPUT=factors/rsi_macd.csv

# 从数据目录计算因子
make factor-compute \
  FACTOR_COMPUTE_FACTORS="rsi_7 zigzag_normalized" \
  FACTOR_COMPUTE_SYMBOL=BTCUSDT \
  FACTOR_COMPUTE_START_DATE=2024-01-01 \
  FACTOR_COMPUTE_END_DATE=2024-12-31 \
  FACTOR_COMPUTE_OUTPUT=factors/selected_factors.csv \
  FACTOR_COMPUTE_FORMAT=parquet

# 直接使用 Python 脚本
python scripts/factor_management/compute_specific_factors.py \
  --factors rsi_7 zigzag_normalized \
  --input data/btcusdt_2024.parquet \
  --output factors/computed.csv \
  --format csv
```

**输出格式**：
- CSV: 适合 Excel 查看
- Parquet: 适合 Python 读取（推荐）
- JSON: 适合 API 调用

### 场景 1: 研究单个因子的效果

## 技术细节

### 因子计算优化

两个工具都使用了 `required_features` 参数：
- 只计算需要的因子，不计算其他因子
- 大幅减少计算时间和内存占用
- 适合实盘场景

### 特征工程自动识别

工具会自动识别因子属于哪个特征模块：
- `baseline`: SR、压缩特征
- `default`: TA-Lib 指标
- `enhanced`: Hurst、Wavelet、Hilbert 等
- `alpha101`: Alpha101 因子

然后只启用需要的模块。

### 因子可用性检查

工具会检查：
1. 因子是否存在于特征工程输出中
2. 如果缺失，会提示可能的原因
3. 只对可用的因子进行分析

## 与现有系统的集成

### 与 dim-compare 集成

```bash
# 1. 运行 dim-compare 得到 top factors
make dim-compare ...

# 2. 测试 top factors 中的某些因子
make factor-test \
  FACTOR_TEST_FACTORS="alpha101_095 cdl_darkcloudcover" \
  ...

# 3. 如果效果好，实盘时只计算这些因子
make factor-compute \
  FACTOR_COMPUTE_FACTORS="alpha101_095 cdl_darkcloudcover" \
  ...
```

### 与 rolling train 集成

```bash
# 1. 测试因子效果
make factor-test FACTOR_TEST_FACTORS="rsi_7 macd" ...

# 2. 创建只包含这些因子的 top_factors.json
python scripts/utils/create_top_factors_json.py \
  --factors rsi_7 macd \
  --output results/custom_top_factors.json

# 3. 使用这些因子进行 rolling train
make rolling \
  ROLLING_USE_TOP_FACTORS=results/custom_top_factors.json \
  ...
```

## 最佳实践

1. **因子研究流程**：
   ```
   发现因子 → 测试单个因子 → 对比多个因子 → 加入 top_factors → 实盘使用
   ```

2. **实盘优化**：
   - 只计算真正需要的因子
   - 使用 Parquet 格式存储（更快）
   - 定期更新因子值

3. **因子选择**：
   - 优先选择 IC 值高且稳定的因子
   - 考虑因子的计算成本
   - 避免高度相关的因子

## 常见问题

### Q: 如何知道有哪些因子可用？

A: 运行一次完整的特征工程，查看输出列名：
```bash
python -c "
from src.data_tools.comprehensive_feature_engineering import ComprehensiveFeatureEngineer
import pandas as pd
df = pd.DataFrame({'open': [1], 'high': [1], 'low': [1], 'close': [1], 'volume': [1]})
engineer = ComprehensiveFeatureEngineer()
df_features = engineer.engineer_all_features(df)
print('\n'.join(sorted(df_features.columns)))
"
```

### Q: 因子计算失败怎么办？

A: 检查：
1. 因子名称是否正确
2. 是否属于指定的 `feature_type`
3. 数据是否完整（OHLCV）

### Q: 如何批量测试所有因子？

A: 先获取所有因子列表，然后分批测试：
```bash
# 获取所有因子
ALL_FACTORS=$(python scripts/utils/list_all_factors.py)

# 分批测试（每批 10 个）
for i in {0..9}; do
  FACTORS=$(echo $ALL_FACTORS | cut -d' ' -f$((i*10+1))-$((i+1)*10))
  make factor-test FACTOR_TEST_FACTORS="$FACTORS" ...
done
```
