# 1) 为什么感觉“没用” — 本质原因（简短）

* **标签不一致**：4H 模型的目标是 forward=24（≈4天），而 1H 特征最有信息量时可能映射到短期 forward（3bars）。短期信号不一定对 4 天后的回报有预测力。

* 当 1H 包含 **lead 信息**（能在 4H 指标变化前先行变化），并且你能把这种 lead 信息转换为对 4H 目标有意义的特征（例如：短期相位急剧反转的计数、短期能量爆发后持续 X 小时）。

## 🧠 一、核心理念：模型 ≠ 策略

一个模型训练的周期、forward bars、特征空间，都决定了它的**感知时间尺度（Temporal Horizon）**。

| 模型       | 代表时间感知  | 擅长任务               | 不适合任务       |
| -------- | ------- | ------------------ | ----------- |
| **1H模型** | 微观（短周期） | 入场时机、日内波动捕捉、结构突破识别 | 趋势确认、趋势延续判断 |
| **4H模型** | 中观（中周期） | 趋势确认、趋势延续、风险过滤     | 早期建仓、短线捕捉   |

👉 所以：

> “1H 是短线猎人，4H 是趋势守卫。”

如果你强行让一个模型做两个时间尺度的任务（比如既要早开仓，又要稳确认），那它的损失函数就会陷入**方向冲突 (Objective Conflict)**：

* 过早入场 → 容易止损；
* 太晚确认 → 错过波段；
  模型梯度在这两者之间会“撕裂”，最终谁都做不好。

---

## ⚙️ 二、正确的架构方式：**多模型多策略分工制**

### ✅ 模型分层

| 层级      | 模型           | 时间尺度 | 目标函数           | 输出        |
| ------- | ------------ | ---- | -------------- | --------- |
| L1      | `Model_1h`   | 短周期  | 捕捉短期方向变化、波动率膨胀 | 早期信号、先手入场 |
| L2      | `Model_4h`   | 中周期  | 识别趋势确认、过滤假突破   | 趋势确认信号    |
| L3 (可选) | `Model_meta` | 融合层  | 动态评估1H和4H信号可信度 | 最终权重或仓位调整 |

### ✅ 策略分层（重点）

1. **短线策略（Scalper / Intraday）**

   * 只使用 `Model_1h`
   * 高频入场，日内止盈止损
   * 不依赖4H趋势
   * 注重胜率与微结构特征（Hilbert、ZigZag、CVD）

2. **趋势策略（Swing / Position）**

   * 使用 `Model_4h`
   * 持仓周期 1~3天，关注趋势确认与延续
   * 用资金曲线过滤假信号
   * 止盈止损可用ATR或结构止损

3. **混合策略（Smart Router / Multi-Agent）**

   * `Model_1h` 发出“潜在信号”；
   * `Model_4h` 决定“是否跟进 / 加仓”；
   * 若短线信号先行、趋势信号随后确认 → 自动升级为趋势单；
   * 若趋势信号缺席 → 保持短线思维，及时止盈退出。
