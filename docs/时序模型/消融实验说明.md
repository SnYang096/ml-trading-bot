# 消融实验（Ablation Study）说明

## 什么是消融实验？

**消融实验（Ablation Study）** 是一种系统性的特征重要性评估方法。通过逐步移除或添加特征组，对比不同配置下的模型性能，来量化每个特征组对最终结果的贡献。

### 核心思想

```
完整特征集 → 移除特征组A → 对比性能差异 → 评估特征组A的贡献
```

### 在交易策略中的应用

在量化交易中，消融实验用于回答：
- **哪些特征真正有效？** 移除某个特征组后，性能下降多少？
- **特征组合的协同效应？** 某些特征组合是否产生1+1>2的效果？
- **特征冗余性？** 哪些特征可以安全移除而不影响性能？

## 使用方法

### 命令

```bash
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="baseline=config/features/baseline.yaml full=config/features/full.yaml"
```

### 参数说明

- `STRAT_COMPARE_CONFIG`: 基础策略配置目录
- `STRAT_COMPARE_SYMBOL`: 交易对
- `STRAT_COMPARE_OVERRIDES`: 特征变体列表，格式为 `name=path/to/features.yaml`

### 工作流程

1. **加载基础策略配置**：从 `STRAT_COMPARE_CONFIG` 读取策略配置
2. **创建特征变体**：
   - `base`: 使用原始配置
   - 其他变体：通过 `--feature-overrides` 指定的特征配置
3. **训练和评估**：对每个变体执行完整的训练-评估流程
4. **对比分析**：生成对比报告（CSV + JSON）

### 输出结果

- `results/strategy_compare/strategy_feature_compare_summary.csv`: 对比摘要表
- `results/strategy_compare/strategy_feature_compare_summary.json`: 详细结果

### 示例场景

#### 场景1：评估 WPT 特征的贡献

```bash
# 创建 baseline.yaml（不含 WPT 特征）
# 创建 with_wpt.yaml（包含 WPT 特征）

make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="no_wpt=config/features/baseline.yaml with_wpt=config/features/with_wpt.yaml"
```

**解读结果**：
- 如果 `with_wpt` 的 Rank IC 显著高于 `no_wpt`，说明 WPT 特征有效
- 如果差异很小，说明 WPT 特征可能冗余

#### 场景2：评估多组特征的协同效应

```bash
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="only_sr=config/features/sr_only.yaml \
                           sr_volume=config/features/sr_volume.yaml \
                           full=config/features/full.yaml"
```

**解读结果**：
- `only_sr` vs `sr_volume`: 评估 Volume 特征的增量贡献
- `sr_volume` vs `full`: 评估其他特征组的增量贡献

## 最佳实践

1. **从简单到复杂**：先测试单个特征组，再测试组合
2. **控制变量**：每次只改变一个特征组，保持其他配置不变
3. **多次验证**：在不同时间段、不同交易对上重复实验
4. **关注稳定性**：不仅看平均性能，还要看性能的稳定性（标准差）

## 与 dim-compare 的区别

| 工具 | 目的 | 方法 |
|------|------|------|
| **ts-strategy-feature-compare** (消融实验) | 评估特征组的贡献 | 对比不同特征配置的完整策略性能 |
| **ts-dim-compare** | 特征选择 | 从大量特征中自动选择最优子集 |

**使用建议**：
- 先用 `ts-dim-compare` 做特征选择，得到 top factors
- 再用 `ts-strategy-feature-compare` 做消融实验，验证特征组的重要性
