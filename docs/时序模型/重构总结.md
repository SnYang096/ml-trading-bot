## 新架构特点

### Config-Driven
- 所有命令都使用策略配置目录（`config/strategies/{strategy_name}/`）
- 支持四个策略：`sr_reversal`, `sr_breakout`, `compression_breakout`, `trend_following`
- 统一的特征加载和标签生成流程

### 命令对比

| 旧命令 | 新命令 | 变化 |
|--------|--------|------|
| `make rolling` (旧实现) | `make rolling` (新实现) | 使用 `ROLLING_CONFIG` 参数，支持策略配置 |
| `make ts-dim-compare` (旧实现) | `make ts-dim-compare` (新实现) | 使用 `DIM_COMPARE_CONFIG` 参数，支持策略配置 |
| - | `make ts-strategy-feature-compare` | 新增：消融实验命令 |

## 使用示例

### Rolling 训练
```bash
make rolling \
  ROLLING_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  INITIAL_TRAIN_MONTHS=6
```

### Dim-Compare
```bash
make ts-dim-compare \
  DIM_COMPARE_CONFIG=config/strategies/sr_reversal \
  SYMBOL=BTCUSDT \
  START_DATE=2024-01-01 \
  END_DATE=2024-12-31
```

### 消融实验
```bash
make ts-strategy-feature-compare \
  STRAT_COMPARE_CONFIG=config/strategies/sr_reversal \
  STRAT_COMPARE_SYMBOL=BTCUSDT \
  STRAT_COMPARE_OVERRIDES="baseline=config/features/baseline.yaml full=config/features/full.yaml"
```
