## 一、`prepare_rank_ic_labels` 的作用

### 核心功能

`prepare_rank_ic_labels` **不是训练模型的函数**，而是**准备标签（label）的函数**。

它的作用是：
1. 计算目标变量（target）
2. 计算辅助指标（volatility、quantile 等）
3. 生成交易掩码（tradable mask）
4. 计算样本权重（trend strength）

### 工作流程

```python
def prepare_rank_ic_labels(
    df: pd.DataFrame,
    use_risk_reward_label: bool = False,  # 是否使用 R/R 标签
    split_by_reaction_type: bool = False,  # 是否按反应类型分类
    ...
) -> pd.DataFrame:
    """
    准备标签，返回包含以下列的 DataFrame：
    - volatility_normalized_target: 目标变量（用于训练）
    - future_return: 未来收益率（用于评估）
    - return_quantile: 历史分位数标签（用于评估）
    - tradable: 可交易掩码（用于过滤样本）
    - trend_strength: 趋势强度（用作样本权重）
    - sr_reaction: SR 反应类型（reversal/breakout，如果启用）
    - rr_reversal_achieved: 反转型 R/R 标签（如果启用）
    - rr_breakout_achieved: 突破型 R/R 标签（如果启用）
    """
```

### 标签生成逻辑

#### 1. 传统标签（`use_risk_reward_label=False`）

```python
# 计算未来收益率
future_return[t] = (close[t+1+horizon] - close[t+1]) / close[t+1]

# 计算波动率归一化目标
volatility_normalized_target = future_return / rolling_vol
```

#### 2. R/R 标签（`use_risk_reward_label=True`）

```python
# 计算 R/R 标签（是否实现 ≥2R）
rr_achieved = compute_rr_label(
    df,
    signal_col="signal",
    rr_ratio=2.0,
    max_holding_bars=24,
)

# 如果启用反应类型分类
if split_by_reaction_type:
    # 分类 SR 反应类型
    sr_reaction = classify_sr_reaction(df, signal_col="signal")
    
    # 分别计算反转型和突破型 R/R 标签
    rr_reversal_achieved = compute_rr_label_by_reaction(
        df, reaction_type="reversal"
    )
    rr_breakout_achieved = compute_rr_label_by_reaction(
        df, reaction_type="breakout"
    )
    
    # 设置主目标变量
    volatility_normalized_target = rr_reversal_achieved  # 或 rr_breakout_achieved
```

## 二、模型训练流程

### 训练函数：`train_rank_ic_model`

```python
def train_rank_ic_model(
    df: pd.DataFrame,
    feature_cols: List[str],  # 特征列表
    target_col: str = "volatility_normalized_target",  # 目标变量
    n_splits: int = 5,  # CV 折数
    ...
) -> Tuple[List[lgb.Booster], float, pd.DataFrame, List[str]]:
    """
    使用 LightGBM 训练模型，返回：
    - models: 训练好的模型列表（每个 CV fold 一个）
    - avg_rank_ic: 平均 Rank IC
    - results_df: 预测结果和指标
    - used_feature_cols: 实际使用的特征列表
    """
```

### 模型架构

**所有策略都使用相同的模型架构：LightGBM（梯度提升树）**

```python
# 默认 LightGBM 参数
default_params = {
    "objective": "regression",  # 回归任务
    "metric": "rmse",  # 评估指标（但实际用 Rank IC）
    "boosting_type": "gbdt",
    "num_leaves": 31,
    "learning_rate": 0.05,
    "feature_fraction": 0.8,
    "bagging_fraction": 0.8,
    "bagging_freq": 5,
    "verbose": -1,
}
```

### 训练流程

```python
# 1. 准备数据
X = df[feature_cols].values
y = df[target_col].values
weights = df["trend_strength"].values  # 样本权重

# 2. 时间序列交叉验证
tscv = TimeSeriesSplit(n_splits=5)
for fold, (train_idx, val_idx) in enumerate(tscv.split(X)):
    # 3. 过滤可交易样本
    train_mask = tradable[train_idx]
    X_train = X[train_idx][train_mask]
    y_train = y[train_idx][train_mask]
    
    # 4. 创建 LightGBM 数据集
    train_data = lgb.Dataset(X_train, label=y_train, weight=weights)
    
    # 5. 训练模型
    model = lgb.train(
        default_params,
        train_data,
        num_boost_round=1000,
        valid_sets=[val_data],
        callbacks=[lgb.early_stopping(50)],
    )
    
    # 6. 评估（使用 Rank IC）
    predictions = model.predict(X_val)
    rank_ic = compute_rank_ic(predictions, y_val_true_ret)
```

## 三、不同策略的差异

### 核心问题：不同策略是否使用相同的模型？

**答案：模型架构相同（都是 LightGBM），但以下方面不同：**

1. **特征集不同**（通过关键词筛选）
2. **标签不同**（不同的 R/R 标签）
3. **训练样本不同**（只使用对应类型的样本）

### 详细对比

| 维度 | 反转型模型 | 突破型模型 | 压缩突破模型 |
|------|-----------|-----------|-------------|
| **模型架构** | LightGBM | LightGBM | LightGBM |
| **特征选择** | `select_reversal_features()` | `select_breakout_features()` | `select_compression_breakout_features()` |
| **特征数量** | ~80 个（SR 强度、反转信号等） | ~80 个（突破动量、趋势延续等） | ~60 个（压缩指标、突破动量等） |
| **标签** | `rr_reversal_achieved` | `rr_breakout_achieved` | `rr_achieved` |
| **训练样本** | 只使用 `sr_reaction == "reversal"` | 只使用 `sr_reaction == "breakout"` | 只使用 `compression_reaction == "compression_breakout"` |
| **目标变量** | `volatility_normalized_target = rr_reversal_achieved` | `volatility_normalized_target = rr_breakout_achieved` | `volatility_normalized_target = rr_achieved` |

### 训练流程对比

#### 反转型模型训练

```python
# 1. 特征工程
df_features = engineer.engineer_all_features(df_raw)

# 2. 特征选择（只选反转型相关特征）
reversal_features = select_reversal_features(df_features, all_features)
# 结果：~80 个特征（sqs_*, volume_ratio, rsi_*, ...）

# 3. 准备标签
df_labeled = prepare_rank_ic_labels(
    df_features,
    use_risk_reward_label=True,
    split_by_reaction_type=True,  # 启用反应类型分类
)
# 结果：生成 sr_reaction, rr_reversal_achieved, rr_breakout_achieved

# 4. 过滤样本（只使用反转型样本）
df_train_reversal = df_labeled[df_labeled["sr_reaction"] == "reversal"]

# 5. 训练模型
models = train_rank_ic_model(
    df_train_reversal,
    feature_cols=reversal_features,  # 只用反转型特征
    target_col="volatility_normalized_target",  # = rr_reversal_achieved
)
```

#### 突破型模型训练

```python
# 1. 特征工程（相同）
df_features = engineer.engineer_all_features(df_raw)

# 2. 特征选择（只选突破型相关特征）
breakout_features = select_breakout_features(df_features, all_features)
# 结果：~80 个特征（breakout_speed, momentum, adx_*, ...）

# 3. 准备标签（相同，但使用不同的标签）
df_labeled = prepare_rank_ic_labels(
    df_features,
    use_risk_reward_label=True,
    split_by_reaction_type=True,
)
# 结果：生成 sr_reaction, rr_reversal_achieved, rr_breakout_achieved

# 4. 过滤样本（只使用突破型样本）
df_train_breakout = df_labeled[df_labeled["sr_reaction"] == "breakout"]

# 5. 训练模型
models = train_rank_ic_model(
    df_train_breakout,
    feature_cols=breakout_features,  # 只用突破型特征
    target_col="volatility_normalized_target",  # = rr_breakout_achieved
)
```

#### 压缩突破模型训练

```python
# 1. 特征工程（相同）
df_features = engineer.engineer_all_features(df_raw)

# 2. 特征选择（只选压缩突破相关特征）
compression_features = select_compression_breakout_features(df_features, all_features)
# 结果：~60 个特征（compression_score, bb_width, ...）

# 3. 分类压缩突破机会
df_features["compression_reaction"] = classify_compression_breakout(df_features)

# 4. 准备标签（不使用反应类型分类）
df_labeled = prepare_rank_ic_labels(
    df_features,
    use_risk_reward_label=True,
    split_by_reaction_type=False,  # 压缩突破不需要 SR 反应分类
)
# 结果：生成 rr_achieved

# 5. 过滤样本（只使用压缩突破样本）
df_train_compression = df_labeled[df_labeled["compression_reaction"] == "compression_breakout"]

# 6. 训练模型
models = train_rank_ic_model(
    df_train_compression,
    feature_cols=compression_features,  # 只用压缩突破特征
    target_col="volatility_normalized_target",  # = rr_achieved
)
```

## 四、为什么使用相同的模型架构？

### 优点

1. **统一性**：所有模型使用相同的训练流程，便于维护
2. **可比较性**：模型架构相同，可以直接比较不同策略的表现
3. **灵活性**：LightGBM 是通用模型，可以学习各种模式
4. **效率**：不需要为每个策略实现不同的模型架构

### 关键差异点

虽然模型架构相同，但通过以下方式实现策略差异化：

1. **特征工程**：不同策略使用不同的特征集
2. **样本过滤**：只使用对应类型的样本训练
3. **标签设计**：使用不同的 R/R 标签作为目标

### 类比

这就像：
- **相同的工具**（LightGBM）
- **不同的材料**（特征）
- **不同的目标**（标签）
- **不同的训练数据**（样本）

## 五、完整训练流程

### 流程图

```
原始数据
  ↓
特征工程（生成所有特征）
  ↓
特征选择（根据策略选择相关特征）
  ├── 反转型 → select_reversal_features()
  ├── 突破型 → select_breakout_features()
  └── 压缩突破 → select_compression_breakout_features()
  ↓
标签准备（prepare_rank_ic_labels）
  ├── 分类反应类型（classify_sr_reaction）
  ├── 计算 R/R 标签（compute_rr_label）
  └── 生成目标变量（volatility_normalized_target）
  ↓
样本过滤（只使用对应类型的样本）
  ├── 反转型 → sr_reaction == "reversal"
  ├── 突破型 → sr_reaction == "breakout"
  └── 压缩突破 → compression_reaction == "compression_breakout"
  ↓
模型训练（train_rank_ic_model）
  ├── 时间序列交叉验证（TimeSeriesSplit）
  ├── LightGBM 训练（每个 fold 一个模型）
  └── Rank IC 评估
  ↓
模型集成（ensemble）
  └── 多个 fold 的模型组合
  ↓
信号生成（generate_ensemble_signals）
  └── 生成交易信号
```

## 六、总结

### 核心答案

1. **`prepare_rank_ic_labels` 的作用**：
   - 准备标签（不是训练模型）
   - 计算目标变量、辅助指标、样本权重等

2. **模型训练方式**：
   - 使用 `train_rank_ic_model` 函数
   - 模型架构：LightGBM（梯度提升树）
   - 训练方法：时间序列交叉验证 + 集成学习

3. **不同策略是否使用相同模型**：
   - **模型架构相同**：都是 LightGBM
   - **特征集不同**：通过关键词筛选
   - **标签不同**：不同的 R/R 标签
   - **训练样本不同**：只使用对应类型的样本

### 关键设计思想

**"相同的工具，不同的应用"**

- 使用统一的模型架构（LightGBM）
- 通过特征选择、样本过滤、标签设计实现策略差异化
- 这样既保持了系统的统一性，又实现了策略的专门化
