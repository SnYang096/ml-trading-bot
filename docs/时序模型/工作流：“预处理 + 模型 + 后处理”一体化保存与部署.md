在使用 LightGBM（或其他模型）进行量化建模时，要实现 “预处理 + 模型 + 后处理”一体化保存与部署，核心目标是：
✅ 确保训练时和预测时的数据流完全一致，且支持定期 retrain 与自动更新

虽然 LightGBM 本身不直接管理预处理逻辑，但我们可以通过 封装 + 序列化 实现端到端的 pipeline。以下是几种工程上成熟、可维护性强的做法。

✅ 推荐方案：自定义 QuantModel 类（最灵活、最清晰）

我们将预处理（如 Winsorize）、模型、后处理（如信号计算）全部封装在一个类中，并实现 .fit() 和 .predict() 方法，同时支持 save/load。
🔧 示例代码

python
import joblib
import numpy as np
import pandas as pd
import lightgbm as lgb
from typing import Optional, Tuple, Dict, Any

class QuantModel:
def __init__(
self,
k: float = 3.5,
target_quantile: float = 0.5,
feature_cols: Optional[list] = None,
lgb_params: Optional[Dict] = None
):
self.k = k
self.target_quantile = target_quantile
self.feature_cols = feature_cols
self.lgb_params = lgb_params or {
"objective": "quantile",
"alpha": self.target_quantile,
"metric": "quantile",
"num_leaves": 64,
"learning_rate": 0.05,
"verbose": -1
}

# 预处理参数（从训练集学习）
self.median_: float = 0.0
self.sigma_: float = 1.0

# 模型
self.model_ = None

def _robust_scale_params(self, y: np.ndarray):
"""仅用训练标签 y 计算 median 和 sigma（MAD-based）"""
y = y[np.isfinite(y)]
if len(y) == 0:
raise ValueError("No valid labels for preprocessing.")

median = np.nanmedian(y)
mad = np.nanmedian(np.abs(y - median))
if mad == 0 or np.isnan(mad):
sigma = np.nanstd(y, ddof=1)
if np.isnan(sigma) or sigma == 0:
sigma = (np.nanmax(y) - np.nanmin(y)) / 6.0 if len(y) > 1 else 1.0
else:
sigma = mad 1.4826
return float(median), float(sigma)

def _winsorize(self, y: np.ndarray) -> np.ndarray:
"""应用 Winsorize（使用已学习的 median/sigma）"""
lower = self.median_ - self.k self.sigma_
upper = self.median_ + self.k * self.sigma_
return np.clip(y, lower, upper)

def fit(self, X: pd.DataFrame, y: pd.Series):
"""训练全流程：预处理 + 模型训练"""
# 1. 学习预处理参数（仅用 y_train）
self.median_, self.sigma_ = self._robust_scale_params(y.values)

# 2. 清洗标签
y_clean = self._winsorize(y.values)

# 3. 准备特征
if self.feature_cols is None:
self.feature_cols = X.columns.tolist()
X_use = X[self.feature_cols].copy()

# 4. 训练 LightGBM
train_data = lgb.Dataset(X_use, label=y_clean)
self.model_ = lgb.train(
self.lgb_params,
train_data,
num_boost_round=500,
callbacks=[lgb.early_stopping(stopping_rounds=50, verbose=False)]
)
return self

def predict(self, X: pd.DataFrame) -> np.ndarray:
"""预测原始尺度的收益（注意：模型输出已是清洗后的尺度）"""
X_use = X[self.feature_cols].copy()
pred_clean = self.model_.predict(X_use)
# 注意：Winsorize 是不可逆的，所以预测值保持在清洗后空间
# 如果你需要“反 Winsorize”，通常不推荐（会引入偏差）
return pred_clean

def get_signal_strength(self, X: pd.DataFrame, vol_pred: np.ndarray) -> np.ndarray:
"""后处理：计算 signal strength = q50 / vol"""
q50 = self.predict(X)
return np.abs(q50) / (vol_pred + 1e-8)

def save(self, path: str):
"""保存整个模型（含预处理参数）"""
joblib.dump(self, path)

@classmethod
def load(cls, path: str):
"""加载模型"""
return joblib.load(path)

📦 其他可选方案对比

方案 优点 缺点
------ ------ ------
自定义类（推荐） 灵活、可扩展、易测试、支持任意预处理 需自己写代码
sklearn Pipeline + 自定义 Transformer 与 sklearn 生态兼容 LightGBM 的 quantile 目标需额外处理，Pipeline 对标签变换支持弱
单独保存参数 + 模型 简单 容易出错（版本不一致、漏存参数）
MLflow / DVC 支持实验跟踪、版本管理 学习成本高，适合大团队
💡 对于量化个人或小团队，自定义类 + joblib 是最佳平衡点。

⚠️ 关键注意事项

1. 不要尝试“反 Winsorize”预测值
Winsorize 是有损压缩，强行还原会引入偏差。预测值应保持在清洗后的空间。

2. 波动率模型需同步更新
如果你有单独的 VolatilityModel，它也应封装并一起 retrain/save。

3. 特征一致性
确保 feature_cols 在训练和预测时完全一致（建议显式指定，而非依赖 X.columns）。

4. 时间对齐
在 retrain 时，确保 y_train 是基于当时可用信息计算的（避免未来函数）。

✅ 总结
用一个自定义的 QuantModel 类，将 Winsorize 参数（median/sigma）、LightGBM 模型、后处理逻辑全部封装起来，通过 joblib 一键保存/加载，是最简单、可靠、可维护的方式。

这样，每次 retrain 都会生成一个完整的、自包含的模型包，部署时只需加载一个文件，彻底避免参数错配或数据泄露风险。
