🔢 一、最优风险比例（R%）：不是固定值，而是动态函数
❌ 常见误区
“固定用 1%” → 在低波动牛市中过于保守，在高波动熊市中仍可能爆仓。
“用 Kelly 公式全仓” → 加密市场噪声大，Kelly 极易过拟合。
✅ 推荐方案：波动率自适应风险预算
核心思想：
账户承担的名义风险（position × vol） 应与当前市场状态匹配。
公式：
\[
R_t = R_0 \cdot \frac{\sigma_{\text{target}}}{\sigma_{\text{market}, t}}
\]

其中：
\( R_0 \)：基准风险比例（如 1%）
\( \sigma_{\text{target}} \)：目标波动率（如 0.02 / 天 ≈ 年化 30%）
\( \sigma_{\text{market}, t} \)：全市场当前波动率（可用 BTC 30日滚动波动率代理）
实现步骤：
python
1. 计算市场波动率（以 BTC 为代表）
btc_vol_30d = btc_returns.rolling(3024).std() # 假设 hourly data
2. 设定目标年化波动率（例如 30%）
target_annual_vol = 0.30
target_hourly_vol = target_annual_vol / np.sqrt(365 24)
3. 动态调整 R%
R_base = 0.01 # 基准 1%
R_dynamic = R_base (target_hourly_vol / (btc_vol_30d + 1e-6))
4. 设置上下限（防极端）
R_dynamic = np.clip(R_dynamic, 0.005, 0.02) # 最低 0.5%，最高 2%
✅ 效果：
牛市低波动 → 自动加仓（R% ↑）
熊市高波动 → 自动减仓（R% ↓）
长期维持目标波动率，提升夏普比率

📉 二、信号阈值（COST_THRESHOLD）：动态覆盖交易成本
❌ 固定阈值问题
设 COST_THRESHOLD = 0.002（0.2%）→ 在低波动期有效，但在高波动期会错过大机会。
✅ 推荐方案：成本比 + 波动率归一化

你的信号 s = (2p−1)· r̂ /σ̂ 已是无量纲的风险调整收益，可直接与成本比较。

但交易成本也应按波动率标准化：
定义：
trading_cost：总成本（手续费 + 滑点），例如 0.0015（0.15%）
cost_in_sigma_units = trading_cost / σ̂
动态阈值：
\[
\text{COST\_THRESHOLD}_i = k \cdot \frac{\text{trading\_cost}_i}{\hat{\sigma}_i}
\]

其中 k 是安全边际（推荐 k = 2.0，即信号需覆盖 2 倍成本）。
代码实现：
python
trading_cost = 0.0015 # 0.15%

for i, coin in enumerate(coins):
vol = vol_pred[i]
signal = signals[i]

# 成本在波动率单位下的大小
cost_in_sigma = trading_cost / (vol + 1e-6)

# 动态阈值：需至少 2 倍覆盖
if abs(signal) < 2.0 cost_in_sigma:
signals[i] = 0.0
✅ 优势：
高波动币种（如小币）：σ̂ 大 → cost_in_sigma 小 → 更容易交易
低波动币种（如稳定币附近）：自动过滤无效信号

🌪️ 三、黑天鹅熔断规则：三层防护体系

加密市场常见黑天鹅：交易所宕机、监管打击、LUNA 式崩盘、闪电贷攻击等。
你需要提前识别 + 快速响应 + 安全恢复。
🛡️ 第一层：波动率极端值熔断（预防性）

触发条件（任一满足即熔断）：
当前 σ̂_i > 99% 分位数 of historical σ̂
或 r̂ / σ̂ > 5（信号异常强，可能是噪声）

动作：
该币种仓位 → 0
暂停新开仓 1~6 小时（等待市场冷静）

python
vol_99 = historical_vol.quantile(0.99)
if vol_pred[i] > vol_99 or abs(signal) > 5.0:
positions[i] = 0
set_cooling_period(coin, hours=2)

🛡️ 第二层：价格跳跃熔断（实时保护）

触发条件（基于最新 tick）：
价格在 1 分钟内变动 > 10%
或买卖价差 > 5%

动作：
立即平仓该币种所有头寸
切换至“只平不建”模式
💡 可通过 WebSocket 监控订单簿实现。

🛡️ 第三层：账户级风险熔断（终极防线）

触发条件（每日检查）：
单日回撤 > 5%
或连续 3 日亏损

动作：
所有仓位降至 10%
切换至保守模式（R% × 0.3）
触发模型健康检查（是否需 retrain？）
