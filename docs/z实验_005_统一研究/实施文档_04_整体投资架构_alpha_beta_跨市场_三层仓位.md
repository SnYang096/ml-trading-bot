# 实施文档 04：整体投资架构 — α/β 分离 × 跨市场 × 加密四层仓位

> 本文把原「三层仓位架构（现货 / 合约 / 彩票）」的内部设计，向外扩展到 **收益来源分层（α vs β）** 与 **跨市场资产配置**，并在加密内部细化为 **四层（L1 现货 β / L2 合约 swing α / L3 C 多腿 / L4 彩票）**。
> 取代旧文件：`实施文档_04_三层仓位架构_现货_合约_彩票.md`。
> **与 `docs/strategy/` 对齐**：L1↔A（`spot_accum`）、L2↔B（PCM trend swing）、L3↔C（`chop_grid` + `trend_scalp`）、L4↔利润彩票（100x ME + meme）。

---

## 0. 问题背景

当前系统的现状与痛点：

1. **合约 α 系统已成形**：TPC / BPC / ME / FBF / SRB 五条子线覆盖了不同阶段/形态的趋势抓取，但本质上都是 **加密合约 + 双边 + 短中周期**，属于主动择时。
2. **仓位常常"不如全仓现货"**：合约加仓不满 → 错过大段趋势 → "有系统但赚不到大钱"。
3. **没有显式的 β 敞口**：整个体系默认 "有信号才持仓"，无法在 crypto 长期牛市里拿满 β。
4. **没有跨市场（crypto 以外）敞口**：遇到加密熊市或结构性下行时，组合没有对冲/轮动的出路。

核心矛盾：
> **系统化合约交易的精细风控** vs **趋势行情需要的持仓耐心与规模** vs **单一市场的系统性风险**。

解决思路分三层（从外到内）：

- **收益来源层**：先把"主动 α"和"资产类别 β"分开，避免一个账户同时承担两种相反的心理/仓位约束。
- **资产类别层**：β 不只来自 crypto，可以向 美股 ETF / 黄金 / A股 扩展，做跨市场强弱轮动。
- **加密内部层**：**四层** **现货 / 合约 swing / C 多腿 / 彩票**（本文第 4 节），处理持有、结构 alpha、震荡现金流与利润博倍数。

## 1. 收益来源：α 与 β 必须分开

### 1.1 两种收益来源

| 类型 | 定义 | 依赖 | 本系统对应 |
|------|------|------|-----------|
| **α（主动超额）** | 主动择时/择标的带来的超额收益 | 信号质量、胜率 × 赔率 | 合约策略（TPC/BPC/ME/FBF/SRB） |
| **β（资产类别）** | 整个资产类别向上的 beta 收益 | 持有时间、不过早出场 | 现货长期持有（BTC/ETH/优质标的） |

两者**不是互斥**，健康组合通常两者都要。常见误区是"想用 α 系统完全取代 β"：结果就是牛市被甩下车、熊市吃满信号失败。

### 1.2 为什么合约不适合承担"长期持有 β"

| 维度 | 合约 | 现货 |
|------|------|------|
| **资金费率** | 长期牛市里多头付费，年化可能 -10% ~ -30% | 无 |
| **爆仓风险** | 一次临时暴跌即可强平 | 不存在爆仓 |
| **Rollover/价差** | 季度合约有基差与展期成本 | 无 |
| **交易所风险** | 永续放在交易所，风控压力大 | 可自托管（冷钱包） |
| **税务/合规** | 许多地区衍生品税率更高 | 相对清晰 |

**结论**：长期看多 BTC / ETH / 其它核心标的，**必须现货**。合约只做"主动择时 α 敞口"。

### 1.3 α / β 账户层级建议

- **物理隔离**：α 与 β 最好在**不同账户或不同子账户**，避免 α 的回撤影响 β 的持仓心态（或反之）。
- **风险预算隔离**：α 系统内的止损/回撤限额，**不能让 β 底仓"来救援"**；β 底仓的浮亏也**不算入 α 的可用保证金**。
- **利润流向**：α 利润可以定期划到 β 做增配，反之一般不做（β 只减仓不"输血"α）。

## 2. 资产类别：crypto 不是唯一 β

### 2.1 为什么要考虑跨市场

- 降低单一市场（加密）的系统性风险
- **跨市场动量/强弱轮动** 在历史上是稳健超额来源（如 Gary Antonacci 的 Dual Momentum）
- 黄金 / 长债在风险偏好切换时具有对冲作用

### 2.2 候选市场 × 可行性

| 市场 | 优点 | 难点 | 推荐优先级 |
|------|------|------|-----------|
| **加密现货（BTC/ETH+）** | 已有数据、流动性好、24/7 | 单一市场 β 过度集中 | **P0**（已在做） |
| **美股 ETF**（SPY/QQQ/GLD/TLT） | 数据好取、工具成熟（IB/券商）、30年回测有效 | 夜盘、税务、汇率 | **P1** |
| **黄金**（GLD / XAU 现货 / GC 期货） | 抗通胀、风险对冲 | 产品形态差异大，需区分 | **P1**（含在美股 ETF 即可） |
| **A股** | 本币、免汇率 | T+1、涨跌停、做空几乎没有、散户结构、政策扰动 | **P3** |
| **港股** | 与美股/A股都能映射 | 流动性极端分化 | **P3** |

**建议**：优先做到 **crypto 现货 + 美股 ETF 月度动量** 这一层就够覆盖绝大部分 β 场景；A股/港股除非有特别信息优势，优先级放最低。

### 2.3 跨市场动量落地（P1 可执行版）

对四资产 `{SPY, QQQ, GLD, TLT}`（或加 `BTC`）：

1. 每月初计算过去 **12 个月** 的 **绝对收益** 与 **12M - 1M 动量**。
2. 只持有得分最高的 **1~2 个**（或动量为负者换现金 / 短债）。
3. 月度再平衡，**不做日内**。

特点：实现极简、可复现、几乎无参数、与 crypto 合约 α 完全解耦。

## 3. 长期持有 BTC 的具体做法

### 3.1 现货 + 定投

- 现货，**不用合约**（见 1.2）。
- 定投可以是：
  - **固定金额周 / 月度定投**（零择时）
  - **条件增强定投**：BTC 在 VWAP1200 / 200D MA 之下时倍投，之上时减速（不是择顶底，只是纪律化）

### 3.2 仓位约束

- **长期持有敞口不超过你能接受 50% 回撤的资金**（历史上 BTC 有三次 -70% 以上回撤）。
- **冷钱包自托管**部分（长期不动的）与**交易所热仓**（参与策略/再平衡）要分开。

### 3.3 与 α 系统的关系

- **α 系统不要对冲 β 底仓**：不要因为 BPC 出了做空信号就去"保护"现货；这会让两个系统互相污染。
- α 利润达到一定额度，按**固定比例**（比如每季度 20%）划到 β 现货增配；这是"α 养 β"的单向机制。

## 4. 加密内部：四层仓位架构（现货 / 合约 swing / C 多腿 / 彩票）

> 上面的 α/β 与跨市场是外层；本节是加密内部的细化。四层与 **A/B/C 战略账本**一一对应（详见 `docs/strategy/A系统.md`、`B系统.md`、`C系统.md`）。

### 4.1 四层架构概览

| 层 | 账本 | 资金占比 / 来源 | 策略 | 目标 |
|----|------|----------------|------|------|
| **L1 — 现货底仓** | **A** | ~50% NAV | `spot_accum_simple`；买入持有 + 盘整内小仓对冲 | 跟上大趋势、不被甩下车（**β**） |
| **L2 — 合约 swing** | **B** | ~50% NAV（PCM trend 槽位） | BPC/TPC/ME/FBF/SRB + 指数加仓 | 趋势段 structural continuation（**α**） |
| **L3 — C 多腿** | **C** | 独立 `multi_leg` 子账户预算（与 L2 物理隔离） | `chop_grid` ↔ `trend_scalp`（**per symbol 互斥**） | 震荡网格 + 低 chop 趋势 scalp；**现金流 / 短周期 alpha** |
| **L4 — 高杠杆彩票** | — | **仅 L2 已实现利润** | 100x ME + Meme token bot（子预算独立） | 用已赚利润博极端倍数；meme 与主策略解耦 |

**L3 不是 meme**：meme / 100x 彩票已明确为 **L4**；L3 专指宪法 `multi_leg` 下的 C 系统（`chop_grid` 收割 semantic chop，`trend_scalp` 吃非 chop 趋势段）。

### 4.2 全局 / 板块宏观条件特征（选板块 + L4 门控）

对 **标的** 与 **板块指数** 各算一套同构标量：

```text
# 相对锚位置（距离 pct），P 为收盘价或标记价，VWAP1200 为 1200 周期 VWAP
d = (P - VWAP1200) / VWAP1200

# 锚的趋势强度：VWAP1200 对时间的斜率，归一化到有界范围
a = normalize( slope(VWAP1200) )   # 例如 tanh / atan / 分位数映射

# 组合宏观分
M = d * a
```

- **全局**：对 **基准标的**（BTC 或 BTC+ETH 合成）算 **`M_glob`**。
- **板块**：对每个 **板块指数序列**（成分 + 再平衡规则需固定）算 **`M_sector`**。

用途：

| 用途 | 建议用法 |
|------|---------|
| **操作哪一板块（L2 资金/信号权重）** | 用 `M_sector` 排序或分档；与单币 BPC/ME 信号是 **AND**，避免单币强但板块逆风重仓。 |
| **是否进入彩票阶段（L4 门控）** | **`M_glob`** 与 **目标板块 `M_sector`** 同时同向且超过幅度阈值，才允许 L4 从 L2 利润划款开火，否则 L4 整层关闭。 |

合理性要点：

1. `d` 管"位置"、`a` 管"趋势"，`M = d·a` 同时强调"有趋势 + 已离开锚"，比单指标更能过滤横盘噪声。
2. 板块层有必要：单币信号在板块 β 逆风时假突破多；板块 `M` 作为第二层 prior 与现有 prefilter 分工一致。
3. 放进 **L4** 门控特别合适：彩票最怕逆宏观裸奔；用 `M_glob + M_sector` 双阈作"允许亏，不允许在明显逆风时亏"的开关。**L3** 的门控走 C 系统自身的 regime（`bpc_semantic_chop` / `trend_confidence`），不与 `M_glob` 混用。

工程注意：

- **量纲与饱和**：`d`、`a` 两侧分别 clip / winsorize 后再相乘；最终 `M` 再做 `tanh` 或分位数映射到 `[-1, 1]`。
- **符号约定**：做多彩票要求 `d > 0` 且 `a > 0` 同向；否则走象限规则。
- **与现有 VWAP 特征去重**：`M_glob` 主要驱动 **L4** 总开关，`M_sector` 主要驱动板块选择/权重；L2 单币 gate、L3 chop/trend regime 各自维持不变。
- **板块指数契约**：成分 / 再平衡 / 缺失填充必须固定可复现。

### 4.3 L1：50% 现货底仓（BTC, SOL, HYPER, BNB, OKX 等）

做法：
- 买入后**不因第一次盘整出场**，即便跌破 VWAP1200
- 盘整期用 **≤ 1/10 仓位**做短期多空降低持仓成本
- 出场：**第二次盘整 + 大周期 CVD 背离**

合理性：
- 解决合约频繁出场错过大段的问题
- "第一次盘整不出" 符合趋势经验，第一次回调/横盘常是中继
- 盘整小仓对冲保持参与感

风险与注意：
- 现货选标的即隐含方向判断（全多），宏观错会承受完整下行
- "第二次盘整"必须**客观定义**
- 平台币（HYPER/OKX）流动性与相关性需单独评估

### 4.4 L2：50% 合约 + 指数加仓

做法：
- 用现有系统（BPC/TPC/ME/FBF/SRB），改加仓为**指数递增**
- 加仓条件：VWAP1200 斜率向上 + top20 token VWAP 斜率向上分位

合理性：
- 解决加仓不满的问题
- VWAP1200 斜率与 BPC 叙事一致（不在横盘/下行时放大）

风险与注意：
- **指数加仓 + 合约 = 尾部风险极大**；必须有总仓位硬顶与单次加仓上限
- 指数因子不宜过大（1.2x~1.5x 每层），并与 ATR / vol 缩放

### 4.5 L3：C 多腿（`chop_grid` + `trend_scalp`）

**账户**：宪法 `multi_leg` 独立子账户（`run_multi_leg_live.py`），与 L2 PCM trend **API / 保证金隔离**。

**子策略互斥**（同一 symbol 只能跑其一）：

| 策略 | Regime | 赚钱 | 亏钱 |
|------|--------|------|------|
| `chop_grid` | `bpc_semantic_chop` 高（开 ≥0.50，关 <0.32） | 无方向震荡、网格回转 | 单边趋势突破、库存堆积 |
| `trend_scalp` | `trend_confidence` 高 + chop 低 | 趋势延续、fee-aware 短持 | 震荡、频繁翻转 |

与 L2 的协作（见 `docs/strategy/C系统.md`）：

- B 拒高 chop（`semantic_chop > 0.4`）→ 震荡段由 **L3 `chop_grid`** 承接，与 TPC 在 chop 市天然对冲。
- `trend_scalp` 与 TPC 部分 regime 重叠，但执行模型不同（多腿库存 vs PCM 单仓结构止损），**不替代** B 的高置信 swing。
- L3 **不**承担 A 的 fattail、**不**用宽止损替 B 扛 beta。

风控锚点（宪法 `multi_leg`，可随 equity 缩放）：

- `unit_notional` 每格/每腿名义；`max_drawdown_pct` 账户硬停（示意 5%）
- 组合 gross/net 与单币 gross/net 上限（示意 20% / 10% / 8% / 4%）

### 4.6 L4：利润分 6 份 × 100x + Meme token bot

L4 **只用 L2 已实现利润**；ME 100x 与 meme bot **名义不混用**。**不得**从 L1/L3 保证金或 L3 多腿预算挪用。

ME 彩票：

- 回测筛选"极端条件下胜率/赔率极高"的场景
- 分 6 份 × 100x，**6 次机会用完即停**
- 受 §4.2 **`M_glob` + `M_sector`** 双阈门控

Meme bot：

- 资金来自 L2 利润（或 L4 内 meme 子预算，如 ME/meme 70/30）
- 专跑**小市值、高波动、叙事驱动**标的；独立账户 / 监控列表
- 单笔上限、日熔断、流动性门槛、时间止损比主策略更严
- **预算耗尽停火**，不摊平

### 4.7 L1 / L2 / L3 / L4 层间规则

1. **方向协同**：L2 在 L1 持仓方向上**只做同向**（趋势叠加）；反向信号仅用于 **L1 的小仓对冲**。L3 可为中性网格或短持顺势，**不与 L1 方向叙事绑定**。
2. **资金流向**：
   - L1 浮盈**不算**入 L2 / L3 可用保证金
   - L3 使用 **multi_leg 独立预算**，不从 L2 利润拨款（与 L4 区分）
   - L2 已实现利润按比例划到 **L4**（总预算内再拆 ME / meme）
   - L4 预算耗尽即停，L2 再产生利润才补
3. **portfolio drawdown limit**：四层叠加总资产回撤达到 **25%** 时全面减仓（**L4 停火、L3 多腿 halt、L2 降杠杆、L1 不动**）。

## 5. 整体组合建议

综合 α/β + 跨市场 + 加密四层：

| 层 | 资产 | 策略 | 目标 |
|----|------|------|------|
| **跨市场 β** | SPY / QQQ / GLD / TLT（或含 BTC） | 月度动量轮动 | 穿越牛熊的基础 β |
| **加密 β（L1 / A）** | BTC / ETH / 选定标的现货 | `spot_accum` + 慢出场 | 加密类别内的长持 |
| **加密 swing α（L2 / B）** | 合约 PCM | TPC/BPC/ME/FBF/SRB + 指数加仓 | 高置信 trend swing |
| **加密 C（L3）** | 合约 multi-leg | `chop_grid` / `trend_scalp` 二选一 | 震荡现金流 + 短趋势 scalp |
| **彩票（L4）** | 合约 + meme | 100x ME + meme bot | 用 L2 利润博极端倍数 |

## 6. 推荐优先级（统一版）

| 优先级 | 动作 | 原因 |
|------|------|------|
| **P0** | α 与 β **账户物理隔离**（子账户/钱包分开） | 防止心态与仓位互相污染 |
| **P0** | 定义 L1 出场的「第二次盘整 + CVD 背离」为**可计算条件** | 避免主观漂移 |
| **P0** | 给 L2 指数加仓设**总仓位硬顶 + 单次加仓上限** | 防尾部风险失控 |
| **P1** | BTC/ETH 现货**月度定投脚本**落地（VWAP1200 之下倍投可选） | 把 β 做出来 |
| **P1** | 纸上/手动拼一次 **L1+L2+L3** 组合级别回测 | 验证四层叠加回撤可接受 |
| **P1** | 定义 **portfolio drawdown limit**（总回撤 25% 全面减仓） | 四层叠加的兜底 |
| **P1** | 美股 ETF 月度动量（SPY/QQQ/GLD/TLT）**回测脚本** | 跨市场 β 扩展 |
| **P1** | L3 **`chop_grid` / `trend_scalp`** 多腿实盘 OMS 对齐（若未 first-class） | C 层与宪法预算一致 |
| **P2** | 实现第 4.2 节的 **`M_glob` / `M_sector`** 特征 + 阈值 | 主要服务 **L4** 宏观门控 |
| **P2** | 用事件回测框架筛选 **L4 准入条件** | 条件越严越好，宁可不触发 |
| **P2** | L1 盘整期小仓对冲的**规则化**（接 ME/FER 或手写规则） | 避免盘整期乱操作 |
| **P2** | L3 regime 阈值定期复核（`semantic_chop` / `trend_confidence`） | C 层 edge 衰减监控 |
| **P2** | **Meme bot（L4）**：白名单 + 流动性/价差门槛 + 子预算 + 熔断 | 彩票扩容不外溢 |
| **P3** | L4 Meme 独立监控 + 审计日志 | 便于事后关策略或调预算 |
| **P3** | A股 / 港股接入评估 | 除非有特别信息优势，最后做 |

## 7. 一句话结论

**先分离收益来源（α vs β），再分散资产类别（加密 + 美股 ETF 等），最后在加密内部做四层（现货 L1 / 合约 swing L2 / C 多腿 L3 / 彩票 L4）；每一层的退出、仓位上限、层间再平衡都要是可执行的客观规则。**

- **L3 = C 系统**（`chop_grid` + `trend_scalp`），**L4 = 彩票**（100x ME + meme）；勿再把 meme 写在 L3。
