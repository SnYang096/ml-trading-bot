- 大多数 ME 特征是反信号 (high=bad), 与 BPC 相反

### Step 3: Direction 验证

ME 方向由 `direction.yaml` 确定性规则决定 (sign(wpt_energy) 等)。
训练前需验证方向模块不是有害的。

```bash
# 全量验证 (所有策略)
python z实验_005_统一研究/direction_strict_validation.py

# 单策略验证 (指定 predictions.parquet)
python z实验_005_统一研究/direction_strict_validation.py \
  --logs results/train_final_xxx/me/predictions.parquet \
  --strategy me
```

**Phase 1 通过标准** (覆盖率)：
- 覆盖率 100% (每个 bar 都有方向)
- Short 占比 > 15%

**Phase 2 通过标准** (方向质量)：
- `median(rr_in_direction) > median(forward_rr_long)` → 方向模块优于 always-long
- `mean(rr_in_direction) > 0` → 按方向交易总体不亏
- Bad rate(按方向) < Bad rate(always-long) → 踩坑率降低
- Short 子集样本量 >= 1080 → 统计可信

### Step 4: Gate 训练 (方向感知)

**关键变更**: ME 不是 long-only 策略。当前 labels_rr_extreme.yaml
硬编码 `direction: long`，但 ME 会产生做空信号。

**数学关系**: `forward_rr_short = -forward_rr_long`
- 做多踩坑 (forward_rr_long < -0.8) = 做空大赚 (forward_rr_short > 0.8)
- 如果 Gate 只学做多失败模式，会把做空的好机会也 veto 掉

**已实现**: 方向感知标签翻转 (`direction_aware: true`)。

训练时根据 direction.yaml 规则计算 per-bar 方向，翻转 forward_rr:
- `forward_rr = forward_rr_long × direction` → 再判断 < -0.8
- Short 时 MFE/MAE 互换: `MFE_short = MAE_long, MAE_short = MFE_long`
- Gate/Evidence/no_opportunity 三种标签均已启用方向感知
- 配置: `labels_rr_extreme.yaml`, `labels_return_tree.yaml`, `labels_no_opportunity.yaml`

```bash

  --config config/strategies/me \
  --features config/strategies/me/features_gate.yaml \
  --labels config/strategies/me/labels_rr_extreme.yaml \
  --symbol BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT,XRPUSDT,ADAUSDT \
  --timeframe 240T --data-path data/parquet_data \
  --start-date 2023-01-01 --end-date 2026-01-01 \

# Gate 应用 archetype 规则
```
