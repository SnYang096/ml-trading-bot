# Percent Risk Budget with Dynamic Leverage

## 1. Objectives

- Use one unified admission model: `risk_budget_mode: percent`.
- Keep baseline conservative, then expand `bpc-long` risk budget only when portfolio is already in a favorable state.
- Add hard shock-stress protection to avoid hidden gap risk when many positions are at breakeven.
- Apply tiered deleveraging instead of all-at-once liquidation.
- Keep live and event-backtest behavior aligned.

## 2. Core Budget Model

All limits are fractions of equity.

- `max_total_risk_pct`: portfolio total risk budget.
- `max_family_risk_pct`: family budget (`bpc`, `me`, `fer`, `lv`).
- `max_symbol_risk_pct`: single symbol budget.

Per-intent risk increment:

`delta_risk = risk_per_trade(strategy) * size_multiplier`

Admission check:

- `symbol_before + delta_risk <= symbol_cap`
- `family_before + delta_risk <= family_cap`
- `total_before + delta_risk <= total_cap`

Reject logs always print:

- `before`, `delta`, `after`, `cap`, `remain`

## 3. Dynamic Leverage (Profit-Up, Drawdown-Down)

### 3.1 Profit-Up Expansion

Use `expansion` policy to scale caps upward for target families (default `bpc`):

- Count released-risk slots (slots whose risk was reduced to breakeven residual).
- Expand caps in steps:
  - `multiplier = 1 + floor((released_slots - trigger_released_slots) / step_released_slots + 1) * step_multiplier`
  - bounded by `max_family_multiplier`, `max_total_multiplier`.

This means leverage grows only when earlier positions already locked profit.

### 3.2 Drawdown-Down Shrink

Use `shrink.by_drawdown` tiers:

- each tier: `{ drawdown_gte, cap_multiplier }`
- apply the strongest matched multiplier (smallest cap).

This contracts risk budget automatically during adverse periods.

## 4. Shock Stress Budget (Hard Guard)

Estimate gap loss under one-step shock:

- For each slot: `notional ~= slot_risk / stop_pct`
- `stress_loss_slot ~= notional * shock_pct`
- `stress_total = sum(stress_loss_slot) + stress_loss_new_intent`

Hard condition:

- `stress_total <= max_stress_loss_pct`

If breached, reject intent even if normal risk budget still passes.

## 5. Tiered Deleveraging Actions

Deleveraging evaluates total-risk usage ratio:

`usage_ratio = total_risk / total_cap`

Policy:

- `freeze_new_entries_ratio`: above this, block non-add new entries.
- `tiers`:
  - each tier has `{ trigger_ratio, reduce_to_ratio }`
  - if `usage_ratio >= trigger_ratio`, reduce risk down to `reduce_to_ratio * total_cap`.

Execution style:

- Progressive close, worst-loss first, not full liquidation.
- Ranking default:
  1. larger unrealized loss in R first (loss_r desc)
  2. higher slot risk first

## 6. Add Position Compatibility

- Slot quota gating is disabled.
- Virtual slot mapping remains for continuity:
  - same `symbol+archetype` signal becomes `add_position`.
- Therefore add-position behavior is preserved without old slot admission restrictions.

## 7. Runtime Stats and Explainability

Expose runtime block:

- effective caps (`total/family/symbol`)
- current risk usage (`total/family/symbol/slot`)
- stress usage (`before`, `after_intent`, `cap`)
- reject counters by reason
- deleveraging events and emitted evictions

## 8. Rollout

Phase 1:

- Enable dynamic expansion for `bpc-long` family only.
- Keep `me/fer` on conservative caps.

Phase 2:

- Extend policy to `me/fer` only after regression passes:
  - no uncontrolled drawdown increase
  - improved symbol participation under trend regimes.
