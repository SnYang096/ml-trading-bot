# 百分比风险预算与动态杠杆（设计说明）

## 1. 目标

- 统一准入模型：仅使用 `risk_budget_mode: percent`，不再用多套 notional/slot 门槛互相打架。
- 基线保守；仅在组合已处于有利状态（已有仓位锁盈释放预算）时，再对 `bpc-long` 等目标家族放大风险上限。
- **硬性的跳空/冲击压力保护**：避免「账面风险很小、实际缺口风险很大」的盲区，尤其在多仓已保本时。
- **分级去杠杆**：按使用率阶梯收紧/减仓，而不是一次性全平。
- **实盘与事件回测行为一致**：同一套 PCM 逻辑、同一套配置字段，避免「回测能过、实盘另一套」。

### 1.1 目标要点举例说明

**（1）硬性冲击压力（shock-stress）——多仓保本时的「隐性缺口风险」**

- **含义**：正常预算只看「止损到价时亏多少 R」（`slot_risk`）。若止损很紧或已移到保本附近，**单笔记账风险**可以压得很低；但真实持仓的**名义规模**仍约为 `风险 / 止损幅度`。若隔夜或消息导致价格**跳空越过止损**，实际亏损可能远大于「按止损价成交」的假设。
- **例子**：10 个币种各记 0.5% 风险，止损距约 2%，则每笔名义约 `0.5%/2% = 25%` 权益的敞口；合计名义敞口很大。若 `shock_pct=4%`（一次跳空按权益 4% 的跌幅去估），则估算的**冲击亏损**可能超过你设的 `max_stress_loss_pct`，此时应**拒绝新开仓**，即使 `total_risk` 仍低于 `max_total_risk_pct`。
- **与保本的关系**：保本后 `breakeven_release` 会把**预算占用**降到 residual，容易让很多仓位同时「看似低风险」通过总预算；但**名义仍在**，跳空时一起受伤。shock-stress 用 `notional ≈ risk/stop_pct` 把这部分「看不见的名义」算进一条硬上限里。

**（2）分级去杠杆（tiered deleveraging）**

- **含义**：当 `usage_ratio = 当前总风险占用 / 总 cap` 超过阶梯阈值时，不是一键全平，而是先**冻结部分新开**，再按规则**逐笔减仓**（本设计默认：**浮亏 R 最大者优先**）。
- **例子**：`total_cap` 对应 10% 风险预算；`freeze_new_entries_ratio=1.8` 表示占用率达到 180% 时先不再开非加仓新仓。`tiers` 如 `trigger_ratio=2.0 → reduce_to_ratio=1.8`：占用 ≥200% 时，把总占用压回约 180% cap，只平掉最亏的一批，而不是全部平仓。

**（3）实盘与事件回测对齐**

- **含义**：`LivePCM` 的 `decide`、风险记账、`notify_position_closed`、`update_slot_loss_r`、去杠杆驱逐列表等，在 `event_backtest` 与 `order_flow_listener` 里走同一套语义。
- **例子**：回测里某 bar 因 `family_cap` 拒绝的 intent，在实盘同一时刻、同一 `slot_risk_frac` 状态下也应得到相同拒绝；加仓失败时应对预占风险做回滚（避免「假满仓」），两边行为一致便于排查。

---

## 2. 核心预算模型

所有上限均为**相对权益的分数**（与配置中口径一致）。

- `max_total_risk_pct`：组合总风险预算。
- `max_family_risk_pct`：家族预算（`bpc` / `me` / `fer` / `lv`）。
- `max_symbol_risk_pct`：单币种预算。

单笔意图的风险增量：

`delta_risk = risk_per_trade(策略) * size_multiplier`

准入检查：

- `symbol_before + delta_risk <= symbol_cap`
- `family_before + delta_risk <= family_cap`
- `total_before + delta_risk <= total_cap`

拒绝日志会打印：`before`、`delta`、`after`、`cap`、`remain`，并在预算类拒绝时附带 `slot_risk_frac` 明细便于核对占用来源。

## 3. 动态杠杆（顺风放大、逆风收缩）

### 3.1 顺风放大（expansion）

对目标家族（默认 `bpc`）在 caps 上乘以放大系数：

- 统计「已释放风险」的 slot 数量（风险已压到保本 residual 附近）。
- 按阶梯放大 `family/total/symbol` 有效 cap，并受 `max_family_multiplier`、`max_total_multiplier` 等上限约束。

含义：**只有前面仓位已经锁盈、释放预算后，才允许后面把 bpc 的风险上限抬上去**，避免一上来就拉满。

### 3.2 逆风收缩（shrink）

使用 `shrink.by_drawdown` 多层：

- 每层：`{ drawdown_gte, cap_multiplier }`
- 取**匹配到的最严重回撤**对应的乘子（cap 收得最紧）。

含义：回撤扩大时自动收紧可用预算，无需手工改配置。

---

## 4. 冲击压力预算（硬性护栏，可选）

配置块：`risk_budget_policy.stress`（`enabled: true` 时生效）。

对「一步冲击」下的缺口损失做**保守估计**：

- 对每个 slot：`notional ≈ slot_risk / stop_pct`
- `stress_loss_slot ≈ notional * shock_pct`
- `stress_total = 各 slot 之和 + 本笔 intent 若成交带来的增量`

硬条件：

- `stress_total <= max_stress_loss_pct`

若违反：**直接拒绝该意图**，即使常规 `total/family/symbol` 预算仍通过。

**再举一例**：某 slot 记账风险 0.01，有效止损约 1%，则名义约 1 倍权益；`shock_pct=0.04` 时该 slot 约贡献 `1 * 4% = 4%` 权益的冲击估算。多 slot 累加后一旦超过 `max_stress_loss_pct`，就触发拒绝——这正是「保本多仓仍占名义」场景下要防的盲区。

---

## 5. 分级去杠杆动作

去杠杆用总风险**使用率**：

`usage_ratio = total_risk / total_cap`

策略要点：

- `freeze_new_entries_ratio`：超过该比例时，拦截**非加仓**新开仓。
- `tiers`：每层 `{ trigger_ratio, reduce_to_ratio }`；若 `usage_ratio >= trigger_ratio`，将总风险压到约 `reduce_to_ratio * total_cap`。

执行风格：

- **渐进减仓**，优先平**浮亏 R 最大**的仓位，再考虑同亏下风险占用更高的 slot。
- 不是「一次全平」，除非后续业务规则另行定义。

---

## 6. 与加仓（add_position）的兼容

- 关闭 slot 名额式准入门控（`disable_slot_gate`）。
- 仍保留「虚拟 slot」映射：同一 `symbol + archetype` 上再次信号 → 标记为 `add_position`，由执行层与宪法加仓规则处理。
- 因此加仓路径保留，又不会被旧的「slot 满不准进」误伤。

---

## 7. 运行时统计与可解释性

建议对外或日志中可见：

- 当前有效 caps（total / family / symbol）
- 当前风险占用（total / family / symbol / 各 slot）
- stress 估算（before、加本 intent 后 after、cap）
- 按原因分类的拒绝计数
- 去杠杆触发与驱逐列表

---

## 8. 上线节奏建议

**阶段 1**

- 仅对 `bpc-long` 家族开启动态放大；`me` / `fer` 保持保守 cap。
- 先验证 shock-stress、去杠杆关闭或极宽时的基线行为。

**阶段 2**

- 回归通过后再考虑把同类政策扩展到 `me` / `fer`：
  - 组合回撤未失控
  - 趋势行情下多币种参与度符合预期

## 似乎百分比预算风险管理，比slot管理更擅长做这些预算管理，更加高级。
可以这么理解，但更准确的说法是：两者解决的问题不一样。

Slot：管的是「名额 / 并发笔数」，规则简单、好讲给人听；在固定每笔记账、少加仓、少家族时够用。
百分比预算：管的是「风险占权益的比例」这一层，天然适合：多策略不同 R、加仓、保本释放、家族/币种分层、顺风放大/逆风收缩、冲击估算——这些都不是「数几笔仓位」能精确表达的。
所以不是「高级」与否的鄙视链，而是：你的系统已经有多层风险语义时，用 percent 当主账本更贴切；slot 仍可保留成你心里「一笔≈一格」的展示层，和 percent 并不矛盾。

一句话：slot 像数椅子，percent 像管总热量；菜品种类多了，用热量更靠谱。
