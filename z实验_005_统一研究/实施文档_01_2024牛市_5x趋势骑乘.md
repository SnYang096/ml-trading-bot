# 实施文档 01：2024 牛市 5x 趋势骑乘

## 1. 目标定义

在 2024 牛市窗口中验证：

- BPC/ME 顺势策略可以在趋势段通过加仓把总杠杆推进到接近 `5x`
- 加仓过程仍满足无风险加仓约束（先保本再加仓）
- 最终收益由趋势段贡献，而不是靠高频小波动

## 2. 范围与前提

- 时间范围：`2024-01-01 ~ 2024-12-31`
- 策略建议：`bpc-long-240T, me-long-120T`（可加 `me-long-60T` 对照）
- 数据：`data/parquet_data`
- 当前代码已支持：
  - `event_backtest` 导出 `add_position_stats.max_observed_leverage`
  - `rolling_sim` 跨月续仓

## 3. 配置检查清单（先做）

### 3.1 宪法与策略开关

- `config/constitution/constitution.yaml`
  - 对应策略 `allow_add_position: true`
  - `max_add_times` 不要过小（建议 BPC 保持 >=3）

### 3.2 execution 加仓约束

- `config/strategies/bpc-long-240T/archetypes/execution.yaml`
- `config/strategies/me-long-120T/archetypes/execution.yaml`

确认：

- `add_position.max_total_leverage: 5.0`
- `add_position.max_add_times` 与宪法不冲突（更严格者生效）
- `lock_profit_breakeven_trigger_r` 不要过高（避免永远加不上）

### 3.3 管线方向与滚动模式

- `config/prod_train_pipeline_2h.yaml`
  - `strategy_scope.direction: long`
  - `pipeline_mode: rolling_live_sim`

## 4. 实施步骤

### Step A：单策略事件回测确认“能到 5x”

```bash
python scripts/event_backtest.py \
  --strategy bpc-long-240T \
  --start-date 2024-01-01 --end-date 2024-12-31 \
  --output /tmp/bpc_long_2024_event.json \
  --export /tmp/bpc_long_2024_trades.csv \
  --trading-map /tmp/bpc_long_2024_map.html
```

同样跑 `me-long-120T` 一次，重点看：

- `add_position_stats.max_observed_leverage`
- `add_position_stats.add_count`
- `trades` 中 `is_add_position=true` 的占比

### Step B：月度滚动续仓（更贴近实盘）

```bash
mlbot pipeline run --all \
  --config config/prod_train_pipeline_2h.yaml \
  --stage rolling_sim
```

输出目录重点看：

- `_rolling_sim/<run_id>/monthly_ledger.jsonl`
- `fast_month_YYYY-MM/*/end_state.json`
- `stitched_summary.json`

### Step C：月度质量与状态复盘

```bash
mlbot pipeline report-side-state --run-id <run_id> --config config/prod_train_pipeline_2h.yaml
mlbot pipeline debug-quality --run-id <run_id> --month 2024-06 --config config/prod_train_pipeline_2h.yaml
```

## 5. 验收标准（必须量化）

- 杠杆目标：
  - 至少一个主策略在牛市段 `max_observed_leverage >= 4.5`
  - 理想目标 `>= 5.0` 且无明显超限
- 趋势骑乘质量：
  - 加仓交易 `add_trades > 0`
  - 趋势月份（如 2024Q1/Q4）`total_r` 为正且贡献明显
- 风险约束：
  - 不出现“靠频繁近止损换收益”的结构（`near_stop_rate` 不显著恶化）

## 6. 失败诊断与修正顺序

若达不到 5x，按顺序排查：

1. `allow_add_position` 是否开启
2. `max_total_leverage` 是否确为 `5.0`
3. `lock_profit_breakeven_trigger_r` 是否过高（导致不触发加仓）
4. `max_add_times` 是否太小
5. 方向过滤是否误伤（`strategy_scope.direction` 与策略不一致）

## 7. 上线门槛（本目标）

满足以下 3 条才进入下一阶段：

- 连续 2 次重跑结果里，`max_observed_leverage` 与加仓次数稳定
- `rolling_sim` 拼接收益不依赖单月偶然峰值
- 交易地图可见“趋势段放大、震荡段收缩”的直观行为
