# 实施文档 02：2025 震荡市控风险 + FER 接入（精简版）

## 1. 目标

- 在 2025 震荡窗口验证组合是否从“进攻”切到“控回撤”。
- 验证 FER 接入后是否提升稳定性（优先看回撤和波动，不只看收益）。

## 2. 模式选择

### 模式 A：`slow_realistic`（主实验）
- 季度慢变量更新（过去 12 个月）
- 月度 3 个月阈值校准 + 当月测试

### 模式 B：`turbo_fixed_features`（快实验）
- 固定特征，不做特征搜索
- 月度 3 个月阈值校准 + 当月测试

```bash
mlbot pipeline run --all --config config/prod_train_pipeline_2h_turbo_2024bull_thresholds_only_fer_only.yaml --stage rolling_sim --skip-shap 2>&1 | tee log.fer.txt


mlbot pipeline run --all --config config/prod_train_pipeline_2h_turbo_2024bull_thresholds_only_fer_only.yaml --stage fast_month --month 2024-07,2024-08,2024-09 --skip-shap 2>&1 | tee log.fer.txt
```

## 3. FER 诊断日志（支撑/阻力未开仓 & 过早平仓）

开启后追加写入 **JSONL**（每行一条 JSON），默认文件为项目根目录 `fer_sr_diag.jsonl`；建议显式指定路径以免弄混多次运行。

```bash
export MLBOT_FER_DIAG=1
export MLBOT_FER_DIAG_LOG=/home/yin/trading/ml_trading_bot/results/fer_sr_diag_202407-09.jsonl
# 可选：仅抽样记录「未成交」的 entry_eval（0~1，默认 1=全量）
# export MLBOT_FER_DIAG_SAMPLE=0.2

mlbot pipeline run --all --config config/prod_train_pipeline_2h_turbo_2024bull_thresholds_only_fer_only.yaml --stage fast_month --month 2024-07,2024-08,2024-09 --skip-shap 2>&1 | tee log.fer.txt
```

| 字段 | 含义 |
|------|------|
| `kind=entry_eval` | 每次 FER 评估：`outcome` 如 `prefilter_deny`、`no_direction`、`gate_deny`、`entry_filter_deny`、`signal`，以及 PCM 侧 `pcm_timeframe_missing` / `pcm_ema_filter_deny` |
| `sr_snapshot` | `dist_to_nearest_sr`、`sqs_hal_low`、`sr_strength_max` 等与 SR/prefilter 相关的特征快照 |
| `prefilter_reason` | 含 **阈值与实际值**（例如 `actual=2.1` 对比 `<=1.2`） |
| `kind=exit` | FER 平仓：`exit_reason_raw` / `exit_reason_norm`、`hold_minutes`、`time_stop_limit_minutes`、`trailing_activated`、`breakeven_locked` 等，用于对照「是否时间止损 / 保本 / 移动止损过早」 |

跑完后可粗略聚合：

```bash
jq -r 'select(.kind=="entry_eval" and .outcome=="prefilter_deny") | .prefilter_reason' "$MLBOT_FER_DIAG_LOG" | sort | uniq -c | sort -rn | head -20
jq -r 'select(.kind=="exit") | "\(.exit_reason_norm) hold_min=\(.hold_minutes)"' "$MLBOT_FER_DIAG_LOG" | sort | uniq -c
```
