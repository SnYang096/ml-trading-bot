# 实施文档 02：2025 震荡市控风险 + FER 接入（精简版）

## 1. 目标

- 在 2025 震荡窗口验证组合是否从“进攻”切到“控回撤”。
- 验证 FER 接入后是否提升稳定性（优先看回撤和波动，不只看收益）。

## 2. 模式选择

### 模式 A：`slow_realistic`（主实验）
- 季度慢变量更新（过去 12 个月）
- 月度 3 个月阈值校准 + 当月测试

### 模式 B：`turbo_fixed_features`（快实验）
- 固定特征，不做特征搜索
- 月度 3 个月阈值校准 + 当月测试

## 3. 推荐配置

- 策略方向建议先做 short 侧：`strategy_scope.direction: short`
- `symbol_policy.mode: hybrid_carry_forward`
- `symbol_policy.carry_forward_ttl_months: 1~2`

建议复制两份配置分别跑 strict/turbo：

```bash
cp config/prod_train_pipeline_2h.yaml config/prod_train_pipeline_2h_2025_range_strict.yaml
cp config/prod_train_pipeline_2h.yaml config/prod_train_pipeline_2h_2025_range_turbo.yaml
```

## 4. 命令

### 4.1 主跑（两种模式对照）

```bash
# strict
mlbot pipeline run --all \
  --config config/prod_train_pipeline_2h_2025_range_strict.yaml \
  --stage rolling_sim

# turbo
mlbot pipeline run --all \
  --config config/prod_train_pipeline_2h_2025_range_turbo.yaml \
  --stage rolling_sim
```

### 4.2 复盘

```bash
mlbot pipeline report-side-state --run-id <run_id> --config <config.yaml>
mlbot pipeline debug-quality --run-id <run_id> --month 2025-03 --config <config.yaml>
mlbot pipeline debug-quality --run-id <run_id> --month 2025-09 --config <config.yaml>
```

### 4.3 组合回测（可选）

```bash
mlbot pipeline run --all --config <config.yaml> --stage pcm_joint
```

## 5. 验收标准

- `stitched_total_r > 0`（最低门槛）
- `max_drawdown_r` 相比 2024 牛市实验明显收敛
- FER 接入后组合回撤/波动下降（即使收益仅小幅提升也算通过）

## 6. 失败排查顺序

1. 方向范围是否匹配震荡结构（short-only / all）。  
2. ME 止损是否过于进攻。  
3. FER 样本是否过少（`n_trades` 低）。  
4. slot 配置是否长期挤掉 FER。
