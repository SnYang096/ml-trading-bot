# 实施文档 04：三层仓位架构 — 现货底仓 + 合约系统 + 高杠杆彩票

## 0. 问题背景

当前合约系统总仓位常常 **不如全仓现货大**，加仓不满时甚至更小；
频繁出场导致 **错过大段趋势**；结果「有系统但赚不到大钱」。

核心矛盾：**系统化合约交易的精细风控 vs 趋势行情需要的持仓耐心与规模**。

---

## 1. 三层架构概览

| 层                  | 资金占比        | 策略                                           | 目标                                    |
| ------------------- | --------------- | ---------------------------------------------- | --------------------------------------- |
| **L1 — 现货底仓**   | ~50%            | 买入持有 + 盘整内小仓对冲                      | 跟上大趋势、不被甩下车                  |
| **L2 — 合约系统**   | ~50%            | 现有 BPC/ME + 指数加仓                         | 趋势阶段放大收益                        |
| **L3 — 高杠杆彩票** | L2 利润的一部分 | **100x ME** + **Meme token bot**（独立子预算） | 用已赚利润博极端倍数；meme 与主策略解耦 |

## 1.1 全局与板块宏观条件特征（操作板块 + 进入彩票门控）

> **预览说明**：`\( \)`、`\dfrac` 等是 LaTeX，**默认 Markdown 预览常常不渲染**。下面改用 **代码块 / 行内代码** 写同一套定义；若在 GitHub 或装了 KaTeX/Math 的预览里，也可自行改成 `$...$` 形式。

### 定义（建议写进特征/门控配置，量纲需统一）

对 **标的** 与 **板块指数** 各算一套同构标量（再分别用于「选板块 / 权重」与「L3 是否允许开火」）：

```text
# 相对锚位置（距离 pct），P 为收盘价或标记价，VWAP1200 为 1200 周期 VWAP
d = (P - VWAP1200) / VWAP1200

# 锚的趋势强度：VWAP1200 对时间的斜率（再归一化）；「角度」建议定义为与斜率单调对应的有界量
# 例如：a = tanh(slope) 或 a = atan(slope) 缩放后的标量，避免量纲随价格尺度漂移
a = normalize( slope(VWAP1200) )

# 组合宏观分（用户原意：距离 pct × 角度/斜率）
M = d * a
```

1. **d**：上式第一行；若多空语义易混，可拆成 **符号(sign)** 与 **|d|** 分开进规则。  
2. **a**：上式第二行，**normalize** 的具体函数写死（clip / tanh / 分位数），全样本可比。  
3. **M**：上式第三行；上线前应对 **d、a 做 winsorize** 再相乘，避免极端 bar 把 M 打爆。

- **全局**：对 **基准标的**（如 BTC 或 BTC+ETH 合成）算 **`M_glob`**。  
- **板块**：对每个 **板块指数序列**（成分与权重规则写死）同样算 **`M_sector`**。

### 用途（与三层对齐）

| 用途 | 建议用法 |
|------|----------|
| **操作哪一板块（L2 资金与信号权重）** | 在多个板块间，用 **`M_sector` 排序或分档**（只做 top 档、或对末档降杠杆）；与单币 BPC/ME 信号 **与** 关系，避免单币强但板块逆风的重仓。 |
| **是否进入彩票阶段（L3 门控）** | **硬条件建议**：**`M_glob`** 与 **目标板块 `M_sector`** 同时满足符号与幅度阈值（例如同向且 **`abs(M)`** 超过分位数阈值）才允许从 L2 利润划款到 L3 / 才允许 meme bot 与 100x ME 开火；否则 **L3 整层关闭**，利润留存或回滚 L2。 |

### 合理性分析（结论先说）

**合理之处**

1. **语义清晰**：**`d`** 管「相对 VWAP 的位置」，**`a`** 管「锚自身是否在走趋势」；**`M = d * a`** 强调 **「有趋势 + 价格已离开锚」**，比单看斜率或单看偏离更能过滤「横盘微斜率噪声」。  
2. **板块层有必要**：单币信号在 **板块 beta** 逆风时假突破多；板块版 **`M`** 作为 **第二层 prior** 与现有 prefilter 分工一致（全局/板块 = 慢变量语境，单币 = 执行层）。  
3. **放进 L3 门控特别合适**：彩票层最怕 **逆宏观、逆板块** 的裸奔；用 **`M_glob` + `M_sector`** 做 **允许亏、不允许在明显逆风时亏** 的开关，和「利润博赔率」叙事一致。

**需要注意的问题**

1. **量纲与饱和**：**`d`** 与 **`a`** 直接相乘，任一侧噪声或极端值会把 **`M`** 打爆；建议 **两侧分别 clip / winsorize** 再乘，或对 **`M`** 再做 **tanh** 或分位数映射到 **[-1, 1]**。  
2. **多空符号**：做多彩票时，若 **`d`** 已带符号，**`a`** 也应约定 **同向为正**；否则会出现「数学上很大、交易语义很怪」的象限，需要 **象限规则**（例如仅允许 **`d > 0` 且 `a > 0`** 做多 meme）。  
3. **与 L2 内已有 VWAP 特征的相关性**：BPC/ME 里若已有大量 VWAP 类特征，全局 **`M`** 再进 L2 可能 **信息重复**；更干净的分工是：**`M_sector` 主要驱动「板块选择/权重」，`M_glob` 主要驱动「L3 总开关」**，L2 单币仍走现有 gate。  
4. **板块指数定义**：必须 **固定成分、再平衡频率、缺失填充**，否则回测与实盘不可复现。  
5. **滞后**：VWAP1200 与斜率均 **滞后**；作 L3 门控时可以接受（彩票本就不追求抄底），但不要指望它 **精确逃顶**。

**总评**：把 **「全局宏观 × 板块宏观」** 作为 **选板块依据 + L3 准入依据** 在逻辑上 **站得住**，且能补当前「系统化单币」对 **beta/板块** 表达不足的一块；工程上关键是 **归一化、符号约定、与 L2 特征去重、板块指数契约**。

## 2. 各层分析

### L1：50% 现货底仓（BTC, SOL, HYPER, BNB, OKX）

**做法**：
- 买入后 **不因第一次盘整出场**，即便跌破 VWAP1200
- 盘整期用 **≤1/10 仓位** 做短期多空，降低持仓成本
- 出场条件：**第二次盘整 + 大周期 CVD 背离**

**合理性**：
- 解决了「合约频繁出场错过大段」的核心问题 — 现货天然持有，不吃资金费、不爆仓
- 「第一次盘整不出」符合趋势交易经验：大行情里第一次回调/横盘常是中继
- 盘整期小仓对冲（而非空仓等待）保持参与感、降低心理成本
- CVD 背离作为大周期退出信号，语义清晰

**风险与注意**：
- 现货标的选择即隐含 **方向判断**（全多）；若宏观趋势判断错，50% 底仓会承受完整下行
- 「第二次盘整」需要 **客观定义**（不能靠感觉），否则容易变成「一直不出」
- 1/10 仓位对冲的 **执行纪律** 要明确：什么条件开、什么条件平，避免变成「盘整里反复亏小钱」
- HYPER / OKX 等平台币流动性和相关性要单独评估，避免集中风险

### L2：50% 合约 + 指数加仓

**做法**：
- 仍用现有系统（BPC/FER/ME），但改 **加仓方式为指数递增**（越加越大）
- 加仓条件：**VWAP1200 斜率向上**，top20token vwap斜率向上pct

**合理性**：
- 解决了「加仓不满 → 仓位不如现货」的问题 — 指数加仓在趋势确认后放大敞口
- VWAP1200 斜率作为趋势确认条件，与 BPC 叙事一致（不在横盘/下行时放大）
- 和 L1 形成互补：L1 保底仓不出，L2 在趋势加速段放大

**风险与注意**：
- **指数加仓 + 合约 = 尾部风险极大**。趋势末端若回调，最大仓位在最高价，回撤会远超线性加仓
- 必须有 **总仓位上限**（例如名义不超过本金 N 倍）和 **单次加仓上限**
- VWAP1200 斜率的 **滞后性**：趋势末段斜率仍正，但价格已在分配；需配合其它信号（如 CVD 背离）限制继续加仓
- 建议：**指数因子不宜过大**（如 1.2x~1.5x 每层），且配合 **ATR / vol 缩放**，避免低波时名义仓位爆炸

### L3：利润分 6 份 × 100x 杠杆 + **Meme token bot**

**做法（ME 彩票，不变）**：
- 仅用 **L2 已实现利润**
- 回测筛选「哪些极端条件下 ME 或其它策略胜率/赔率极高」
- 分成 6 份，每份开 100x，6 次机会

**做法（Meme token bot — 并入彩票层）**：
- **资金**：仍只来自 **L2 已实现利润**（或 L3 总预算内再划一块 **meme 子预算**，与 6 份 ME **不混用名义**）
- **定位**：专门跑 **高波动、小市值、叙事驱动** 的 meme；与 BPC/FER 主叙事 **隔离**，避免污染 L2 风控
- **执行**：独立 bot（监控列表、上线/下架规则、单笔上限、日熔断、流动性门槛）；**不允许**从 L1 底仓或 L2 主保证金直接挪用
- **退出**：比 ME 100x 更严 — 建议 **时间止损 + 流动性枯竭即砍 + 单币最大亏损占 meme 子预算比例**（meme 尾部比主流合约更肥）

**合理性**：
- **风险隔离做得好**：用的是已赚的钱，不是本金；6 次分配 = 每次只赌 1/6 利润
- Meme bot 放进 **L3** 而不是 L2：符合「彩票」语义，不把主系统的 Sharpe/回撤假设搞脏
- 从行为金融角度：把「想博暴利」的冲动 **制度化**，避免在 L1/L2 里冲动放大杠杆
- 如果条件筛选做得好（极少触发、历史 payoff 极正偏），期望值可以为正

**风险与注意**：
- 100x 杠杆 **几乎 = 全输或暴利**，滑点和资金费在极端行情里会吃掉很多
- **Meme 特有风险**：拉盘/砸盘极快、合约深度差、**插针爆仓**、部分标的 **合约不存在或下架**；需白名单 + 最小 24h 成交额 + 最大价差
- 「哪些条件可以开 100x」— 回测里能选出极端正偏的条件，但 **样本一定极少**，统计显著性存疑；**meme 回测更易过拟合**，更要 **纸上 → 极小资金** 两阶
- 建议 **先用模拟 / 极小真金白银** 跑一段，验证「极端条件触发频率」和「实际滑点」
- 6 次机会 **用完就停**，直到 L2 再产生利润 — 这条纪律必须写死；**meme 子预算耗尽同样停**，不补仓摊平

## 3. 组合层面评价

### 优点

1. **解决了核心矛盾**：L1 保证跟上大趋势（仓位够大、不轻易出），L2 精细化放大，L3 处理「博暴利」冲动
2. **风险分层清晰**：现货 → 合约 → 彩票，风险递增但资金递减
3. **利润再投资逻辑合理**：L3 用的是 L2 利润，不侵蚀本金

### 需要补的

1. **各层之间的资金再平衡规则**
   - L1 现货浮盈是否算入 L2 可用保证金？
   - L2 亏损到什么程度缩减仓位或暂停？
   - L3 的「利润」怎么计算 — 已实现的还是浮动的？

2. **L1 与 L2 的方向冲突处理**
   - L1 全多持仓 + L2 如果出空信号 — 是否允许？算对冲还是矛盾？
   - 建议：L2 在 L1 持仓方向上 **只做同向**（趋势叠加），反向信号仅用于 **L1 的小仓对冲**

3. **熊市/大幅回调的整体预案**
   - L1 若跌 30%~50%，是否有 **硬止损 / 减仓线**？还是真的「死扛到第二次盘整」？
   - 建议至少有一个 **portfolio-level drawdown limit**（例如总资产回撤 25% 时全面减仓）

4. **L3 的条件定义**
   - 需要从历史数据中筛选「极端正偏条件」，建议用 **事件回测框架**（已有）做
   - 条件越严格越好（宁可不触发，不要乱触发）
5. **L3 内 ME 与 Meme bot 的预算切分**
   - 例如：L3 总预算 = 某期 L2 利润的 X%；其中 **ME 6 份** 与 **meme 子预算** 比例写死（如 70/30），避免 meme 把利润烧光后 ME 无子弹

6. **全局 / 板块 `M` 特征落地**
   - 在特征管道或单独 macro 配置中实现 **第 1.1 节** 的 `d`、`a`、`M`；输出 **全局序列 + 各板块序列**；L2 只做排序/降权时读板块 **`M_sector`**，L3 门控读 **`M_glob` + 目标板块 `M_sector`** 双阈值
   - 与 `macro_tp_vwap_1200_position` 等现有键对齐命名与 bar 对齐，避免双写口径

## 4. 推荐优先级

| 优先级 | 动作                                                                                | 原因                     |
| ------ | ----------------------------------------------------------------------------------- | ------------------------ |
| **P0** | 定义 L1 出场的「第二次盘整」+ CVD 背离为 **可计算条件**                             | 避免主观判断漂移         |
| **P0** | 给 L2 指数加仓设 **总仓位硬顶** + 单次加仓上限                                      | 防止尾部风险失控         |
| **P1** | 用现有系统跑一段 L1+L2 的 **纸上组合回测**（哪怕手动拼）                            | 验证整体回撤是否可接受   |
| **P1** | 定义 **portfolio drawdown limit**                                                   | 三层叠加后的最坏情况兜底 |
| **P2** | 用事件回测框架筛选 L3 条件                                                          | 不急，等 L2 有利润再做   |
| **P2** | L1 盘整期小仓对冲的 **规则化**（接入 ME/FER 或手工规则）                            | 避免盘整期乱操作         |
| **P2** | **Meme token bot**：白名单 + 流动性/价差门槛 + 子预算 + 熔断，与 L2 账户/策略树隔离 | 彩票层扩容但风险不外溢   |
| **P3** | Meme 独立监控与 **审计日志**（触发原因、标的、PnL）                                 | 便于事后关策略或调预算   |
| **P2** | 实现 **第 1.1 节** 全局/板块 **`M`**，并写 **L3 准入** 与 **板块权重** 两套阈值（先纸上验收） | 把宏观门控从叙事落到可测规则 |

## 5. 一句话结论

三层架构的 **思路合理**：现货解决「跟上趋势」，合约解决「放大收益」，彩票处理「博暴利冲动」（含 **ME 极端杠杆** 与 **meme 专用 bot**，均在 L3、吃利润不占本金）。**全局 / 板块 `M`（距离 pct × VWAP1200 斜率）** 作为 **选板块** 与 **L3 总开关** 在逻辑上可补强 beta 语境，但须做好归一化与指数契约。
关键不在于「想法对不对」，而在于 **每一层的退出条件、仓位上限、层间再平衡** 是否有可执行的客观规则。
先把 L1 出场条件和 L2 加仓上限定死，再跑一段组合级别回测，比直接上线更稳。
