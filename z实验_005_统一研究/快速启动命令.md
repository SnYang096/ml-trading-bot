
**漂移报告输出**:
- Metrics 对比表 (Sharpe / Trades / WinRate / MeanR 变化百分比)
- 逐文件分析: prefilter 阈值漂移、gate 规则增删、execution 参数变化等

> **注意**: 每次 pipeline 跑完时，如果存在上一次实验，会**自动输出漂移报告**，无需手动 `--diff`。

## 四、DEPLOY (config → live)

> 将研究确认的 config/strategies/ 部署到 live/highcap/config/strategies/
> 脚本: `scripts/deploy_config_to_live.py`

### 4.0 Deploy 触发条件

**不是每次研究变了就 deploy。** 核心逻辑：“旧的不行了，新的更好” 才 deploy。

规则配置在 `config/research_pipeline.yaml` 的 `deploy_gate:` 中：

**触发条件 (OR, 至少一个满足才值得 deploy):**

| 规则 | 条件 | 含义 |
|------|------|------|
| sharpe_improve | 提升 >= 5% | 新版本显著更好 |
| drift_level | >= HIGH 且 \|Sharpe变化\| > 3% | 参数大幅漂移 + 性能确实有变 |

> **稳定性保护**: 即使漂移达 HIGH，若 Sharpe 变化 <= 3%，则不触发——防止数据窗口滑动导致的自然漂移误触发。

**安全门禁 (AND, 全部满足才允许 deploy):**

| 规则 | 条件 | 含义 |
|------|------|------|
| require_adopt | 决策 = ADOPT | 新版不比旧版差 |
| min_trades | >= 50 | 统计显著性保障 |
| human_confirm | 默认 true | 部署前需人工确认 |

**决策矩阵:**

| 漂移\\Sharpe变化 | 提升>=5% | \|变化\|>3% | \|变化\|<=3% |
|---|---|---|---|
| HIGH | ✅ DEPLOY | ✅ DEPLOY | ❌ SKIP (漂移大但性能没变) |
| < HIGH | ✅ DEPLOY | ❌ SKIP | ❌ SKIP |

pipeline 跑完会自动显示：
- `🚀 Deploy: ✅ 值得且允许 deploy` + deploy 命令
- `⏭️  Deploy: SKIP` — Sharpe 提升不足 且 漂移较小/性能稳定, 不需要动实盘
- `🚫 Deploy: ❌` — 有触发但安全门禁未通过 + 具体原因

结果保存在 `report.json` 的 `deploy_gate` 字段中。

### 4.1 查看差异 (不部署)

```bash
# 查看所有策略的 config/ vs live/ 差异
python scripts/deploy_config_to_live.py --diff

# 只看某个策略
python scripts/deploy_config_to_live.py --diff --strategy bpc
```

### 4.2 执行部署

```bash
# 部署所有策略 (交互确认)
python scripts/deploy_config_to_live.py --deploy

# 部署指定策略
python scripts/deploy_config_to_live.py --deploy --strategy bpc me

# 部署 + 自动 git commit
python scripts/deploy_config_to_live.py --deploy --git-commit

# 非交互模式 (自动化/CI)
python scripts/deploy_config_to_live.py --deploy --yes --git-commit
```

### 4.3 回滚

```bash
# 查看回滚指引 + 最近的部署 commit
python scripts/deploy_config_to_live.py --rollback

# 手动回滚上一次部署
git revert HEAD

# 手动恢复到指定版本
git checkout <commit-hash> -- live/highcap/config/strategies/
```

### 4.4 完整部署流程 (研究 → 实盘)

```bash
# Step 1: 运行研究 pipeline
python scripts/auto_research_pipeline.py --strategy fer

# Step 2: 确认决策 (应显示 ADOPT)
python scripts/auto_research_pipeline.py --strategy fer --list

# Step 3: 查看 config/ vs live/ 差异
python scripts/deploy_config_to_live.py --diff --strategy fer

# Step 4: 部署到 live/
python scripts/deploy_config_to_live.py --deploy --strategy fer --git-commit

# Step 5: 验证部署
ls live/highcap/config/strategies/fer/archetypes/
```

## 九、常见问题

### Q: Feature Store 太慢怎么办?
A: Feature Store 是增量构建的，已有月份自动跳过。首次全量构建耗时较长 (~1h/策略)，后续增量只需几分钟。

### Q: ADOPT 后如何回退?
A: `config/strategies/` 在 git 中，`git revert` 即可回退。

### Q: DEPLOY 后如何回退?
A: `python scripts/deploy_config_to_live.py --rollback` 查看指引，或直接 `git revert HEAD`。

### Q: 如何只重跑某一步?
A: 参考第六节手动分步执行，找到对应 Step 的命令单独运行。

### Q: ME 的 timeframe 是什么?
A: ME 使用 60T (1小时), BPC/FER 使用 240T (4小时)。pipeline 配置中已自动区分。
