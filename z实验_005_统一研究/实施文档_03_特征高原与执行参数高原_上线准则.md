# 实施文档 03：高原参数与上线准则（精简版）

## 1. 目标

- 找到“高原解”而非尖峰最优：
  - 慢变量结构高原（特征/规则）
  - 快变量 execution 高原
  - 组合 slot 高原
- 形成可回滚、可复现的上线参数集。

## 2. 原则

- 优先稳健性：高原内低回撤、低近止损优先于最高 Sharpe。
- 必须跨环境稳定：至少覆盖 2024（趋势）+ 2025（震荡）。

## 3. 两种运行模式

- `slow_realistic`：季度慢变量 + 月度 3 个月阈值校准 + 当月测试（主流程）
- `turbo_fixed_features`：固定特征 + 月度 3 个月阈值校准 + 当月测试（快流程）

## 4. 命令流程

### 4.1 慢变量结构（可选专项）

```bash
mlbot pipeline run --all --config <config.yaml> --stage slow_snapshot
```

### 4.2 execution 高原（可选专项）

```bash
mlbot pipeline run --all --config <config.yaml> --stage execution_opt
mlbot pipeline run --all --config <config.yaml> --stage event_backtest
```

### 4.3 组合回测（可选专项）

```bash
mlbot pipeline run --all --config <config.yaml> --stage pcm_joint
```

### 4.4 最终总验收（必须）

```bash
# strict
mlbot pipeline run --all --config <strict_config.yaml> --stage rolling_sim

# turbo
mlbot pipeline run --all --config <turbo_config.yaml> --stage rolling_sim
```

## 5. 验收与评分卡

- `monthly_ledger.jsonl` 与 `stitched_summary.json` 一致。
- 高原判定通过：
  - 邻域参数不明显退化
  - `max_drawdown_r`、`near_stop_rate` 可接受
- 跨 2024/2025 都通过：
  - 不依赖单月爆发
  - 发生退化有明确回滚路径

建议评分卡：

- 慢变量高原：通过/不通过
- execution 高原：通过/不通过
- slot 高原：通过/不通过
- 2024 目标：通过/不通过
- 2025 目标：通过/不通过

任一关键项不通过，不上线。
