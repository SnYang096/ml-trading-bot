# 实施文档 03：特征高原 + 执行层参数高原 + 上线准则

## 1. 目标定义

建立一套可复现流程，找到：

- 特征层“平坦高原”（不是单点最优）
- execution 参数“平坦高原”
- 可上线参数集（稳健优先，不追求单次最高分）

## 2. 核心原则

- 只接受“高原内稳定解”，拒绝“尖峰最优”
- 指标必须跨时间窗稳定（至少 bull + range 两种环境）
- 上线参数必须可解释、可回滚、可复现

## 3. 产物契约

每次实验至少保留以下文件：

- 特征筛选输出（prefilter/gate 相关实验报告）
- execution 网格输出：`event_exec_opt.json`
- 组合层高原输出：`pcm_slot_grid_report.json`
- 滚动拼接输出：`monthly_ledger.jsonl` + `stitched_summary.json`

## 4. 实施步骤

### Step A：特征层高原（慢变量）

建议流程：

1. 跑 `slow_snapshot` 固定慢变量快照
2. 对 prefilter/gate 做“小范围网格”而非大范围暴搜
3. 每组参数至少在 2 个窗口测试（2024 + 2025）

命令入口：

```bash
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage slow_snapshot
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage prefilter
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage gate
```

高原判定（建议）：

- Top-N 方案中，Sharpe/TotalR 相差不大（例如在最佳值 90~95% 区间）
- `max_drawdown_r`、`near_stop_rate` 更低者优先

### Step B：execution 参数高原（快变量）

对每个策略运行事件优化：

```bash
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage execution_opt
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage event_backtest
```

或单策略手动：

```bash
python scripts/optimize_event_execution.py \
  --strategy me-short-120T \
  --start-date 2024-01-01 --end-date 2025-12-31 \
  --sym-r 1.0:0.5:4.0 \
  --output /tmp/me_short_exec_opt.json
```

高原判定（建议）：

- 不选单点最高 Sharpe
- 选“邻域参数”都不过度退化的点
- 同时约束：
  - `near_stop_rate`
  - `max_drawdown_r`
  - `n_trades`（避免样本太少）

### Step C：组合层高原（slot 与资源分配）

```bash
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage pcm_slot_grid
```

从 `pcm_slot_grid_report.json` 选“高原推荐 case”，不是最高分 case：

- 先看 `score` 在高原范围内
- 再按 `max_drawdown_r`、`slot_full_rate`、`sharpe_daily` 排序

### Step D：上线前滚动总验收

```bash
mlbot pipeline run --all --config config/prod_train_pipeline_2h.yaml --stage rolling_sim
```

要求最终 `stitched_summary.json` 与月度账本一致，且地图可解释。

## 5. 上线评分卡（建议模板）

上线前给每组参数打分（通过/不通过）：

- 特征高原稳定性：通过/不通过
- execution 高原稳定性：通过/不通过
- slot 高原稳定性：通过/不通过
- 2024 牛市目标：通过/不通过
- 2025 震荡目标：通过/不通过

只要任一关键项“不通过”，不上线。

## 6. 回滚与灰度

- 回滚优先级：
  1. execution 参数
  2. slot case
  3. 特征结构
- 灰度建议：
  - 先单策略小资金 shadow
  - 再多策略低风险实盘
  - 最后恢复目标风险预算

## 7. 最终上线判据（硬门槛）

满足以下条件再上线：

- 多窗口（2024/2025）都在可接受回撤内
- 不是靠单月爆发支撑整体
- 参数在邻域内稳定，具备高原特性
- 发生退化时有明确回滚路径
