# 实盘Tick数据存储验证脚本使用说明

## 📋 脚本概述

提供了两个脚本用于验证实盘tick数据存储：

1. **`scripts/verify_live_tick_storage.py`** - 一次性完整验证
2. **`scripts/monitor_live_tick_storage.py`** - 实时持续监控

---

## 🔍 脚本1: 完整验证 (`scripts/verify_live_tick_storage.py`)

### 功能说明

执行三项完整验证：
1. **Tick数据格式验证** - 检查格式、频率、完整性
2. **VPIN特征计算验证** - 验证VPIN能正确使用新tick数据
3. **存储性能监控** - 监控磁盘占用和读写性能

### 使用方法

#### 基本用法（使用默认参数）

```bash
cd /home/yin/trading/ml_trading_bot
python scripts/verify_live_tick_storage.py
```

#### 指定参数

```bash
# 验证特定币种和日期
python scripts/verify_live_tick_storage.py \
  --symbol ETHUSDT \
  --date 2026-02-12 \
  --base-path live/highcap/data
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--symbol` | BTCUSDT | 交易对符号 |
| `--date` | 今天 | 验证日期 (YYYY-MM-DD) |
| `--base-path` | live/highcap/data | 数据基础路径 |

## 📊 脚本2: 实时监控 (`scripts/monitor_live_tick_storage.py`)

### 功能说明

持续监控实盘tick数据存储，提供：
1. **实时写入速率监控** - 每分钟写入多少条
2. **存储空间趋势** - 存储增长速度
3. **系统资源监控** - CPU、内存、磁盘使用情况
4. **异常告警** - 写入速率异常、空间不足等

### 使用方法

#### 基本用法（监控1小时）

```bash
cd /home/yin/trading/ml_trading_bot
python scripts/monitor_live_tick_storage.py
```

#### 后台运行（推荐用于长时间监控）

```bash
# 监控24小时，每60秒检查一次
nohup python scripts/monitor_live_tick_storage.py \
  --duration 1440 \
  --interval 60 \
  > monitor_output.log 2>&1 &

# 查看监控输出
tail -f monitor_output.log
```

#### 指定参数

```bash
# 监控特定币种，每30秒检查一次，运行2小时
python scripts/monitor_live_tick_storage.py \
  --symbol ETHUSDT \
  --duration 120 \
  --interval 30 \
  --base-path live/highcap/data
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--symbol` | BTCUSDT | 交易对符号 |
| `--duration` | 60 | 监控时长（分钟） |
| `--interval` | 60 | 检查间隔（秒） |
| `--base-path` | live/highcap/data | 数据基础路径 |

## 🔧 常见问题

### Q1: 验证脚本报错"未找到tick数据"

**原因**: 
- 实盘系统还未运行
- 指定的日期没有数据
- 数据路径不正确

**解决方案**:
```bash
# 检查数据目录
ls -lh live/highcap/data/ticks/BTCUSDT/

# 使用正确的日期
python scripts/verify_live_tick_storage.py --date 2026-02-11
```

### Q2: VPIN计算失败

**原因**:
- tick数据格式不正确
- 数据量不足（VPIN需要至少50个桶的数据）

**解决方案**:
```bash
# 检查tick数据格式
python -c "
import pandas as pd
ticks = pd.read_parquet('live/highcap/data/ticks/BTCUSDT/2026-02-12.parquet')
print(ticks.head())
print(ticks.dtypes)
"
```

### Q3: 监控脚本占用资源过高

**解决方案**:
```bash
# 降低检查频率
python scripts/monitor_live_tick_storage.py --interval 300  # 5分钟检查一次
```

### Q4: 存储空间增长过快

**分析**:
1. 查看监控输出的"预测月度存储需求"
2. 如果超过预期（~8MB/月/币种），检查：
   - 是否有重复数据
   - 是否tick频率异常高

**排查命令**:
```bash
# 检查文件大小
du -sh live/highcap/data/ticks/*/
du -sh live/highcap/data/bars/*/

# 检查tick数量
python -c "
import pandas as pd
from pathlib import Path
for f in Path('live/highcap/data/ticks/BTCUSDT/').glob('*.parquet'):
    df = pd.read_parquet(f)
    print(f'{f.name}: {len(df)} 条')
"
```

## 🚀 快速开始

### 1分钟快速验证

```bash
cd /home/yin/trading/ml_trading_bot

# 完整验证
python scripts/verify_live_tick_storage.py

# 如果全部通过，启动监控
python scripts/monitor_live_tick_storage.py --duration 60
```

### 生产环境部署

```bash
# 1. 添加到systemd服务（持续监控）
sudo tee /etc/systemd/system/tick-monitor.service > /dev/null <<EOF
[Unit]
Description=Tick Storage Monitor
After=network.target

[Service]
Type=simple
User=yin
WorkingDirectory=/home/yin/trading/ml_trading_bot
ExecStart=/usr/bin/python3 scripts/monitor_live_tick_storage.py --duration 999999 --interval 60
Restart=always

[Install]
WantedBy=multi-user.target
EOF

# 2. 启动服务
sudo systemctl daemon-reload
sudo systemctl enable tick-monitor
sudo systemctl start tick-monitor

# 3. 查看状态
sudo systemctl status tick-monitor
sudo journalctl -u tick-monitor -f
```
