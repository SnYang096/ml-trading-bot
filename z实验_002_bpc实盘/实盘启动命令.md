### 1. 准备 warmup 数据（启动前执行，替代旧的 build_feature_store）

```bash
cd /home/yin/trading/ml_trading_bot

# 下载最近 6 个月 monthly + 最近 1 个月 daily aggTrades
# 转换为 1min 聚合 ticks/bars 写入 live/highcap/data/
bash live/scripts/prepare_warmup_ticks.sh highcap 6
```

- **作用**：为 `highcap` universe 中的全部币种下载并转换最近 6 个月的 1min 聚合 ticks 和 bars。
- **产出目录**：`live/highcap/data/ticks/` 和 `live/highcap/data/bars/`
- **首次运行**：约 10-30 分钟（需下载），后续重新运行跳过已下载文件
- **可指定币种**：

```bash
python live/scripts/prepare_warmup_ticks.py --universe highcap --months 6 --symbols BTCUSDT,ETHUSDT
```

```bash

# 密钥已从 config/local/binance_mainnet.env 复制到 live/binance_mainnet.env
```

- **作用**：实盘系统需要 Binance API 密钥来访问市场数据和下单。

  - 观察模式（`TRADE_SIZE=0.0`）不需要 API 密钥，但实际交易必须配置

### 3. 可选：单独运行依赖自检（预检查配置与数据）

```bash
cd /home/yin/trading/ml_trading_bot
# 方式1：使用 bash 脚本（推荐，会自动加载 API 密钥）
bash live/scripts/check_dependencies.sh highcap

# 方式2：Python 脚本（需手动设置环境变量）
python live/scripts/check_dependencies.py \
  --symbols "BTCUSDT,ETHUSDT,BNBUSDT,SOLUSDT,XRPUSDT,ADAUSDT" \
  --live-root live/highcap
```

- **作用**：在不真正启动实盘的情况下，检查：
  - `live/highcap/config/` 中的 constitution 与 BPC 配置是否完整；
  - `live/highcap/data/ticks/` 是否存在历史数据（无则警告）；
  - Binance API 密钥是否配置。
- **检查项目**：全部通过才能启动实盘。

> 提示：第 4 步的启动脚本内部也会自动调用依赖自检；本步主要用于提前排查问题。

### 4. 启动 highcap universe 实盘（带依赖自检 + 质量闸门）

```bash
cd /home/yin/trading/ml_trading_bot
./live/scripts/start_live.sh highcap
```

- **内部流程摘要（策略B）**：
  - 解析 `universe = highcap`，从 `live/highcap/universe.yaml` 读取币种列表；
  - **加载 `live/binance_mainnet.env` 中的 API 密钥**；
  - 设置 `PYTHONPATH` 与一系列 `MLBOT_LIVE_*` 环境变量；
  - 调用 `check_dependencies.py` 做依赖自检（失败则直接退出 OFFLINE）；
  - 调用 `scripts/run_live.py`：
    - 初始化 `BPCLiveStrategy` 与多币种 `MultiSymbolManager`；
    - 加载 `live/highcap/data/ticks/` 中的历史 1min ticks（由 `prepare_warmup_ticks.sh` 预先准备）；
    - 通过 `on_bar()` 逐条喂入 `IncrementalFeatureComputer`，重建流式特征状态；
    - 启动 WebSocket 接收实时 ticks，开始累积实时数据；
    - **基于实时数据累积量判定模式**（不依赖 Feature Store）：
      - 阈值：≥240 bars (4h) → NORMAL，120-240 bars (2-4h) → DEGRADED，<120 bars → OFFLINE
    - NORMAL → 正常交易（约启动后 4h）；
    - DEGRADED → 仅观察不交易（2-4h 之间）；
    - OFFLINE → 继续接收数据，等待累积（<2h）。
