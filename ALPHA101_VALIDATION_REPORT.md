# Alpha101 因子验证报告 - BTC & ETH

## 执行摘要

本报告验证了 4 个 Alpha101 因子在加密货币（BTC/ETH）时序策略中的有效性，并提供了针对单资产时序策略的优化版本。

## 核心发现

### 1. Alpha #001 (alpha101_001)

**原始版本问题：**
- ❌ 不在报告中（因为 `rank()` 操作导致所有值都是常数 -0.5）
- `rank()` 是横截面操作，在单资产下无效

**时序适配版本：**
- ✅ 表现优秀
- **BTC**: 最佳相关性 0.0573 (6 bars, p=9.6e-08) ✅ 显著
- **ETH**: 平均相关性 0.0173
- **改进**: 从无效（常数）变为有效（波动率）
- **实现**: 直接用滚动波动率替代原始公式

### 2. Alpha #022 (alpha101_022)

**原始版本：**
- 平均绝对相关性: 0.0091
- 最佳相关性: -0.0415 (ETH, 120T, 6 bars, p=6.2e-03)

**时序适配版本：**
- ✅ 表现优秀
- **ETH**: 最佳相关性 0.0508 (3 bars, p=2.3e-06) ✅ 显著
- **BTC**: 平均相关性 0.0205
- **改进**: +220.3%
- **实现**: 
  - 使用更稳定的窗口（10 vs 5）
  - 移除 `rank()` 操作，直接用波动率
  - 用变化率代替 delta

### 3. Alpha #043 (alpha101_043)

**原始版本：**
- 平均绝对相关性: 0.0106
- 最佳相关性: 0.0511 (BTC, 240T, 12 bars, p=1.8e-02)

**时序适配版本：**
- ✅ 表现良好
- **BTC**: 最佳相关性 0.0362 (12 bars, p=7.7e-04) ✅ 显著
- **ETH**: 平均相关性 0.0088
- **改进**: +45.1%
- **实现**: 
  - 保留 `ts_rank()`（时间序列内的归一化，有意义）
  - 优化窗口参数

### 4. Alpha #066 (alpha101_066)

**原始版本：**
- 平均绝对相关性: 0.0147
- 最佳相关性: 0.0700 (ETH, 240T, 24 bars, p=1.2e-03) ✅ 显著
- 显著相关性数量: 11/40 (27.5%)

**时序适配版本：**
- ⚠️ 在 5 分钟级别表现较差
- 最佳相关性: -0.0185 (ETH, 12 bars, p=8.6e-02) ❌ 不显著
- 显著相关性数量: 0/8 (0%)
- **建议**: 在更高时间框架（如 240T）测试，或使用不同实现

## 关键改进

### 1. 移除横截面 rank() 操作

**问题：**
- `rank()` 是横截面操作，在单资产下无效
- 导致因子值为常数（如 alpha101_001 的所有值都是 -0.5）

**解决方案：**
- 移除 `rank()` 操作
- 直接使用原始值或使用 `ts_rank()`（时间序列内的归一化）

### 2. 优化窗口参数

**问题：**
- Alpha101 默认参数偏小（如 5, 7）
- 在加密货币市场中噪声较大

**解决方案：**
- 增大窗口参数（如 10, 20）
- 根据市场特性调整参数

### 3. 保留时间序列内的归一化

**保留：**
- `ts_rank()`: 时间序列内的归一化，有意义
- 能捕捉极端值，适合时序模型

**可选：**
- 使用 z-score 替代（更平滑）
- 根据策略需求选择

## 实施建议

### 1. 集成到特征工程

```python
from data_tools.alpha_factors.alpha101_timeseries_adapted import (
    compute_adapted_alpha101_factors
)

# 计算时序适配的 Alpha101 因子
factors = compute_adapted_alpha101_factors(df, use_ts_rank=True)
```

### 2. 参数优化

根据具体策略和市场特性调整窗口参数：

```python
factors = compute_adapted_alpha101_factors(
    df,
    alpha001_window=5,           # 波动率窗口
    alpha022_corr_window=10,     # 相关性窗口
    alpha022_delta_window=5,     # 变化率窗口
    alpha022_vol_window=20,      # 波动率窗口
    alpha043_vol_rank_window=20, # 成交量排名窗口
    alpha043_mom_rank_window=8,  # 动量排名窗口
    alpha043_adv_window=20,      # 平均成交量窗口
    alpha043_mom_period=7,       # 动量周期
    use_ts_rank=True             # 使用 ts_rank 或 z-score
)
```

### 3. 组合使用

建议结合多个因子使用，而不是单独依赖某个因子：

```python
# 组合因子
df['alpha101_combined'] = (
    0.4 * df['alpha101_001_ts'] +
    0.3 * df['alpha101_022_ts'] +
    0.3 * df['alpha101_043_ts']
)
```

## 注意事项

1. **相关性较低**: 这些因子在加密货币市场的相关性仍然较低（< 0.1），这是正常现象
2. **组合使用**: 建议结合多个因子使用，而不是单独依赖某个因子
3. **时间框架**: 不同时间框架的表现差异较大，需要根据具体策略选择
4. **市场特性**: 加密货币市场的特性可能与传统市场不同，需要持续优化
