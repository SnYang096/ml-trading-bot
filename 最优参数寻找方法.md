# 平坦高原搜索 vs Optuna 对比

## 核心区别

### 1. **优化方法**

#### 平坦高原搜索（Plateau Optimization）
- **方法**: 网格搜索/穷举扫描
- **策略**: 在阈值范围内**系统性地扫描**所有可能的值（按步长）
- **选择标准**: 找到**"平坦高原"**区间（Sharpe在某个阈值区间内保持稳定）
- **目标**: 找到**鲁棒性最强**的阈值（在多个分桶维度下都表现稳定）

```python
# 平坦高原搜索的工作方式
for threshold in np.arange(min_threshold, max_threshold, step=0.02):
    sharpe = evaluate(threshold)
    # 找到连续的最大区间，其中 sharpe >= min_sharpe_threshold
```

#### Optuna
- **方法**: 贝叶斯优化（TPE算法）
- **策略**: **智能采样**，根据历史trial结果选择下一个最有可能最优的参数
- **选择标准**: 找到**全局最优解**（最大化目标函数）
- **目标**: 找到**性能最好**的参数组合

```python
# Optuna的工作方式
def objective(trial):
    threshold = trial.suggest_float("threshold", 0.0, 1.0)
    sharpe = evaluate(threshold)
    return sharpe  # 最大化sharpe

study = optuna.create_study(direction="maximize")
study.optimize(objective, n_trials=100)  # 智能采样100次
```

## 2. **优化维度**

### 平坦高原搜索
- **单参数优化**: 一次只优化一个规则的一个阈值
- **顺序优化**: 按优先级顺序逐一优化，每个规则优化完成后**冻结**
- **Hard-Gate System**: 不允许联合优化，确保规则独立性

```python
# 优化流程
1. 优化规则A（阈值1）→ 冻结
2. 优化规则B（阈值2）→ 冻结（基于规则A过滤后的数据）
3. 优化规则C（阈值3）→ 冻结（基于规则A+B过滤后的数据）
```

### Optuna
- **多参数联合优化**: 可以同时优化多个参数
- **全局优化**: 考虑参数之间的交互作用
- **灵活**: 可以优化任意参数组合

```python
# 优化流程
def objective(trial):
    threshold1 = trial.suggest_float("threshold1", 0.0, 1.0)
    threshold2 = trial.suggest_float("threshold2", 0.0, 1.0)
    threshold3 = trial.suggest_float("threshold3", 0.0, 1.0)
    # 同时优化所有参数
    return evaluate(threshold1, threshold2, threshold3)
```

## 3. **目标函数**

### 平坦高原搜索
- **目标**: Robustness Score = **min Sharpe across buckets**
- **分桶维度**: Archetype × Vol (low/mid/high)
- **理念**: 找到在所有分桶下都表现稳定的阈值（**最差情况最好**）

```python
robustness_score = min(
    sharpe(TC_low), sharpe(TC_mid), sharpe(TC_high),
    sharpe(TE_low), sharpe(TE_mid), sharpe(TE_high),
    ...
)
# 选择robustness_score最高的阈值
```

### Optuna
- **目标**: 直接优化业务指标（Sharpe、总收益等）
- **理念**: 找到**平均性能最好**的参数

```python
def objective(trial):
    # 优化平均Sharpe
    return mean_sharpe_across_all_data
```

## 4. **计算成本**

### 平坦高原搜索
- **计算量**: 高（需要扫描所有阈值）
- **时间**: 慢（每个规则需要扫描50-100个阈值）
- **优势**: 结果可解释性强，能看到完整的阈值-Sharpe曲线

```python
# 示例：扫描50个阈值
thresholds = np.arange(0.0, 1.0, 0.02)  # 50个值
for threshold in thresholds:
    evaluate(threshold)  # 50次评估
```

### Optuna
- **计算量**: 低（智能采样，通常只需要20-100次trial）
- **时间**: 快（只评估最有希望的区域）
- **优势**: 效率高，适合高维参数空间

```python
# 示例：智能采样30次
study.optimize(objective, n_trials=30)  # 只评估30次，但可能找到最优解
```

## 5. **适用场景**

### 平坦高原搜索 ✅ 适合：
1. **Gate规则优化**（当前使用场景）
   - 规则有明确的优先级顺序
   - 需要确保规则独立性
   - 需要找到鲁棒性强的阈值（在所有市场条件下都稳定）

2. **需要可解释性**
   - 需要看到完整的阈值-Sharpe曲线
   - 需要理解每个阈值的影响

3. **参数空间小**
   - 每个规则只有一个阈值需要优化
   - 参数空间维度低（1维）

### Optuna ✅ 适合：
1. **模型超参数优化**
   - 需要同时优化多个超参数（learning_rate, num_leaves, max_depth等）
   - 参数空间维度高（10+维）

2. **联合优化**
   - 需要同时优化多个相关参数
   - 参数之间有交互作用

3. **计算资源有限**
   - 每次评估成本高（如需要重新训练模型）
   - 需要快速找到近似最优解

## 7. **为什么Gate规则用平坦高原而不是Optuna？**

### 原因1: Hard-Gate System要求
- Gate规则需要**按优先级顺序优化**
- 每个规则优化完成后**冻结**，不允许后续修改
- Optuna的联合优化会破坏这个约束

### 原因2: 鲁棒性优先
- Gate规则需要**在所有市场条件下都稳定**
- 平坦高原的"最差情况最好"策略更适合
- Optuna的"平均性能最好"可能在某些市场条件下失效

### 原因3: 可解释性
- 需要看到每个规则的阈值-Sharpe曲线
- 需要理解为什么选择某个阈值
- 平坦高原提供完整的可视化信息

### 原因4: 参数空间简单
- 每个规则只有一个阈值（1维）
- 不需要复杂的联合优化
- 网格搜索已经足够高效

🔍 一、为什么“平坦高原方法太慢”？——以及 Optuna 是否更快？

平坦高原法（Flat Plateau Search）的瓶颈：
- 通常需要网格扫描 + 多次回测（如 quantile 从 0.1 到 0.9，步长 0.05）
- 每个点都要跑完整 pipeline（label → execute → metric）
- 若有 10 个参数，组合爆炸 → O(n^k) 时间复杂度

Optuna 的优势：
- 使用 TPE（Tree-structured Parzen Estimator） 等贝叶斯优化器
- 智能采样：优先探索高潜力区域，避免穷举
- 通常 50~200 次 trial 就能收敛，远少于网格搜索的 1000+ 次

✅ 结论：Optuna 确实比平坦高原网格搜索快得多（尤其高维时）

⚠️ 二、但 Optuna 找到的阈值会不会“太敏感”？

这是完全合理的担忧。Optuna 只看目标函数（如 Sharpe），可能返回：

- 在训练集上极优，但样本外崩溃的阈值（如 quantile=0.8732）
- 对某个 symbol 过拟合（如只适合 BTC，不适合 ETH）
- 位于性能曲面的“尖峰”而非“高原” → 微小扰动就失效

🛡️ 如何让 Optuna 找到稳健、通用的阈值？

✅ 策略 1：在目标函数中加入“稳健性惩罚”
def objective(trial):
    params = suggest_params(trial)
    sharpe_by_symbol = []
    for symbol in symbols:
        perf = backtest(symbol, params)
        sharpe_by_symbol.append(perf.sharpe)
    
    mean_sharpe = np.mean(sharpe_by_symbol)
    std_sharpe = np.std(sharpe_by_symbol)
    
    # 惩罚跨品种不稳定性
    robust_score = mean_sharpe - 0.5 * std_sharpe
    
    # 额外：剔除明显负收益的 symbol（你已计划）
    if any(s < -0.2 for s in sharpe_by_symbol):
        return -10  # hard reject
    
    return robust_score

✅ 策略 2：约束搜索空间为“合理区间”
不要让 Optuna 自由探索 [0.0, 1.0]，而是限定在逻辑合理的高原附近：
例如：wick exhaustion 通常在 0.5~0.7 有效
wick_thresh = trial.suggest_float("wick_exhaustion", 0.5, 0.7)

→ 这样即使 Optuna 找到 0.63，也大概率在高原内。

✅ 策略 3：后处理：round + plateau check
- Optuna 返回 0.632 → 手动 round 到 0.6
- 再验证 0.55 / 0.6 / 0.65 三档性能是否相近
- 如果是 → 说明找到了高原，安全！

🌐 三、如何实现“多 symbol 通用”？

你提到：“如果明显负的我会剔除，希望能找到尽量通用的” —— 这很聪明。

推荐做法：两阶段筛选
阶段   目标   方法
Phase 1: 品种过滤   剔除根本不适用的 symbol   若某 symbol 在 baseline 参数下 Sharpe < -0.1，直接 exclude

Phase 2: 参数优化   找跨剩余 symbol 通用的参数   Optuna 目标 = mean(Sharpe) - λ·std(Sharpe)

✅ 这样既保留了通用性，又避免被“毒品种”拖垮。
